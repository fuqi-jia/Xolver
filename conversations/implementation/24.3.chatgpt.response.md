**结论：不接受当前版本。**

这版已经非常接近可实现，上一轮要求的大部分点都修了：范围保持 **IncrementalEGraph performance optimization only**，明确不做 propagation/model/mixed fallback；`RollbackSignatureTable` 的逆操作、`eraseIfOwner`、`ProofForest` inactive edge、snapshot queue invariant、`refreshSignature` 的一致性说明、debug invariant 都补上了。文档也明确当前只替换 signature table、增加 `currentSig_`、用 `ProofForest` 加速 explanation，不改变 `EufSolver` 公共行为。

但还有一个关键 blocker：**递归 explanation 的 `ExplainContext` 没有真正贯穿递归调用**。

# 1. 最大 blocker：`explainEdge()` 调用 `explainEquality(ai, bi)` 会重建新的 context

当前伪代码是：

```cpp
ExplainResult IncrementalEGraph::explainEquality(EufTermId a, EufTermId b) {
    ...
    ExplainContext ctx;
    ...
    auto sub = explainEdge(eid, ctx);
}
```

但在 `explainEdge()` 里递归解释 congruence 参数时写的是：

```cpp
auto sub = explainEquality(ai, bi);
```

这会重新创建一个新的 `ExplainContext ctx`。结果是：

```text
1. memo 不能跨递归层复用；
2. activePairs 不能跨递归层检测正在解释的 pair；
3. 文档声称的 recursive cycle guard 实际无效；
4. 遇到递归 explanation 环时，仍可能无限递归或重复解释。
```

必须改成 context 贯穿的版本：

```cpp
ExplainResult explainEquality(EufTermId a, EufTermId b) {
    ExplainContext ctx;
    return explainEquality(a, b, ctx);
}

ExplainResult explainEquality(EufTermId a,
                              EufTermId b,
                              ExplainContext& ctx) {
    if (a == b) return {true, {}};
    if (!same(a, b)) return {false, {}};

    auto key = canonicalPairKey(a, b);
    if (auto it = ctx.memo.find(key); it != ctx.memo.end()) {
        return it->second;
    }
    if (!ctx.activePairs.insert(key).second) {
        return {false, {}};
    }

    auto edgeIds = proofForest_.path(a, b);
    if (edgeIds.empty()) {
        ctx.activePairs.erase(key);
        return {false, {}};
    }

    std::vector<SatLit> reasons;
    for (size_t eid : edgeIds) {
        auto sub = explainEdge(eid, ctx);
        if (!sub.ok) {
            ctx.activePairs.erase(key);
            return {false, {}};
        }
        appendDedup(reasons, sub.reasons);
    }

    ctx.activePairs.erase(key);
    ExplainResult result{true, std::move(reasons)};
    ctx.memo[key] = result;
    return result;
}
```

然后 `explainEdge()` 里必须调用：

```cpp
auto sub = explainEquality(ai, bi, ctx);
```

而不是无参 context 版本。

---

# 2. `pairKey` 应该 canonicalize，不能直接 `(u << 32) | v`

当前：

```cpp
uint64_t pairKey = ((uint64_t)edge.u << 32) | edge.v;
```

这是有方向的。`explainEquality(a,b)` 和 `explainEquality(b,a)` 应该共享同一个 memo/active key，因为 equality 是无向的。

建议：

```cpp
uint64_t canonicalPairKey(EufTermId a, EufTermId b) {
    if (a > b) std::swap(a, b);
    return (uint64_t(a) << 32) | uint64_t(b);
}
```

如果 `EufTermId` 将来超过 32 位，就改成结构体 key，不要手工拼。

---

# 3. `ProofForest::addEdge()` 最好加 `ur.merged` 断言

当前 `processMergeQueue()` 里：

```cpp
auto ur = uf_.unite(ra, rb);
EClassId dst = ur.winner;
EClassId src = ur.loser;

proofForest_.addEdge(m.a, m.b, m.reason);
```

前面已经有 `if (same(m.a, m.b)) continue;`，所以通常 `ur.merged` 应为 true。但这里最好写成硬防御：

```cpp
auto ur = uf_.unite(ra, rb);
if (!ur.merged) {
    continue;
}

proofForest_.addEdge(m.a, m.b, m.reason);
```

或者至少：

```cpp
assert(ur.merged);
```

因为你的 hard rule 是 “`addEdge` only called after a successful UF unite”。实现里也要体现出来。

---

# 4. debug invariant 注释写反了

文档写：

```text
Debug invariant (only in NDEBUG builds)
```

这句话应改成：

```text
Debug invariant (only in non-NDEBUG/debug builds)
```

代码里是：

```cpp
#ifndef NDEBUG
    checkSignatureTableInvariant();
#endif
```

所以文字不要写反，避免实现模型误解。

---

# 5. `congruenceQueue_` 要么删除，要么 snapshot 不再 assert 它

文档后面说：

```text
The congruenceQueue_ can be removed entirely
```

但前面 `EGraphSnapshot` 仍然写：

```cpp
assert(congruenceQueue_.empty());
```

如果本版真的移除 `congruenceQueue_`，那 snapshot 里不要再引用它；如果保留字段但不使用，也要明确：

```text
congruenceQueue_ is removed in this patch.
snapshot only asserts mergeQueue_.empty().
```

否则实现时会出现接口残留。

---

# 最小修改后可以接受

只要改下面 5 点，我认为下一版可以接受：

```text
1. 增加 explainEquality(a,b, ExplainContext&) 内部版本；
   explainEdge 递归时必须传同一个 ctx。
2. memo/activePairs 使用 canonical unordered pair key。
3. proofForest_.addEdge 前检查或 assert ur.merged。
4. 修正文档：debug invariant 是 non-NDEBUG/debug builds。
5. congruenceQueue_ 的处理统一：删除则不要再 snapshot assert；保留则明确永远为空。
```

# 最终意见

**不接受当前版本。**

不是架构问题，而是 explanation context 的实现细节仍有 blocker。当前写法虽然定义了 `memo + activePairs`，但递归时重新创建 context，导致 cycle guard 和 memo 实际失效。修掉这个后，这版优化方案基本可以进入实现。
