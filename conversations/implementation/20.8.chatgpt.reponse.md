还不能 Approve。**最大 blocker 仍然没修掉：`LinearizationResult` 还是 `vector<TheoryLemma>`，但后面仍然调用 `markEmitted(lemma)`。** 这在接口上实现不了。

# 1. `markEmitted(lemma)` 还是错误的

计划里 `LinearizationResult` 仍然是：

```cpp
struct LinearizationResult {
    LinearizationStatus status;
    std::vector<TheoryLemma> lemmas;
};
```

但 cache 需要的信息是：

```cpp
NonlinearTermKey key
SatLit nonlinearReason
vector<mpq_class> boundValues
int cutIndex
```

`TheoryLemma` 里面没有这些信息，所以这句仍然不成立：

```cpp
linAdapter_->markEmitted(lemma);
```

必须改成：

```cpp
struct CutCacheKey {
    NonlinearTermKey key;
    SatLit nonlinearReason;
    std::vector<mpq_class> boundValues;
    int cutIndex;
};

struct PendingLinearizationLemma {
    TheoryLemma lemma;
    std::optional<CutCacheKey> cacheKey;
};

struct LinearizationResult {
    LinearizationStatus status;
    std::vector<PendingLinearizationLemma> lemmas;
};
```

然后 adapter 里：

```cpp
for (auto& item : lr.lemmas) {
    if (!lemmaDb.contains(item.lemma)) {
        lemmaDb.insertIfNew(item.lemma);
        pendingLinLemmas_.push_back(std::move(item.lemma));

        if (item.cacheKey) {
            linAdapter_->markEmitted(*item.cacheKey);
        }
    }
}
```

`abstraction lemma` 可以没有 `cacheKey`；McCormick / square cut lemma 必须带 `cacheKey`。

---

# 2. `NiaLinearizationAdapter::run` 参数不够，无法生成 mirror lemmas

计划说 adapter 要扫描 active assignments，生成 mirror lemmas：

```text
original linear bound -> NIA
mirror effective bound -> LIA
```

但现在接口是：

```cpp
LinearizationResult run(
    const std::vector<NormalizedNiaConstraint>& constraints,
    const DomainStore& domains,
    TheoryLemmaDatabase& lemmaDb);
```

这里没有 active assignments，也没有原始 linear atoms 的 literal/value 信息。
所以 adapter 根本不知道：

```text
哪个 linear atom 当前 active；
它的 SAT literal 是什么；
它是 true 还是 false；
false 时要取反成什么 effective relation。
```

要么把 mirror 拆成单独步骤：

```cpp
generateMirrorLemmas(activeAssignments, registry, TheoryId::LIA)
```

要么修改 adapter 接口：

```cpp
LinearizationResult run(
    const std::vector<NormalizedNiaConstraint>& nonlinearConstraints,
    const std::vector<ActiveTheoryAssignment>& activeAssignments,
    const DomainStore& domains,
    TheoryLemmaDatabase& lemmaDb);
```

我更建议拆成两个阶段：

```text
9.4 mirror effective active linear bounds to LIA/LRA
9.5 run IncrementalLinearizer on nonlinear constraints
```

这样职责清楚。

---

# 3. D4 的 bound guard 表述仍有风险

计划里仍写：

```text
If bounds are unconditional (no reasons), omit bound guards.
```

这句话容易被实现成：

```cpp
if (reasons.empty()) omitGuard();
```

但后面的红线又说：

```text
If finite bounds have incomplete reasons and are not global, skip the cut.
```

后者才是正确的。

应该改成：

```text
Only omit bound guards when the bound is explicitly marked global.
Empty reason vectors alone do not imply unconditional bounds.
In V1, if finite bounds are not global and reasonComplete=false, skip bounds-dependent cuts.
```

# 4. “Missing bounds -> NoChange” 测试仍然不准确

计划测试里还写：

```text
Missing bounds -> NoChange | x∈[0,10], y unbounded → NoChange
```

这只能用于 `McCormickGenerator` 单元测试，不能用于完整 `IncrementalLinearizer::run()`。

因为 full linearizer 即使缺少 `y` bound，也可能仍然生成：

```text
abstraction lemma
square nonnegativity lemma
```

应改成：

```text
Missing product bounds:
  McCormickGenerator returns no envelope cuts.
  Full IncrementalLinearizer may still emit abstraction lemmas.
```

---

# 5. `x²+y²≤1, x=2 -> UNSAT` 需要明确 singleton bound

这个测试可以成立，但前提是 `x=2` 被转换成 LIA 可见的 singleton bounds：

```text
x >= 2
x <= 2
```

这样 tangent midpoint 才是 `a=2`，可推出：

```text
s_x >= 4
```

否则如果只是 mirror 了 equality，但 `SquareCutGenerator` 从 `BoundStore` 里拿不到 `x∈[2,2]`，它不会生成足够强的 tangent cut。

测试说明建议改成：

```text
x=2 must be mirrored and also represented in BoundStore as singleton bound [2,2].
Then tangent at midpoint a=2 gives s_x >= 4.
```
