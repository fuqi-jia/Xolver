建议选 **[4] Revise**。这个计划总体方向对：IDL/RDL 独立走 difference graph + Bellman-Ford/SPFA，比 Simplex 简洁；动态 disequality split、model validation、QF_IDL/QF_RDL 独立注册也都是正确方向。计划里也已经明确使用 `TheoryId::IDL/RDL`、复用 `LinearAtomPayload`、负环生成 conflict、RDL 用 `(c, deltaCoeff)` 处理 strict，这些是合理的。

但现在还不能直接 Approve，主要有几个 correctness / execution 问题。

---

# 1. 最大问题：没有明确处理 false literal 的 effective relation

`assertLit` 里写了 `ActiveAssignment { lit, atom, value }`，但是后续提取约束时只说“从 `LinearAtomPayload` 提取 difference constraint”，没有明确：

```text
如果 value=false，必须先取反 relation。
```

这会直接导致 soundness bug。

例如原 atom 是：

```text
x - y <= 3
```

如果 SAT assignment 给的是 false，那么 theory 看到的约束应是：

```text
x - y > 3
```

在 IDL 中要归一化为：

```text
y - x <= -4
```

在 RDL 中要归一化为：

```text
y - x < -3
```

必须补一节：

```text
Before normalization, compute effective relation:
  value=true  -> original relation
  value=false -> negated relation

Negation:
  <= -> >
  <  -> >=
  >= -> <
  >  -> <=
  =  -> !=
  != -> =
```

否则所有带 `not` 的 IDL/RDL 公式都会出错。

---

# 2. 单变量差分约束提取不完整

计划写：

```text
1 var: treat x as x - ZERO, coefficient +1.
```

这只覆盖了：

```text
x <= c
```

但还要处理：

```text
-x <= c
```

它应解释为：

```text
ZERO - x <= c
```

也就是边：

```text
x -> ZERO weight c
```

所以差分提取规则应该是：

```text
linear form: a*x + const rel 0

if a = +1:
  x - ZERO rel -const

if a = -1:
  ZERO - x rel -const

otherwise unsupported
```

对于双变量也类似，必须从：

```text
a*x + b*y + const rel 0
```

提取：

```text
x - y rel -const
```

或者：

```text
y - x rel -const
```

要求 `{a,b}={+1,-1}`。否则 `x + 2 <= y`、`y - x >= 3` 这类会很容易符号写错。

---

# 3. IDL 的 rational RHS 规则要补齐，尤其 Eq/Neq

计划说 IDL 做 floor/ceil 归一化，但没有明确 equality/disequality 的 rational 情况。

必须加：

```text
IDL:
  x-y = c:
    if c not integer -> conflict
    else add both bounds

  x-y != c:
    if c not integer -> tautology / NoChange
    else disequality split

  x-y <= c:
    x-y <= floor(c)

  x-y < c:
    x-y <= ceil(c)-1

  x-y >= c:
    y-x <= -ceil(c)

  x-y > c:
    y-x <= -(floor(c)+1)
```

尤其：

```text
x - y != 1/2
```

在 IDL 里永真，不能生成 split。

---

# 4. RDL 的一个测试期望是错的

计划里写：

```text
SAT: x - y < 1 ∧ y - x < -1
```

这个其实是 **UNSAT**。

因为：

```text
y - x < -1
等价于
x - y > 1
```

所以两条合起来是：

```text
x - y < 1
x - y > 1
```

不可能同时成立。

可以改成 SAT 测试：

```text
x - y < 1 ∧ y - x < 2
```

例如 `x=y=0` 满足。

或者：

```text
x - y < 1 ∧ y - x <= 0
```

例如 `x=y=0` 满足第一条 `0<1` 和第二条 `0<=0`。

---

# 5. Disequality split 要明确处理 dynamic atom 的 theory routing

计划说复用：

```cpp
TheoryAtomRegistry::getOrCreateLinearBoundAtom
```

但要确保新 atom 的 `TheoryId` 是 IDL/RDL，而不是默认 LIA/LRA。计划前面提到“same TheoryId”，但这里最好写死：

```text
IDL diseq split creates IDL-owned LinearAtomPayload.
RDL diseq split creates RDL-owned LinearAtomPayload.
New literals must route back to IdlSolver/RdlSolver.
```

否则 split lemma 生成后，SAT 选中分支，可能被 LIA/LRA solver 接走，IDL/RDL 看不到新边。

---

# 6. 增量 SPFA 和 out-of-scope 自相矛盾

Phase 1 设计了：

```cpp
runIncremental(...)
```

Phase 2 `check()` 也用了：

```cpp
bf_.runIncremental(graph_, newEdges_, currentDist_)
```

但后面 Out-of-Scope 又写：

```text
Incremental SPFA with dirty rebuild optimization
```

这会让实现范围混乱。建议第一版直接改成：

```text
V1: 每次 check 全量 rebuild graph + full Bellman-Ford。
V2: 再做 incremental SPFA。
```

或者如果你确实要做 incremental，就不要把它列为 out-of-scope。

我建议 **V1 只做 full rebuild**。IDL/RDL 很快，先保证正确。

---

# 7. Unsupported atom 的处理要和主 CDCL loop 对齐

计划写：

```text
return TheoryCheckResult::unknown()
```

这可以，但要确认 `TheoryManager` / `Solver::checkSatInternal` 看到 theory unknown 后会立即返回 top-level `Unknown`，而不是继续 CDCL 搜索。

建议加验收测试：

```smt2
(set-logic QF_IDL)
(declare-const x Int)
(declare-const y Int)
(assert (<= (+ x y) 3))
(check-sat)
```

期望：

```text
unknown
```

不能 SAT，不能 UNSAT，不能死循环。

---

# 8. RDL epsilon 选择不能只写“ε=1 or upper bound”

这个方向对，但要补一句：**如果 validate 失败，不能返回 SAT**。计划风险里提到了 validation，但 model construction 里最好明确：

```text
buildRdlModel:
  construct candidate epsilon;
  validate all active constraints;
  if validation fails, try smaller epsilon;
  if still fails, return Unknown, never SAT.
```

更稳的做法是第一版直接用 delta-weight 模型做 validation，再选 rational epsilon 时保守取：

```text
ε = 1 / (2 * (1 + maxAbsDeltaCoeff + maxAbsRationalDenominatorBound))
```

当然实现可以简单，但红线要明确。
