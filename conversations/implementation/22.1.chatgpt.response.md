还要 **Revise**，而且这版不是 “minor”。
主要问题是：**文档内部把上一版已经确定的安全设计又写回了旧方案**，导致实现模型会不知道到底按哪套做。

# 总体判断

这版开头的 Red Lines 写得很好，但后面 Phase 里多处违反 Red Lines。最严重的是这些：

```text
1. Atomizer-side dedup 和 Registry-side get-or-create 同时存在，互相矛盾。
2. true != false 检查又放回了 equality merge 之前，会漏 UNSAT。
3. explanation cycle 又写成 ok=true + empty reasons，违反红线。
4. closeCongruence 声明返回 CloseStatus，但实现又是 void。
5. Solver.cpp 里又写死 boolSortId = 0，违反红线。
6. EGraph 仍然缺 termClass_，find(t) 无法可靠实现。
```

所以这版不能直接实现，需要先把内部矛盾删干净。

---

# 1. 最大 blocker：EUF atom dedup 方案自相矛盾

前面 Red Line 写：

```text
Atomizer owns EUF dedup via unordered_map<EufAtomKey, SatLit>.
Registry only records atoms.
```

Phase 2 也是这个设计：

```cpp
std::unordered_map<EufAtomKey, SatLit, EufAtomKeyHash> eufAtomDedup_;
```

但后面 Phase 4 又写：

```cpp
SatLit getOrCreateEufAtom(const EufAtomPayload& payload, ExprId originExpr);
```

并且说 Registry 内部调用：

```text
atomizer_->registerDynamicAtom(...)
```

这又回到了 Registry 创建 / dedup SAT var 的旧设计。两套设计不能同时存在。

建议固定为上一版确定的方案：

```text
Atomizer 负责 freshVar + EUF dedup；
Registry 只负责 registerParsedTheoryAtom / findBySatVar；
删除 Registry::getOrCreateEufAtom；
删除 registry 内部 eufLookup_；
删除 atomizer_->registerDynamicAtom 反向依赖。
```

最终保留：

```cpp
SatLit Atomizer::getOrCreateEufAtom(EufAtomPayload payload, ExprId originExpr);
```

不要再有：

```cpp
TheoryAtomRegistry::getOrCreateEufAtom(...)
```

---

# 2. `true != false` 检查位置又错了

当前 `check()` 顺序是：

```text
1. intern terms
2. add terms
2b. check true != false
3. assert equalities
4. close congruence
5. check disequalities
```

这会漏掉：

```smt2
(assert (= true false))
```

因为检查 true/false 时，它们还没有被 equality merge 合并。

必须改成：

```text
1. intern terms
2. add terms
3. assert equalities
4. close congruence
5. check true != false
6. check disequalities
```

也就是：

```cpp
// after equality merge + closeCongruence
if (trueTerm exists && falseTerm exists &&
    egraph_.same(trueTerm, falseTerm)) {
    auto er = egraph_.explainEquality(trueTerm, falseTerm, termManager_);
    ...
}
```

这个是 soundness blocker。

---

# 3. explanation cycle 又违反红线

Red Line 写：

```text
Explanation cycle returns failure (ok=false), not empty reasons.
```

但代码里写：

```cpp
if (visitedPairs.count(pk)) return {true, {}};  // cycle in explanation graph
```

这必须改成：

```cpp
if (visitedPairs.count(pk)) {
    return {false, {}};
}
```

否则 explanation 可能缺少必要原因，学到过强 conflict clause。

---

# 4. `closeCongruence` 前后签名不一致

头文件写：

```cpp
CloseStatus closeCongruence(const EufTermManager& tm);
```

但后面实现写：

```cpp
void EGraph::closeCongruence(const EufTermManager& tm)
```

而且 `check()` 里也没有检查返回值：

```cpp
egraph_.closeCongruence(termManager_);
```

应该统一成：

```cpp
CloseStatus EGraph::closeCongruence(const EufTermManager& tm);
```

调用处：

```cpp
auto cs = egraph_.closeCongruence(termManager_);
if (cs != CloseStatus::Ok) {
    return TheoryCheckResult::unknown();
}
```

内部遇到 sort mismatch：

```cpp
if (status == MergeStatus::SortMismatch) {
    return CloseStatus::SortMismatch;
}
```

不要只写注释 “should never happen”。

# 5. `Solver.cpp` 又写死了 `boolSortId = 0`

Red Line 写：

```text
Bool sort ID obtained from SOMTParser sort system, not ConstBool scanning.
```

但 Phase 6 又写：

```cpp
atomizer.setBoolSortId(0);
```

并且注释里还说：

```text
For now, assume SortId 0 is Bool.
```

这必须删掉。应该写成：

```cpp
SortId boolSort = adapter.getBoolSortId();
if (boolSort == NullSort) {
    return Result::Unknown; // or internal unsupported
}
atomizer.setBoolSortId(boolSort);
```

不要再出现：

```cpp
atomizer.setBoolSortId(0);
```

也不要扫描 ConstBool。

# 6. `EGraph` 仍然缺少 `termClass_`

头文件里有：

```cpp
std::vector<EClass> classes_;
std::vector<ProofEdge> proofEdges_;
```

但没有：

```cpp
std::vector<EClassId> termClass_;
```

这样 `find(EufTermId t)` 无法可靠实现。必须加：

```cpp
std::vector<EClassId> termClass_;
```

并明确：

```cpp
void EGraph::addTerm(EufTermId t) {
    if (t >= termClass_.size()) {
        termClass_.resize(t + 1, NullEClass);
    }
    if (termClass_[t] != NullEClass) return;

    EClassId c = classes_.size();
    classes_.push_back(EClass{c, c, {t}});
    termClass_[t] = c;
}

EClassId EGraph::find(EufTermId t) const {
    assert(t < termClass_.size());
    assert(termClass_[t] != NullEClass);
    return ufFind(termClass_[t]);
}
```

否则 `same(a,b)`、`merge(a,b)`、`explainEquality(a,b)` 都没有可靠基础。

# 7. `pop()` 仍然缺少 clamp

当前：

```cpp
trail_.resize(scopeLimits_.back());
```

上一版已经要求改成防御式：

```cpp
size_t lim = std::min(scopeLimits_.back(), trail_.size());
trail_.resize(lim);
```

否则 CDCL backtrack 后再 pop，仍然可能 resize 回扩。

应改成：

```cpp
void EufSolver::pop(uint32_t n) {
    while (n-- && !scopeLimits_.empty()) {
        size_t lim = std::min(scopeLimits_.back(), trail_.size());
        trail_.resize(lim);
        scopeLimits_.pop_back();
    }
}
```

# 8. Synthetic allocator 还没补 reset / 防撞

当前：

```cpp
class SyntheticExprIdAllocator {
    ExprId nextId_ = std::numeric_limits<ExprId>::max() - 100;
public:
    ExprId next() { return nextId_--; }
};
```

建议补：

```cpp
class SyntheticExprIdAllocator {
    static constexpr ExprId Start =
        std::numeric_limits<ExprId>::max() - 100;
    ExprId nextId_ = Start;

public:
    ExprId next() {
        assert(nextId_ != TrueSentinelExpr);
        assert(nextId_ != FalseSentinelExpr);
        assert(nextId_ > 0);
        return nextId_--;
    }

    void reset() {
        nextId_ = Start;
    }
};
```

同时 `Atomizer::reset()` 必须清：

```cpp
eufAtomDedup_.clear();
synthExprAlloc_.reset();
```

# 9. `distinct` 的 0/1 元情况没有处理

`distinct` 的 0 或 1 个参数应当是 vacuously true。

建议在 n-ary distinct helper 里写：

```cpp
if (e.children.size() <= 1) {
    return satTrueLit();
}
```

`=` 如果少于 2 个参数，可以 parse error / unsupported；但 `distinct` 不应错误处理。


# 最终判断

这版不能说 implementation-ready，因为存在多个“红线写对了、后面实现又写错了”的冲突。
建议你不要继续扩展内容，而是做一次**清理型修订**：删除旧设计残留，统一接口语义。修完上面 10 点后，就可以进入实现。
