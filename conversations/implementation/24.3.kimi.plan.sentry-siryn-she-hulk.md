## Scope: Incremental EUF Baseline Optimization ONLY

**In scope:** Rollbackable signature table + ProofForest explanation speedup.

**Strictly out of scope:** Theory propagation, model construction, mixed fallback.

## What This Plan Actually Changes

The ONLY changes are internal to `IncrementalEGraph` performance:

1. **Replace** `sigTable_` (plain unordered_map + `sigTableDirty_` flag + full rebuild) with `RollbackSignatureTable` (trail-based rollback, no rebuild).
2. **Add** `currentSig_` per-app signature tracking with rollback trail.
3. **Replace** `explainEquality()` (full-scan `mergeRecords_` + BFS) with `ProofForest` (incremental adjacency + tree-path BFS).
4. **Add** `ExplainContext` with both **memo** and **activePairs** cycle guard.

No changes to `EufSolver` public behavior, `EufTypes` semantics, `RollbackUnionFind`, or `EufTermManager`.

## New Files

### `src/theory/euf/RollbackSignatureTable.h/.cpp`

Rollbackable hash map. Every mutating operation pushes a `Change` record. Rollback replays inverse operations in reverse.

```cpp
class RollbackSignatureTable {
public:
    std::optional<EufTermId> find(const AppSignature& sig) const;
    void insertOrAssign(const AppSignature& sig, EufTermId owner);
    void eraseIfOwner(const AppSignature& sig, EufTermId owner);
    size_t snapshot() const;
    void rollback(size_t snap);
    void clear();

private:
    enum class ChangeKind { InsertNew, Overwrite, Erase };
    struct Change {
        ChangeKind kind;
        AppSignature sig;
        std::optional<EufTermId> previousOwner;
    };
    std::unordered_map<AppSignature, EufTermId, AppSignatureHash> table_;
    std::vector<Change> trail_;
};
```

**Hard rule: rollback inverse operations**

| ChangeKind | `previousOwner` | Rollback action |
|-----------|-----------------|-----------------|
| `InsertNew` | `nullopt` | `table_.erase(sig)` |
| `Overwrite` | old owner | `table_[sig] = previousOwner` |
| `Erase` | erased owner | `table_[sig] = previousOwner` |

**Hard rule: `eraseIfOwner` must check owner equality before pushing trail.**

If `table_[sig] != owner`, `eraseIfOwner` does **nothing** and pushes **nothing**. Otherwise it records `Change{Erase, sig, table_[sig]}` then erases.

This prevents one app's refresh from deleting another valid app's entry.

### `src/theory/euf/ProofForest.h/.cpp`

Rollbackable proof adjacency for explanation. Edges are append-only; rollback shrinks `activeEdgeCount_`.

```cpp
class ProofForest {
public:
    void addEdge(EufTermId u, EufTermId v, const MergeReason& reason);
    size_t snapshot() const;
    void rollback(size_t snap);
    void clear();
    std::vector<size_t> path(EufTermId u, EufTermId v) const;
    size_t activeEdgeCount() const;

private:
    struct Edge {
        EufTermId u;
        EufTermId v;
        MergeReason reason;
    };
    std::vector<Edge> edges_;
    std::vector<std::vector<std::pair<EufTermId, size_t>>> adj_;
    size_t activeEdgeCount_ = 0;
};
```

**Hard rule: `addEdge` only called after a successful UF unite.**

```cpp
auto ur = uf_.unite(ra, rb);
if (ur.merged) {
    proofForest_.addEdge(lhs, rhs, reason);  // lhs/rhs are the original terms, not roots
}
```

Never call `addEdge` on enqueue, attempt, or failed merge. This guarantees the active subgraph is acyclic (a forest).

**Hard rule: `path()` skips inactive edges.**

```cpp
for (auto [v, edgeId] : adj_[u]) {
    if (edgeId >= activeEdgeCount_) continue;
    // ... BFS continues
}
```

**Memory management:** `edges_` and `adj_` are append-only. After rollback, old inactive edges remain in memory. To prevent unbounded growth during long CDCL searches:

- V1: allow growth; BFS skips inactive edges.
- If profiling shows memory pressure: add compaction threshold. When `inactiveEdges() > threshold`, rebuild `edges_`/`adj_` from active edges only.

Documented in plan but threshold compaction is post-V1 polish.

#### 2. `ensureTerm(EufTermId t)`

```cpp
void IncrementalEGraph::ensureTerm(EufTermId t) {
    if (t >= members_.size()) {
        size_t oldSize = members_.size();
        members_.resize(t + 1);
        for (size_t i = oldSize; i <= t; ++i) {
            uf_.addNode();
            members_[i].push_back(static_cast<EufTermId>(i));
        }
    }
    if (t >= currentSig_.size()) {
        currentSig_.resize(t + 1, std::nullopt);
    }
}
```

No `sigTableDirty_ = true`. New app terms will be refreshed via `refreshSignature` when first seen or when their parents are affected by a merge.

#### 3. `refreshSignature(EufTermId app)` — core of the design

**Hard rule: `refreshSignature` may only be called inside `processMergeQueue` while processing to fixpoint.**

```cpp
void IncrementalEGraph::refreshSignature(EufTermId app) {
    if (app >= tm_.termCount()) return;
    if (tm_.node(app).args.empty()) return;

    for (EufTermId arg : tm_.node(app).args) {
        ensureTerm(arg);
    }
    ensureTerm(app);

    // 1. Compute NEW signature before touching old state
    AppSignature newSig = computeSignature(app);

    // 2. Look up existing owner for this signature
    auto ownerOpt = sigTable_.find(newSig);

    if (ownerOpt) {
        EufTermId owner = *ownerOpt;

        // 3. Stale owner check
        bool ownerIsFresh = currentSig_[owner].has_value() &&
                            *currentSig_[owner] == newSig;

        if (ownerIsFresh && !same(owner, app)) {
            // Congruence merge needed. Do NOT unregister old signature yet.
            // The merge will change reps; re-processing will fix everything.
            MergeReason cr;
            cr.kind = MergeReasonKind::Congruence;
            cr.lhsApp = owner;
            cr.rhsApp = app;
            const auto& n1 = tm_.node(owner);
            const auto& n2 = tm_.node(app);
            for (size_t i = 0; i < n1.args.size(); ++i) {
                cr.argPairs.push_back({n1.args[i], n2.args[i]});
            }
            mergeQueue_.push_back({owner, app, cr});
            return;
        }
    }

    // 4. No congruence merge needed. Safe to update registration.
    if (currentSig_[app].has_value()) {
        sigTable_.eraseIfOwner(*currentSig_[app], app);
    }

    sigTable_.insertOrAssign(newSig, app);
    setCurrentSig(app, newSig);
}
```

**Why this is safe:** When a congruence merge is discovered, the old signature registration is left untouched. The outer `processMergeQueue` loop will eventually process the new merge, which will trigger `collectParents` and `refreshSignature` again for affected apps. At that point the reps will have changed and the correct final registration will be established.

**Hard rule: after `processMergeQueue` returns, every app has `currentSig_[app]` consistent with `sigTable_` and current UF reps.**

#### 4. `setCurrentSig` / `rollbackCurrentSig`

```cpp
void IncrementalEGraph::setCurrentSig(EufTermId app,
                                       std::optional<AppSignature> sig) {
    if (currentSig_[app] == sig) return;
    currentSigTrail_.push_back({app, currentSig_[app]});
    currentSig_[app] = std::move(sig);
}

void IncrementalEGraph::rollbackCurrentSig(size_t snap) {
    while (currentSigTrail_.size() > snap) {
        auto ch = currentSigTrail_.back();
        currentSigTrail_.pop_back();
        currentSig_[ch.first] = std::move(ch.second);
    }
}
```

#### 5. `processMergeQueue()`

```cpp
MergeStatus IncrementalEGraph::processMergeQueue() {
    while (!mergeQueue_.empty()) {
        auto m = mergeQueue_.front();
        mergeQueue_.pop_front();

        if (same(m.a, m.b)) continue;

        EClassId ra = rep(m.a);
        EClassId rb = rep(m.b);

        if (tm_.node(m.a).sort != tm_.node(m.b).sort &&
            tm_.node(m.a).sort != NullSort && tm_.node(m.b).sort != NullSort) {
            return MergeStatus::SortMismatch;
        }

        // Collect affected parents BEFORE unite
        std::vector<EufTermId> affected = collectParents(ra);
        auto affectedB = collectParents(rb);
        affected.insert(affected.end(), affectedB.begin(), affectedB.end());

        // Record merge
        MergeRecord rec;
        rec.id = static_cast<MergeId>(mergeRecords_.size());
        rec.lhs = m.a;
        rec.rhs = m.b;
        rec.lhsRootBefore = ra;
        rec.rhsRootBefore = rb;
        rec.reason = m.reason;
        mergeRecords_.push_back(rec);

        // Unite
        auto ur = uf_.unite(ra, rb);
        EClassId dst = ur.winner;
        EClassId src = ur.loser;

        // Proof forest: ONLY after successful merge
        proofForest_.addEdge(m.a, m.b, m.reason);

        // Merge members
        memberTrail_.push_back({dst, src, members_[dst].size()});
        members_[dst].insert(members_[dst].end(),
                             members_[src].begin(), members_[src].end());

        // Refresh signatures of all affected parent applications
        for (EufTermId app : affected) {
            refreshSignature(app);
        }
    }
    return MergeStatus::Ok;
}
```

**Key differences from old code:**
- No `ensureFreshSignatureTable()` call
- `refreshSignature(app)` called directly for each affected parent (replaces `congruenceQueue_` push + separate `processCongruenceQueue`)
- `proofForest_.addEdge()` called immediately after successful `uf_.unite()`

The `congruenceQueue_` can be removed entirely since `refreshSignature` is called synchronously during merge processing.

#### 7. `rollback()`

```cpp
void IncrementalEGraph::rollback(EGraphSnapshot snap) {
    mergeQueue_.resize(snap.mergeQueueSize);

    proofForest_.rollback(snap.proofForestSnap);
    mergeRecords_.resize(snap.mergeRecordSize);

    rollbackCurrentSig(snap.currentSigSnap);
    sigTable_.rollback(snap.sigTableSnap);

    while (memberTrail_.size() > snap.memberTrailSize) {
        auto ch = memberTrail_.back();
        memberTrail_.pop_back();
        members_[ch.dstRoot].resize(ch.oldDstSize);
    }

    uf_.rollback(snap.ufTrailSize);

#ifndef NDEBUG
    checkSignatureTableInvariant();
#endif
}
```

**Debug invariant (only in NDEBUG builds):**

```cpp
void IncrementalEGraph::checkSignatureTableInvariant() const {
    for (const auto& [sig, owner] : sigTable_.internalTable()) {
        assert(currentSig_[owner].has_value());
        assert(*currentSig_[owner] == sig);
        auto computed = computeSignature(owner);
        assert(computed == sig);
    }
    for (EufTermId t = 0; t < currentSig_.size(); ++t) {
        if (currentSig_[t]) {
            auto it = sigTable_.find(*currentSig_[t]);
            assert(it.has_value());
            assert(*it == t || same(*it, t));
        }
    }
}
```

(Expose `internalTable()` as a debug-only accessor on `RollbackSignatureTable`.)

#### 8. `explainEquality()` — rewritten with ProofForest

```cpp
struct ExplainContext {
    std::unordered_map<uint64_t, ExplainResult> memo;
    std::unordered_set<uint64_t> activePairs;
};

ExplainResult IncrementalEGraph::explainEquality(EufTermId a, EufTermId b) {
    if (a == b) return {true, {}};
    if (!same(a, b)) return {false, {}};

    auto edgeIds = proofForest_.path(a, b);
    if (edgeIds.empty() && a != b) return {false, {}};

    ExplainContext ctx;
    std::vector<SatLit> reasons;

    for (size_t eid : edgeIds) {
        auto sub = explainEdge(eid, ctx);
        if (!sub.ok) return {false, {}};
        appendDedup(reasons, sub.reasons);
    }
    return {true, std::move(reasons)};
}
```

#### 9. `explainEdge()` — replaces `explainMergeReason()`

```cpp
ExplainResult IncrementalEGraph::explainEdge(size_t edgeId,
                                               ExplainContext& ctx) {
    const auto& edge = proofForest_.edge(edgeId);

    if (edge.reason.kind == MergeReasonKind::AssertedEquality) {
        return {true, {edge.reason.assertedLit}};
    }

    uint64_t pairKey = ((uint64_t)edge.u << 32) | edge.v;
    if (auto it = ctx.memo.find(pairKey); it != ctx.memo.end()) {
        return it->second;
    }

    // Cycle guard
    if (!ctx.activePairs.insert(pairKey).second) {
        return {false, {}};  // cycle in recursive explanation
    }

    std::vector<SatLit> out;
    for (const auto& [ai, bi] : edge.reason.argPairs) {
        if (ai == bi) continue;
        auto sub = explainEquality(ai, bi);
        if (!sub.ok) {
            ctx.activePairs.erase(pairKey);
            return {false, {}};
        }
        appendDedup(out, sub.reasons);
    }

    ctx.activePairs.erase(pairKey);
    ExplainResult result{true, std::move(out)};
    ctx.memo[pairKey] = result;
    return result;
}
```

**Hard rule: `explainEdge` uses both memo (completed results) and activePairs (in-progress cycle guard).** Cycle guard returns `{false, {}}`, never empty reasons as success.
