# CDCAC 求解器重构计划：从常数检查器到完整 NRA 核心

## 目标

将当前仅支持常数约束的 `CdcacSolver` 骨架，重构为模块化的 **Conflict-Driven Cylindrical Algebraic Covering (CDCAC)** 引擎，使 QF_NRA 逻辑能够求解非线性实数约束。

## 当前状态

- `CdcacSolver::check()` 遇到非常数多项式直接返回 `Unknown`
- `NraSolver` 是薄 facade，未连接 `NraLinearizationAdapter`
- 多项式基础设施（`PolynomialKernel`, `LibPolyKernel`, `PolynomialConverter`）已成熟
- NIA 求解器有完整 pipeline 可供参考

## 架构总览

保留 `CdcacSolver` 外壳作为 CDCL(T) facade，引入三层内部结构：

```
NraSolver (TheorySolver facade)
  └── CdcacSolver (trail + active constraints + effort dispatch)
        └── CdcacCore (recursive covering/search algorithm)
              ├── AlgebraBackend   (exact root isolation / sign eval / projection)
              ├── ProjectionEngine (McCallum / Lazard / Conservative)
              ├── CoveringManager  (cells, intervals, union, sample selection)
              └── ReasonManager    (label collection, conflict minimization)
```

`libpoly` 只出现在 `LibpolyBackend` 实现中，核心算法不直接依赖它。

### 关键接口设计

```cpp
// CdcacTypes.h
enum class TheoryEffort { Cheap, Standard, Full };
enum class CdcacStatus { Sat, Unsat, Unknown };
struct CdcacResult {
    CdcacStatus status;
    std::optional<SamplePoint> model;
    std::vector<SatLit> reasons;  // for Unsat
};
struct ModelSeed {
    std::unordered_map<std::string, mpq_class> values;
};

// AlgebraBackend.h
class AlgebraBackend {
public:
    virtual ~AlgebraBackend() = default;
    virtual RootSet isolateRealRoots(const std::vector<mpz_class>& coeffs) = 0;
    virtual Sign signAt(PolyId p, const std::unordered_map<std::string, mpq_class>& sample) = 0;
    virtual std::vector<std::vector<mpz_class>> projectionPolys(
        const std::vector<std::vector<mpz_class>>& polys, int varIdx) = 0;
};

// CdcacCore.h (skeleton)
class CdcacCore {
public:
    CdcacCore(PolynomialKernel* kernel, AlgebraBackend* algebra);
    CdcacResult solve(const CdcacInput& input);
private:
    CdcacResult solveLevel(int k, SamplePoint& prefix, const CdcacInput& input);
    CdcacResult checkFullSample(const SamplePoint& s);
    Cell buildLeafConflictCell(const ActiveConstraint& c, const SamplePoint& s, int varIdx);
};
```

### 数据结构（P0 简化版）

```cpp
struct RealAlg {
    bool isRational;
    mpq_class rational;
    // P0 只支持 rational； algebraic root 字段留占位
};
struct SamplePoint {
    std::vector<std::string> varOrder;  // x1, x2, ..., xn
    std::vector<RealAlg> values;
};
enum class CellKind { Sector, Section, Point, FullLine };
struct Bound {
    enum Kind { NegInf, PosInf, Rational } kind;
    mpq_class value;  // valid if kind == Rational
    bool open;
};
struct Cell {
    int varIdx;               // index into varOrder
    CellKind kind;
    Bound lower, upper;
    std::vector<SatLit> reasons;
    std::vector<PolyId> guards;
};
struct Covering {
    int varIdx;
    std::vector<Cell> cells;
    bool coversAllLine() const;
    std::optional<RealAlg> chooseSampleOutside(const mpq_class* preferred) const;
};
```

### CdcacSolver::check() 新流程

```cpp
TheoryCheckResult CdcacSolver::check(TheoryEffort effort, const ModelSeed* seed) {
    if (pendingConflict_ && isStillActive(pendingConflict_->reasons))
        return TheoryCheckResult::mkConflict(ReasonManager::toConflict(pendingConflict_->reasons));

    auto trivial = checkConstantsOnly();
    if (!trivial.isConsistent()) return trivial;

    if (effort != TheoryEffort::Full)
        return TheoryCheckResult::consistent();  // defer

    CdcacInput input = buildInput(seed);
    CdcacResult res = core_.solve(input);

    switch (res.status) {
        case CdcacStatus::Sat:
            lastModel_ = res.model;
            return TheoryCheckResult::consistent();
        case CdcacStatus::Unsat:
            pendingConflict_ = PendingConflict{currentMaxLevel(), res.reasons};
            return TheoryCheckResult::mkConflict(ReasonManager::toConflict(res.reasons));
        case CdcacStatus::Unknown:
            return TheoryCheckResult::unknown();
    }
}
```

## 阶段 P1：单变量 CDCAC（3-4 天）

**目标**：完整支持一元非线性实数约束（`x^2 - 2 = 0`, `x^2 < 0`, `x != 0` 等）。

### 算法

1. 收集所有 active polynomials，统一变量顺序（单变量时只有一个）
2. 对每个多项式，用 `AlgebraBackend::isolateRealRoots` 隔离实根
3. 将实轴切分为 sections + sectors
4. 每个 cell 取有理 sample
5. 检查是否存在一个 cell 满足所有约束
6. 若不存在，cells 构成 covering，收集 reasons 返回 conflict

### 核心递归算法（P2 版）

```cpp
CdcacResult CdcacCore::solveLevel(int k, SamplePoint& prefix, const CdcacInput& input) {
    if (k == numVars) {
        return checkFullSample(prefix);
    }
    Covering cover(k);
    while (true) {
        auto sample = cover.chooseSampleOutside(seed ? seed->at(varOrder[k]) : nullptr);
        if (!sample) {
            auto reasons = reasonMgr_.minimize(cover);
            return CdcacResult::unsat(reasons);
        }
        prefix.values.push_back(*sample);
        auto child = solveLevel(k + 1, prefix, input);
        prefix.values.pop_back();

        if (child.status == CdcacStatus::Sat) return child;
        if (child.status == CdcacStatus::Unknown) return child;

        // child is Unsat: generalize to parent cell
        Cell generalized = generalizeConflictCell(k, *sample, child, input);
        cover.add(generalized);
        cover.normalizeAndMerge();
    }
}
```

## 阶段 P3：代数数 Section 支持（4-5 天）

**目标**：支持变量等于 projection polynomial 的根（algebraic sample），打通真正的 CDCAC。

### 关键工作

1. **RealAlg 扩展**：加入 `AlgebraicRoot` 结构（defining poly + root index + isolating interval）
2. **代数数比较**：实现基于 interval refinement 的 `<`, `=`, `>` 比较

4. **Section cell**：`CellKind::Section` 的上下界重合于一个代数根

## 阶段 P4：Lazard / Nullification Fallback（2-3 天）

**目标**：处理 McCallum projection 的 nullification 问题，保证 soundness。

### 策略

- 默认 `ProjectionMode::Conservative`（包含 McCallum 全集 + 额外 guards）
- 检测 nullification（投影多项式在某 prefix 上恒为零）
- 触发 `ProjectionMode::Lazard` fallback
- 仍失败则返回 `Unknown`

## 阶段 P5：Reason Minimization 与 Pruning（2-3 天）

**目标**：减小 conflict clause 大小，提升 CDCL 学习效率。

### 策略

1. **Cell merge**：相邻且 reasons/guards 兼容的 cell 合并
2. **Subsumed cell deletion**：被其他 cell 覆盖的删除
3. **Greedy set-cover**：从 covering labels 中贪心选择最小 reason set
4. **Deletion-based minimization**：逐个尝试删除 reason，检查 covering 是否仍完整

## 阶段 P6：与 CDCL / LRA / Linearizer 深度集成（3-4 天）

**目标**：CDCAC 不再孤立运行，而是与线性求解器协同。

### 工作项

1. **Effort 分层**：
   - `Cheap`：常数 + duplicate + 简单 bound contradiction
   - `Standard`：incremental linearization + LRA model refinement + McCormick cuts
   - `Full`：CDCAC complete check
2. **Model seed**：从 LRA 获取 candidate model，传给 CDCAC 作为 preferred sample
3. **Lemma 回灌**：CDCAC conflict 直接成为 CDCL clause；linearizer cuts 作为 lemmas
4. **Resource budget**：max cells / max depth / timeout → 超出则 Unknown
5. **统计**：`CdcacStats` 计数器（calls / sat / unsat / unknown / cells / projection calls / root isolations / libpoly failures）

## 最小可交付路径（MVP）

如果资源有限，最低要求是完成 **P0 + P1**：
- 架构正确，facade 稳定
- 单变量 CDCAC 完整可用
- 多变量返回 Unknown（但不 crash）

这使得 QF_NRA 中大量单变量问题可以求解，同时架构支持后续扩展。

## Soundness 红线

1. **后端失败 → Unknown**：libpoly 返回异常、root isolation 验证失败、algebraic sign 不确定时，绝不返回 Sat/Unsat
2. **nullification 未处理 → Unknown**：McCallum projection 在 nullification 点未验证前，不能返回 Unsat
3. **所有 Sat 必须经过 sample 验证**：`checkFullSample` 必须逐项检查所有 active constraints
4. **Conflict clause 只能含 active literals**：projection polynomial 本身不能成为 reason
