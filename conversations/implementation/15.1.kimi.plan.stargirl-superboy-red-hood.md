## Chosen Approach: Phase 5 + Framework Extraction

### Part A: PolynomialKernel::terms() Interface

Extend `PolynomialKernel` with a monomial term iterator:

```cpp
struct MonomialTerm {
    mpz_class coefficient;           // exact integer coefficient
    std::vector<std::pair<std::string, int>> powers; // [(var, exp), ...]
};

// Return all monomial terms of a polynomial.
// Empty vector if constant.
virtual std::optional<std::vector<MonomialTerm>> terms(PolyId a) const = 0;
```

**Implementation in LibPolyKernel:**
- libpoly C API has `lp_polynomial_traverse_f` callback type and `lp_polynomial_to_monomial`.
- C++ `polyxx` wrapper does **not** expose term iteration.
- We add a C-compatible callback inside `LibPolyKernel::terms()` that uses the C API on the underlying `lp_polynomial_t*` (accessible via `poly::Polynomial::get_internal()`).
- Each monomial gives us `lp_integer_t` coefficient + `(lp_variable_t, degree)[]` array.
- Map `lp_variable_t` back to string via `revVarMap_`.
- Fallback: if C API unavailable or poly not monomial-friendly, return `std::nullopt`.

**Files modified:**

### Part B: NIA Algebraic Strengthening

#### B1. Multivariable GCD Conflict

Current `checkGcdConflict` only works for univariate polynomials. Extend it:

**Algorithm:**
1. Use `terms()` to get all monomials.
2. Collect the coefficient of each non-constant monomial.
3. Compute `g = gcd(|coeff_1|, |coeff_2|, ..., |coeff_n|)`.
4. If `g > 0` and constant term `c % g != 0` â†’ UNSAT.

**Example:** `2x + 4y = 3` â†’ coefficients [2, 4], gcd=2, 3%2â‰ 0 â†’ UNSAT.

**Example:** `xy + x + y = 0` â†’ coefficients [1, 1, 1], gcd=1 â†’ no conflict.

**Files modified:**
- `src/theory/arith/nia/AlgebraicIntegerReasoner.cpp` â€” rewrite `checkGcdConflict`

#### B2. Factor-Split Lemmas

For constraints of the form `p * q = 0`:

**Algorithm:**
1. Detect if the normalized polynomial is a product of two non-constant polynomials.
   - Use `PolynomialKernel::factor()` or detect syntactic product (`mul(a, b)` in construction history).
   - For now: try `lp_polynomial_factor_square_free` via C API.
2. If `p*q = 0` is asserted as an equality, emit lemma:
   - `p = 0 âˆ¨ q = 0`
3. Register dynamic atoms for `p = 0` and `q = 0` via `TheoryAtomRegistry`.

**Note:** libpoly's factorization works over Q. For integer polynomials this is valid.
**Safety:** Only emit if both factors are non-constant and degree â‰¥ 1.

**Files modified:**
- `src/theory/arith/nia/AlgebraicIntegerReasoner.h/.cpp` â€” implement `checkFactorRules`
- `src/theory/arith/nia/NiaSolver.cpp` â€” pass `registry_` to `algebraic_` if needed

#### B3. Enhanced Modular Reasoning

Current `checkModular` brute-forces residues for 1â€“2 variables. Enhance:

**Algorithm:**
1. For each equality constraint, get `terms()`.
2. For each variable, compute modular bounds from individual monomials.
   - e.g., `x^2 + y^2 = 2` â†’ mod 4: squares are 0 or 1 mod 4. Sum of two squares = 2 mod 4 only if both â‰¡1 mod 4. This constrains both variables to odd.
3. Emit domain restriction lemmas: e.g., `x â‰¡ 1 (mod 2)` as `x = 2k+1`.
   - For now: restrict to simple parity/odd constraints via `DomainStore::excludeValue` or `restrictToFiniteSet`.

**Alternative (simpler):** Keep brute-force but use `terms()` to compute tighter variable bounds before enumeration. If a monomial `a*x^d` has `a % m == 0`, that term vanishes mod m, reducing the search space.

**Files modified:**
- `src/theory/arith/nia/AlgebraicIntegerReasoner.cpp` â€” enhance `checkModular`

#### C2. IntervalTypes.h

```cpp
struct IntervalZ {
    mpz_class lo; // inclusive
    mpz_class hi; // inclusive
    bool isEmpty() const { return lo > hi; }
};

enum class IntervalReasonKind {
    DomainBound,      // from DomainStore lower/upper bound
    Constraint,       // from a normalized constraint
    Propagation       // from interval arithmetic propagation
};

struct IntervalReason {
    IntervalReasonKind kind;
    std::optional<SatLit> lit; // for Constraint/Propagation
};

struct ReasonedInterval {
    IntervalZ interval;
    std::vector<IntervalReason> reasons;
};
```

#### C3. ReasonedBox

```cpp
class ReasonedBox {
public:
    // Get/set interval for a variable
    std::optional<ReasonedInterval> get(const std::string& var) const;
    void set(const std::string& var, ReasonedInterval ri);

    // Narrow: intersect with new interval, merge reasons

    // Check for empty (conflict)

    // Reset

private:

};
```
