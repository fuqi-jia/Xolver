建议 **再 Revise**。这版已经非常接近了，但还不能 Approve。大的方向都对：co-register LIA/LRA、mirror effective active bounds、V1 只做 `x*y`/`x^2`、abstraction lemma 与 cut lemma 分离、square nonnegativity、aux var 内部化，这些都正确。

但仍有一个真正 blocker 和几个小矛盾。

---

# 1. 最大 blocker：`markEmitted(lemma)` 仍然不对

计划前面定义 cache API 时需要：

```cpp
markEmitted(
  NonlinearTermKey key,
  SatLit nonlinearReason,
  vector<mpq_class> boundValues,
  int cutIndex
)
```

但 `NiaSolver::check()` 伪代码里仍然写：

```cpp
pendingLinLemmas_.push_back(std::move(lemma));
linAdapter_->markEmitted(lemma);
```

这还是无法实现。`TheoryLemma` 里没有：

```text
NonlinearTermKey
nonlinearReason
boundValues
cutIndex
```

所以 adapter 无法从 `lemma` 反推出 cache key。这个问题之前已经反复出现，现在仍然残留。

必须改成：

```cpp
struct PendingLinearizationLemma {
    TheoryLemma lemma;
    std::optional<CutCacheKey> cacheKey;
};

struct LinearizationResult {
    LinearizationStatus status;
    std::vector<PendingLinearizationLemma> lemmas;
};
```

然后：

```cpp
for (auto& item : lr.lemmas) {
    if (!lemmaDb.contains(item.lemma)) {
        lemmaDb.insertIfNew(item.lemma);
        pendingLinLemmas_.push_back(std::move(item.lemma));

        if (item.cacheKey) {
            linAdapter_->markEmitted(*item.cacheKey);
        }
    }
}
```

或者让 `IncrementalLinearizer::run()` 内部只对真正进入 `result.lemmas` 的 lemma 标记 emitted。不要再让 adapter 调 `markEmitted(lemma)`。

---

# 2. `D4` 仍有一句容易误导的话

计划写：

```text
If bounds are unconditional (no reasons), omit bound guards.
```

后面又写：

```text
Empty reasons does not mean unconditional.
```

后者才是正确原则。空 reasons 不能自动表示 unconditional，可能只是 reason 丢失。

建议改成：

```text
Only omit bound guards when the bound is explicitly marked global.
Empty reason vectors alone do not imply unconditional bounds.
```

否则 agent 很可能在 `lowerReasons.empty()` 时直接生成 unguarded McCormick cut，这会有 soundness 风险。

---

# 3. `Missing bounds -> NoChange` 测试仍然不准确

测试表里还有：

```text
Missing bounds -> NoChange | x∈[0,10], y unbounded → NoChange
```

但完整 `IncrementalLinearizer::run()` 即使缺少 `y` bound，也可能仍然生成：

```text
abstraction lemma: ¬C ∨ p_L rel 0
square nonnegativity: s >= 0
```

所以这个测试不应该写成 full linearizer 的 `NoChange`。

应改成：

```text
Missing product bounds:
  McCormickGenerator returns no envelope cuts;
  full IncrementalLinearizer may still emit abstraction lemmas.
```

如果要测 `NoChange`，只测 `McCormickGenerator`，不要测完整 `IncrementalLinearizer::run()`。

---

# 4. `x²+y²≤1, x=2 -> UNSAT` 这个测试依赖 tangent，可行但要加前提

这个测试现在写：

```text
mirror x=2 to LIA, square nonnegativity s_x>=0,s_y>=0, plus x=2→s_x>=4 via tangent/secant
```

这个思路可以成立，但前提是：

```text
x=2 被 mirror 到 LIA；
SquareCutGenerator 为 s_x=x² 生成 tangent at a=2；
或者有 bounds 让 tangent midpoint 恰好不够时仍能推出足够强 cut。
```

当前 `SquareCutGenerator` 的 tangent 点如果没有 model，就用 midpoint；如果 bounds 不充分，不一定能推出 `s_x>=4`。所以这个测试要么：

```text
明确 modelX=2 可用；
```

要么换成更稳的测试，例如先给 `x` 单点 bound：

```text
x >= 2, x <= 2
```

然后 tangent 点用 midpoint `a=2`。

---

# 5. mirror lemmas 应该作为独立 step，不要和 linearizer run 混在一起

计划说：

```text
The NIA/NRA adapter scans active assignments and generates mirror lemmas alongside linearizer cuts.
```

这可以，但最好明确成两个步骤：

```text
Step A: mirror effective active linear bounds to LIA/LRA
Step B: run IncrementalLinearizer on nonlinear constraints
```

这样职责清楚，测试也清楚：

```text
mirror test 单独测；
linearizer cut test 单独测；
integration 再测两者配合。
```
