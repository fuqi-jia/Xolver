下面给你一版 **IncrementalLinearizer 详细实现方案**。核心定位：

```text
IncrementalLinearizer 不是 NIA/NRA solver；
不是 ICP；
不是 local search；
不是 MCSAT。

它是一个“非线性约束 → 线性松弛/线性割”的增量组件。
```

它负责把：

```text
x*y
x^2
x^k
monomial
polynomial constraint
```

在已有变量区间下转成 **sound linear constraints / cuts / lemmas**，交给 LIA/LRA 继续处理。

---

# 0. 总体目标

IncrementalLinearizer 的目标是：

```text
给定：
  active nonlinear constraints
  current bounds / domains
  current model candidate
  dynamic atom registry

输出：
  sound linear lemmas / cuts
  或 NoChange / UnknownBudget
```

典型例子：

```text
z = x*y
x ∈ [lx, ux]
y ∈ [ly, uy]
```

生成 McCormick envelope：

```text
z ≥ lx*y + ly*x - lx*ly
z ≥ ux*y + uy*x - ux*uy
z ≤ ux*y + ly*x - ux*ly
z ≤ lx*y + uy*x - lx*uy
```

这些是线性的，可以交给 LRA/LIA。

# 2. 核心职责边界

IncrementalLinearizer 做：

```text
1. 抽象非线性子项；
2. 为非线性子项创建 auxiliary variable；
3. 根据当前 bounds 生成 sound linear envelopes；
4. 根据当前 model violation 生成 separating cut；
5. 维护增量缓存，避免重复生成同一 cut；
6. 通过 TheoryAtomRegistry 注册 dynamic linear atoms；
7. 返回 TheoryLemma / LinearCutSet。
```

不做：

```text
1. 不直接返回 SAT；
2. 不用 sampling 证明 UNSAT；
3. 不自己做 CDCL；
4. 不自己实现 ICP；
5. 不直接修改 SAT solver；
6. 不生成 fake ExprId；
7. 不在没有 bounds 的情况下乱线性化。
```

# 3. 输入输出接口

```cpp
struct LinearizationInput {

```

```cpp
enum class LinearizationStatus {
```

# 4. 基础数据结构

## 4.1 非线性子项 key

```cpp
enum class NonlinearKind {
    Product,
    Square,
    Power,
    Monomial,
    Polynomial
};

struct NonlinearTermKey {
    NonlinearKind kind;
    std::vector<PolyId> children;
    uint32_t exponent = 0;

    bool operator==(const NonlinearTermKey&) const;
};
```

例子：

```text
x*y      -> Product(x,y)
x^2      -> Square(x)
x^5      -> Power(x,5)
x^2*y^3  -> Monomial({x:2,y:3})
```

---

## 4.2 辅助变量

每个非线性子项创建一个 auxiliary variable：

```text
t_xy = x*y
t_x2 = x^2
t_m  = x^2*y^3
```

结构：

```cpp
struct AuxTerm {
    std::string auxName;
    PolyId auxPoly;              // variable poly
    NonlinearTermKey key;
    SortKind sort;               // Int or Real
};
```

命名：

```text
__nl_aux_0
__nl_aux_1
...
```

外部 model 输出必须过滤：

```text
__nl_aux_*
```

---

## 4.3 LinearCut

```cpp
struct LinearCut {
    LinearForm lhs;       // Σ ai xi + c
    Relation rel;         // <=, >=, =
    mpq_class rhs;        // usually 0
    std::vector<SatLit> reasons;
    std::string debugName;
};
```

最终必须转成 dynamic linear atom：

```cpp
SatLit lit = registry->getOrCreateLinearAtom(
    lhs,
    rel,
    rhs,
    owner == NIA ? TheoryId::LIA : TheoryId::LRA
);
```

注意：

```text
NIA nonlinear cut -> LIA linear atom
NRA nonlinear cut -> LRA linear atom
```

但 lemma 的 guard 仍然来自 nonlinear constraint reason。

# 5. 抽象化策略

## 5.1 Product abstraction

对于：

```text
x*y
```

创建：

```text
t = x*y
```

然后原多项式中用 `t` 替换 `x*y`。

例如：

```text
x*y + z <= 5
```

变成：

```text
t + z <= 5
```

并额外生成 McCormick envelope 来约束 `t`。

---

## 5.2 Square abstraction

```text
x^2
```

创建：

```text
s = x^2
```

然后：

```text
x^2 + y <= 10
```

变成：

```text
s + y <= 10
```

并生成 square cuts。

---

## 5.3 Power abstraction

```text
x^k
```

两种方式：

### 方式 A：直接 power aux

```text
p = x^k
```

适合 V1 简化。

### 方式 B：递归 product

```text
x^4 = s*s, s=x^2
x^5 = x*x^4
```

适合后续增强。

V1 建议：

```text
只支持 x^2 和 x^3；
更高幂先 NoChange 或递归 product。
```

# 6. McCormick envelope

## 6.1 前提

对 product：

```text
t = x*y
```

需要 bounds：

```text
x ∈ [lx, ux]
y ∈ [ly, uy]
```

如果任意一边无界：

```text
NoChange
```

不能生成 McCormick。

---

## 6.2 Real McCormick

四个约束：

```text
t ≥ lx*y + ly*x - lx*ly
t ≥ ux*y + uy*x - ux*uy
t ≤ ux*y + ly*x - ux*ly
t ≤ lx*y + uy*x - lx*uy
```

转成标准线性形式：

```text
-t + lx*y + ly*x - lx*ly <= 0
-t + ux*y + uy*x - ux*uy <= 0
 t - ux*y - ly*x + ux*ly <= 0
 t - lx*y - uy*x + lx*uy <= 0
```

这四条对 `t=x*y` 是 sound over-approx。

---

## 6.3 Integer case

对于 NIA，McCormick 对整数变量仍然 valid，因为整数点是实数点子集。

但可以加整数强化：

```text
x,y,t are Int
```

如果 `x,y` 都是整数，`t` 也应是整数。辅助变量 `t` 应注册为 Int sort。

第一版不做更强 integer hull。

---

# 7. Square cuts

对于：

```text
s = x^2
x ∈ [l,u]
```

## 7.1 Secant upper bound

如果 `x^2` 是 convex，在 `[l,u]` 上：

```text
x^2 <= (l+u)x - l*u
```

所以：

```text
s <= (l+u)x - l*u
```

线性化：

```text
s - (l+u)x + l*u <= 0
```

这是上界 cut。

---

## 7.2 Tangent lower bound

对任意 point `a ∈ [l,u]`：

```text
x^2 >= 2a*x - a^2
```

所以：

```text
s >= 2a*x - a^2
```

线性化：

```text
-s + 2a*x - a^2 <= 0
```

选择 `a`：

```text
1. 当前 model value of x；
2. midpoint；
3. l；
4. u。
```

V1 建议用当前 model，如果没有 model，则 midpoint。

---

## 7.3 对约束本身的直接 cut

对于：

```text
x^2 <= c
```

不要通过 linearizer 做主力，这个应交给 ICP / SquareBoundReasoner。

Linearizer 可以辅助生成 secant/tangent，但不要替代 interval reasoning。

---

# 8. Product separating cut

如果当前 model 给出：

```text
x = x0
y = y0
t = t0
```

且：

```text
t0 != x0*y0
```

可以生成当前 box 下违反的 McCormick inequality。

流程：

```text
1. 用当前 bounds 生成四个 McCormick cuts；
2. 在当前 model 上 evaluate；
3. 找出被违反最多的 cut；
4. 返回该 cut。
```

这比一次性返回四条更温和。

配置：

```cpp
bool emitAllMcCormickCuts = false;
```

V1 可以默认：

```text
emit violated cuts only
```

如果没有 model：

```text
emit all envelope cuts
```

---

# 9. Polynomial linearization

对于约束：

```text
p(x) rel 0
```

流程：

```text
1. 分解 p 的 terms；
2. 抽象所有 nonlinear terms；
3. 得到 linearized polynomial p_L；
4. 对 p_L rel 0 生成一个 guarded linear atom；
5. 对每个 aux term 生成 envelope cuts。
```

例子：

```text
x*y + z <= 5
```

抽象：

```text
t = x*y
t + z <= 5
```

返回 lemmas：

```text
C_active -> (t + z <= 5)
C_active -> McCormick(t,x,y)
```

更具体地：

```text
¬C ∨ lit(t+z<=5)
¬C ∨ lit(mccut1)
¬C ∨ lit(mccut2)
...
```

注意：`t+z<=5` 本身不是原公式的等价替代，只有配合 envelope 才是 relaxation。

---

# 10. Soundness 关键点

对原约束：

```text
C: p(x) <= 0
```

抽象出 `p_L(x,t)`。

如果只加入：

```text
p_L(x,t) <= 0
```

和 McCormick envelope，它是原问题的 **relaxation**，即所有真实解满足它，但它可能包含伪解。

这对于 pruning 是安全的：

```text
如果 relaxation UNSAT，则原问题 UNSAT。
```

但如果 relaxation SAT，不能推出原问题 SAT。

所以 Linearizer 只能用于：

```text
1. 加强 linear relaxation；
2. 发现 linear conflict；
3. 生成 cuts；
4. 排除当前 spurious model。
```

不能直接 SAT。

# 11. Lemma 形式

## 11.1 Guarded linearization lemma

对于 active nonlinear atom `C`：

```text
C -> L
```

转成 clause：

```text
¬C ∨ L
```

其中 `L` 是 dynamic linear atom。

例如：

```text
C: x*y + z <= 5
L: t_xy + z <= 5
```

lemma：

```text
¬C ∨ L
```

---

## 11.2 Guarded envelope lemma

```text
C -> McCormickCut
```

lemma：

```text
¬C ∨ cut
```

如果 cut 依赖 bounds，例如：

```text
x ∈ [lx,ux]
y ∈ [ly,uy]
```

则必须 guarded by bound reasons：

```text
¬C ∨ ¬B_lx ∨ ¬B_ux ∨ ¬B_ly ∨ ¬B_uy ∨ cut
```

第一版如果 bounds 来自 unconditional asserted constraints，可以 include all bound reasons。

不要生成不带 bound reasons 的 McCormick cut，除非 bounds 是 global static bounds。

---

# 12. Bound reason 设计

```cpp
struct BoundInfo {
    mpq_class lower;
    mpq_class upper;
    std::vector<SatLit> lowerReasons;
    std::vector<SatLit> upperReasons;
};
```

生成 product cut reasons：

```text
reasons =
  nonlinear constraint reason
  + x lower reasons
  + x upper reasons
  + y lower reasons
  + y upper reasons
```

Square cut reasons：

```text
nonlinear constraint reason
+ x lower reasons
+ x upper reasons
```

# 13. Incremental cache

## 13.1 Aux cache

```cpp
std::unordered_map<NonlinearTermKey, AuxTerm> auxCache_;
```

同一个 `x*y` 不重复创建 aux。

要 canonicalize：

```text
x*y == y*x
```

Product key 应排序 children。

---

## 13.2 Cut cache

```cpp
std::unordered_set<LinearCutKey> emittedCuts_;
```

避免重复返回同一 cut。

Key 包含：

```text
cut linear form canonical representation
relation
rhs
guard reasons maybe omitted
```

注意：如果 bounds 变了，McCormick cut 也变了，所以 key 里要包含 bound values：

```cpp
struct McCormickCutKey {
    AuxTermId aux;
    Var x;
    Var y;
    mpq_class lx, ux, ly, uy;
    int cutIndex;
};
```

---

## 13.3 Bound versioning

每个变量 bounds 有 version：

```cpp
struct BoundVersion {
    uint64_t lowerVersion;
    uint64_t upperVersion;
};
```

当 bounds 更新时，重新生成相关 cuts。

# 14. 增量调度

Linearizer 不应该每次扫描所有多项式。

建立 watchers：

```text
nonlinear term depends on variables x,y
constraint depends on nonlinear terms t1,t2
```

当：

```text
1. 新 nonlinear constraint active；
2. 某个 variable bound 更新；
3. 当前 model 发生变化；
```

触发相关 term 重新 linearize。

```cpp
class LinearizationWorklist {
    void pushConstraint(ConstraintId);
    void pushTerm(NonlinearTermKey);
    std::optional<Task> pop();
};
```

V1 可以简单全量扫描；V2 加 worklist。

# 15. 与 LIA/LRA 交互

## 15.1 生成 linear atom

```cpp
SatLit cutLit = registry->getOrCreateLinearAtom(
    LinearForm form,
    Relation::Leq,
    mpq_class(0),
    linearTheoryId
);
```

其中：

```text
NIA -> LIA
NRA -> LRA
```

---

## 15.2 返回 lemma

```cpp
TheoryLemma lemma;
lemma.lits = {
    nonlinearReason.negated(),
    boundReasons.negated()...,
    cutLit
};
```

注意：如果 `boundReasons` 是 positive asserted literals，那么 guard 中放它们的 negation。

---

## 15.3 Linear solver route

新增 dynamic linear atom 必须能被 LIA/LRA assert：

```text
cutLit true -> LIA/LRA receives linear constraint
```

不要让它回到 NIA/NRA。

# 16. 与 ICP 的关系

ICP 和 Linearizer 的分工：

```text
ICP:
  收缩变量区间；
  empty box conflict；
  split suggestion。

Linearizer:
  根据区间生成线性 relaxation；
  交给 LIA/LRA 做 conflict / propagation。
```

推荐 pipeline：

```text
NIA/NRA check:
  1. Linear domain / basic bound
  2. ICP contraction
  3. IncrementalLinearizer generates cuts
  4. LIA/LRA relaxation check
  5. Local search / exact solver / branch
```

不要让 Linearizer 自己做 contraction。

---

# 17. 与 Local Search 的关系

Local Search 找到 candidate 后：

```text
candidate violates t = x*y
```

Linearizer 可以根据该 point 生成 separating cut。

例如当前：

```text
x0,y0,t0
```

违反某个 McCormick inequality，则返回该 cut。

如果所有 McCormick 都不违反，但 `t != x*y`，说明 relaxation 太弱，需要：

```text
1. tighter bounds from ICP；
2. split box；
3. nonlinear exact check。
```

Linearizer 本身不解决这个问题。

# 18. 支持范围分阶段

## V1：最小强版本

支持：

```text
1. Product x*y with finite bounds；
2. Square x^2 with finite bounds；
3. Single active polynomial constraint abstraction；
```

```text
x*y*z
x^2*y

```

```text
t1 = x*y

```

```text
s = x^2

```

## V3：convex cuts

支持：

```text
x^2

```

生成 tangent / secant cuts。

---

## V4：integer strengthening

支持：

```text
integer McCormick strengthening
modular cuts
parity cuts
RLT-style cuts
```

## 19.1 McCormick basic

```text
x∈[0,10], y∈[0,10], t=x*y
```

检查四条 cut：

```text
t >= 0
t >= 10x + 10y -100
t <= 10x
t <= 10y
```

## 19.2 McCormick signed interval

```text
x∈[-2,3], y∈[4,5]
```

检查公式系数正确。

---

## 19.3 Square secant

```text
s=x^2, x∈[-2,3]
```

secant：

```text
s <= (1)x + 6
```

因为：

```text
(l+u)=1
-lu=6
```

即：

```text
s - x - 6 <= 0
```

---

## 19.4 Tangent

at `a=2`：

```text
s >= 4x - 4
```

即：

```text
-s + 4x - 4 <= 0
```

---

## 19.5 Guard reasons

如果 bounds 来自：

```text
B1: x>=0
B2: x<=10
B3: y>=0
B4: y<=10
C: x*y+z<=5
```

cut lemma 必须包含：

```text
¬C ∨ ¬B1 ∨ ¬B2 ∨ ¬B3 ∨ ¬B4 ∨ cut
```

---

## 19.6 No bound no cut

如果 `x` 或 `y` 缺上界/下界：

```text
NoChange
```

---

# 20. Correctness red lines

```text
1. 没有 finite bounds，不生成 McCormick。
2. McCormick cut 必须带 bound reasons。
3. Linearizer 不返回 SAT。
4. Linearizer 不用 sampling 证明 UNSAT。
5. 所有 cuts 必须是原非线性约束在当前 bounds 下的 sound relaxation。
6. Dynamic atoms 必须真实注册。
7. Cut lit 必须 route 到 LIA/LRA。
8. Aux vars 不暴露到 public model。
9. 如果当前 bounds 后续变宽，旧 cut 必须由 guard reasons 保护，不能无条件保留。
10. Budget exhausted -> NoChange / UnknownBudget。
```

# 23. 最终建议

这个模块适合独立 agent 并行写，但一定要让它保持边界：

```text
IncrementalLinearizer = nonlinear term abstraction + sound linear cuts.
```

第一版只做：

```text
McCormick for x*y
Square secant/tangent for x^2
guarded dynamic linear lemmas
NIA/NRA adapter
```

不要让它扩展成 ICP、NIA solver 或 local search。这样并行开发最安全，也最容易验收。
