有办法，而且我建议**不要只做 `MAX_SINGLE_BOUND_SPLITS=3`**。那个只是止血，会让无限链停下来，但也会很快把本来可解的问题变成 Unknown。你现在遇到的问题本质是：**branch 只会切半空间，但没有把非线性约束转成有限候选域/搜索盒**。日志里已经看到单边界变量会生成 `x≤-1∨x≥0`、`x≤-2∨x≥-1` 这种无限链，导致 SAT 深度搜索负整数。

我建议做一个三层方案。

---

# 方案 A：立即止血，限制 single-bound / unbounded 分裂

这个可以马上做，但只作为安全阀。

```cpp
static constexpr int MAX_UNBOUNDED_SPLITS_PER_VAR = 1;
static constexpr int MAX_SINGLE_BOUND_SPLITS_PER_VAR = 3;
```

规则：

```text
both-bound 变量：允许继续二分，因为区间会收敛；
unbounded 变量：只允许 k=0 中心分裂一次；
lower-only / upper-only 变量：最多分裂 3 次；
超过预算后跳过该变量；
所有变量都超过预算后返回 Unknown。
```

这能防止：

```text
x <= 0
x <= -1
x <= -2
...
```

但它不能让 `x²+y²=65` 稳定 SAT。它只是避免无限搜索。

---

# 方案 B：更应该做的是“非线性约束导出搜索盒”

对 `x²+y²=65`，branch 不该从无界开始盲切。这个约束本身就隐含：

```text
|x| <= floor(sqrt(65))
|y| <= floor(sqrt(65))
```

也就是：

```text
-8 <= x <= 8
-8 <= y <= 8
```

一旦拿到这个 box，`BoundedNiaSolver` 就能直接枚举，最多 `17*17=289` 个点，很快找到 `(1,8),(4,7),(7,4),(8,1)` 等解。日志里也提到这个例子理论上有解但 branch 搜索可能很深，这正是缺少 sum-of-squares bound 的表现。

所以真正该加的是：

```text
SumOfSquaresBoundReasoner
```

先不要做完整 ICP，就做一个很安全的规则：

```text
Σ xi² <= c, c >= 0
  => 对每个 xi 加 bound [-floor(sqrt(c)), floor(sqrt(c))]

Σ xi² = c, c >= 0
  => 因为 Σ xi² <= c，所以同样加 bound [-floor(sqrt(c)), floor(sqrt(c))]

Σ xi² = c, c < 0
  => UNSAT

Σ xi² <= c, c < 0
  => UNSAT
```

这样对：

```text
x² + y² = 65
```

normalized 后是：

```text
x² + y² - 65 = 0
```

识别为：

```text
x² + y² = 65
```

然后加：

```text
x,y ∈ [-8,8]
```

接下来 bounded enumeration 直接解决，不需要 branch 深搜。

这个规则 sound、简单、收益巨大。

---

# 方案 C：branch 前先尝试“搜索盒构造”

我建议把 `buildBranchLemma` 前面加一个步骤：

```text
tryDeriveSearchBox()
```

顺序变成：

```text
LinearDomain
→ SquareBound
→ Univariate
→ Algebraic
→ SumOfSquaresBound
→ EmptyDomain
→ BoundedEnumeration
→ LocalSearch
→ Branch
→ Unknown
```

也就是说，branch 前必须尽量把变量变成 finite domain。你的目标不是“让 SAT 盲目切整数空间”，而是：

```text
先用非线性结构推 domain；
domain finite 后交给 bounded solver；
只有完全推不出 domain 时才 branch。
```

---

# 具体怎么实现

## 1. 新增 `SumOfSquaresBoundReasoner`

文件：

```text
src/theory/arith/nia/SumOfSquaresBoundReasoner.h/.cpp
```

接口：

```cpp
class SumOfSquaresBoundReasoner {
public:
    NiaReasoningResult run(
        const std::vector<NormalizedNiaConstraint>& constraints,
        DomainStore& domains
    );
};
```

只识别这类 normalized polynomial：

```text
x1² + x2² + ... + xn² - c rel 0
```

其中：

```text
每个平方项系数为 +1；
没有交叉项；
没有一次项；
常数项为 -c；
变量全是整数。
```

支持：

```text
rel = Leq:
  x1² + ... + xn² <= c

rel = Eq:
  x1² + ... + xn² = c
```

规则：

```text
若 c < 0:
  Leq/Eq 都 UNSAT

若 c >= 0:
  对每个 xi:
    lower = -floorSqrt(c)
    upper =  floorSqrt(c)
  DomainUpdated
```

`Geq` 暂时不处理，`Neq` 暂时不处理。

---

## 2. 接入位置

放在 `SquareBoundReasoner` 后面、`BoundedNiaSolver` 前面：

```text
normalize
→ trivial
→ linearDomain
→ squareBound
→ sumOfSquaresBound
→ univariate
→ algebraic
→ emptyDomain
→ bounded
→ localSearch
→ branch
→ unknown
```

如果 `x²+y²=65`，它会在 `sumOfSquaresBound` 处把 `x,y` 都变成有限域，然后 bounded enumeration 接管。

---

## 3. branch 改成“最后手段”

`buildBranchLemma` 只在这些条件之后触发：

```text
1. 所有 domain reasoner 都跑完；
2. domains_.isEmpty() 为 false；
3. bounded solver 不能运行或 UnknownBudget；
4. local search 没找到模型；
5. 没有新的 domain update；
6. branch budget 未超。
```

不要让 branch 在 domain propagation 之前触发。

---

## 4. single-bound branch 仍然加预算

即使有 sum-of-squares，仍然要防无限链：

```cpp
bool isBothBounded(const IntDomain& d) {
    return d.hasLower && d.hasUpper;
}

bool isSingleBounded(const IntDomain& d) {
    return d.hasLower ^ d.hasUpper;
}

bool isUnbounded(const IntDomain& d) {
    return !d.hasLower && !d.hasUpper;
}
```

选择变量时：

```text
优先 both-bounded；
其次 single-bounded，但受预算限制；
最后 unbounded，只允许中心 split。
```

伪代码：

```cpp
std::optional<BranchCandidate> chooseBranchVar(...) {
    // 1. both-bounded: best
    for (v : varsByLargestFiniteInterval) {
        if (hasBothBounds(v) && !isSingleton(v)) {
            return {v, midpoint(v)};
        }
    }

    // 2. single-bound: limited
    for (v : varsByActivity) {
        if (hasSingleBound(v) &&
            branchCountPerVar_[v] < maxSingleBoundSplits) {
            return {v, singleBoundSplitK(v)};
        }
    }

    // 3. unbounded: center only
    for (v : varsByActivity) {
        if (isUnbounded(v) &&
            branchCountPerVar_[v] < maxUnboundedSplits) {
            return {v, 0};
        }
    }

    return std::nullopt;
}
```

# 对 `x²+y²=65` 的正确处理路径

不要指望 branch 主导。理想路径应该是：

```text
input:
  x² + y² = 65

normalize:
  x² + y² - 65 = 0

SumOfSquaresBound:
  x,y ∈ [-8,8]

BoundedNiaSolver:
  enumerate 289 assignments

validator:
  finds (1,8), (4,7), ...

return SAT
```

这才是合理的 NIA solver 行为。

# 最终建议

有办法，而且最推荐的是：

```text
不要让 branch 单独解决无界非线性；
先从非线性约束里推出搜索盒；
有盒子就 bounded enumeration；
没盒子才 branch；
branch 必须带预算。
```

所以当前下一步我建议从 **SumOfSquaresBoundReasoner + branch safety** 开始。
这比单纯加 `MAX_SINGLE_BOUND_SPLITS=3` 更有效，也更符合 SMT/NIA 求解器的架构。
