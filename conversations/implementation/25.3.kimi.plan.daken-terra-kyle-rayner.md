## 2. Critical Design Fixes from Review v2

### Fix 1: Shared equalities/disequalities are `TheoryId::Combination`, not owned by a single theory

After purification:

```text
t = x + 1        // LIA bridge atom (owner = LIA)
u = f(t)         // EUF bridge atom (owner = EUF)
u = y            // Combination shared equality (owner = Combination)
```

`u = y` is an interface equality between two shared constants. It must:
1. Be registered as a `TheoryId::Combination` atom.
2. Have its own `SatVar` via `TheoryAtomRegistry::getOrCreateSharedEqualityAtom(u, y)`.
3. Be routed by `TheoryManager::assertTheoryLit()` to `SharedEqualityManager` (not to EUF or LIA directly).
4. Only when SAT assigns it true/false does `SharedEqualityManager` union them and forward to owner theories.

Invariant:
```text
After purification:
1. Every non-interface theory atom has exactly one owner theory.
2. Equality between shared constants is represented as a TheoryId::Combination atom.
3. Bridge definitions are routed to the theory in whose signature the RHS is pure.
4. No mixed compound term reaches any theory solver.
```

`PurifiedAtom.owner` values:
```cpp
TheoryId::EUF
TheoryId::LRA
TheoryId::LIA
TheoryId::NRA
TheoryId::NIA
TheoryId::Combination   // ONLY for shared equalities / disequalities
```

### Fix 2: Stage B requires verified SAT decision hook; no unit-clause fallback

**Hard requirement**: `QF_UFLIA` is enabled **only if** arrangement literals can be made SAT-decision-visible without constraining one branch.

Acceptance for B-2:
1. `getOrCreateSharedEqualityAtom(a,b)` creates a real `SatVar`.
2. The SAT backend can branch on this variable via decision callback / user decision hook / assumption-based controlled split.
3. **No arbitrary unit clause `Eq(a,b)` or `¬Eq(a,b)` may be emitted as arrangement fallback.**
4. **If no valid decision mechanism exists, `QF_UFLIA` remains disabled.**

This is a non-negotiable precondition for Stage B. It must be verified before Stage B implementation begins.

### Fix 3: EUF deduced equality only for registered SharedTermId constants

EUF does **not** propagate arbitrary internal compound term equalities to the combination layer.

Correct behavior:
```text
1. interface x=y causes e-graph merge.
2. if EUF has asserted disequality f(x)≠f(y), then conflict with congruence explanation.
   (This is internal to EUF; no shared equality propagation needed.)
3. getDeducedSharedEqualities() ONLY reports equalities between registered SharedTermId constants.
   Example: if both f(x) and f(y) were purified into shared constants u=f(x), v=f(y),
   then EUF reports Eq(u,v) when u and v are in the same e-graph class.
```

### Fix 4: EUF model returns equality partition, not numeric concrete values

```cpp
struct EufTheoryModel {
    // Equality partition over shared terms
    std::vector<std::vector<SharedTermId>> sharedPartitions;

    // Disequality requirements between classes
    std::vector<std::pair<size_t, size_t>> disequalities;

    // Uninterpreted function interpretation (optional, lazy)
    // ...

```

Numeric values for Int/Real sorts come from LRA/LIA models. EUF only guarantees equivalence class consistency. The combination layer reconciles partitions with numeric assignments.

### Fix 5: Pending events carry decision level and rollback correctly

```cpp

    SharedTermId a, b;
    bool isEquality;
    SatLit reasonLit;
    int decisionLevel;  // MUST be recorded
```

`TheoryManager::backtrackToLevel(level)`:
```cpp
void TheoryManager::backtrackToLevel(int level) {
    // Remove pending events from levels above target
    auto it = std::remove_if(
        pendingSharedEqEvents_.begin(),
        pendingSharedEqEvents_.end(),
        [level](const auto& ev) { return ev.decisionLevel > level; });
    pendingSharedEqEvents_.erase(it, pendingSharedEqEvents_.end());

    sharedEqMgr_.rollback(snapshots_.back().sharedEqSnap);
    arrangementMgr_.rollback(snapshots_.back().arrangementSnap);
    for (auto* solver : solvers_) solver->backtrackToLevel(level);
}
```

### Phase A-1: Purifier + SharedTermRegistry

**Goal**: Decompose mixed formulas into pure theory atoms + bridge definitions + shared equality atoms.

**Purifier algorithm**:
```
Input: CoreIr assertions for mixed logic
Output: vector<PurifiedAtom> + SharedTermRegistry populated

For each assertion ExprId e:
    Walk e bottom-up
    If node is theory atom T(...):
        For each child c that is alien to T:
            If c not yet purified:
                fresh = createPurificationVar(c)
                emit bridge atom: fresh = c  (owner = theory_of(c))
                register fresh in SharedTermRegistry
                add owner theory_of(c) to fresh.owners
            replace c with fresh
        emit purified atom e' with owner = theory_of(T)
        for each shared term s in e': add owner theory_of(T) to s.owners
    If node is equality between two shared constants a, b:
        emit shared equality atom with owner = TheoryId::Combination
```

**Key invariant**: After purification, every atom is either:
- A pure theory atom with exactly one owner, OR
- A shared equality/disequality atom with `TheoryId::Combination`.

### Phase A-4: Theory solver NO hooks (EUF + LRA)

**Goal**: Both solvers accept interface equalities/disequalities and report deduced equalities between registered shared constants.

```cpp
class TheorySolver {
public:
    virtual bool supportsCombination() const { return false; }

    virtual TheoryCheckResult assertInterfaceEquality(
        SharedTermId a, SharedTermId b, SatLit reason) {
        return TheoryCheckResult::consistent();
    }

    virtual TheoryCheckResult assertInterfaceDisequality(
        SharedTermId a, SharedTermId b, SatLit reason) {
        return TheoryCheckResult::consistent();
    }

    virtual std::vector<SharedEqualityPropagation>
    getDeducedSharedEqualities() { return {}; }

    virtual std::optional<TheoryModel>
    getModel(const SharedTermRegistry& sharedTerms) const { return std::nullopt; }
};
```

**EufSolver**:
- `assertInterfaceEquality(a,b,reason)`: intern shared terms into e-graph, merge classes.
- `assertInterfaceDisequality(a,b,reason)`: store disequality via existing `ActiveDisequality`.
- `getDeducedSharedEqualities()`: after `check()`, scan e-graph. For each pair of **registered SharedTermId constants** in same class, if not already in `SharedEqualityManager`, report with `explainEquality` reasons.
  - **Never** report arbitrary internal compound terms like `Eq(f(x), f(y))` unless both were explicitly registered as shared constants.
- `getModel()`: return equality partition over shared terms (`EufTheoryModel` with partitions, not numeric values).

**LraSolver**:
- `assertInterfaceEquality(a,b,reason)`: treat as `x - y = 0` bound in simplex.
- `assertInterfaceDisequality(a,b,reason)`: if bounds already imply `x=y`, return conflict; else store disequality.
- `getDeducedSharedEqualities()`: after simplex check, for any pair of shared constants with tight bounds (`x-y ≤ 0` and `y-x ≤ 0`), report equality with bound reasons.
- `getModel()`: return rational values for shared variables.

**Files to modify**:
- `src/theory/TheorySolver.h`
- `src/theory/euf/EufSolver.h/.cpp`
- `src/theory/arith/lra/LraSolver.h/.cpp`

**Tests**:
- EUF: assert `x=y` interface → `same(x,y)` true; if `f(x)` and `f(y)` are shared constants, `getDeduced` reports `Eq(f(x), f(y))`
- EUF: assert `x≠y` interface → `same(x,y)` false; no conflict if not already same
- EUF: internal disequality `f(x)≠f(y)` + interface `x=y` → internal conflict (no shared propagation needed)
- LRA: assert `x≤y`, `y≤x` → `getDeduced` reports `Eq(x,y)`
- LRA: assert `x=y` interface → simplex has `x-y=0` bound
- Backtrack interface equality in both solvers → state restored

**Estimated**: 4–5 days.

### Phase A-5: TheoryManager NO coordinator (convex)

**Goal**: Single-pass CDCL(T) coordinator. Pending events carry decision level.

```cpp
struct PendingSharedEqEvent {
    SharedTermId a, b;
    bool isEquality;
    SatLit reasonLit;
    int decisionLevel;  // Recorded at enqueue time
};

TheoryCheckResult TheoryManager::check(TheoryLemmaDatabase& lemmaDb) {
    // 1. Process pending SAT-assigned shared equalities/disequalities
    for (auto& ev : pendingSharedEqEvents_) {
        if (ev.isEquality) {
            sharedEqMgr_.assertEquality(ev.a, ev.b, ev.reasonLit);
            for (auto* solver : solversOwning(ev.a, ev.b)) {
                auto r = solver->assertInterfaceEquality(ev.a, ev.b, ev.reasonLit);
                if (r.isConflictOrLemma()) return r;
            }
        } else {
            sharedEqMgr_.assertDisequality(ev.a, ev.b, ev.reasonLit);
            for (auto* solver : solversOwning(ev.a, ev.b)) {
                auto r = solver->assertInterfaceDisequality(ev.a, ev.b, ev.reasonLit);
                if (r.isConflictOrLemma()) return r;
            }
        }
    }
    pendingSharedEqEvents_.clear();

    // 2. Check shared equality/disequality conflict
    if (auto c = sharedEqMgr_.checkDisequalityConflict()) {
        return TheoryCheckResult::mkConflict(*c);
    }

    // 3. Run each theory check
    for (auto* solver : solvers_) {
        auto r = solver->check(lemmaDb);
        if (r.isConflictOrLemma()) return r;
    }

    // 4. Collect theory-propagated shared equalities
    for (auto* solver : solvers_) {
        for (auto& prop : solver->getDeducedSharedEqualities()) {
            SatLit eqLit = registry_.getOrCreateSharedEqualityAtom(prop.a, prop.b);
            if (!sat_.isAssignedTrue(eqLit)) {
                TheoryLemma lemma;
                for (auto& reason : prop.reasons) {
                    lemma.lits.push_back(reason.negated());
                }
                lemma.lits.push_back(eqLit);
                if (lemmaDb.insertIfNew(lemma)) {
                    return TheoryCheckResult::mkLemma(std::move(lemma));
                }
            }
        }
    }

    // 5. Model combination (Stage A: convex)
    auto combo = tryCombineModels();
    if (combo.ok) return TheoryCheckResult::consistent();
    if (combo.conflict) return TheoryCheckResult::mkConflict(*combo.conflict);
    return TheoryCheckResult::unknown();
}
```

`assertTheoryLit()` routing:
```cpp
void TheoryManager::assertTheoryLit(const TheoryAtomRecord& atom,
                                     bool value, int level, SatLit lit) {
    if (atom.theory == TheoryId::Combination) {
        auto& payload = std::get<SharedEqualityPayload>(atom.payload);
        pendingSharedEqEvents_.push_back({
            payload.a, payload.b, value, lit, level
        });
        return;
    }
    auto* solver = solverByTheory_[atom.theory];
    if (solver) solver->assertLit(atom, value, level, lit);
}
```

## 6. Acceptance Criteria

1. **Tautology split is never used.**
2. **Only SAT-assigned equalities enter `SharedEqualityManager` union-find.**
3. **Every propagated equality has reconstructable reason clause.**
4. **Sampling/model validation never produces a lemma.**
5. **Purified bridge atoms route to exactly one owner theory; shared equalities route to `TheoryId::Combination`.**
6. **Shared equality atoms are canonical and SAT-visible.**
7. **Backtracking restores**: shared equality UF, arrangement decisions, theory-local interface assertions, pending queues (by decision level).
8. **Final `Sat` validates original unpurified formula.**
9. **Single-theory paths unchanged.** `QF_LRA`, `QF_UF` regression tests pass with zero changes.
10. **`QF_UFNIA`/`QF_UFNRA` conservative:** return `Unknown` when unsure.
11. **`QF_UFLRA` acceptance:**
    - `(= x y)` + `(not (= (f x) (f y)))` → **unsat**
    - `(<= x y)` + `(<= y x)` + `(not (= (f x) (f y)))` → **unsat**
    - `(= x y)` + `(= (f x) (f y))` → **sat**
12. **Pending event rollback:** backtrack before `check()` processes events → events removed, no stale processing.
13. **EUF deduced equality:** only between registered `SharedTermId` constants.
14. **EUF model:** returns equality partition, not numeric values.
15. **Stage B precondition:** verified SAT decision hook before `QF_UFLIA` enabled.
