# EUF Solver V1 Cleanup Plan — Revision 5

## 1. Goal

Fix all correctness bugs and internal contradictions in the existing EUF V1 implementation. **No new features** (no model construction, no theory propagation, no mixed-theory support). Pure cleanup of what already exists.

## 2. Current State

NLColver has a working EUF V1 skeleton for pure `QF_UF`:
- `EufTermManager`, `EGraph`, `EufSolver`, `Atomizer` integration
- 17 regression tests + 18 unit tests pass

This plan fixes the bugs that could cause **false SAT**, **invalid explanation**, or **internal inconsistency**.

## 3. Correctness Invariants (These Are the Rules)

Every file change below must satisfy all of these. No exceptions.

| # | Invariant |
|---|-----------|
| I1 | **Bool sort ID** comes from `adapter.getBoolSortId()`. Never hardcoded `0`. Never inferred by scanning `ConstBool`. |
| I2 | **Bool is a two-valued sort.** For every Bool-valued term `b`, the canonical atom is `lit(b) := (= b true)`. `lit(b)=false` means `b=false`. Bool equality/distinct are encoded propositionally, not as ordinary EUF atoms. |
| I3 | **Atomizer owns EUF dedup.** `Atomizer::getOrCreateEufAtom` creates/dedups SAT vars. `TheoryAtomRegistry` only records atoms. No `Registry::getOrCreateEufAtom`. No `atomizer_->registerDynamicAtom` from Registry. |
| I4 | **`EGraph` has `termClass_`** mapping `EufTermId → EClassId`. `find(t)` uses it. `addTerm(t)` initializes it. |
| I5 | **`closeCongruence` returns `CloseStatus`** (Ok / SortMismatch). Header signature matches implementation. Callers check the return value. |
| I6 | **`check()` order**: intern terms → `addTerm` → assert equalities → `closeCongruence` → check true≠false → check disequalities. |
| I7 | **Explanation cycle returns `{false, {}}`** (failure). BFS uses visited-node skip (continue), not abort. Recursive explanation uses `activePairs` guard. |
| I8 | **`pop()` clamps** with `min(scopeLimit, trailSize)`. |
| I9 | **Synthetic allocator** has `reset()` and asserts against `TrueSentinelExpr` / `FalseSentinelExpr`. `Atomizer::reset()` clears `eufAtomDedup_` and resets allocator. |
| I10 | **`distinct` with ≤1 arg** returns SAT true literal. |
| I11 | **C++20** required in CMake. |
| I12 | **Mixed theories** (`QF_UFLIA`, `QF_UFLRA`, `QF_UFNIA`, `QF_UFNRA`) return `Unknown`. No fallback. Explicit logic names only. |

## 4. Change 1: Bool Semantics (Soundness Fix)

**Files**: `src/sat/Atomizer.cpp`, `src/theory/euf/EufSolver.cpp`

### 4a. Bool formula-position atoms

In `EufSolver::check()`, when processing a trail entry for a Bool formula-position atom `lit(b) := (= b true)`:

```cpp
if (value == true) {
    // b = true
    merge(b, trueConstant, reason);
} else {
    // lit(b) = false  =>  b = false
    merge(b, falseConstant, reason);
}
```

This ensures Bool terms are always in either the `true` class or the `false` class. The existing `true != false` check after `closeCongruence` will then correctly detect contradictions.

### 4b. Bool equality / distinct propositionally

In `Atomizer::atomize()`, when `defaultTheory_ == TheoryId::EUF` and an `Eq` or `Distinct` node has Bool sort on all arguments:

**Bool equality**:
```text
(= b1 b2 ... bn)
=> ∧_{i=1}^{n-1} (bi ↔ b_{i+1})
```

Encode pairwise `↔` directly as SAT clauses:
```text
(¬bi ∨ b_{i+1}) ∧ (bi ∨ ¬b_{i+1})
```

Each `bi` is the canonical Bool atom `lit(bi)`. Do **not** register as EUF atom.

**Bool distinct**:
```text
n <= 1  => true
n = 2   => (b1 XOR b2)
n >= 3  => false
```

Encode `XOR` as SAT clauses:
```text
(b1 ∨ b2) ∧ (¬b1 ∨ ¬b2)
```

## 7. Change 4: Explanation BFS Fix

**File**: `src/theory/euf/EGraph.cpp`

### 7a. `explainPath` — standard BFS

Replace `visitedPairs` + early-abort with standard BFS:

```cpp
ExplainResult EGraph::explainPath(
    EufTermId start, EufTermId target,
    const EufTermManager& tm,
    std::unordered_set<uint64_t>& activePairs) {

    std::unordered_map<EufTermId, EufTermId> parentTerm;
    std::unordered_map<EufTermId, size_t> parentEdge;
    std::queue<EufTermId> q;
    std::unordered_set<EufTermId> visited;

    q.push(start);
    visited.insert(start);
    parentTerm[start] = start;

    while (!q.empty()) {
        EufTermId cur = q.front(); q.pop();
        for (size_t ei : adj[cur]) {
            const auto& edge = proofEdges_[ei];
            EufTermId next = (edge.u == cur) ? edge.v : edge.u;
            if (visited.count(next)) continue;  // skip, do NOT abort
            visited.insert(next);
            parentTerm[next] = cur;
            parentEdge[next] = ei;

            if (next == target) {
                // reconstruct
                std::vector<SatLit> reasons;
                EufTermId node = target;
                while (node != start) {
                    size_t eidx = parentEdge[node];
                    auto exp = explainMergeReason(
                        proofEdges_[eidx].reason, tm, activePairs);
                    if (!exp.ok) return {false, {}};
                    appendUnique(reasons, exp.reasons);
                    node = parentTerm[node];
                }
                return {true, std::move(reasons)};
            }
            q.push(next);
        }
    }
    return {false, {}};
}
```

### 7b. `explainMergeReason` — recursive guard

```cpp
ExplainResult EGraph::explainMergeReason(
    const MergeReason& reason,
    const EufTermManager& tm,
    std::unordered_set<uint64_t>& activePairs) {

    if (reason.kind == MergeReasonKind::AssertedEquality) {
        return {true, {reason.assertedLit}};
    }

    std::vector<SatLit> reasons;
    for (auto [argA, argB] : reason.argPairs) {
        if (argA == argB) continue;
        uint64_t pk = pairKey(argA, argB);
        if (!activePairs.insert(pk).second) {
            return {false, {}};  // cycle in recursive explanation
        }
        auto sub = explainEquality(argA, argB, tm, activePairs);
        activePairs.erase(pk);
        if (!sub.ok) return {false, {}};
        appendUnique(reasons, sub.reasons);
    }
    return {true, std::move(reasons)};
}
```
