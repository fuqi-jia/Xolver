# CDCAC P2b–P4 完善计划

## 总体目标

在 P2a 已经完成的基础上，把 CDCAC 的代数层补成一个可靠模块：

```text
P2b：精确 RealAlg 比较 + checked mergeRoots + CoverageResult coversAllLine
P2c：AlgebraicRoot provenance metadata
P2d：mixed sample 的 signAt 分层支持
P3 ：Algebraic Tower reduction / exact zero detection
P4 ：algebraic-prefix root isolation + nullification contract
```

**核心红线**：
- same isolating interval 绝不能作为代数根相等判定；
- Unknown comparison 绝不能进入 sort / merge / coversAllLine；
- exact zero 不能靠 interval refinement 猜，必须靠 gcd / reduction / exact sign 证明。

---

## P2b：精确 RealAlg 比较与 root merge 可靠化

### 状态审计

| 项 | 状态 | 需补 |
|---|---|---|
| CompareResult 枚举 | ✅ 已有 | 无 |
| same poly+index → Equal | ✅ 已有 | 无 |
| disjoint intervals → Less/Greater | ✅ 已有 | 无 |
| rational-algebraic eval+refine | ✅ 已有 | 无 |
| algebraic-algebraic gcd+locate+refine | ✅ 已有 | 无 |
| mergeRoots insertion-style | ✅ 已有 | **Unknown 时必须失败，不能 tie-break** |
| coversAllLine 走 compareRealAlg | ✅ 已有 | **需从 bool 改 CoverageResult 三值** |
| boundSame 对 open/closed 的检查 | ❌ | **需补：相邻 cell 同边界但双开 => 不覆盖** |

### 1. mergeRoots：Unknown 时失败

当前 mergeRoots 在 `Unknown` 时 fallback 到 `lower bound` tie-break。这破坏了 soundness——
如果 comparator 无法判定顺序，排序结果不可信。

修改：
```cpp
std::optional<RootSet> mergeRoots(const std::vector<RootSet>& rootSets);
```
任一比较返回 `Unknown` 时，返回 `std::nullopt`。
调用者 `solveLevel` 收到 `nullopt` 时返回 `CdcacResult::mkUnknown(...)`。

### 2. coversAllLine：bool → CoverageResult

```cpp
enum class CoverageResult { Covers, DoesNotCover, Unknown };
CoverageResult coversAllLine(AlgebraBackend* algebra, const Covering& cover);
```

`boundSame` 逻辑：
1. 用 `compareRealAlg` 判定边界值是否 Equal；
2. 若 `Unknown`，整体返回 `Unknown`；
3. 若 Equal，还需检查至少一侧 closed（sector→section 或 section→sector 的相邻关系天然满足，但 sector→sector 双开不行）。

### 3. boundSame：open/closed 契约

相邻 cell 覆盖全线的条件：
```
prev.upper.value == curr.lower.value   (exact same point)
&& !(prev.upper.open && curr.lower.open)   (至少一侧闭合)
```

当前 `Bound::open` 语义：
- sector lower/upper: open=true（开区间）
- section lower/upper: open=false（闭区间，单点）
- negInf/posInf: open=true（约定）

sector(-inf, r) + section[r, r]：upper open=true, lower open=false → OK
section[r, r] + sector(r, +inf)：upper open=false, lower open=true → OK
sector(-inf, r) + sector(r, +inf)：upper open=true, lower open=true → **不 OK**

## P2c：AlgebraicRoot Provenance Metadata

### 数据结构

```cpp
struct RootOrigin {
    PolyId liftedDefiningPoly = NullPoly;
    std::string mainVar;
    int level = -1;
};

struct AlgebraicRoot {
    UniPolyId definingPoly = NullUniPolyId;
    int rootIndex = -1;
    mpq_class lower;
    mpq_class upper;
    std::vector<RootOrigin> origins;   // 支持多个来源多项式
};
```

### 创建时填充

在 `solveLevel(k, prefix, input)` 中，对每个约束的 polynomial：
```cpp
for (auto& r : roots.roots) {
    if (r.isAlgebraic()) {
        r.root.origins.push_back({c.poly, var, k});
    }
}
```

### merge 时保留

`mergeRoots` 去重时，保留的根的 `origins` 应合并被丢弃根的 `origins`（去重后 push_back）。

## P2d：signAt 分层支持

### 分层结构

```cpp
Sign signAt(PolyId p, const SamplePoint& s) {
    if (s.allRational())      return signAtRational(p, s);
    if (s.numAlgebraic() == 1) return signAtOneAlgebraic(p, s);
    return signAtAlgebraicTower(p, s);   // P3 前返回 Unknown
}
```

### 1. signAtRational

现有逻辑不变：`kernel_->sgn(p, rationalMap)`。

### 2. signAtOneAlgebraic

现有逻辑基本正确，需确保 zero detection 走 gcd：
```cpp
UniPolyId g = specializeToUnivariate(p, rationalPrefix, algVar);
return signUnivariateAtAlgebraic(g, sample.values[algIdx].root);
```

`signUnivariateAtAlgebraic` 流程：
1. `isConstantUni(g)` → 非零常数符号；
2. `d = gcdUni(alpha.definingPoly, g)`；
3. `!isConstantUni(d) && rootBelongsTo(alpha, d)` → `Sign::Zero`；
4. 否则 interval endpoint eval + refine 直到符号恒定。

### 3. signAtAlgebraicTower（P3 前）

**P3 前返回 Unknown**。不要直接调用 libpoly evaluate 结果决定 Unsat。

P3 实现后替换为：
```cpp
auto reduced = reduceModuloTower(p, s);
if (reduced.isZero()) return Sign::Zero;
if (reduced.isUnknown()) return Sign::Unknown;
return certifiedIntervalSign(reduced.poly, s);
```

## P3：Algebraic Tower Reduction

### 目标

支持如 `x²−2=0, y−x=0, y−x≠0` 的 exact zero 判定。

### reduceModuloTower

```cpp
struct ReductionResult {
    enum class Status { Zero, NonZero, Unknown } status;
    PolyId poly = NullPoly;   // valid if NonZero
};

ReductionResult reduceModuloTower(PolyId p, const SamplePoint& s);
```

算法（从最高层向最低层）：
```cpp
PolyId q = p;
for (int i = s.numVars() - 1; i >= 0; --i) {
    const std::string& x = s.varOrder[i];
    const RealAlg& a = s.values[i];

    if (a.isRational()) {
        q = substituteRational(q, x, a.rational);
        continue;
    }

    // Algebraic: need lifted defining polynomial
    const auto& root = a.root;
    PolyId f = chooseDefiningPoly(root, i);
    if (f == NullPoly) return {Unknown, NullPoly};

    // Nullification check
    PolyId lc = leadingCoefficient(f, x);
    Sign lcSign = signAt(lc, s.prefix(i));
    if (lcSign == Sign::Zero || lcSign == Sign::Unknown)
        return {Unknown, NullPoly};

    q = pseudoRemainder(q, f, x);
    // primitivePart / normalize if needed
}

if (kernel_->isZero(q)) return {Zero, NullPoly};
return {NonZero, q};
```

### chooseDefiningPoly

从 `root.origins` 中选择 `level == targetLevel` 且 `mainVar == varOrder[targetLevel]` 的 origin。如果多个，取第一个（后续可扩展为选当前 cell guard 中有效的）。

### pseudoRemainder

需要 `PolynomialKernel` 或 `LibPolyKernel` 暴露多变量多项式除法/伪余式操作。
libpoly C API 有 `lp_polynomial_pdivrem` / `lp_polynomial_spdivrem`。
需要在 `LibPolyKernel` 中包装这些操作。

### certifiedIntervalSign

如果 reduction 后得到非零多项式 `q`：
1. 对 `q` 做 interval evaluation；
2. interval > 0 → Pos, interval < 0 → Neg；
3. interval 跨 0 → refine 相关 algebraic intervals；
4. 超预算 → Unknown。

**关键**：interval sign 只用于证明非零正负；exact zero 必须靠 reduction。

## P4：Algebraic-Prefix Root Isolation

### 目标

支持 `x²−2=0, y²−x=0`（prefix 含代数数）。

### 接口

```cpp
struct RootIsolationResult {
    enum class Status { Success, Unknown, BackendFailure } status;
    RootSet roots;
};

// AlgebraBackend
virtual RootIsolationResult isolateRealRoots(
    PolyId p, const SamplePoint& prefix, const std::string& mainVar) = 0;
```

逐步废弃只接受 `UniPolyId` 的旧接口，或保留为 rational-prefix 快路径。

### LibpolyBackend 实现

若 libpoly 支持 `isolate_real_roots(Polynomial, Assignment)`：
1. 用 prefix 构造 `poly::Assignment`（rational → Value(Rational), algebraic → reconstruct AlgebraicNumber）；
2. 调用 `poly::isolate_real_roots(libKernel_->get(p), assignment)`；
3. 转换结果 `Value` → `RealAlg`：
   - rational: `fromRational`
   - algebraic: extract defining UPolynomial, convert to UniPolyId, extract interval, find rootIndex
4. Validate：每个 root 代回 signAt 为 Zero；intervals 不重叠；可排序。
5. 任何异常 → `Unknown`。

### solveLevel 调用

```cpp
UniPolyId up = algebra_->specializeToUnivariate(p, prefix, var);
RootSet roots;
if (up != NullUniPolyId) {
    roots = algebra_->isolateRealRoots(up);
} else if (prefix.hasAlgebraic()) {
    auto res = algebra_->isolateRealRoots(p, prefix, var);
    if (res.status != Success) return CdcacResult::mkUnknown(...);
    roots = std::move(res.roots);
} else {
    continue; // skip
}
```

### Nullification Contract

在 projection / root isolation 中：
- 若某个 guard polynomial 在当前 prefix 下恒为 0；
- 且 projection mode 不能证明 delineability；
- 则当前 branch 不能返回 Unsat（返回 Unknown 或切换 projection mode）。

## 必须新增测试

### P2b
1. `sqrt(2)` vs `sqrt(3)`，初始 interval 都设 `[1,2]`：compare 不能 Equal，refine 后 Less；
2. `x²−2` 的 `sqrt(2)` vs `(x²−2)(x+1)` 的 `sqrt(2)`：compare 必须 Equal（gcd 路径）；
3. rational `3/2` vs `sqrt(2)`，interval `[1,2]`：compare 必须 Greater；
4. mergeRoots 遇 Unknown：返回 Unknown，不构造 cells；
5. coversAllLine 双开 sector：sector(-inf,r) + sector(r,+inf) → DoesNotCover。

### P3
1. `x²−2=0, y−x=0, y−x≠0` → conflict；
2. `x²−2=0, y−x=0, y²−2=0` → sat；
3. `x²−2=0, y+x=0, y²−2=0` → sat；
4. `x²−2=0, y−x=0, y>1` → sat；
5. `x²−2=0, y−x=0, y>2` → conflict。

### P4
1. `x²−2=0, y²−x=0` → sat；
2. `x²−2=0, y²−x=0, y=0` → conflict；
3. `x²−2=0, y²−x<0` → conflict；
4. `x²−2=0, y²−x≥0` → sat。
