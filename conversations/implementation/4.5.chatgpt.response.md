这版**仍然不能 Approve**。它已经修掉了很多架构问题，但现在还残留一个**明确的 LIA disequality soundness bug**，以及一个 default registration 冲突。

最严重的是 Step 6 这里：

```cpp
LinearFormKey lhs = form.withoutConstant(); // terms only
auto lit1 = registry_.getOrCreateLinearBoundAtom(lhs, LEQ, -g);
auto lit2 = registry_.getOrCreateLinearBoundAtom(lhs, GEQ,  g);
```

如果 `form` 表示的是：

```text
p(x) = a·x + c
```

那么 `p <= -g` 应该注册成：

```text
a·x + c <= -g
```

也就是：

```text
a·x <= -g - c
```

而不是：

```text
a·x <= -g
```

同理：

```text
p >= g
```

应该是：

```text
a·x >= g - c
```

所以现在这段会生成**错误 lemma**。文件里已经写了 `form = a·x + c`，但生成 bound 时直接用了 `form.withoutConstant()` 和 `±g`，常数项被丢掉了，这是正确性问题。

举个反例：

```text
p(x) = 2x + 2
p(x) != 0
```

这里：

```text
g = 2, c = 2
```

正确 split 是：

```text
2x + 2 <= -2  ∨  2x + 2 >= 2
```

等价于：

```text
2x <= -4  ∨  2x >= 0
```

也就是：

```text
x <= -2  ∨  x >= 0
```

但当前 plan 会生成：

```text
2x <= -2  ∨  2x >= 2
```

也就是：

```text
x <= -1  ∨  x >= 1
```

这会排除 `x = 0`。然而 `x=0` 时 `p=2`，确实满足 `p != 0`。所以这是 **unsound lemma**。

---

应该改成：

```cpp
// p(x) = a·x + c
LinearFormKey lhs = form.withoutConstant(); // a·x only

mpq_class leRhs = -g - c;  // a·x <= -g - c
mpq_class geRhs =  g - c;  // a·x >=  g - c

auto lit1 = registry_.getOrCreateLinearBoundAtom(lhs, LEQ, leRhs);
auto lit2 = registry_.getOrCreateLinearBoundAtom(lhs, GEQ, geRhs);

return {
    DiseqResult::Kind::Lemma,
    std::nullopt,
    TheoryLemma{{d.lit.negated(), lit1, lit2}}
};
```

并在注释里写清楚：

```text
p != 0
where p = a·x + c

If g | -c, then the valid integer-strength split is:
    p <= -g ∨ p >= g

When dynamic bound atoms use lhs=a·x and rhs form, this becomes:
    a·x <= -g - c
    a·x >=  g - c
```
