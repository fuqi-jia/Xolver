我建议这版可以 **[1] Approve**，但执行前补一个很小但重要的实现修正：`pendingUnknown_` 不能“任意 backtrack 就清空”。

整体上，这版已经把我上一轮指出的关键 correctness 问题都修掉了：

1. **QF_NRA 下 linear atom 也走 polynomial path**：plan 明确写了 `defaultTheory_ == TheoryId::NRA` 时，所有算术比较 atom，包括线性约束，都注册为 `PolynomialAtomPayload` 并路由到 `TheoryId::NRA`，避免 linear payload 在 NRA 中 orphan。

2. **`PolynomialKernel` 共享问题修正了**：`Atomizer` 只借用 raw pointer，`CdcacSolver` 通过 `unique_ptr` 持有同一个 kernel；plan 也明确 `PolynomialId` 是 kernel-local，不能跨 kernel 实例。

3. **非整数有理常数处理是 sound 的**：Phase NRA-1 暂时拒绝非整数 rational constant，返回 `Unknown`，避免 `LibPolyKernel::mkConst` 丢分母导致 unsound。

4. **payload mismatch 不再伪造 conflict**：现在 mismatch 进入 `pendingUnknown_`，返回 `Unknown`，而不是学习 `¬reason` 这种不合法 theory conflict。

5. **测试补得比较完整**：包括 constant sat/unsat、non-constant unknown、false literal negation、`distinct 1 1`、`not (= 1 1)`、非整数 rational constant unknown。

---

唯一需要改的小点是这里：

```cpp
void CdcacSolver::backtrackToLevel(int level) {
    while (!trail_.empty() && trail_.back().level > level) {
        active_.resize(trail_.back().activeSizeBefore);
        trail_.pop_back();
    }
    if (pendingConflict_ && pendingConflict_->level > level) {
        pendingConflict_.reset();
    }
    pendingUnknown_ = false;  // reset on any backtrack
}
```

`pendingUnknown_ = false` 不能在任意 backtrack 时无条件清掉。假设某个 payload mismatch 或 unsupported 状态来自 level 1，而 SAT 只从 level 3 回溯到 level 2，这个 unsupported assignment 仍然活跃，此时清掉 `pendingUnknown_` 就可能让 solver 忽略一个 active unsupported theory atom，进而错误返回 SAT。

建议改成 level-tagged：

```cpp
struct PendingUnknown {
    int level;
};

std::optional<PendingUnknown> pendingUnknown_;
```

`assertLit()` 中：

```cpp
if (!payload) {
    pendingUnknown_ = PendingUnknown{level};
    return;
}
```

`check()` 中：

```cpp
if (pendingUnknown_) {
    return TheoryCheckResult::unknown();
}
```

`backtrackToLevel()` 中：

```cpp
if (pendingUnknown_ && pendingUnknown_->level > level) {
    pendingUnknown_.reset();
}
```

或者更保守地，把 payload mismatch 视为 internal unsupported state，在当前 `check-sat` 生命周期内不清除，直接一直 `Unknown`。Phase NRA-1 里我更推荐 level-tagged，因为和 `pendingConflict_` 语义一致。

给 Kimi 的最终确认可以这样写：

```text
Approve with one required implementation fix:

pendingUnknown_ must be level-tagged, like pendingConflict_.
Do not clear pendingUnknown_ on every backtrack.

Use:

struct PendingUnknown { int level; };
std::optional<PendingUnknown> pendingUnknown_;

Set pendingUnknown_ = PendingUnknown{level} on payload mismatch / internal unsupported state.
In backtrackToLevel(target), clear it only if pendingUnknown_->level > target.
In check(), if pendingUnknown_ is set, return Unknown.

After this fix, proceed with the plan.
```

结论：**选 [1] Approve**，但把上面这个 `pendingUnknown_` level-tag 修正作为执行时的硬要求。
