下面给你一版**最终版 NRA 约束求解算法设计**。我按你现在已经确定的统一架构来写：

```text
CaDiCaL ExternalPropagator
  -> TheoryAtomRegistry
  -> TheoryManager
  -> CdcacSolver
  -> PolynomialKernel / CDCAC engine / LocalSearch engine
```

目标是：**QF_NRA 多项式实数约束求解，正确性优先，local search 可以有，但只能作为找模型的加速器，不能作为 unsat 证明。**

---

# 1. 支持范围与语义

第一版 NRA 只支持**量词自由非线性实数算术**：

```smt2
(set-logic QF_NRA)
```

支持 atom：

```text
p(x1,...,xn) = 0
p(x1,...,xn) != 0
p(x1,...,xn) < 0
p(x1,...,xn) <= 0
p(x1,...,xn) > 0
p(x1,...,xn) >= 0
```

其中 `p` 是有理系数多项式。

不支持的内容直接返回 `Unknown`，不能静默降级：

```text
sin, cos, exp, log 等超越函数；
变量作除数的有理函数；
混合 Int/Real 的 NIRA；
未实现的 UF / Array / String 等跨理论组合；
未能抽取成 polynomial payload 的 theory atom。
```

NRA solver 只处理实数变量。整数变量不应该进入 `CdcacSolver`。如果 future 支持 NIRA，应建立独立 `NiaSolver` 或 `NiraSolver`，不要让 `CdcacSolver` 偷偷把 Int 当 Real。

# 3. TheoryAtomRecord 与 PolynomialAtomPayload

NRA atom 必须在 atomization 阶段注册进 `TheoryAtomRegistry`：

```cpp
struct PolynomialAtomPayload {
    PolynomialId poly;  // normalized p(x)
    Relation rel;       // Eq, Neq, Lt, Leq, Gt, Geq
    mpq_class rhs;      // usually 0 after normalization
};

struct TheoryAtomRecord {
    SatVar satVar;
    TheoryId theory;
    bool isDynamic;
    ExprId exprId;      // diagnostic only
    TheoryAtomPayload payload;
};
```

建议 normalize 成：

```text
p rel 0
```

即：

```cpp
PolynomialAtomPayload {
    poly = lhs - rhs,
    rel = rel,
    rhs = 0
}
```

这样 `CdcacSolver` 内部只需要处理 `p ⋈ 0`。

# 4. False literal 的有效关系

SAT 赋值为 false 时，NRA solver 不能保存原始 relation，而要保存**有效 relation**。

```cpp
Relation effectiveRel(Relation rel, bool value) {
    if (value) return rel;

    switch (rel) {
        case Relation::Eq:  return Relation::Neq;
        case Relation::Neq: return Relation::Eq;
        case Relation::Lt:  return Relation::Geq;
        case Relation::Leq: return Relation::Gt;
        case Relation::Gt:  return Relation::Leq;
        case Relation::Geq: return Relation::Lt;
    }
}
```

例如：

```text
¬(p <= 0)  ==>  p > 0
¬(p = 0)   ==>  p != 0
¬(p != 0)  ==>  p = 0
```

这个必须是 `assertLit()` 的第一步。

# 5. CdcacSolver 的增量状态

```cpp
struct ActivePolynomialConstraint {
    PolynomialId poly;
    Relation rel;     // effective relation
    SatLit reason;    // assigned SAT literal
};

struct CdcacTrailEntry {
    int level;
    size_t activeSizeBefore;
};

class CdcacSolver : public TheorySolver {
private:
    PolynomialKernel& kernel_;

    std::vector<ActivePolynomialConstraint> active_;
    std::vector<CdcacTrailEntry> trail_;

    std::optional<TheoryConflict> pendingConflict_;

    CdcacEngine engine_;
    NraLocalSearch localSearch_;
    NraModel currentModel_;
};
```

`assertLit()`：

```cpp
void CdcacSolver::assertLit(
    const TheoryAtomRecord& atom,
    bool value,
    int level,
    SatLit reason
) {
    const auto* payload =
        std::get_if<PolynomialAtomPayload>(&atom.payload);

    if (!payload) {
        pendingConflict_ = unsupportedTheoryPayloadConflictOrUnknown(...);
        return;
    }

    size_t oldSize = active_.size();

    Relation rel = effectiveRel(payload->rel, value);

    active_.push_back({
        .poly = payload->poly,
        .rel = rel,
        .reason = reason
    });

    trail_.push_back({
        .level = level,
        .activeSizeBefore = oldSize
    });
}
```

`backtrackToLevel()`：

```cpp
void CdcacSolver::backtrackToLevel(int level) {
    while (!trail_.empty() && trail_.back().level > level) {
        active_.resize(trail_.back().activeSizeBefore);
        trail_.pop_back();
    }

    if (pendingConflict_ && pendingConflict_->level > level) {
        pendingConflict_.reset();
    }

    engine_.backtrackToLevel(level);
}
```

# 6. PolynomialKernel 设计

NRA 的核心不是 Boolean search，而是多项式内核。不要把多项式直接写成裸 `std::map<monomial, mpq_class>` 到处传。

建议建立：

```cpp
class PolynomialKernel {
public:
    PolynomialId makePolynomial(const Polynomial& p);
    const Polynomial& get(PolynomialId id) const;

    PolynomialId add(PolynomialId a, PolynomialId b);
    PolynomialId sub(PolynomialId a, PolynomialId b);
    PolynomialId mul(PolynomialId a, PolynomialId b);
    PolynomialId pow(PolynomialId a, int k);

    PolynomialId primitivePart(PolynomialId p);
    PolynomialId squareFreePart(PolynomialId p);
    PolynomialId derivative(PolynomialId p, VarId x);

    PolynomialId resultant(PolynomialId f, PolynomialId g, VarId x);
    PolynomialId discriminant(PolynomialId f, VarId x);
    std::vector<PolynomialId> subresultantCoefficients(
        PolynomialId f,
        PolynomialId g,
        VarId x
    );

    Sign signAt(
        PolynomialId p,
        const AlgebraicPoint& sample
    );

    RootIsolationResult isolateRealRoots(
        PolynomialId p,
        VarId mainVar,
        const PartialAlgebraicPoint& prefix
    );
};
```

多项式必须 canonicalize：

```text
1. 合并同类项；
2. 去掉 0 系数；
3. monomial 按固定变量序排序；
4. 有理数系数规范化；
5. content / primitive part 可缓存；
6. PolynomialId interning，避免重复对象。
```


# 8. 总体 check 流程

`CdcacSolver::check()`：

```cpp
TheoryCheckResult CdcacSolver::check(
    TheoryLemmaDatabase& lemmaDb
) {
    if (pendingConflict_) {
        return TheoryCheckResult::mkConflict(*pendingConflict_);
    }

    if (active_.empty()) {
        return TheoryCheckResult::consistent();
    }

    auto normalized = normalizeActiveConstraints(active_);

    auto trivial = checkTrivialConstants(normalized);
    if (trivial.isConflict()) {
        return mkConflict(trivial.conflict);
    }

    // 1. Incomplete but sound SAT acceleration
    auto local = localSearch_.tryFindModel(normalized);
    if (local.hasModel()) {
        if (validateModelExact(local.model, normalized)) {
            currentModel_ = local.model;
            return TheoryCheckResult::consistent();
        }
    }

    // 2. Complete core
    auto result = engine_.check(normalized);

    if (result.kind == CdcacResult::Sat) {
        if (!validateModelExact(result.model, normalized)) {
            return TheoryCheckResult::unknown();
        }
        currentModel_ = result.model;
        return TheoryCheckResult::consistent();
    }

    if (result.kind == CdcacResult::Unsat) {
        TheoryConflict c = buildConflictClause(result.coreReasons);
        return TheoryCheckResult::mkConflict(c);
    }

    return TheoryCheckResult::unknown();
}
```

这里 `localSearch_` 只能找 SAT model。它不能证明 UNSAT。

# 9. Local Search 在 NRA 中的位置

local search 必须是：

```text
可选加速器；
只用于找 candidate model；
candidate 必须 exact validation；
失败没有任何结论。
```

推荐目标函数：

对每个约束 `p ⋈ 0` 定义 violation：

```text
p <= 0: max(0, p)
p <  0: max(0, p + margin)
p >= 0: max(0, -p)
p >  0: max(0, -p + margin)
p =  0: |p|
p != 0: max(0, ε - |p|)
```

总目标：

```text
F(x) = Σ weight_i * violation_i(x)^2
```

算法：

```text
1. 多随机初值；
2. 坐标下降 / quasi-Newton / gradient-free search；
3. interval box shrinking；
4. 找到 near-zero violation 后 rationalize；
5. exact sign validation；
6. validation 成功才返回 SAT。
```

本地搜索输出：

```cpp
struct LocalSearchResult {
    bool found;
    AlgebraicPoint candidate;
};
```

禁止：

```text
local search 失败 -> unsat；
floating result 未验证 -> sat。
```

# 10. 完备核心：CDCAC / CAC

推荐 NRA 完备核心用 **CDCAC：Conflict-Driven Cylindrical Algebraic Covering**。

它比 full CAD 更适合 SMT，因为它不是先构造全空间 CAD，而是对当前 SAT assignment 的 conjunction 做 model-guided search，并在失败处构造覆盖和 conflict。

---

# 11. CDCAC 的基本对象

变量序：

```text
x1 ≺ x2 ≺ ... ≺ xn
```

固定后不能在一次 check 中随便改变。

Cell：

```cpp
enum class CellKind {
    Sector,   // interval between roots
    Section   // exactly a root

struct Cell {

    CellKind kind;

    AlgebraicNumber lower;

    // for section

    int rootIndex;

};
```

Cover：

```cpp
struct Cover {
    int level;
    std::vector<Cell> cells;
    std::vector<SatLit> reasons;

```

CDCAC result：

```cpp
struct CdcacResult {
    enum class Kind { Sat, Unsat, Unknown } kind;

```

# 12. CDCAC 主递归

核心递归：

```cpp
CdcacResult CdcacEngine::check(
    const std::vector<ActivePolynomialConstraint>& constraints
) {
    ProjectionContext ctx(variableOrder_, constraints);
    PartialSample sample;

    auto r = searchLevel(0, sample, ctx);

    if (r.kind == SearchResult::Model) {
        return CdcacResult::sat(r.model);
    }

    if (r.kind == SearchResult::Covering) {
        return CdcacResult::unsat(r.cover.reasons);
    }

    return CdcacResult::unknown();
}
```

递归层：

```cpp
SearchResult searchLevel(
    int k,
    PartialSample& sample,
    ProjectionContext& ctx
) {
    if (k == numVars_) {
        return checkFullSample(sample, ctx.constraints);
    }

    LineCover cover;

    while (!cover.coversRealLine()) {
        AlgebraicNumber a = chooseSampleOutsideCover(k, sample, cover, ctx);

        sample.push(a);

        auto child = searchLevel(k + 1, sample, ctx);

        sample.pop();

        if (child.kind == Model) {
            return child;
        }

        // child returns an infeasible cell over x_k
        Cell blocked = liftChildCoverToCurrentLevel(child.cover, k, sample, ctx);

        cover.add(blocked);
    }

    return SearchResult::covering(cover.toCover());
}
```

直观含义：

```text
在第 k 层，尝试给 x_k 选一个不被已知 infeasible cells 覆盖的样本；
向下递归；
如果下面失败，就把失败原因提升为 x_k 上的一个 cell；
不断加入 cell；
如果 x_k 的整条实线都被 infeasible cells 覆盖，则当前 prefix 不可扩展。
```

# 13. Full sample 检查

当所有变量都有 sample：

```cpp
SearchResult checkFullSample(
    const AlgebraicPoint& sample,
    const std::vector<ActivePolynomialConstraint>& constraints
) {
    for (const auto& c : constraints) {
        Sign s = kernel_.signAt(c.poly, sample);

        if (!relationSatisfied(s, c.rel)) {
            Cell cell = explainViolationAtSample(c, sample);
            return SearchResult::covering(cell);
        }
    }

    return SearchResult::model(sample);
}
```

关系判断：

```cpp
bool relationSatisfied(Sign s, Relation rel) {
    switch (rel) {
        case Eq:  return s == Zero;
        case Neq: return s != Zero;
        case Lt:  return s == Negative;
        case Leq: return s == Negative || s == Zero;
        case Gt:  return s == Positive;
        case Geq: return s == Positive || s == Zero;
    }
}
```

# 14. 违反约束时如何构造 cell

假设在完整 sample `α = (α1,...,αn)` 上，约束 `p rel 0` 被违反。

需要构造一个包含当前 sample 的 cell，使得在该 cell 内此约束仍然违反。

对最后一个变量 `x_n`：

1. 固定 prefix：

```text
α1,...,α_{n-1}
```

2. 将 `p` 特化为 univariate polynomial：

```text
q(x_n) = p(α1,...,α_{n-1}, x_n)
```

3. 隔离 q 的所有实根。
4. 找到包含 `α_n` 的 sector 或 section。
5. 在该 cell 内 q 的 sign 不变，因此该约束 truth value 不变。

例如：

```text
p(α) > 0，但约束 p <= 0
```

则选择包含 `α_n` 的 sign-invariant cell，整个 cell 内 `p > 0`，因此 `p <= 0` 恒假。

对于 `p != 0` 被违反：

```text
p(α) = 0
```

则 cell 是 root section：

```text
x_n = root_i(q)
```

在这个 section 上 `p = 0`，因此 `p != 0` 恒假。

对于 `p = 0` 被违反：

```text
p(α) != 0
```

则选择 sector，保证 `p` 不为 0，因此 equality 恒假。

# 15. Projection / lifting 的正确性

为了把子层的 infeasible cell 提升到父层，必须保证 root 结构在父 cell 上稳定。需要 projection polynomials。

Projection set 至少包含：

```text
1. polynomial leading coefficients；
2. discriminants；
3. resultants between relevant polynomials；
4. subresultant coefficients；
5. squarefree factors。
```

建议实现一个统一的 projection engine：

```cpp
class ProjectionEngine {
public:
    ProjectionSet projectionFor(
        const std::vector<PolynomialId>& polys,
        VarId eliminateVar
    );

    ProjectionSet projectionForCellExplanation(
        const Cell& cell,
        const std::vector<PolynomialId>& involvedPolys,
        VarId eliminateVar
    );
};
```

每个 projection polynomial 都要带 provenance：

```cpp
struct ProjectedPolynomial {
    PolynomialId poly;
    std::vector<SatLit> reasons;
};
```

原因是最终 conflict clause 只能引用 SAT literals，不能引用内部 projection polynomial。

# 16. Conflict clause 构造

CDCAC 证明当前 active constraints 不可满足后，会得到一个 cover，其每个 cell 都带 reasons。

最终 conflict clause：

```cpp
TheoryConflict buildConflictClause(
    const std::vector<SatLit>& coreReasons
) {
    TheoryConflict c;

    for (SatLit r : minimize(coreReasons)) {
        c.clause.push_back(r.negated());
    }

    return c;
}
```

形式：

```text
¬l1 ∨ ¬l2 ∨ ... ∨ ¬lk
```

其中 `l1,...,lk` 是当前 SAT assignment 中导致 NRA 不可满足的 theory literals。

正确性条件：

```text
如果 l1 ∧ ... ∧ lk 在 NRA 中不可满足，
那么 clause ¬l1 ∨ ... ∨ ¬lk 是 theory-valid。
```

必须保证每个 cell 的 reason 都来自 active constraints 或由这些 constraints 投影得到的 provenance。

# 17. Conflict core 最小化

初始 core 可以很大。可以做 deletion-based minimization：

```cpp
std::vector<SatLit> minimizeCore(
    std::vector<SatLit> core
) {
    for each lit in core:
        trial = core without lit
        if CDCAC_check(trial constraints) is Unsat:
            core = trial
    return core;
}
```

第一版可以限制预算：

```text
maxCoreMinimizeRounds = 32
```

这个优化不影响正确性，只影响学习质量。

# 18. Dynamic polynomial split lemma

CDCAC 能直接处理 `p != 0`，所以第一版不必强制拆 disequality。

但为了 SAT 层更好搜索，可以支持可选 split：

```text
p != 0  ==>  p < 0 ∨ p > 0
```

guarded lemma：

```text
¬(p != 0) ∨ (p < 0) ∨ (p > 0)
```

这需要 registry 支持 dynamic polynomial atom：

```cpp
SatLit getOrCreatePolynomialAtom(
    PolynomialId poly,
    Relation rel,
    mpq_class rhs,
    TheoryId theory
);
```

但这不是 correctness 必需。建议放 Phase 2。

第一版 CDCAC 内部直接处理 `Neq` 更稳。

# 19. SAT 模型返回前必须 exact validation

无论 model 来自 local search 还是 CDCAC，返回 `consistent()` 前必须：

```cpp
bool validateModelExact(
    const AlgebraicPoint& model,
    const std::vector<ActivePolynomialConstraint>& constraints
) {
    for (const auto& c : constraints) {
        Sign s = kernel_.signAt(c.poly, model);
        if (!relationSatisfied(s, c.rel)) {
            return false;
        }
    }
    return true;
}
```

如果 validation 失败：

```text
debug: assert false
release: return Unknown
```

不能返回 SAT。

# 20. Variable order

NRA 的变量序对性能影响巨大，但不影响正确性。

第一版固定变量序：

```text
按变量 id 升序；
或按出现次数 / 最高次数 / 总次数 heuristic。
```

建议 heuristic：

```text
1. degree 高的变量后消元；
2. 出现次数多的变量后消元；
3. objective-independent；
4. tie-break by var id。
```

一旦进入 `check()`，变量序固定，不要中途改变。

# 22. CdcacSolver::check() 的完整结构

最终伪代码：

```cpp
TheoryCheckResult CdcacSolver::check(
    TheoryLemmaDatabase& lemmaDb
) {
    if (pendingConflict_) {
        return TheoryCheckResult::mkConflict(pendingConflict_->conflict);
    }

    if (active_.empty()) {
        return TheoryCheckResult::consistent();
    }

    auto constraints = normalize(active_);

    auto trivial = checkTrivial(constraints);
    if (trivial.conflict) {
        return TheoryCheckResult::mkConflict(*trivial.conflict);
    }

    auto local = localSearch_.tryFindModel(constraints);
    if (local.found) {
        if (validateModelExact(local.model, constraints)) {
            currentModel_ = local.model;
            return TheoryCheckResult::consistent();
        }
    }

    CdcacResult result = engine_.check(constraints);

    if (result.kind == CdcacResult::Sat) {
        if (!validateModelExact(result.model, constraints)) {
            return TheoryCheckResult::unknown();
        }

        currentModel_ = result.model;
        return TheoryCheckResult::consistent();
    }

    if (result.kind == CdcacResult::Unsat) {
        TheoryConflict c = buildConflictClause(result.coreReasons);
        return TheoryCheckResult::mkConflict(c);
    }

    return TheoryCheckResult::unknown();
}
```

# 23. 完备性说明

在以下条件下，`CdcacSolver` 是 QF_NRA 完备的：

```text
1. 输入约束都是有理系数多项式 sign constraints；
2. 使用固定变量序；
3. ProjectionEngine 生成足够的 projection polynomials；
4. root isolation 和 sign evaluation 是 exact；
5. CDCAC search 不设置不完备资源截断；
6. 每个 infeasible cell 的 lifting / projection explanation 正确。
```

则：

```text
如果存在实数解，CDCAC 最终找到一个 algebraic sample；
如果不存在实数解，CDCAC 最终构造覆盖全空间的 infeasible cover；
该 cover 的 reasons 形成 theory-valid conflict clause。
```

如果引入预算：

```text
maxCells
maxProjectionPolys
maxDegree
maxTime
```

超过预算只能返回 `Unknown`。

# 24. 第一阶段实现顺序

建议这样做：

```text
Phase NRA-1: Polynomial payload + CdcacSolver shell
  - PolynomialAtomPayload
  - Atomizer polynomial extraction
  - CdcacSolver assertLit/backtrack/check
  - check 暂时只 exact validate trivial / return Unknown

Phase NRA-2: PolynomialKernel
  - canonical polynomial
  - arithmetic
  - derivative
  - squarefree
  - resultant/discriminant/subresultants
  - rational sign evaluation

Phase NRA-3: Local search SAT acceleration
  - double candidate search
  - rationalization
  - exact validation

Phase NRA-4: Univariate root isolation
  - Sturm sequence
  - isolating intervals
  - sign at rational/algebraic sample

Phase NRA-5: Basic CAD-style recursive search
  - full projection set
  - cell decomposition
  - sample selection
  - complete SAT/UNSAT for small instances

Phase NRA-6: CDCAC optimization
  - lazy projection
  - conflict-driven covering
  - reason/provenance tracking
  - conflict clause minimization

Phase NRA-7: Dynamic polynomial lemmas
  - optional disequality split
  - optional theory propagation
```

```smt2
(declare-const x Real)
(assert (< (* x x) 0))
(check-sat)
```

应为 unsat。

Circle SAT：

```smt2
(set-logic QF_NRA)
(declare-const x Real)
(declare-const y Real)
(assert (= (+ (* x x) (* y y)) 1))
(check-sat)
```

Circle UNSAT：

```smt2
(set-logic QF_NRA)
(declare-const x Real)
(declare-const y Real)
(assert (< (+ (* x x) (* y y)) 0))
(check-sat)
```

Disequality：

```smt2
(set-logic QF_NRA)
(declare-const x Real)
(assert (not (= (* x x) 0)))
(check-sat)
```

SAT, e.g. `x = 1`.

Boolean combination:

```smt2
(set-logic QF_NRA)
(declare-const x Real)
(assert (or (< (* x x) 1) (> x 10)))
(check-sat)
```

由 SAT 层拆 Boolean，NRA 层只看当前 active conjunction。

Conflict learning test：

```smt2
(set-logic QF_NRA)
(declare-const x Real)
(assert (<= (* x x) 0))
(assert (not (= x 0)))
(check-sat)
```

应为 unsat，conflict clause 应包含这两个 active literals 的原因。

---

# 26. 关键禁忌

不要做：

```text
1. double local search 找到近似解就返回 SAT；
2. local search 失败返回 UNSAT；
3. polynomial atom 抽取失败后静默忽略；
4. strict inequality 近似成 non-strict；
5. p != 0 简化成 p > 0；
6. CDCAC conflict 不带 reason；
7. projection polynomial 没有 provenance；
8. CdcacSolver 走旧 scan path；
9. dynamic polynomial atom 使用 fake ExprId 路由；
10. 返回 SAT 前不 exact validate。
```

# 总结

最终 NRA 求解器应该是：

```text
CdcacSolver =
  incremental polynomial constraint manager
  + exact PolynomialKernel
  + incomplete local-search model finder
  + complete CDCAC engine
  + exact model validator
  + conflict-clause generator
```

其中：

```text
local search 只找 SAT；
CDCAC 负责完整 SAT/UNSAT；
PolynomialKernel 保证 exactness；
TheoryAtomRegistry 保证 CDCL(T) 路由；
conflict clause 保证 SAT 学习；
ExternalPropagator 保证统一集成。
```

这样设计出来的 NRA solver 才能和你当前的 LIA/LRA 架构统一，同时满足“正经 solver”的正确性要求。
