## Phase 1：bounded-first NIA 能力闭环（当前阶段）

### 1.1 DomainStore 完善

当前 DomainStore 已实现基础功能（intervals + finite sets + exclusions + reasons）。需要补充：

1. **DomainStore 与 DomainStore 组合时的冲突检测**
   - `x ∈ {1,2,3}` + `x >= 4` → empty domain
   - `x ≠ 0,1,2` + `0 <= x <= 2` → empty domain
   - 多个 bound 组合：`x >= 5` + `x <= 3`

2. **Conflict clause 完整性验证**
   - 确保 `buildEmptyDomainConflict()` 收集所有相关 reason
   - 已部分覆盖（test_domain_store.cpp），但可补充更复杂的组合场景

### 1.2 SquareBoundReasoner（新建组件）

从 `UnivariateIntegerReasoner::handleSquareBound` 的骨架中独立出来，专门处理平方约束：

**支持模式：**

| 约束形式 | 处理 |
|---------|------|
| `x² <= c` | 若 `c < 0` → UNSAT；若 `c >= 0` → `x ∈ [-⌊√c⌋, ⌊√c⌋]` |
| `x² = c` | 若 `c < 0` → UNSAT；若 `c` 非完全平方 → UNSAT；若 `c = r²` → `x ∈ {-r, r}` |
| `x² != c` | 若 `c = r²` → `x ≠ r, x ≠ -r`；否则 tautology |
| `x² >= c` | 暂时 NoChange（是 disjunction，Phase 2 处理） |

**实现要求：**
- 使用 GMP 精确整数运算：`mpz_sqrt`, `mpz_perfect_square_p`
- 只处理单变量、degree=2、首项系数为 1 的形式
- 结果更新到 DomainStore

**测试用例：**
- `x² <= 4` → `x ∈ {-2,-1,0,1,2}`
- `x² <= -1` → UNSAT
- `x² = 49` → `x ∈ {-7, 7}`
- `x² = 50` → UNSAT
- `x² != 4` → `x ≠ 2, x ≠ -2`

### 1.3 BoundedNiaSolver 完善

当前已实现：
- ✅ 触发条件：`domains.allFinite(vars) && totalSize <= 10000`
- ✅ 直接枚举 Cartesian product

- ✅ `UnsatComplete` + coarse conflict

**需要补充：**

2. **枚举冲突优化**
   - 当前 conflict 是所有 active reasons 的否定（coarse 但 sound）
   - Phase 1 保持 coarse，Phase 2+ 再优化

## Phase 2：最小整数 branch lemma

### 2.1 目标

当 domain 不完整、BoundedNiaSolver 不能启动时，让 SAT/CDCL 能继续切整数空间。

### 2.2 规则

只实现全局有效的整数分裂：

```
x <= k  ∨  x >= k+1
```

不实现：
- 有限值枚举分裂（`x=v1 ∨ x=v2 ∨ ...`）— 太复杂，容易 unsound
- 非线性分裂（`x²<=c ∨ x²>=c+1`）— 需要 guarded reasons

### 2.3 变量选择策略

```
1. 已有 finite lower/upper 的变量（分裂区间最小）
2. 当前 domain 区间最大的变量
3. 出现在最多 constraints 中的变量
4. fallback: 第一个变量
```

### 2.4 k 的选择

```
有 lower 和 upper:   k = floor((lower + upper) / 2)
只有 lower:          k = lower
只有 upper:          k = upper - 1
无界:                k = 0
```

### 2.5 动态 atom 注册

Branch lemma 需要新的多项式原子：
- `x <= k`
- `x >= k+1`

必须通过 `TheoryAtomRegistry::getOrCreatePolynomialAtom(...)` 注册为真实 SAT var。

### 2.6 接入 NiaSolver 管道

在 `buildBranchLemma` 中实现，取代当前的 `return std::nullopt`。

## Phase 3：基础 ICP / 区间传播

### 3.1 IntervalZ 数据结构

```cpp
struct IntervalZ {
    mpz_class lo;  // 下界（包含）
    mpz_class hi;  // 上界（包含）
    bool isEmpty() const { return lo > hi; }
};
```

### 3.2 IntervalEvaluator

对多项式在区间上的 over-approx evaluation：

| 操作 | 规则 |
|------|------|
| `add` | `[a,b] + [c,d] = [a+c, b+d]` |
| `sub` | `[a,b] - [c,d] = [a-d, b-c]` |
| `neg` | `-[a,b] = [-b,-a]` |
| `mul` | `[a,b] * [c,d] = [min(ac,ad,bc,bd), max(ac,ad,bc,bd)]` |
| `pow2` | `[a,b]²: 若 0∈[a,b] → [0, max(a²,b²)]; 否则 → [min(a²,b²), max(a²,b²)]` |
| `pow3` | `[a,b]³ = [a³,b³]` |

### 3.3 约束区间状态检查

对 normalized constraint `p rel 0`：

| rel | definitely violated | definitely satisfied |
|-----|---------------------|---------------------|
| `<=` | `interval.lo > 0` | `interval.hi <= 0` |
| `>=` | `interval.hi < 0` | `interval.lo >= 0` |
| `=` | `0 ∉ [lo,hi]` | `[lo,hi] == [0,0]` |

### 3.4 传播规则

**Sum-of-squares：**
- `x² + y² <= c` → 对每个变量：`xi ∈ [-⌊√c⌋, ⌊√c⌋]`

**Product pruning（可选）：**
- `x * y = c` → 若已知 `x ∈ [a,b]`，可推 `y ∈ [ceil(c/b), floor(c/a)]`（需处理零和符号）

### 3.5 接入管道

在 `NiaSolver::check()` 中，AlgebraicIntegerReasoner 之后、EmptyDomain 之前插入 IntervalEvaluator：

```
... → algebraic → intervalICP → emptyDomain → bounded → ...
```

## Phase 4：Interval B&B

### 4.1 目标

当 `allFinite(vars)` 但 `totalSize > threshold` 时，不直接 UnknownBudget，而是递归分裂区间。

### 4.2 算法

```cpp
solveBox(box, depth):
    if depth > maxDepth or budget > maxBudget:
        return UnknownBudget

    status = evaluateConstraintsByInterval(constraints, box)
    if any violated:
        return UnsatComplete(boxConflict)

    if box is singleton:
        if validate(exactAssignment): return Sat
        else: return UnsatComplete

    chooseSplitVar(box)
    left = splitLowerHalf(box, var)
    right = splitUpperHalf(box, var)

    r1 = solveBox(left, depth+1)
    if r1 == Sat: return Sat

    r2 = solveBox(right, depth+1)
    if r2 == Sat: return Sat

    if r1 == UnsatComplete && r2 == UnsatComplete:
        return UnsatComplete
    else:
        return UnknownBudget
```

### 4.3 正确性

- **Sat**: 找到精确满足赋值 → 必须 validate
- **UnsatComplete**: 所有子区间都 closed → sound
- **UnknownBudget**: 预算耗尽 → 不声称 UNSAT

## Phase 5：代数推理增强

等 Phase 1-4 稳定后再做。

### 5.1 多变量 GCD conflict

对 normalized equality `p = 0`：
- 设 `p = q + c`，`q` 无常数项
- `g = gcd(所有 q 的系数)`
- 若 `g ∤ -c` → UNSAT

### 5.2 Factor rules

- `p*q = 0` → lemma: `¬(pq=0) ∨ (p=0) ∨ (q=0)`
- `p*q != 0` → lemmas: `¬(pq≠0) ∨ (p≠0)`, `¬(pq≠0) ∨ (q≠0)`
- 需要 dynamic polynomial atom

### 5.3 Modular reasoning 扩展

- 当前仅支持 1-2 变量的穷举 mod 检查
- 扩展到 `evaluateMod` 通用接口
- 支持更多模数

## Phase 6：Local Search 只做 SAT finder

当前 NiaLocalSearch 只是尝试几个固定候选点。增强：

1. **初始点选择**: domain midpoint, small integers, univariate roots, random
2. **Objective**: `Σ violation²`
3. **Move**: 单变量小步调整（±1）
4. **Restart**: 随机重启避免局部最优
5. **Budget**: 固定迭代次数
6. **任何找到的 model 必须 exact-validate**
7. **失败是 NoChange/Unknown，绝不 UNSAT**

---

## 正确性红线（不可违反）

1. **No incomplete reasoning may return UNSAT.**
   - UnsatComplete 只能来自：constant contradiction, empty roots, modular contradiction, GCD contradiction, finite-domain exhaustion, interval B&B complete closure.
2. **Stub polynomial backend returns nullopt, never fake values.**
3. **Domain conflicts must include complete reasons.**
4. **Dynamic atoms must be real registered atoms.**
   - `getOrCreatePolynomialAtom` 必须创建真正的 SAT var，route 回 `NiaSolver::assertLit`。
5. **Every SAT result must exact validate.**
   - `IntegerModelValidator::validate()` 必须对所有 normalized constraints 通过。
6. **Branch lemma must be globally valid.**
   - `x <= k ∨ x >= k+1` 对所有整数 x 成立，与当前赋值无关。
