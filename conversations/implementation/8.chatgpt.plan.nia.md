下面给你一版 **NIA 约束求解算法计划**，按照你当前最终架构来设计：

```text id="33e8rb"
CaDiCaL ExternalPropagator
  -> TheoryAtomRegistry
  -> TheoryManager
  -> NiaSolver
  -> PolynomialKernel / Relaxation / IntegerReasoner / FiniteSolver / LocalSearch
```

重点原则：

```text id="h63tnn"
1. QF_NIA 一般不可判定，不能承诺完整总能终止。
2. SAT 必须 exact validate。
3. UNSAT 必须来自 sound conflict / sound finite-domain exhaustion / sound relaxation unsat。
4. local search 只能找模型，不能证明 UNSAT。
5. bounded NIA 可以完整求解。
6. unbounded nonlinear integer case 可以返回 Unknown。
```

---

# 0. 先明确理论边界

`QF_NIA` 是量词自由非线性整数算术，本质上包含 existential Diophantine problem。一般情形不可判定。

所以最终 `NiaSolver` 的结果语义必须是：

```text id="q1xjdw"
Sat:
  给出一个整数赋值，exact validation 通过。

Unsat:
  给出 sound reason，说明当前 active NIA conjunction 无整数解。

Unknown:
  算法预算耗尽、超出支持片段、或无法完成证明。
```

绝对不要写成：

```text id="zrpw8u"
NiaSolver 总是 complete。
```

正确说法是：

```text id="w0r1yo"
NiaSolver is sound, complete on selected decidable fragments
such as LIA and bounded finite-domain NIA, and may return Unknown
for general unbounded NIA.
```

# 1. 支持范围

第一版 `NiaSolver` 支持：

```smt2 id="qf2guw"
(set-logic QF_NIA)
```

理论 atom：

```text id="hm9ykx"
p(x1,...,xn) = 0
p(x1,...,xn) != 0
p(x1,...,xn) < 0
p(x1,...,xn) <= 0
p(x1,...,xn) > 0
p(x1,...,xn) >= 0
```

其中：

```text id="sad65h"
p 是有理系数多项式；
x_i 是 Int 变量；
Boolean structure 由 CaDiCaL 处理；
NiaSolver 只看当前 SAT trail 下的 active conjunction。
```

不支持时：

```text id="jrlc0b"
return Unknown
```

不能静默忽略。

暂不支持：

```text id="pmd5rb"
1. Real/Int 混合 NIRA；
2. 除法中变量作分母；
3. 超越函数 sin/cos/exp/log；
4. UF/Array/String 与 NIA 的组合；
5. 非 polynomial expression。
```

# 2. TheoryAtomRegistry payload

新增或复用 polynomial payload，但 theory id 区分 `NRA` 和 `NIA`。

```cpp id="4iexcf"
struct PolynomialAtomPayload {
```

对于 QF_NIA，建议统一 normalize 成：


```cpp id="b5ab7u"

payload.rhs  = 0;
```

```text id="bh8tlm"

Atomizer、NiaSolver 必须共享同一个 PolynomialKernel。
```

# 5. False literal 的 effective relation

`assertLit()` 不能直接存原 relation。必须先按 SAT assignment 得到 effective relation。

```cpp id="s99ysp"
Relation negateRelation(Relation r) {
    switch (r) {
        case Relation::Eq:  return Relation::Neq;
        case Relation::Neq: return Relation::Eq;
        case Relation::Lt:  return Relation::Geq;
        case Relation::Leq: return Relation::Gt;
        case Relation::Gt:  return Relation::Leq;
        case Relation::Geq: return Relation::Lt;
    }
}
```

`assertLit()`：

```cpp id="qywu4h"
void NiaSolver::assertLit(
    const TheoryAtomRecord& atom,
    bool value,
    int level,
    SatLit reason
) {
    size_t oldActive = active_.size();

    const auto* payload =
        std::get_if<PolynomialAtomPayload>(&atom.payload);

    if (!payload) {
        pendingUnknown_ = PendingUnknown{level};
        return;
    }

    Relation rel = value ? payload->rel : negateRelation(payload->rel);

    active_.push_back({
        .poly = payload->poly,
        .rel = rel,
        .reason = reason
    });

    trail_.push_back({
        .level = level,
        .activeSizeBefore = oldActive
    });
}
```

# 6. backtrack

```cpp id="4a8cb3"
void NiaSolver::backtrackToLevel(int level) {
    while (!trail_.empty() && trail_.back().level > level) {
        active_.resize(trail_.back().activeSizeBefore);
        trail_.pop_back();
    }

    if (pendingUnknown_ && pendingUnknown_->level > level) {
        pendingUnknown_.reset();
    }

    if (pendingConflict_ && pendingConflict_->level > level) {
        pendingConflict_.reset();
    }

    relaxation_.backtrackToLevel(level);
}
```

# 7. Normalization：整数多项式规范化

NIA 内部最好全部转成整数系数 polynomial。

输入可能有有理系数：

```text id="o4nb5g"
(1/2)x^2 + (3/2)y - 5 <= 0
```

乘以 positive LCM：

```text id="a537o2"
x^2 + 3y - 10 <= 0
```

因为乘的是正数，relation 不变。

Normalization 规则：

```text id="irrm9x"
p rel 0
  -> clear denominators:
       q = lcm_denominators(p) * p
       q has integer coefficients
       q rel 0

strict integer relation:
  q < 0  <=> q <= -1
  q > 0  <=> q >= 1

non-strict:
  q <= 0, q >= 0 unchanged

equality:
  q = 0 unchanged

disequality:
  q != 0 unchanged
```

所以 `NiaNormalizer` 输出：

```cpp id="k7x7vx"
struct NormalizedNiaConstraint {
    PolynomialId poly;   // integer coefficients
    Relation rel;        // Eq, Neq, Leq, Geq only after strict normalization
    SatLit reason;
};
```

建议把 `<` / `>` 在 NIA 中都转成 `<=` / `>=`，减少后续算法复杂度。

# 8. check() 总流程

最终 `NiaSolver::check()`：

```cpp id="xam6dv"
TheoryCheckResult NiaSolver::check(
    TheoryLemmaDatabase& lemmaDb
) {
    if (pendingUnknown_) {
        return TheoryCheckResult::unknown();
    }

    if (pendingConflict_) {
        return TheoryCheckResult::mkConflict(
            pendingConflict_->conflict);
    }

    if (active_.empty()) {
        return TheoryCheckResult::consistent();
    }

    auto constraints = normalizer_.normalize(active_);
    if (!constraints) {
        return TheoryCheckResult::unknown();
    }

    // 1. Trivial constant reasoning
    auto trivial = checkTrivialConstants(*constraints);
    if (trivial.conflict) {
        return TheoryCheckResult::mkConflict(*trivial.conflict);
    }

    // 2. Exact integer local search for SAT
    auto ls = localSearch_.tryFindModel(*constraints);
    if (ls.found && validateModelExact(ls.model, *constraints)) {
        setModel(ls.model);
        return TheoryCheckResult::consistent();
    }

    // 3. LIA / monomial relaxation
    auto lin = relaxation_.checkLinearRelaxation(*constraints, lemmaDb);
    if (lin.kind != Consistent) return lin;

    // 4. NRA relaxation
    auto nra = relaxation_.checkRealRelaxation(*constraints, lemmaDb);
    if (nra.kind != Consistent) return nra;

    if (nra.hasModel) {
        if (isIntegral(nra.model) &&
            validateModelExact(nra.model.asIntegerModel(), *constraints)) {
            setModel(nra.model.asIntegerModel());
            return TheoryCheckResult::consistent();
        }
    }

    // 5. Modular / gcd / factor reasoning
    auto arith = integerReasoner_.run(*constraints, lemmaDb);
    if (arith.kind != Consistent) return arith;

    // 6. Bounds inference
    auto bnd = inferBoundsAndTighten(*constraints, lemmaDb);
    if (bnd.kind != Consistent) return bnd;

    // 7. Bounded complete solving
    if (allVariablesHaveFiniteBounds(*constraints)) {
        auto fin = boundedSolver_.solve(*constraints, lemmaDb);
        if (fin.kind != Unknown) return fin;
    }

    // 8. Branch split fallback
    auto br = buildBranchLemma(*constraints, lemmaDb);
    if (br.kind != Consistent) return br;

    return TheoryCheckResult::unknown();
}
```

这个顺序可以微调，但原则是：

```text id="fyxfnp"
SAT acceleration before expensive proof；
relaxation UNSAT 可直接 UNSAT；
candidate SAT 必须 exact validate；
bounded domain 可以 complete；
general unbounded nonlinear integer case 可以 Unknown。
```

# 9. Trivial constants

常量约束直接判断。

```cpp id="cbsysq"
std::optional<TheoryConflict>
checkConstantConstraint(
    const NormalizedNiaConstraint& c
) {
    if (!kernel_.isConstant(c.poly)) return std::nullopt;

    mpz_class v = kernel_.constantValue(c.poly);

    bool ok = relationSatisfied(v, c.rel);

    if (!ok) {
        return TheoryConflict{{c.reason.negated()}};
    }

    return std::nullopt;
}
```

例如：

```smt2 id="wdx0tg"

(assert (not (= 1 1))) ; unsat
(assert (distinct 1 1)) ; unsat
```

# 10. SAT exact validation

任何 SAT model 必须 exact validate。

```cpp id="1f9jmy"
bool validateModelExact(
    const IntegerModel& model,
    const std::vector<NormalizedNiaConstraint>& constraints
) {
    for (const auto& c : constraints) {
        mpz_class value = kernel_.evalInteger(c.poly, model);
        if (!relationSatisfied(value, c.rel)) {
            return false;
        }
    }
    return true;
}
```

禁止：

```text id="ch8q02"
double 近似满足 -> SAT
local search near-zero -> SAT
NRA relaxation real model -> SAT
```

都必须 integer exact validation。

# 11. Local search：只用于找 SAT

NIA local search 是非常有用的，但必须严格限定：

```text id="nzcgm5"
local search found model + exact validation -> SAT
local search failed -> no conclusion
```

目标函数：

对于整数赋值 `M`，定义 violation：

```text id="cle7xt"
p <= 0: max(0, p(M))
p >= 0: max(0, -p(M))
p = 0 : |p(M)|
p != 0: 0 if p(M)!=0 else penalty
```

总目标：

```text id="j12vw8"
F(M) = Σ weight_i * violation_i(M)^2
```

搜索策略：

```text id="7701fr"
1. 多随机初值；
2. 使用当前 relaxation model round 后作为初值；
3. 坐标下降；
4. 小邻域搜索；
5. large neighborhood search；
6. conflict-driven variable weighting；
7. 找到 F=0 后 exact validation。
```

变量初值来源：

```text id="a5ybwb"
1. 当前 NRA relaxation model 四舍五入；
2. LIA/monomial relaxation model；
3. 约束中常数附近；
4. 小范围 [-10,10]；
5. 随机 box。
```

# 12. LIA / monomial relaxation

非线性单项式引入 abstract variable：

```text id="s9h4ha"
m = x*y
z_m 代表 m
```

把 polynomial constraint 变成 linear constraint over：

```text id="7dwq6n"
original variables + monomial abstraction variables
```

例如：

```text id="60074g"
x^2 + 2xy - y <= 3
```

变成：

```text id="1w6zfx"
z_x2 + 2 z_xy - y <= 3
```

这是 original NIA 的 over-approximation，因为 `z_x2, z_xy` 暂时是自由整数变量。

性质：

```text id="z606id"
linear relaxation UNSAT => original NIA UNSAT
linear relaxation SAT   => no conclusion
```

如果 relaxation unsat，返回 conflict：

```text id="3m2gqh"
¬l1 ∨ ... ∨ ¬lk
```

其中 `l_i` 是 involved active constraints。

如果变量有 finite bounds，可以加入 McCormick / product envelope：

对于：

```text id="5f9jlx"
z = x*y
x ∈ [lx, ux]
y ∈ [ly, uy]
```

加入 valid inequalities：

```text id="jslt68"
z >= lx*y + ly*x - lx*ly
z >= ux*y + uy*x - ux*uy
z <= lx*y + uy*x - lx*uy
z <= ux*y + ly*x - ux*ly
```

这些是 real valid over box，因此对 integer points 也 valid。

注意：

```text id="ul12e3"
这些 constraints 是 relaxation 内部使用；
如果要作为 theory lemma 返回，必须能表达成原始变量上的 valid polynomial/linear atom，
并通过 TheoryAtomRegistry 注册。
```

第一版建议只内部使用。

# 13. NRA relaxation

把所有 Int 变量当 Real，交给 `CdcacSolver` 或内部 `NraRelaxationEngine`。

性质：

```text id="tix4pr"
NRA relaxation UNSAT => NIA UNSAT
NRA relaxation SAT   => no conclusion unless model is integer and exact validation passes
```

例如：

```smt2 id="5973qo"
(assert (= (+ (* x x) 1) 0))
```

NRA UNSAT，所以 NIA UNSAT。

如果 NRA 得到 model：

```text id="y2e4u0"
x = 3/2
```

则不能 SAT。可以用它指导 branch：

```text id="ca0diw"
x <= 1 ∨ x >= 2
```

# 14. Branch split

对于 integer variable `x` 和 rational value `q`：

```text id="y8jgga"
x <= floor(q) ∨ x >= ceil(q)
```

如果 `q` 非整数，这是全局 valid NIA lemma。

如果 `q` 是整数，可以选择邻近切分：

```text id="wmmo0i"
x <= q ∨ x >= q+1
```

或者根据当前 bounds split midpoint。

生成 dynamic atoms：

```cpp id="gkgw5m"
SatLit left = registry.getOrCreatePolynomialAtom(
    poly = x - floor(q),
    rel = Leq,
    rhs = 0,
    theory = TheoryId::NIA
);

SatLit right = registry.getOrCreatePolynomialAtom(
    poly = x - ceil(q),
    rel = Geq,
    rhs = 0,
    theory = TheoryId::NIA
);

TheoryLemma{{left, right}};
```

对于 integer tautology，不需要 premise。

去重：

```cpp id="29s7jm"
if (lemmaDb.contains(lemma)) skip;
```

# 15. Modular reasoning

这是 NIA 非常重要的 sound UNSAT 方法。

## 15.1 Equality modulo m

对于 active equality：

```text id="p68anx"
p(x) = 0
```

必要条件：

```text id="qtfjme"
存在 residue assignment r_i ∈ Z_m，使 p(r) ≡ 0 mod m
```

如果对某个 modulus `m` 没有 residue solution，则 UNSAT。

例如：

```text id="mt652z"
x^2 = 2
```

mod 4 下平方只可能是 0 或 1，所以 `x^2 ≡ 2 mod 4` 无解，UNSAT。

算法：

```cpp id="trzux0"
ModularResult checkModulo(
    const std::vector<NormalizedNiaConstraint>& constraints,
    int modulus
) {
    // enumerate residues for selected variables
    // or use constraint propagation over residues
}
```

不能直接枚举所有变量的大 modulus。第一版：

```text id="6t37wv"
m ∈ {2,3,4,5,7,8,9,11}
变量数小于阈值时枚举 residues；
否则用局部 propagation / skip。
```

若发现 contradiction：

```cpp id="fz23yp"
return TheoryCheckResult::mkConflict(
    conflictFromReasons(involvedConstraints)
);
```

## 15.2 Disequality modulo

对于：

```text id="fy89s7"
p != 0
```

modular reasoning 较弱。可以推出：

```text id="4zohe0"
如果 p mod m 恒为 0，不代表 p=0；
所以不能用来证明 p!=0 false。
```

但可以辅助 local search 和 branching，不作为 unsat 证明，除非有额外 bounds。

# 16. GCD / divisibility reasoning

对于 linear case：

```text id="bbl04s"
a1*x1 + ... + an*xn = c
```

如果：

```text id="d8glw4"
g = gcd(a_i)
g ∤ c
```

则 UNSAT。

对于 polynomial equality：

```text id="f9puuv"
Σ c_i * monomial_i = c
```

如果所有 monomial_i 在整数上都是整数，左边属于 gcd(coefficients) 的倍数。

若：

```text id="97q63c"
g = gcd(c_i)
g ∤ c
```

则 UNSAT。

例子：

```text id="gjnl3w"
2*x^2 + 4*y*z = 3
```

左边永远偶数，右边奇数，UNSAT。

实现：

```cpp id="4xw28e"
std::optional<TheoryConflict>
checkCoefficientGcdEquality(
    const NormalizedNiaConstraint& c
);
```

只对 equality 使用。对 inequalities 要谨慎，可做 tightening：

```text id="dmv0yf"
g | p(x)
p <= c  =>  p <= g * floor(c/g)
p >= c  =>  p >= g * ceil(c/g)
```

这个可以生成 guarded lemma。

# 17. Factor reasoning

整数环是 integral domain。

## 17.1 Product equality

```text id="hwauol"
p*q = 0  =>  p = 0 ∨ q = 0
```

生成 guarded split lemma：

```text id="ew3kk7"
¬(p*q = 0) ∨ (p = 0) ∨ (q = 0)
```

## 17.2 Product disequality

```text id="x96sd2"
p*q != 0  =>  p != 0 ∧ q != 0
```

可生成两个 guarded lemmas：

```text id="mbsc13"
¬(p*q != 0) ∨ (p != 0)
¬(p*q != 0) ∨ (q != 0)
```

## 17.3 Square constraints

```text id="l74h7f"
x^2 >= 0    always true
x^2 < 0     unsat
x^2 = 0     x = 0
x^2 != 0    x != 0
```

更一般：

```text id="cirgso"
Σ squares <= 0  => every square = 0
Σ squares < 0   => unsat
```

例如：

```text id="xk4d0g"
x^2 + y^2 <= 0
```

推出：

```text id="ss6mxg"
x = 0 ∧ y = 0
```

可作为 guarded lemmas。

# 18. Bounds inference

很多 NIA 问题能否求解取决于能否得到 finite bounds。

来源：

```text id="mm1cja"
1. active linear constraints；
2. polynomial inequalities；
3. square constraints；
4. domain split lemmas；
5. user assertions；
6. relaxation model。
```

常见推理：

```text id="j5mntv"
x^2 <= c, c >= 0  =>  -floor(sqrt(c)) <= x <= floor(sqrt(c))

x^2 >= c 不能直接给 finite bound。

x*y = 1 over integers:
  x ∈ {1,-1}, y ∈ {1,-1}

x*y = 0:
  x=0 ∨ y=0

x^2 + y^2 <= c:
  |x| <= floor(sqrt(c))
  |y| <= floor(sqrt(c))
```

生成 bound tightening lemma：

```text id="5e92u7"
¬premise ∨ (x <= k)
¬premise ∨ (x >= k)
```

如果 premise 是 active constraint。

# 19. Bounded complete solving

如果所有变量都有 finite integer bounds：

```text id="ei2bxq"
x_i ∈ [l_i, u_i]
```

则 NIA becomes finite-domain and decidable.

可以用两种 backend。

---

## 19.1 Interval branch-and-bound

Box：

```cpp id="kh3pao"
struct IntBox {
    std::unordered_map<VarId, IntervalZ> domains;
};
```

Interval evaluation：

```cpp id="7k69o5"
IntervalZ evalInterval(PolynomialId p, const IntBox& box);
```

Pruning：

```text id="h38sfi"
p <= 0:
  if lower(p) > 0 -> box infeasible
  if upper(p) <= 0 -> constraint definitely satisfied

p >= 0:
  if upper(p) < 0 -> infeasible
  if lower(p) >= 0 -> definitely satisfied

p = 0:
  if 0 notin interval(p) -> infeasible

p != 0:
  if 0 notin interval(p) -> definitely satisfied
```

Search：

```cpp id="ug42t1"
BoundedResult solveBox(IntBox box) {
    auto prune = intervalPrune(box);
    if (prune.infeasible) return UnsatBox;

    if (box.isSingleton()) {
        IntegerModel m = box.singlePoint();
        if (validateModelExact(m, constraints)) return Sat(m);
        return UnsatBox;
    }

    VarId x = chooseSplitVar(box);
    auto [left, right] = split(box, x);

    solveBox(left);
    solveBox(right);
}
```

若所有 boxes 都 unsat：

```text id="9u2vqh"
active conjunction unsat over finite domain
```

返回 conflict clause：

```text id="42pxlo"
¬l1 ∨ ... ∨ ¬lk
```

用所有 active constraints 的 reasons 即可，虽然不最小但 sound。

---

## 19.2 Bit-blasting finite domain

对于 bounded variable：

```text id="pbxc3d"
x ∈ [l,u]
```

编码：

```text id="x = l + unsigned_bits"
```

位宽：

```text id="5jmho0"
w = ceil(log2(u-l+1))
```

并加 range constraint：

```text id="5t7jby"
bits_value <= u-l
```

多项式求值要 exact，不能 overflow。需要为每个 polynomial 计算值域：

```text id="esx9ne"
range(p, box) = [minPossible, maxPossible] over interval arithmetic
```

选择足够大的 signed bit-width：

```text id="cwcm7k"
W >= max bit length of all intermediate polynomial values + safety margin
```

更稳的是使用 arbitrary precision bit-vector encoding 或 SAT-level arithmetic circuit with exact width.

Bit-blast result：

```text id="kdutvw"
SAT:
  decode integer assignment；
  exact validate；
  if valid -> SAT；
  else encoding bug -> Unknown/internal error。

UNSAT:
  finite domain exhausted -> theory conflict over active reasons。
```

第一版建议 interval B&B 先做，bit-blasting 后续加。

# 20. Dynamic polynomial atoms

为了生成 NIA lemmas，需要 registry 支持 dynamic polynomial atom：

# 21. Conflict clause

当 NIA 子过程证明当前 active constraints unsat：

```cpp id="4g327k"
TheoryConflict buildConflict(
    std::vector<SatLit> reasons
) {
    TheoryConflict c;
    for (SatLit r : minimize(reasons)) {
        c.clause.push_back(r.negated());
    }
    return c;
}
```

形式：

```text id="uj0gir"
¬l1 ∨ ¬l2 ∨ ... ∨ ¬lk
```

正确性条件：

```text id="pc19zm"
l1 ∧ ... ∧ lk has no integer solution.
```

如果 reason 不完整，不允许返回 conflict。

# 22. Lemma 类型

NIA solver 会产生这些 lemmas：

## 22.1 Branch lemma

```text id="4xnr7g"
x <= k ∨ x >= k+1
```

全局 valid for integer x。

## 22.2 Guarded tightening

```text id="hbng5e"
¬premise ∨ tighter_bound
```

例如：

```text id="xr1hns"
2x <= 5  =>  x <= 2
lemma: ¬(2x <= 5) ∨ (x <= 2)
```

## 22.3 Factor split

```text id="4acxbd"
¬(p*q=0) ∨ (p=0) ∨ (q=0)
```

## 22.4 Modular conflict

```text id="ouzird"
¬l1 ∨ ... ∨ ¬lk
```

## 22.5 Bounded exhaustion conflict

```text id="q3gjsk"
¬l1 ∨ ... ∨ ¬lk
```

# 23. No-op / duplicate guards

每个 lemma 生成前必须检查：

```text id="tp7vxx"
1. lemmaDb.contains(candidate)；
2. tautology：¬p ∨ p；
3. conclusion already active；
4. self-tightening；
5. opposite-literal suspicious case。
```

对于 guarded lemma：

```text id="esxv6w"
¬premise ∨ conclusion
```

如果：

```text id="8o9mzj"
conclusion == premise
```

则是 tautology，跳过。

# 24. NiaSolver check 阶段推荐顺序

最终建议顺序：

```text id="lnjba5"
1. pendingUnknown / pendingConflict
2. normalize active constraints
3. trivial constants
4. exact local search SAT
5. linear subset / monomial LIA relaxation
6. NRA relaxation
7. modular/gcd/factor reasoning
8. bound inference
9. bounded complete solver
10. branch split
11. Unknown
```

为什么不是先 branch？

因为 branch 会增加 SAT 搜索树。更好的策略是：

```text id="tupok1"
先用 cheap sound reasoning 尽量 SAT/UNSAT；
再用 complete finite-domain；
最后才 branch。
```

# 27. 正确性准则

`NiaSolver` 每个结论必须满足：

## SAT

```text id="ubhl9t"
有完整 integer assignment；
所有 active constraints exact eval 通过；
才返回 consistent。
```

## UNSAT

必须来自：

```text id="we2zvp"
1. trivial constant contradiction；
2. LIA relaxation UNSAT；
3. NRA relaxation UNSAT；
4. modular/gcd/factor sound proof；
5. finite-domain complete exhaustion；
6. exact conflict core。
```

## UNKNOWN

出现：

```text id="hb6qg4"
1. unsupported atom；
2. unbounded nonlinear case无法证明；
3. budget exhausted；
4. incomplete backend；
5. validation failure；
6. algebraic relaxation unknown。
```

# 28. 最终总结

`NiaSolver` 不应该是一个“单算法”求解器，而应该是一个组合式 nonlinear integer engine：

```text id="2m4b9z"
NiaSolver =
  exact polynomial normalization
  + exact model validator
  + integer local search SAT finder
  + LIA/monomial relaxation
  + NRA relaxation
  + modular/gcd/factor reasoning
  + bound inference
  + bounded complete solver
  + branch split
```

核心策略：

```text id="c6avw"
SAT:
  local search / bounded solver / relaxation integral model
  -> exact validation

UNSAT:
  relaxation unsat / modular-gcd proof / finite exhaustion
  -> conflict clause

Otherwise:
  branch or Unknown
```

这才是一个 sound 的 QF_NIA solver 计划。它不会假装解决一般不可判定问题，但在工程上可以覆盖大量实际 SMT-LIB NIA 实例。
