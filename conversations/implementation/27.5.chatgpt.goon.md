## 1. 最大 blocker：`definingPoly = NullUniPolyId` 不能接受

你写：

```text
代数根的 definingPoly 设为 NullUniPolyId。根隔离本身正确。
```

这对 **root isolation 本身** 可能暂时没问题，但对 CDCAC 来说不够。

因为后面这些都依赖 `definingPoly`：

```text
compareRealAlg
mergeRoots
signUnivariateAtAlgebraic
rootBelongsTo
gcd zero detection
refineRootInterval
coversAllLine 的边界比较
tower reduction
```

也就是说，`isolateRealRootsAlgebraic` 返回的 root 如果没有 `definingPoly`，它只能作为“临时数值根”使用，不能进入完整 CDCAC cell/reason/coverage 流程。

### 正确处理

不要从 `poly::get_defining_polynomial(an)` 硬拿。如果它会 SIGSEGV，先不要依赖这个 API。

对于 algebraic-prefix root isolation，应该自己保存 defining source：

```cpp
struct AlgebraicRoot {
    UniPolyId definingPoly;        // 若能获得有理系数一元 defining poly
    PolyId liftedDefiningPoly;     // 原始多变量多项式 p
    VarId mainVar;
    int level;
    int rootIndex;
    mpq_class lower;
    mpq_class upper;
    std::vector<RootOrigin> origins;
};
```

P4 返回的 root 即使暂时没有 `UniPolyId definingPoly`，也必须至少有：

```text
liftedDefiningPoly = p
mainVar = 当前变量
level = 当前层
origin = 当前约束多项式
```

然后明确标记：

```cpp
enum class AlgebraicRootKind {
    RationalDefiningPoly,
    AlgebraicPrefixDefiningPoly
};
```

否则不能把它伪装成普通一元有理多项式根。

## 3. `reduceModuloTower` 现在只能算 skeleton

你写：

```text
signAtTower 实现：对 tower 中的每个代数变量，从高层到低层做 pseudoRemainder
```

这个方向对，但现在还不能算 sound，除非已经满足下面条件：

```text
1. 每个 algebraic root 有 liftedDefiningPoly/mainVar/level；
2. chooseLiftedDefiningPoly 能选出当前 prefix 下有效的 defining polynomial；
3. pseudoRemainder 之前检查 degree(f, x) > 0；
4. 检查 leadingCoefficient(f, x) 在 prefix 上非零；
5. pseudoRemainder 的变量 x 确实是 libpoly 当前 top variable，或者 kernel 支持任意变量；
6. remainder 后 primitivePart + normalize；
7. Nullification 时返回 Unknown，而不是继续。
```

如果这些没齐，`reduceModuloTower` 只能叫：

```text
experimental skeleton
```

不能标成 P3 完成。

## 4. `univariateToPoly` 要小心方向和变量

`univariateToPoly` 很容易出错。必须明确：

```text
UniPolyId 系数顺序是 low-to-high 还是 high-to-low；
重建 PolyId 时使用哪个 VarId；
系数是否已经 normalized；
零系数是否删除；
重建后是否 hash-cons/canonicalize。
```

建议加测试：

```text
UniPoly [ -2, 0, 1 ] with var x -> x^2 - 2
UniPoly [ -1, 1 ] with var y -> y - 1
UniPoly [ 0, 0, 1 ] with var z -> z^2
```
