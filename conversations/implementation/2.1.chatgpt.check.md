总体看，这个 plan 的大方向是对的：**tableau 持久化、bounds 增量更新、`check()` 只做 pivot loop、`pop()` 不回退 basis**，这些都符合 Dutertre-de Moura general simplex 的实现思路。文档里也明确把持久结构分成 tableau、bounds、assignment，并强调 `assertLower/assertUpper` 更新 bounds、`check()` 修复 basic-variable violations、backtracking 只恢复 bounds。

但是里面有几个**会导致实现错误或语义不完整**的问题，需要改。

---

# 1. 最大问题：constraint registration 的符号写法不一致

文档写的是：

```text
s = sum a_j x_j - c
```

但又写：

```text
In matrix: 1s + sum(-a_j)*x_j = -c
```

这两个容易混淆。

如果你的 tableau row 语义是：

```cpp
basicVars_[r] = rhs_[r] + sum_j matrix_[r][j] * x_j
```

那么对原子：

```text
a1*x1 + ... + an*xn ~ c
```

定义：

```text
s = a1*x1 + ... + an*xn - c
```

tableau 应该是：

```text
s = -c + a1*x1 + ... + an*xn
```

也就是：

```cpp
rhs[row] = -c;
matrix[row][x1] = a1;
matrix[row][x2] = a2;
...
basicVars[row] = s;
```

不是 `matrix = -a_j`。

如果你采用完整等式形式：

```text
s - a1*x1 - ... - an*xn = -c
```

那才会出现 `-a_j`。但是进入 tableau 后，basic variable `s` 已经被解出来了，所以 row 内部应该存：

```text
s = -c + Σ a_j x_j
```

建议把 plan 中这一段改成：

```text
For atom p(x) ~ c, define s = p(x) - c.
Add tableau row:
    s = -c + Σ a_j x_j
The relation is asserted as a bound on s.
```

然后关系映射为：

```text
p <= c   ⇒   s <= 0
p <  c   ⇒   s <= -δ
p >= c   ⇒   s >= 0
p >  c   ⇒   s >=  δ
p =  c   ⇒   s >= 0 and s <= 0
```

# 2. `Assert BOTH true and false theory literals` 不够精确

文档多次说 TheoryManager 要同时 assert true 和 false theory literals。这个方向是对的，因为 SAT model 中一个 theory atom 为 false 时，theory solver 必须知道它的反约束。

但这里有一个严重细节：**不是所有 false literal 都能转成一个 bound。**

假设 atom 是：

```text
s <= 0
```

那么：

```text
¬(s <= 0)  ⇒  s > 0  ⇒  s >= δ
```

可以转成 bound。

但如果 atom 是：

```text
s = 0
```

那么：

```text
¬(s = 0)  ⇒  s != 0
```

这不是 convex constraint，不能直接用 simplex 的单个 bound 表达。

所以必须分类：

```text
atom: s <= 0
  true  -> upper(s, 0)
  false -> lower(s, δ)

atom: s < 0
  true  -> upper(s, -δ)
  false -> lower(s, 0)

atom: s >= 0
  true  -> lower(s, 0)
  false -> upper(s, -δ)

atom: s > 0
  true  -> lower(s, δ)
  false -> upper(s, 0)

atom: s = 0
  true  -> lower(s,0) + upper(s,0)
  false -> disequality obligation, not a bound

atom: s != 0
  true  -> disequality obligation, not a bound
  false -> lower(s,0) + upper(s,0)
```

`disequality obligation` 的处理方式：

```text
如果当前 model 中 s != 0，没事；
如果当前 model 中 s = 0，需要加 split lemma:
    s < 0 ∨ s > 0
也就是：
    s <= -δ ∨ s >= δ
```

否则你的 LRA solver 会错误处理 `not (= x 0)`、`distinct`、`!=`。

# 3. V1 的 “reset only bounds” 不是完整 incremental DM

文档说 V1 保持当前 CDCL loop，只改 `TheoryManager::check`：

```text
1. NOT reset the theory solver.
2. Reset only bounds.
3. Assert all atoms with their current value.
4. Call check().
```

然后说这就是 correct DM algorithm with minimal CDCL loop changes。

这里要改措辞。这个版本**可以正确**，但它不是完整意义上的 incremental DM；它只是：

```text
tableau / basis incremental
bounds per SAT model rebuilt
```

也就是 **semi-incremental**。

真正的 DM incremental 是 SAT decision level 下：

```text
assertLit(lit)
push decision level
pop decision level
```

bounds 随 SAT trail 增量变化，而不是每个 SAT model 都 `resetBounds()` 后重断言。

所以建议写成：

```text
V1 is tableau-incremental but assignment-level non-incremental:
it preserves tableau and basis, but rebuilds active bounds for each SAT model.
This is acceptable for correctness and simpler integration, but does not fully exploit DM incrementality.
```

否则后面实现和预期性能会对不上。

# 4. `resetBounds()` 后必须明确处理 beta

文档写：

```text
resetBounds(): clear all bounds back to (-inf, +inf)
```

但是如果你 reset 了 bounds，又接着 assert 当前 SAT assignment 的 bounds，必须保证 `beta_` 和 invariant 重新成立。

建议规则：

```cpp
void resetBoundsForModelCheck() {
    lower_ = -inf;
    upper_ = +inf;
    boundReason_.clear();
    betaDirty_ = true;
}
```

下一次 `check()` 前：

```cpp
if (betaDirty_) {
    recomputeBetaFromBasis();
    betaDirty_ = false;
}
```

`recomputeBetaFromBasis()` 不应该随便保持旧值，建议：

```cpp
for each nonbasic x:
    beta[x] = chooseValueWithinBounds(lower[x], upper[x]);
for each basic b:
    beta[b] = rhs[row(b)] + Σ coeff * beta[nonbasic];
```

`chooseValueWithinBounds` 可以是：

```text
0 if 0 in [l,u]
else l if l finite
else u if u finite
else 0
```

否则容易出现：旧 assignment 残留在新的 SAT model 中，导致 non-basic invariant 不稳。

# 5. Bound trail 有 base frame 问题

文档中的 `assertUpper` 直接写：

```cpp
trail_.back().push_back(...)
```

但是如果没有先 `push()`，`trail_` 是空的。

必须保证初始化时有 base frame：

```cpp
GeneralSimplex::GeneralSimplex() {
    trail_.push_back({});
}
```

或者改成：

```cpp
currentTrailFrame().push_back(...)
```

并保证至少有一层。

同时 `resetBounds()` 不应该直接复用 `trail_` 的 push/pop 逻辑，否则会混淆：

```text
user-level push/pop
SAT decision-level push/pop
per-model resetBounds
```

建议分三种状态操作：

```text
reset()                 清空 tableau + atoms + all states
resetActiveBounds()      只清当前 SAT model 的 active bounds
pushUserContext()        用户 push
popUserContext()         用户 pop
pushTheoryLevel()        SAT decision level，未来版本用
popTheoryLevel()         SAT decision level，未来版本用
```

# 6. Conflict explanation 缺少 bound reason

文档写了 lower-bound conflict 的形式：

```text
conflict = { xi >= lower[xi] }
         + { xj <= upper[xj] | xj in N+ }
         + { xj >= lower[xj] | xj in N- }
```

这个数学形式是对的。

但工程上还必须保存每个 bound 的来源：

```cpp
struct Bound {
    BoundKind kind; // finite, -inf, +inf
    DeltaRational value;
    SatLit reasonLit;
    AtomId atom;
};
```

否则你只能得到：

```text
x <= 3
y >= 2
```

却无法翻译成 SAT conflict clause：

```text
¬lit1 ∨ ¬lit2 ∨ ...
```

所以 `lower_` / `upper_` 不能只是值，还要是：

```cpp
struct BoundInfo {
    bool finite;
    DeltaRational value;
    SatLit reason;
};
```

对于 equality：

```text
s = 0
```

它会产生两个 bounds：

```text
s >= 0  reason = lit_eq
s <= 0  reason = lit_eq
```

冲突解释里可能两个都引用同一个 literal。

# 7. `std::optional<DeltaRational>` 不适合表示上下界

文档里 `BoundEntry` 用：

```cpp
std::optional<DeltaRational> oldBound;
```

这能表达 “没有界”，但你后面又写：

```cpp
if (bound >= upper[var]) return OK;
if (bound < lower[var]) return CONFLICT;
```

这里如果 `upper[var]` / `lower[var]` 是 optional，就会很麻烦。

建议直接设计：

```cpp
enum class BoundKind {
    NegInf,
    Finite,
    PosInf
};

struct BoundValue {
    BoundKind kind;
    DeltaRational value;
};
```

比较函数支持：

```text
-inf < finite < +inf
```

这样 `lower_`、`upper_`、trail old value 都统一。

# 8. `pivot()` 的矩阵语义需要写清楚

目前文档有两个语义混在一起：

```text
Row r: basicVars_[r] = rhs_[r] + sum_j matrix_[r][j] * x_j
```

和：

```text
1s + sum(-a_j)*x_j = -c
```

建议只保留 tableau 语义，不要再混用 full equation 语义。

实现上可以用 full dense matrix，但语义必须规定：

```text
matrix_[row][v] 表示 basicVar(row) 对非 basic 变量 v 的系数。
对 basic 变量列，应该视为 0 或不使用。
```

`update(xj,v)` 只允许 `xj` 是 non-basic：

```cpp
assert(!isBasic_[xj]);
```

`pivot(leavingBasic, enteringNonBasic)` 后要保证：

```text
entering becomes basic
leaving becomes non-basic
所有 row 仍然满足：
  basic = rhs + Σ coeff(nonbasic) * nonbasic
```

否则 update 会错。

# 9. 测试用例有一个明显错误

文档测试里写：

```text
push x >= 5 -> check UNSAT -> pop -> check SAT
```

单独 `x >= 5` 不可能 UNSAT。

应该改成：

```text
base: x <= 3
push: x >= 5
check: UNSAT
pop
check: SAT
```

或者：

```text
base: x <= 3
check: SAT
push: x >= 5
check: UNSAT
pop
check: SAT
```

# 10. `Iteration cap with Unknown fallback` 对 LRA 不应该作为正常策略

文档 hardening 里写：

```text
Iteration limit with graceful Unknown fallback
```

作为 debug 防护可以，但不能作为 LRA 正常 solver 设计。因为 LRA 是可判定的，general-simplex 加 Bland's rule 应该终止。

建议改成：

```text
Debug-only iteration guard:
  If exceeded, dump tableau/basis/assignment and abort in debug mode;
  in release mode may return Unknown only under explicit resource limit.
```

否则你的 solver 会在本应完整的 QF_LRA 上不稳定地返回 unknown。

# 11. SAT / theory solver 持久化还有 atom ID 风险

文档提出后续让 SAT solver 和 theory solver 都持久化，并注册 constraints once。 这里要加一个约束：

```text
Atomizer 的 atom id 必须稳定。
TheoryAtom -> auxiliary var 的映射必须按 normalized atom key，而不是按临时 SAT var。
```

否则会出现：

```text
同一个约束重复注册，产生多个 auxiliary var；
旧 SAT literal 对应的 reason 失效；
push/pop 后 stale atom 留在 GeneralSimplex 中。
```

建议：

```cpp
struct NormalizedLinearAtomKey {
    SparseLinearExpr lhsMinusRhs;
    Relation rel;
};

unordered_map<NormalizedLinearAtomKey, AuxVar> atomToAux_;
unordered_map<SatLit, BoundAssertion> litToBound_;
```

其中 `atomToAux_` 按数学表达式 canonical key，`litToBound_` 按当前 SAT abstraction。

# 12. 建议你把最终修正版写成这样

核心改法：

```text
1. Constraint registration:
   s = p(x) - c
   tableau row: s = -c + Σ a_j x_j

2. Literal assertion:
   true/false literal must be translated by relation type.
   Disequality cannot be represented as one bound.

3. Bound representation:
   lower_/upper_ store BoundInfo(value + reason), not just DeltaRational.

4. resetActiveBounds:
   clears active bounds and reasons;
   marks betaDirty = true.

5. push/pop:
   initialize base trail frame;
   distinguish user push/pop, SAT-level push/pop, and per-model reset.

6. conflict:
   return SAT literals from bound reasons, not raw inequalities.

7. V1 description:
   call it tableau-incremental / semi-incremental, not full DM incremental.

8. tests:
   fix invalid push/pop test.
```

# 13. 简短结论

这个 plan **方向正确，但现在不能直接照着实现**。最需要改的是三点：

```text
第一，tableau row 的符号和存储语义要统一；
第二，false theory literals 尤其是 equality/disequality 不能简单转成 bounds；
第三，V1 的 resetBounds 方案只是半增量，必须补 bound reason、betaDirty、stable atom mapping。
```

这些改完之后，它就可以作为 NLColver 的 LRA 基础版本继续推进。
