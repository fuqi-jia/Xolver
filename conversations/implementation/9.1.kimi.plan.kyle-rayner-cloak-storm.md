# Plan: NIA-Core — Univariate + Domain + Bounded Search Pipeline

## Goal
Upgrade `NiaSolver` from "trivial constants only" to a real nonlinear integer engine:
- **Univariate integer equality solving** (`x^2=4` → SAT, `x^2=2` → UNSAT)
- **Square / bound inference** (`x^2<=9` → finite domain)
- **Algebraic reasoning** (GCD equality conflict, square rules, factor rules)
- **Modular reasoning** (`x^2=2` → mod-4 UNSAT)
- **Domain store** (intervals + finite sets + exclusions)
- **Bounded complete solver** (finite-domain enumeration / interval B&B)
- **Exact integer validation** for every SAT result
- **Dynamic polynomial atom support** for lemmas

After NIA-Core, these must NOT be `Unknown`:
```smt2
(assert (= (* x x) 4))                          ; sat
(assert (= (* x x) 2))                          ; unsat
(assert (= (+ (* x x) 1) 0))                    ; unsat
(assert (and (>= x 0) (<= x 10) (= (* x x) 49))) ; sat
(assert (and (>= x 0) (<= x 10) (= (* x x) 50))) ; unsat
```

### IF-1. `PolynomialKernel` — add `evalInteger` and fix `variables`

**Problem**: Current `LibPolyKernel::variables()` only checks `main_variable`, missing multi-var polynomials. Current `sgn()` only evaluates at `mpq_class` samples.

**Fix** in `PolynomialKernel.h`:
```cpp
// Evaluate polynomial at integer assignment (exact integer arithmetic)
virtual mpz_class evalInteger(
    PolyId a,
    const std::unordered_map<std::string, mpz_class>& sample) const = 0;
```

**Fix** `LibPolyKernel::variables()` to iterate all terms and collect all distinct variables. Current `main_variable` check is insufficient for univariate detection.

**Implementation** in `LibPolyKernel.cpp`: Use libpoly term iteration (if available) or track variables during polynomial construction as fallback.

**Stub** in `StubPolyKernel`: Return 0 for `evalInteger`.

### IF-2. `TheoryAtomRegistry` — add `getOrCreatePolynomialAtom`

**Problem**: No dynamic polynomial atom creation. Branch lemmas, factor splits, and bound tightenings need it.

**Add** to `TheoryAtomRegistry.h`:
```cpp
SatLit getOrCreatePolynomialAtom(
    PolyId poly,
    Relation rel,
    const mpq_class& rhs,
    TheoryId theory

```

**Implementation**: Similar to `getOrCreateLinearBoundAtom` but with polynomial lookup key:

struct PolyLookupKey {

Register new atom into `records_`, observe SAT var, return `SatLit`.

## NIA-Core Component Design

### 1. `NiaNormalizer` — `src/theory/arith/nia/NiaNormalizer.h/.cpp`

**Input**: `std::vector<ActiveNiaConstraint>` (from `NiaSolver` trail)
**Output**: `std::vector<NormalizedNiaConstraint>` or `nullopt` (unsupported)

**Rules**:
1. `p rel 0` form (already from Atomizer)
2. Clear denominators: multiply by positive LCM of all coefficients
3. Strict inequality normalization:
   - `p < 0` → `p <= -1`
   - `p > 0` → `p >= 1`
4. Output relations: `Eq`, `Neq`, `Leq`, `Geq` only

```cpp
struct NormalizedNiaConstraint {
    PolyId poly;      // integer coefficients
    Relation rel;     // Eq, Neq, Leq, Geq
    SatLit reason;
};

class NiaNormalizer {
public:
    explicit NiaNormalizer(PolynomialKernel& kernel);
    std::optional<std::vector<NormalizedNiaConstraint>>
    normalize(const std::vector<ActiveNiaConstraint>& active);
private:
    PolynomialKernel& kernel_;
};
```

### 2. `IntegerModelValidator` — `src/theory/arith/nia/IntegerModelValidator.h/.cpp`

**Input**: `IntegerModel` + normalized constraints
**Output**: `bool` (exact validation)

```cpp
using IntegerModel = std::unordered_map<std::string, mpz_class>;

class IntegerModelValidator {
public:
    explicit IntegerModelValidator(PolynomialKernel& kernel);
    bool validate(const IntegerModel& model,
                  const std::vector<NormalizedNiaConstraint>& constraints) const;
private:
    PolynomialKernel& kernel_;
};
```

Uses `kernel_.evalInteger(c.poly, model)` and checks relation.

### 3. `DomainStore` — `src/theory/arith/nia/DomainStore.h/.cpp`

**Per-variable domain**:
```cpp
struct IntDomain {
    bool hasLower = false;
    bool hasUpper = false;
    mpz_class lower;
    mpz_class upper;

    std::optional<std::set<mpz_class>> finiteValues;
    std::set<mpz_class> excludedValues;
};
```

**Operations**:
- `restrictToFiniteSet(var, values)`
- `addLowerBound(var, lb)`
- `addUpperBound(var, ub)`
- `excludeValue(var, v)`
- `isEmpty(var)` / `isEmpty()`
- `allFinite(vars)`
- `totalSize()` (product of domain sizes, for enumeration threshold)

**Conflict detection**: If any domain becomes empty → `TheoryConflict`.

### 4. `UnivariateIntegerReasoner` — `src/theory/arith/nia/UnivariateIntegerReasoner.h/.cpp`

**Core capability**: Solve `p(x)=0` for integer roots.

**Algorithm**:
1. Extract variable names via `kernel_.variables(poly)`
2. If `vars.size() == 1` → univariate
3. For `Eq`:
   - Factor out `x` while `p(0)==0` (collect root 0)
   - Apply Rational Root Theorem: all integer roots divide `|a_0|`
   - Test each divisor `d` and `-d` by `evalInteger`
   - If no roots → UNSAT conflict
   - If roots found → `DomainStore::restrictToFiniteSet(x, roots)`
4. For `Neq`:
   - If no integer roots → constraint is tautology (ignore)
   - If roots found → `DomainStore::excludeValue(x, root)` for each
5. For `Leq` / `Geq`:
   - `x^2 <= c` (c>=0) → `[-floor(sqrt(c)), floor(sqrt(c))]`
   - `x^2 >= 0` → tautology
   - `x^2 < 0` (already normalized to `x^2 <= -1`) → UNSAT

**Note**: RRT requires coefficient extraction. Since `LibPolyKernel` uses libpoly's internal representation, we need a way to extract coefficients for univariate polynomials. If libpoly doesn't expose this directly, we can evaluate at enough points and use finite differences / interpolation, or we can extend `PolynomialKernel` with `getCoefficients(poly)`.

**Recommended extension to `PolynomialKernel`**:
```cpp
// For univariate polynomials only. Returns coefficients from highest to constant.
// Returns empty if not univariate or unsupported.
virtual std::vector<mpz_class> getIntegerCoefficients(PolyId a) const = 0;
```

**Implementation via evaluation + interpolation** (fallback if libpoly doesn't expose coeffs):
- Evaluate `p(0), p(1), p(2), ..., p(n)` where `n = degree`
- Use Newton interpolation to recover integer coefficients
- This works because we know the polynomial is integer-coefficient after normalization

### 5. `AlgebraicIntegerReasoner` — `src/theory/arith/nia/AlgebraicIntegerReasoner.h/.cpp`

**Rules** (each returns `optional<TheoryCheckResult>`):

**A. Square rules**
- `x^2 >= 0`: tautology, ignore
- `x^2 < 0` (normalized to `x^2 <= -1`): UNSAT
- `x^2 <= 0`: implies `x = 0` → `DomainStore::restrictToFiniteSet(x, {0})`
- `x^2 + y^2 <= 0`: implies `x=0, y=0`
- `x^2 + y^2 < 0`: UNSAT

**B. Coefficient GCD equality conflict**
- For `p = 0` where all monomials are integer-valued:
  - `g = gcd(|coeff_i|)`
  - If `g != 0` and constant term `c % g != 0` → UNSAT
  - Example: `2x^2 + 4yz = 3` → UNSAT (left even, right odd)

**C. Factor rules** (syntactic product recognition)
- `p*q = 0` → generate lemma: `¬(p*q=0) ∨ (p=0) ∨ (q=0)`
- `p*q != 0` → generate lemmas: `¬(p*q!=0) ∨ (p!=0)`, `¬(p*q!=0) ∨ (q!=0)`
- Uses `TheoryAtomRegistry::getOrCreatePolynomialAtom`

**D. Modular reasoning** (small modulus)
- For each active equality `p(x)=0`:
  - Test mod `m ∈ {2,3,4,5,7,8,9,11}`
  - If `numVars <= 6`, enumerate all residue assignments
  - If no residue satisfies all equalities simultaneously → UNSAT
  - Example: `x^2 = 2` → mod 4: squares are 0 or 1, never 2 → UNSAT

### 6. `BoundedNiaSolver` — `src/theory/arith/nia/BoundedNiaSolver.h/.cpp`

**Trigger**: `DomainStore::allFinite()` is true.

**Strategy**:
1. If `totalSize() <= threshold` (e.g., 10000): direct enumeration
2. Otherwise: interval branch-and-bound with pruning

**Direct enumeration**:
```cpp
for each assignment in cartesian product of finite domains:
    if validator.validate(assignment, constraints):
        return Sat(assignment);
return Unsat(conflict over all active reasons);
```

**Interval pruning**:
- For each constraint, evaluate `IntervalZ` over current box
- If constraint definitely violated → prune branch
- If definitely satisfied → skip
- Choose split variable (largest domain or most active)
- Split at midpoint
- Recurse

**IntervalZ**:
```cpp
struct IntervalZ {
    mpz_class lo;
    mpz_class hi;
    bool empty = false;
};
```

Operations: `add`, `sub`, `neg`, `mul` with interval arithmetic.
For `pow`: compute bounds by evaluating at extremes.

### 7. `NiaLocalSearch` — `src/theory/arith/nia/NiaLocalSearch.h/.cpp` (skeleton for NIA-Core)

**Phase NIA-Core scope**: Minimal SAT finder only.

```cpp
class NiaLocalSearch {
public:
    explicit NiaLocalSearch(PolynomialKernel& kernel);
    std::optional<IntegerModel> tryFindModel(
        const std::vector<NormalizedNiaConstraint>& constraints,
        const DomainStore& domains);
private:
    PolynomialKernel& kernel_;
};
```

**Algorithm**:
1. Generate initial assignments:
   - Domain midpoint
   - Univariate roots
   - Small integers in `[-10, 10]`
   - Random within bounds
2. Coordinate descent / small neighborhood search
3. Violation objective: sum of squared violations
4. If `F == 0` found → return model
5. Budget: max iterations / time

**Important**: Any returned model must be exact-validated by `IntegerModelValidator` before `NiaSolver` returns SAT.

## NiaSolver Refactoring

### New `NiaSolver` composition

```cpp
class NiaSolver : public TheorySolver {
private:
    // Infrastructure
    std::unique_ptr<PolynomialKernel> kernel_;
    std::unique_ptr<PolynomialConverter> converter_;

    // Trail state
    std::vector<ActiveNiaConstraint> active_;
    std::vector<NiaTrailEntry> trail_;
    std::optional<PendingConflict> pendingConflict_;
    std::optional<PendingUnknown> pendingUnknown_;

    // NIA-Core engines
    NiaNormalizer normalizer_;
    IntegerModelValidator validator_;
    DomainStore domains_;
    UnivariateIntegerReasoner univariate_;
    AlgebraicIntegerReasoner algebraic_;
    BoundedNiaSolver bounded_;
    NiaLocalSearch localSearch_;

    // Current model (if SAT found)
    std::optional<IntegerModel> currentModel_;
};
```

### New `check()` pipeline

```cpp
TheoryCheckResult NiaSolver::check(TheoryLemmaDatabase& lemmaDb) {
    if (pendingUnknown_) return unknown();
    if (pendingConflict_) return mkConflict(pendingConflict_->conflict);
    if (active_.empty()) return consistent();

    // 1. Normalize
    auto normalized = normalizer_.normalize(active_);
    if (!normalized) return unknown();

    // 2. Trivial constants
    if (auto r = checkTrivialConstants(*normalized)) return *r;

    // 3. Reset domains from normalized constraints
    domains_.reset();

    // 4. Univariate reasoning (roots, square bounds)
    if (auto r = univariate_.run(*normalized, domains_, lemmaDb)) return *r;

    // 5. Algebraic reasoning (square, GCD, factor, modular)
    if (auto r = algebraic_.run(*normalized, domains_, lemmaDb)) return *r;

    // 6. Check if domains became empty
    if (domains_.isEmpty()) {
        return mkConflict(buildConflictFromActive(*normalized));
    }

    // 7. Bounded complete solver
    auto allVars = collectVars(*normalized);
    if (domains_.allFinite(allVars)) {
        return bounded_.solve(*normalized, domains_, validator_, lemmaDb);
    }

    // 8. Local search SAT finder (optional, budget-limited)
    if (auto model = localSearch_.tryFindModel(*normalized, domains_)) {
        if (validator_.validate(*model, *normalized)) {
            currentModel_ = *model;
            return consistent();
        }
    }

    // 9. Branch split or Unknown
    if (auto lemma = buildBranchLemma(*normalized, domains_, lemmaDb)) {
        return mkLemma(*lemma);
    }

    return unknown();
}
```

## Correctness Red Lines

1. **Every SAT result must exact-validate** via `IntegerModelValidator`.
2. **UNSAT must come from sound proof**: constant contradiction, empty integer roots, modular contradiction, GCD contradiction, or finite-domain exhaustion.
3. **Local search failure is NOT UNSAT** — it gives no conclusion.
4. **Rational coefficients must be cleared** in `NiaNormalizer` before any reasoning.
5. **Dynamic atoms must register via `TheoryAtomRegistry`** — no fake ExprId routing.
6. **Conflict clauses must include all active reasons** involved in the proof.
7. **Modular reasoning only for equalities** — don't use it to prove `p != 0`.
8. **Factor split lemmas must be guarded** — `¬premise ∨ conclusion`.
9. **Branch lemmas must be globally valid** — `x <= floor(q) ∨ x >= ceil(q)` for integers.
10. **Unknown is acceptable** for general unbounded nonlinear integer cases.

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| `LibPolyKernel` cannot extract univariate coefficients | Use evaluation+interpolation fallback in `getIntegerCoefficients()` |
| `LibPolyKernel::variables()` incomplete | Fix to iterate all terms; if libpoly API limited, track during construction |
| RRT divisor enumeration too large for large `a_0` | Cap divisor enumeration (e.g., max 1000 divisors); fallback to modular reasoning |
| Bounded solver enumeration explosion | Threshold-based: small domains enumerate, large domains use interval B&B |
| Interval arithmetic overflow | Use `mpz_class` intervals; no fixed-width overflow |
| Modular reasoning exponential for many vars | Limit to `numVars <= 6` and `modulus <= 11`; skip otherwise |
| Dynamic atom registration overhead | Lookup table dedup in `TheoryAtomRegistry`; no-op if atom already exists |
| Product recognition misses hidden factors | Syntactic recognition only; missed factors → Unknown (sound) |
