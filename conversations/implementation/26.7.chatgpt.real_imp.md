对，这个方案**还是在偷懒**。主要偷懒点是：它把“mixed algebraic sample 支持”写成了一个低配版本，但当前数据结构根本不够支撑正确实现。尤其这句：

```text
如果有多个 algebraic variable，先实现 reduceModuloTower 判 zero；
如果 reduce 后不是 zero，interval refinement
```

这其实是把最难的东西一句话带过去了。没有 `liftedDefiningPoly / mainVar / level`，根本没法正确做 `reduceModuloTower`。

---

## 直接结论

当前应该先改成：

```text
P2a 只做 rational-prefix + one-algebraic-variable signAt；
P2b 再做 algebraic root metadata；
P2c 再做 reduceModuloTower；
P3 再做 algebraic-coefficient root isolation。
```

不要现在就宣称：

```text
signAt 支持 mixed sample
```

这会变成假的支持。

---

# 1. 当前方案哪里偷懒

## 偷懒 1：只有 `definingPoly` 不够

现在 `AlgebraicRoot` 只有：

```cpp
UniPolyId definingPoly;
int rootIndex;
mpq_class lower;
mpq_class upper;
```

这只够描述：

```text
某个一元有理系数多项式的第几个根
```

但 CDCAC 里的代数根通常不是孤立存在的，而是某一层 lifting 出来的：

```text
x = alpha
y = beta
beta 是 f(alpha, y) 的某个根
```

所以必须知道：

```cpp
PolyId liftedDefiningPoly;   // 原始多变量多项式
VarId mainVar;               // 当前 root 属于哪个变量
int level;                   // 在变量序中的层级
Sample prefixContext;        // 或至少能从 SamplePoint 中恢复 prefix
```

否则你无法知道：

```text
这个 root 是哪个变量的 root；
它依赖哪些前缀变量；
它的 defining polynomial 在 tower 中怎么用；
后续 reduceModuloTower 应该按哪个变量归约。
```

所以这个字段必须补。

---

## 偷懒 2：没有 gcd，就不能可靠判断 algebraic zero

现在所谓“同定义多项式比例检测”只能处理：

```text
g = c * f
```

但实际必须处理：

```text
g = f * h
```

例如：

```text
f = x^2 - 2
g = (x^2 - 2)(x + 1)
```

在 `alpha = sqrt(2)` 上，`g(alpha)=0`。比例检测发现不了。

所以一元代数根上的 sign 必须至少支持：

```text
gcd(f, g)
```

判断 `alpha` 是否也是 `g` 的根。

没有 gcd，就不要说“单变量 algebraic path 完整”。

---

## 偷懒 3：多个 algebraic variable 不能靠普通 interval refinement

如果有：

```text
x = sqrt(2)
y = sqrt(2)
p = y - x
```

真实值是 0。

如果你只做 interval evaluation：

```text
x ∈ [1.4142, 1.4143]
y ∈ [1.4142, 1.4143]
p ∈ [-0.0001, 0.0001]
```

它会永远包含 0。你 refine 再久也不能证明 zero。

所以必须先做 exact reduction：

```text
y - x mod (y - x) = 0
```

也就是 `reduceModuloTower`。没有它，多个 algebraic sample 的 zero 判断就是不完整的。

---

## 偷懒 4：`reduceModuloTower` 没有 metadata 无法实现

要做：

```text
p mod f_i
```

你必须知道每个 algebraic value 的 defining polynomial：

```text
f_i(x_1,...,x_i)
```

以及它对应哪个主变量 `x_i`。

当前只有 `UniPolyId definingPoly`，这是 specialization 后的一元多项式，不足以做多变量 polynomial remainder。

所以必须先扩展：

```cpp
struct AlgebraicRoot {
    UniPolyId definingPoly;
    PolyId liftedDefiningPoly;
    VarId mainVar;
    int level;
    int rootIndex;
    mpq_class lower;
    mpq_class upper;
};
```

---

# 2. 正确的阶段拆分

## P2a：只做 rational-prefix + 单代数变量

目标不要太大。P2a 可以支持：

```text
sample 中最多一个 algebraic value；
其他变量都是 rational。
```

比如：

```text
x = sqrt(2)
检查 x > 1
检查 x^2 - 2 = 0
检查 (x^2 - 2)(x+1) = 0
```

实现：

```cpp
Sign signAt(PolyId p, const SamplePoint& s) {
    if (s.allRational()) {
        return signAtRational(p, s);
    }

    auto algVars = s.algebraicVars();

    if (algVars.size() == 1) {
        VarId a = algVars[0];

        // 把除 a 外的 rational variables 全部代入
        UniPolyId g = specializeAllOtherVarsRational(p, s, a);

        return signUnivariateAtAlgebraic(g, s.valueOf(a).root());
    }

    return Sign::Unknown;
}
```

P2a 不要假装支持多个 algebraic variables。

---

## P2a 必须完成的东西

### 1. rational root canonicalization

这个可以做，而且应该做。

根隔离之后：

```cpp
RealAlg canonicalizeRoot(UniPolyId f, const AlgebraicRoot& r) {
    if (r.lower == r.upper && eval(f, r.lower) == 0) {
        return RealAlg::fromRational(r.lower);
    }
    return RealAlg::fromAlgebraic(r);
}
```

注意：这应该放在 `isolateRealRoots()` 或 `RootSet` 构造里，不应该散落在 `solveLevel()`。

---

### 2. 一元 gcd

需要实现：

```cpp
UniPolyId gcd(UniPolyId f, UniPolyId g);
bool isConstant(UniPolyId f);
bool rootBelongsTo(const AlgebraicRoot& alpha, UniPolyId g);
```

然后：

```cpp
Sign signUnivariateAtAlgebraic(UniPolyId g, const AlgebraicRoot& alpha) {
    UniPolyId f = alpha.definingPoly;

    if (isZero(g)) return Sign::Zero;

    UniPolyId d = gcd(f, g);

    if (!isConstant(d) && rootBelongsTo(alpha, d)) {
        return Sign::Zero;
    }

    return signByIntervalRefinement(g, alpha);
}
```

这个是 P2a 必须补的，不然代数 sign 不稳。

---

### 3. rational-algebraic compare

`compareRealAlg` 至少要支持：

```text
rational-rational
rational-algebraic
algebraic-rational
same definingPoly + same rootIndex
disjoint interval algebraic-algebraic
```

先不要声称支持所有 algebraic-algebraic equality。

P2a 版本：

```cpp
CompareResult compareRealAlg(const RealAlg& a, const RealAlg& b) {
    if (a.isRational() && b.isRational()) {
        return compare(a.rational(), b.rational());
    }

    if (a.isRational() && b.isAlgebraic()) {
        return compareRationalToAlgebraic(a.rational(), b.root());
    }

    if (a.isAlgebraic() && b.isRational()) {
        return invert(compareRationalToAlgebraic(b.rational(), a.root()));
    }

    if (a.root().definingPoly == b.root().definingPoly &&
        a.root().rootIndex == b.root().rootIndex) {
        return CompareResult::Equal;
    }

    if (a.root().upper < b.root().lower) return CompareResult::Less;
    if (b.root().upper < a.root().lower) return CompareResult::Greater;

    return CompareResult::Unknown;
}
```

`Unknown` 时，`mergeRoots` 不能乱合并，也不能乱排序；可以 refine 若干次，仍不行就返回 Unknown。

## P2c：实现真正的 reduceModuloTower

有了 metadata 后，才能做：

```cpp
PolyId reduceModuloTower(PolyId p, const SamplePoint& sample) {
    PolyId q = p;

    for (int i = sample.level() - 1; i >= 0; --i) {
        VarId x = sample.varOrder[i];
        const RealAlg& a = sample.values[i];

        if (a.isRational()) {
            q = substituteRational(q, x, a.rational());
            continue;
        }

        const auto& r = a.root();

        // 必须保证 r.mainVar == x
        if (r.mainVar != x) {
            return Invalid;
        }

        PolyId f = r.liftedDefiningPoly;

        // 检查 leading coefficient 在 prefix 下不为 0
        auto lc = leadingCoefficient(f, x);
        Sign lcSign = signAt(lc, sample.prefix(i));

        if (lcSign != Sign::Pos && lcSign != Sign::Neg) {
            return Unknown; // nullification or unsupported
        }

        q = pseudoRemainder(q, f, x);
        q = primitivePart(q);
        q = normalize(q);
    }

    return q;
}
```

然后：

```cpp
Sign signAtAlgebraicTower(PolyId p, const SamplePoint& sample) {
    PolyId q = reduceModuloTower(p, sample);

    if (q == Unknown) return Sign::Unknown;
    if (kernel_->isZero(q)) return Sign::Zero;

    return signByIntervalRefinement(q, sample);
}
```

这才是多个 algebraic values 的正确方向。
