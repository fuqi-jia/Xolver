Revise strongly.

The current plan is not acceptable as the final architecture because it still keeps a split design: LIA/LRA use the new trail-based path, while NRA/CdcacSolver may stay on a legacy scan-based path, and unknown logic may fall back to registering both LraSolver and CdcacSolver. This is exactly what we do not want.

We are not implementing a full SAT solver. We continue to use CaDiCaL as the SAT/CDCL engine. What we need to implement is the unified CDCL(T) integration layer and the theory-solver interface.

The final architecture must have exactly one CDCL(T) integration path:

```text
CaDiCaL
  -> TheoryAtomRegistry
  -> TheoryBridge / CaDiCaL ExternalPropagator
  -> TheoryManager
  -> TheorySolver interface
  -> LraSolver / LiaSolver / CdcacSolver / future NiaSolver
```

Do not keep a legacy scan-based theory path.

---

## 1. Remove the legacy scan-based CDCL(T) path

Delete the old theory-checking path after migration.

Remove:

```cpp
TheoryCheckResult check(const CoreIr& ir,
                        const std::vector<Atomizer::AtomRecord>& atoms,
                        const SatSolver& sat);
```

Remove:

```cpp
runLegacyCdclLoop(...)
```

Remove:

```cpp
useLegacy
useLegacyCheckPath
```

Do not keep:

```cpp
if (useLegacy) {
    return runLegacyCdclLoop(atomizer);
}
```

The final `Solver::checkSatInternal()` must have one route only.

If a theory is not migrated yet, that logic should return `Unknown` or be temporarily disabled. It must not use the old scan-based path.

## 2. Do not special-case NRA / CdcacSolver with a legacy path

The current plan says:

```text
CdcacSolver may stay on a legacy scan-based path if migration is deferred.
```

This must be removed.

CdcacSolver must implement the same `TheorySolver` interface as LraSolver and LiaSolver:

```cpp
class TheorySolver {
public:
    virtual TheoryId id() const = 0;

    virtual void assertLit(
        const TheoryAtomRecord& atom,
        bool value,
        int level,
        SatLit reason
    ) = 0;

    virtual void backtrackToLevel(int level) = 0;

    virtual TheoryCheckResult check(
        TheoryLemmaDatabase& lemmaDb
    ) = 0;

    virtual void reset() = 0;
};
```

For CdcacSolver:

```cpp
void CdcacSolver::assertLit(
    const TheoryAtomRecord& atom,
    bool value,
    int level,
    SatLit reason
) {
    // Atom must carry PolynomialAtomPayload.
    // Record active nonlinear constraint.
    // Push trail entry with previous active size.
}

void CdcacSolver::backtrackToLevel(int level) {
    // Roll back active nonlinear constraints.
}

TheoryCheckResult CdcacSolver::check(TheoryLemmaDatabase& lemmaDb) {
    // Run current CDCAC/NRA check over active constraints.
}
```

If CdcacSolver cannot be migrated in this refactor, then QF_NRA/NRA should return `Unknown`. It must not use legacy scan mode.

The same rule applies to future NIA.

## 4. TheoryAtomRegistry must support all theory atom payloads, not only linear atoms

The current `TheoryAtomRecord` is too linear-specific:

```cpp
struct TheoryAtomRecord {
    SatVar satVar;
    TheoryId theory;
    bool isDynamic;
    ExprId exprId;
    LinearFormKey lhs;
    Relation rel;
    mpq_class rhs;
};
```

This is fine for LRA/LIA but not enough for NRA/NIA.

Replace it with a payload variant:

```cpp
struct LinearAtomPayload {
    LinearFormKey lhs;
    Relation rel;
    mpq_class rhs;
};

struct PolynomialAtomPayload {
    PolynomialId poly;
    Relation rel;
    mpq_class rhs;
};

struct TheoryAtomRecord {
    SatVar satVar;
    TheoryId theory;
    bool isDynamic;
    ExprId exprId;

    std::variant<
        LinearAtomPayload,
        PolynomialAtomPayload
        // future: UFAtomPayload, BVAtomPayload, StringAtomPayload
    > payload;
};
```

Rules:

```text
LraSolver / LiaSolver accept only LinearAtomPayload.
CdcacSolver / future NiaSolver accept only PolynomialAtomPayload.
Payload mismatch is an internal unsupported-theory error or returns Unknown.
```

All parsed theory atoms and all dynamic atoms must be registered through `TheoryAtomRegistry`.

No fake ExprId.
No solver-local dynamic atom leakage.
No dynamic atom that cannot be routed by SatVar.

## 5. Atomizer must register every parsed theory atom

`TheoryBridge` will only route through:

```cpp
registry.findBySatVar(lit.var())
```

Therefore, every parsed theory atom must be registered in `TheoryAtomRegistry` during atomization.

For linear atoms:

```cpp
registry.registerParsedTheoryAtom(
    satVar,
    exprId,
    TheoryId::LIA or TheoryId::LRA,
    LinearAtomPayload{lhs, rel, rhs}
);
```

For nonlinear polynomial atoms:

```cpp
registry.registerParsedTheoryAtom(
    satVar,
    exprId,
    TheoryId::NRA,
    PolynomialAtomPayload{poly, rel, rhs}
);
```

Dynamic atoms also go through the registry:

```cpp
registry.getOrCreateLinearBoundAtom(...);
registry.getOrCreatePolynomialAtom(...); // future
```

`Atomizer::atoms()` must not be used in the CDCL(T) path.

## 6. Use CaDiCaL, but do not implement a separate SAT solver

We are not writing a full SAT solver.

CaDiCaL remains the SAT/CDCL engine:

```text
CaDiCaL handles:
  Boolean search
  BCP
  conflict analysis
  learned clauses
  restart
  SAT-level propagation
```

Our code handles:

```text
TheoryAtomRegistry
TheoryBridge / ExternalPropagator integration
TheoryManager
Theory solvers
Theory lemmas/conflicts
```

The final integration should use CaDiCaL ExternalPropagator / IPASIR-UP if available in the vendored version. If the current vendored CaDiCaL does not expose this interface, upgrade CaDiCaL.

Do not treat model replay as the final architecture.

## 7. Remove model replay from the final plan

The current plan uses:

```cpp
listener_->onBacktrack(0);
for (SatVar v = 1; v <= maxVar_; ++v) {
    int val = solver_->val(static_cast<int>(v));
    if (val > 0) listener_->onAssign(SatLit::positive(v), 0);
    else         listener_->onAssign(SatLit::negative(v), 0);
}
```

This must not be the final design.

It is still full model replay and effectively preserves the old full-scan cost. Worse, `onBacktrack(0)` does not remove assignments also stored at level 0 if the theory trail removes only entries with `entry.level > targetLevel`.

Final plan should say:

```text
Production integration:
  CaDiCaL ExternalPropagator / IPASIR-UP.

No replay-all-vars backend in final solver.
```

If an adapter is kept briefly during migration, mark it as temporary and delete it before finalizing the refactor. It must not be the endpoint of this plan.

## 8. Solver::checkSatInternal must have one route

Final structure:

```cpp
Result Solver::checkSatInternal() {
    if (!ir || ir->assertions().empty()) {
        return Result::Sat;
    }

    auto sat = createCadicalSolver();

    TheoryAtomRegistry registry;
    TheoryManager theoryManager;
    TheoryLemmaDatabase lemmaDb;

    registerSolversByLogic(theoryManager);

    Atomizer atomizer(*sat, registry);

    for (ExprId a : ir->assertions()) {
        SatLit lit = atomizer.atomize(a, *ir);
        sat->addClause({lit});
    }

    CadicalTheoryPropagator propagator(
        registry,
        theoryManager,
        lemmaDb
    );

    sat->connectExternalPropagator(&propagator);

    auto result = sat->solve();

    if (result == SatResult::Sat) return Result::Sat;
    if (result == SatResult::Unsat) return Result::Unsat;
    return Result::Unknown;
}
```

No:

```cpp
useLegacy
runLegacyCdclLoop
Atomizer::atoms() scan
```

## 9. Fix TheoryLemmaDatabase ownership

Theory solvers must not insert lemmas into the database.

Wrong:

```cpp
if (lemmaDb.insertIfNew(*lemma)) {
    return mkLemma(*lemma);
}
```

Correct:

```cpp
if (!lemmaDb.contains(candidate)) {
    return mkLemma(candidate);
}
skip candidate;
```

Only the solver integration layer inserts when it is actually going to send the lemma to CaDiCaL:

```cpp
if (lemmaDb.insertIfNew(lemma)) {
    addTheoryClauseToCadical(lemma);
} else {
    return internal error or Unknown;
}
```

Reason:

```text
contains()      is for theory solvers to avoid returning duplicates.
insertIfNew()   is for the integration layer immediately before adding the clause.
```

Otherwise a new lemma inserted inside the theory solver will look duplicate when the outer layer tries to add it.

## 10. Fix LiaSolver trail sizes

The current pseudocode stores vector sizes after modification:

```cpp
disequalities_.push_back(...);
trail_.push_back({level, activeAtoms_.size(), disequalities_.size()});
```

This cannot undo the push.

Store sizes before modification:

```cpp
void LiaSolver::assertLit(...) {
    size_t oldActive = activeAtoms_.size();
    size_t oldDiseq  = disequalities_.size();

    int aux = manager_.getOrCreateAuxVar(gs_, payload.lhs, payload.rhs);

    if (isDisequality(payload.rel, value)) {
        disequalities_.push_back(...);
        trail_.push_back({level, oldActive, oldDiseq});
        return;
    }

    auto result = manager_.assertBound(...);

    if (result.immediateConflict) {
        pendingConflict_ = PendingConflict{level, result.conflict};
        return;
    }

    activeAtoms_.push_back(...);
    trail_.push_back({level, oldActive, oldDiseq});
}
```

Backtrack:

```cpp
void LiaSolver::backtrackToLevel(int level) {
    while (!trail_.empty() && trail_.back().level > level) {
        activeAtoms_.resize(trail_.back().activeAtomsSize);
        disequalities_.resize(trail_.back().disequalitiesSize);
        trail_.pop_back();
    }

    if (pendingConflict_ && pendingConflict_->level > level) {
        pendingConflict_.reset();
    }

    gs_.backtrackToLevel(level);
}
```

## 11. pendingConflict_ must be level-tagged

Do not store:

```cpp
std::optional<TheoryConflict> pendingConflict_;
```

Use:

```cpp
struct PendingConflict {
    int level;
    TheoryConflict conflict;
};

std::optional<PendingConflict> pendingConflict_;
```

When backtracking:

```cpp
if (pendingConflict_ && pendingConflict_->level > targetLevel) {
    pendingConflict_.reset();
}
```

Otherwise a conflict caused by a high-level assignment may survive after SAT backtracks and cause a spurious conflict.

## 12. Fix no-op lemma guards

For a guarded lemma:

```text
¬premise ∨ conclusion
```

The tautological no-op case is:

```text
conclusion == premise
```

because:

```text
¬p ∨ p
```

is tautological.

The current plan’s condition:

```text
tightenedLit == lit.negated()
```

is wrong. That would produce:

```text
¬p ∨ ¬p
```

which is not a tautology.

Correct guards:

```cpp
if (conclusion == premise) {
    skip;  // tautological guarded lemma
}

if (conclusion == premise.negated()) {
    return internal error or skip unless explicitly justified;
}
```

Also skip:

```text
self-tightening;
duplicate lemma;
already satisfied conclusion once SatAssignmentView is available.
```

```cpp
atomRegistry_.reset();
```

## Final decision

This plan should not be approved until it removes the legacy scan path and commits to one unified theory integration path for all theories.

The corrected direction is:

```text
Continue using CaDiCaL.
Do not implement a SAT solver.
Use one theory integration interface.
All theory solvers use it.
No legacy scan fallback.
Unknown logic returns Unknown.
```
