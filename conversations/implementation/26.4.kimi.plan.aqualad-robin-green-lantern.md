# CDCAC 求解器重构计划：从常数检查器到完整 NRA 核心

## 目标

将当前仅支持常数约束的 `CdcacSolver` 骨架，重构为模块化的 **Conflict-Driven Cylindrical Algebraic Covering (CDCAC)** 引擎，使 QF_NRA 逻辑能够求解非线性实数约束。

## 当前状态

- `CdcacSolver::check()` 遇到非常数多项式直接返回 `Unknown`
- `NraSolver` 是薄 facade，未连接 `NraLinearizationAdapter`
- 多项式基础设施（`PolynomialKernel`, `LibPolyKernel`, `PolynomialConverter`）已成熟
- NIA 求解器有完整 pipeline 可供参考

## 架构总览

保留 `CdcacSolver` 外壳作为 CDCL(T) facade，引入三层内部结构：

```
NraSolver (TheorySolver facade)
  └── CdcacSolver (trail + active constraints + effort dispatch)
        └── CdcacCore (recursive covering/search algorithm)
              ├── AlgebraBackend   (exact root isolation / sign eval / projection)
              ├── ProjectionEngine (McCallum / Lazard / Conservative)
              ├── CoveringManager  (cells, intervals, union, sample selection)
              └── ReasonManager    (label collection, safe conflict minimization)
```

`libpoly` 只出现在 `LibpolyBackend` 实现中，核心算法不直接依赖它。

**核心设计约束：**
- `CdcacCore` 不依赖 `CdcacSolver` 的 private 类型（如 `ActiveConstraint`），只通过 `CdcacInput` / `CdcacConstraint` 交互
- `AlgebraBackend` 统一使用 `PolyId` / `UniPolyId` / `VarId` / `SamplePoint`，禁止裸 `std::string` map 和 `vector<mpz_class>` 主接口
- `ModelSeed` 从 P1 起就设计进接口，`chooseSampleOutside` 优先使用 seed

### AlgebraBackend 接口（定型版）

```cpp
class AlgebraBackend {
public:
    virtual ~AlgebraBackend() = default;

    // Root isolation for a univariate polynomial (already specialized)
    virtual RootSet isolateRealRoots(UniPolyId p) = 0;

    // Sign of multivariate polynomial at a sample point.
    // SamplePoint may contain algebraic values (P1+).
    virtual Sign signAt(PolyId p, const SamplePoint& sample) = 0;

    // Specialize a multivariate polynomial to univariate by substituting prefix.
    virtual UniPolyId specializeToUnivariate(
        PolyId p,
        const SamplePoint& prefix,
        VarId mainVar
    ) = 0;

    // Projection: eliminate eliminateVar from polys, return new polynomial set.
    virtual std::vector<PolyId> projectionPolys(
        const std::vector<PolyId>& polys,
        VarId eliminateVar,
        ProjectionMode mode
    ) = 0;

    // Self-check: validate that root isolation is correct (Sturm count, disjoint intervals, etc.)
    virtual bool validateRootIsolation(UniPolyId p, const RootSet& roots) = 0;
};
```

### 核心数据结构（P0 版）

```cpp
// CdcacConstraint.h — public, shared between Solver facade and Core
struct CdcacConstraint {
    ConstraintId id;
    PolyId poly;
    Relation rel;
    SatLit reason;
    int level;
};

struct CdcacInput {
    std::vector<CdcacConstraint> constraints;
    std::vector<VarId> varOrder;          // x1, x2, ..., xn
    std::optional<ModelSeed> seed;
};

// CdcacTypes.h
struct ModelSeed {
    std::unordered_map<VarId, mpq_class> values;
};

struct AlgebraicRoot {
    UniPolyId definingPoly;   // univariate after specialization
    int rootIndex;            // ordered real root index
    mpq_class lower;          // isolating interval
    mpq_class upper;
};

struct RealAlg {
    enum class Kind { Rational, AlgebraicRoot };
    Kind kind;
    mpq_class rational;       // valid if kind == Rational
    AlgebraicRoot root;       // valid if kind == AlgebraicRoot
};

struct SamplePoint {
    std::vector<VarId> varOrder;
    std::vector<RealAlg> values;  // parallel to varOrder
};

enum class CellKind { Sector, Section, Point, FullLine };

struct Bound {
    enum class Kind { NegInf, PosInf, Rational, AlgebraicRoot } kind;
    RealAlg value;   // valid if kind == Rational or AlgebraicRoot
    bool open;
};

struct Cell {
    VarId var;
    CellKind kind;
    Bound lower;
    Bound upper;
    std::vector<SatLit> reasons;   // active literals whose negation refutes this cell
    std::vector<PolyId> guards;    // projection polynomials guaranteeing delineability
};

struct Covering {
    VarId var;
    std::vector<Cell> cells;
    bool coversAllLine() const;
    std::optional<RealAlg> chooseSampleOutside(const mpq_class* preferred) const;
};
```

### CdcacSolver::check() 新流程

```cpp
TheoryCheckResult CdcacSolver::check(TheoryEffort effort, const ModelSeed* seed) {
    if (pendingConflict_ && isStillActive(pendingConflict_->reasons))
        return TheoryCheckResult::mkConflict(ReasonManager::toConflict(pendingConflict_->reasons));

    auto trivial = checkConstantsOnly();
    if (!trivial.isConsistent()) return trivial;

    if (effort != TheoryEffort::Full)
        return TheoryCheckResult::consistent();  // defer to full effort

    CdcacInput input = buildInput(seed);
    CdcacResult res = core_.solve(input);

    switch (res.status) {
        case CdcacStatus::Sat:
            lastModel_ = res.model;
            return TheoryCheckResult::consistent();
        case CdcacStatus::Unsat:
            pendingConflict_ = PendingConflict{currentMaxLevel(), res.reasons};
            return TheoryCheckResult::mkConflict(ReasonManager::toConflict(res.reasons));
        case CdcacStatus::Unknown:
            return TheoryCheckResult::unknown();
    }
}
```

### 关键认识

`x^2 - 2 = 0` 没有有理解，因此 P1 **必须**支持一元代数根 sample。P1 不需要多变量代数点求值，但必须支持：
- 单变量多项式的实根隔离（`isolateRealRoots`）
- 代数根的表示与比较（`AlgebraicRoot`，interval refinement）
- 单变量多项式在单一代数根上的符号求值（`signAt` with `SamplePoint` containing one `AlgebraicRoot`）

### P1 核心算法

```cpp
CdcacResult CdcacCore::solveUnivariate(const CdcacInput& input) {
    // 1. Collect all polynomials, get varOrder (size == 1)
    // 2. For each polynomial, isolate real roots -> RootSet
    // 3. Merge all roots, cut real line into cells (sectors + sections)
    // 4. For each cell:
    //      pick sample (rational for sector, algebraic root for section)
    //      if sample satisfies ALL constraints -> Sat
    // 5. If no cell satisfies -> Unsat, cells form covering, collect reasons
    // 6. If root isolation/signAt fails -> Unknown
}
```

```
0 = 0                       → sat
-1 < 0                      → sat
x^2 + 1 = 0                 → unsat

x^2 < 0                     → unsat
```

## 阶段 P2a：多变量递归框架 + Rational Sample（3-4 天）

**目标**：建立多变量递归 `solveLevel` 框架，支持 rational sector sample。Leaf conflict 可生成。暂不做 projection 和 parent-cell generalization。

### 策略

- 固定变量顺序 `varOrder[x1, x2, ..., xn]`
- 第 k 层：对 `xk` 取 rational sample，递归检查 `x_{k+1}...xn`
- Leaf（k == n）：像 P1 一样检查所有约束
- 若 leaf 失败，生成 leaf conflict cell（同 P1 的单变量逻辑，但针对最内层变量）
- **暂不做 projection**：若递归返回 Unsat，直接把这个 specific rational sample 标记为 "tried"，换下一个 rational sample
- 若某层所有 rational sample 都试完仍不行，返回 Unknown（因为还没做 projection/generalization）

### P2a 测试集

```
x*y = 1, x=1, y=1          → sat  (纯 rational)
x*y = 0, x!=0, y!=0        → unsat  (leaf 冲突，无需 projection)
```

## 阶段 P4：Nullification Contract + Lazard Fallback（4-5 天）

**目标**：处理 McCallum projection 的 nullification 问题，保证 soundness。

### Nullification Contract

**定义**：projection polynomial `q(x1,...,xk)` 在当前 prefix `(a1,...,a_{k-1})` 上代入后恒为零，即 `q(a1,...,a_{k-1}, xk)` 是零多项式。

**检测时机**：在 `generalizeConflictCell` 中，得到 projected polynomial 的 roots 后，检查是否有 projection polynomial 在当前 prefix 上 vanish。

**Contract**：

```cpp
// 在 generalizeConflictCell 中：
for (PolyId guard : projectedPolys) {
    if (algebra_.vanishesAtPrefix(guard, currentPrefix)) {
        // McCallum projection 在当前点不可信
        if (mode == ProjectionMode::McCallum) {
            // 1. 切换 Lazard projection
            // 2. 重建 guards
            // 3. 重新生成 parent cell
            // 若 Lazard 仍失败 → Unknown
            return fallbackOrUnknown(k, sample, childConflict);
        }
        // Conservative mode 也应检查，若仍 vanish → Unknown
        return CdcacResult::unknown("nullification under conservative projection");
    }
}
```

**关键规则**：只要检测到 projection polynomial 在当前 prefix 上恒为零，且当前 projection mode 不能证明 delineability，则当前 branch 不得返回 Unsat。

## 阶段 P5：Covering Pruning + Safe Reason Minimization（3-4 天）

**目标**：减小 covering 大小，优化 conflict clause，但**不破坏 soundness**。

### 安全策略（按优先级）

1. **Cell merge**：相邻且 reasons/guards 兼容的 cell 合并
2. **Subsumed cell deletion**：被其他 cell 覆盖的删除
3. **Reason union 去重**：简单收集所有 cell 的 reasons，去重
4. **Deletion-based safe minimization**：逐个尝试删除 reason，检查 covering 是否仍完整，且每个 cell 的 local refutation 仍然有效

**禁止（P5 不做）**：
- 不考虑 cell local proof 有效性的 greedy set-cover
- 可能导致 "covering 形式上覆盖全空间但某 cell 不可满足性已失去约束支撑" 的优化

## 阶段 P6：LRA / Linearizer / CDCL 深度集成（4-5 天）

5. **Resource budget**：max cells（如 10000）、max depth、timeout → 超出则 Unknown
6. **统计**：`CdcacStats`（calls / sat / unsat / unknown / cellsCreated / projectionCalls / rootIsolationCalls / signEvalCalls / libpolyFailures）

## Soundness 红线（不可违反）

1. **后端失败 → Unknown**：libpoly 返回异常、root isolation 验证失败、algebraic sign 不确定、nullification 未处理时，绝不返回 Sat/Unsat
2. **Sat 必须经过 sample 验证**：`checkFullSample` 必须逐项检查所有 active constraints
3. **Conflict clause 只能含 active literals**：projection polynomial 本身不能成为 reason
4. **Nullification 未处理不得 Unsat**：检测到 projection polynomial vanish 且 mode 不可信时，只能 Unknown 或 Lazard fallback
5. **Reason minimization 不得破坏 local proof**：删除 reason 后，每个 cell 的不可满足性仍必须有约束支撑
