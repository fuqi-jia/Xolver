对，**这种每轮遍历 `atomizer.atoms()` 的方式不是最终设计**。它可以作为早期 MVP，但不能作为你要的“正经 SMT solver”的最终架构。

你现在日志里这类逻辑：

```cpp
auto tr = theoryManager_.check(*ir, atomizer.atoms(), *sat);

for (const auto& atom : atoms) {
    if (!atom.isTheory) continue;
    bool val = sat.value(atom.var);
    solver->assertLit({atom.var, atom.expr}, val, ir);
}
```

本质是：**SAT 每给一个 model，就把所有 theory atoms 全扫一遍，再重新 assert 给 theory solver**。这会导致两个问题：第一，随着 dynamic atoms、cuts、branch lemmas 增多，每轮 `check()` 越来越慢；第二，theory solver 被迫半增量甚至重建 active state，重复工作很多。你日志里也已经看到它在讨论 `atomizer.atoms()` 是否包含 dynamic atoms，以及每轮如何把它们喂回 `LiaSolver`。

最终版本应该改成：**SAT assignment trail 驱动的增量 CDCL(T) 接口**，不要每轮全量扫描 atoms。

---

# 最终架构：Trail-based Theory Bridge

核心原则：

```text
Atomizer / TheoryAtomRegistry:
  只负责注册 atom 和 SatVar 映射。
  不负责每轮遍历。

SAT solver:
  每次赋值一个 literal 时，通知 TheoryBridge。

TheoryBridge:
  只把“本轮新赋值的 theory literal”推给对应 theory solver。
  不扫描所有 atoms。

TheorySolver:
  增量维护 active bounds / disequality obligations / reasons。
  backtrack 时按 decision level 回滚。
```

也就是说，最终不要这样：

```cpp
for (auto& atom : atomizer.atoms()) {
    if (sat.value(atom.var) assigned) {
        solver.assertLit(atom, value);
    }
}
```

而要这样：

```cpp
void TheoryBridge::onSatAssign(SatLit lit, int level) {
    auto* atom = registry_.findBySatVar(lit.var());
    if (!atom) return;

    TheorySolver* solver = solverFor(atom->theory);
    solver->assertLit(*atom, lit.isPositive(), level, lit);
}
```

# 最终数据结构

## 1. TheoryAtomRegistry

`TheoryAtomRegistry` 是全局 atom 映射中心。

```cpp
struct TheoryAtomRecord {

    TheoryId theory;

    bool isDynamic;
    ExprId exprId;  // parsed atom 可用；dynamic atom 可为 invalid/synthetic

    std::unordered_map<LinearBoundAtomKey, SatVar> linearAtomToVar_;

```

查找必须是：

```text
O(1) SatVar -> TheoryAtomRecord
```

不要每轮扫 `records_`。

## 2. SAT solver assignment callback

最终 `SatSolver` wrapper 需要暴露 assignment trail 事件：

```cpp
class SatAssignmentListener {
public:
    virtual void onAssign(SatLit lit, int decisionLevel) = 0;
    virtual void onBacktrack(int targetLevel) = 0;
};

class SatSolver {
public:
    void setAssignmentListener(SatAssignmentListener* listener);

    SolveResult solve();
    void addClause(const std::vector<SatLit>& clause);
};
```

当 SAT solver 内部做：

```text
decision literal
unit propagation literal
learned propagation literal
```

时，都调用：

```cpp
listener_->onAssign(lit, currentDecisionLevel());
```

回溯时调用：

```cpp
listener_->onBacktrack(targetLevel);
```

如果你暂时的 SAT backend 不支持 callback，也应该在 wrapper 层模拟 assignment trail，而不是让 `TheoryManager` 扫所有 atoms。接口要先按最终形态设计。

# 3. TheoryBridge

新增一个中间层，不让 `Solver::checkSatInternal()` 直接遍历 atoms。

```cpp
class TheoryBridge : public SatAssignmentListener {
public:
    TheoryBridge(TheoryAtomRegistry& registry,
                 TheoryManager& theoryManager)
        : registry_(registry), theoryManager_(theoryManager) {}

    void onAssign(SatLit lit, int level) override {
        const TheoryAtomRecord* atom = registry_.findBySatVar(lit.var());
        if (!atom) return;

        theoryManager_.assertTheoryLit(*atom, lit, level);
    }

    void onBacktrack(int targetLevel) override {
        theoryManager_.backtrackToLevel(targetLevel);
    }

private:
    TheoryAtomRegistry& registry_;
    TheoryManager& theoryManager_;
};
```

这样，dynamic atom 被 SAT 赋值时，会**立即**进入对应 theory solver，不需要下一轮扫描。

# TheoryManager 最终接口

不要：

```cpp
TheoryCheckResult check(const CoreIr& ir,
                        const std::vector<AtomRecord>& atoms,
                        const SatSolver& sat);
```

改成：

```cpp
class TheoryManager {
public:
    void registerSolver(std::unique_ptr<TheorySolver> solver);

    void assertTheoryLit(
        const TheoryAtomRecord& atom,
        SatLit assignedLit,
        int decisionLevel
    );

    void backtrackToLevel(int level);

    TheoryCheckResult check();

private:
    std::unordered_map<TheoryId, TheorySolver*> solverByTheory_;
};
```

`assertTheoryLit`：

```cpp
void TheoryManager::assertTheoryLit(
    const TheoryAtomRecord& atom,
    SatLit assignedLit,
    int level
) {
    auto* solver = solverByTheory_.at(atom.theory);
    bool value = assignedLit.isPositive();

    solver->assertLit(atom, value, level, assignedLit);
}
```

`check()` 只调用 solver 的当前状态：

```cpp
TheoryCheckResult TheoryManager::check() {
    for (auto& [id, solver] : solverByTheory_) {
        auto r = solver->check();
        if (!r.isConsistent()) return r;
    }
    return TheoryCheckResult::consistent();
}
```

注意：这里没有 atoms 遍历。

# LiaSolver 最终接口

`LiaSolver` 也不能每轮 reset 后重建 active atoms。它应该增量接受 literal。

```cpp
class LiaSolver : public TheorySolver {
public:
    void assertLit(
        const TheoryAtomRecord& atom,
        bool value,
        int decisionLevel,
        SatLit reason
    ) override;

    void backtrackToLevel(int level) override;

    TheoryCheckResult check() override;

private:
    GeneralSimplex gs_;

    struct TrailEntry {
        int level;
        enum class Kind {
            Bound,
            Disequality,
            ActiveAtom
        } kind;

        // enough data to undo
    };

    std::vector<TrailEntry> trail_;
    std::vector<size_t> levelLimits_;

    std::vector<ActiveLinearAtom> activeAtoms_;
    std::vector<DiseqInfo> disequalities_;
};
```

`assertLit` 只处理新增 literal：

```cpp
void LiaSolver::assertLit(
    const TheoryAtomRecord& atom,
    bool value,
    int level,
    SatLit reason
) {
    ensureLevel(level);

    int aux = manager_.getOrCreateAuxVar(gs_, atom.lhs, atom.rhs);

    if (isDisequality(atom.rel, value)) {
        disequalities_.push_back({aux, reason});
        trail_.push_back(...);
        return;
    }

    manager_.assertBound(gs_, aux, atom.rel, value, reason, level);

    activeAtoms_.push_back(...);
    trail_.push_back(...);
}
```

`backtrackToLevel`：

```cpp
void LiaSolver::backtrackToLevel(int level) {
    while (!trail_.empty() && trail_.back().level > level) {
        undo(trail_.back());
        trail_.pop_back();
    }

    gs_.backtrackToLevel(level);
}
```

这样 `activeAtoms_` 和 `GeneralSimplex` 都是增量状态，不需要每轮扫所有 atom 重建。

# Lemma 去重也要全局化，不要每轮清空

你现在卡住的根源之一就是同一个 lemma 每轮返回。最终版本必须有全局 lemma database。

```cpp
class TheoryLemmaDatabase {
public:
    bool insertIfNew(const TheoryLemma& lemma);

private:
    std::unordered_set<ClauseKey> emitted_;
};
```

在 `TheoryManager` 或 `Solver` 层做：

```cpp
if (tr.kind == Lemma) {
    if (lemmaDb_.insertIfNew(*tr.lemma)) {
        sat.addClause(tr.lemma->lits);
        continue;
    }

    // duplicate lemma: theory solver must continue to next reasoning source,
    // not return the same lemma again.
}
```

更好的方式是 theory solver 内部不要返回重复 lemma：

```cpp
std::optional<TheoryLemma> LiaSolver::tryEmitLemma(TheoryLemma lemma) {
    if (lemmaDb_.isKnown(lemma)) return std::nullopt;
    return lemma;
}
```

但无论放哪一层，**不能在 `LiaSolver::reset()` 中清空**。它必须贯穿整个 `check-sat` 调用，甚至可以贯穿 solver lifetime。

# No-op lemma 必须在生成处消灭

最终版本里，以下 lemma 不允许返回：

```text
1. tautology clause:
   ¬p ∨ p

2. conclusion already true:
   ¬premise ∨ conclusion
   且 conclusion 当前已赋 true

3. duplicate lemma:
   已经 addClause 过

4. self-tightening:
   x >= 0  推出 x >= 0
   2x <= 4 推出 2x <= 4
```

所以 `IntegerReasoner` 需要接收一个轻量 SAT assignment view：

```cpp
class SatAssignmentView {
public:
    LBool value(SatLit lit) const;
};
```

生成 lemma 前检查：

```cpp
if (tightenedLit == originalLit) return nullopt;

if (satView.value(tightenedLit) == LBool::True) {
    return nullopt;
}
```

这不是调试逻辑，是最终 solver 的性能与终止性基本要求。

# 最终 `checkSatInternal()` 不应扫描 atoms

最终主循环应该是：

```cpp
Result Solver::checkSatInternal() {
    SatSolver sat;
    TheoryAtomRegistry registry;
    TheoryManager theory;
    TheoryBridge bridge(registry, theory);

    sat.setAssignmentListener(&bridge);

    registerSolversByLogic(theory);

    // atomize assertions once
    for (ExprId assertion : ir->assertions()) {
        SatLit lit = atomizer.atomize(assertion, *ir);
        sat.addClause({lit});
    }

    while (true) {
        auto sr = sat.solve();

        if (sr == Unsat) return Unsat;
        if (sr == Unknown) return Unknown;

        auto tr = theory.check();

        if (tr.kind == Consistent) {
            return Sat;
        }

        if (tr.kind == Conflict) {
            sat.addClause(tr.conflict.lits);
            continue;
        }

        if (tr.kind == Lemma) {
            if (lemmaDb.insertIfNew(tr.lemma)) {
                sat.addClause(tr.lemma.lits);
            }
            continue;
        }

        if (tr.kind == Unknown) {
            return Unknown;
        }
    }
}
```

注意这里没有：

```cpp
theoryManager.check(ir, atomizer.atoms(), sat)
```

也没有：

```cpp
for all atoms
```

# 最终复杂度对比

旧方案：

```text
每次 SAT model:
  遍历所有 atoms: O(#atoms)
  每个 atom 查询 SAT value
  每个 active atom 重新 assert 到 theory
  theory state 重建/半重建

总成本:
  O(#models * #atoms)
```

最终方案：

```text
SAT 每次新赋值 literal:
  若是 theory atom，O(1) 查 registry
  O(1) 路由给 theory solver
  theory solver 增量 assert

每次 theory check:
  不扫描 atoms
  只检查当前 theory state
  simplex 内部用 violation queue / sparse tableau

总成本:
  O(#assigned_theory_literals + theory_check_cost)
```

这才是最终形态。

# 结论

你的直觉是对的：**每轮遍历所有 atoms 是不好的，不能作为最终版本。**

最终版本必须是：

```text
SAT trail 驱动
+ TheoryBridge 增量路由
+ TheorySolver decision-level trail
+ GeneralSimplex 增量 bound trail
+ lemma database 去重
```

而不是：

```text
SAT model 出来后，全量扫描 atomizer.atoms()
```

这也是为什么你现在会遇到重复 lemma、dynamic atom 回灌、active atom 重建这些问题。它们都不是孤立 bug，而是当前全量扫描式 CDCL(T) 接口的结构性问题。
