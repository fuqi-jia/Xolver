# 1. 4.6 里仍残留错误公式，必须删干净

前面 4.5 已经写对了：

```text
Σ frac(a'_j) * y_j >= 1 - frac(b')
```

这是正确的 Gomory fractional cut。

但 4.6 开头仍然写了旧公式：

```text
Σ frac(a'_j) * y_j >= frac(b')
```

这个和后面 `(1 - f0) + ...` 的 RHS 矛盾。

这个必须改掉。否则 Kimi 实现时可能按错误公式写。

直接要求改成：

```text
4.6 把 cut 转回原变量

cut:
    Σ frac(a'_j) * y_j >= 1 - frac(b')

if x_j at lower:
    y_j = x_j - l_j

if x_j at upper:
    y_j = u_j - x_j

therefore:
    Σ_L f_j*(x_j - l_j) + Σ_U f_j*(u_j - x_j) >= 1 - f0

expanded:
    Σ_L f_j*x_j - Σ_U f_j*x_j
      >= (1 - f0) + Σ_L f_j*l_j - Σ_U f_j*u_j
```

并且整段不要再出现：

```text
>= frac(b')
>= f0
```

除非是在说明“这是错误公式”。

# 2. Gomory 可能生成 contradiction cut，需要单独处理

例如：

```text
x_B = 5/2 - x
```

正确 Gomory cut 是：

```text
0 >= 1/2
```

这是一个**矛盾 cut**，不是普通 dynamic atom。计划里测试部分已经意识到这个例子会生成矛盾 cut，而不是 `x >= 1`。

但是实现设计里还默认：

```cpp
SatLit cutLit = registry_.getOrCreateLinearBoundAtom(cut->lhs, cut->rel, cut->rhs);
lemma = ¬premises ∨ cutLit;
```

如果 cut 的 lhs 为空、rhs 为正，例如：

```text
0 >= 1/2
```

就不能普通注册成一个动态 atom。应该直接返回 guarded conflict lemma：

```text
¬premise_1 ∨ ... ∨ ¬premise_k
```

建议补一条：

```text
If a Gomory cut normalizes to a constant contradiction, e.g. 0 >= positive,
do not register a dynamic cut atom.
Return the guarded conflict lemma:
    ¬premise_1 ∨ ... ∨ ¬premise_k

If it normalizes to a tautology, skip the cut.
```

这是 correctness + 工程接口问题。

# 3. Unit Cube Test 仍缺 active constraint 规范化规则

现在 Phase 3 只写了：

```text
A x <= b
a_i · z <= b_i - 1/2 ||a_i||_1
```

但没有明确怎么把 active atoms 统一成 `<=` 系统。这个会影响正确性。需要补：

```text
Before building A x <= b for unit cube test:

1. For a·x <= c:
      keep as is.

2. For a·x >= c:
      convert to -a·x <= -c.

3. For equality a·x = c:
      add both a·x <= c and -a·x <= -c.

4. For strict integer inequality:
      a·x < c  =>  a·x <= ceil(c) - 1
      a·x > c  =>  a·x >= floor(c) + 1
                 => -a·x <= -(floor(c)+1)

5. Disequality must not enter the cube system.
   Disequality obligations are handled before cube test.

6. Clear denominators before integer rounding-related operations.
```

否则 cube test 很容易处理错 `>=`、`=`、`<`、`>`。

# 4. GCD/cheap reasoning 必须作用在 effective active literal 上

Phase 2 里的 gcd equality conflict、gcd tightening、single-variable tightening 都应该作用在 **当前 SAT assignment 下的 effective relation** 上，而不是原始 atom relation。也就是说：

```text
lit true  : use original relation
lit false : use negated relation
```

例如：

```text
¬(a·x <= c)  等价于  a·x > c
```

LIA 下再转成：

```text
a·x >= floor(c) + 1
```

建议补一句：

```text
IntegerReasoner must operate on normalized effective active constraints,
after false-literal negation and strict-integer normalization.
It must not reason directly on the original parsed relation.
```

# 5. Gomory cut expansion 后的 validation 要评估 expanded cut

计划已经要求 cut 在注册 dynamic atom 前展开回原始变量空间，这是对的。 但 cut validation 里要明确：

```text
evaluate(cut, beta) must be done after expansion to original variables,
or using a definition-aware evaluator that expands aux variables consistently.
```

否则 raw cut over GS aux 与 registered cut over original vars 可能不一致，debug 时很难查。
