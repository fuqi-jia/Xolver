还要 **Revise**。这版比上一版明显好，上一轮几个核心 blocker 基本都意识到了：**不注册 mixed logic、term-level proof graph、Bool-valued UF、n-ary =/distinct、full signature、assertedLit、sort check、trail push/pop** 都补上了。

但是现在还有几个新的实现级 blocker，不修直接做会很容易卡住或产生隐蔽错误。

---

# 1. 最大 blocker：n-ary normalize 的实现方案仍然不稳

你现在写的是：

```cpp
registry_->registerParsedTheoryAtom(
    pv, eid, TheoryId::EUF,
    EufAtomPayload{lhs, rhs, Relation::Eq});
```

也就是说，所有从 n-ary equality / distinct 生成的 pairwise atom 都共用原始 `eid`。

这有风险：

```smt2
(assert (= a b c))
```

会生成：

```text
v1 ↔ (= a b)
v2 ↔ (= b c)
```

但它们都记录 `exprId = eid`，如果 `TheoryAtomRegistry` 或后续诊断、查找、dedup、model 映射依赖 `exprId`，就会冲突或覆盖。

更大的问题是：你这里直接写死注册成 `TheoryId::EUF`。如果 Atomizer 是全局通用的，那么 LRA/NRA 里的 n-ary equality 也可能被错误注册成 EUF atom。

建议改成：

```cpp
struct SyntheticTheoryAtomOrigin {
    ExprId originExpr;
    size_t syntheticIndex;
};
```

或者更简单：

```cpp
struct EufAtomPayload {
    ExprId lhsExpr;
    ExprId rhsExpr;
    Relation rel;
    ExprId originExpr;      // 原始 n-ary expr 或 binary expr
    uint32_t syntheticIndex;
};
```

然后 n-ary helper 只能在 `defaultTheory_ == TheoryId::EUF` 时直接生成 EUF atoms。否则要么走对应 theory 的 binary atom extraction，要么先标记 unsupported。

也就是说：

```cpp
if (defaultTheory_ == TheoryId::EUF) {
    return atomizeNaryEufEq(...);
} else {
    return atomizeNaryGenericEq(...); // 或 unsupported
}
```

不要在通用 `atomizeRec` 前置逻辑里无条件注册 EUF。

# 2. Bool 常量只处理 `true` 不够，`false` 和 `true != false` 必须处理

你现在只设计了：

```cpp
constexpr ExprId TrueSentinelExpr = ...
```

以及：

```cpp
Kind::ConstBool with value true → trueConstant()
```

但 QF_UF 里会出现：

```smt2
(assert (= (p a) false))
(assert (distinct (p a) false))
(assert (= true false))
```

如果 EUF term manager 不支持 `false`，这些要么 unknown，要么错误。

建议加：

```cpp
constexpr ExprId TrueSentinelExpr  = std::numeric_limits<ExprId>::max() - 1;
constexpr ExprId FalseSentinelExpr = std::numeric_limits<ExprId>::max() - 2;
```

并在 `EufTermManager` 里有：

```cpp
EufTermId trueConstant();
EufTermId falseConstant();
```

同时，EGraph 初始化后要内置约束：

```text
true != false
```

否则：

```smt2
(assert (= true false))
```

可能被当成普通两个常量合并，错误 SAT。

最简单做法是在 `check()` 里额外检查：

```cpp
if (egraph_.same(termManager_.trueConstant(),
                 termManager_.falseConstant())) {
    // 如果是由某个 asserted equality 导致的，解释 true=false，
    // conflict clause 为这些 equality reasons 的否定。
}
```

如果不想让 EUF 处理 Bool 常量，那么必须在 Atomizer 层把 Bool equality 全部转成 propositional formula，例如：

```smt2
(= (p a) false)  =>  (not (p a))
(= (p a) true)   =>  (p a)
(distinct (p a) false) => (p a)
```

但这更麻烦。V1 更建议支持 `true/false` 两个特殊 EUF 常量。

# 3. TheoryAtomRegistry 的 dedup API 设计不一致

你写了 dedup：

```cpp
std::unordered_map<EufLookupKey, size_t, EufLookupKeyHash> eufLookup_;
```

但注册接口是：

```cpp
void registerEufAtom(SatVar satVar, ExprId exprId, const EufAtomPayload& payload);
```

这有一个问题：Atomizer 已经先 `freshVar()` 了，Registry 如果发现这个 atom 已存在，怎么办？

现在有两种合法设计，必须选一个。

## 方案 A：Registry 负责 get-or-create，返回 SAT literal

推荐。

```cpp
SatLit TheoryAtomRegistry::getOrCreateEufAtom(
    const EufAtomPayload& payload,
    ExprId originExpr,
    Atomizer& atomizer);
```

如果已经存在，直接返回已有 SAT var；如果不存在，再 fresh var 并注册。

## 方案 B：允许重复 SAT var，但加等价约束

如果两个 SAT var 表示同一个 EUF atom：

```text
v1 ↔ v2
```

需要加入：

```cpp
sat_.addClause({¬v1, v2});
sat_.addClause({v1, ¬v2});
```

否则 SAT 层可能给同一个 theory atom 两个不同布尔值，虽然 EUF 最后能冲突，但会产生大量无意义 conflict。

V1 建议用方案 A。

# 4. `push/pop` 和 `backtrackToLevel` 组合有隐藏 bug

你现在是：

```cpp
void push() {
    scopeLimits_.push_back(trail_.size());
}

void pop(uint32_t n) {
    trail_.resize(scopeLimits_.back());
    scopeLimits_.pop_back();
}

void backtrackToLevel(int level) {
    remove_if(trail_, level > target);
}
```

问题是：`backtrackToLevel()` 可能把 `trail_` 缩短到比某些 `scopeLimits_` 更小。之后 `pop()` 再：

```cpp
trail_.resize(scopeLimits_.back());
```

就可能把 trail 重新扩张，产生默认构造的垃圾 assignment。

例如：

```text
push: scopeLimits = [10]
CDCL backtrack: trail size 变成 5
pop: resize(10)  // 错
```

建议改成：

```cpp
void EufSolver::backtrackToLevel(int level) {
    trail_.erase(
        std::remove_if(trail_.begin(), trail_.end(),
            [level](const auto& a) { return a.level > level; }),
        trail_.end());

    for (auto& lim : scopeLimits_) {
        lim = std::min(lim, trail_.size());
    }
}
```

更干净的是分开两类 trail：

```text
user assertion scope trail
CDCL assignment trail
```

V1 可以先用 `min(limit, trail_.size())` 修掉。

# 5. explanation 不能在失败时静默返回空原因

你现在的逻辑里：

```cpp
if (visitedPairs.count(pair)) return {};
```

以及 BFS 找不到 path 的情况没有明确错误处理。

这很危险。因为返回空 reasons 会导致 conflict clause 缺少必要前提，可能学到过强甚至不正确的子句。

建议 explanation 返回结构改成：

```cpp
struct ExplainResult {
    bool ok;
    std::vector<SatLit> reasons;
};
```

然后：

```cpp
auto r = explainEquality(a, b, tm);
if (!r.ok) {
    return TheoryCheckResult::unknown(); // 或 internal error
}
```

不要让 “解释失败” 退化成 “无需原因”。

另外 `visitedPairs` 最好 canonicalize：

```cpp
auto key = std::minmax(from, to);
```

因为 proof graph 是无向图，`(a,b)` 和 `(b,a)` 应视为同一个 explanation query。

# 6. sort mismatch 不能只是 `return false`

现在：

```cpp
if (tm.node(a).sort != tm.node(b).sort) {
    return false;
}
```

这个不够安全。因为 `return false` 和 “两个 term 已经同类，所以无需 merge” 的语义混在一起了。

建议：

```cpp
enum class MergeStatus {
    Merged,
    AlreadySame,
    SortMismatch
};
```

然后：

```cpp
auto status = egraph_.merge(...);
if (status == MergeStatus::SortMismatch) {
    return TheoryCheckResult::unknown(); // 或 internal error
}
```

否则 sort mismatch equality 可能被静默忽略。

# 7. `UF` logic 不建议注册，先只注册 `QF_UF`

你现在写：

```cpp
} else if (logic == "QF_UF" || logic == "UF") {
```

```cpp
logic == "QF_UF"
```

`UF` 可能包含量词，除非你已经在 parser/solver 层明确禁止 quantifier，否则不要注册。

# 8. `Solver.cpp` 不要对未知 logic 默认注册 LRA

你写的是：

```cpp
} else {
    // Default: LRA (no EUF)
    theoryManager.registerSolver(std::make_unique<LraSolver>());
}
```

这也有风险。比如用户写：

```smt2
(set-logic QF_UFLIA)
```

如果你不支持 mixed theory，正确行为应该是：

```text
unsupported / unknown
```

而不是悄悄走 LRA。

建议改成：

```cpp
if (logic == "QF_UF") {
    register EUF;
} else if (logic == "QF_LRA") {
    register LRA;
} else if (...) {
    ...
} else {
    markUnsupportedLogic(logic);
}
```

尤其是：

```text
QF_UFLIA
QF_UFLRA
QF_UFNIA
QF_UFNRA
```

必须显式 unsupported，不能 fallback。

Relation::Neq

# 10. 建议补充的测试

你已有测试不错，但还缺几个关键 regression。

## Bool false

```smt2
(set-logic QF_UF)
(declare-sort U 0)
(declare-fun p (U) Bool)
(declare-const a U)
(assert (= (p a) false))
(assert (p a))
(check-sat)
; UNSAT
```

## true / false 冲突

```smt2
(set-logic QF_UF)
(assert (= true false))
(check-sat)
; UNSAT
```

## repeated same atom dedup

```smt2
(set-logic QF_UF)
(declare-sort U 0)
(declare-const a U)
(declare-const b U)
(assert (or (= a b) (= a b)))
(assert (distinct a b))
(check-sat)
; SAT or UNSAT depending on first assertion assignment, but registry must not corrupt duplicate atoms
```

更直接：

```smt2
(set-logic QF_UF)
(declare-sort U 0)
(declare-const a U)
(declare-const b U)
(assert (= a b))
(assert (not (= a b)))
(check-sat)
; UNSAT
```

## n-ary atom 不同 pair 不能共享 exprId 出错

```smt2
(set-logic QF_UF)
(declare-sort U 0)
(declare-const a U)
(declare-const b U)
(declare-const c U)
(assert (= a b c))
(assert (distinct b c))
(check-sat)
; UNSAT
```

## unsupported mixed logic

```smt2
(set-logic QF_UFLIA)
(declare-fun f (Int) Int)
(declare-const x Int)
(assert (= (f x) (f x)))
(check-sat)
; should return unknown/unsupported, not silently use LRA
```

# 总体判断

这版可以保留大框架，但还不能直接实现。

最需要改的是这几处：

```text
1. n-ary normalize 不要共用 exprId，也不要无条件注册 EUF；
2. Bool 常量要同时支持 true / false，并维护 true != false；
3. TheoryAtomRegistry 的 dedup API 要改成 get-or-create 或加 SAT 等价约束；
4. push/pop 与 CDCL backtrack 的 trail limit 要防止 resize 回扩；
5. explanation 失败不能返回空 reasons；
6. unsupported logic 不能 fallback 到 LRA。
```

改完这些后，这个 V1 就比较稳了，可以作为 **sound QF_UF baseline** 开始交给实现模型。
