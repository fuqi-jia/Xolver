# Plan: CDCAC P2b–P4 (Algebraic Tower & Mixed Sample Support)

## Context

P2a is complete: univariate CDCAC with exact algebraic root comparison works (214 tests green).
The next stages extend CDCAC to **mixed algebraic samples** (tower of algebraic numbers) and
**algebraic-coefficient root isolation**.

Key libpoly capability discovered during planning:
- `poly::evaluate(Polynomial, Assignment)` supports `AlgebraicNumber` values ✅
- `poly::sgn(Value)` returns exact sign (-1/0/1) ✅
- `poly::isolate_real_roots(Polynomial, Assignment)` supports algebraic assignments ✅
- `poly::to_univariate(Polynomial, Assignment)` is **integer-only** ❌
- `poly::UPolynomial` is **integer-coefficient only** ❌

This means:
- **P3** (tower sign) can be implemented directly via libpoly evaluate — no manual tower reduction needed.
- **P4** (algebraic-coefficient root isolation) can use `isolate_real_roots(Polynomial, Assignment)` directly,
  bypassing the need for algebraic-coefficient `UPolynomial`s.

## Implementation Order

### Phase 1 — P3: Algebraic Tower Sign (`signAt` for multiple algebraic vars)

**Goal**: `LibpolyBackend::signAt` supports samples with >1 algebraic variable.

**Files**:
- `src/theory/arith/nra/LibpolyBackend.cpp` — implement `signAt` algebraic-tower path
- `src/theory/arith/nra/LibpolyBackend.h` — add private helpers

**Steps**:
1. Add `poly::Value realAlgToPolyValue(const RealAlg&)` helper:
   - Rational → `poly::Value(poly::Rational(q))`
   - Algebraic → reconstruct `poly::AlgebraicNumber` from `AlgebraicRoot` (need `UPolynomial` + `DyadicInterval`)
2. Add `poly::Assignment buildAssignment(const SamplePoint&)` helper:
   - For each `(varName, RealAlg)` in sample, resolve `poly::Variable` via `libKernel_->resolveVar()`
   - Set in `poly::Assignment`
3. Modify `signAt`:
   - Current logic: all-rational → `kernel_->sgn`; exactly-one-algebraic → `specializeToUnivariate` + `signUnivariateAtAlgebraic`; >1 algebraic → `Unknown`
   - New logic: if >1 algebraic (or any algebraic), try `poly::evaluate(p, assignment)` + `poly::sgn(result)`
   - If libpoly returns non-zero sign, map to `Sign::Pos`/`Sign::Neg`/`Sign::Zero`
   - Keep existing paths as fallback for when libpoly is unavailable
4. Add unit test: `x^2=2, y^2=3, x+y>0` → sat (multi-algebraic sign)

**Rationale**: This is the highest-impact, lowest-risk change. It immediately unlocks multi-variable algebraic reasoning.

### Phase 2 — P2b: AlgebraicRoot Metadata Extension

**Goal**: Track provenance of algebraic roots (original poly, variable, level).

**Files**:
- `src/theory/arith/nra/CdcacTypes.h` — extend `AlgebraicRoot`
- `src/theory/arith/nra/CdcacCore.cpp` — set metadata in `solveLevel`
- `src/theory/arith/nra/CdcacCore.cpp` — preserve metadata in `mergeRoots`

**Steps**:
1. Extend `AlgebraicRoot`:
   ```cpp
   PolyId liftedDefiningPoly = NullPoly;   // original multivariate polynomial
   std::string mainVar;                     // variable this root belongs to
   int level = -1;                          // index in varOrder
   ```
2. In `solveLevel(k, prefix, input)`:
   - After `isolateRealRoots(up)`, iterate roots and set metadata:
     ```cpp
     for (auto& r : roots.roots) {
         if (r.isAlgebraic()) {
             r.root.liftedDefiningPoly = p;
             r.root.mainVar = var;
             r.root.level = k;
         }
     }
     ```
3. In `mergeRoots`:
   - When dropping a duplicate candidate, ensure the surviving root has metadata (either one is fine)
4. Add unit test: verify metadata is populated after `solveLevel` on an algebraic case

**Rationale**: Infrastructure for P2c/P3/P4; needed for tower construction and conflict explanation.

### Phase 3 — P4: Algebraic-Coefficient Root Isolation

**Goal**: `solveLevel` works when prefix contains algebraic values.

**Files**:
- `src/theory/arith/nra/AlgebraBackend.h` — add `isolateRealRootsAlgebraic`
- `src/theory/arith/nra/LibpolyBackend.h` — implement it
- `src/theory/arith/nra/LibpolyBackend.cpp` — implement it
- `src/theory/arith/nra/CdcacCore.cpp` — use it when `specializeToUnivariate` fails on algebraic prefix

**Steps**:
1. Add interface to `AlgebraBackend`:
   ```cpp
   virtual RootSet isolateRealRootsAlgebraic(
       PolyId p, const SamplePoint& prefix, const std::string& mainVar) = 0;
   ```
2. Implement in `LibpolyBackend`:
   - Build `poly::Assignment` from prefix (same as P3)
   - Call `poly::isolate_real_roots(libKernel_->get(p), assignment)` → `std::vector<poly::Value>`
   - For each `Value`:
     - If rational: `RealAlg::fromRational(convertToMpq(value))`
     - If algebraic:
       a. `an = poly::as_algebraic_number(value)`
       b. `defPoly = poly::get_defining_polynomial(an)`
       c. Convert `defPoly` to `std::vector<mpz_class>` (low→high reversed to high→low)
       d. `UniPolyId upId = allocUni(coeffs)`
       e. Extract interval from `an` via `get_lower_bound`/`get_upper_bound`
       f. Re-isolate `upId` to find `rootIndex` (match by interval overlap)
       g. Create `AlgebraicRoot` with metadata (set in caller `solveLevel`)
   - Return `RootSet`
3. Modify `solveLevel`:
   - When `specializeToUnivariate(p, prefix, var)` returns `NullUniPolyId`:
     - Try `algebra_->isolateRealRootsAlgebraic(p, prefix, var)` instead of skipping
   - Keep `validateRootIsolation` call
4. Add unit tests:
   - `x^2=2, y^2=x` → sat (y = ±2^(1/4))
   - `x^2=2, y^2=x, y>0` → sat
   - `x^2=2, y^2=x, y<0` → sat
   - `x^2=2, y^2=x, y=0` → unsat

**Rationale**: This completes the multivariate CDCAC pipeline. Once P3 and P4 are in place, CDCAC can handle arbitrary QF_NRA formulas with algebraic solutions (e.g., x²−2=0, y²−x=0).

## Acceptance Criteria

After all phases:
- `signAt` must support arbitrary tower depth (test: 3+ algebraic variables)
- `solveLevel` must successfully isolate roots when prefix contains algebraic values
- All 214 existing tests remain green
- New tests added for:
  - Multi-algebraic sign evaluation (P3)
  - Algebraic-coefficient root isolation (P4)
  - Metadata presence (P2b)

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| `poly::evaluate` on complex algebraic towers is slow | Cache assignment objects; timeout guard |
| `poly::isolate_real_roots` with algebraic assignment returns `Value` not `AlgebraicNumber` | Implement robust `Value → RealAlg` conversion (Phase 3 step 2) |
| `poly::AlgebraicNumber` reconstruction from `AlgebraicRoot` is fragile | Use `UPolynomial` + `DyadicInterval` constructor; validate with `sgn` |
| `mergeRoots` metadata loss on dedup | Ensure metadata is copied from either root when equal |
