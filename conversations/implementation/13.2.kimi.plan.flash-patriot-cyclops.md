# Phase 3: Interval Evaluation + Pruning (Revised)

## 关键修正

**上一版计划存在 soundness 风险：角点采样不是 over-approx，会导致 false UNSAT。**

反例：
```
x ∈ [-2, 2]
constraint: x² - 1 <= 0
```
角点采样：x=-2→3, x=2→3，min=3>0 → 误判 violated（false UNSAT）。
实际上 x=0 时 p(0)=-1<=0，约束可满足。

**因此：Phase 3 必须基于真正的 interval arithmetic，绝不使用采样来判断 UNSAT。**

---

## 目标

构建 IntervalZ + 单变量多项式 IntervalEvaluator，在 NiaSolver check 管道中加入一轮 **sound over-approx interval evaluation**。

Phase 3 明确**不做**多变量 interval evaluation（需要 PolynomialKernel::terms() 接口扩展，放到 Phase 4+）。

## 设计

### 3.1 IntervalZ 数据结构

```cpp
struct IntervalZ {
    mpz_class lo;  // inclusive lower bound
    mpz_class hi;  // inclusive upper bound
    bool isEmpty() const { return lo > hi; }
};
```

### 3.2 单变量多项式 IntervalEvaluator

**只处理单变量多项式**（基于 `PolynomialKernel::getIntegerCoefficients`）。多变量 → NoChange。

**核心操作（sound over-approx）：**

| 操作 | 规则 |
|------|------|
| `add` | `[a,b] + [c,d] = [a+c, b+d]` |
| `sub` | `[a,b] - [c,d] = [a-d, b-c]` |
| `neg` | `-[a,b] = [-b,-a]` |
| `mul` | `[a,b] * [c,d] = [min(ac,ad,bc,bd), max(ac,ad,bc,bd)]` |
| `pow(even, 0∈[l,u])` | `[0, max(l^k, u^k)]` |
| `pow(even, 0∉[l,u])` | `[min(l^k, u^k), max(l^k, u^k)]` |
| `pow(odd)` | `[l^k, u^k]` |

**多项式 evaluation：**
- 系数数组 `[a_n, a_{n-1}, ..., a_0]` 表示 `a_n·x^n + ... + a_1·x + a_0`
- 对每个 monomial `a_i·x^i`，计算 `intervalMul([a_i,a_i], intervalPower([x], i))`
- 所有 monomial interval 相加得到最终 interval

**definitely violated 判断：**

| rel | violated iff |
|-----|-------------|
| `<=` | `interval.lo > 0` |
| `>=` | `interval.hi < 0` |
| `=` | `0 ∉ [lo,hi]`（即 lo > 0 或 hi < 0）|
| `!=` | `[lo,hi] == [0,0]` |

**注意：** interval arithmetic 的 over-approx 只会导致 interval 变宽（lo 更小、hi 更大），因此 `interval.lo > 0` 意味着真正的最小值 > 0，约束对所有赋值都 violated。这是 **sound** 的。

**如果变量没有 bound：** 跳过 interval evaluation（无限区间无法安全处理）。

### 3.3 不处理的情况（返回 NoChange）

- 多变量多项式
- 单变量但 `getIntegerCoefficients` 失败
- 变量没有 lower 或 upper bound
- 不是 definitely violated（这是绝大多数情况，正常）

### 3.4 禁止

- **角点采样**（corner sampling）用于 UNSAT 判断
- **反向传播**（从 constraint interval 反推变量 bound）
- **Product pruning**
- **多变量 interval evaluation**（Phase 3 范围外）

### 3.5 接入位置

放在 `AlgebraicIntegerReasoner` 之后、`EmptyDomain` 之前：

```
... → algebraic → intervalEvaluator → emptyDomain → bounded → ...
```

### 3.6 测试

**必须包含的 soundness 测试：**
- `x ∈ [-2,2], x² - 1 <= 0` → interval = `[-1, 3]` → **不能判断 violated**（hi=3>0, lo=-1<0）→ NoChange ✓
- `x ∈ [0,2], x³ + 1 <= 0` → interval = `[1, 9]` → lo=1>0 → **violated** → UNSAT ✓
- `x ∈ [0,2], x³ - 8 <= 0` → interval = `[-8, 0]` → hi=0<=0（satisfied），lo=-8<0（不 violated）→ NoChange ✓

**端到端 SMT2：**
- `x >= 0, x³ + 1 <= 0` → UNSAT（interval `[1,∞)` 的 lo>0）
- `0 <= x <= 2, x³ - 8 <= 0` → SAT（BoundedNiaSolver 枚举或 LocalSearch 找到 x=2）

## 正确性红线

1. **Interval evaluation 必须是真正的 over-approx，不是采样。**
2. **绝不使用角点采样、端点采样或任何 sampling 来判断 UNSAT。**
3. **多变量多项式 Phase 3 不处理（返回 NoChange）。**
4. **不做反向传播。**
5. **No incomplete reasoning may return UNSAT。**
