# Plan: QF_NIA Phase NIA-1 — Polynomial Payload + NiaSolver Shell

## Goal
Unblock the QF_NIA end-to-end pipeline:
- `Atomizer` extracts polynomial constraints for QF_NIA (same payload as NRA).
- `NiaSolver::check()` handles **trivial constant polynomial constraints** exactly.
- Non-constant constraints return `Unknown` (sound).
- No more `unsupportedTheorySeen_` blocking on every NIA atom.

This is **Phase NIA-1** from the canonical design doc (§25).

---

**Blocker**: `Solver.cpp` has no `QF_NIA` branch → `NiaSolver` is never registered, and `Atomizer` has no `defaultTheory_ == NIA` path.

---

## NIA Theory Boundary (per design doc §0–§1)

`QF_NIA` is **quantifier-free nonlinear integer arithmetic**. General case is **undecidable**.

`NiaSolver` result semantics:
- **Sat**: exact integer assignment, all active constraints pass exact evaluation.
- **Unsat**: sound conflict / sound finite-domain exhaustion / sound relaxation unsat.
- **Unknown**: budget exhausted, unsupported fragment, or unbounded nonlinear case.

Phase NIA-1 **must not** claim completeness. It only unblocks the pipeline + handles constants.

---

## Approach

Single bounded phase: implement NIA-1 exactly per design doc §25.
No alternatives needed — scope is clearly bounded (unblock pipeline + trivial constants + strict normalization).

#include "theory/arith/poly/PolynomialKernel.h"
#include "theory/arith/poly/PolynomialConverter.h"
#include <gmpxx.h>
#include <vector>
#include <memory>
#include <optional>

class NiaSolver : public TheorySolver {
public:
    explicit NiaSolver(std::unique_ptr<PolynomialKernel> kernel);

    TheoryId id() const override { return TheoryId::NIA; }

    void push() override;

    void reset() override;

private:
    struct ActiveNiaConstraint {
        PolynomialId poly;
        Relation rel;       // effective relation
        SatLit reason;
    };

    struct NiaTrailEntry {
        int level;
        size_t activeSizeBefore;
    };

    struct PendingConflict {
        int level;
        TheoryConflict conflict;
    };

        int level;

    std::unique_ptr<PolynomialConverter> converter_;

    std::vector<ActiveNiaConstraint> active_;
    std::vector<NiaTrailEntry> trail_;
    std::optional<PendingConflict> pendingConflict_;
    std::optional<PendingUnknown> pendingUnknown_;

#### `assertLit()` — per design doc §5
```cpp
void NiaSolver::assertLit(const TheoryAtomRecord& atom, bool value,
                          int level, SatLit reason) {
    const auto* payload = std::get_if<PolynomialAtomPayload>(&atom.payload);
    if (!payload) {
        pendingUnknown_ = PendingUnknown{level};
        return;
    }

    size_t oldSize = active_.size();
    Relation rel = value ? payload->rel : negateRelation(payload->rel);

    active_.push_back({payload->poly, rel, reason});
    trail_.push_back({level, oldSize});
}
```

#### `backtrackToLevel()` — per design doc §6
```cpp
void NiaSolver::backtrackToLevel(int level) {
    while (!trail_.empty() && trail_.back().level > level) {
        active_.resize(trail_.back().activeSizeBefore);
        trail_.pop_back();
    }
    if (pendingConflict_ && pendingConflict_->level > level) {
        pendingConflict_.reset();
    }
    if (pendingUnknown_ && pendingUnknown_->level > level) {
        pendingUnknown_.reset();
    }
}
```

#### `check()` — trivial constant handling per design doc §9

Phase NIA-1 **does not** implement local search, relaxation, modular reasoning, or branch split. It only handles trivial constants.

```cpp
TheoryCheckResult NiaSolver::check(TheoryLemmaDatabase& /*lemmaDb*/) {
    if (pendingConflict_) {
        return TheoryCheckResult::mkConflict(pendingConflict_->conflict);
    }

    if (pendingUnknown_) {
        return TheoryCheckResult::unknown();
    }

    if (active_.empty()) {
        return TheoryCheckResult::consistent();
    }

    std::vector<SatLit> conflictLits;
    bool hasNonConstant = false;

    for (const auto& c : active_) {
        if (!kernel_->isConstant(c.poly)) {
            hasNonConstant = true;
            continue;
        }
        mpq_class val = kernel_->toConstant(c.poly);
        if (!relationSatisfied(val, c.rel)) {
            conflictLits.push_back(c.reason.negated());
        }
    }

    if (!conflictLits.empty()) {
        return TheoryCheckResult::mkConflict(TheoryConflict{conflictLits});
    }

    if (!hasNonConstant) {
        return TheoryCheckResult::consistent();
    }

    return TheoryCheckResult::unknown();
}
```

#### `relationSatisfied()`
```cpp
bool NiaSolver::relationSatisfied(const mpq_class& val, Relation rel) const {
    switch (rel) {
        case Relation::Eq:  return val == 0;
        case Relation::Neq: return val != 0;
        case Relation::Lt:  return val <  0;
        case Relation::Leq: return val <= 0;
        case Relation::Gt:  return val >  0;
        case Relation::Geq: return val >= 0;
    }
    return false;
}
```

#### `reset()`
```cpp
void NiaSolver::reset() {
    active_.clear();
    trail_.clear();
    pendingConflict_.reset();
    pendingUnknown_.reset();
}
```

### 5. `tests/unit/test_nia.cpp` (new, ~120 lines)

Eight test cases:

**Test 1 — Trivial constant unsat**: `(assert (= 1 0))` → `Result::Unsat`
- Atomizer extracts constant polynomial `Eq 0`.
- NiaSolver `check()` sees constant, val=1, Eq unsatisfied → conflict.

**Test 2 — Trivial constant sat**: `(assert (> 1 0))` → `Result::Sat`
- Constant polynomial `1`, relation `Gt` → satisfied.

**Test 3 — Non-constant returns Unknown**: `(assert (= (* x x) 2))` → `Result::Unknown`
- Atomizer extracts polynomial payload (not blocked).
- NiaSolver `check()` sees non-constant → `Unknown`.

**Test 4 — False literal negation**: `(assert (not (> 1 0)))` → `Result::Unsat`
- `not (> 1 0)` becomes `<=`; constant `1 <= 0` is false → conflict.

**Test 5 — Distinct constant unsat**: `(assert (distinct 1 1))` → `Result::Unsat`
- `distinct 1 1` maps to `Neq`; constant `1 != 1` is false → conflict.

**Test 6 — Eq negation constant unsat**: `(assert (not (= 1 1)))` → `Result::Unsat`
- `not (= 1 1)` becomes `Neq`; constant `1 != 1` is false → conflict.

**Test 7 — Non-integer rational constant**: `(assert (= (/ 1 2) 0))` → `Result::Unknown`
- `PolynomialConverter` rejects non-integer rational → unsupported → `Unknown`.

**Test 8 — Linear NIA atom**: `(assert (>= x 0))` in QF_NIA → `Result::Unknown`
- Linear atom goes through polynomial path (not `extractLinearConstraint`).
- NiaSolver sees non-constant → `Unknown`.
- Guards against linear payload orphaning in QF_NIA.

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| `PolynomialConverter` fails on division / non-integer exponents | Mark unsupported; sound |
| `PolynomialConverter` rejects non-integer rational constants | Returns `Unknown` (sound) until rational coefficient handling is fixed in NIA-2+ |
| `PolynomialId` cross-kernel misuse | **Atomizer and NiaSolver share the same `PolynomialKernel` instance**; `Atomizer` holds a raw borrow, `NiaSolver` holds `unique_ptr` ownership. No second instance. |
| SAT var routing to wrong solver | `registerParsedTheoryAtom` receives explicit `TheoryId::NIA`; `TheoryManager` dispatches by `record.theory` |
| Payload mismatch in `assertLit` | Treated as `pendingUnknown_` (returns `Unknown`), level-tagged like `pendingConflict_`. Safe and sound. |
| Binary-only `distinct` support | Documented limitation; n-ary distinct deferred to NIA-2+. |
| QF_NIA linear atom orphaning | All arithmetic comparisons under QF_NIA go through `extractPolynomialConstraint`, registered as `PolynomialAtomPayload` + `TheoryId::NIA`. No linear payload orphaning. |
