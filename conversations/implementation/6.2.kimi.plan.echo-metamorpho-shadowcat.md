# Plan: Trail-Based CDCL(T) Refactor (Revised)

## Goal
Replace the current scan-all-atoms CDCL(T) loop with a trail-based, incrementally-driven theory integration architecture. The current design scans `atomizer.atoms()` every SAT model, calls `solver->reset()`, and re-asserts all theory literals from scratch. This refactor removes that bottleneck while keeping all 25 existing unit tests passing.

## Core Principle
**All theory atoms — parsed from SMT input or dynamically created — must live in `TheoryAtomRegistry`. `TheoryBridge` uses *only* `registry.findBySatVar()` to route SAT assignments to theory solvers. No `Atomizer::atoms()` scan remains in the CDCL(T) path.**

---

## Phase 1: Unified Atom Registration (`TheoryAtomRegistry`)

### 1.1 `TheoryAtomRecord` (new struct)
```cpp
struct TheoryAtomRecord {
    SatVar satVar;
    TheoryId theory;
    bool isDynamic;
    ExprId exprId;        // original parsed expr or synthetic id
    LinearFormKey lhs;    // empty for non-arithmetic atoms
    Relation rel;
    mpq_class rhs;
};
```

For parsed theory atoms, `lhs/rel/rhs` are populated eagerly during `atomize()` so `assertLit` never re-parses CoreIr.

### 1.2 `TheoryAtomRegistry` internal storage
Replace `std::vector<AtomRecord> records_` with:
```cpp
std::vector<TheoryAtomRecord> records_;           // dense index
std::unordered_map<SatVar, size_t> satVarToIdx_;  // O(1) lookup
std::unordered_map<LookupKey, size_t, LookupKeyHash> lookup_;
```

Add:
```cpp
const TheoryAtomRecord* findBySatVar(SatVar v) const;
```

### 1.3 `Atomizer::atomize()` must register parsed theory atoms
In `Atomizer::atomizeRec()`, when a theory atom is detected (`Eq`, `Lt`, `Leq`, `Gt`, `Geq`, `Distinct`), after allocating `SatVar v`:
```cpp
// Extract normalized lhs/rel/rhs via LinearAtomManager helper
LinearFormKey lhs;
Relation rel;
mpq_class rhs;
if (extractLinearConstraint(eid, ir, lhs, rhs, rel)) {
    registry.registerParsedTheoryAtom(v, eid, theoryId, lhs, rel, rhs);
}
```

Dynamic atoms continue to use:
```cpp
registry.getOrCreateLinearBoundAtom(lhs, rel, rhs, theory);
```

Both paths populate `satVarToIdx_` so `findBySatVar` works for all atoms.

### 1.4 `Atomizer::atoms()` is deprecated for CDCL(T)
It may remain for debugging or legacy paths, but `Solver::checkSatInternal` no longer uses it.

## Phase 2: SAT Assignment Listener Interface

### 2.1 `SatAssignmentListener`
```cpp
class SatAssignmentListener {
public:
    virtual ~SatAssignmentListener() = default;

    // Signed literal semantics:
    //   onAssign(positive(var), level) means var is assigned TRUE.
    //   onAssign(negative(var), level) means var is assigned FALSE.
    virtual void onAssign(SatLit lit, int decisionLevel) = 0;
    virtual void onBacktrack(int targetLevel) = 0;
};
```

### 2.2 `SatSolver` gains
```cpp
virtual void setAssignmentListener(SatAssignmentListener* listener) = 0;
```

### 2.3 `CadicalBackend` temporary adapter (explicitly marked)
`CadicalBackend::solve()` will simulate assignment events because the current CaDiCaL version does not expose fine-grained callbacks.

```cpp
SolveResult CadicalBackend::solve() {
    int res = solver_->solve();
    if (res != CaDiCaL::SATISFIABLE) { ... }

    if (listener_) {
        // Temporary model-delta simulation.
        // Final architecture requires true assignment/backtrack events.
        listener_->onBacktrack(0);
        for (SatVar v = 1; v <= maxVar_; ++v) {
            int val = solver_->val(static_cast<int>(v));
            if (val > 0) listener_->onAssign(SatLit::positive(v), 0);
            else         listener_->onAssign(SatLit::negative(v), 0);
        }
    }
    return SolveResult::Sat;
}
```

**This is a compatibility adapter, not the final implementation.**
The architecture must isolate this behind `SatAssignmentListener`. When the backend is upgraded to one with native callbacks, only `CadicalBackend` changes.

A **fake callback-capable SAT backend** must be added in `tests/` to validate true incremental behavior without the full replay cost.

## Phase 3: `TheoryBridge` and `TheoryManager`

### 3.1 `TheoryBridge` (new)
```cpp
class TheoryBridge : public SatAssignmentListener {
public:
    TheoryBridge(TheoryAtomRegistry& registry, TheoryManager& tm)
        : registry_(registry), tm_(tm) {}

    void onAssign(SatLit lit, int level) override {
        const auto* atom = registry_.findBySatVar(lit.var());
        if (!atom) return;
        tm_.assertTheoryLit(*atom, lit, level);
    }

    void onBacktrack(int targetLevel) override {
        tm_.backtrackToLevel(targetLevel);
    }
};
```

### 3.2 `TheoryManager` new interface
Remove:
```cpp
TheoryCheckResult check(const CoreIr& ir,
                        const std::vector<Atomizer::AtomRecord>& atoms,
                        const SatSolver& sat);
```

Add:
```cpp
void assertTheoryLit(const TheoryAtomRecord& atom, SatLit assignedLit, int level);
void backtrackToLevel(int level);
TheoryCheckResult check(TheoryLemmaDatabase& lemmaDb);
```

`assertTheoryLit` routes to the correct solver by `atom.theory`.
`backtrackToLevel` calls `solver->backtrackToLevel(level)` for all solvers.
`check()` calls `solver->check(lemmaDb)` for all solvers.

## Phase 4: `TheorySolver` New Interface

```cpp
class TheorySolver {
public:
    virtual ~TheorySolver() = default;
    virtual TheoryId id() const = 0;

    // Scope management (for Solver::push/pop API only, not CDCL loop)
    virtual void push() = 0;
    virtual void pop(uint32_t n) = 0;

    // Incremental assertion from SAT trail
    virtual void assertLit(const TheoryAtomRecord& atom,
                           bool value,
                           int level,
                           SatLit reason) = 0;

    // Backtrack to target decision level
    virtual void backtrackToLevel(int level) = 0;

    // Check current incremental state
    virtual TheoryCheckResult check(TheoryLemmaDatabase& lemmaDb) = 0;

    // Reset ONCE per fresh check-sat call (clears all incremental state)
    virtual void reset() = 0;
};
```

Key change: `check()` takes `TheoryLemmaDatabase&` so theory solvers can query whether a candidate lemma was already emitted, and skip it *before* returning.

---

## Phase 5: `TheoryLemmaDatabase` (new)

```cpp
class TheoryLemmaDatabase {
public:
    bool insertIfNew(const TheoryLemma& lemma);
    bool contains(const TheoryLemma& lemma) const;

private:
    std::unordered_set<uint64_t> emitted_;
    static uint64_t computeKey(const TheoryLemma& lemma);
};
```

- Lives in `Solver::checkSatInternal`, **survives the entire check-sat call**.
- Never cleared between SAT models.
- `TheorySolver::check(lemmaDb)` uses it to skip duplicate lemmas internally.
- `Solver` layer adds the clause only if `insertIfNew` returns true.
- If a theory solver returns a duplicate lemma, the `Solver` layer treats it as an error (returns `Unknown`) rather than looping.

## Phase 6: `GeneralSimplex` Level-Aware Backtracking

Current `push()/pop()` use a stack of bound-update vectors. We need `backtrackToLevel(int level)`.

### 6.1 `TrailEntry` upgrade
```cpp
struct TrailEntry {
    int level;
    int var;
    bool isLower;
    BoundInfo oldBound;
    DeltaRational oldBeta;
};
```

### 6.2 `assertLower/Upper` gain `int level`
```cpp
bool assertLower(int var, const BoundInfo& info, int level);
bool assertUpper(int var, const BoundInfo& info, int level);
```

Each stores `level` in the `TrailEntry`.

### 6.3 `backtrackToLevel(int level)`
```cpp
void GeneralSimplex::backtrackToLevel(int level) {
    while (!trail_.empty() && trail_.back().level > level) {
        auto& e = trail_.back();
        if (e.isLower) vars_[e.var].lower = e.oldBound;
        else           vars_[e.var].upper = e.oldBound;
        vars_[e.var].beta = e.oldBeta;
        trail_.pop_back();
    }
    betaDirty_ = true;
    conflict_.clear();
    hasImmediateConflict_ = false;
}
```

### 6.4 Deprecate `push()/pop()` for CDCL loop
Keep for `Solver::push()/pop()` legacy API, but the CDCL loop uses `backtrackToLevel`.

## Phase 7: `LiaSolver` Incremental Refactor

### 7.1 Internal trail
`LiaSolver` trails only `activeAtoms_` and `disequalities_`. **Bound undo is delegated to `GeneralSimplex::backtrackToLevel()`** — no double bookkeeping.

```cpp
struct LiaTrailEntry {
    int level;
    size_t activeAtomsSize;
    size_t disequalitiesSize;
};

std::vector<LiaTrailEntry> trail_;
```

### 7.2 `assertLit`
```cpp
void LiaSolver::assertLit(const TheoryAtomRecord& atom, bool value,
                          int level, SatLit reason) {
    ensureLevel(level);

    int aux = manager_.getOrCreateAuxVar(gs_, atom.lhs, atom.rhs);

    if (isDisequality(atom.rel, value)) {
        disequalities_.push_back({aux, atom.lhs, atom.rhs, reason});
        trail_.push_back({level, activeAtoms_.size(), disequalities_.size()});
        return;
    }

    bool immediateConflict = manager_.assertBound(gs_, aux, atom.rel, value, reason, level);
    if (immediateConflict) {
        // Store immediate conflict for check() to report
        pendingConflict_ = manager_.buildImmediateConflict(gs_);
        return;
    }

    activeAtoms_.push_back({atom.exprId, aux, atom.rel, value,
                            atom.lhs, atom.rhs, reason});
    trail_.push_back({level, activeAtoms_.size(), disequalities_.size()});
}
```

### 7.3 `backtrackToLevel`
```cpp
void LiaSolver::backtrackToLevel(int level) {
    while (!trail_.empty() && trail_.back().level > level) {
        auto& e = trail_.back();
        activeAtoms_.resize(e.activeAtomsSize);
        disequalities_.resize(e.disequalitiesSize);
        trail_.pop_back();
    }
    gs_.backtrackToLevel(level);
}
```

### 7.4 `check(TheoryLemmaDatabase&)`
```cpp
TheoryCheckResult LiaSolver::check(TheoryLemmaDatabase& lemmaDb) {
    if (pendingConflict_) return mkConflict(*pendingConflict_);

    auto r = gs_.check();
    if (r == Unsat) return mkConflict(manager_.translateConflict(gs_));
    if (r == Unknown) return unknown();

    // 1. Disequality
    auto dr = handleDisequalities(lemmaDb);
    if (dr.kind != Consistent) return dr;

    // 2. Integrality check FIRST
    auto ir = checkIntegrality();
    if (ir.kind == Consistent) {
        // Integral model — validate before returning SAT
        if (validator_.validateLiaModel(...)) {
            return consistent();
        }
        return unknown();
    }

    // 3. Fractional: cheap integer reasoning
    if (!activeAtoms_.empty()) {
        auto cr = integerReasoner_.run(activeAtoms_);
        if (cr) {
            if (cr->kind == Lemma && cr->lemmaOpt) {
                if (lemmaDb.insertIfNew(*cr->lemmaOpt)) return *cr;
                // Duplicate — skip and fall through
            } else {
                return *cr;
            }
        }
    }

    // 4. Branch split
    if (ir.lemmaOpt) {
        if (lemmaDb.insertIfNew(*ir.lemmaOpt)) return ir;
    }

    return unknown();
}
```

### 7.5 `reset()`
```cpp
void LiaSolver::reset() {
    activeAtoms_.clear();
    disequalities_.clear();
    trail_.clear();
    pendingConflict_ = std::nullopt;
    gs_.resetActiveBounds();
    manager_.resetBoundReasons();
    // NOTE: emittedLemmas_ removed — deduplication is now in TheoryLemmaDatabase.
}
```

`reset()` is called **once per fresh check-sat initialization**, never between SAT models.

## Phase 8: `LraSolver` Migration

Same pattern as `LiaSolver`:
- Incremental `assertLit` with trail.
- `backtrackToLevel` delegates to `GeneralSimplex::backtrackToLevel`.
- `check(TheoryLemmaDatabase&)` uses lemmaDb for dedup.
- No per-check `reset()` in the CDCL loop.

## Phase 9: `CdcacSolver` Path

`CdcacSolver` (NRA) must either:
1. **Fully migrate** to new interface (incremental `assertLit`, `backtrackToLevel`, `check(lemmaDb)`), or
2. **Stay on a legacy scan-based path** if migration is deferred.

Since NRA is not the current focus, **Option 2 is acceptable temporarily**, but the Plan must make it explicit:

```cpp
if (logic == "QF_NRA" || logic == "NRA") {
    // Legacy scan-based path until CdcacSolver is migrated.
    theoryManager_.registerSolver(std::make_unique<CdcacSolver>(...));
    useLegacyCheckPath = true;
}
```

`Solver::checkSatInternal` can branch:
```cpp
if (useLegacyCheckPath) {
    // Old scan-based loop
} else {
    // New trail-based loop with TheoryBridge
}
```

This avoids breaking NRA while LIA/LRA move to the new architecture.

## Phase 10: `Solver::checkSatInternal` Rewrite

```cpp
Result checkSatInternal() {
    if (!ir || ir->assertions().empty()) return Result::Sat;

    sat = createSatSolver();
    Atomizer atomizer(*sat);
    atomRegistry_.setContext(sat.get(), &atomizer);

    theoryManager_.clearSolvers();
    bool useLegacy = false;

    if (logic == "QF_LIA" || logic == "LIA") {
        auto lia = std::make_unique<LiaSolver>();
        lia->setRegistry(&atomRegistry_);
        theoryManager_.registerSolver(std::move(lia));
    } else if (logic == "QF_LRA" || logic == "LRA") {
        auto lra = std::make_unique<LraSolver>();
        lra->setRegistry(&atomRegistry_);
        theoryManager_.registerSolver(std::move(lra));
    } else if (logic == "QF_NRA" || logic == "NRA") {
        theoryManager_.registerSolver(
            std::make_unique<CdcacSolver>(createPolynomialKernel()));
        useLegacy = true;
    } else {
        // Fallback
        theoryManager_.registerSolver(std::make_unique<LraSolver>());
        theoryManager_.registerSolver(
            std::make_unique<CdcacSolver>(createPolynomialKernel()));
        useLegacy = true;
    }

    // Atomize assertions (registers parsed theory atoms into registry)
    for (ExprId a : ir->assertions()) {
        SatLit lit = atomizer.atomize(a, *ir);
        sat->addClause({lit});
    }

    if (useLegacy) {
        return runLegacyCdclLoop(atomizer);
    }

    // ---- Trail-based CDCL(T) loop ----
    TheoryBridge bridge(atomRegistry_, theoryManager_);
    sat->setAssignmentListener(&bridge);
    TheoryLemmaDatabase lemmaDb;

    while (true) {
        auto sr = sat->solve();
        if (sr == Unsat) return Result::Unsat;
        if (sr == Unknown) return Result::Unknown;

        auto tr = theoryManager_.check(lemmaDb);

        if (tr.kind == Consistent) return Result::Sat;

        if (tr.kind == Conflict) {
            sat->addClause(tr.conflict.clause);
            continue;
        }

        if (tr.kind == Lemma) {
            if (lemmaDb.insertIfNew(*tr.lemma)) {
                sat->addClause(tr.lemma->lits);
            } else {
                // Duplicate lemma without new clause → potential loop
                return Result::Unknown;
            }
            continue;
        }

        return Result::Unknown;
    }
}
```

## Phase 11: No-Op Lemma Guards

All theory lemma generators must skip:

1. **Tautology**: `tightenedLit == lit.negated()`.
2. **Self-tightening**: `tightenedLit.var == lit.var` (canonical registry returns same atom).
3. **Duplicate**: query `lemmaDb.contains(candidate)` before returning.
4. **Already satisfied**: deferred until `SatAssignmentView` is available.

## Testing Strategy

2. Add `FakeSatBackend` (implements `SatSolver` with true callbacks) in `tests/fakes/` to validate the trail-based interface without Cadical replay.
3. Add `test_general_simplex_trail.cpp`: bound assertion, backtrack, multi-level.

5. Add `test_lemma_dedup.cpp`: duplicate lemma prevention.

## Risks and Mitigations

| Risk | Mitigation |
|------|------------|
| `CadicalBackend` replay is O(#vars) | Temporary; architecture isolates it behind `SatAssignmentListener`. Fake backend tests validate true incremental interface. |
| `GeneralSimplex` backtrack bug | Extensive unit tests on bound trail; `debugCheckInvariants` after every backtrack. |
| Parsed atoms not registered | Assert in `TheoryBridge::onAssign` if a theory-var lookup fails; crash early. |
| Duplicate lemma loop | `Solver` returns `Unknown` on duplicate lemma; `TheorySolver` queries `lemmaDb` before returning. |
| NRA legacy path divergence | Explicit `useLegacy` branch in `checkSatInternal`; NRA tests run on legacy path until migrated. |
