# 总体架构

`NiaSolver::check()` 不要只扫 constants。下一版应该变成：

```cpp
TheoryCheckResult NiaSolver::check(TheoryLemmaDatabase& lemmaDb) {
    if (pendingUnknown_) return unknown();
    if (pendingConflict_) return mkConflict(pendingConflict_->conflict);
    if (active_.empty()) return consistent();

    auto normalized = normalizer_.normalize(active_);
    if (!normalized) return unknown();

    // 1. 常量矛盾
    if (auto r = trivialReasoner_.check(*normalized)) return *r;

    // 2. 一元多项式推理：x^2=4, x^2=2, x^2+1=0
    if (auto r = univariateReasoner_.run(*normalized, lemmaDb)) return *r;

    // 3. 平方/因式/GCD/模推理
    if (auto r = algebraicReasoner_.run(*normalized, lemmaDb)) return *r;

    // 4. 边界推导
    auto domains = boundInferer_.infer(*normalized, lemmaDb);

    // 5. 有限域完备搜索
    if (domains.allFinite()) {
        return boundedSolver_.solve(*normalized, domains, lemmaDb);
    }

    // 6. local search 找 SAT，只能 exact validate 后返回 sat
    if (auto model = localSearch_.tryFindModel(*normalized, domains)) {
        if (validator_.validate(*model, *normalized)) {
            currentModel_ = *model;
            return consistent();
        }
    }

    // 7. branch / relaxation / unknown
    if (auto lemma = brancher_.tryBuildUsefulSplit(*normalized, domains, lemmaDb)) {
        return mkLemma(*lemma);
    }

    return unknown();
}
```

这不是“小步推进”，而是把 NIA solver 的核心骨架一次性补上。

# 1. NiaNormalizer：统一整数多项式形式

当前 Phase NIA-1 先把非整数 rational constant 返回 Unknown，这可以短暂保留。但下一步应该修掉 rational 系数问题，全部转成整数系数多项式。

输入统一成：

```text
p(x) rel 0
```

其中 `p` 是整数系数多项式，`rel ∈ {=, !=, <=, >=}`。

严格不等式在整数语义下转成非严格：

```text
p < 0   ==>   p <= -1
p > 0   ==>   p >= 1
```

有理系数清分母：

```text
(1/2)x^2 + (3/2)y - 5 <= 0
```

乘以正的 lcm：

```text
x^2 + 3y - 10 <= 0
```

类设计：

```cpp
struct NormalizedNiaConstraint {
    PolynomialId poly;   // integer coefficients
    Relation rel;        // Eq, Neq, Leq, Geq
    SatLit reason;
};

class NiaNormalizer {
public:
    std::optional<std::vector<NormalizedNiaConstraint>>
    normalize(const std::vector<ActiveNiaConstraint>& active);
};
```

如果遇到不能清分母、变量除法、非整数指数、非 polynomial term，返回 `Unknown`。

# 2. IntegerModelValidator：所有 SAT 必须 exact validate

下一步必须有统一 validator，后面 local search、bounded search、univariate solver 都用它。

```cpp
class IntegerModelValidator {
public:
    bool validate(
        const IntegerModel& model,
        const std::vector<NormalizedNiaConstraint>& constraints
    ) const;
};
```

实现：

```cpp
bool validate(...) {
    for (const auto& c : constraints) {
        mpz_class v = kernel_.evalInteger(c.poly, model);
        if (!relationSatisfied(v, c.rel)) {
            return false;
        }
    }
    return true;
}
```

任何 SAT 结论都必须经过它。

# 3. UnivariateIntegerReasoner：必须解决 `x^2=4`

这是下一步最关键的模块。不要等 local search，也不要等完整 CDCAC。`x^2=4` 这种一元整数方程必须由专门模块直接处理。

## 3.1 识别一元约束

对于每个 normalized constraint，计算变量集合：

```cpp
std::set<VarId> vars = kernel_.variables(c.poly);
```

如果：

```text
vars.size() == 1
```

就是一元约束。

例如：

```text
x^2 - 4 = 0
x^2 - 2 = 0
x^2 + 1 = 0
x^2 <= 9
```

---

## 3.2 一元等式 `p(x)=0`

对于整数系数一元多项式：

```text
p(x) = a_n x^n + ... + a_0
```

整数根必须满足 rational root theorem：

```text
integer root r divides constant term a_0
```

特殊情况：

```text
a_0 = 0
```

则 `x=0` 是根，之后把 `x` 因子除掉继续。

算法：

```cpp
IntegerRootResult findIntegerRoots(UnivariatePolynomial p) {
    roots = {};

    while (p(0) == 0) {
        roots.insert(0);
        p = p / x;
    }

    if (degree(p) == 0) {
        if (p == 0) return AllIntegers;
        return roots;
    }

    divisors = allDivisors(abs(constantTerm(p)));

    for (d : divisors) {
        for (r : {d, -d}) {
            if (p(r) == 0) roots.insert(r);
        }
    }

    return roots;
}
```

然后：

```text
p(x)=0
```

如果 `roots` 为空，则直接 UNSAT，conflict 是：

```text
¬reason
```

如果 `roots` 非空，则该约束把变量 domain 限制成 finite set：

```text
x ∈ roots
```

例子：

```text
x^2 - 4 = 0
```

得到：

```text
x ∈ {-2, 2}
```

于是可以进入有限域搜索，最终返回 SAT。

```text
x^2 - 2 = 0
```

没有整数根，直接 UNSAT。

```text
x^2 + 1 = 0
```

没有整数根，直接 UNSAT。

---

## 3.3 一元 disequality `p(x)!=0`

如果 `p` 没有整数根，那么：

```text
p(x) != 0
```

恒真，可以忽略。

如果有有限整数根 `{r1,...,rk}`，则记录 excluded values：

```text
x ∉ {r1,...,rk}
```

这有助于 finite-domain search。

---

## 3.4 一元 inequality

先实现常见强规则，不必一开始做完整一元 CAD。

### 平方上界

```text
x^2 <= c
```

如果 `c < 0`，UNSAT。

如果 `c >= 0`：

```text
-floor(sqrt(c)) <= x <= floor(sqrt(c))
```

例如：

```text
x^2 <= 9
```

得到：

```text
-3 <= x <= 3
```

### 平方严格/非严格下界

```text
x^2 < 0
```

UNSAT。

```text
x^2 >= 0
```

恒真。

### 单项幂

```text
x^(2k) <= c
```

也可推 finite bound。

先做 degree 2 square 规则就够覆盖大量例子。

# 4. DomainStore：统一保存整数域

需要一个 domain 管理器，不要到处临时传 set。

```cpp
struct IntDomain {
    bool hasLower = false;
    bool hasUpper = false;

```

组合规则：

```text
finite set ∩ interval ∩ exclusions
```

如果 domain 为空，返回 UNSAT conflict。

# 5. AlgebraicIntegerReasoner：平方、因式、GCD、模推理

这一组规则必须和 univariate 一起上，否则 NIA 只会处理 `x^2=4`，很快又陷入 unknown。

---

## 5.1 平方规则

识别：

```text
x^2
```

规则：

```text
x^2 >= 0      tautology
x^2 < 0       unsat
x^2 <= c      finite bound
x^2 = 0       x = 0
x^2 != 0      x != 0
```

更一般：

```text
x^2 + y^2 <= 0
```

推出：

```text
x = 0
y = 0
```

若：

```text
x^2 + y^2 < 0
```

UNSAT。

---

## 5.2 因式规则

如果 `PolynomialKernel` 能提供 factorization，就做；如果不能，先做 syntactic product recognition。

### Product equality

```text
p*q = 0
```

生成 lemma：

```text
¬(p*q=0) ∨ (p=0) ∨ (q=0)
```

对于 active conjunction，也可以直接在 bounded/domain solver 中利用。

### Product disequality

```text
p*q != 0
```

推出：

```text
p != 0
q != 0
```

生成：

```text
¬(p*q!=0) ∨ (p!=0)
¬(p*q!=0) ∨ (q!=0)
```

这能解决：

```smt2
(assert (= (* x y) 0))
(assert (not (= x 0)))
(assert (not (= y 0)))
```

---

## 5.3 coefficient GCD equality

对于：

```text
Σ c_i * monomial_i = c
```

如果所有 monomial 在整数上都是整数，则左边是：

```text
g = gcd(c_i)
```

的倍数。若：

```text
g ∤ c
```

UNSAT。

例子：

```text
2*x^2 + 4*y*z = 3
```

左边永远偶数，右边奇数，UNSAT。

---

## 5.4 Modular reasoning

这是解决 `x^2=2` 的关键补充。

虽然一元整数根可以处理 `x^2=2`，但多变量场景需要 modular reasoning。

第一版只做 small modulus：

```text
m ∈ {2, 3, 4, 5, 7, 8, 9, 11}
```

对所有 active equalities：

```text
p(x) = 0
```

检查是否存在 residue assignment 使所有 equality 满足。

如果变量数少：

```text
numVars <= 6
```

可以枚举 residues。

如果无 residue solution，则 UNSAT。

例子：

```text
x^2 = 2
```

mod 4 下平方只可能是 0 或 1，无解，UNSAT。

# 6. BoundedNiaSolver：有限域完备搜索

一旦 domain store 中所有变量 finite，就应该进入完备搜索，而不是 unknown。

## 6.1 输入

```cpp
BoundedResult solve(
    const std::vector<NormalizedNiaConstraint>& constraints,
    const DomainStore& domains
);
```

## 6.2 搜索策略

如果 domain size 小：

```text
productSize <= threshold
```

直接枚举。

否则用 interval branch-and-bound。

### Direct enumeration

```cpp
for each assignment in finite domains:
    if validator.validate(assignment, constraints):
        return Sat(assignment);

return Unsat(conflict over active reasons);
```

这能覆盖：

```smt2
(assert (= (* x x) 4))
```

因为 univariate equality 已经把：

```text
x ∈ {-2, 2}
```

所以枚举立刻找到 SAT。

---

## 6.3 Interval pruning

如果 domain 较大，用 interval evaluation：

```cpp
IntervalZ evalInterval(PolynomialId p, IntBox box);
```

判断：

```text
p <= 0:
  lower(p) > 0  => infeasible
  upper(p) <= 0 => satisfied

p >= 0:
  upper(p) < 0  => infeasible
  lower(p) >= 0 => satisfied

p = 0:
  0 ∉ interval(p) => infeasible

p != 0:
  0 ∉ interval(p) => satisfied
```

递归 split 最大 domain 的变量。

# 7. Local search：作为 SAT finder，但不替代 exact solver

目标函数：

```text

p != 0: 0 if p != 0 else penalty
```

初值：

```text

2. univariate roots；
3. relaxation rounding；
4. [-10,10] 小整数；
5. random。
```

但注意：`x^2=4` 不应该依赖 local search，应该由 univariate + bounded solver 直接解决。
