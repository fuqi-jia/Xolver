# Plan: Incremental GeneralSimplex (Dutertre-de Moura) for NLColver LRA

## Context & Goals

Replace NLColver's current single-variable-bound `SimplexSolver` with a Dutertre-de Moura General Simplex. The tableau and basis persist; bounds are updated incrementally by `assertLit`; `check()` only runs the pivot loop; `pop()` only restores bounds (not basis).

Reference: `20260510_2_lira.md` (lira.md) sections 1-4.

## Core Design: Tableau-Incremental / Semi-Incremental V1

V1 is **semi-incremental**: the tableau and basis persist, but active bounds are rebuilt for each SAT model. This is simpler to integrate with NLColver's current CDCL loop while preserving the correct DM algorithm.

Future V2 will make it fully incremental at the SAT decision level (bounds follow SAT trail without per-model reset).

Persistent structures:

1. **Tableau** (`matrix_`, `rhs_`, `basicVars_`, `isBasic_`): built once, mutated only by `pivot()`.
2. **Bounds** (`lower_`, `upper_`): updated incrementally; trail stack for push/pop.
3. **Assignment** (`beta_`): updated by `update()` and `pivotAndUpdate()`; recomputed after `resetActiveBounds()` or `pop()`.

## Data Structures

### DeltaRational

```cpp
struct DeltaRational {
    mpq_class a;  // real part
    mpq_class b;  // delta coefficient
};
// x > c  -> lower = DeltaRational(c,  1)
// x < c  -> upper = DeltaRational(c, -1)
// x >= c -> lower = DeltaRational(c,  0)
// x <= c -> upper = DeltaRational(c,  0)
```

### BoundValue

```cpp
enum class BoundKind { NegInf, Finite, PosInf };

struct BoundValue {
    BoundKind kind;
    DeltaRational value;
    bool operator<(const BoundValue& rhs) const;
    bool operator>(const BoundValue& rhs) const;
};
```

### BoundInfo

```cpp
struct BoundInfo {
    BoundValue bound;
    std::optional<SatLit> reason;  // valid only for finite bounds
};
```

For equality `s = 0`, both `lower_` and `upper_` are set to `0` with the same `reason`.

### Tableau

```cpp
// Row r:  basicVars_[r] = rhs_[r] + Σ_j matrix_[r][j] * x_j
// matrix_[r][j] is meaningful only for non-basic vars (basic var columns are 0).
std::vector<std::vector<mpq_class>> matrix_;  // m x n
std::vector<mpq_class> rhs_;                  // m
std::vector<int> basicVars_;                  // m
std::vector<bool> isBasic_;                   // n
std::vector<DeltaRational> beta_;             // n (assignment)
```

### Bound Trail

```cpp
struct BoundEntry {
    int var;
    bool isLower;
    BoundInfo oldBound;
};
std::vector<std::vector<BoundEntry>> trail_;
```

Initialized with a base frame: `trail_.push_back({})` in constructor.

## Constraint Registration

For each theory atom `p(x) ~ c` where `p(x) = Σ a_j x_j`:

1. Ensure original vars `x_j` exist.
2. Create auxiliary var `s`.
3. Add tableau row:
   ```
   s = -c + Σ a_j x_j
   ```
   In tableau storage:
   ```cpp
   rhs_[row] = -c;
   matrix_[row][xj] = a_j;
   basicVars_[row] = s;
   ```
4. Initially `s` is basic (coefficient 1 in its own row, 0 elsewhere after refactorization).

Initially all bounds are `(-inf, +inf)`. The constraint is inactive until `assertLit` sets a bound on `s`.

## Literal Assertion

`assertLit` translates theory atoms to bounds on auxiliary vars. It must handle both true and false literals.

| Atom form | True literal | False literal |
|-----------|-------------|---------------|
| `s <= 0` | `upper(s, 0)` | `lower(s, δ)` |
| `s < 0` | `upper(s, -δ)` | `lower(s, 0)` |
| `s >= 0` | `lower(s, 0)` | `upper(s, -δ)` |
| `s > 0` | `lower(s, δ)` | `upper(s, 0)` |
| `s = 0` | `lower(s,0) + upper(s,0)` | **disequality obligation** |
| `s != 0` | **disequality obligation** | `lower(s,0) + upper(s,0)` |

**Disequality obligation**: `s != 0` cannot be expressed as a single bound. The solver must:
- If current model has `s != 0`, accept.
- If current model has `s = 0`, return a **split lemma**: `s < 0 ∨ s > 0` (i.e., `s <= -δ ∨ s >= δ`).

This is handled at the `SimplexSolver` wrapper layer, not inside `GeneralSimplex`.

### assertUpper / assertLower

```cpp
AssertResult assertUpper(int var, BoundInfo info) {
    assert(info.bound.isFinite() && info.reason.has_value());
    if (info.bound <= upper_[var].bound) return OK;
    if (info.bound < lower_[var].bound) return CONFLICT;
    trail_.back().push_back({var, false, upper_[var]});
    upper_[var] = info;
    if (!isBasic_[var] && violatesUpper(var)) {
        update(var, info.bound.value);
    }
    return OK;
}
```

```cpp
void update(int xj, DeltaRational v) {
    assert(!isBasic_[xj]);
    DeltaRational delta = v - beta_[xj];
    for (int row = 0; row < m; ++row) {
        beta_[basicVars_[row]] += DeltaRational(matrix_[row][xj]) * delta;
    }
    beta_[xj] = v;
}
```

## check() - Pivot Loop

```cpp
bool violatesLower(int x) {
    return lower_[x].bound.isFinite() && beta_[x] < lower_[x].bound.value;
}
bool violatesUpper(int x) {
    return upper_[x].bound.isFinite() && beta_[x] > upper_[x].bound.value;
}
bool canIncrease(int x) {
    return !upper_[x].bound.isFinite() || beta_[x] < upper_[x].bound.value;
}
bool canDecrease(int x) {
    return !lower_[x].bound.isFinite() || beta_[x] > lower_[x].bound.value;
}
bool atUpper(int x) {
    return upper_[x].bound.isFinite() && beta_[x] == upper_[x].bound.value;
}
bool atLower(int x) {
    return lower_[x].bound.isFinite() && beta_[x] == lower_[x].bound.value;
}

Result check() {
    if (betaDirty_) { recomputeBeta(); betaDirty_ = false; }

    while (true) {
        int xi = findViolatedBasicVar();
        if (xi == -1) return Sat;

        if (violatesLower(xi)) {
            int xj = findEnteringVarToIncrease(xi);
            if (xj == -1) { explainLowerConflict(xi); return Unsat; }
            pivotAndUpdate(xi, xj, lower_[xi].bound.value);
        } else if (violatesUpper(xi)) {
            int xj = findEnteringVarToDecrease(xi);
            if (xj == -1) { explainUpperConflict(xi); return Unsat; }
            pivotAndUpdate(xi, xj, upper_[xi].bound.value);
        }
    }
}
```

### findEnteringVarToIncrease(xi)

Iterate non-basic vars `xj` in row(xi). Let `a = matrix_[row(xi)][xj]`.
- If `a > 0` and `beta_[xj] < upper_[xj].bound`, `xj` can increase.
- If `a < 0` and `beta_[xj] > lower_[xj].bound`, `xj` can decrease.
- Bland's rule: choose smallest index eligible var.

### pivotAndUpdate(xi, xj, target)

```cpp
void pivotAndUpdate(int xi, int xj, DeltaRational target) {
    int r = rowOf(xi);
    mpq_class aij = matrix_[r][xj];
    DeltaRational theta = (target - beta_[xi]) / aij;

    beta_[xi] = target;
    beta_[xj] += theta;
    for (int k = 0; k < m; ++k) {
        if (basicVars_[k] == xi) continue;
        beta_[basicVars_[k]] += DeltaRational(matrix_[k][xj]) * theta;
    }

    pivot(xi, xj);
}
```

### pivot(leaving, entering)

Gauss-Jordan elimination on the tableau. After pivot:
- `entering` becomes basic, `leaving` becomes non-basic.
- Every row maintains: `basic = rhs + Σ coeff(nonbasic) * nonbasic`.

## Conflict Explanation

For lower-bound conflict on basic `xi` with row `xi = Σ a_ij x_j`:

```cpp
N+ = { xj | a_ij > 0, atUpper(xj) }
N- = { xj | a_ij < 0, atLower(xj) }

conflictLits = { *lower_[xi].reason }
             + { *upper_[xj].reason | xj in N+ }
             + { *lower_[xj].reason | xj in N- }
```

All reasons used in conflict explanation must be finite bounds with valid reasons.

The wrapper translates these `SatLit` reasons into a `TheoryConflict` clause.

## Scope Management

### push() / pop()

```cpp
void push() { trail_.push_back({}); }

void pop() {
    for (auto it = trail_.back().rbegin(); it != trail_.back().rend(); ++it) {
        if (it->isLower) lower_[it->var] = it->oldBound;
        else             upper_[it->var] = it->oldBound;
    }
    trail_.pop_back();
    betaDirty_ = true;
}
```

### resetActiveBounds()

Called at the start of each new SAT model check. Clears bounds and reasons, marks beta dirty.

```cpp
void resetActiveBounds() {
    for (auto& l : lower_) l = {BoundValue::negInf()};
    for (auto& u : upper_) u = {BoundValue::posInf()};
    betaDirty_ = true;
}
```

### recomputeBeta()

```cpp
void recomputeBeta() {
    for (int j = 0; j < n; ++j) {
        if (isBasic_[j]) continue;
        beta_[j] = chooseValueWithinBounds(j);
    }
    for (int r = 0; r < m; ++r) {
        int b = basicVars_[r];
        beta_[b] = DeltaRational(rhs_[r]);
        for (int j = 0; j < n; ++j) {
            if (isBasic_[j]) continue;
            beta_[b] += DeltaRational(matrix_[r][j]) * beta_[j];
        }
    }
}
```

```cpp
DeltaRational chooseValueWithinBounds(int var) {
    auto& l = lower_[var].bound;
    auto& u = upper_[var].bound;
    if (l.isFinite() && u.isFinite()) {
        DeltaRational mid = (l.value + u.value) / 2;
        if (mid >= l.value && mid <= u.value) return mid;
        return l.value;
    }
    if (l.isFinite()) return l.value;
    if (u.isFinite()) return u.value;
    return DeltaRational(0);
}
```

## SimplexSolver Wrapper

`SimplexSolver` owns a `GeneralSimplex` and maps `CoreExpr` / `TheoryAtom` to GS indices.

**Stable atom mapping**: Separate `LinearFormKey` (maps to auxiliary var) from `AtomKey` (maps to assertion behavior). Multiple relations on the same linear form share one auxiliary var.

```cpp
struct LinearFormKey {
    std::vector<std::pair<std::string, mpq_class>> terms;  // sorted by var name
    mpq_class constant;  // p(x) - c
};

struct AtomKey {
    LinearFormKey form;
    Relation rel;
};

std::unordered_map<LinearFormKey, int, LinearFormKeyHash> formToAux_;
std::unordered_map<AtomKey, AtomInfo, AtomKeyHash> atomInfo_;
```

### assertLit

1. Extract linear constraint from `CoreExpr`.
2. Look up (or create) auxiliary var `s` via `atomToAux_`.
3. Translate `value` (true/false) and `relation` to bounds on `s`.
4. Call `gs_.assertLower(s, info)` / `gs_.assertUpper(s, info)`.

### check

1. Call `gs_.check()`.
2. If `Unsat`: translate `BoundReason`s to `TheoryConflict` using stored `reason` SatLits.
3. Return `TheoryResult`.

### reset

Call `gs_.resetActiveBounds()`. **Do not** rebuild tableau.

## Integration with CDCL(T) Loop

Current NLColver recreates the SAT solver per `checkSatInternal()`. V1 keeps this pattern but modifies `TheoryManager::check`:

1. **Do NOT** reset the theory solver between SAT iterations.
2. Call `solver->resetActiveBounds()` to clear active bounds.
3. Assert **all** theory atoms (both true and false) with their current SAT values.
4. Call `solver->check()`.

This is semi-incremental: tableau persists, bounds are rebuilt per model.

Future V2 will persist the SAT solver and atomizer too, making the entire solver incremental.

## Implementation Phases

### Phase 1: GeneralSimplex Core
- `DeltaRational.h`
- `GeneralSimplex.h/.cpp`: addVar, addConstraint, assertLower/Upper, update, check, pivot, conflict explanation, push/pop, resetActiveBounds.
- Unit tests: 2-var SAT, 2-var UNSAT, strict inequalities, disequality split.

### Phase 2: SimplexSolver Wrapper + TheoryManager Fix
- Rewrite `SimplexSolver` to own `GeneralSimplex`.
- Implement stable `AtomKey` mapping.
- Fix `TheoryManager::check`: assert both true and false literals; do not reset solver.
- End-to-end tests: multi-var LRA benchmarks.

### Phase 3: Scope + Disequality
- Test push/pop with bound trail.
- Test disequality (`!=`) split lemma generation.

### Phase 4: Hardening
- Bland's rule for anti-cycling.
- Debug-only iteration guard (LRA is decidable; should not normally hit).
- Degeneracy diagnostics.

## Testing Strategy

| Test | Expected |
|------|----------|
| x + y <= 3, x >= 1, y >= 1 | SAT |
| x + y <= 3, x >= 2, y >= 2 | UNSAT |
| 2x + 3y <= 5, x - y >= 1 | SAT |
| x > 3, x < 3 | UNSAT (strict) |
| x + y = 5, x - y = 1 | SAT |
| base: x <= 3, push: x >= 5, check UNSAT, pop, check SAT | SAT after pop |
| x != 0, x >= 0, x <= 0 | disequality split lemma -> UNSAT after split |
| assert x <= 2, then assert x >= 3 | UNSAT on second assert |

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Cycling | Bland's rule. Debug-only iteration guard (not a normal fallback). |
| Singular basis | Should not happen with auxiliary initial basis. Add assertion. |
| Disequality handling | Wrapper generates guarded split lemma: `¬(s!=0) ∨ (s<0) ∨ (s>0)`. Not an immediate conflict unless both branches contradicted. |
| Atomizer instability | Stable `AtomKey` mapping prevents duplicate auxiliary vars. |
| Semi-incremental perf | Acceptable for V1. Full incrementality is V2. |
