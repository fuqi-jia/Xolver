建议 **再 Revise 一小轮**。这版已经把最大 blocker 修掉了：`LinearizationResult` 现在已经改成 `PendingLinearizationLemma + CutCacheKey`，adapter 里也按 `item.cacheKey` 做 `markEmitted`，不再从 `TheoryLemma` 反推 cache metadata。这点终于对了。

但现在还有两个执行级问题，修完就可以 Approve。

# 1. `NiaLinearizationAdapter::run` 仍然没有 active assignments，无法生成 mirror lemmas

计划里已经明确需要 mirror effective active linear bounds：

```text
原始 linear bound -> NIA/NRA
mirror effective linear bound -> LIA/LRA
```

但 Step 9 的 adapter 接口仍然是：

```cpp
LinearizationResult run(
    const std::vector<NormalizedNiaConstraint>& constraints,
    const DomainStore& domains,
    TheoryLemmaDatabase& lemmaDb);
```

这个接口没有：

```text
active assignments
原始 literal
value=true/false
effective relation
```

所以它没法生成：

```text
¬B_NIA ∨ B_LIA
```

尤其 `B=false` 时，还需要先取反并规范化 relation。缺 active assignment，做不了这个步骤。

建议明确拆成两个函数：

```cpp
std::vector<TheoryLemma> mirrorActiveLinearBounds(
    const std::vector<ActiveTheoryAssignment>& activeAssignments,
    TheoryId targetLinearTheory);

LinearizationResult runLinearizer(
    const std::vector<NormalizedNiaConstraint>& nonlinearConstraints,
    const DomainStore& domains,
    TheoryLemmaDatabase& lemmaDb);
```

然后在 `NiaSolver::check()` 里顺序是：

```text
A. 先返回 pending mirror lemmas；
B. 生成/入队 mirror effective active linear bounds；
C. 再运行 IncrementalLinearizer 生成 nonlinear cuts；
D. 返回 pending linearization lemmas。
```

NRA 同理。

---

# 2. Bound reason policy 仍有一句二选一，V1 应该写死为 skip

Step 8 里仍然写：

```text
If a finite bound has no complete reason and is not marked global, either:
- Skip the cut, or
- Use a coarse all-active guard.
```

但后面的红线又写：

```text
If finite bounds have incomplete reasons and are not global, skip the cut.
```

V1 不要留二选一，否则 agent 可能实现 coarse all-active guard，把 scope 扩大。建议直接统一成：

```text
V1 policy:
If a finite bound is not global and reasonComplete=false, skip the bounds-dependent cut.
Do not implement coarse all-active guard in V1.
```

---

# 3. `markEmitted` 方法需要在 Adapter 接口里显式暴露

`NiaSolver::check()` 里写了：

```cpp
linAdapter_->markEmitted(*item.cacheKey);
```

但 `NiaLinearizationAdapter` 类目前只展示了：

```cpp
explicit NiaLinearizationAdapter(...);
LinearizationResult run(...);
```

需要补：

```cpp
void markEmitted(const CutCacheKey& key);
```

或者让 adapter 内部在 enqueue 时处理。既然现在设计是 solver 成功入队后再 mark，那 adapter 必须公开这个方法并转发到内部 `IncrementalLinearizer/LinearizationCache`。
