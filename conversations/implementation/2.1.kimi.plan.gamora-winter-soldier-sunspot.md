# Plan: Incremental GeneralSimplex (Dutertre-de Moura) for NLColver LRA

## Context & Goals

Replace NLColver's current single-variable-bound `SimplexSolver` with a fully incremental Dutertre-de Moura General Simplex. The core requirement is that the tableau persists across `assertLit` / `check` / `push` / `pop` calls; `assertLit` performs live bound updates (including `update()` on non-basic vars), and `check()` only runs the pivot loop.

Reference: `20260510_2_lira.md` (lira.md) sections 1-4 contain the full DM algorithm.

## Core Design: Pure DM Incremental Simplex

The solver maintains three persistent structures:

1. **Tableau** (`matrix_`, `rhs_`, `basicVars_`, `isBasic_`): built when constraints are first registered, mutated only by `pivot()`.
2. **Bounds** (`lower_`, `upper_`): updated incrementally by `assertLit`; trail stack for `push/pop`.
3. **Assignment** (`beta_`): updated by `update()` and `pivotAndUpdate()`; lazily recomputed after `pop()`.

### Why Not "Rebuild on check"

DM simplex is explicitly designed so that:
- `assertLower`/`assertUpper` only modify bounds.
- If a non-basic var violates its tightened bound, `update()` restores the invariant without rebuilding.
- `check()` only repairs basic-var violations via pivot.
- Backtracking only restores bounds; tableau and basis never revert.

## Data Structures

### DeltaRational

    struct DeltaRational {
        mpq_class a;  // real part
        mpq_class b;  // delta coefficient
    };
    // x > c  -> lower = DeltaRational(c,  1)
    // x < c  -> upper = DeltaRational(c, -1)

### Tableau

    // Row r:  basicVars_[r] = rhs_[r] + sum_j matrix_[r][j] * x_j
    std::vector<std::vector<mpq_class>> matrix_;  // m x n
    std::vector<mpq_class> rhs_;                  // m
    std::vector<int> basicVars_;                  // m
    std::vector<bool> isBasic_;                   // n
    std::vector<DeltaRational> beta_;             // n

### Bound Trail (for push/pop)

    struct BoundEntry {
        int var;
        bool isLower;
        std::optional<DeltaRational> oldBound;
    };
    std::vector<std::vector<BoundEntry>> trail_;

On pop: walk back entries, restore bounds, mark betaDirty_ = true.

## Algorithm Details

### 1. Constraint Registration (One-time)

For each theory atom sum a_j*x_j ~ c:
- Ensure original vars x_j exist.
- Create auxiliary var s.
- Add tableau row: s = sum a_j*x_j - c.
  In matrix: 1*s + sum(-a_j)*x_j = -c.
- s starts as basic (coeff 1 in its row).

Initially all bounds are (-inf, +inf). The constraint is inactive until assertLit sets a bound on s.

### 2. assertLit / assertLower / assertUpper (Incremental)

assertLower / assertUpper implement the DM rules from lira.md section 3:

    AssertResult assertUpper(int var, DeltaRational bound) {
        if (bound >= upper[var]) return OK;        // not tighter
        if (bound < lower[var]) return CONFLICT;   // contradicts lower
        trail_.back().push_back({var, false, upper[var]});
        upper[var] = bound;
        if (!isBasic_[var] && beta_[var] > bound) {
            update(var, bound);                    // restore non-basic invariant
        }
        return OK;
    }

update(var, v) from lira.md section 1.3:

    void update(int xj, DeltaRational v) {
        DeltaRational delta = v - beta_[xj];
        for (int row = 0; row < m; ++row) {
            beta_[basicVars_[row]] += DeltaRational(matrix_[row][xj]) * delta;
        }
        beta_[xj] = v;
    }

### 3. check() - Pivot Loop Only

    Result check() {
        if (betaDirty_) { recomputeBeta(); betaDirty_ = false; }

        while (true) {
            int xi = findViolatedBasicVar();
            if (xi == -1) return Sat;

            if (beta_[xi] < lower_[xi]) {
                int xj = findEnteringVarToIncrease(xi);
                if (xj == -1) { explainLowerConflict(xi); return Unsat; }
                pivotAndUpdate(xi, xj, *lower_[xi]);
            } else if (beta_[xi] > upper_[xi]) {
                int xj = findEnteringVarToDecrease(xi);
                if (xj == -1) { explainUpperConflict(xi); return Unsat; }
                pivotAndUpdate(xi, xj, *upper_[xi]);
            }
        }
    }

findEnteringVarToIncrease(xi) (lira.md section 1.4):
- Iterate non-basic vars xj in row(xi).
- If coeff > 0 and beta[xj] < upper[xj], can increase xj to increase xi.
- If coeff < 0 and beta[xj] > lower[xj], can decrease xj to increase xi.
- Bland's rule: choose smallest index eligible var.

pivotAndUpdate(xi, xj, target) (lira.md section 1.3):
- Compute theta = (target - beta[xi]) / coeff.
- Update beta: beta[xi] = target; beta[xj] += theta; sync other basic vars.
- Call pivot(xi, xj) to swap basic/non-basic status in tableau.

pivot(leaving, entering) performs Gauss-Jordan elimination.

### 4. Conflict Explanation (lira.md section 2)

For lower-bound conflict on basic xi with row xi = sum a_ij*x_j:

    N+ = { xj | a_ij > 0, beta[xj] at upper bound }  // cannot increase
    N- = { xj | a_ij < 0, beta[xj] at lower bound }  // cannot decrease
    conflict = { xi >= lower[xi] }
             + { xj <= upper[xj] | xj in N+ }
             + { xj >= lower[xj] | xj in N- }

### 5. push() / pop()

    void push() { trail_.push_back({}); }

    void pop() {
        for (auto it = trail_.back().rbegin(); it != trail_.back().rend(); ++it) {
            if (it->isLower) lower_[it->var] = it->oldBound;
            else             upper_[it->var] = it->oldBound;
        }
        trail_.pop_back();
        betaDirty_ = true;
    }

After pop, the next check() recomputes beta from scratch.

### 6. reset() vs resetBounds()

- resetBounds(): clear all bounds back to (-inf, +inf). Called at the start of each new SAT assignment phase.
- reset(): full reset including tableau (rarely needed; only when constraint set changes).

## SimplexSolver Wrapper Design

SimplexSolver owns a GeneralSimplex and maps between CoreExpr / TheoryAtom and GeneralSimplex indices.

Key methods:
- assertLit: extracts linear constraint, looks up auxiliary var, calls gs_.assertLower/Upper.
- check: calls gs_.check(), translates BoundReasons to SatLit clause.
- push/pop: forwards to gs_.
- conflict: returns the translated TheoryConflict.

## Integration with CDCL(T) Loop

Current NLColver recreates the SAT solver on each checkSatInternal(). To exploit incremental simplex, the theory solver must persist.

### Proposed Change to Solver::checkSatInternal()

Current (non-incremental):

    sat = createSatSolver();
    Atomizer atomizer(*sat);
    // ... CDCL loop with theoryManager.check() resetting each time

Proposed (incremental at theory level):

    if (!sat) sat = createSatSolver();
    Atomizer atomizer(*sat);
    // Register constraints once
    for (auto& atom : atomizer.atoms()) {
        if (atom.isTheory) theoryManager.registerAtom(atom);
    }
    // CDCL loop
    while (true) {
        auto sr = sat->solve();
        if (sr == Unsat) return Result::Unsat;
        // Update theory bounds (incremental)
        theoryManager.updateAssignments(*sat);
        auto tr = theoryManager.check();
        if (tr == Sat) return Result::Sat;
        // Add conflict clause and continue
    }

However, this is a significant architectural change. For V1, we keep the current CDCL loop but modify TheoryManager::check to:
1. NOT reset the theory solver.
2. Reset only bounds (not tableau).
3. Assert all atoms with their current value.
4. Call check().

This gives us the correct DM algorithm with minimal CDCL loop changes.

## Implementation Phases

### Phase 1: GeneralSimplex Core
- DeltaRational.h: complete operators, comparison, arithmetic.
- GeneralSimplex.h/.cpp:
  - addVar, addConstraint (tableau building)
  - assertLower, assertUpper (with update)
  - check() (pivot loop)
  - pivot(), pivotAndUpdate()
  - conflict explanation
  - push/pop trail
- Unit tests: 2-var SAT, 2-var UNSAT, strict inequalities.

### Phase 2: SimplexSolver Wrapper + TheoryManager Fix
- Rewrite SimplexSolver to own GeneralSimplex.
- Fix TheoryManager::check:
  - Do NOT reset theory solver between calls.
  - Reset only bounds.
  - Assert BOTH true and false theory literals.
- End-to-end tests: multi-var LRA via SMT-LIB files.

### Phase 3: CDCL Loop Integration
- Modify Solver::checkSatInternal to avoid recreating theory state.
- Test push/pop scope nesting with checkSat.

### Phase 4: Hardening
- Bland's rule (smallest index) for anti-cycling.
- Iteration limit with graceful Unknown fallback.
- Degeneracy diagnostics.

## Testing Strategy

| Test | Expected |
|------|----------|
| x + y <= 3, x >= 1, y >= 1 | SAT |
| x + y <= 3, x >= 2, y >= 2 | UNSAT |
| 2x + 3y <= 5, x - y >= 1 | SAT |
| x > 3, x < 3 | UNSAT (strict via delta) |
| x + y = 5, x - y = 1 | SAT (x=3, y=2) |
| push x >= 5 -> check UNSAT -> pop -> check SAT | SAT after pop |
| assert x <= 2, assert x >= 3 (incremental) | UNSAT on second assert |

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Cycling in pivot loop | Bland's rule (smallest eligible index). Iteration cap (e.g., 10000) with Unknown fallback. |
| Singular basis | Should not happen with auxiliary-variable initial basis. Add assertion and fallback. |
| Integration complexity | Phase 2 keeps minimal CDCL loop changes; only TheoryManager::check is modified. |
| Performance with dense matrix | Acceptable for V1. Sparse revised simplex (FOMT-Solver) is a future upgrade path. |
