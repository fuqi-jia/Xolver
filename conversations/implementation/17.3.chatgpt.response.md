# 1. V1 说 full rebuild，但 `assertLit` 里仍写了 add edge

计划里 `assertLit` 仍然写：

```text
Add edge to graph, record EdgeId in newEdges_.
```

但 V1 的 `check()` 又写：

```cpp
graph_.clear();
for each active assignment:
    extract difference constraint, normalize, add edge to graph_
auto r = bf_.runFull(graph_);
```

这两个设计冲突。既然 V1 明确“不使用 incremental SPFA，每次 check 全量重建”，那么 `assertLit` **不应该修改 graph**，也不需要 `newEdges_`。

V1 应该改成：

```cpp
TheoryCheckResult IdlSolver::assertLit(...) {
    activeAssignments_.push_back({level, lit, atom, value});
    graphDirty_ = true;
    return TheoryCheckResult::consistent();
}
```

然后所有这些工作都放在 `check()` 中做：

```cpp
graph_.clear();
disequalities_.clear();

for (const auto& a : activeAssignments_) {
    auto r = extractAndNormalize(a);

    if (r.kind == NormalizeKind::Unsupported) {
        return TheoryCheckResult::unknown();
    }

    if (r.kind == NormalizeKind::ImmediateConflict) {
        return TheoryCheckResult::mkConflict(
            TheoryConflict{{a.lit.negated()}}
        );
    }

    if (r.kind == NormalizeKind::Edges) {
        add edges to graph_;
    }

    if (r.kind == NormalizeKind::Disequality) {
        disequalities_.push_back(...);
    }

    if (r.kind == NormalizeKind::Tautology) {
        continue;
    }
}

auto bf = bf_.runFull(graph_);
...
```

这样 backtrack、重复 check、disequality 重建都干净。
