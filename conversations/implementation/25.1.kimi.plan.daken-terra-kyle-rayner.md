## 1. Architecture Decision

**Single recommended approach: Extend `TheoryManager` into an NO-aware coordinator.**

Rationale:
- `TheoryManager` already supports multiple solvers (NRA+LRA co-registration).
- `check()` is the natural location for the NO fixpoint loop.
- `assertTheoryLit()` is the natural routing point for shared equalities.
- Single-theory paths remain unchanged when only one solver is registered.
- Minimal disruption to `Solver.cpp`, `CadicalTheoryPropagator`, and `Atomizer`.

Alternative (rejected): A separate `NelsonOppenManager` wrapping `TheoryManager`. This would add unnecessary indirection and complicate the SAT-theory bridge. The NO logic belongs in the coordination layer that already exists.

### Phase NO-4: Convex Nelson-Oppen (EUF + LRA / EUF + RDL)

**Goal**: Full NO combination for convex theories via equality propagation.

**What to implement**:

1. **`TheoryManager::check()` NO fixpoint**
   Replace the sequential loop with:
   ```cpp
   TheoryCheckResult TheoryManager::check(TheoryLemmaDatabase& lemmaDb) {
       bool changed = true;
       while (changed) {
           changed = false;

           // 1. Check shared equality/disequality conflicts
           if (auto c = sharedEqMgr_.checkDisequalityConflict()) {
               return TheoryCheckResult::mkConflict(*c);
           }

           // 2. Send newly known shared equalities/disequalities to owning theories
           for (auto& eq : pendingSharedEqualities_) {
               for (auto* solver : solversOwning(eq.a, eq.b)) {
                   auto r = solver->assertInterfaceEquality(eq.a, eq.b, eq.reasonLit);
                   if (r.isConflictOrLemma()) return r;
               }
           }
           for (auto& neq : pendingSharedDisequalities_) {
               for (auto* solver : solversOwning(neq.a, neq.b)) {
                   auto r = solver->assertInterfaceDisequality(neq.a, neq.b, neq.reasonLit);
                   if (r.isConflictOrLemma()) return r;
               }
           }
           pendingSharedEqualities_.clear();
           pendingSharedDisequalities_.clear();

           // 3. Run each theory check
           for (auto* solver : solvers_) {
               auto r = solver->check(lemmaDb);
               if (r.isConflictOrLemma()) return r;
           }

           // 4. Collect theory-deduced shared equalities
           for (auto* solver : solvers_) {
               for (auto& prop : solver->getDeducedEqualities()) {
                   SatLit eqLit = registry_.getOrCreateSharedEqualityAtom(prop.a, prop.b);
                   if (!sat_.isAssignedTrue(eqLit)) {
                       // Generate lemma: Â¬r1 âˆ¨ ... âˆ¨ Â¬rk âˆ¨ Eq(a,b)
                       TheoryLemma lemma;
                       for (auto& reason : prop.reasons) lemma.lits.push_back(reason.negated());
                       lemma.lits.push_back(eqLit);
                       if (lemmaDb.insertIfNew(lemma)) {
                           return TheoryCheckResult::mkLemma(std::move(lemma));
                       }
                   }
               }
           }
       }
       return TheoryCheckResult::consistent();
   }
   ```

## 5. Red Lines (Invariants)

1. **Purification mandatory**: No mixed term reaches any theory solver unpurified.
2. **Shared equality canonicalization**: `Eq(a,b)` uses canonical key (`a < b`).
3. **Every propagated shared equality needs a lemma** with full `SatLit` reasons.
4. **Every combination conflict needs full reasons** from all contributing theories.
5. **Non-convex theories require lazy arrangement splits**: Do not enumerate all arrangements upfront.
6. **Final SAT validates original unpurified formula**: `FullModelValidator` runs before returning `Sat`.
7. **Internal purification variables hidden** from public `getModel()` / `getValue()`.
8. **Backtracking rolls back all state**: `SharedEqualityManager`, `ArrangementManager`, theory solvers, `TheoryManager`.
9. **Single-theory path unchanged**: When only one solver is registered, existing behavior is identical.
10. **NO code is gated**: All NO-specific logic is behind `if (isCombinationMode_)` or similar; no overhead in pure-logic paths.

## 7. Risk Mitigation

| Risk | Mitigation |
|---|---|
| EUF e-graph doesn't scale to large shared term sets | Limit EUF to uninterpreted-function terms only; arithmetic variables enter EUF only as leaves |
| LRA/LIA simplex interface equality causes performance regression | Interface equality is just one additional bound (`x-y=0`); simplex already handles disequalities |
| Purification produces too many bridge variables | Reuse bridge variables for identical alien subterms; memo table in Purifier |
| Arrangement split explosion | Lazy generation + relevant pair filtering; only split when theory requests it |
| Model validation too expensive | Only run on candidate SAT; cache subevaluations |
| Backtracking complexity | `SharedEqualityManager` uses same trail pattern as `RollbackUnionFind`; well-tested |
