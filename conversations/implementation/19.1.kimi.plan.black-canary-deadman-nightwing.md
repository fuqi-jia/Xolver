# ICP (Interval Constraint Propagation) Framework Implementation Plan

## Goal

Transform the existing NIA-private interval evaluation into a **theory-agnostic ICP common framework** under `src/theory/arith/`. NIA, NRA, IncrementalLinearizer, BoundedNiaSolver, and LocalSearchAdvisor will all be clients.

## Design Principles

1. **Over-approx only**: every interval operation must be sound over-approximation.
2. **No direct SAT claims**: ICP never returns `Sat` directly; candidates go through exact validator.
3. **Reason tracking**: every contraction carries `SatLit` reasons; empty box conflicts are explainable.
4. **Budget bounded**: `maxIterations` / `maxContractors` limits; budget exhaustion → `UnknownBudget`.
5. **Client owns split/B&B**: ICP returns `SplitSuggestion`; recursive search stays in NIA/NRA.

## Current State

Already exists under `src/theory/arith/interval/`:
- `IntervalZ` — integer interval `[lo, hi]`
- `IntervalOperations` — add, sub, neg, mul, pow (all over-approx)
- `ReasonedInterval` — interval + `std::vector<SatLit>` reasons
- `ReasonedBox` — per-variable map of `ReasonedInterval`
- `IntervalEvaluator` — single-var polynomial forward eval (multi-var → `NoChange`)

NIA already uses: `IntervalEvaluator`, `SquareBoundReasoner`, `SumOfSquaresBoundReasoner`.

## Key Data Structures

### IntervalQ
```cpp
struct IntervalQ {
    mpq_class lo, hi;
    bool loOpen = false, hiOpen = false;
    bool isEmpty() const;
    bool contains(const mpq_class& v) const;
    mpq_class width() const;
};
```
V1 only handles **finite intervals**; infinity → `NoChange` or require domain reasoner to give finite bounds first.

### IcpConstraint
```cpp
struct IcpConstraint {
    PolyId poly;       // normalized p rel 0
    Relation rel;      // Eq, Neq, Lt, Leq, Gt, Geq
    SatLit reason;     // active literal
    TheoryId owner;    // NIA / NRA
};
```

### IcpResult
```cpp
enum class IcpStatus {
    Conflict, DomainUpdate, Candidate, SplitSuggestion, NoChange, UnknownBudget
};
struct IcpResult {
    IcpStatus status;
    std::optional<TheoryConflict> conflict;
    std::vector<BoundUpdate> updates;
    std::optional<SplitSuggestion> split;
    std::optional<ModelCandidate> candidate;
};
```

### Contractor Interface
```cpp
class Contractor {
public:
    virtual ContractorResult contract(ReasonedBoxQ& box) = 0;
    virtual std::vector<std::string> vars() const = 0;
    virtual SatLit reason() const = 0;
};
```

## Implementation Order

### Phase 1: Interval Foundation (1-2 days)
1. `IntervalQ.h/.cpp` + `IntervalQOps.h` — rational interval ops with mpq
2. `ReasonedBoxQ` alongside existing `ReasonedBoxZ` (rename current to ReasonedBoxZ)
3. Unit tests for IntervalQ ops (add, sub, mul, div-by-zero handling, pow, sqrt)

### Phase 2: Relation Forward Conflict (1 day)
4. `RelationContractor` — forward eval only
   - `p <= 0`: if `p.lo > 0` → `Empty` conflict
   - `p >= 0`: if `p.hi < 0` → `Empty` conflict
   - `p = 0`: if `0 ∉ p.interval` → `Empty` conflict
   - `p != 0`: if `p.interval == [0,0]` → `Empty`; otherwise `NoChange`
5. `IcpEngine` skeleton with full worklist loop (no backward yet)
6. `Worklist` + watcher map

### Phase 3: Primitive Backward Contractors (2-3 days)
7. `AddContractor` / `SubContractor` — forward + backward
8. `MulContractor` — forward + backward; div-by-zero interval → `NoChange`
9. `SquareContractor` — forward + backward; `Z.hi < 0` → `Empty`
10. `PowContractor` — even/odd power handling
11. `SumSquaresContractor` — `Σ xi^2 <= c` direct bound
12. `ContractorFactory` — pattern match constraints to contractors

### Phase 4: NIA Adapter (1-2 days)
13. `NiaIcpAdapter`
    - Build `ReasonedBoxZ` from `DomainStore`
    - Convert active constraints to `IcpConstraint`
    - Run `IcpEngine`
    - Translate results back to `DomainStore` updates / conflicts
14. Wire into `NiaSolver::check()` pipeline (after algebraic reasoner, before bounded solver)
15. Replace/remove old `SquareBoundReasoner` / `SumOfSquaresBoundReasoner` direct logic; delegate to ICP

### Phase 5: NRA Adapter (1 day, optional for V1)
16. `NraIcpAdapter` — real bounds → `ReasonedBoxQ`
17. Wire as pre-pruning stage in `NraSolver`

### Phase 6: Splitter & Integration (1-2 days)
18. `Splitter` — choose variable by max width / activity
19. `SplitSuggestion` integration in NIA adapter
20. `IcpConfig` tuning (max iterations, min width, budget)

### Phase 7: Testing (2-3 days)
21. Unit tests for each contractor in isolation
22. Integration tests: NIA ICP on benchmark subset
23. Regression tests: ensure existing NIA verified cases still pass
24. Soundness checks: no false UNSAT on known-SAT instances

## Integration Points

### NIA Solver Pipeline (post-ICP)
```
assertLit → normalize → linear → square/univariate → algebraic → ICP → empty? → bounded → local search → split
```

### CDCL(T) TheoryCheckResult
```cpp
IcpResult r = icpEngine.run(...);
switch (r.status) {
  case Conflict:    return TheoryCheckResult::mkConflict(r.conflict);
  case DomainUpdate: apply to DomainStore; return NoChange;
  case Candidate:   if (validator.validate(r.model)) return SatCandidate; else NoChange;
  case SplitSuggestion: return TheoryCheckResult::mkLemma(makeSplitLemma(r.split));
  case UnknownBudget: return Unknown;
}
```

## Red Lines

1. All interval ops over-approx; no corner sampling for UNSAT.
2. Division by interval containing zero → `NoChange`, never contract.
3. NIA contractor results rounded to integers: `[ceil(a), floor(b)]`.
4. Every conflict includes all used bound reasons.
5. Budget exhaustion → `UnknownBudget`, never `Unsat`.
6. Candidate model must pass exact validator before returned as `Sat`.
7. ICP core has **zero dependency** on NIA/NRA types (only generic `TheoryId`).

## Expected Impact

| Capability | Before | After |
|------------|--------|-------|
| x² ≤ c pruning | NIA-only hardcoded | Generic SquareContractor |
| Σxᵢ² ≤ c pruning | NIA-only hardcoded | Generic SumSquaresContractor |
| Add/Sub/Mul contraction | None | Add/Sub/Mul contractors |
| NRA nonlinear pruning | None (CDCAC only) | ICP pre-pruning via adapter |
| Split suggestion | NIA ad-hoc | Generic Splitter |
| Reason tracking | Partial | Every bound has SatLit reasons |
