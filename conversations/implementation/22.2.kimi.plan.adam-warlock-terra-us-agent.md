# EUF Solver Enhancement Plan (Revision 2)

## 1. Current State

NLColver has a **working EUF V1** for pure `QF_UF`:
- `EufTermManager`: hash-consed interning of Variable, UFApply, ConstBool
- `EGraph`: union-find + full-rebuild congruence closure via repeated signature-table scan
- `EufSolver`: CDCL(T) TheorySolver interface, full-rebuild `check()`, conflict generation
- `Atomizer`: EUF atom extraction with n-ary `=` / `distinct` support
- 17 regression tests + 18 unit tests all pass

**One confirmed correctness bug**: `EGraph::explainPath()` BFS aborts with `{false,{}}` on benign revisits in undirected explanation graphs (e.g., transitivity cycles).

**Missing for production readiness**: model construction, theory propagation.

**Out of scope for this revision**: full Nelson-Oppen combination (QF_UFLIA/UFLRA/UFNIA/UFNRA remain `Unknown`).

## 2. Design Invariants (Non-Negotiable)

These are fixed for all implementation below; the plan never contradicts them.

| # | Invariant |
|---|-----------|
| I1 | **Atomizer owns EUF dedup** via `getOrCreateEufAtom(EufAtomPayload, ExprId)`. Registry only records via `registerParsedTheoryAtom`. No `Registry::getOrCreateEufAtom` exists. |
| I2 | **Bool sort ID** comes from `adapter.getBoolSortId()`, never hardcoded `0` and never inferred by scanning `ConstBool`. |
| I3 | **`EGraph` owns `termClass_`** mapping `EufTermId → EClassId`. `find(t)` reads `termClass_`; `addTerm` initializes it. |
| I4 | **`check()` order**: intern terms → `addTerm` → assert equalities → `closeCongruence` → check true≠false → check disequalities. |
| I5 | **`closeCongruence` returns `CloseStatus`** (Ok / SortMismatch). Header and implementation signatures match. Callers check the return value. |
| I6 | **Explanation cycle** returns `{false, {}}` (failure), never `{true, {}}`. |
| I7 | **`pop()` clamps** with `min(scopeLimit.back(), trail_.size())`. |
| I8 | **Synthetic ExprId allocator** has `reset()` and collision asserts against `TrueSentinelExpr` / `FalseSentinelExpr`. `Atomizer::reset()` clears `eufAtomDedup_` and resets the allocator. |
| I9 | **`distinct` with ≤1 arg** is vacuously true (returns SAT true literal). |
| I10 | **C++20** is required in CMake (`CMAKE_CXX_STANDARD 20`, `STANDARD_REQUIRED ON`). |

## 3. Phase 1: Fix Explanation BFS Bug

**File**: `src/theory/euf/EGraph.cpp`

**Root cause**: `explainPath()` uses a `visitedPairs` set that returns `{false,{}}` when any undirected edge is re-encountered. In an undirected explanation graph, a term can have multiple proof edges; revisiting a node via a different edge is normal and should not abort.

**Fix**:
- Replace `visitedPairs` + early-abort with standard BFS visited-node tracking.
- `parentTerm` and `parentEdge` already track the BFS tree; simply skip `next` if it already has a parent.
- Remove the `pairKey` collision detection entirely.

**Pseudocode of corrected BFS**:
```cpp
std::unordered_map<EufTermId, EufTermId> parentTerm;
std::unordered_map<EufTermId, size_t> parentEdge;
std::queue<EufTermId> q;
std::unordered_set<EufTermId> visited;

q.push(start);
visited.insert(start);
parentTerm[start] = start;

while (!q.empty()) {
    EufTermId cur = q.front(); q.pop();
    for (size_t edgeIdx : adj[cur]) {
        const auto& edge = proofEdges_[edgeIdx];
        EufTermId next = (edge.u == cur) ? edge.v : edge.u;
        if (visited.count(next)) continue;
        visited.insert(next);
        parentTerm[next] = cur;
        parentEdge[next] = edgeIdx;
        if (next == target) { /* reconstruct path */ }
        q.push(next);
    }
}
```

**New test** (`tests/unit/test_euf.cpp`):
```smt2
(set-logic QF_UF)
(declare-sort U 0)
(declare-const a U) (declare-const b U) (declare-const c U)
(declare-fun f (U) U)
(assert (= a b))
(assert (= b c))
(assert (= c a))
(assert (distinct (f a) (f c)))
(check-sat)
```
→ `unsat`. Before the fix, `explainPath` would abort on the cycle and the solver would incorrectly return `unknown`.

## 4. Phase 2: EUF Model Construction

### 4a. Model data structures (`src/theory/euf/EufTypes.h`)

Add:
```cpp
struct EufModel {
    // One value string per equivalence class, e.g. "ufval_0"
    std::unordered_map<EClassId, std::string> classValues;

    // Function interpretations: symbol → (argClassValues → resultClassValue)
    struct FuncEntry {
        std::vector<std::string> argValues;
        std::string resultValue;
    };
    struct FuncInterp {
        std::vector<FuncEntry> entries;
        std::string defaultValue;
    };
    std::unordered_map<FuncSymbolId, FuncInterp> functions;
};
```

### 4b. EGraph model builder (`src/theory/euf/EGraph.cpp`)

Implement `EufModel buildModel(const EufTermManager& tm) const`:
1. For each root equivalence class (where `ufFind(cid) == cid`), assign `classValues[cid] = "ufval_" + std::to_string(idx)`.
2. For each application term `t = f(args)` in `tm`:
   - `EClassId resultClass = find(t)`.
   - Build `argValues` by mapping each arg through `find → classValues`.
   - Append `{argValues, classValues[resultClass]}` to `functions[f].entries`.
3. Congruence closure guarantees uniqueness of entries per arg-signature; assert no collision.

Note: `buildModel` is `const` and does not modify the graph. It must be called after consistency is confirmed.

### 4c. EufSolver integration

- `EufSolver::check()` calls `model_ = egraph_.buildModel(termManager_)` after confirming consistency.
- Expose `const EufModel& eufModel() const`.

### 4d. Solver API wiring (`src/api/Solver.cpp`)

`Solver::getModel()` currently returns an empty `Model{}`. For `QF_UF`, after SAT is confirmed:
1. Look up the `EufSolver` from `TheoryManager`.
2. Convert `EufModel` to public `Model` format:
   - For each user-visible constant (variable in SMT-LIB), map its `EufTermId → EClassId → classValue`.
   - For each uninterpreted function, include its finite interpretation table.
3. Store in `Solver::Impl` for `getModel()` to return.

**Acceptance**: `nlcolver solve --produce-models` on `euf_001_sat_basic_eq.smt2` outputs a model where `a` and `b` share the same `ufval_N`.

## 5. Phase 3: Theory Propagation

### 5a. Track registered atoms

In `EufSolver::check()`, after building `equalities_` / `disequalities_`, collect all unique equality atoms seen (deduplicated by `satVar`):

```cpp
std::vector<EufAtom> candidateProps_;
for (const auto& a : trail_) {
    const auto& p = std::get<EufAtomPayload>(a.atom.payload);
    if (effectiveRel(p, a.value) == Relation::Eq) {
        EufAtom atom{lhs, rhs, Relation::Eq, a.lit};
        // dedup by satVar
        candidateProps_.push_back(atom);
    }
}
removeDuplicatesBySatVar(candidateProps_);
```

### 5b. Detect entailed unassigned equalities

After `closeCongruence()`, iterate `candidateProps_`:

```cpp
for (const auto& atom : candidateProps_) {
    if (atom already assigned true in current SAT model) continue;
    if (egraph_.same(atom.lhs, atom.rhs)) {
        auto er = egraph_.explainEquality(atom.lhs, atom.rhs, termManager_);
        if (!er.ok) continue;
        std::vector<SatLit> clause;
        for (SatLit r : er.reasons) clause.push_back(r.negated());
        clause.push_back(atom.assertedLit);
        return TheoryCheckResult::mkLemma(TheoryLemma{std::move(clause)});
    }
}
```

If the SAT solver assignment interface is not available to query "already assigned true", skip that filter; the SAT solver will ignore redundant lemmas.

### 5c. No negative propagation

EUF does not naturally entail disequalities (except true≠false, already handled). Skip negative propagation.

**Acceptance**: Add an indirect propagation test:
```smt2
(set-logic QF_UF)
(declare-sort U 0)
(declare-const a U) (declare-const b U)
(declare-fun f (U) U)
(assert (= a b))
(assert (not (= (f a) (f b))))
(check-sat)
```
→ `unsat`. The solver should detect this either via propagation lemma or via the disequality conflict path. The test verifies the overall result is correct.

## 6. Phase 4: Mixed-Theory Skeleton (Prep Only)

**File**: `src/api/Solver.cpp`

No full Nelson-Oppen in this revision. Only two changes:

1. **Purification fallback**: For QF_UFLIA/UFLRA/UFNIA/UFNRA, before returning `Unknown`, check if the problem is actually pure:
   - Scan `ir->assertions()` for any `UFApply`, uninterpreted sort, or EUF-specific equality.
   - If none → fall back to pure arithmetic solver path.
   - If only UF with no arithmetic ops → fall back to pure QF_UF path.
   - Only return `Unknown` when both theories are genuinely present.

## 10. Red Lines (Reiterated)

1. **No Registry-side EUF atom creation**. Only `Atomizer::getOrCreateEufAtom` creates/dedups.
2. **`true != false` check runs after `closeCongruence`**, not before.
3. **Explanation cycle always returns failure** (`ok=false`), never empty success.
4. **`closeCongruence` header and implementation signatures match** and return `CloseStatus`.
5. **`boolSortId` from `adapter.getBoolSortId()`**, never `0`.
6. **`EGraph` has `termClass_`** and `find(t)` uses it reliably.
7. **`pop()` clamps** to `min(scopeLimit, trailSize)`.
8. **Synthetic allocator has `reset()` and collision asserts**.
9. **`distinct` ≤1 arg is vacuously true**.
10. **C++20 required** in CMake.
11. **Mixed theories return `Unknown`** unless pure fallback applies.
12. **All existing tests pass** after every change.
