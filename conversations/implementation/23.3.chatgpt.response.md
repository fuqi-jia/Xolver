**结论：不接受当前版本。**

这版标题写的是 **Baseline Only**，Scope 也说只做 incremental e-graph replacement，model construction 和 theory propagation 都是 TODO-only。但后面又完整写了 **Phase 6: Theory Propagation** 和 **Phase 7: Model Construction**，而且 execution order 里也把 model 和 propagation 放进了实现步骤。这导致方案内部目标不一致，不能直接交给实现模型。

更重要的是，它仍然有几个 soundness blocker。

# 1. Scope 自相矛盾：说 Baseline Only，但又实现 propagation/model

开头写：

```text
Out of scope:
- Model construction
- Theory propagation
```

但后面又写：

```text
Phase 6: Theory Propagation
Phase 7: Model Construction
Execution Order:
8. Model construction
9. Propagation
```

这必须删干净。否则实现模型会不知道到底做不做这些功能。

建议本版只保留：

```text
Revision A: Incremental EUF baseline only

In scope:
1. RollbackUnionFind
2. IncrementalEGraph
3. incremental assertLit / backtrack / check
4. congruence closure
5. disequality conflict
6. explanation
7. Bool formula-position handling
8. true != false invariant

Out of scope:
1. model construction
2. theory propagation
3. mixed theory fallback
```

直接删除 Phase 6、Phase 7，以及 execution order 里的 model/propagation。

# 2. Bool 二值语义仍然没有写实

`assertLit()` 里写：

```cpp
// handle bool special case (= t true) with value=false → t = false
...
```

这个不能用省略号。Bool 语义是 soundness 关键点，必须明确写完整。

需要固定规则：

```text
For BoolTermAsFormula atom:
  lit(b) := (= b true)

If lit(b) assigned true:
  merge(b, true)

If lit(b) assigned false:
  merge(b, false)
```

也就是说，`lit(b)=false` 不能解释成普通 EUF disequality：

```text
b != true
```

而必须解释成：

```text
b = false
```

否则下面这个会假 SAT：

```smt2
(set-logic QF_UF)
(declare-const p Bool)
(declare-const q Bool)
(assert (distinct p true))
(assert (distinct q true))
(assert (distinct p q))
(check-sat)
```

正确结果是 UNSAT，因为 `p=false` 且 `q=false`。

建议补充 payload 标记：

```cpp
enum class EufAtomKind {
    Equality,
    Disequality,
    BoolTermAsFormula
};
```

然后在 `assertLit()` 中明确处理：

```cpp
if (payload.kind == EufAtomKind::BoolTermAsFormula) {
    EufTermId b = intern(payload.lhs);
    EufTermId target = value ? trueTerm : falseTerm;
    egraph_.merge(b, target, MergeReason::asserted(assertedLit));
    return;
}
```

# 3. `true != false` invariant 没有进入 `check()`

Scope 里说有 built-in `true ≠ false` invariant，但 `check()` 里只检查：

```text
1. pendingConflict_
2. disequalities_
3. optional propagation
4. consistent
```

没有看到：

```text
if egraph.same(trueTerm, falseTerm) => conflict
```

这会漏掉：

```smt2
(set-logic QF_UF)
(assert (= true false))
(check-sat)
```

建议在每次 merge 后，或者至少在 `check()` 中加入：

```cpp
if (egraph_.hasTrueFalseConflict()) {
    auto er = egraph_.explainEquality(trueTerm, falseTerm);
    if (!er.ok) return TheoryCheckResult::unknown();

    std::vector<SatLit> clause;
    for (auto lit : er.reasons) {
        clause.push_back(lit.negated());
    }
    return TheoryCheckResult::mkConflict({clause});
}
```

`check()` 顺序应固定为：

```text
1. pending conflict
2. true != false conflict
3. asserted disequality conflicts
4. consistent
```

如果保留 propagation，将来也应该放在 conflict 检查之后。

# 4. `pendingConflict_` 没有随 backtrack 回滚

`assertLit()` 里可能设置 `pendingConflict_`，但 `backtrackToLevel()` 没有清理它。

典型错误：

```text
level 1: assert (= a b)
level 2: assert (distinct a b)  -> pendingConflict_
backtrackToLevel(1)
check()
```

如果 `pendingConflict_` 没有清空，solver 会在回退后仍然返回 conflict。

建议二选一。

更简单：

```cpp
void EufSolver::backtrackToLevel(int level) {
    ...
    pendingConflict_.reset();
}
```

然后 `check()` 重新扫描当前有效 disequalities。

更精细：

```cpp
struct PendingConflict {
    int level;
    TheoryConflict conflict;
};
```

回退时：

```cpp
if (pendingConflict_ && pendingConflict_->level > level) {
    pendingConflict_.reset();
}
```

V1 建议直接清空并重扫。

# 5. `pop()` 写得太模糊，容易和 egraph snapshot 不一致

现在只写：

```text
push() stores scopeLimits_.push_back(trail_.size())
pop(n) restores trail size and rolls back egraph accordingly
```

这不够。Incremental e-graph 里 `pop()` 必须同时恢复：

```text
trail_
disequalities_
egraph snapshot
pendingConflict_
scope stack
```

建议增加：

```cpp
std::vector<size_t> scopeLimits_;
std::vector<EGraphSnapshot> scopeSnapshots_;
```

`push()`：

```cpp
void EufSolver::push() {
    scopeLimits_.push_back(trail_.size());
    scopeSnapshots_.push_back(egraph_.snapshot());
}
```

`pop()`：

```cpp
void EufSolver::pop(uint32_t n) {
    while (n-- && !scopeLimits_.empty()) {
        size_t lim = std::min(scopeLimits_.back(), trail_.size());
        EGraphSnapshot snap = scopeSnapshots_.back();

        trail_.resize(lim);
        removeDisequalitiesAfterTrailLimit(lim);
        egraph_.rollback(snap);
        pendingConflict_.reset();

        scopeLimits_.pop_back();
        scopeSnapshots_.pop_back();
    }
}
```

不要只靠 `trail_.resize()`，否则 egraph 仍然保留已经回退的 merges。

# 6. `levelSnapshots_` 设计容易 off-by-one

当前写：

```cpp
ensureSnapshotForLevel(level);
...
if (level < levelSnapshots_.size()) {
    egraph_.rollback(levelSnapshots_[level]);
}
```

这个设计有隐患，因为它默认：

```text
decision level 连续；
每个 level 只有一个固定 snapshot；
snapshot 是在该 level 第一条 assignment 之前记录的；
backtrackToLevel(L) 应该保留 level L 的 assignment，移除 >L。
```

但如果 snapshot 取错时机，就会把 level L 自己的 merge 也回滚掉。

建议明确：

```text
snapshot for level L is taken before the first assignment at level L.
backtrackToLevel(L) restores to the state after all assignments with level <= L.
```

更稳的设计是按 trail limit 管理：

```cpp
struct LevelSnapshot {
    int level;
    size_t trailLimitBeforeLevel;
    EGraphSnapshot snapBeforeLevel;
};
```

然后 `backtrackToLevel(L)`：

```text
remove assignments with level > L
rollback to snapshot before first removed assignment
```

如果暂时不想复杂化，可以本版只支持 `push/pop` snapshots，CDCL level backtrack 先用 full recompute 或明确测试覆盖。

# 7. Propagation 候选来源仍然错

Phase 6 写：

```text
Maintain registeredAtoms_ — all Eq atoms that have ever been seen by assertLit
```

这还是错的。`assertLit()` 只能看到已经被 SAT 赋值的 atom。Theory propagation 要找的是：

```text
已经注册但尚未赋值、且被 EUF 当前等价类蕴含的 equality atom
```

所以候选必须来自：

```text
TheoryAtomRegistry / Atomizer 里所有 EUF atoms
```

不是 `assertLit()` 见过的 atoms。

如果本版按 Baseline Only，就直接删除 Phase 6。
如果以后做 propagation，必须是：

```cpp
for (const auto& rec : registry.recordsForTheory(TheoryId::EUF)) {
    if (rec.payload.rel != Relation::Eq) continue;

    SatLit atomLit = SatLit::positive(rec.satVar);
    auto val = satAssignment.value(atomLit);

    if (val == True) continue;
    if (val == False) return conflict(negReasons + atomLit);
    if (val == Undef && egraph_.same(lhs, rhs)) {
        return lemma(negReasons + atomLit);
    }
}
```

没有 SAT assignment query，就不要实现 propagation。

# 8. Model construction 仍然是错误的 naive `ufval_N`

Phase 7 写：

```text
Each class gets a distinct UF value index
```

这对 Bool sort 不对，对不同 uninterpreted sorts 也不严谨。

如果以后做 model，应当是：

```text
Bool sort:
  class(true)  -> true
  class(false) -> false
  other Bool class -> false by default

Uninterpreted sort S:
  class C -> S!val_N
```

函数默认值也必须按 result sort：

```text
result sort Bool -> false
result sort S    -> S!val_default_N
```

不能所有 class 都统一 `ufval_N`。

本版既然声明 model out of scope，建议直接删除 Phase 7，只留 TODO：

```text
TODO Revision B:
  add sort-correct EUF model construction.
```

# 9. Explanation 需要返回 `ExplainResult`，不是 `vector<SatLit>`

当前 API 写：

```cpp
ExplainResult explainEquality(EufTermId a, EufTermId b);
```

但后面的文字又说：

```text
auto reasons = egraph_.explainEquality(...)
```

像是直接返回 vector。这里要统一。

必须强制：

```cpp
struct ExplainResult {
    bool ok;
    std::vector<SatLit> reasons;

```

调用处：

```cpp
auto er = egraph_.explainEquality(lhs, rhs);
if (!er.ok) {
    return TheoryCheckResult::unknown();
}

for (auto lit : er.reasons) {
    clause.push_back(lit.negated());
}
```

不要让 explanation failure 退化成空 reasons。

# 10. `MergeStatus` 无法表达 true/false conflict

`IncrementalEGraph::merge()` 当前返回：

```cpp
MergeStatus
```

但它只描述 merge 过程，比如 merged / already same / sort mismatch。
如果 merge 后触发：

```text
true and false become same
```

这个应该被 EufSolver 转成 conflict。

建议不要把 conflict 塞进 `MergeStatus`，而是在 `assertLit()` 每次 merge 后检查：

```cpp
auto st = egraph_.merge(...);
if (st == MergeStatus::SortMismatch) return unknown/internal error;

if (egraph_.hasTrueFalseConflict()) {
    pendingConflict_ = buildTrueFalseConflict(level);
}
```

如果该 merge 还导致已存在 disequality 被违反，也可以在 `check()` 重扫 disequalities，不一定 assertLit 立即找。

# 11. Signature table rollback 还需要更明确

你写了：

```text
Signature table lazy rebuild after backtrack
findFresh() recomputes signature of stored term to verify
```

方向可以，但 rollback 后队列和 signature table 的关系需要明确：

```text
rollback 后：
  sigTableDirty_ = true
  congruenceQueue_ 清空或回滚到 snapshot
  mergeQueue_ 回滚到 snapshot
```

更稳的规则：

```cpp
void rollback(EGraphSnapshot snap) {
    ...
    mergeQueue_.resize(snap.mergeQueueSize);
    congruenceQueue_.clear();
    sigTableDirty_ = true;
}
```

然后下一次 `merge()`：

```cpp
ensureFreshSignatureTable();
```

`ensureFreshSignatureTable()` 应当：

```text
clear sigTable_
iterate all current application terms
insert fresh signatures
detect existing congruent apps and enqueue merges if necessary
processToFixpoint if new congruence discovered
```

否则 rollback 后可能出现 signature table fresh 了，但已经存在的 congruent app 没有被重新合并。

# 12. `members_` rollback 还不够具体

你定义了：

```cpp
struct MemberChange {
    EufClassId dstRoot;
    EufClassId srcRoot;
    size_t oldDstSize;
};
```

但没有明确 loser class 的 members 如何处理。

建议采用最简单策略：

```text
只维护 root class 的 members 有效；
merge 时把 src members append 到 dst；
不清空 src members；
rollback 时 resize dst 到 oldDstSize；
src members 不需要恢复，因为从未修改。
```

并在文档里写清楚：

```text
members_[nonRoot] may be stale and must never be used unless its class is current root.
```

否则实现模型可能清空 src，然后 rollback 时恢复不回来。

# 13. `RollbackUnionFind::unite()` 对 no-op 也 push trail，不一定必要

当前规则：

```text
if ra == rb, push {merged=false}
```

这会让每次 redundant merge 都污染 trail。不是 soundness bug，但会让 snapshot / rollback trail 变大。

可以接受，但建议明确原因：

```text
Push no-op changes only if snapshots depend on exact operation count.
Otherwise no-op unite should not push trail.
```

更简单是：

```cpp
if (ra == rb) return false;
```

`snapshot()` 只是 `trail_.size()`，不要求 no-op 也入 trail。建议删掉 no-op push，减少回滚负担。
