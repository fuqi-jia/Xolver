## 2. Critical Design Fixes from Review

### Fix 1: Arrangement split is a SAT decision, not a tautology lemma

`Eq(a,b) ∨ ¬Eq(a,b)` as a lemma is useless. Instead:

```cpp
class ArrangementManager {
public:
    // Ensure Eq(a,b) atom exists, register it as observed/decision var,
    // enqueue into arrangementDecisionQueue.
    void requestDecision(SharedTermId a, SharedTermId b);

    // Called by CadicalTheoryPropagator when SAT asks for a decision
    // (or via theory-decision hook if available).
    std::optional<SatLit> pickNextDecision();

    // Called when SAT assigns Eq(a,b) or ¬Eq(a,b).
    void noteDecision(SharedTermId a, SharedTermId b, bool equal);
};
```

Flow:
1. Non-convex theory (or model-combination failure) needs `a=b` decided.
2. `requestDecision(a,b)` creates `Eq(a,b)` atom, registers with CaDiCaL, adds to queue.
3. SAT eventually branches on `Eq(a,b)` (true/false).
4. Assignment callback routes to `SharedEqualityManager` and owner theories.

**No tautology clauses.**

### Fix 2: Purification routes bridge atoms to exactly one owner

```cpp
struct PurifiedAtom {
    TheoryId owner;          // ONLY this theory sees this atom
    ExprId atom;             // guaranteed pure for owner theory
    std::vector<SharedTermId> sharedTerms; // symbols appearing in this atom
};

struct SharedTerm {
    ExprId symbol;           // original user var or purification var
    SortId sort;
    bool isInternal;         // true if created by Purifier
    std::bitset<TheoryId> owners;
};
```

Red line: **No mixed atom reaches any theory solver. Bridge definitions are routed only to the theory in whose signature they are pure. Only purified constants are shared across theories.**

### Fix 3: Only SAT-assigned equalities enter SharedEqualityManager union-find

```cpp
enum class EqualitySource { SatAssigned, TheoryPropagated };
```

- `SatAssigned`: `Eq(a,b)` is true in SAT → `SharedEqualityManager::assertEquality(a,b)` unions them.
- `TheoryPropagated`: LRA proves `a=b` → returns lemma `¬r1 ∨ ... ∨ ¬rk ∨ Eq(a,b)`. SAT must assign it before union.

**Never union a theory-propagated equality before SAT assignment.**

### Fix 4: NIA/NRA first-version hooks are skeleton only

| Capability | EUF | LRA | LIA | NRA | NIA |
|---|---|---|---|---|---|
| Accept interface equality | ✅ | ✅ | ✅ | ✅ | ✅ |
| Accept interface disequality | ✅ | ✅ | ✅ | ✅ | ✅ |
| Report deduced equality (lemma) | ✅ | ✅ | ⚠️ conservative | ❌ | ❌ |
| Request arrangement | N/A | N/A | ✅ | ❌ | ❌ |

NRA/NIA first version:
- Can accept `assertInterfaceEquality/Disequality`.
- **Never** return a lemma from sampling, model validation, or grid search.
- If model cannot be combined, return `Unknown`.
- Full nonlinear equality propagation deferred to Stage C.

### Fix 6: LRA interface disequality handling

```cpp
TheoryCheckResult LraSolver::assertInterfaceDisequality(a, b, reason) {
    if (impliesEquality(a, b)) {
        // Conflict: x≠y but bounds imply x=y
        return conflict(...);
    }
    storeDisequality(a, b, reason);
    return Consistent;
}
```

If final model gives `model(x) == model(y)` despite disequality:
- Dense reals: try perturbation.
- If perturbation fails or not desired: return `Unknown` (never return `Sat` with conflicting model).

### Fix 7: `check()` is single-pass CDCL(T) style, not an internal fixpoint

Remove `while (changed)` loop. `check()` runs once per SAT model check:
1. Process pending shared equality events.
2. Propagate to theories.
3. Run theory checks.
4. Return first conflict/lemma/decision.
5. If all consistent, try model combination.

The SAT engine drives the loop via `cb_check_found_model` → `TheoryManager::check()` → return → CaDiCaL continues.

**Phase A-1: Purifier + SharedTermRegistry**

Implement `src/theory/combination/Purifier.h/.cpp` and `SharedTermRegistry.h/.cpp`.

Purifier algorithm:
```
Input: CoreIr assertions for mixed logic
Output: vector<PurifiedAtom> + SharedTermRegistry populated

For each assertion ExprId e:
    Walk e bottom-up
    If node is theory atom T(...):
        For each child c that is alien to T:
            If c not yet purified:
                fresh = createPurificationVar(c)
                emit bridge atom: fresh = c  (owner = theory_of(c))
                register fresh in SharedTermRegistry
                add owner theory_of(c) to fresh.owners
            replace c with fresh
        emit purified atom e' with owner = theory_of(T)
        for each shared term s in e': add owner theory_of(T) to s.owners
```

Key invariant: after purification, every atom has exactly one owner theory.

**EufSolver implementation**:
- `assertInterfaceEquality(a,b,reason)`: intern shared terms into e-graph, merge classes.
  - Shared terms may be variables or purification vars. Treat them as e-graph nodes.
- `assertInterfaceDisequality(a,b,reason)`: store disequality (existing `ActiveDisequality` mechanism).
- `getDeducedSharedEqualities()`: after `check()`, scan e-graph. For each pair of shared terms in same class, if not already in `SharedEqualityManager`, report with `explainEquality` reasons.
- `getModel()`: return equivalence class representatives + concrete values.

- `getDeducedSharedEqualities()`: after simplex check, for any pair with tight bounds (`x-y ≤ 0` and `y-x ≤ 0`), report equality with bound reasons.
- `getModel()`: return rational values for shared variables.

**Phase A-5: TheoryManager NO coordinator (convex only)**

Rewrite `TheoryManager::check()`:

```cpp
TheoryCheckResult TheoryManager::check(TheoryLemmaDatabase& lemmaDb) {
    // 1. Process any newly SAT-assigned shared equalities/disequalities
    //    (These were queued by CadicalTheoryPropagator::notify_assignment)
    for (auto& ev : pendingSharedEqEvents_) {
        if (ev.isEquality) {
            sharedEqMgr_.assertEquality(ev.a, ev.b, ev.reasonLit);
            for (auto* solver : solversOwning(ev.a, ev.b)) {
                auto r = solver->assertInterfaceEquality(ev.a, ev.b, ev.reasonLit);
                if (r.isConflictOrLemma()) return r;
            }
        } else {
            sharedEqMgr_.assertDisequality(ev.a, ev.b, ev.reasonLit);
            for (auto* solver : solversOwning(ev.a, ev.b)) {
                auto r = solver->assertInterfaceDisequality(ev.a, ev.b, ev.reasonLit);
                if (r.isConflictOrLemma()) return r;
            }
        }
    }
    pendingSharedEqEvents_.clear();

    // 2. Check shared equality/disequality conflict
    if (auto c = sharedEqMgr_.checkDisequalityConflict()) {
        return TheoryCheckResult::mkConflict(*c);
    }

    // 3. Run each theory check
    for (auto* solver : solvers_) {
        auto r = solver->check(lemmaDb);
        if (r.isConflictOrLemma()) return r;
    }

    // 4. Collect theory-propagated shared equalities
    for (auto* solver : solvers_) {
        for (auto& prop : solver->getDeducedSharedEqualities()) {
            SatLit eqLit = registry_.getOrCreateSharedEqualityAtom(prop.a, prop.b);
            // Only generate lemma if SAT hasn't already assigned it true
            if (!sat_.isAssignedTrue(eqLit)) {
                TheoryLemma lemma;
                for (auto& reason : prop.reasons) {
                    lemma.lits.push_back(reason.negated());
                }
                lemma.lits.push_back(eqLit);
                if (lemmaDb.insertIfNew(lemma)) {
                    return TheoryCheckResult::mkLemma(std::move(lemma));
                }
            }
        }
    }

    // 5. Model combination (Stage A: convex, should succeed if all consistent)
    auto combo = tryCombineModels();
    if (combo.ok) return TheoryCheckResult::consistent();
    if (combo.conflict) return TheoryCheckResult::mkConflict(*combo.conflict);
    return TheoryCheckResult::unknown();
}
```

    theoryMgr->enableCombinationMode();
    atomizer.setDefaultTheory(TheoryId::NRA);
} else if (logic == "QF_UFNIA" || logic == "UFNIA") {
    theoryMgr->registerSolver(std::make_unique<EufSolver>());
    theoryMgr->registerSolver(std::make_unique<NiaSolver>());
    theoryMgr->enableCombinationMode();
    atomizer.setDefaultTheory(TheoryId::NIA);
}

## 5. Acceptance Criteria (Strengthened)

1. **Tautology split is never used** as the only arrangement mechanism.
2. **Theory-propagated equality is not inserted into union-find before SAT assignment.** Only `SatAssigned` equalities enter `SharedEqualityManager`.
3. **Every propagated equality has a reconstructable reason clause.** Lemma form: `¬r1 ∨ ... ∨ ¬rk ∨ Eq(a,b)`.
4. **Sampling/model validation never produces a lemma.** NRA/NIA `getDeducedSharedEqualities()` returns empty in Stage C.
5. **Purified bridge atoms are routed to exactly one owner theory.** No mixed atom reaches any solver.
6. **Shared equality atoms are canonical and SAT-visible.** `Eq(a,b)` and `Eq(b,a)` share one `SatVar`; registered as observed var.
7. **Backtracking restores all state:** shared equality UF, arrangement decisions, theory-local interface assertions, pending queues.
8. **Final `Sat` validates original unpurified formula.** `FullModelValidator` runs before returning.
9. **Single-theory paths produce unchanged behavior.** `QF_LRA`, `QF_UF`, `QF_LIA` regression tests pass with zero changes.
10. **`QF_UFNIA`/`QF_UFNRA` remain conservative.** Return `Unknown` when nonlinear hooks cannot prove satisfiability.
11. **`QF_UFLRA` acceptance:**
    - `(= x y)` + `(not (= (f x) (f y)))` → **unsat**
    - `(<= x y)` + `(<= y x)` + `(not (= (f x) (f y)))` → **unsat**
    - `(= x y)` + `(= (f x) (f y))` → **sat**
12. **`QF_UFLIA` acceptance:**
    - `(= x y)` + `(not (= (f x) (f y)))` → **unsat**
    - Case requiring arrangement split completes and returns correct result.
