# 关键问题：`extractPolynomialConstraint` 不能 hard-code `TheoryId::NRA`

计划里说：

```cpp
if (defaultTheory_ == TheoryId::NRA || defaultTheory_ == TheoryId::NIA) {
    if (polyKernel_ && extractPolynomialConstraint(eid, ir, v)) {
        // registered inside extractPolynomialConstraint
    }
}
```

这里 `extractPolynomialConstraint` 内部必须根据 `defaultTheory_` 注册：

```cpp
registry_->registerParsedTheoryAtom(
    v, eid, defaultTheory_,
    PolynomialAtomPayload{diff, rel, mpq_class(0)}
);
```

不能沿用 NRA 版本里的：

```cpp
registry_->registerParsedTheoryAtom(
    v, eid, TheoryId::NRA,
    PolynomialAtomPayload{diff, rel, mpq_class(0)}
);
```

否则在 QF_NIA 下，所有 polynomial atoms 会被注册成 `TheoryId::NRA`，`TheoryManager` 会把它们路由给 `CdcacSolver`，而不是 `NiaSolver`。这会破坏整个 Phase NIA-1 的目标。

所以要让 Kimi 改成：

```text
Revise one point before approve:

extractPolynomialConstraint must register the atom with the current default theory.

Under QF_NRA:
    register TheoryId::NRA

Under QF_NIA:
    register TheoryId::NIA

Do not hard-code TheoryId::NRA inside extractPolynomialConstraint.

Suggested signature:

bool Atomizer::extractPolynomialConstraint(
    ExprId eid,
    const CoreIr& ir,
    SatVar v,
    TheoryId theory
);

Call:

if ((defaultTheory_ == TheoryId::NRA || defaultTheory_ == TheoryId::NIA)
    && polyKernel_
    && extractPolynomialConstraint(eid, ir, v, defaultTheory_)) {
    ...
}

Inside:

registry_->registerParsedTheoryAtom(
    v, eid, theory,
    PolynomialAtomPayload{diff, rel, mpq_class(0)}
);
```
