# Incremental EUF E-Graph Implementation Plan

## Context: Existing Codebase State

NLColver already has a **functional full-rebuild EUF solver** (`src/theory/euf/`):
- `EufTermManager` — term interning with symbol + application hash-consing
- `EGraph` — union-find + proof forest + `closeCongruence()` via repeated full signature-table scan
- `EufSolver` — CDCL(T) interface with `assertLit` storing trail, `check()` rebuilding from scratch
- **Conflict explanation** already works via BFS on proof edges + recursive congruence expansion
- **20 unit tests + 21 regression SMT2 tests** passing

**This plan replaces the full-rebuild approach with an incremental e-graph.**

---

## Design Decision: Replace EGraph with IncrementalEGraph

The existing `EGraph` (255 lines) is intentionally simple. The incremental version is a ground-up redesign of the same abstraction. We will:
1. Keep `EufTermManager` but extend it with immutable `parentsOf()` use-lists.
2. Create `RollbackUnionFind` and `IncrementalEGraph` as new classes.
3. Evolve `EufSolver` to call `egraph.merge()` incrementally in `assertLit` instead of batching in `check()`.
4. Remove or deprecate the old `EGraph` files once the incremental version is fully wired and tested.

---

## Phase 1: EufTermManager Extension — Immutable Parent Use-Lists

**Goal:** Every term must know which applications contain it as an argument. This is immutable and needs no rollback.

**Files:** `src/theory/euf/EufTermManager.h`, `.cpp`

**Changes:**
- Add `std::vector<std::vector<EufTermId>> parents_` parallel to `nodes_`.
- In `createNode()`, for each argument `arg`, push the new node id into `parents_[arg]`.
- Add `const std::vector<EufTermId>& parentsOf(EufTermId t) const` accessor.

**Invariant:** `parentsOf(t)` never changes after term creation. No rollback needed.

## Phase 2: RollbackUnionFind

**Goal:** Union-find without path compression, with union-by-size, supporting `snapshot()` and `rollback()`.

**New files:** `src/theory/euf/RollbackUnionFind.h`, `.cpp`

**Interface:**
```cpp
class RollbackUnionFind {
public:
    EufClassId addNode();                    // new singleton class
    EufClassId find(EufClassId x) const;     // no path compression
    bool same(EufClassId a, EufClassId b) const;
    bool unite(EufClassId a, EufClassId b);  // returns true if merged
    size_t snapshot() const;                 // trail size marker
    void rollback(size_t snap);              // undo merges back to marker
    uint32_t classSize(EufClassId root) const;
private:
    std::vector<EufClassId> parent_;
    std::vector<uint32_t> size_;
    struct Change { EufClassId childRoot; EufClassId parentRoot; uint32_t oldParentSize; bool merged; };
    std::vector<Change> trail_;
};
```

**Key rule:** No path compression. `find()` walks parent links read-only. Rollback simply restores `parent_` and `size_` from `trail_`.

## Phase 3: IncrementalEGraph — Core Data Structure

**Goal:** Incrementally maintain congruence closure with merge queues, signature table, and full rollback support.

**New files:** `src/theory/euf/IncrementalEGraph.h`, `.cpp`

**Internal structures:**
```cpp
struct MergeRecord {
    MergeId id;
    EufTermId lhs, rhs;
    EufClassId lhsRootBefore, rhsRootBefore;
    MergeReason reason;
};

struct EGraphSnapshot {
    size_t ufSnap;
    size_t memberTrailSize;
    size_t mergeRecordSize;
    size_t mergeQueueSize;
    size_t congruenceQueueSize;
};
```

**State:**
- `EufTermManager& tm_` (immutable reference)
- `RollbackUnionFind uf_`
- `std::vector<std::vector<EufTermId>> members_` — class members (rollbackable via trail)
- `std::vector<MergeRecord> mergeRecords_`
- `std::deque<PendingMerge> mergeQueue_`
- `std::deque<EufTermId> congruenceQueue_`
- `SignatureTable sigTable_` (not strictly rollbackable in V1)
- `bool sigTableDirty_ = false`

**Key methods:**
- `ensureTerm(EufTermId t)` — add singleton class if not present
- `MergeStatus merge(EufTermId a, EufTermId b, MergeReason reason)` — enqueue + process
- `bool same(EufTermId a, EufTermId b) const`
- `EufClassId rep(EufTermId t) const`
- `EGraphSnapshot snapshot() const`
- `void rollback(EGraphSnapshot snap)`
- `std::vector<SatLit> explainEquality(EufTermId a, EufTermId b)` — BFS on `mergeRecords_` + recursive `explainMergeReason`

**Congruence closure algorithm (V1):**
1. `merge(a,b,reason)` enqueues a `PendingMerge`.
2. `processMergeQueue()` pops merges, skips if already same.
3. For each actual merge: record merge, unite UF, merge members, collect affected parents from both old roots, enqueue them to `congruenceQueue_`.
4. `processCongruenceQueue()` pops apps, computes `AppSignature` from current reps.
5. `sigTable_.findFresh(sig)` checks the table; if entry exists, verifies `computeSignature(entryTerm) == sig` to reject stale entries.
6. If fresh entry found and not same → enqueue congruence merge with `MergeReason::congruence(...)`.
7. Congruence merges loop back into `processMergeQueue()`.

**Stale signature handling (V1):**
- Forward path: signature table is incrementally maintained.
- After `rollback()`: set `sigTableDirty_ = true`. Next `merge()` or `check()` calls `rebuildSignatureTable()` which rescans all applications from scratch.
- This avoids complex signature-table rollback while still being incremental during forward search.

## Phase 4: MergeReason & Explanation

**Goal:** Produce sound conflict explanations for disequalities and propagation lemmas.

**Files:** `src/theory/euf/EufTypes.h` (extend existing), `IncrementalEGraph.cpp`

**MergeReason design:**
```cpp
enum class MergeReasonKind { AssertedEquality, Congruence };

struct MergeReason {
    MergeReasonKind kind;
    SatLit assertedLit;                          // for AssertedEquality
    EufTermId app1 = NullEufTerm;               // for Congruence
    EufTermId app2 = NullEufTerm;
    std::vector<std::pair<EufTermId, EufTermId>> argPairs; // for Congruence
};
```

**Explanation algorithm:**
1. Build temporary adjacency from `mergeRecords_` (only records up to current snapshot).
2. BFS from `a` to `b` on term-level graph (`lhs` ↔ `rhs`).
3. For each edge on the path, call `explainMergeReason()`:
   - `AssertedEquality` → returns `{assertedLit}`
   - `Congruence` → recursively calls `explainEquality(ai, bi)` for each arg pair, deduplicating literals
4. Cycle guard: `std::unordered_set<uint64_t> activePairs` (pair-key) to prevent infinite recursion on cyclic proof graphs.

**Note:** This is the same explanation style as the existing `EGraph`, but built on `MergeRecord` trail instead of `ProofEdge` vector. We can reuse the BFS + recursion pattern.

## Phase 5: EufSolver Integration — Incremental Assert & Backtrack

**Goal:** Replace full-rebuild `check()` with incremental `assertLit` + lightweight `check()`.

**Files:** `src/theory/euf/EufSolver.h`, `.cpp`

**New behavior:**

### `assertLit()`
```cpp
void EufSolver::assertLit(const TheoryAtomRecord& atom, bool value, int level, SatLit reason) {
    // 1. ensure snapshot exists for this level
    ensureSnapshotForLevel(level);

    // 2. compute effective relation
    const auto& payload = std::get<EufAtomPayload>(atom.payload);
    bool isEq = (payload.rel == Relation::Eq && value) || (payload.rel == Relation::Neq && !value);

    // 3. intern terms (monotonic, no rollback)
    EufTermId lhs = termManager_.intern(payload.lhs, *coreIr_);
    EufTermId rhs = termManager_.intern(payload.rhs, *coreIr_);
    if (lhs == NullEufTerm || rhs == NullEufTerm) return;

    // 4. ensure terms exist in egraph
    egraph_.ensureTerm(lhs);
    egraph_.ensureTerm(rhs);

    // 5. handle bool special case (= t true) with value=false → t = false
    ...

    // 6. store assignment
    trail_.push_back({level, reason, atom, value});

    // 7. incremental merge or disequality store
    if (isEq) {
        auto status = egraph_.merge(lhs, rhs, MergeReason::asserted(reason));
        // status may indicate immediate conflict from disequality
    } else {
        disequalities_.push_back({lhs, rhs, Relation::Neq, reason, level});
        if (egraph_.same(lhs, rhs)) {
            // immediate disequality conflict — store pendingConflict_
        }
    }
}
```

### `backtrackToLevel()`
```cpp
void EufSolver::backtrackToLevel(int level) {
    // 1. remove trail entries above level
    auto it = std::remove_if(trail_.begin(), trail_.end(), [level](auto& a){ return a.level > level; });
    trail_.erase(it, trail_.end());

    // 2. remove disequalities above level
    auto dit = std::remove_if(disequalities_.begin(), disequalities_.end(), [level](auto& d){ return d.level > level; });
    disequalities_.erase(dit, disequalities_.end());

    // 3. rollback egraph to snapshot for this level
    if (level < static_cast<int>(levelSnapshots_.size())) {
        egraph_.rollback(levelSnapshots_[level]);
    }

    // 4. clean scope limits
    while (!scopeLimits_.empty() && scopeLimits_.back() > trail_.size()) {
        scopeLimits_.pop_back();
    }
}
```

### `check()`
```cpp
TheoryCheckResult EufSolver::check(TheoryLemmaDatabase& lemmaDb) {
    // 1. check pending conflict from assertLit (disequality violated by incremental merge)
    if (pendingConflict_) return *pendingConflict_;

    // 2. verify all disequalities (lightweight; could optimize to only check touched)
    for (auto& d : disequalities_) {
        if (egraph_.same(d.lhs, d.rhs)) {
            auto reasons = egraph_.explainEquality(d.lhs, d.rhs);
            // negate reasons + negate disequality lit
            ...
            return TheoryCheckResult::mkConflict(...);
        }
    }

    // 3. optional: theory propagation
    if (enablePropagation_) {
        if (auto lemma = findPropagation(lemmaDb)) {
            return TheoryCheckResult::mkLemma(*lemma);
        }
    }

    // 4. model ready (egraph reps are current)
    return TheoryCheckResult::consistent();
}
```

### `push()` / `pop()`
- `push()` stores `scopeLimits_.push_back(trail_.size())`.
- `pop(n)` restores trail size and rolls back egraph accordingly.

## Phase 6: Theory Propagation

**Goal:** When egraph entails a registered equality that is unassigned in SAT, emit a propagation lemma.

**Implementation:**
- Maintain `registeredAtoms_` — all `Eq` atoms that have ever been seen by `assertLit` (deduplicated by `SatVar`).
- In `check()`, after disequality check:
  ```cpp
  for (auto& atom : registeredAtoms_) {
      if (satIsAssigned(atom.lit)) continue;
      if (atom.rel == Relation::Eq && egraph_.same(atom.lhs, atom.rhs)) {
          auto reasons = egraph_.explainEquality(atom.lhs, atom.rhs);
          // lemma = ¬reasons ∨ atom.lit
          if (!lemmaDb.contains(lemma)) {
              lemmaDb.insertIfNew(lemma);
              return TheoryCheckResult::mkLemma(lemma);
          }
      }
  }
  ```
- **Note:** SAT assignment query may need a new interface to `CadicalBackend` or we can store assigned vars locally. If no SAT query interface exists in V1, skip propagation and leave a TODO.

## Phase 7: Model Construction

**Goal:** Build a concrete EUF model from the egraph after successful `check()`.

**Files:** `src/theory/euf/EufModel.h`, `.cpp` (new)

**Design:**
```cpp
struct EufModel {
    // Each class gets a distinct UF value index
    std::unordered_map<EufClassId, uint64_t> classValue;
    // Function table: symbol -> list of (arg-class-tuple -> result-class)
    std::unordered_map<EufSymbolId, std::vector<std::pair<std::vector<EufClassId>, EufClassId>>> functions;
};
```

**Build steps:**
1. Iterate all classes; assign sequential `ufval_N`.
2. Iterate all application terms; for each, record `(symbol, rep(args)) -> rep(app)`.
3. Verify consistency: same `(symbol, arg-reps)` must map to same result class (congruence invariant).

**Integration:**
- `EufSolver::model()` returns `EufModel`.
- `Solver::checkSatInternal()` for `QF_UF` can optionally validate the model before returning `Sat`.
- `ModelValidator` can be extended to evaluate EUF `Eq`/`Distinct` using `EufModel`.

### Unit tests to add/modify (`tests/unit/test_euf.cpp`):

| Test | Purpose |
|------|---------|
| Incremental merge basic | `assert a=b` then `egraph.same(a,b)` |
| Transitivity incremental | `a=b`, then `b=c`, then `same(a,c)` |

## Key Red Lines

1. **No path compression in RollbackUnionFind** — simplifies rollback correctness.
2. **Signature table lazy rebuild after backtrack** — avoid complex sig-table trail in V1.
3. **Merge queue is non-recursive** — use `std::deque` for both merge and congruence queues.
4. **Explanation must work after rollback** — `mergeRecords_` must be truncated on `rollback()`.
5. **Class members must rollback** — `members_` trail restores old state.
6. **Stale signature entries must be detected** — `findFresh()` recomputes signature of stored term to verify.
7. **Sort mismatch is still an error** — `merge()` checks `sort(t1) == sort(t2)`.
8. **Disequality conflict includes full explanation** — all equality reasons + disequality reason, all negated.
9. **Term manager never rolls back** — terms are monotonic; only equality state rolls back.
10. **All existing 20 unit tests must still pass** — incremental rewrite must not regress.
