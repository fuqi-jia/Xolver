对，**你要的是完备方案，那就不能再保留 “same isolating interval ⇒ same value” 这种规则**。完整实现应该把 `compareRealAlg` 做成一个真正的 **real algebraic number comparator**，核心原则是：

```text id="o4e3d2"
两个代数数相等，必须由代数证明得到；
不能由当前隔离区间形状得到。
```

也就是说：

```text id="zdk5o7"
same interval        不能证明相等；
same polynomial/root 可以证明相等；
gcd/common-root      可以证明相等；
rational exact root  可以证明相等；
否则必须 refine 到区间分离。
```

---

# 1. 先定义必须满足的 invariant

`AlgebraicRoot` 不能只是一个区间。它必须表示：

```text id="ad8o3l"
alpha = definingPoly 的第 rootIndex 个实根，并且 lower < alpha < upper
```

建议 invariant：

```cpp id="l2c4af"
struct AlgebraicRoot {
    UniPolyId definingPoly;  // canonical primitive squarefree polynomial
    int rootIndex;           // index among real roots of definingPoly
    mpq_class lower;
    mpq_class upper;
};
```

其中 `definingPoly` 必须规范化：

```text id="b7x4jb"
1. primitive part
2. positive leading coefficient
3. squarefree part
4. zero leading coefficients trimmed
5. proportional polynomials canonicalized to same representation
```

否则：

```text id="94nd7s"
x^2 - 2
2x^2 - 4
-x^2 + 2
```

会被当成不同 defining polynomial，后面比较会变复杂。

---

# 2. rational root 必须提前 canonicalize

如果 root isolation 得到：

```text id="w0ccpa"
lower == upper
```

并且：

```text id="ps18o3"
eval(definingPoly, lower) == 0
```

那就不要创建 `AlgebraicRoot`，直接创建：

```cpp id="hhq5ro"
RealAlg::fromRational(lower)
```

所以正式实现里不应该长期存在：

```text id="ik6x9x"
algebraic root [0, 0]
algebraic root [-1, -1]
```

这类对象。它们应该全部变成 rational。

# 3. compareRealAlg 的完整语义

定义返回值：

```cpp id="to7ojw"
enum class CompareResult {
    Less,
    Equal,
    Greater,
    Unknown
};
```

数学完备版本理论上不需要 `Unknown`，因为 refine 一定会终止；但工程里要有 resource budget，所以保留 `Unknown` 是对的。

主入口：

```cpp id="ti30nt"
CompareResult compareRealAlg(const RealAlg& a, const RealAlg& b) {
    if (a.isRational() && b.isRational()) {
        return compareRational(a.rational, b.rational);
    }

    if (a.isRational() && b.isAlgebraic()) {
        return compareRationalAlgebraic(a.rational, b.root);
    }

    if (a.isAlgebraic() && b.isRational()) {
        return invert(compareRationalAlgebraic(b.rational, a.root));
    }

    return compareAlgebraicAlgebraic(a.root, b.root);
}
```

# 4. rational vs algebraic 的完整比较

不能只看：

```cpp id="akjlwr"
if (q < lower) return Less;
if (q > upper) return Greater;
```

因为 `q` 可能落在隔离区间内部。

完整逻辑：

```cpp id="nxl5m4"
CompareResult compareRationalAlgebraic(mpq_class q, AlgebraicRoot alpha) {
    while (true) {
        if (q < alpha.lower) {
            return CompareResult::Less;     // q < alpha
        }

        if (q > alpha.upper) {
            return CompareResult::Greater;  // q > alpha
        }

        // q lies inside the isolating interval.
        // Check whether q is exactly alpha.
        Sign s = evalUniAtRational(alpha.definingPoly, q);

        if (s == Sign::Zero) {
            return CompareResult::Equal;
        }

        // q is not the root, but the interval is too wide.
        // Refine alpha until q is excluded.
        if (!refineRootInterval(alpha)) {
            return CompareResult::Unknown;
        }
    }
}
```

这个逻辑可以正确处理：

```text id="vyixnn"
alpha = sqrt(2), interval [1,2]
q = 3/2
```

因为 `3/2` 在 `[1,2]` 内，但不是 `x^2-2` 的根，所以继续 refine，直到区间变成：

```text id="3lnjfl"
[1.375, 1.4375]
```

这时 `3/2 > upper`，返回 `q > alpha`。

# 5. algebraic vs algebraic 的完整比较

这个是关键。不能写：

```cpp id="unl4s9"
if (same interval) return Equal;
```

完整逻辑应该是：

```cpp id="l07e8l"
CompareResult compareAlgebraicAlgebraic(AlgebraicRoot a, AlgebraicRoot b) {
    // 1. Same canonical polynomial
    if (a.definingPoly == b.definingPoly) {
        if (a.rootIndex == b.rootIndex) {
            return CompareResult::Equal;
        }
        return a.rootIndex < b.rootIndex
            ? CompareResult::Less
            : CompareResult::Greater;
    }

    // 2. Fast path: disjoint intervals
    if (a.upper < b.lower) return CompareResult::Less;
    if (b.upper < a.lower) return CompareResult::Greater;

    // 3. Algebraic equality proof by gcd
    UniPolyId d = gcdUni(a.definingPoly, b.definingPoly);

    if (!isConstantUni(d)) {
        auto pa = locateRootInPolynomial(a, d);
        auto pb = locateRootInPolynomial(b, d);

        if (pa.status == RootLocateStatus::Belongs &&
            pb.status == RootLocateStatus::Belongs) {

            if (pa.rootIndexInTarget == pb.rootIndexInTarget) {
                return CompareResult::Equal;
            }

            return pa.rootIndexInTarget < pb.rootIndexInTarget
                ? CompareResult::Less
                : CompareResult::Greater;
        }
    }

    // 4. Not proven equal. Refine until intervals separate.
    for (int i = 0; i < maxRefineIters_; ++i) {
        if (a.upper < b.lower) return CompareResult::Less;
        if (b.upper < a.lower) return CompareResult::Greater;

        bool okA = refineRootInterval(a);
        bool okB = refineRootInterval(b);

        if (!okA || !okB) {
            return CompareResult::Unknown;
        }
    }

    return CompareResult::Unknown;
}
```

这里真正重要的是第 3 步：**用 gcd 证明共同根**。

# 6. locateRootInPolynomial 的完整实现

`rootBelongsTo(alpha, d)` 只返回 bool 还不够。为了比较两个不同 defining polynomial 的共同根，最好返回：

```cpp id="vz3gqb"
enum class RootLocateStatus {
    Belongs,
    NotBelongs,
    Unknown
};

struct RootLocateResult {
    RootLocateStatus status;
    int rootIndexInTarget = -1;
};
```

目标：

```text id="i62qsg"
给定 alpha 是 f 的某个根；
判断 alpha 是否也是 h 的根；
如果是，返回它是 h 的第几个实根。
```

实现：

```cpp id="6o4nd5"
RootLocateResult locateRootInPolynomial(AlgebraicRoot alpha,
                                         UniPolyId h) {
    if (isConstantUni(h)) {
        return {RootLocateStatus::NotBelongs, -1};
    }

    h = squarefreePart(h);

    while (true) {
        // alpha.interval = (lower, upper)
        // 要求 endpoints 不是 h 的根；如果是，需要 refine alpha interval。
        if (evalUniAtRational(h, alpha.lower) == Sign::Zero ||
            evalUniAtRational(h, alpha.upper) == Sign::Zero) {
            if (!refineRootInterval(alpha)) {
                return {RootLocateStatus::Unknown, -1};
            }
            continue;
        }

        int countInside = countRealRootsInInterval(h, alpha.lower, alpha.upper);

        if (countInside == 0) {
            return {RootLocateStatus::NotBelongs, -1};
        }

        if (countInside == 1) {
            int rootsBefore = countRealRootsLessThan(h, alpha.lower);
            return {RootLocateStatus::Belongs, rootsBefore};
        }

        // More than one h-root lies in alpha's current interval.
        // Refine alpha's isolating interval until at most one remains.
        if (!refineRootInterval(alpha)) {
            return {RootLocateStatus::Unknown, -1};
        }
    }
}
```

为什么这是正确的？

因为如果：

```text id="p8sc2b"
d = gcd(f, g)
```

那么 `d` 的根一定也是 `f` 的根。`alpha` 的 interval 隔离了 `f` 的一个根。不断 refine 后，这个 interval 中要么没有 `d` 的根，要么只有一个 `d` 的根。若有一个，它只能是 `alpha`。

# 7. 对当前例子的正确处理

测试：

```text id="qx66wk"
x^2 - 2 = 0
(x^2 - 2)(x + 1) != 0
```

两个 defining polynomials：

```text id="b0b0yu"
f = x^2 - 2
g = x^3 + x^2 - 2x - 2
```

比较它们的正根：

```text id="n0v84s"
alpha = sqrt(2), root of f
beta  = sqrt(2), root of g
```

完整算法：

```text id="6h49mx"
gcd(f, g) = x^2 - 2
locate alpha in gcd -> rootIndex = 1
locate beta  in gcd -> rootIndex = 1
=> Equal
```

比较负根：

```text id="f9b7ll"
alpha = -sqrt(2), root of f
beta  = -sqrt(2), root of g
```

```text id="fdfwvc"
gcd(f, g) = x^2 - 2
locate alpha in gcd -> rootIndex = 0
locate beta  in gcd -> rootIndex = 0
=> Equal
```

比较 `-1`：

```text id="td5zug"
q = -1
f(-1) = -1 != 0
```

所以它不会和 `±sqrt(2)` 合并。

# 8. mergeRoots 不能直接用 std::sort + Unknown comparator

如果 `compareRealAlg` 可能返回 `Unknown`，不能这样写：

```cpp id="qll1ex"
std::sort(roots.begin(), roots.end(), [&](const RealAlg& a, const RealAlg& b) {
    return compareRealAlg(a, b) == CompareResult::Less;
});
```

因为 `std::sort` 需要严格弱序。`Unknown` 会破坏排序。

完整方案：写一个可以返回失败的排序。

```cpp id="dzdzfn"
std::optional<std::vector<RealAlg>>
sortAndMergeRoots(std::vector<RealAlg> roots) {
    std::vector<RealAlg> out;

    for (RealAlg r : roots) {
        bool inserted = false;

        for (auto it = out.begin(); it != out.end(); ++it) {
            CompareResult c = compareRealAlg(r, *it);

            if (c == CompareResult::Unknown) {
                return std::nullopt;
            }

            if (c == CompareResult::Equal) {
                inserted = true;
                break;
            }

            if (c == CompareResult::Less) {
                out.insert(it, r);
                inserted = true;
                break;
            }
        }

        if (!inserted) {
            out.push_back(r);
        }
    }

    return out;
}
```

如果返回 `nullopt`：

```text id="zgskd7"
不能继续构造 cells；
必须返回 Unknown。
```

不要带着不确定排序继续 CDCAC，否则可能 unsound。

# 9. boundEqual / coversAllLine 也必须走 compareRealAlg

不要让 `Bound` 自己比较：

```cpp id="etvjpx"
a.value.root == b.value.root
```

因为不同 defining polynomial 可以表示同一个代数数。

应该：

```cpp id="f3gocs"
bool boundEqual(const Bound& a, const Bound& b) {
    if (a.kind != b.kind) {
        // rational/algebraic 混合也可能相等
        if (a.isFinite() && b.isFinite()) {
            return compareRealAlg(a.value, b.value) == CompareResult::Equal;
        }
        return false;
    }

    if (a.isNegInf() || a.isPosInf()) {
        return a.kind == b.kind;
    }

    return compareRealAlg(a.value, b.value) == CompareResult::Equal;
}
```

如果 `compareRealAlg` 返回 `Unknown`：

```text id="e6j1tz"
coversAllLine 不能返回 true；
应该返回 Unknown 或 false 并向上报告 Unknown。
```

最好把接口从：

```cpp id="ay8i58"
bool coversAllLine(const Covering& cover)
```

改成：

```cpp id="s8paz9"
enum class CoverageResult {
    Covers,
    DoesNotCover,
    Unknown
};

CoverageResult coversAllLine(const Covering& cover);
```

# 10. signUnivariateAtAlgebraic 也要用同一套 gcd/root locate

现在你有：

```text id="1n9aci"
signUnivariateAtAlgebraic:
  gcd zero detection → interval refinement
```

这里也不能只做比例检测。

完整逻辑：

```cpp id="3s54gj"
Sign signUnivariateAtAlgebraic(UniPolyId g,
                               AlgebraicRoot alpha) {
    if (isZeroUni(g)) {
        return Sign::Zero;
    }

    UniPolyId f = alpha.definingPoly;
    UniPolyId d = gcdUni(f, g);

    if (!isConstantUni(d)) {
        auto loc = locateRootInPolynomial(alpha, d);

        if (loc.status == RootLocateStatus::Belongs) {
            return Sign::Zero;
        }

        if (loc.status == RootLocateStatus::Unknown) {
            return Sign::Unknown;
        }
    }

    // alpha is not a root of g.
    // Refine alpha interval until g has constant sign on the interval.
    while (true) {
        if (evalUniAtRational(g, alpha.lower) == Sign::Zero ||
            evalUniAtRational(g, alpha.upper) == Sign::Zero) {
            if (!refineRootInterval(alpha)) return Sign::Unknown;
            continue;
        }

        int gRoots = countRealRootsInInterval(g, alpha.lower, alpha.upper);

        if (gRoots == 0) {
            mpq_class m = midpoint(alpha.lower, alpha.upper);
            return evalUniAtRational(g, m);
        }

        if (!refineRootInterval(alpha)) {
            return Sign::Unknown;
        }
    }
}
```

这个可以完整处理：

```text id="6yyjxm"
x^2 - 2 = 0
(x^2 - 2)(x+1) != 0
```

因为 `gcd(f,g)=f`，直接得到 zero。

# 11. refineRootInterval 的要求

`refineRootInterval(alpha)` 必须保持 invariant：

```text id="2zi4si"
alpha.lower < alpha < alpha.upper
且 interval 中恰好有 definingPoly 的一个实根，即 alpha。
```

伪代码：

```cpp id="a52m2w"
bool refineRootInterval(AlgebraicRoot& alpha) {
    UniPolyId f = alpha.definingPoly;

    mpq_class mid = midpoint(alpha.lower, alpha.upper);

    Sign sm = evalUniAtRational(f, mid);

    if (sm == Sign::Zero) {
        // Found rational root. Canonicalization should happen outside.
        alpha.lower = mid;
        alpha.upper = mid;
        return true;
    }

    int leftCount = countRealRootsInInterval(f, alpha.lower, mid);

    if (leftCount == 1) {
        alpha.upper = mid;
        return true;
    }

    int rightCount = countRealRootsInInterval(f, mid, alpha.upper);

    if (rightCount == 1) {
        alpha.lower = mid;
        return true;
    }

    return false;
}
```

如果发现 rational root，最好返回一个信号，让上层把 `RealAlg` canonicalize 成 rational。

# 12. 最终完整实现顺序

你现在应该要求 coding agent 按这个顺序做，而不是继续 patch：

```text id="qsdcyv"
1. UniPoly canonicalization:
   primitive, squarefree, positive leading coefficient, trimmed coefficients.

2. Rational root canonicalization:
   singleton interval roots become RealAlg::fromRational.

3. Sturm/root counting utilities:
   countRealRootsInInterval
   countRealRootsLessThan
   endpoint-root handling.

4. refineRootInterval:
   maintain one-root invariant.

5. locateRootInPolynomial:
   decide whether alpha is a root of another polynomial h;
   return rootIndexInTarget.

6. signUnivariateAtAlgebraic:
   gcd + locateRootInPolynomial for zero;
   refinement for positive/negative.

7. compareRationalAlgebraic:
   eval at rational if rational is inside interval;
   refine until outside or equal.

8. compareAlgebraicAlgebraic:
   same canonical polynomial/rootIndex;
   gcd + locate in gcd;
   refine until disjoint;
   Unknown only on resource failure.

9. mergeRoots:
   no std::sort with Unknown;
   use insertion sort / checked sort that can return Unknown.

10. coversAllLine:
    Bound equality uses compareRealAlg;
    CoverageResult supports Unknown.
```

# 14. 判断标准

完整方案的判断标准是：

```text id="tm0eru"
不会因为两个不同代数根当前区间相同而误合并；
不会因为两个相同代数根来自不同 defining polynomial 而无法合并；
不会在 compare Unknown 的情况下继续 sort/merge/coversAllLine；
不会用 interval refinement 判断 exact zero；
exact zero 必须由 gcd / reduction / root membership 证明。
```

一句话：**完整实现不是“same interval + refine”，而是“canonical polynomial + gcd common-root proof + certified refinement”。**
