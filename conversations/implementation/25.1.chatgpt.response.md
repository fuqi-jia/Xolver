## 结论：**不接受，需要 Revise**

当前计划方向是对的：把 `TheoryManager` 扩展成 Nelson-Oppen 协调层，而不是另起一个 `NelsonOppenManager`，这个架构选择可以保留；计划里也已经列出了 `Purifier`、`SharedTermRegistry`、`SharedEqualityManager`、`ArrangementManager` 等核心组件，整体分层是合理的。

但现在这版还不能直接执行，因为有几个 **soundness / completeness blocker**。

---

## 1. 最大 blocker：`Eq(a,b) ∨ ¬Eq(a,b)` 作为 lemma 基本无效

计划中非凸理论 arrangement 的做法是生成：

```cpp
Eq(a,b) ∨ ¬Eq(a,b)
```

这只是一个重言式。它最多引入一个 SAT 变量，不会真正推动 CDCL 做 case split；如果 SAT 端忽略 tautological clause，甚至可能完全没有效果。

应该改成：

```cpp
ArrangementManager::requestDecision(a, b)
```

然后通过 SAT 决策接口让 `Eq(a,b)` 成为候选 branching literal。也就是说：

```cpp
requestArrangement(a,b)
  -> ensure Eq(a,b) atom exists
  -> observe/register it in CaDiCaL
  -> put into arrangementDecisionQueue
  -> cb_decide / theory decision hook returns Eq(a,b) or ¬Eq(a,b)
```

不要把 arrangement split 理解成“加一条 tautology lemma”。它本质是 **引入共享等式原子并强制 SAT 对它做决策**。

---

## 2. Purification 的路由语义还不够精确

计划里说 bridge atoms “registered with both owner theories”。这个说法危险。桥接等式不是都发给两个理论，而是：

```text
t = x + 1      -> LIA/LRA atom
u = f(t)       -> EUF atom
u > 0          -> LIA/LRA atom
```

共享的是 `t`、`u` 这些 purified constants，不是每个 bridge atom 都双发。否则很容易把非纯 atom 送进错误 theory solver。

建议明确成：

```cpp
struct PurifiedAtom {
    TheoryId owner;
    ExprId atom;          // guaranteed pure for owner theory
    std::vector<SharedTermId> sharedTerms;
};

struct SharedTerm {
    ExprId symbol;        // original variable or purification variable
    SortId sort;
    bool isInternal;
    std::bitset<TheoryId> owners;
};
```

红线应改成：

```text
No mixed atom reaches any theory solver.
Bridge definitions are routed only to the theory in whose signature they are pure.
Only purified constants are shared across theories.
```

---

## 3. Theory-propagated equality 的时序没有写对

当前 NO fixpoint 里，理论发现 `a=b` 后直接生成 lemma：

```cpp
¬r1 ∨ ... ∨ ¬rk ∨ Eq(a,b)
```

这个方向对，但要注意：**在 SAT 真正赋值 `Eq(a,b)` 之前，不能把 `a,b` 当作全局已合并共享等式使用。**

正确流程应该是：

```text
Theory T proves a=b
  -> return propagation lemma / theory propagation
  -> SAT assigns Eq(a,b)
  -> assertTheoryLit(Eq(a,b)) is called
  -> SharedEqualityManager unions a,b
  -> Eq(a,b) is sent to all owner theories
```

否则你会在 SAT 尚未接受该等式时，把它提前传播给其他理论，可能破坏 CDCL(T) 的因果关系。

因此 `SharedEqualityManager` 里要区分两类边：

```cpp
enum class EqualitySource {
    SatAssigned,        // equality literal is true in SAT
    TheoryPropagated    // lemma/proof exists, but not necessarily assigned yet
};
```

更简单的第一版：**只允许 SAT-assigned equality 进入 SharedEqualityManager 的 union-find**。理论推导出来的 equality 只返回 lemma，不立刻 union。

---

## 4. NIA/NRA 的 deduced equality 不能来自 sampling/model validation

计划里写 NIA/NRA 可以从 “model validation / grid sampling” report deduced equalities。这个不接受。

Sampling、候选模型验证只能说明“当前模型里相等”，不能说明“公式蕴含相等”。如果把它作为 lemma：

```text
reasons -> Eq(a,b)
```

就是 unsound。

建议第一版这样处理：

```text
EUF: 可以 report equality，但必须能 explain congruence reason。
LRA/RDL: 可以 report equality，但必须能给出 tight-bound explanation。
LIA: 谨慎 report；只复用 LRA relaxation 中可证明的 equality。
NIA/NRA: 第一版不要 report deduced equality，只接受 interface equality / disequality，并在无法组合模型时返回 Unknown 或请求 arrangement。
```

也就是说，`NiaSolver/NraSolver` 的 NO hooks 第一版可以存在，但只能是 skeleton，不能产生理论 lemma。

---

## 5. 非凸 arrangement 的 completeness 条件太弱

计划里说“when solver cannot determine shared term relationship, request arrangement”。这不够完整。非凸组合不是等某个 theory 主动说“不知道”才 split，而是最终需要保证共享变量之间的 arrangement 足以组合模型。

建议加入一个最终检查：

```text
All theory solvers consistent
  -> tryCombineModels()
  -> if shared arrangement incomplete and needed:
        request next arrangement decision
     else if models disagree:
        return conflict / request split
     else:
        SAT
```

也就是说，NO-5 不应该只挂在 `getArrangementRequests()` 上，还要挂在 **model combination failure** 上。

---

## 6. LRA disequality 处理需要补充

计划里说 interface disequality 发送给 owner theories，但 LRA 的 disequality 不是简单一个 bound。

对于 LRA：

```text
assertInterfaceDisequality(x,y)
```

至少要做到：

```text
如果 LRA 已经蕴含 x = y，则 conflict；
否则记录 x ≠ y，用于最终 model construction / validation。
```

否则 SAT 可能决定 `¬Eq(x,y)`，LRA 当前模型却给出 `x=y`，最终 model validator 会失败。

第一版可以保守处理：

```cpp
if (impliesEquality(x,y)) return conflict;
else storeDisequality(x,y);
```

最终模型如果刚好让二者相等，在 dense real 情况下可以尝试扰动；不想做扰动的话，返回 Unknown，但不能返回 Sat。

---

## 7. `TheoryManager::check()` 里的 `changed` 目前是假的

当前伪代码：

```cpp
bool changed = true;
while (changed) {
    changed = false;
    ...
}
```

但里面几乎没有地方设置 `changed = true`。实际上当前设计更像“每次 callback 产生一个 lemma，然后交还 SAT”，不是内部真正 fixpoint。

建议二选一：

### 方案 A：CDCL(T) 风格，推荐第一版

```text
check()
  -> process currently assigned shared equalities
  -> run each theory once
  -> if one lemma/conflict found, return
  -> if arrangement decision needed, request decision
  -> otherwise consistent
```

不要叫内部 fixpoint。

### 方案 B：真正 fixpoint

那就必须允许 theory propagation 直接进入 pending queue，并设置：

```cpp
changed = true;
```

但这个更容易和 SAT assignment 时序打架，第一版不建议。

---

## 8. Phase 顺序建议重排

当前 NO-1 到 NO-7 太平均，容易同时改太多文件。建议改成更稳的三段：

### Stage A：只做 EUF + LRA，先保证 sound

1. Purifier + SharedTermRegistry
2. Shared equality atom registry
3. SharedEqualityManager，只接受 SAT-assigned equality
4. EUF/LRA interface equality hooks
5. Theory-propagated equality lemma，带完整 reasons
6. Final model validator

目标只开放：

```text
QF_UFLRA
QF_UFRDL
```

暂时不要开放：

```text
QF_UFLIA
QF_UFNIA
QF_UFNRA
```

### Stage B：加入 non-convex LIA arrangement

1. Arrangement atom creation
2. SAT decision hook / branching queue
3. `assertInterfaceDisequality`
4. model-combination-driven split request
5. 开放 `QF_UFLIA`

### Stage C：NIA/NRA 只做保守集成

1. 接受 interface equality
2. 不产生 sampling-based equality lemma
3. 不能证明时返回 Unknown
4. 最后再考虑真正的 nonlinear arrangement / lemma generation

---

## 9. Acceptance Criteria 需要加强

当前 acceptance criteria 太少。建议至少加这些：

```text
1. Tautology split is not used as the only arrangement mechanism.
2. Theory-propagated equality is not inserted into union-find before SAT assignment.
3. Every propagated equality has a reconstructable reason.
4. Sampling/model validation never produces a lemma.
5. Purified bridge atoms are routed to exactly one owner theory.
6. Shared equality atoms are canonical and SAT-visible.
7. Backtracking restores:
   - shared equality UF
   - arrangement decisions
   - theory-local interface assertions
   - pending queues
8. Final Sat must validate original unpurified formula.
9. QF_LRA/QF_UF single-theory paths produce byte-level or result-level unchanged behavior.
10. QF_UFNIA/QF_UFNRA remain disabled or return Unknown until sound hooks exist.
```

---

## 最终建议

这版计划 **不能接受为实现计划**，但可以作为架构草稿保留。

我建议你让下一版重点修 5 件事：

1. 把 purification 的 owner-routing 写精确。
2. 删除 `Eq ∨ ¬Eq` 作为 arrangement lemma 的方案，改成 SAT decision hook。
3. 明确 theory equality propagation 的 SAT assignment 时序。
4. 禁止 NIA/NRA sampling 产生 lemma。
5. 第一阶段只承诺 `EUF + LRA/RDL`，不要一口气开放 `UFLIA/UFNIA/UFNRA`。
