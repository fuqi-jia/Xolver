## Phase 2：Branch Lemma + CDCL(T) 搜索闭环（当前阶段）

### 2.1 NiaSolver 注册 TheoryAtomRegistry

- 给 `NiaSolver` 添加 `setRegistry(TheoryAtomRegistry*)` 接口
- 在 `Solver.cpp` 中创建 NiaSolver 后调用 `setRegistry`

### 2.2 Branch lemma：全局有效整数分裂

只做 `x ≤ k ∨ x ≥ k+1`，不做其他分裂形式。

构造：
```cpp
PolyId xPoly = kernel_->mkVar(x);
SatLit le = registry_->getOrCreatePolynomialAtom(
    xPoly, Relation::Leq, mpq_class(k), TheoryId::NIA);
SatLit ge = registry_->getOrCreatePolynomialAtom(
    xPoly, Relation::Geq, mpq_class(k + 1), TheoryId::NIA);
return TheoryLemma{{le, ge}};
```

Dynamic atom 路由回 `assertLit` 时：
- `x ≤ k` → `x - k ≤ 0`（assertLit 已有 poly-rhs 转换）✓
- `x ≥ k+1` → `x - (k+1) ≥ 0` ✓

### 2.3 Duplicate suppression

维护 `BranchSplitKey(var, k)` 集合。若已生成过：
- 尝试另一变量/k
- 无其他选择则返回 Unknown

使用 `TheoryLemmaDatabase::contains` / `insertIfNew` 做 lemma 级去重。

### 2.4 变量选择

1. 有 lower 和 upper 的最大区间变量
2. 只有 lower 或 upper 的变量
3. 出现频率最高的变量
4. fallback：第一个变量

### 2.5 k 值选择

```
有 lower 和 upper:   k = floor((lower + upper) / 2)
只有 lower:          k = lower
只有 upper:          k = upper - 1
无界:                k = 0
```

### 2.6 测试策略

不要声称 branch 能证明 `x²=2` UNSAT（应由 univariate/modular 处理）。

Branch 测试应验证：
- `buildBranchLemma` 正确生成 `x≤k ∨ x≥k+1`
- dynamic atoms 正确注册并路由回 NiaSolver
- duplicate split 不重复生成
- 有界大域问题在 branch 后 domain 能收紧

## Phase 3：Interval Evaluation + 基础 ICP

### 3.1 IntervalZ 数据结构

```cpp
struct IntervalZ {
    mpz_class lo;  // inclusive
    mpz_class hi;  // inclusive
    bool isEmpty() const { return lo > hi; }
};
```

### 3.2 IntervalEvaluator

Over-approx interval arithmetic（绝不允许 under-approx）：

```
add:  [a,b] + [c,d] = [a+c, b+d]
sub:  [a,b] - [c,d] = [a-d, b-c]
neg:  -[a,b] = [-b,-a]
mul:  [a,b] * [c,d] = [min(ac,ad,bc,bd), max(ac,ad,bc,bd)]
pow2: 0∈[l,u] → [0, max(l²,u²)]; else → [min(l²,u²), max(l²,u²)]
pow3: [l³, u³]
```

### 3.3 Constraint interval status

| rel | definitely violated | definitely satisfied |
|-----|---------------------|---------------------|
| `≤` | `interval.lo > 0` | `interval.hi ≤ 0` |
| `≥` | `interval.hi < 0` | `interval.lo ≥ 0` |
| `=` | `0 ∉ [lo,hi]` | `[lo,hi] == [0,0]` |
| `≠` | `0 ∉ [lo,hi]` | `[lo,hi] == [0,0]` |

### 3.4 Sum-of-squares propagation

- `x² + y² ≤ c, c < 0` → UNSAT
- `x² + y² ≤ c, c ≥ 0` → 每个变量 `∈ [-⌊√c⌋, ⌊√c⌋]`

### 3.5 禁止 Product reverse pruning

Phase 3 明确不做 `xy=c` 的反向乘法传播。原因：跨 0、负数、除法边界、不等式方向都容易写错。

## Phase 4：Interval B&B

### 4.1 入口

`allFinite(vars) == true` 且 `totalSize > enumerationThreshold`

### 4.2 内部状态（与顶层区分）

```cpp
enum class BoxResult { Sat, Closed, Unknown };
```

内部只返回 `Closed`/`Sat`/`Unknown`。
顶层只有全部 boxes `Closed` 时才返回 `UnsatComplete(allActiveReasonsConflict)`。

### 4.3 算法

```cpp
BoxResult solveBox(box, depth):
    if budget exceeded: return Unknown

    if any constraint definitely violated by interval eval:
        return Closed

    if box is singleton:
        if exactValidate(assignment): return Sat
        return Closed

    split box into left/right by variable midpoint

    r1 = solveBox(left, depth+1)
    if r1 == Sat: return Sat

    r2 = solveBox(right, depth+1)
    if r2 == Sat: return Sat

    if r1 == Closed && r2 == Closed: return Closed
    return Unknown
```

### 4.4 Conflict 策略（第一版）

- 不尝试在子 box 内部构造精细的 boxConflict
- `solveBox` 内部只返回 `Closed`/`Sat`/`Unknown`
- 顶层若所有 boxes 都 `Closed`，才返回 `UnsatComplete(allActiveReasonsConflict)`
- 使用 coarse conflict（所有 active normalized reasons 的否定）

## Phase 5：代数推理增强

等 Phase 1-4 稳定后再做。

### 5.1 多变量 GCD conflict

对 normalized equality `p = 0`：
- `p = q + c`，`q` 无常数项
- `g = gcd(所有 q 的系数)`
- 若 `g ∤ -c` → UNSAT

### 5.2 Factor rules

- `p*q = 0` → lemma: `¬(pq=0) ∨ (p=0) ∨ (q=0)`
- `p*q != 0` → lemmas: `¬(pq≠0) ∨ (p≠0)`, `¬(pq≠0) ∨ (q≠0)`
- 需要 dynamic polynomial atom

### 5.3 Modular reasoning 扩展

扩展到 `evaluateMod` 通用接口，支持更多变量和模数。

## Phase 6：Local Search SAT Finder

当前 NiaLocalSearch 只尝试固定候选点。增强：

1. **初始点**: domain midpoint, small integers, univariate roots, random
2. **Objective**: `Σ violation²`
3. **Move**: 单变量小步调整（±1），按 violation 最大约束挑变量
4. **Restart**: 随机重启避免局部最优
5. **Budget**: 固定迭代次数
6. **任何找到的 model 必须 exact-validate**
7. **失败是 NoChange/Unknown，绝不 UNSAT**

---

## 正确性红线（不可违反）

1. **No incomplete reasoning may return UNSAT.**
   - UnsatComplete 只能来自：constant contradiction, empty roots, modular contradiction, GCD contradiction, finite-domain exhaustion, interval B&B complete closure.
2. **Stub polynomial backend returns nullopt, never fake values.**
3. **Domain conflicts must include complete reasons.**
4. **Dynamic atoms must be real registered atoms.**
   - `getOrCreatePolynomialAtom` 必须创建真正的 SAT var，route 回 `NiaSolver::assertLit`。
5. **Every SAT result must exact validate.**
   - `IntegerModelValidator::validate()` 必须对所有 normalized constraints 通过。
6. **Branch lemma must be globally valid.**
   - `x ≤ k ∨ x ≥ k+1` 对所有整数 x 成立，与当前赋值无关。
7. **Interval evaluation must be over-approx only.**
   - 不允许 under-approx；宁可保守不要激进。
8. **Product reverse pruning is prohibited in Phase 3.**
   - Phase 3 只做 interval evaluation 和 safe pruning。
