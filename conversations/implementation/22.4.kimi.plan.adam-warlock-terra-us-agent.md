## 1. Decision

**Scope: Strengthen pure QF_UF only.**

Mixed theories (QF_UFLIA / QF_UFLRA / QF_UFNIA / QF_UFNRA) return `Unknown` unconditionally. No pure fallback. No Nelson-Oppen.

NLColver has a **working EUF V1** for pure `QF_UF`:
- `EufTermManager`: hash-consed interning of Variable, UFApply, ConstBool
- `EGraph`: union-find + full-rebuild congruence closure via repeated signature-table scan

## 3. Design Invariants (Non-Negotiable)

## 4. Phase 1: Fix Explanation BFS

**File**: `src/theory/euf/EGraph.cpp`

**Problem**: `explainPath()` uses `visitedPairs` that returns `{false,{}}` on benign revisits in undirected explanation graphs.

**Fix**:
- Replace `visitedPairs` + early-abort with **standard BFS visited-node tracking**.
- Skip `next` if it already has a `parentTerm`; do not abort.
- **Keep recursive explanation protection**: `explainMergeReason` maintains an `activePairs` set. Before recursing into `explainEquality(argA, argB)`, push `(argA,argB)`; pop on return. If already active, return `{false,{}}`.

**Pseudocode**:
```cpp
ExplainResult EGraph::explainMergeReason(
    const MergeReason& reason,
    const EufTermManager& tm,
    std::unordered_set<uint64_t>& activePairs) {
    if (reason.kind == MergeReasonKind::AssertedEquality) {
        return {true, {reason.assertedLit}};
    }
    std::vector<SatLit> reasons;
    for (auto [argA, argB] : reason.argPairs) {
        if (argA == argB) continue;
        uint64_t pk = pairKey(argA, argB);
        if (!activePairs.insert(pk).second) return {false, {}};
        auto sub = explainEquality(argA, argB, tm, activePairs);
        activePairs.erase(pk);
        if (!sub.ok) return {false, {}};
        appendUnique(reasons, sub.reasons);
    }
    return {true, std::move(reasons)};
}

ExplainResult EGraph::explainPath(
    EufTermId start, EufTermId target,
    const EufTermManager& tm,
    std::unordered_set<uint64_t>& activePairs) {
    std::unordered_map<EufTermId, EufTermId> parentTerm;
    std::unordered_map<EufTermId, size_t> parentEdge;
    std::queue<EufTermId> q;
    std::unordered_set<EufTermId> visited;

    q.push(start); visited.insert(start); parentTerm[start] = start;
    while (!q.empty()) {
        EufTermId cur = q.front(); q.pop();
        for (size_t ei : adj[cur]) {
            const auto& edge = proofEdges_[ei];
            EufTermId next = (edge.u == cur) ? edge.v : edge.u;
            if (visited.count(next)) continue;
            visited.insert(next);
            parentTerm[next] = cur;
            parentEdge[next] = ei;
            if (next == target) {
                // reconstruct path: collect reasons from parentEdge chain
                std::vector<SatLit> reasons;
                EufTermId node = target;
                while (node != start) {
                    size_t eidx = parentEdge[node];
                    auto exp = explainMergeReason(
                        proofEdges_[eidx].reason, tm, activePairs);
                    if (!exp.ok) return {false, {}};
                    appendUnique(reasons, exp.reasons);
                    node = parentTerm[node];
                }
                return {true, std::move(reasons)};
            }
            q.push(next);
        }
    }
    return {false, {}};
}
```

## 5. Phase 2: Bool Two-Valued Semantics

### 5a. Bool atom convention

For every Bool-valued term `b`:
- **Canonical SAT atom**: `lit(b) := (= b true)`.
- This atom is registered as a EUF Bool-term atom.
- Model construction and EUF state interpret:
  - `lit(b) = true`  → `b = true`
  - `lit(b) = false` → `b = false`

### 5b. Atomizer: rewrite Bool equality / distinct propositionally

**File**: `src/sat/Atomizer.cpp`

When `defaultTheory_ == TheoryId::EUF` and all arguments of `=` or `distinct` have Bool sort:

**Bool equality** (n-ary):
```text
(= b1 b2 ... bn)
=> ∧_{i=1}^{n-1} (bi ↔ b_{i+1})

Clause encoding for (x ↔ y):
  (¬x ∨ y) ∧ (x ∨ ¬y)
```
Each `bi` is the canonical Bool atom `lit(bi)`.

**Bool distinct** (n-ary):
```text
n <= 1     => true (vacuously)
n = 2      => (b1 XOR b2)
n >= 3     => false

Clause encoding for (x XOR y):
  (x ∨ y) ∧ (¬x ∨ ¬y)
```

These are compiled to SAT clauses directly, **not** registered as ordinary EUF atoms.

**Important**: For formula-position Bool terms like `(p a)` or bare `p`, Atomizer already converts them to `(= (p a) true)` or `(= p true)`. This is the canonical Bool atom. The Bool equality/distinct rewrite above operates on these canonical atoms.

### 5c. EufSolver: Bool formula-position atoms merge true/false

In `EufSolver::check()`, when processing trail entries for Bool formula-position atoms:

```cpp
// lit(b) is the atom (= b true)
if (value == true) {
    // b = true
    merge(b, trueConstant, reason);
} else {
    // lit(b) = false means b = false
    merge(b, falseConstant, reason);
}
```

This ensures EGraph state is complete: `b` is always either in the `true` class or the `false` class. The existing `true != false` check after `closeCongruence` will catch contradictions.

## 6. Phase 3: Sort-Correct Model Construction

### 6a. Model data structures

**File**: `src/theory/euf/EufTypes.h`

```cpp
struct EufClassValue {
    SortId sort;
    std::string value;
};

struct EufFuncEntry {
    std::vector<std::string> argValues;
    std::string resultValue;
};

struct EufFuncInterp {
    std::vector<EufFuncEntry> entries;
    std::string defaultValue;
};

struct EufModel {
    std::unordered_map<EClassId, EufClassValue> classValues;
    std::unordered_map<FuncSymbolId, EufFuncInterp> functions;
};

struct BuildModelResult {
    bool ok;
    EufModel model;
    std::string error;
};
```

### 6b. EGraph model builder

**File**: `src/theory/euf/EGraph.cpp`

Implement `BuildModelResult buildModel(const EufTermManager& tm) const`:

1. **Determine class membership**: Group terms by root class. For each class, verify all members have the same `sort`. If a class is sort-mixed, return `ok=false` with error.

2. **Assign class values by sort**:
   - **Bool sort**:
     - If class contains `trueConstant` → `"true"`
     - If class contains `falseConstant` → `"false"`
     - Otherwise → `"false"` (default choice)
   - **Uninterpreted sort S**:
     - `"S!val_N"` where `N` is a per-sort counter.
   - **Other sorts** (should not appear in pure QF_UF; if they do, return `ok=false`).

3. **Function interpretations**: For each application term `t = f(args)`:
   - `resultClass = find(t)`.
   - `argValues` = map each arg through `find → classValues.value`.
   - Append entry to `functions[f].entries`.
   - **Default value** for `f`:
     - If result sort is Bool → `"false"`
     - If result sort is uninterpreted S → `"S!val_default_N"`

4. **Collision check**: If two entries have same `argValues` but different `resultValue`, return `ok=false` (egraph bug).

### 6c. EufSolver integration

- `EufSolver::check()` calls `buildModel()` after confirming consistency.
- If `!result.ok`, log diagnostic but **still return `Consistent`** (per I13).
- Store successful model in `eufModel_`.
- Expose `const EufModel& eufModel() const`.

### 7b. Propagation detection (after all conflict checks)

In `EufSolver::check()`, the order is:
```cpp
// 1. assert equalities
// 2. closeCongruence
// 3. check true != false conflict
// 4. check asserted disequality conflict
// 5. THEN propagation over registered atoms
```

Propagation loop:
```cpp
for (const auto& rec : allRegisteredAtoms_) {
    const auto& p = std::get<EufAtomPayload>(rec.payload);
    if (p.rel != Relation::Eq) continue;

    EufTermId lhs = termManager_.intern(p.lhs, *coreIr_);
    EufTermId rhs = termManager_.intern(p.rhs, *coreIr_);
    if (lhs == NullEufTerm || rhs == NullEufTerm) continue;
    if (!egraph_.same(lhs, rhs)) continue;

    SatLit atomLit = SatLit::positive(rec.satVar);

    // MANDATORY: query SAT assignment
    auto satVal = satAssignment_.value(rec.satVar);  // or however SAT state is queried
    if (satVal == LBool::True) {
        continue;  // already satisfied
    }
    if (satVal == LBool::False) {
        // Conflict: reasons imply atomLit, but SAT says ¬atomLit
        auto er = egraph_.explainEquality(lhs, rhs, termManager_);
        if (!er.ok) continue;
        std::vector<SatLit> clause;
        for (SatLit r : er.reasons) clause.push_back(r.negated());
        clause.push_back(atomLit);
        return TheoryCheckResult::mkConflict(TheoryConflict{std::move(clause)});
    }
    // satVal == Undef
    if (emittedPropLemmas_.contains(rec.satVar)) continue;

    auto er = egraph_.explainEquality(lhs, rhs, termManager_);
    if (!er.ok) continue;
    std::vector<SatLit> clause;
    for (SatLit r : er.reasons) clause.push_back(r.negated());
    clause.push_back(atomLit);
    emittedPropLemmas_.insert(rec.satVar);
    return TheoryCheckResult::mkLemma(TheoryLemma{std::move(clause)});
}
```

**If SAT assignment query is not available in the current interface**, Phase 4 must be **deferred entirely** — do not implement propagation without it. Document as TODO.
