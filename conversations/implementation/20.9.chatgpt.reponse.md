还要 **Revise**。这版虽然已经补了 co-register、mirror bounds、effective relation、guarded extension 这些大方向，但**最大 blocker 仍然存在**：`LinearizationResult` 仍然是 `std::vector<TheoryLemma>`，没有 `CutCacheKey` metadata，后面的 cache mark 逻辑仍然无法正确实现。

---

# 1. `LinearizationResult` 仍然不对

文件里 Step 2 仍然写的是：

```cpp
struct LinearizationResult {
    LinearizationStatus status;
    std::vector<TheoryLemma> lemmas;
};
```

但你的 cache API 需要：

```cpp
NonlinearTermKey term;
SatLit nonlinearReason;
std::vector<mpq_class> boundValues;
int cutIndex;
```

`TheoryLemma` 里没有这些信息。所以一旦 adapter 收到一个 `TheoryLemma`，它无法知道该调用哪个：

```cpp
markEmitted(term, nonlinearReason, boundValues, cutIndex)
```

这个问题必须改成：

```cpp
struct CutCacheKey {
    NonlinearTermKey term;
    SatLit nonlinearReason;
    std::vector<mpq_class> boundValues;
    int cutIndex;
};

struct PendingLinearizationLemma {
    TheoryLemma lemma;
    std::optional<CutCacheKey> cacheKey; // abstraction lemma 可为 nullopt
};

struct LinearizationResult {
    LinearizationStatus status;
    std::vector<PendingLinearizationLemma> lemmas;
};
```

然后 adapter 里处理的是 `item`，不是 `lemma`：

```cpp
for (auto& item : lr.lemmas) {
    const auto& lemma = item.lemma;

    if (!lemmaDb.contains(lemma)) {
        lemmaDb.insertIfNew(lemma);
        pendingLinLemmas_.push_back(std::move(item.lemma));

        if (item.cacheKey) {
            linAdapter_->markEmitted(*item.cacheKey);
        }
    }
}
```

---

# 2. 伪代码里仍然把 `lemma` 和 `item.cacheKey` 混在一起

你现在的集成伪代码仍然是类似这种形态：

```cpp
for (auto& lemma : lr.lemmas) {
    ...
    pendingLinLemmas_.push_back(std::move(lemma));
    linAdapter_->markEmitted(*item.cacheKey);
}
```

这里 `item` 根本没有定义。这个就是因为结果类型还没改成 `PendingLinearizationLemma`。所以这一段必须整体改掉，而不是只改一句。

---

# 3. mirror step 已经对了，但要确认接口也有 active assignments

这版已经写清楚：adapter 先扫描 active assignments 生成 mirror lemmas，再运行 linearizer，这个方向正确。

但要确保接口真的包含 active assignment 输入，而不是仍然只有：

```cpp
run(normalized, domains_, lemmaDb)
```

建议固定为两个独立函数：

```cpp
std::vector<TheoryLemma> mirrorActiveLinearBounds(
    const std::vector<ActiveTheoryAssignment>& activeAssignments,
    TheoryId targetLinearTheory);

LinearizationResult runLinearizer(
    const std::vector<NormalizedNiaConstraint>& nonlinearConstraints,
    const DomainStore& domains,
    TheoryLemmaDatabase& lemmaDb);
```

这样不会把 mirror 和 nonlinear cut generation 搅在一起。

---

