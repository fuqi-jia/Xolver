# Plan: Optimized Incremental E-Graph for NLColver EUF

## Goal
Upgrade the EUF e-graph from the current "correct but slow" baseline (full signature-table rebuild on backtrack + global BFS over mergeRecords for explanation) to a production-grade design with:

1. **Rollbackable signature table** — no full rebuild on backtrack
2. **Per-app current-signature tracking with rollback** — correct stale-owner handling
3. **ProofForest** — incremental adjacency for O(tree-size) explanation instead of O(#merges) BFS
4. **Recursive explanation with local memo + reason dedup** — compact conflict clauses

## Current Baseline (what exists today)

Files involved:
- `src/theory/euf/IncrementalEGraph.h/.cpp` — congruence closure engine
- `src/theory/euf/EufTypes.h` — types (EGraphSnapshot, MergeRecord, MergeReason, etc.)
- `src/theory/euf/EufSolver.h/.cpp` — CDCL(T) integration

Current behavior:
- `sigTable_` is a plain `unordered_map<AppSignature, EufTermId>`
- `rollback()` clears `sigTable_` and sets `sigTableDirty_ = true`
- `processMergeQueue()` calls `ensureFreshSignatureTable()` which does **O(A × arity)** full rebuild
- `explainEquality()` scans **all** `mergeRecords_` to build adjacency, then BFS — **O(#merges)**

## Implementation Approach (single recommended design)

### Phase A: RollbackSignatureTable + currentSig (signature table optimization)

**New file: `src/theory/euf/RollbackSignatureTable.h/.cpp`**

```cpp
class RollbackSignatureTable {
public:
    std::optional<EufTermId> find(const AppSignature& sig) const;
    void insertOrAssign(const AppSignature& sig, EufTermId owner);
    void eraseIfOwner(const AppSignature& sig, EufTermId owner);
    size_t snapshot() const;
    void rollback(size_t snap);
    void clear();

private:
    enum class ChangeKind { InsertNew, Overwrite, Erase };
    struct Change {
        ChangeKind kind;
        AppSignature sig;
        std::optional<EufTermId> oldOwner;
    };
    std::unordered_map<AppSignature, EufTermId, AppSignatureHash> table_;
    std::vector<Change> trail_;
};
```

**Modify: `src/theory/euf/EufTypes.h`**
- Extend `EGraphSnapshot` with `sigTableSnap`, `currentSigSnap`, `proofForestSnap`
- Keep existing `MergeRecord`, `MergeReason`, `ExplainResult` structs

**Modify: `src/theory/euf/IncrementalEGraph.h`**
- Replace `sigTable_` + `sigTableDirty_` with `RollbackSignatureTable sigTable_`
- Add `std::vector<std::optional<AppSignature>> currentSig_` + `currentSigTrail_`
- Add `ProofForest proofForest_` (forward-declare or include)
- Add `refreshSignature(EufTermId app)` method
- Remove `ensureFreshSignatureTable()` (no longer needed)
- Remove `sigTableDirty_`
- Update `snapshot()` and `rollback()` signatures

**Modify: `src/theory/euf/IncrementalEGraph.cpp`**

1. `clear()`: clear `sigTable_`, `currentSig_`, `currentSigTrail_`, `proofForest_`
2. `ensureTerm()`: resize `currentSig_` alongside `members_`; after adding a new app term, call `refreshSignature(t)` instead of setting `sigTableDirty_`
3. Implement `refreshSignature(app)`:
   - If `currentSig_[app]` exists, `sigTable_.eraseIfOwner(*currentSig_[app], app)` then `setCurrentSig(app, nullopt)`
   - Compute new signature via `computeSignature(app)`
   - Look up owner in `sigTable_`
   - If no owner → `insertOrAssign(sig, app)`, `setCurrentSig(app, sig)`
   - If owner exists but `currentSig_[owner] != sig` (stale) → `insertOrAssign(sig, app)`, `setCurrentSig(app, sig)`
   - If owner exists and is valid but `!same(owner, app)` → enqueue congruence merge, **do not register app yet**
   - If same class → `insertOrAssign(sig, owner)`, `setCurrentSig(app, sig)`
4. `processMergeQueue()`: **remove** `ensureFreshSignatureTable()` call; process merges directly
5. `processCongruenceQueue()`: for each app, call `refreshSignature(app)` (replaces the old inline logic)
6. `merge()`: **remove** `ensureFreshSignatureTable()` call
7. `snapshot()`: capture `sigTable_.snapshot()`, `currentSigTrail_.size()`, `proofForest_.snapshot()`
8. `rollback()`: rollback in dependency order:
   ```
   mergeQueue_.resize()
   congruenceQueue_.resize()
   proofForest_.rollback()
   mergeRecords_.resize()
   rollbackCurrentSig()
   sigTable_.rollback()
   rollbackMembers()
   uf_.rollback()
   ```

### Phase B: ProofForest (explanation optimization)

**New file: `src/theory/euf/ProofForest.h/.cpp`**

```cpp
class ProofForest {
public:
    void addEdge(EufTermId u, EufTermId v, const MergeReason& reason);
    size_t snapshot() const;
    void rollback(size_t snap);
    void clear();

    // Returns edge IDs along the path u→v in the active forest.
    // Empty if no path (should not happen for same-class terms).
    std::vector<size_t> path(EufTermId u, EufTermId v) const;

    size_t activeEdgeCount() const { return activeEdgeCount_; }

private:
    struct Edge {
        EufTermId u;
        EufTermId v;
        MergeReason reason;
    };
    std::vector<Edge> edges_;
    std::vector<std::vector<std::pair<EufTermId, size_t>>> adj_;
    size_t activeEdgeCount_ = 0;
};
```

**Rollback semantics:** `rollback(snap)` sets `activeEdgeCount_ = snap`. Adjacency lists are append-only; BFS skips `edgeId >= activeEdgeCount_`.

**Modify: `src/theory/euf/IncrementalEGraph.cpp`**

1. In `processMergeQueue()`, after verifying `same(m.a, m.b)` is false, before `uf_.unite()`, call:
   ```cpp
   proofForest_.addEdge(m.a, m.b, m.reason);
   ```
2. Replace `explainEquality()` implementation:
   - If `a == b` → return `{true, {}}`
   - If `!same(a, b)` → return `{false, {}}`
   - Call `proofForest_.path(a, b)` to get edge IDs
   - For each edge ID, call `explainEdgeReason(edge.reason)` recursively
   - Deduplicate reasons via `ReasonSet`
3. Replace `explainMergeReason()` with `explainEdgeReason(edgeId, ExplainContext& ctx)`:
   - Uses `ExplainContext` local memo to cache `(a,b) → reasons` within a single top-level call
   - For `AssertedEquality` → returns `{assertedLit}`
   - For `Congruence` → recursively explains each arg pair, merges reasons
   - Uses `activePairs` cycle guard (same as today)
4. Remove the old `explainMergeReason()` BFS-over-all-mergeRecords logic entirely

## Key design invariants (red lines)

1. `RollbackSignatureTable::eraseIfOwner` must check owner equality before erasing
2. `currentSig_` must be rolled back alongside `sigTable_` — never leave them out of sync
3. `ProofForest::addEdge` only called when `!same(a,b)` — active graph is always a forest
4. Congruence `argPairs` frozen at merge time — never recompute from current reps during explanation
5. `refreshSignature` does **not** register an app when it discovers a congruence merge — the merge will change reps, then re-processing will register correctly
6. Backtrack order: high-level structures first (`mergeQueue`, `proofForest`, `sigTable`), then `uf_`
