# EUF Solver Enhancement Plan — Revision 3

## 1. Decision

**Scope: Strengthen pure QF_UF only.**

Mixed theories (QF_UFLIA / QF_UFLRA / QF_UFNIA / QF_UFNRA) remain `Unknown` in this revision. No pure fallback. No Nelson-Oppen.

## 2. Current State

NLColver has a **working EUF V1** for pure `QF_UF`:
- `EufTermManager`: hash-consed interning of Variable, UFApply, ConstBool
- `EGraph`: union-find + full-rebuild congruence closure via repeated signature-table scan
- `EufSolver`: CDCL(T) TheorySolver interface, full-rebuild `check()`, conflict generation
- `Atomizer`: EUF atom extraction with n-ary `=` / `distinct` support
- 17 regression tests + 18 unit tests all pass

**One confirmed correctness bug**: `EGraph::explainPath()` BFS aborts with `{false,{}}` on benign revisits in undirected explanation graphs.

## 3. Design Invariants (Non-Negotiable)

| # | Invariant |
|---|-----------|
| I1 | **Atomizer owns EUF dedup** via `getOrCreateEufAtom`. Registry only records. No `Registry::getOrCreateEufAtom`. |
| I2 | **Bool sort ID** comes from `adapter.getBoolSortId()`, never hardcoded. |
| I3 | **`EGraph` owns `termClass_`** mapping `EufTermId → EClassId`. |
| I4 | **`check()` order**: intern terms → `addTerm` → assert equalities → `closeCongruence` → check true≠false → check disequalities. |
| I5 | **`closeCongruence` returns `CloseStatus`**; header and implementation match; callers check it. |
| I6 | **Explanation cycle returns `{false, {}}`** (failure), never `{true, {}}`. |
| I7 | **`pop()` clamps** with `min(scopeLimit, trailSize)`. |
| I8 | **Synthetic allocator** has `reset()` and collision asserts against sentinels. `Atomizer::reset()` clears `eufAtomDedup_` and resets allocator. |
| I9 | **`distinct` with ≤1 arg** is vacuously true. |
| I10 | **C++20** required in CMake. |
| I11 | **Bool is a two-valued sort.** Bool equality / distinct must enforce this cardinality, not treat Bool as an infinite uninterpreted domain. |
| I12 | **Model construction never changes SAT/UNSAT result.** If model build fails after consistency is confirmed, return SAT with empty/diagnostic model, not UNSAT or crash. |
| I13 | **Mixed theories return `Unknown`.** No pure fallback in this revision. |

## 4. Phase 1: Fix Explanation BFS

**File**: `src/theory/euf/EGraph.cpp`

**Problem**: `explainPath()` uses `visitedPairs` that returns `{false,{}}` when any undirected edge is re-encountered. In an undirected explanation graph, multiple paths are normal.

**Fix**:
- Replace `visitedPairs` + early-abort with **standard BFS visited-node tracking**.
- `parentTerm` / `parentEdge` already track the BFS tree; simply skip `next` if it already has a parent.
- **Keep recursive explanation protection**: in `explainMergeReason`, before recursing into `explainEquality(argA, argB)`, push `(argA, argB)` onto an `activePairs` set; pop on return. If the pair is already active, return `{false,{}}` to prevent infinite recursion from cyclic congruence reasoning.

**Pseudocode**:
```cpp
ExplainResult EGraph::explainMergeReason(
    const MergeReason& reason,
    const EufTermManager& tm,
    std::unordered_set<uint64_t>& activePairs) {
    if (reason.kind == MergeReasonKind::AssertedEquality) {
        return {true, {reason.assertedLit}};
    }
    std::vector<SatLit> reasons;
    for (auto [argA, argB] : reason.argPairs) {
        if (argA == argB) continue;
        auto sub = explainEquality(argA, argB, tm, activePairs);
        if (!sub.ok) return {false, {}};
        appendUnique(reasons, sub.reasons);
    }
    return {true, std::move(reasons)};
}

ExplainResult EGraph::explainPath(
    EufTermId start, EufTermId target,
    const EufTermManager& tm,
    std::unordered_set<uint64_t>& activePairs) {
    // Standard BFS
    std::unordered_map<EufTermId, EufTermId> parentTerm;
    std::unordered_map<EufTermId, size_t> parentEdge;
    std::queue<EufTermId> q;
    std::unordered_set<EufTermId> visited;

    q.push(start); visited.insert(start); parentTerm[start] = start;
    while (!q.empty()) {
        EufTermId cur = q.front(); q.pop();
        for (size_t ei : adj[cur]) {
            const auto& edge = proofEdges_[ei];
            EufTermId next = (edge.u == cur) ? edge.v : edge.u;
            if (visited.count(next)) continue;
            visited.insert(next);
            parentTerm[next] = cur;
            parentEdge[next] = ei;
            if (next == target) { /* reconstruct */ }
            q.push(next);
        }
    }
    return {false, {}};  // no path found
}
```

## 5. Phase 2: Sound Bool Semantics

**Problem**: Treating Bool as an infinite uninterpreted sort is unsound.

Counter-example:
```smt2
(declare-const p Bool) (declare-const q Bool)
(assert (distinct p true))
(assert (distinct q true))
(assert (distinct p q))
```
→ Must be `unsat` (Bool has only two values).

### 5a. Atomizer: rewrite Bool equality / distinct propositionally

**File**: `src/sat/Atomizer.cpp`

When `defaultTheory_ == TheoryId::EUF` and both sides of `=` / `distinct` have Bool sort:

**Bool equality**:
```text
(= b1 b2)     →  (b1 ↔ b2)   via SAT encoding
(= b1 b2 b3)  →  (b1↔b2) ∧ (b2↔b3)
```
Each `bi` is first atomized as a Bool atom (its own SAT variable). The equality is compiled to SAT clauses directly, **not** registered as a EUF atom.

**Bool distinct**:
```text
(distinct b1)       →  true
(distinct b1 b2)    →  (b1 XOR b2)   via SAT encoding
(distinct b1 b2 b3) →  false        (Bool cardinality = 2)
```

**Implementation**: In `atomize()` or a new `atomizeBoolEq` helper:
- Detect if `lhs` and `rhs` sorts are Bool (using `ir.get(eid).sort == boolSortId_`).
- If yes, generate SAT clauses directly instead of calling `getOrCreateEufAtom`.
- For n-ary Bool equality: encode pairwise `↔` via Tseitin.
- For n-ary Bool distinct: encode pairwise `XOR` via Tseitin; if arity > 2, return `false` literal.

### 5b. EufSolver: treat true/false as distinct Bool values

`EufTermManager` already interns `true` and `false` as special constants. `EufSolver::check()` already checks `same(trueConstant, falseConstant)` after `closeCongruence`.

This is correct for the true/false conflict path. The new Bool semantics in the Atomizer ensure that `distinct p true` is never seen as a EUF disequality — it becomes a SAT-level constraint that forces `p=false`.

### 5c. Tests

Add to `tests/unit/test_euf.cpp`:

| Test | SMT2 | Expected |
|------|------|----------|
| Bool distinct cardinality UNSAT | `(distinct p true), (distinct q true), (distinct p q)` | `unsat` |
| Bool ternary distinct UNSAT | `(distinct p q r)` | `unsat` |
| Bool eq false UNSAT | `p, (= p false)` | `unsat` |
| Bool distinct true SAT | `(distinct p true)` | `sat`, model has `p=false` |

## 6. Phase 3: Sort-Correct Model Construction

### 6a. Model data structures

**File**: `src/theory/euf/EufTypes.h`

```cpp
struct EufClassValue {
    SortId sort;
    std::string value;
};

struct EufFuncEntry {
    std::vector<std::string> argValues;
    std::string resultValue;
};

struct EufFuncInterp {
    std::vector<EufFuncEntry> entries;
    std::string defaultValue;
};

struct EufModel {
    std::unordered_map<EClassId, EufClassValue> classValues;
    std::unordered_map<FuncSymbolId, EufFuncInterp> functions;
};
```

### 6b. EGraph model builder

**File**: `src/theory/euf/EGraph.cpp`

Implement `BuildModelResult buildModel(const EufTermManager& tm) const`:

1. **Collect root classes**: For each class `c` where `ufFind(c) == c`, assign a value based on sort:
   - If sort is Bool:
     - If `c` contains `trueConstant` → `"true"`
     - If `c` contains `falseConstant` → `"false"`
     - Otherwise → this is an error (Bool class without true/false means Bool constraints were incomplete); return `ok=false`.
   - If sort is uninterpreted `S` → `"S!val_N"` where `N` is a per-sort counter.

2. **Function interpretations**: For each application term `t = f(args)`:
   - `resultClass = find(t)`.
   - `argValues` = map each arg through `find → classValues.value`.
   - Append entry to `functions[f].entries`.
   - `defaultValue` for `f` is a fresh value of `f`'s result sort.

3. **Collision check**: If two entries in `functions[f]` have the same `argValues` but different `resultValue`, return `ok=false` (this would be an egraph bug).

```cpp
struct BuildModelResult {
    bool ok;
    EufModel model;
    std::string error;
};
```

### 6c. EufSolver integration

- `EufSolver::check()` calls `buildModel()` after confirming consistency.
- If `!result.ok`, log diagnostic but still return `Consistent` (per I12). Do not turn SAT into UNSAT.
- Store successful model in member `eufModel_`.
- Expose `const EufModel& eufModel() const`.

### 6d. Solver API wiring

**File**: `src/api/Solver.cpp`

In `checkSatInternal()`, after SAT is confirmed for QF_UF:
- Look up `EufSolver` from `TheoryManager`.
- If `eufModel()` is available, convert to public `Model`.
- `getModel()` returns the stored model.

### 7b. Propagation detection

In `EufSolver::check()`, after `closeCongruence()`:

```cpp
for (const auto& rec : allRegisteredAtoms_) {
    const auto& p = std::get<EufAtomPayload>(rec.payload);
    if (p.rel != Relation::Eq) continue;

    EufTermId lhs = termManager_.intern(p.lhs, *coreIr_);
    EufTermId rhs = termManager_.intern(p.rhs, *coreIr_);
    if (lhs == NullEufTerm || rhs == NullEufTerm) continue;
    if (!egraph_.same(lhs, rhs)) continue;

    SatLit atomLit = SatLit::positive(rec.satVar);

    // Optional: if SAT assignment query available:
    //   if assigned true → skip
    //   if assigned false → this is actually conflict (handle separately)

    auto er = egraph_.explainEquality(lhs, rhs, termManager_);
    if (!er.ok) continue;

    std::vector<SatLit> clause;
    for (SatLit r : er.reasons) clause.push_back(r.negated());
    clause.push_back(atomLit);
    return TheoryCheckResult::mkLemma(TheoryLemma{std::move(clause)});
}
```

**Critical**: Use `atomLit = SatLit::positive(rec.satVar)`, not `assertedLit` from trail. `assertedLit` only makes sense for already-assigned literals.
