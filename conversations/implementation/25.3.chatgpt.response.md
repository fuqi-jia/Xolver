## 结论：**接受 Stage A；整体计划有条件接受**

这一版可以进入实现。前两轮的关键 blocker 基本都修掉了：shared equality 已经改成 `TheoryId::Combination`，Stage B 禁止 unit-clause fallback，EUF 只传播 registered shared constants，EUF model 改成 partition，pending events 也补了 decision level 和 rollback。

但我建议实现前再补 **3 个小但关键的细节**。这些不是推翻方案的问题，而是防止 Stage A 实现时出现隐蔽 bug。

---

## 1. Stage A 目标里包含 `QF_UFRDL`，但 RDL hooks 没写

当前 Stage A 说目标是：

```text
QF_UFLRA and QF_UFRDL fully functional
```

但 Phase A-4 只写了：

```text
EufSolver
LraSolver
```

没有写 `RdlSolver` 的 NO hooks。

这里必须二选一：

### 推荐方案：Stage A 先只启用 `QF_UFLRA`

```text
Stage A target:
QF_UFLRA fully functional.
QF_UFRDL remains disabled until RdlSolver implements interface hooks.
```

### 或者补充 RDL hooks

```cpp
class RdlSolver : public TheorySolver {
public:
    TheoryCheckResult assertInterfaceEquality(...) override;
    TheoryCheckResult assertInterfaceDisequality(...) override;
    std::vector<SharedEqualityPropagation> getDeducedSharedEqualities() override;
    std::optional<TheoryModel> getModel(...) const override;
};
```

如果不补这一点，`QF_UFRDL` 会变成“Solver.cpp 注册了，但理论层并不完整”。

---

## 2. Purifier 对 shared equality 的识别最好改成两阶段

当前 Purifier 伪代码里写：

```text
If node is equality between two shared constants a, b:
    emit shared equality atom with owner = TheoryId::Combination
```

这个方向对，但实现时有一个时序问题：一个变量是否 shared，可能要等整条公式或整批 assertions 扫完之后才知道。

例如：

```smt2
(assert (= x y))
(assert (= (f x) z))
(assert (<= y 0))
```

在处理第一条 `(= x y)` 时，可能还不知道 `x` 会进入 EUF，`y` 会进入 LRA。如果过早把 `(= x y)` 归给 LRA 或 EUF，后面就难以修正。

建议改成：

```text
Pass 1: purification + owner collection
  - 生成 pure atoms / bridge atoms
  - 收集每个 SharedTerm 的 owner set

Pass 2: classify equality/disequality atoms
  - 如果两边都是 shared/interface constants，则 owner = Combination
  - 否则归给唯一 theory owner
```

或者做一个 `finalizePurifiedAtoms()`，在 owner set 稳定后再把 equality promote 成 `TheoryId::Combination`。

这是 Stage A 最容易踩的点之一。

---

## 3. `assertTheoryLit()` 里的 reason literal 要明确是“实际赋值文字”

当前伪代码：

```cpp
pendingSharedEqEvents_.push_back({
    payload.a, payload.b, value, lit, level
});
```

这里要明确：`lit` 必须是 SAT 当前赋值为 true 的那个 literal。

也就是说：

```text
value = true   -> reasonLit = Eq(a,b)
value = false  -> reasonLit = ¬Eq(a,b)
```

不要传固定的正 literal 再用 `value` 区分。否则后面 disequality 的解释会错，可能生成：

```text
Eq(a,b) -> conflict
```

而实际 reason 应该是：

```text
¬Eq(a,b) -> disequality
```

建议写成：

```cpp
SatLit assignedLit = value ? atom.positiveLit() : atom.negativeLit();

pendingSharedEqEvents_.push_back({
    payload.a,
    payload.b,
    value,
    assignedLit,
    level
});
```

这个对 explanation、rollback、conflict clause 都很重要。

---

## 4. `solversOwning(a,b)` 需要定义清楚

Phase A-5 里使用：

```cpp
for (auto* solver : solversOwning(ev.a, ev.b))
```

这个函数不能含糊。对于 shared equality，应该发送给所有需要知道这两个 interface constants 的 theory solver。

建议定义为：

```text
solversOwning(a,b)
  = union(owners(a), owners(b))
    filtered by solver.supportsCombination()
```

同时要求：

```text
如果某个 solver 收到 interface equality a=b，
但本地还没有 intern a 或 b，
则必须先 intern 成该 theory 的 interface constant。
```

不要用 intersection。用 intersection 可能漏传；比如某个 term 原本只在 EUF 侧出现，另一个只在 LRA 侧出现，但 equality 把它们连接起来，两边都需要知道这个关系。

---

## 5. pending events 处理时建议即时检查冲突

现在流程是：

```cpp
处理所有 pending events
pendingSharedEqEvents_.clear()
再 checkDisequalityConflict()
```

更稳的写法是每处理一个 equality/disequality 后立刻检查一次：

```cpp
sharedEqMgr_.assertEquality(...);
if (auto c = sharedEqMgr_.checkDisequalityConflict())
    return mkConflict(*c);
```

否则可能在已经出现 `Eq(x,y)` 与 `¬Eq(x,y)` 冲突后，还继续把不一致状态转发给 theory solver。虽然最终也能查出来，但调试会更乱。

---

## 最终判断

可以接受，但建议把文档最后改成：

```text
Stage A accepted implementation target:
- QF_UFLRA fully functional.
- QF_UFRDL only enabled if RdlSolver NO hooks are implemented; otherwise remains Unknown.

Before coding:
1. Purifier uses two-pass classification or finalize phase.
2. assertTheoryLit stores the assigned literal as reasonLit.
3. solversOwning(a,b) is defined as union of owners with lazy interning.
4. SharedEqualityManager checks equality/disequality conflict immediately after each pending event.
```

改完这几点，就可以正式让实现者开工。整体方案已经从“架构草稿”提升到“可执行实现计划”。
