# 1. `BoundUpdateZ` 不能只表示区间

计划里 `BoundUpdateZ` 是：

```cpp
struct BoundUpdateZ {
    std::string var;
    IntervalZ newInterval;
    std::vector<SatLit> reasons;
};
```

但 `x^2 = c` 的 NIA 规则写了：

```text
c = r^2 -> finite set {-r, r}
```

这不是区间 update，而是 finite-set update。当前类型表达不了。

建议改成：

```cpp
enum class BoundUpdateKindZ {
    Interval,
    FiniteSet,
    ExcludeValue
};

struct BoundUpdateZ {
    std::string var;
    BoundUpdateKindZ kind;
    IntervalZ interval;
    std::vector<mpz_class> finiteSet;
    std::vector<mpz_class> excluded;
    std::vector<SatLit> reasons;
};
```

或者 V1 明确：

```text
ICP V1 只返回 interval update；
x²=c 的 finite-set refinement 继续留给旧 SquareBoundReasoner；
ICP 的 SquareContractorZ 对 x²=c 只做 conflict / interval [-r,r]，不返回 finite set。
```

二选一。否则类型和规则不一致。

# 4. NIA strict normalization 要说明前提：整数系数已清分母

计划写：

```text
NIA: p < 0 -> p <= -1
```

这只在 `p` 是整数值多项式时成立。也就是：

```text
Int variables + integer coefficients after clearDenominators
```

如果 `p` 还有 rational coefficients，这个规约不成立。

建议补一句：

```text
Before ICP, NIA constraints must be normalized to integer-coefficient polynomial p rel 0.
Only then Lt/Gt can be rewritten as p<=-1 / p>=1.
If integer normalization fails, ICP returns NoChange.
```

---

# 5. `RelationContractorQ` strict 返回 NoChange 是对的，但测试也要覆盖

计划已经写了 NRA strict 不实现时必须 `NoChange`，不能近似成非 strict。这个很好。建议加回归：

```text
NRA: x ∈ [0,0], constraint x < 0
ICP V1 must return NoChange, not Conflict.
```

因为如果错误当成 `x<=0`，就会漏掉 strict violation；如果错误当成 conflict/pruning 又可能产生边界问题。
