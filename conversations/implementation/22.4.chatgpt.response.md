**结论：不接受当前版本。**

这版不能交给实现模型。问题不是“还差一点优化”，而是文档内部又混入了旧方案，导致多条 Red Lines 被后文实现直接违反。按这个版本实现，仍然可能出现 **假 SAT、错误 explanation、编译接口不一致、Bool 语义不 sound**。

# 1. 最大 blocker：Bool sort 仍然不是二值语义，QF_UF 会假 SAT

当前方案把 Bool-valued term 处理成：

```text
p  ==>  (= p true)
```

但当这个 literal 为 false 时，`check()` 只是把它解释成：

```text
p != true
```

普通 EUF 会把 Bool 当成无限 uninterpreted sort，这不对。Bool 只有 `true/false` 两个值，所以：

```text
p != true  ==>  p = false
```

当前方案没有保证这一点。

反例：

```smt2
(set-logic QF_UF)
(declare-const p Bool)
(declare-const q Bool)
(assert (distinct p true))
(assert (distinct q true))
(assert (distinct p q))
(check-sat)
```

正确结果是 **UNSAT**。
因为 `p=false`，`q=false`，所以 `distinct p q` 不成立。
但当前 EUF 可能把 `p`、`q` 当作两个不同的普通 UF 值，错误返回 SAT。

必须加入 Bool 二值语义。推荐改法：

```text
For every Bool-valued term b:
  lit(b) := (= b true)

If lit(b) assigned true:
  merge(b, true)

If lit(b) assigned false:
  merge(b, false)
```

并且 Bool equality / distinct 不应作为普通 EUF equality / disequality 处理，而应命题化：

```text
(= b1 b2)        => b1 ↔ b2
(distinct b1 b2) => b1 XOR b2
(distinct b1 b2 b3) => false
```

这是 soundness blocker。

# 2. `true != false` 检查位置又错了

Red Line 写：

```text
true != false is enforced after congruence closure.
```

但 `EufSolver::check()` 里实际是：

```text
2. add terms
2b. check true != false
3. assert equalities
4. close congruence
5. check disequalities
```

这会漏掉：

```smt2
(set-logic QF_UF)
(assert (= true false))
(check-sat)
```

因为检查 `true` 和 `false` 是否同类时，还没有执行 equality merge。

必须改成：

```text
1. intern terms
2. add terms
3. assert equalities
4. closeCongruence
5. check true != false
6. check disequalities
```

并且要检查 `closeCongruence` 的返回值。

# 4. Explanation 又写回了错误逻辑

Red Line 写：

```text
Explanation cycle returns failure (ok=false), not empty reasons.
```

但实际伪代码里又写了：

```cpp
if (visitedPairs.count(pk)) return {true, {}};  // cycle in explanation graph
```

这是严重错误。
返回 `{true,{}}` 等价于“这个等式不需要任何原因”，会生成过强 conflict clause。

必须改成：

```cpp
if (activePairs.count(pk)) {
    return {false, {}};
}
```

同时要区分两种 visited：

```text
BFS visited node:
  遇到已访问 term，continue，不失败。

递归 explanation active pair:
  同一个 pair 正在解释中再次进入，返回 failure。
```

不要把普通 BFS revisit 和递归解释环混在一起。

# 6. EGraph 仍然缺 `termClass_`

Red Line 写：

```text
EGraph owns termClass_ mapping EufTermId → EClassId.
```

但 `EGraph.h` 里只看到：

```cpp
std::vector<EClass> classes_;
```

没有：

```cpp
std::vector<EClassId> termClass_;
```

没有这个，`find(EufTermId t)` 无法可靠实现。

必须加：

```cpp
std::vector<EClassId> termClass_;
```

并实现：

```cpp
void EGraph::addTerm(EufTermId t) {
    if (t >= termClass_.size()) {
        termClass_.resize(t + 1, NullEClass);
    }
    if (termClass_[t] != NullEClass) return;

    EClassId c = classes_.size();
    classes_.push_back(EClass{c, c, {t}});
    termClass_[t] = c;
}

EClassId EGraph::find(EufTermId t) const {
    assert(t < termClass_.size());
    assert(termClass_[t] != NullEClass);
    return ufFind(termClass_[t]);
}
```

```cpp
void EufSolver::pop(uint32_t n) {

}
```
