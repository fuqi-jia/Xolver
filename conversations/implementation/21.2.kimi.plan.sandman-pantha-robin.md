# EUF Solver Implementation Plan for NLColver — Revised V1

## Goal
Implement a **sound** QF_UF (equality-only fragment with uninterpreted functions) theory solver. V1 is strictly limited to pure QF_UF — no mixed theories.

## Critical V1 Constraints (Red Lines)

1. **NO mixed logics in V1.** Do NOT register QF_UFLIA/UFLRA/UFNIA/UFNRA in `Solver.cpp`. Without Nelson-Oppen / equality sharing, these would be **unsound**.
2. **Term-level proof graph for explanation.** Never use current class root as BFS endpoint — union-find compression makes this unreliable.
3. **Bool-valued UF predicates must work.** `(p a)` where `p` returns Bool must be converted to `(= (p a) true)` internally.
4. **N-ary equality/distinct must normalize.** `(= a b c)` and `(distinct a b c d)` are legal SMT-LIB.
5. **Function symbol key includes full signature** (name + arg sorts + result sort).
6. **Conflict clause uses `assertedLit` negation** — must be the actual true-assigned SAT literal.
7. **Sort mismatch is an internal error** — EGraph::merge defensively checks sorts.
8. **Trail-based push/pop** — O(1) push, O(backtrack-items) pop.

---

## Architecture Context

NLColver uses:
- **CoreIr** as lightweight dense array IR (`CoreExpr` nodes with `Kind`, `SortId`, `children`, `Payload`)
- **SOMTParser** for parsing; `FrontendAdapter` maps SOMTParser `Node` → `CoreExpr`
- **Atomizer** converts `CoreExpr` assertions to SAT clauses + `TheoryAtomRecord`s
- **TheoryAtomRegistry** deduplicates theory atoms (`LinearAtomPayload`, `PolynomialAtomPayload` variants)
- **TheoryManager** dispatches `assertLit`/`check` to registered `TheorySolver`s by `TheoryId`
- **Solver::Impl** registers solvers based on `(set-logic ...)` string
- V1 theory solvers use **full-rebuild** `check()`: `assertLit` stores active assignments, `check()` rebuilds from scratch

## Phase 2: N-ary Normalize + Bool-Predicate in Atomizer

### 2.1 N-ary Equality / Distinct Normalization

SMT-LIB allows n-ary `=` and `distinct`:
```smt2
(= a b c)         ; means a=b ∧ b=c
(distinct a b c)  ; means a≠b ∧ a≠c ∧ b≠c
```

**Where to normalize:** In `Atomizer::atomizeRec`, before extracting theory atoms.

For `Kind::Eq` with `children.size() > 2`:
```cpp
// Convert (= t1 t2 ... tn) to (and (= t1 t2) (= t2 t3) ... (= t_{n-1} tn))
// Each binary (= ti t_{i+1}) gets its own SAT variable via recursive atomizeRec
```

For `Kind::Distinct` with `children.size() > 2`:
```cpp
// Convert (distinct t1 t2 ... tn) to (and (distinct t1 t2) (distinct t1 t3) ...)
// Generate all pairwise distinct atoms, AND them via Tseitin
```

**Implementation approach:** Add helper methods in Atomizer:
```cpp
SatLit atomizeNaryEq(ExprId eid, const CoreIr& ir);
SatLit atomizeNaryDistinct(ExprId eid, const CoreIr& ir);
```

These create new `CoreExpr`-like structures inline (using synthetic ExprIds) or directly build the SAT CNF. Simpler: directly construct the SAT encoding without creating new CoreExpr nodes:

```cpp
SatLit Atomizer::atomizeNaryEq(ExprId eid, const CoreIr& ir) {
    const auto& e = ir.get(eid);
    SatVar andVar = freshVar();
    for (size_t i = 0; i + 1 < e.children.size(); ++i) {
        // Create a synthetic binary equality and atomize it
        // Need to register it as a theory atom
        // Approach: directly call the binary equality extraction path
    }
    // Tseitin: andVar → each pairwise eq, each pairwise eq → andVar
}
```

**Alternative simpler approach:** Add a pre-rewrite step before atomization that flattens n-ary `=`/`distinct` into `And`/`Or` combinations of binary atoms. But since CoreIr is dense and read-only after import, it's easier to handle directly in `atomizeRec`.

### 2.2 Bool-Valued UF Predicate Support

When a `Kind::UFApply` with Bool sort appears in formula position (i.e., it's the root of an assertion, or a child of `And`/`Or`/`Not`), the Atomizer must treat it as an equality atom:

```text
(p a)  where p: U → Bool
=>  (= (p a) true)
```

**Implementation:** In `atomizeRec`, before the main `switch(e.kind)`, check:
```cpp
if (e.kind == Kind::UFApply && e.sort == boolSortId) {
    // Convert to (= this_expr true_const)
    // Find or create a true constant CoreExpr
    // Atomize the resulting Eq
}
```

But we need `boolSortId`. The `CoreIr` doesn't have a global sort registry. Options:
1. Pass `boolSortId` to Atomizer from Solver (obtained from FrontendAdapter or hardcoded as 0 if BOOL is always SortId 0)
2. Check if the expression has no atom parent (i.e., it stands alone as formula)

Actually, looking at `Atomizer::atomizeRec`, any expression that is not a boolean connective falls through to the `default` case and gets a fresh SAT variable. For a Bool-valued UFApply:
- It would get a SAT variable
- But it wouldn't be registered as a theory atom (the `isTheory` check only matches `Eq/Distinct/Lt/Leq/Gt/Geq`)
- So it would be treated as a boolean atom with no theory semantics

**Fix:** In the `default` case, after the `isTheory` check, add:
```cpp
if (e.kind == Kind::UFApply && isBoolSort(e.sort)) {
    // This is a predicate application p(a)
    // Convert to (= p(a) true) and atomize that
    // We need the true constant's ExprId
}
```

However, the true constant may not exist in CoreIr. Simpler: handle it directly in Atomizer by creating a synthetic binary equality:

```cpp
// p(a) is equivalent to (= p(a) true)
// Create a theory atom: EufAtomPayload{lhs=p(a), rhs=true_expr, rel=Eq}
```

But `true_expr` needs to be a valid `ExprId`. In SMT-LIB, `true` is a constant. If the parser already creates it, we can reference it. If not, we need to handle the RHS specially.

**Simpler approach for V1:** In `Atomizer`, when encountering a Bool-valued `UFApply`:
1. Treat it like an equality atom where LHS is the UFApply and RHS is implicitly `true`
2. Register as `EufAtomPayload{lhsExpr=eid, rhsExpr=NullExpr, rel=Eq}` with a convention that `NullExpr` RHS means `true`
3. In `EufSolver`, `intern()` on `NullExpr` returns a special "true" constant term

Or even simpler: the Atomizer creates the `true` constant in CoreIr on demand.

**Recommended V1 approach:**
- Solver passes `boolSortId` to Atomizer
- In `atomizeRec`, Bool-valued `UFApply` falls through to `default` case
- In `default`, detect `e.kind == Kind::UFApply && e.sort == boolSortId_`
- Create/register a `EufAtomPayload` with `lhsExpr = eid` and a special `rhsExpr = TRUE_EXPR_SENTINEL`
- `EufSolver` knows `TRUE_EXPR_SENTINEL` → intern as a special true constant with its own `FuncSymbolId`

Actually, looking more carefully: the `default` case creates a fresh SAT var. For a Bool-valued UFApply, we want it to be a theory atom (so EUF solver handles it). So we should handle it BEFORE the default case:

```cpp
case Kind::UFApply: {
    if (e.sort == boolSortId_) {
        // Predicate application: treat as (p(args) = true)
        SatVar v = freshVar();
        result = SatLit::positive(v);
        if (registry_ && defaultTheory_ == TheoryId::EUF) {
            // Register as EUF atom: lhs = this UFApply, rhs = true sentinel
            registry_->registerParsedTheoryAtom(
                v, eid, TheoryId::EUF,
                EufAtomPayload{eid, TRUE_SENTINEL_EXPR, Relation::Eq});
        } else {
            atoms_.push_back({v, eid, false, TheoryId::Bool});
        }
    } else {
        // Non-bool UFApply in formula position — this is an error
        // (e.g., user wrote (assert (f a)) where f returns Int)
        registry_->setUnsupportedTheorySeen();
    }
    break;
}
```

Wait, but `UFApply` can also appear as an argument to another function, e.g. `(f (g a))`. In that case it's not in formula position. But `atomizeRec` is only called on formula-level expressions (assertions and their boolean subexpressions). Terms inside equality/arguments are handled by the EUF term manager's `intern()`, not by `atomizeRec`. So a standalone `UFApply` case in `atomizeRec` only triggers when the UFApply IS the formula atom. That's correct.

### 3.3 `EufTermManager`

Interns EUF terms from CoreExpr subgraphs.

```cpp
class EufTermManager {
public:
    struct ENode {
        EufTermId id;
        FuncSymbolId symbol;
        std::vector<EufTermId> args;  // empty for constants
        SortId sort;
        ExprId origin;  // corresponding CoreExpr id
    };

    struct FuncSymbolKey {
        std::string name;
        std::vector<SortId> argSorts;
        SortId resultSort;

        bool operator==(const FuncSymbolKey&) const = default;
    };
    struct FuncSymbolKeyHash {
        std::size_t operator()(const FuncSymbolKey& k) const;
    };

    // Intern a CoreExpr subtree. Returns NullEufTerm if non-UF term encountered.
    EufTermId intern(ExprId exprId, const CoreIr& ir);

    const ENode& node(EufTermId id) const;
    bool isApplication(EufTermId id) const;
    size_t termCount() const;

    void clear();

    // Special true constant (for Bool-valued predicate support)
    EufTermId trueConstant() const { return trueConstant_; }

private:
    std::vector<ENode> nodes_;
    std::unordered_map<FuncSymbolKey, FuncSymbolId, FuncSymbolKeyHash> symbolMap_;

    struct TermKey {
        FuncSymbolId symbol;
        std::vector<EufTermId> args;
        bool operator==(const TermKey&) const = default;
    };
    struct TermKeyHash {
        std::size_t operator()(const TermKey& k) const;
    };
    std::unordered_map<TermKey, EufTermId, TermKeyHash> termMap_;

    EufTermId trueConstant_ = NullEufTerm;

    FuncSymbolId internSymbol(const FuncSymbolKey& key);
    EufTermId createNode(FuncSymbolId sym, std::vector<EufTermId> args, SortId sort, ExprId origin);
    EufTermId internTrueConstant();
};
```

**Interning logic** (`internRec`):
- `Kind::Variable` with any sort → constant (0-ary function, symbol = var name, sort = variable's sort)
- `Kind::UFApply` → intern symbol from payload name + children's sorts + node's sort, intern each child recursively, then intern application
- `Kind::ConstBool` with value `true` → return `trueConstant()`
- Arithmetic `Kind::Add/Sub/Mul/...` → return `NullEufTerm`
- `TrueSentinelExpr` → return `trueConstant()`
- Nested `Kind::Eq/Distinct` inside arguments → return `NullEufTerm` (EUF only handles equality at top level)

**Important:** Even Int/Real-typed variables are interned as constants. The EUF solver doesn't care about arithmetic semantics — it just sees them as uninterpreted constants. The issue is only when arithmetic **operations** appear (Add, Mul, etc.), which EUF cannot handle.

### 3.4 `EGraph` with Term-Level Proof Graph

**Critical design:** The proof graph records edges between **original term endpoints**, not between current class representatives. This is essential for correct explanation.

```cpp
class EGraph {
public:
    void clear();
    void addTerm(EufTermId t);

    // Merge two terms' equivalence classes. Returns true if merge happened.
    // Passes EufTermManager for sort checking and congruence key building.
    bool merge(EufTermId a, EufTermId b, const MergeReason& reason,
               const EufTermManager& tm);

    bool same(EufTermId a, EufTermId b) const;
    EClassId find(EufTermId t) const;

    // Close congruence: repeated scan until fixpoint.
    void closeCongruence(const EufTermManager& tm);

    // Explain why two terms are in the same class.
    // Uses the term-level proof graph (NOT class-level BFS).
    std::vector<SatLit> explainEquality(EufTermId a, EufTermId b,
                                        const EufTermManager& tm) const;

    // Build model: one value per class
    std::unordered_map<EClassId, std::string> buildModel() const;

    size_t classCount() const;

private:
    struct EClass {
        EClassId id;
        EClassId parent;
        std::vector<EufTermId> members;
    };

    struct ProofEdge {
        EufTermId u;        // original endpoint (NOT class rep)
        EufTermId v;        // original endpoint (NOT class rep)
        MergeReason reason;
    };

    std::vector<EClass> classes_;
    std::vector<ProofEdge> proofEdges_;

    // Union-Find
    EClassId ufFind(EClassId x) const;
    void ufUnite(EClassId a, EClassId b);

    // Congruence detection
    struct AppKey {
        FuncSymbolId symbol;
        std::vector<EClassId> argClasses;
        bool operator==(const AppKey&) const = default;
    };
    struct AppKeyHash {
        std::size_t operator()(const AppKey& k) const;
    };

    std::vector<std::pair<EufTermId, EufTermId>> findCongruentPairs(
        const EufTermManager& tm) const;

    // Explanation helpers
    std::vector<SatLit> explainMergeReason(const MergeReason& reason,
                                           const EufTermManager& tm,
                                           std::unordered_set<std::pair<EufTermId, EufTermId>, PairHash>& visitedPairs) const;

    std::vector<SatLit> explainPath(EufTermId from, EufTermId to,
                                    const EufTermManager& tm,
                                    std::unordered_set<std::pair<EufTermId, EufTermId>, PairHash>& visitedPairs) const;
};
```

**Merge algorithm:**
```cpp
bool EGraph::merge(EufTermId a, EufTermId b, const MergeReason& reason,
                   const EufTermManager& tm) {
    // Sort check
    if (tm.node(a).sort != tm.node(b).sort) {
        return false;  // or assert — this should never happen
    }

    if (same(a, b)) return false;

    // Record proof edge BEFORE uniting (captures original endpoints)
    proofEdges_.push_back({a, b, reason});

    EClassId ca = find(a);
    EClassId cb = find(b);
    ufUnite(ca, cb);
    return true;
}
```

**Explanation algorithm:**
```cpp
std::vector<SatLit> EGraph::explainEquality(EufTermId a, EufTermId b,

    if (a == b) return {};

    return explainPath(a, b, tm, visited);
}

std::vector<SatLit> EGraph::explainMergeReason(const MergeReason& reason,
                                               const EufTermManager& tm,
                                               std::unordered_set<std::pair<EufTermId, EufTermId>, PairHash>& visitedPairs) const {
    if (reason.kind == MergeReasonKind::AssertedEquality) {
        return {reason.assertedLit};
    }

    if (reason.kind == MergeReasonKind::Congruence) {
        std::vector<SatLit> result;
        for (const auto& [ai, bi] : reason.argPairs) {
            auto sub = explainPath(ai, bi, tm, visitedPairs);
            appendUnique(result, sub);
        }
        return result;
    }

    return {};
}
```

**Congruence closure (V1 simple repeated-scan):**
```cpp
void EGraph::closeCongruence(const EufTermManager& tm) {
    bool changed = true;
    while (changed) {
        changed = false;
        auto pairs = findCongruentPairs(tm);
        for (auto [a, b] : pairs) {
            if (same(a, b)) continue;

            const auto& na = tm.node(a);
            const auto& nb = tm.node(b);

            MergeReason reason;
            reason.kind = MergeReasonKind::Congruence;
            reason.lhsApp = a;
            reason.rhsApp = b;
            for (size_t i = 0; i < na.args.size(); ++i) {
                reason.argPairs.push_back({na.args[i], nb.args[i]});
            }

            if (merge(a, b, reason, tm)) {
                changed = true;
            }
        }
    }
}
```

**Problem:** `atomizeRec` expects to work with `ExprId`s from CoreIr. For n-ary normalization, we need to create synthetic binary equalities. Since CoreIr is a dense vector, we CAN add new synthetic expressions to it. But CoreIr is owned by the Solver, not the Atomizer.

**Better approach:** Handle n-ary normalization at the **Atomizer level without creating new CoreExpr nodes**. Build the Tseitin CNF directly:

```cpp
SatLit Atomizer::atomizeNaryEq(ExprId eid, const CoreIr& ir) {
    const auto& e = ir.get(eid);
    SatVar andVar = freshVar();
    SatLit andLit = SatLit::positive(andVar);

    std::vector<SatLit> pairwiseLits;
    for (size_t i = 0; i + 1 < e.children.size(); ++i) {
        // Create a synthetic EUF atom for each pairwise equality
        SatVar pv = freshVar();
        SatLit plit = SatLit::positive(pv);
        pairwiseLits.push_back(plit);

        ExprId lhs = e.children[i];
        ExprId rhs = e.children[i + 1];
        if (lhs > rhs) std::swap(lhs, rhs);

        if (registry_) {
            registry_->registerParsedTheoryAtom(
                pv, eid, TheoryId::EUF,
                EufAtomPayload{lhs, rhs, Relation::Eq});
        }
    }

    // Tseitin: andVar → each pairwise
    for (SatLit pl : pairwiseLits) {
        sat_.addClause({SatLit::negative(andVar), pl});
    }
    // Tseitin: any pairwise → andVar
    std::vector<SatLit> clause;
    clause.push_back(SatLit::positive(andVar));
    for (SatLit pl : pairwiseLits) {
        clause.push_back(pl.negated());
    }
    sat_.addClause(clause);

    return andLit;
}

SatLit Atomizer::atomizeNaryDistinct(ExprId eid, const CoreIr& ir) {
    const auto& e = ir.get(eid);
    SatVar andVar = freshVar();
    SatLit andLit = SatLit::positive(andVar);

    std::vector<SatLit> pairwiseLits;
    for (size_t i = 0; i < e.children.size(); ++i) {
        for (size_t j = i + 1; j < e.children.size(); ++j) {
            SatVar pv = freshVar();
            SatLit plit = SatLit::positive(pv);
            pairwiseLits.push_back(plit);

            ExprId lhs = e.children[i];
            ExprId rhs = e.children[j];
            if (lhs > rhs) std::swap(lhs, rhs);

            if (registry_) {
                registry_->registerParsedTheoryAtom(
                    pv, eid, TheoryId::EUF,
                    EufAtomPayload{lhs, rhs, Relation::Neq});
            }
        }
    }

    // Tseitin AND encoding same as above
    // ...

    return andLit;
}
```

**Note:** The synthetic atoms share the same `exprId = eid` (the original n-ary expression). This is acceptable for diagnostics; routing uses `satVar`.

### 5.2 `assertLit`

```cpp
void EufSolver::assertLit(const TheoryAtomRecord& atom, bool value, int level, SatLit assertedLit) {
    ActiveAssignment a;
    if (!extractFromPayload(atom, value, assertedLit, a)) {
        return;  // Should not happen if routing is correct
    }
    trail_.push_back(a);
}
```

### 5.3 `backtrackToLevel`

```cpp
void EufSolver::backtrackToLevel(int level) {
    auto it = std::remove_if(trail_.begin(), trail_.end(),
        [level](const auto& a) { return a.level > level; });
    trail_.erase(it, trail_.end());
}
```

### 5.4 `push`/`pop`

```cpp
void EufSolver::push() {
    scopeLimits_.push_back(trail_.size());
}

void EufSolver::pop(uint32_t n) {
    while (n-- && !scopeLimits_.empty()) {
        trail_.resize(scopeLimits_.back());
        scopeLimits_.pop_back();
    }
}
```

### 5.5 `check()` — Full Rebuild

```cpp
TheoryCheckResult EufSolver::check(TheoryLemmaDatabase&) {
    if (!coreIr_) {
        return TheoryCheckResult::unknown();
    }

    termManager_.clear();
    egraph_.clear();
    equalities_.clear();
    disequalities_.clear();

    // 1. Build effective atoms and intern terms
    for (const auto& a : trail_) {
        Relation effRel = a.value ? a.rawRel
                        : (a.rawRel == Relation::Eq ? Relation::Neq : Relation::Eq);

        EufTermId lhs = termManager_.intern(a.lhsExpr, *coreIr_);
        EufTermId rhs = termManager_.intern(a.rhsExpr, *coreIr_);

        if (lhs == NullEufTerm || rhs == NullEufTerm) {
            return TheoryCheckResult::unknown();
        }

        if (effRel == Relation::Eq) {
            equalities_.push_back({lhs, rhs, effRel, a.assertedLit});
        } else {
            disequalities_.push_back({lhs, rhs, effRel, a.assertedLit});
        }
    }

    // 2. Register all terms in EGraph
    for (EufTermId t = 0; t < termManager_.termCount(); ++t) {
        egraph_.addTerm(t);
    }

    // 3. Assert equalities
    for (const auto& eq : equalities_) {
        MergeReason reason;
        reason.kind = MergeReasonKind::AssertedEquality;
        reason.assertedLit = eq.assertedLit;
        egraph_.merge(eq.lhs, eq.rhs, reason, termManager_);
    }

    // 4. Close congruence
    egraph_.closeCongruence(termManager_);

    // 5. Check disequalities
    for (const auto& neq : disequalities_) {
        if (egraph_.same(neq.lhs, neq.rhs)) {
            auto eqReasons = egraph_.explainEquality(neq.lhs, neq.rhs, termManager_);

            std::vector<SatLit> clause;
            for (auto lit : eqReasons) {
                clause.push_back(lit.negated());
            }
            clause.push_back(neq.assertedLit.negated());

            return TheoryCheckResult::mkConflict(TheoryConflict{clause});
        }
    }

    return TheoryCheckResult::consistent();
}
```

| Explanation graph | Class-root BFS on `explainEdges_` | **Term-level proof graph** — edges store original term endpoints |

| N-ary =/distinct | Assumed binary | **Normalized** in Atomizer to pairwise combinations |
| Func symbol key | Name only | **Full signature** — name + arg sorts + result sort |
| Variable naming | `SatLit reason` | **`SatLit assertedLit`** — always the true-assigned literal |
| Sort checking | Interface didn't pass `EufTermManager` | **`merge()` takes `tm` parameter** for defensive sort check |
| Push/pop | Copy whole `activeAssignments_` | **Trail + scope limits** — O(1) push |
