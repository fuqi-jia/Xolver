建议 **[4] Revise**，只剩一处核心问题没真正改完：**DomainStore 的 reason 机制仍然没有落到接口上**。

这版已经修掉了上一轮主要问题：`evalInteger` 改成 `optional`，`StubPolyKernel` 返回 `nullopt`，`degree()` 已经加入 Phase A，`FatalUnknown` 替代普通 `Unknown`，bounded solver 区分 `Sat/Unsat/Unknown`，RRT incomplete 不再证明 UNSAT，这些都正确。

但 DomainStore 这里仍然矛盾：

计划文字说：

```text
If any domain becomes empty, DomainStore builds TheoryConflict from all reasons that contributed to the emptiness.
Never emit conflict from an incomplete reason set.
```

可是接口仍然是：

```text
restrictToFiniteSet(var, values)
addLowerBound(var, lb)
addUpperBound(var, ub)
excludeValue(var, v)
```

这些 API 没有 `SatLit reason` 参数，所以 DomainStore 无法知道每个 domain restriction 来自哪个 active constraint。后面 `domains_.buildEmptyDomainConflict()` 也没有传入 active reasons。这样实现时要么拿不到 reason，要么很容易生成不完整 conflict。

必须改成二选一。

---

## 推荐改法 A：DomainStore 记录精确 reason

把接口改成：

```cpp
restrictToFiniteSet(var, values, SatLit reason);
addLowerBound(var, lb, SatLit reason);
addUpperBound(var, ub, SatLit reason);
excludeValue(var, v, SatLit reason);
```

每个 `IntDomain` 记录：

```cpp
std::vector<SatLit> lowerReasons;
std::vector<SatLit> upperReasons;
std::vector<SatLit> finiteSetReasons;
std::vector<SatLit> exclusionReasons;
```

空域时返回相关 reasons 的否定 clause。

---

## 可接受改法 B：先用 all active reasons，粗但 sound

如果现在不想做精确 reason tracking，那就明确写成：

```cpp
domains_.buildEmptyDomainConflict(allActiveReasons(*normalized));
```

并在计划里写清楚：

```text
Phase NIA-Core does not yet track precise domain-update reasons.
For empty-domain conflicts, it uses all active constraint reasons.
This is coarse but sound.
```

这样不能声称 “from all reasons that contributed to the emptiness”，只能说 “from all active reasons”。

## 另一个小文字问题

```text
Only Conflict, Lemma, or FatalUnknown are terminal.
```

这不是大问题，但建议一起改。
