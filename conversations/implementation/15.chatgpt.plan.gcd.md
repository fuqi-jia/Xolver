# Phase A1：Multi-variable GCD conflict

处理 normalized equality：

```text
p = 0
```

把多项式拆成：

```text
p = q + c
```

其中 `c` 是常数项，`q` 是非零项。

令：

```text
g = gcd(all coefficients of q)
```

如果：

```text
g ∤ -c
```

则无整数解。

例子：

```text
6x + 9y = 5
g = 3
3 ∤ 5
=> UNSAT
```

非线性也一样：

```text
2x^2 + 4yz - 3 = 0
g = 2
2 ∤ 3
=> UNSAT
```

正确性要求：

```text
1. 必须能精确提取所有 monomial coefficients；
2. 如果 term decomposition 不可用，返回 NoChange；
3. 不能靠字符串/采样猜；
4. conflict reason 是当前 equality literal 的 negation。
```

如果当前 `PolynomialKernel` 没有 term API，就加：

```cpp
struct PolyTerm {
    mpz_class coeff;
    std::vector<std::pair<std::string, int>> powers;
};

std::optional<std::vector<PolyTerm>>
PolynomialKernel::terms(PolyId p);
```

# Phase A2：Factor split lemmas

只做 syntactic product，不做代数因式分解。

识别：

```text
p*q = 0
```

生成 lemma：

```text
¬(p*q = 0) ∨ (p = 0) ∨ (q = 0)
```

识别：

```text
p*q != 0
```

生成两个 lemma：

```text
¬(p*q != 0) ∨ (p != 0)
¬(p*q != 0) ∨ (q != 0)
```

关键要求：

```text
1. 只在明确知道该 polynomial 是 syntactic product 时做；
2. p=0、q=0、p!=0、q!=0 都必须通过 TheoryAtomRegistry 注册真实 dynamic atom；
3. 不允许 fake ExprId；
4. dynamic atom 回到 assertLit 时继续走 poly-rhs normalization。
```

这个对：

```text
xy = 0 ∧ x != 0 ∧ y != 0
```

很有用。

# Phase A3：Modular equality reasoning

只用于 equalities：

```text
p1 = 0, ..., pk = 0
```

选小模数：

```text
2, 3, 4, 5, 7, 8, 9, 11
```

对所有变量枚举 residue：

```text
x ∈ Z/mZ
```

如果没有 residue assignment 同时满足所有 equality：

```text
UNSAT
```

conflict：

```text
¬reason(p1=0) ∨ ... ∨ ¬reason(pk=0)
```

注意：

```text
1. 只用于 equality；
2. 不用来证明 disequality false；
3. 变量数太多时跳过；
4. 模数太大或预算耗尽时 NoChange；
5. evaluateMod 必须是 exact modular arithmetic。
```

例子：

```text
x^2 + y^2 = 3
```

mod 4：

```text
square ∈ {0,1}
x^2+y^2 ∈ {0,1,2}
不能等于 3
=> UNSAT
```
