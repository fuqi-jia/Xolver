你这里说的“不相等的不等式”，本质是 **linear disequality**：

```text
p(x) != 0
```

或者 SMT-LIB 里的：

```smt2
(not (= p 0))
(distinct p 0)
```

结论先说：**Kimi 当前这个“assertLit 时忽略 disequality，check 后 violated 就临时试 `s < 0` / `s > 0`”的方向可以做成 sound 的工程捷径，但不是最专业的 CDCL(T) 处理方式。**更标准、更可扩展、更 proof-friendly 的做法是：**把 disequality 当作 obligation；如果当前 LRA model 违反它，就返回 guarded split lemma，而不是在 theory solver 内部偷偷完成分支。**

你前面 plan 里其实已经写了正确方向：`s != 0` 不能表达成单个 bound；若当前 model 里 `s = 0`，应返回 split lemma `s < 0 ∨ s > 0`，并且 disequality 在 wrapper 层处理，不放进 GeneralSimplex 内核。

# 1. 为什么 disequality 不能直接转成 simplex bound

对线性形式：

```text
s = p(x)
```

不等式：

```text
s <= 0
s >= 0
s < 0
s > 0
```

都是 convex half-space，可以直接转成 simplex bound：

```text
s <= 0
s >= 0
s <= -δ
s >= δ
```

但 disequality：

```text
s != 0
```

等价于：

```text
s < 0 ∨ s > 0
```

这是两个 half-space 的析取，不是一个 convex constraint。Simplex 处理的是线性等式 + 上下界的 conjunction，所以不能直接塞进去。

因此：

```text
s != 0
```

不能在 `assertLit` 时调用一次 `assertLower` 或 `assertUpper` 完成。

---

# 2. 最专业的 CDCL(T) 处理：obligation + guarded split lemma

假设 SAT 当前 assignment 里有一个 theory literal：

```text
d := (s != 0)
```

并且 `d = true`。

在 `assertLit(d)` 时：

```cpp
disequalities_.push_back({auxVar: s, lit: d});
```

不向 GeneralSimplex 加 bound。

然后 `gs_.check()` 得到一个 LRA model `β`。

分两种情况：

## 情况 A：`β(s) != 0`

那这个 disequality 已经被当前 model 满足，可以接受。

```text
β(s) < 0 或 β(s) > 0
```

此时不需要产生 lemma。

## 情况 B：`β(s) = 0`

当前 LRA model 不满足 `s != 0`。这时不要直接返回 `Unknown`，也不要直接返回 `Unsat`，而应该返回一个 theory lemma：

```text
¬(s != 0) ∨ (s < 0) ∨ (s > 0)
```

用 δ 表示：

```text
¬d ∨ (s <= -δ) ∨ (s >= δ)
```

也就是：

```cpp
Clause lemma = {
    negate(d_lit),
    getOrCreateLit(s <= -delta),
    getOrCreateLit(s >=  delta)
};
```

这条 lemma 是 theory-valid 的，因为：

```text
s != 0  ⇒  s < 0 ∨ s > 0
```

这才是 CDCL(T) 里的标准处理方式。

# 3. Kimi 现在的“临时试两个分支”有什么问题

它说：

```text
check() 返回 Sat 后，如果 disequality violated，
尝试添加 s < 0 bound 并重新 check；
如果不 Sat，尝试 s > 0 bound；
如果都不 Sat，返回 Unsat。
```

这个思路**作为内部优化可以做**，但如果直接这么实现，有三个风险。

---

## 问题一：如果找到某一支 Sat，必须返回该支的 model

例如：

```text
x != 0
```

GeneralSimplex 忽略 disequality 后可能给：

```text
x = 0
```

然后你临时试：

```text
x < 0
```

得到：

```text
x = -1
```

这时返回 Sat 是 sound 的，前提是你最终返回的是 `x=-1` 这个 model，而不是原来的 `x=0` model。

所以必须：

```cpp
if (leftSat) {
    currentModel_ = leftModel;
    return Sat;
}
if (rightSat) {
    currentModel_ = rightModel;
    return Sat;
}
```

并且最后统一调用 model validator。

---

## 问题二：如果两个分支都 Unsat，不能随便返回 conflict

如果：

```text
Γ ∧ s < 0  unsat
Γ ∧ s > 0  unsat
```

那么确实可以推出：

```text
Γ ∧ s != 0 unsat
```

但 SMT solver 需要一个 conflict clause，不能只说 `Unsat`。

正确 conflict 需要包含：

```text
当前 active constraints 的 reasons
+ disequality literal d
```

最简单安全做法不是自己合成复杂 conflict，而是先返回 split lemma：

```text
¬d ∨ s<0 ∨ s>0
```

让 SAT solver 分别探索两个分支。每个分支冲突时，GeneralSimplex 自己会给出标准 bound conflict。

这比你在 theory solver 里面合并两个分支的 proof 简单得多。

---

## 问题三：内部 hidden branch 会绕开 SAT solver 的学习

如果你在 theory solver 内部偷偷做：

```text
try s < 0
try s > 0
```

SAT solver 不知道这个 split，也学不到 clause。后续遇到类似结构还会重复做同样的内部搜索。

专业 solver 更倾向于：

```text
theory solver 发现当前 model 不满足 disequality
  ↓
返回 split lemma
  ↓
SAT solver 正式分支、学习、回溯
```

这样和 CDCL 机制一致。

# 4. 推荐实现方案：V1 用 split lemma，不要 local branch

我建议你现在直接让 Kimi 改成这个版本：

```text
Neq true:
  记录 disequality obligation，不 assert bound。

Eq false:
  等价于 Neq true，也记录 disequality obligation。

check():
  1. 先 gs_.check()
  2. 如果 gs_ unsat，返回 simplex conflict
  3. 如果 gs_ sat，检查所有 disequality obligations
  4. 若某个 s != 0 在 model 中 s = 0：
       返回 guarded split lemma:
         ¬d ∨ (s <= -δ) ∨ (s >= δ)
  5. 若所有 disequality 都满足：
       返回 Sat
```

伪代码：

```cpp
TheoryResult SimplexSolver::check() {
    auto r = gs_.check();

    if (r == GeneralSimplex::Unsat) {
        lastConflict_ = translateConflict();
        return TheoryResult::Conflict;
    }

    auto model = gs_.model();

    for (const DiseqInfo& d : disequalities_) {
        DeltaRational v = model.value(d.auxVar);

        if (v.isZero()) {
            SatLit left  = getOrCreateBoundLit(d.auxVar, Relation::LT, 0);
            SatLit right = getOrCreateBoundLit(d.auxVar, Relation::GT, 0);

            TheoryLemma lemma;
            lemma.lits.push_back(negate(d.lit));
            lemma.lits.push_back(left);   // s < 0, internally upper(s, -δ)
            lemma.lits.push_back(right);  // s > 0, internally lower(s, δ)

            return TheoryResult::Lemma(lemma);
        }
    }

    currentModel_ = model;
    return TheoryResult::Sat;
}
```

这里 `getOrCreateBoundLit` 必须能动态注册新的 theory atom：

```text
s <= -δ
s >= δ
```

如果你暂时不想把 `δ` 暴露成 atom，可以用原始表达式建：

```text
p(x) < 0
p(x) > 0
```

然后通过 Atomizer 生成 SAT literal。

# 5. false literal 的关系映射要保持统一

你现在写的映射思路是对的：

```text
Eq true:
  s = 0  → lower(s,0) + upper(s,0)

Eq false:
  s != 0 → disequality obligation

Neq true:
  s != 0 → disequality obligation

Neq false:
  s = 0  → lower(s,0) + upper(s,0)
```

对普通不等式：

```text
s <= 0 false  → s > 0  → lower(s, δ)
s <  0 false  → s >= 0 → lower(s, 0)
s >= 0 false  → s < 0  → upper(s, -δ)
s >  0 false  → s <= 0 → upper(s, 0)
```

这一套是正确的。

建议代码里不要写很多 if-else，而是先做 relation negate：

```cpp
Relation negateRelation(Relation r) {
    switch (r) {
        case Relation::LEQ: return Relation::GT;
        case Relation::LT:  return Relation::GEQ;
        case Relation::GEQ: return Relation::LT;
        case Relation::GT:  return Relation::LEQ;
        case Relation::EQ:  return Relation::NEQ;
        case Relation::NEQ: return Relation::EQ;
    }
}
```

然后：

```cpp
Relation effectiveRel = value ? rel : negateRelation(rel);
assertEffectiveRelation(aux, effectiveRel, reasonLit);
```

其中：

```cpp
void assertEffectiveRelation(int s, Relation r, SatLit lit) {
    switch (r) {
        case Relation::LEQ:
            gs_.assertUpper(s, BoundInfo(delta(0,0), lit));
            break;

        case Relation::LT:
            gs_.assertUpper(s, BoundInfo(delta(0,-1), lit));
            break;

        case Relation::GEQ:
            gs_.assertLower(s, BoundInfo(delta(0,0), lit));
            break;

        case Relation::GT:
            gs_.assertLower(s, BoundInfo(delta(0,1), lit));
            break;

        case Relation::EQ:
            gs_.assertLower(s, BoundInfo(delta(0,0), lit));
            gs_.assertUpper(s, BoundInfo(delta(0,0), lit));
            break;

        case Relation::NEQ:
            disequalities_.push_back({s, lit});
            break;
    }
}
```

这个写法最干净。

# 6. “忽略 disequality，violated 返回 Unknown”不推荐

Kimi 提到：

```text
disequality 在 assertLit 时直接忽略；
check() 后如果发现 violated，返回 Unknown。
```

这当然 sound，因为不会错误返回 Sat。但它太弱了。

例如：

```smt2
(assert (not (= x 0)))
(assert (>= x 0))
(check-sat)
```

这个显然 sat，`x=1` 即可。

如果 simplex 先给 `x=0`，你就返回 Unknown，会让 LRA 很多简单问题都不完整。

所以不建议。至少也要返回 split lemma。

# 7. “内部试两个分支”什么时候可以做

可以作为优化，但不是 V1 最推荐。

如果要做，应该这样：

```text
1. 先发现 violated disequality。
2. 优先返回 split lemma 给 SAT。
3. 如果 options.local_diseq_branch = true，
   可以在返回 lemma 前试探一个分支，用于快速产生 model。
```

但必须遵守：

```text
如果某个分支 sat：
  返回该分支 model，并 validator 检查。

如果两个分支 unsat：
  不建议直接返回 conflict；
  除非你能合并两个分支的 conflict proof。
  否则仍返回 split lemma。
```

更安全的内部 branch 伪代码：

```cpp
TheoryResult handleDisequality(DiseqInfo d) {
    if (!model.value(d.auxVar).isZero()) {
        return SatContinue;
    }

    if (options_.tryLocalDiseqBranch) {
        auto left = tryTemporaryBound(d.auxVar, LT_ZERO);
        if (left.sat() && validate(left.model())) {
            currentModel_ = left.model();
            return Sat;
        }

        auto right = tryTemporaryBound(d.auxVar, GT_ZERO);
        if (right.sat() && validate(right.model())) {
            currentModel_ = right.model();
            return Sat;
        }

        // 两边都 unsat，也先不要自己合并 conflict
    }

    return Lemma(buildDiseqSplitLemma(d));
}
```

这样是安全的。

# 8. 多个 disequality 怎么处理

如果有多个：

```text
s1 != 0
s2 != 0
...
sk != 0
```

当前 model 可能同时违反多个。

专业做法：

```text
一次只返回一个 split lemma。
```

选择策略：

```text
1. 选择最简单的 linear form；
2. 选择 activity 最高的 disequality；
3. 选择出现变量最少的 disequality；
4. 选择当前值最“接近 0”的 disequality；这里所有 violated 都是 0，所以看复杂度即可。
```

不要一次性返回很多 split lemma，容易污染 SAT。

# 9. 对 LIA 的特殊注意

如果是 LIA，`s != 0` 且 `s` 是整数线性表达式，那么还有一个更强的离散 split：

```text
s != 0  ⇔  s <= -1 ∨ s >= 1
```

不需要 δ。

所以：

## LRA 中：

```text
s != 0  ⇒  s <= -δ ∨ s >= δ
```

## LIA 中：

```text
s != 0  ⇒  s <= -1 ∨ s >= 1
```

这点很重要。因为如果你在 LIA 里仍用 δ，虽然逻辑上也可以表达 strict，但没有利用整数性。

所以建议：

```cpp
if (logic == QF_LIA && linearFormIsIntegerValued(s)) {
    split = (s <= -1) || (s >= 1);
} else {
    split = (s < 0) || (s > 0);
}
```

这比 δ split 更高效。

例如：

```text
2x + 4y != 0
```

左边永远是偶数，甚至可以更强：

```text
2x + 4y <= -2 ∨ 2x + 4y >= 2
```

一般化：

```text
g = gcd(coefficients)
s != 0  ⇒  s <= -g ∨ s >= g
```

这就是 LIA disequality 的专业优化。

# 10. 更专业高效的 LIA disequality split

对整数线性形式：

```text
s = a1*x1 + ... + an*xn + c
```

如果所有 `x_i ∈ Z`，那么 `a1*x1 + ... + an*xn` 是 `g = gcd(a_i)` 的倍数。

如果 `c` 也纳入表达式，`s` 的可取值落在某个 residue class。对于最简单的 `s != 0`，可以先看 `s` 是否能等于 0：

## 10.1 如果 gcd already excludes zero

例如：

```text
2x + 4y + 1 != 0
```

因为：

```text
2x + 4y + 1
```

永远是奇数，不可能等于 0。

那么这个 disequality 永远成立，不需要 split。

判断：

```text
g = gcd(2,4)=2
constant = 1
要使 2x+4y+1 = 0，需要 2x+4y = -1
但 2 ∤ -1
所以 equality impossible
```

所以：

```text
2x + 4y + 1 != 0
```

是 tautological under integrality。

## 10.2 如果 zero possible

例如：

```text
2x + 4y != 0
```

最小非零绝对值不是 1，而是 2。

所以 split 用：

```text
2x + 4y <= -2 ∨ 2x + 4y >= 2
```

一般：

```text
p(x) != 0
g = gcd(coefficients)
if p(x) can equal 0:
    p(x) <= -g ∨ p(x) >= g
else:
    already satisfied
```

注意如果表达式有 constant：

```text
p(x) = a·x + c
```

可取值是：

```text
c mod g
```

如果 `0` 不在这个 residue class，即：

```text
g ∤ -c
```

则 `p(x) = 0` 不可能，disequality 恒真。

如果：

```text
g | -c
```

则非零值和 0 之间的 gap 是 `g`。

所以 LIA disequality split 是：

```text
p(x) <= -g ∨ p(x) >= g
```

这比 `p < 0 ∨ p > 0` 更适合整数算术。

# 11. 建议你给 Kimi 的修改意见

可以直接贴这个：

```text
Disequality handling should not be implemented as “ignore and return Unknown”.
Use the standard CDCL(T) obligation + split-lemma scheme.

Implementation requirements:

1. assertLit:
   Convert false literals by negateRelation().
   For effective relation NEQ, do not assert a simplex bound.
   Record DiseqInfo {auxVar/form, original literal, linear form}.

2. check():
   Run GeneralSimplex first.
   If UNSAT, return the normal simplex conflict.
   If SAT, inspect disequality obligations.
   If all are satisfied by the current model, return SAT.
   If one obligation p != 0 is violated, return a guarded split lemma.

3. LRA split:
      ¬lit(p != 0) ∨ lit(p < 0) ∨ lit(p > 0)
   internally:
      ¬lit ∨ lit(p <= -δ) ∨ lit(p >= δ)

4. LIA optimized split:
   If p(x)=a·x+c is integer-valued:
      g = gcd(|a_i|)
      if g does not divide -c:
          the equality p=0 is impossible, so the disequality is already satisfied.
      else:
          return
             ¬lit(p != 0) ∨ lit(p <= -g) ∨ lit(p >= g)

5. Do not return direct UNSAT from local disequality branches unless conflict clauses/proofs from both branches are correctly merged.
   For V1, prefer returning the split lemma and let SAT/CDCL branch.

6. Optional optimization:
   You may try local disequality branches to quickly find a satisfying model, but if no branch is found, fall back to the split lemma.
```

# 12. 最终判断

Kimi 当前方案里这句：

```text
Neq true: 记录到 disequalities_，不 assert bound
false literal: 取反关系后 assert
check() 运行 GeneralSimplex check，然后处理 disequalities
```

这个大方向是对的。

但这句：

```text
check() 后如果 disequality violated，尝试添加 s < 0 bound 重新 check；
如果不 Sat，尝试 s > 0 bound；
如果都不 Sat，返回 Unsat
```

不够专业，容易在 conflict/proof/CDCL 集成上出问题。

建议改成：

```text
violated disequality -> 返回 guarded split lemma
```

LRA 用：

```text
¬d ∨ s < 0 ∨ s > 0
```

LIA 用更强的：

```text
¬d ∨ s <= -g ∨ s >= g
```

其中：

```text
g = gcd(coefficients of s)
```

这就是更专业、高效、适合 SMT/CDCL(T) 的处理方案。
