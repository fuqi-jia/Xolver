# CDCAC P2b–P4 完善计划

## 总体目标

在 P2a 已经完成的基础上，下一步目标不是继续堆特判，而是把 CDCAC 的代数层补成一个可靠模块：

```text
P2b：精确实代数数比较 + root merge / covering 可靠化
P2c：AlgebraicRoot provenance metadata
P2d：mixed sample 的 signAt 支持
P3 ：algebraic tower zero detection / sign evaluation
P4 ：algebraic-prefix root isolation + nullification contract
```

其中最重要的红线是：

```text
same isolating interval 绝不能作为代数根相等判定；
Unknown comparison 绝不能进入 sort / merge / coversAllLine；
exact zero 不能靠 interval refinement 猜，必须靠 gcd / reduction / libpoly exact sign / tower reduction 证明。
```

# P2b：精确 RealAlg 比较与 root merge 可靠化

## 目标

把当前 `compareRealAlg / mergeRoots / coversAllLine` 改成 sound 版本。

## 1. 删除不安全规则

必须删除：

```cpp
if (a.root.lower == b.root.lower && a.root.upper == b.root.upper) {
    return 0;
}
```

它不 sound。两个不同代数根可能暂时有同一个隔离区间。

只允许保留：

```cpp
if (a.root.lower == a.root.upper &&
    b.root.lower == b.root.upper &&
    a.root.lower == b.root.lower) {
    return 0;
}
```

但如果 rational root canonicalization 已经正确做了，这种情况理论上不会再出现在 algebraic-algebraic 比较里。

---

## 2. 定义 CompareResult

```cpp
enum class CompareResult {
    Less,
    Equal,
    Greater,
    Unknown
};
```

不要再用 `-1 / 0 / 1 / 2` 这种魔法数字。

---

## 3. compareRealAlg 完整逻辑

```cpp
CompareResult compareRealAlg(const RealAlg& a, const RealAlg& b) {
    if (a.isRational() && b.isRational())
        return compareRational(a.rational, b.rational);

    if (a.isRational() && b.isAlgebraic())
        return compareRationalAlgebraic(a.rational, b.root);

    if (a.isAlgebraic() && b.isRational())
        return invert(compareRationalAlgebraic(b.rational, a.root));

    return compareAlgebraicAlgebraic(a.root, b.root);
}
```

---

## 4. rational-algebraic 比较

完整流程：

```text
q 与 alpha 比较：

1. 如果 q < alpha.lower，返回 Less；
2. 如果 q > alpha.upper，返回 Greater；
3. 如果 q 在 interval 内：
   3.1 eval(definingPoly, q)
   3.2 若为 0，返回 Equal；
   3.3 若非 0，refine alpha interval；
   3.4 重复直到 q 被排除或资源耗尽。
```

伪代码：

```cpp
CompareResult compareRationalAlgebraic(mpq_class q, AlgebraicRoot alpha) {
    for (;;) {
        if (q < alpha.lower) return CompareResult::Less;
        if (q > alpha.upper) return CompareResult::Greater;

        Sign s = evalUniAtRational(alpha.definingPoly, q);
        if (s == Sign::Zero) return CompareResult::Equal;

        if (!refineRootInterval(alpha))
            return CompareResult::Unknown;
    }
}
```

---

## 5. algebraic-algebraic 比较

完整流程：

```text
alpha, beta 比较：

1. same canonical defining polynomial：
   - rootIndex 相同 => Equal
   - rootIndex 不同 => 按 rootIndex 比较

2. interval 已分离：
   - a.upper < b.lower => Less
   - b.upper < a.lower => Greater

3. 计算 gcd(f, g)：
   - 若 gcd 非常数：
       locate alpha in gcd
       locate beta  in gcd
       若二者都属于 gcd 且 rootIndexInGcd 相同 => Equal
       若属于 gcd 但 index 不同 => 按 index 比较

4. 若不能证明 equal，则 refine 两个区间直到分离；
5. 若资源耗尽，返回 Unknown。
```

伪代码：

```cpp
CompareResult compareAlgebraicAlgebraic(AlgebraicRoot a,
                                         AlgebraicRoot b) {
    if (a.definingPoly == b.definingPoly) {
        if (a.rootIndex == b.rootIndex)
            return CompareResult::Equal;
        return a.rootIndex < b.rootIndex
            ? CompareResult::Less
            : CompareResult::Greater;
    }

    for (;;) {
        if (a.upper < b.lower) return CompareResult::Less;
        if (b.upper < a.lower) return CompareResult::Greater;

        UniPolyId d = gcdUni(a.definingPoly, b.definingPoly);
        if (!isConstantUni(d)) {
            auto la = locateRootInPolynomial(a, d);
            auto lb = locateRootInPolynomial(b, d);

            if (la.belongs && lb.belongs) {
                if (la.rootIndexInTarget == lb.rootIndexInTarget)
                    return CompareResult::Equal;
                return la.rootIndexInTarget < lb.rootIndexInTarget
                    ? CompareResult::Less
                    : CompareResult::Greater;
            }

            if (la.unknown || lb.unknown)
                return CompareResult::Unknown;
        }

        bool okA = refineRootInterval(a);
        bool okB = refineRootInterval(b);
        if (!okA || !okB)
            return CompareResult::Unknown;
    }
}
```

## 6. locateRootInPolynomial

这个函数必须实现。`rootBelongsTo(alpha, h)` 只返回 bool 不够。

```cpp
struct RootLocateResult {
    bool belongs = false;
    bool unknown = false;
    int rootIndexInTarget = -1;
};
```

逻辑：

```text
给定 alpha 是 f 的某个根，判断 alpha 是否也是 h 的根：

1. 若 h 常数，NotBelongs；
2. refine alpha.interval，使 h 的端点不为 0；
3. 用 Sturm / root counting 统计 h 在 alpha.interval 内有几个根；
4. 0 个 => NotBelongs；
5. 1 个 => Belongs，并返回它是 h 的第几个实根；
6. 多个 => refine alpha.interval；
7. 超预算 => Unknown。
```

## 7. mergeRoots 不能用 std::sort + Unknown comparator

不能这样：

```cpp
std::sort(roots.begin(), roots.end(), [&](auto& a, auto& b) {
    return compareRealAlg(a, b) == CompareResult::Less;
});
```

因为 `Unknown` 会破坏严格弱序。

改成 checked insertion sort：

```cpp
std::optional<std::vector<RealAlg>>
sortAndMergeRoots(std::vector<RealAlg> roots) {
    std::vector<RealAlg> out;

    for (auto r : roots) {
        bool inserted = false;

        for (auto it = out.begin(); it != out.end(); ++it) {
            CompareResult c = compareRealAlg(r, *it);

            if (c == CompareResult::Unknown)
                return std::nullopt;

            if (c == CompareResult::Equal) {
                inserted = true;
                break;
            }

            if (c == CompareResult::Less) {
                out.insert(it, r);
                inserted = true;
                break;
            }
        }

        if (!inserted)
            out.push_back(r);
    }

    return out;
}
```

如果返回 `nullopt`：

```text
当前 CDCAC branch 返回 Unknown，不能继续构造 cells。
```

---

## 8. coversAllLine 改成三值

当前：

```cpp
bool coversAllLine(const Covering& cover);
```

应该改成：

```cpp
enum class CoverageResult {
    Covers,
    DoesNotCover,
    Unknown
};

CoverageResult coversAllLine(const Covering& cover);
```

`boundEqual` 必须用 `compareRealAlg`：

```cpp
CoverageResult boundEqual(const Bound& a, const Bound& b) {
    if (a.isInfinity() || b.isInfinity()) {
        return sameInfinity(a, b) ? Covers : DoesNotCover;
    }

    CompareResult c = compareRealAlg(a.value, b.value);
    if (c == CompareResult::Equal) return CoverageResult::Covers;
    if (c == CompareResult::Unknown) return CoverageResult::Unknown;
    return CoverageResult::DoesNotCover;
}
```

并且相邻 cell 的边界必须满足：

```text
same boundary point
并且至少一侧闭合
```

否则：

```text
(-inf,0) open + (0,+inf) open
```

会被错误当成覆盖全线。

# P2c：AlgebraicRoot provenance metadata

## 目标

让每个 algebraic root 不只是“某个一元多项式的根”，还知道它来自哪个原始多变量多项式、哪个变量、哪一层。

## 修改结构

```cpp
struct AlgebraicRoot {
    UniPolyId definingPoly;       // 当前 prefix 下 specialize 后的一元多项式
    PolyId liftedDefiningPoly;    // 原始多变量多项式
    VarId mainVar;                // 当前根对应的变量
    int level;                    // varOrder 中的位置

    int rootIndex;
    mpq_class lower;
    mpq_class upper;
};
```

如果 `VarId` 现在还是 string，可以先用 string，但建议最终改成 interned id。

---

## 创建 root 时填 metadata

在 `solveLevel(k, prefix, input)` 中：

```cpp
for (auto& root : roots) {
    if (root.isAlgebraic()) {
        root.root.liftedDefiningPoly = constraint.poly;
        root.root.mainVar = varOrder[k];
        root.root.level = k;
    }
}
```

注意：如果一个 root 来自多个 polynomial，merge 时要保存 provenance 列表，而不是只保留一个。

推荐：

```cpp
struct RootOrigin {
    PolyId liftedDefiningPoly;
    VarId mainVar;
    int level;
};

struct AlgebraicRoot {
    UniPolyId definingPoly;
    int rootIndex;
    mpq_class lower;
    mpq_class upper;
    std::vector<RootOrigin> origins;
};
```

这样后续 conflict explanation / guard collection 更稳。

# P2d：signAt 支持 mixed sample

## 目标

分阶段支持混合 sample，不再假装一步到位。

## signAt 分层

```cpp
Sign signAt(PolyId p, const SamplePoint& s) {
    if (s.allRational())
        return signAtRational(p, s);

    if (s.numAlgebraicValues() == 1)
        return signAtOneAlgebraic(p, s);

    return signAtAlgebraicTower(p, s);
}
```

---

## 1. exactly-one-algebraic-variable

已经接近完成，但要保证 zero detection 用 gcd。

```cpp
Sign signAtOneAlgebraic(PolyId p, const SamplePoint& s) {
    VarId a = s.onlyAlgebraicVar();

    UniPolyId g = specializeAllOtherVarsRational(p, s, a);
    if (g == NullUniPolyId)
        return Sign::Unknown;

    return signUnivariateAtAlgebraic(g, s.valueOf(a).root);
}
```

---

## 2. signUnivariateAtAlgebraic

完整流程：

```text
1. g == 0 => Zero
2. d = gcd(f, g)
3. 若 d 非常数：
   locate alpha in d
   若 belongs => Zero
4. 若 alpha 不是 g 的根：
   refine alpha interval，直到 g 在 interval 上没有根；
   取 interval 内 rational sample 判断正负。
```

---

## 3. multiple algebraic variables

不要直接靠 interval refinement。

第一版：

```cpp
Sign signAtAlgebraicTower(PolyId p, const SamplePoint& s) {
    auto reduced = reduceModuloTower(p, s);

    if (reduced.status == ReductionStatus::Zero)
        return Sign::Zero;

    if (reduced.status == ReductionStatus::Unknown)
        return Sign::Unknown;

    return signByCertifiedIntervalEvaluation(reduced.poly, s);
}
```

如果暂时没有 `reduceModuloTower`，就返回 Unknown。不要让 libpoly evaluate 的结果未经验证直接决定 Unsat。

# P3：Algebraic Tower reduction

## 目标

支持：

```text
x^2 - 2 = 0
y - x = 0
y - x != 0
```

这种 exact zero 需要 tower reduction 才能证明。

## reduceModuloTower

```cpp
ReductionResult reduceModuloTower(PolyId p, const SamplePoint& s) {
    PolyId q = p;

    for (int i = s.level() - 1; i >= 0; --i) {
        VarId x = s.varOrder[i];
        RealAlg a = s.values[i];

        if (a.isRational()) {
            q = substituteRational(q, x, a.rational);
            continue;
        }

        const auto& root = a.root;

        if (root.mainVar != x || root.level != i)
            return ReductionResult::unknown();

        PolyId f = chooseLiftedDefiningPoly(root, s.prefix(i));
        if (f == NullPoly)
            return ReductionResult::unknown();

        PolyId lc = leadingCoefficient(f, x);
        Sign lcSign = signAt(lc, s.prefix(i));

        if (lcSign == Sign::Zero || lcSign == Sign::Unknown)
            return ReductionResult::unknown();  // nullification / non-regular

        q = pseudoRemainder(q, f, x);
        q = primitivePart(q);
        q = normalize(q);
    }

    if (kernel_->isZero(q))
        return ReductionResult::zero();

    return ReductionResult::nonzeroCandidate(q);
}
```

这里 `chooseLiftedDefiningPoly` 可以先取第一个 origin，但长期要选择当前 cell guard 中有效的那个。

---

## certified interval sign

如果 reduction 后不是零多项式：

```text
1. 对 q 做 interval evaluation；
2. 如果 interval > 0，返回 Pos；
3. 如果 interval < 0，返回 Neg；
4. 如果 interval 跨 0，refine 相关 algebraic intervals；
5. 超预算则 Unknown。
```

注意：interval sign 只用于证明非零正负。exact zero 必须靠 reduction。

# P4：Algebraic-prefix root isolation + nullification

## 目标

支持：

```text
x^2 - 2 = 0
y^2 - x = 0
```

这需要在 `x = sqrt(2)` 的 prefix 下隔离 `y^2 - sqrt(2)` 的根。

## 接口扩展

```cpp
enum class RootIsolationStatus {
    Success,
    Unknown,
    BackendFailure
};

struct RootIsolationResult {
    RootIsolationStatus status;
    RootSet roots;
};

virtual RootIsolationResult isolateRealRoots(
    PolyId p,
    const SamplePoint& prefix,
    VarId mainVar
) = 0;
```

逐步废弃只接受 `UniPolyId` 的接口，或者把它作为 rational-prefix 快路径。

---

## Libpoly 路径

如果当前 libpoly 版本确实支持：

```text
poly::isolate_real_roots(Polynomial, Assignment)
```

且 assignment 支持 algebraic values，那么可以做：

```text
1. 用 SamplePoint 构造 libpoly Assignment；
2. 调用 isolate_real_roots(poly, assignment)；
3. 转换 roots 为 RealAlg；
4. 对 rational singleton 做 canonicalization；
5. validate roots；
6. 返回 RootIsolationResult。
```

但要加 validator：

```text
- roots 按 compareRealAlg 可排序；
- intervals 不重叠；
- 每个 section sample 代回 signAt(defining poly) 为 Zero；
- 相邻 sector sample 符号稳定；
- 后端异常或验证失败 => Unknown。
```

---

## Nullification contract

在 parent generalization 和 root isolation 中都要检查：

```text
若某个 projection / guard polynomial 在当前 prefix 下恒为 0，
且当前 projection mode 不能证明 delineability，
则当前 branch 不能返回 Unsat。
```

返回策略：

```text
McCallum mode:
  nullification -> Lazard fallback

Conservative mode:
  unresolved nullification -> Unknown

Lazard mode:
  validation 失败 -> Unknown
```

# 当前计划中要删除或改写的旧表述

必须删除：

```text
same isolating interval from different polys -> same value
```

改成：

```text
overlapping intervals trigger gcd/root-locate or refinement;
same interval alone never proves equality.
```

必须删除：

```text
P3 can be implemented directly via libpoly evaluate — no manual tower reduction needed.
```

改成：

```text
libpoly exact evaluation can be used as backend oracle,
but CDCAC core still needs a validated zero/sign path;
for exact zero in mixed samples, tower reduction or trusted libpoly exact proof path is required.
```

必须删除：

```text
P4 can bypass algebraic-coefficient polynomial handling entirely.
```

改成：

```text
P4 may call libpoly algebraic-assignment root isolation as primary backend,
but must validate roots and preserve Unknown-on-failure semantics.
```
