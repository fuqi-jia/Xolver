# NLColver 完整演进计划：Stage A 完善 → Stage K

## 0. 前提条件与依赖状态

### 0.1 依赖管理策略：Vendor Build via Git Submodule

所有外部依赖（CaDiCaL、libpoly、SOMTParser）均以 **git submodule** 管理，通过 CMake **vendor build** 集成，不依赖系统包管理器，不污染系统目录。

**已配置的 submodules：**

| Submodule | 路径 | 构建方式 | CMake 目标 |
|-----------|------|----------|------------|
| SOMTParser | `third_party/SOMTParser` | `add_subdirectory` | `somtparser` |
| CaDiCaL | `third_party/cadical` | `ExternalProject_Add` (`./configure -O && make`) | `cadical` (IMPORTED STATIC) |
| libpoly | `third_party/libpoly` | `add_subdirectory` (禁用 Python/tests) | `poly` / `poly_static` |

**降级策略：**
- 若 submodule 目录为空（未 `git submodule update --init`），CMake 打 **WARNING** 并继续使用 stub 实现，**不报 FATAL_ERROR**
- 这使得新 clone 的开发者可以在不拉取所有 submodule 的情况下编译 Stage A 骨架

**CMake 集成代码（根 CMakeLists.txt）：**

```cmake
# ---------------------------------------------------------------------------
# CaDiCaL (git submodule + ExternalProject_Add)
# ---------------------------------------------------------------------------
if(EXISTS ${CMAKE_SOURCE_DIR}/third_party/cadical/src/cadical.cpp)
    include(ExternalProject)
    ExternalProject_Add(
        cadical_external
        SOURCE_DIR ${CMAKE_SOURCE_DIR}/third_party/cadical
        CONFIGURE_COMMAND ${CMAKE_SOURCE_DIR}/third_party/cadical/configure -O
        BUILD_COMMAND ${MAKE_EXECUTABLE}
        BUILD_IN_SOURCE 1
        INSTALL_COMMAND ""
    )
    add_library(cadical STATIC IMPORTED)
    set_target_properties(cadical PROPERTIES
        IMPORTED_LOCATION ${CMAKE_SOURCE_DIR}/third_party/cadical/build/libcadical.a
        INTERFACE_INCLUDE_DIRECTORIES ${CMAKE_SOURCE_DIR}/third_party/cadical/src
    )
    add_dependencies(cadical cadical_external)
    set(CADICAL_FOUND TRUE)
else()
    set(CADICAL_FOUND FALSE)
    message(WARNING "CaDiCaL submodule not found. SAT backend will be stubbed.")
endif()

# ---------------------------------------------------------------------------
# libpoly (git submodule + add_subdirectory)
# ---------------------------------------------------------------------------
if(EXISTS ${CMAKE_SOURCE_DIR}/third_party/libpoly/src/CMakeLists.txt)
    set(LIBPOLY_BUILD_PYTHON_API OFF CACHE BOOL "" FORCE)
    set(LIBPOLY_BUILD_TEST OFF CACHE BOOL "" FORCE)
    add_subdirectory(third_party/libpoly)
    set(LIBPOLY_FOUND TRUE)
else()
    set(LIBPOLY_FOUND FALSE)
    message(WARNING "libpoly submodule not found. Polynomial kernel will be stubbed.")
endif()
```

**src/CMakeLists.txt 链接方式：**

```cmake
if(CADICAL_FOUND)
    target_link_libraries(nlcolver_core PUBLIC cadical)
    target_compile_definitions(nlcolver_core PUBLIC NLCOLVER_HAS_CADICAL)
endif()

if(LIBPOLY_FOUND)
    target_link_libraries(nlcolver_core PUBLIC poly_static)
    target_compile_definitions(nlcolver_core PUBLIC NLCOLVER_HAS_LIBPOLY)
endif()
```

**CaDiCaL API 映射：**
```cpp
class CadicalBackend : public SatSolver {
    std::unique_ptr<CaDiCaL::Solver> solver;
public:
    SatVar newVar() override { solver->add(0); return ++maxVar; }
    void addClause(const std::vector<SatLit>& clause) override {
        for (auto lit : clause) solver->add(lit.sign ? lit.var : -lit.var);
        solver->add(0);
    }
    SolveResult solve() override {
        int res = solver->solve();
        return res == 10 ? Sat : res == 20 ? Unsat : Unknown;
    }
    bool value(SatVar v) const override { return solver->val(v) > 0; }
};
```

### 1.2 Rewriter 集成

**设计：**
- 不复写 SOMTParser 的 Rewriter，而是**在其基础上注册 NLColver-specific rules**
- 在 `FrontendAdapter::importProblem()` 中，parse 后先 `rewrite()` 再 `importNode()`

```cpp
SOMTParser::Rewriter rewriter(parser.getNodeManager());
installDefaultRewriteRules(rewriter);  // SOMTParser 自带
installNlcolverRewriteRules(rewriter); // 我们的扩展
for (auto& assertion : assertions) {
    assertion = rewriter.rewrite(assertion);
}
```

**NLColver-specific rules（Stage A 最小集）：**
- Arithmetic normalization: `p - q ⋈ 0` 统一
- `>` / `>=` 翻转符号 → `<` / `<=`
- Flatten associative `and` / `or` / `add` / `mul`

**文件：**
- `src/expr/rewriter.h`
- `src/expr/rewriter.cpp`

### 1.3 Push/Pop Scope 管理

**设计：**
- `CoreIr` 中 assertions 不直接存储 vector，而是存储 `(ScopeLevel, ExprId)` 对
- `push()` 增加当前 scope level
- `pop(n)` 移除 scope level > current - n 的 assertions
- Atomizer 的 memo_ 和 SAT solver 的 clause DB 同样需要 scope 感知

**关键修改：**
```cpp
// CoreIr
std::vector<std::pair<ScopeLevel, ExprId>> scopedAssertions_;
void addAssertion(ExprId id, ScopeLevel level);
void popScopes(ScopeLevel targetLevel);
```

**文件：**

## 2. Phase 1: Stage B — Polynomial Kernel

### 2.1 自有 Polynomial 表示

**数据结构（参考 plan.md §3.1）：**
```cpp
struct Monomial {
    SmallVector<std::pair<VarId, uint32_t>, 4> powers;
    bool operator<(const Monomial& o) const; // lex ordering for canonicalization
    bool operator==(const Monomial& o) const;
};

struct Term {
    mpq_class coeff;   // rational coefficient
    Monomial mono;
};

struct Polynomial {
    SmallVector<Term, 8> terms;  // sorted by monomial ordering
    uint32_t total_degree;
    uint64_t hash;
};
```

**Canonicalization 规则：**
1. 合并同类项（相同 monomial 的系数相加）
2. 删除系数为 0 的项
3. 按 monomial 字典序排序
4. 首项系数为正（若多项式非零，乘以 -1）

**文件：**
- `src/theory/arith/poly/Polynomial.h`
- `src/theory/arith/poly/Polynomial.cpp`

### 2.2 PolynomialKernel 接口层

### 4.2 CAlC Solver

**核心算法（参考 plan.md §8.3）：**
```cpp
NraResult CdcacSolver::check(const std::vector<TheoryAtom>& atoms) {
    VarOrder order = orderManager_.choose(atoms);
    Covering cov(order);

    while (true) {
        auto sample = sampleManager_.nextUncovered(cov, atoms);
        if (!sample) {
            // Covering complete → UNSAT
            return unsat(cov.toConflictClause(), cov.toCertificate());
        }

        EvalResult er = evaluator_.evaluate(atoms, *sample);
        if (er.allSatisfied()) {
            return sat(Model::fromSample(*sample));
        }

        ConflictSet cs = conflictSelector_.select(er);
        Cell cell = cellBuilder_.construct(order, *sample, cs);
        Certificate cert = cellCertifier_.certify(cell, cs);

        cov.add(cell, cert);
        trace_.onNraCellGenerated(cell, cs, cert);
    }
}
```

**组件分解：**
- `SampleManager`：sample 选择策略
- `Evaluator`：在 sample point 处评估 polynomial
- `ConflictSelector`：选择 violated constraint subset
- `CellBuilder`：围绕 sample 构建 cell
- `CellCertifier`：验证 cell 正确性

## 5. Phase 4: Stage E — CDCL(T) Integration

### 5.1 完整 CDCL(T) Loop

**Solver::checkSatInternal() 的完整版本：**
```cpp
Result checkSatInternal() {
    // 1. Preprocess / rewrite
    // 2. Atomize → SAT clauses
    // 3. CDCL loop:
    while (true) {
        SatResult sr = sat->solve();
        if (sr == Unsat) return Unsat;
        if (sr == Unknown) return Unknown;

        // 4. Theory check under SAT assignment
        TheoryCheckResult tr = theoryManager.check(sat->getAssignment());
        if (tr == Sat) return Sat;
        if (tr == Conflict) {
            sat->addClause(tr.conflictClause);
            continue;
        }
        if (tr == LemmaGenerated) {
            for (auto& lemma : tr.lemmas) sat->addClause(lemma);
            continue;
        }
    }
}
```
