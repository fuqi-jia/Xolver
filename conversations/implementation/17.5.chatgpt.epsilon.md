这段是在把 **RDL 的 infinitesimal 模型**：

```text
x = a + bδ
```

具体化成普通有理数模型：

```text
x = a + bε
```

其中 `ε > 0` 是一个足够小的有理数。

所以代码里的：

```cpp
mpq_class epsilon = 1;
for (int iter = 0; iter < 100; ++iter) {
    ...
    epsilon /= 2;
}
```

本质是在做：

```text
先试 ε=1；
不行就 ε=1/2；
再不行 ε=1/4；
最多试 100 次。
```

这属于**试探式实现**，不适合严肃求解器。RDL 这里应该精确计算出一个足够小的 `ε`，而不是靠 magic number。

---

# 这段代码的问题

## 1. `epsilon = 1` 是拍脑袋

对于 RDL strict constraint，例如：

```text
x - y < 1
```

内部用：

```text
x - y <= 1 - δ
```

Bellman-Ford 得到的是 `a + bδ` 形式的 symbolic distance。最后必须把 `δ` 替换成一个足够小的正有理数 `ε`。

但 `ε=1` 不一定合法。比如某些约束要求：

```text
ε < 1/1000000
```

那从 `1` 开始试就会失败很多次。

---

## 2. `iter < 100` 是 magic number

理论上，如果合法的 ε 很小，例如：

```text
ε < 2^-200
```

那试 100 次还不够，代码就会返回 `{}`。

但是这不是因为系统无解，而是因为实现自己没试够。

所以这段注释：

```cpp
// Feasible epsilon should always exist for a satisfiable system.
return {};
```

本身就说明设计不严谨：既然理论上存在，就应该**计算出来**，而不是靠循环碰运气。

---

## 3. `return {}` 含义不清

```cpp
return {};
```

如果返回类型是：

```cpp
std::unordered_map<std::string, mpq_class>
```

那 `{}` 既可能表示：

```text
失败
```

也可能表示：

```text
模型为空，因为没有变量
```

这不专业。

应该改成：

```cpp
std::optional<std::unordered_map<std::string, mpq_class>>
```

失败返回：

```cpp
std::nullopt
```

成功返回：

```cpp
model
```

---

## 4. 这里不应该用 `graph.edges()` 里的所有边

如果图里支持 inactive edge，那么必须只检查 active edges。

否则 backtrack 后的旧边可能还在 `graph.edges()` 中，导致模型被错误判失败。

应该用：

```cpp
for (const auto& e : graph.activeEdges())
```

或者在 `edges()` 内部保证只返回 active edge。

---

## 5. `mpq_model` 看起来是 typo

代码里有：

```cpp
mpq_model lhs = ...
mpq_model rhs = ...
```

这大概率应该是：

```cpp
mpq_class lhs = ...
mpq_class rhs = ...
```

除非你们自己定义了 `mpq_model`，否则这会直接编译不过。

---

# 正确做法：一次性精确计算 ε

RDL 中每个 distance 是：

```text
dist[v] = A_v + B_v δ
```

每条边：

```text
u -> v, weight = C + Dδ
```

含义是：

```text
dist[v] <= dist[u] + weight
```

把 `δ` 替换为 `ε` 后，需要满足：

```text
(A_v + B_v ε) <= (A_u + B_u ε) + C + D ε
```

移项：

```text
P + Qε <= 0
```

其中：

```text
P = A_v - A_u - C
Q = B_v - B_u - D
```

因为 Bellman-Ford 已经在 lexicographic order 下保证：

```text
(P, Q) <= (0, 0)
```

所以只需要处理几种情况。

---

## 情况 1：`P < 0`

```text
P + Qε <= 0
```

如果：

```text
Q <= 0
```

那么任意 `ε > 0` 都满足。

如果：

```text
Q > 0
```

则需要：

```text
ε <= -P / Q
```

所以收集一个 upper bound：

```text
epsilonUpper = min(epsilonUpper, -P / Q)
```

---

## 情况 2：`P == 0`

此时必须有：

```text
Q <= 0
```

否则说明 symbolic distance 本身不满足边约束，Bellman-Ford 或 weight 比较实现有 bug。

如果 `Q <= 0`，任意 `ε > 0` 都满足。

---

## 情况 3：`P > 0`

这不应该发生。

说明：

```text
dist[v] <= dist[u] + weight
```

在 rational part 上已经失败了。应该返回 internal error / nullopt，而不是继续试 ε。

---

# 推荐实现

把函数改成这样：

```cpp
inline std::optional<std::unordered_map<std::string, mpq_class>>
buildRdlModel(
    const std::vector<RdlWeight>& dist,
    int zeroNode,
    const DifferenceGraph<RdlWeight>& graph
) {
    std::optional<mpq_class> upper;

    for (const auto& e : graph.activeEdges()) {
        const RdlWeight& dv = dist[e.to];
        const RdlWeight& du = dist[e.from];

        mpq_class P = dv.c - du.c - e.weight.c;
        int Q = dv.deltaCoeff - du.deltaCoeff - e.weight.deltaCoeff;

        if (P > 0) {
            return std::nullopt;
        }

        if (P == 0) {
            if (Q > 0) {
                return std::nullopt;
            }
            continue;
        }

        // P < 0
        if (Q > 0) {
            mpq_class bound = (-P) / Q;
            if (!upper || bound < *upper) {
                upper = bound;
            }
        }
    }

    mpq_class epsilon;
    if (upper) {
        if (*upper <= 0) {
            return std::nullopt;
        }
        epsilon = *upper / 2;
    } else {
        epsilon = 1;
    }

    std::unordered_map<std::string, mpq_class> model;

    for (int i = 0; i < graph.numNodes(); ++i) {
        if (i == zeroNode) continue;

        const auto name = graph.nodeName(i);
        if (name.empty()) continue;

        model[name] =
            (dist[i].c - dist[zeroNode].c)
            + (dist[i].deltaCoeff - dist[zeroNode].deltaCoeff) * epsilon;
    }

    return model;
}
```

这个版本没有：

```text
epsilon = 1 然后试 100 次
```

而是从所有边约束中直接算出：

```text
ε 的最大允许上界
```

再取一半。

# 还需要一个最终 validate

即使你精确算了 ε，也必须 validate 一遍：

```cpp
auto model = buildRdlModel(...);
if (!model) {
    return Unknown;
}

if (!validateRdlModel(*model, activeAtoms)) {
    return Unknown;
}

return Sat;
```

不能直接相信构造过程。

原因是：

```text
1. graph 里可能漏了某些 active atom；
2. disequality 可能没进边；
3. dynamic atom 可能 route 错；
4. strict weight 或 epsilon 计算可能有 bug。
```

所以 RDL 最终 SAT 前必须 exact validation。

```text
这个 buildRdlModel 不能用 epsilon=1 然后循环除以 2 试 100 次。
这是 magic-number search，不适合 solver。

请改成 exact epsilon computation。

For each active edge u -> v with weight C + Dδ and distances:
  dist[v] = A_v + B_vδ
  dist[u] = A_u + B_uδ

After substituting δ=ε, we need:
  (A_v - A_u - C) + (B_v - B_u - D) ε <= 0

Let:
  P = A_v - A_u - C
  Q = B_v - B_u - D

Cases:
  P > 0:
    internal failure / nullopt
  P == 0:
    require Q <= 0, otherwise nullopt
  P < 0 and Q > 0:
    collect ε <= -P/Q
  P < 0 and Q <= 0:
    no restriction

Choose:
  ε = min(all collected positive bounds) / 2
or ε = 1 if no bound exists.

Also:
  - return optional<model>, not empty map on failure;
  - iterate only active edges;
  - replace mpq_model with mpq_class unless mpq_model is intentionally defined;
  - validate the final rational model before returning SAT.
```

# 总结

这段代码想解决的问题是对的：
RDL 用 `δ` 表示 strict inequality，最后要实例化成具体有理数 `ε`。

但当前实现方式不对：

```text
epsilon = 1;
循环除以 2；
最多 100 次；
失败 return {};
```

这是临时 hack。

正确做法是：

```text
从所有边约束精确推出 ε 的上界；
选一个足够小的正有理数；
构造模型；
最终 validate。
```

这样才是可靠的 RDL solver 实现。
