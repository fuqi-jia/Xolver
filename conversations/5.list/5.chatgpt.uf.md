QF_UF 比 LIA/NIA/NRA 简单很多。**如果只看纯 QF_UF，也就是 ground EUF，没有量词、没有算术、没有数组，那么最小超越集非常清楚：增量 E-graph + backtrackable union-find + signature table + explanation forest + disequality watch + CDCL(T) theory propagation + model construction。**

Z3 文档里对 UF 的核心解释也是 congruence closure：当同一函数的参数相等时，可以推出函数应用相等；cvc5 系统论文也说明其 Equality Engine 负责 congruence closure、冲突检测，并通过 inference manager 发出 lemma、conflict 和 propagation。([GitHub微软][1]) 你给的列表也正好覆盖了 UF/EUF 核心、预处理、CDCL(T)、Nelson-Oppen、量词和高级优化几块。
