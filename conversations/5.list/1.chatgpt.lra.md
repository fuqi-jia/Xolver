结论：**不存在“做完某几个功能就数学上必然超过 SOTA”的保证**。LRA 的 SOTA 差距主要不在“功能是否有”，而在 **CDCL(T) 集成、增量回溯、稀疏精确计算、冲突解释质量、pivot/传播策略、benchmark 工程细节**。不过，如果按“最小技术闭环”来选，你这 80 项里真正值得作为“超越 SOTA LRA solver”的最小集合，大概是下面这些。

当前可比对象里，2025 SMT-COMP 的 QF_LRA Parallel Track 是 **SMTS** 最强，后面有 Yices2 和 Z3-Parti-Z3pp；2024 Single Query 的 QF_LinearRealArith 则 OpenSMT、Yices2、cvc5、Z3-alpha 都很强。也就是说，目标不是“超过一个玩具 Simplex”，而是要压过这些成熟 CDCL(T)+LRA 工程实现。([SMT-COMP][1])

## 最小超越集：建议锁定 36 项

### A. 核心求解内核：必须做

这些决定你是不是一个真正的 SMT-LRA solver，而不是普通 LP solver：

1. **Simplex**
2. **Dual Simplex**
3. **Incremental Simplex**
4. **Backtrackable Simplex**
5. **Tableau-based Simplex**
6. **Sparse Tableau**
7. **Revised Simplex**
8. **Basic / Non-basic Variable Management**
9. **Pivot Selection Heuristic**
10. **Degeneracy Handling**
11. **Cycling Prevention**
12. **Incremental Feasibility Maintenance**

这里的核心是：**incremental + backtrackable + sparse + revised/tableau hybrid + stable pivot**。Dutertre–de Moura 的经典 LRA SMT simplex 关键优势就是能高效接入 DPLL(T)、快速回溯、预简化、理论传播，并处理 strict inequalities；cvc5 也明确是在这个方向上扩展其线性算术 solver。

### B. 传播与约简：必须做

9. **Bound Propagation**
10. **Bound Tightening**
11. **Difference Logic Detection**
12. **Bound Constraint Detection**
13. **Strict Inequality Handling**
14. **Epsilon / Infinitesimal Reasoning**
15. **Disequality Handling**
16. **Constraint Normalization**
17. **Coefficient Normalization**
18. **Integer Coefficient Scaling**
19. **Denominator Clearing**
20. **Tautology Elimination**
21. **Trivial Conflict Detection**
22. **Duplicate Constraint Detection**
23. **Propagation Queue**
24. **Watch-based Bound Propagation**
25. **Implied Bound Detection**
26. **Fixed Variable Detection**

这里不是为了“功能完整”，而是为了减少 Simplex 调用次数。SOTA LRA solver 很多优势来自搜索过程中尽早传播、尽早发现 infeasible bound、避免重复构造约束。Yices 的论文也强调其 sparse linear arithmetic solver 在 full linear arithmetic 和 difference logic fragment 上都很高效。

### C. 冲突解释与 lemma：必须做

30. **Conflict Analysis**
31. **Farkas Lemma Explanation**
32. **Minimal Conflict Extraction**
33. **Theory Lemma Generation**
34. **Explanation Cache**
35. **Infeasible Bound Detection**

这是能不能超过 SOTA 的分水岭。LRA 不是只要判定 SAT/UNSAT，还要向 SAT core 返回**短、稳定、可复用**的 theory conflict / theory propagation lemma。否则即使 Simplex 很快，CDCL 层也会学到垃圾 clause。

### D. 增量回溯状态：必须做

37. **Incremental Assertion**
38. **Push / Pop Backtracking**
39. **Trail-based State Restoration**
40. **Basis Restoration**
41. **Sparse Matrix Update**
42. **Bound Cache**

这里优先级高于很多“高级数值优化技巧”。SMT 场景里频繁 assert / retract / backjump，如果每次重建 tableau 或 basis，就很难赢成熟 solver。

### E. 精确算术与模型：必须做

35. **Model Construction**
36. **Model Validation**
37. **Rational Arithmetic**
38. **Exact Arithmetic**
39. **Numeral Canonicalization**
40. **Feasible Assignment Repair**

LRA 不能靠 double。要超过 SOTA，至少要做到：内部可以用快速表示，但最终冲突、传播、模型验证必须 exact。否则 benchmark 上迟早 false sat / false unsat。

### F. CDCL(T) / theory combination 接口：必须做

67. **Phase-aware Model Repair**
68. **Theory Combination Interface**
69. **Nelson-Oppen Equality Propagation**

如果只做纯 QF_LRA，可以暂时弱化 70/71；但如果目标是 SMT 求解器整体超过 SOTA，而不是单独 LRA checker，这两个必须做。cvc5 的体系里，各 theory solver 通过统一 inference manager 发出 lemma、conflict 和 propagation，这类接口工程本身就是 SOTA 的一部分。([Computer Science][2])

---

## 我会删掉或后置的项

这些不是不重要，而是**不属于最小超越集**：

12. **Gaussian Elimination**：可以作为 equality preprocessing，但不是主闭环。
13. **Bland’s Rule**：保底规则可以有，但不是性能核心。
14. **Devex Pricing**、28. **Steepest-edge Pricing**、29. **Bound-flipping Ratio Test**：这些是增强项，不是第一批最小集。
15. **Tableau Snapshotting**：通常不如 trail/basis restoration 精细。
16. **Eta Matrix Update**、44. **LU Factorization**：如果你的 revised simplex 已经成熟再上；第一版不必硬做。
17. **Redundant Constraint Removal**、52. **Constraint Subsumption**：有用，但可能代价大，先轻量化。
18. **Variable Elimination**：谨慎，容易破坏 explanation 和 incrementality。
19. **Optimization Support**、66. **Objective Bound Propagation**：这是 OMT，不是最小 LRA-SMT 超越集。
20. **LRA-LIA Interface**、69. **LRA-NRA Abstraction Interface**：对混合理论重要，但不是纯 LRA 最小集。
21. **Proof Logging**、73. **Proof-producing Farkas Certificates**：论文/可信验证很重要，但如果目标是性能超越，可以后置。
