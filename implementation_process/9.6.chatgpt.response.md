建议继续选 **[4] Revise**。这版比上一版成熟很多：已经补了 effective literal、DomainStore reasons、BoundedSolveStatus、dynamic polynomial atom route、modular conflict reasons、interpolation conservative fallback、branch lemma 约束等关键点。整体方向可以保留。

但还有几个**正确性硬问题**，尤其第 1 条必须改。

---

# 1. `NiaNormalizer` 的 false literal 取反映射有 soundness bug

现在 plan 写：

```text
Leq → Geq (since ¬(p <= 0) ≡ p > 0)
Geq → Leq (since ¬(p >= 0) ≡ p < 0)
```

这里不对。`¬(p <= 0)` 是 `p > 0`，在整数上应归一化为：

```text
p >= 1
```

如果直接映射成 `Geq`，就会变成：

```text
p >= 0
```

这是弱了一档，可能把不满足原公式的模型放进来。

正确映射必须保留 strict relation：

```text
Eq  false -> Neq
Neq false -> Eq
Leq false -> Gt
Geq false -> Lt
Lt  false -> Geq
Gt  false -> Leq
```

然后再做 strict normalization：

```text
Gt: p > 0  -> p >= 1
Lt: p < 0  -> p <= -1
```

必须让 Kimi 改成：

```text
Do not map false Leq directly to Geq.
Do not map false Geq directly to Leq.
Use Gt/Lt first, then normalize strict integer inequalities.
```

这是当前最大 blocker。

---

# 2. `DomainStore` 的 reason 仍然不够强

现在 `IntDomain` 里：

```cpp
SatLit finiteSetReason;
std::map<mpz_class, SatLit> excludedValues;
```

这还不够。因为 domain update 可能连续发生：

```text
x ∈ {1,2,3}   reason A
x ∈ {2,3,4}   reason B
x != 2        reason C
x != 3        reason D
```

最终空域的 conflict 需要：

```text
¬A ∨ ¬B ∨ ¬C ∨ ¬D
```

单个 `finiteSetReason` 无法表达多个 finite-set restriction 的交集来源。

建议改成：

```cpp
std::vector<SatLit> finiteSetReasons;
std::map<mpz_class, std::vector<SatLit>> excludedValueReasons;
```

或者更统一：

```cpp
struct DomainRestriction {
    enum Kind { Lower, Upper, FiniteSet, Exclude };
    ...
    std::vector<SatLit> reasons;
};
```

`buildEmptyDomainConflict()` 必须收集导致空域的**完整 restriction reasons**，不能只取最后一次更新的 reason。

---

# 3. 需要显式加入线性 domain inference

现在有 `DomainStore`，但没有一个明确组件把 normalized linear constraints 写入 domain。

NIA 里会经常出现：

```smt2
(assert (>= x 0))
(assert (<= x 10))
(assert (= (* x x) 49))
```

虽然 `x^2=49` 可以给出 `{ -7, 7 }`，然后 validator 能筛出 `7`，但更一般的 bounded search 需要线性约束填充 domain：

```text
x <= c      -> addUpperBound(x,c)
x >= c      -> addLowerBound(x,c)
x = c       -> restrictToFiniteSet(x,{c})
x != c      -> excludeValue(x,c)
```

建议增加一个组件：

```cpp
LinearNiaDomainReasoner
```

输入 normalized NIA constraints，只处理单变量一次线性形式：

```text
a*x + c rel 0
```

推出 integer domain update。它应在 univariate/algebraic 之前或之后运行都可以，但必须在 bounded solver 前运行。

---

# 4. Algebraic GCD / square / factor 规则需要多项式 term 接口，否则只能 skip

Plan 只给了：

```cpp
degree(poly, var)
getIntegerCoefficients(poly, var)
```

这足够做 univariate RRT，但不够做：

```text
2x^2 + 4yz = 3 的 coefficient GCD
x^2 + y^2 <= 0 的 square recognition
p*q = 0 的 syntactic product recognition
```

这些需要至少一种接口：

```cpp
getTerms(poly)
getMonomials(poly)
getConstantTerm(poly)
isSyntacticProduct(poly)
```

或者说明这些规则只能基于 `PolynomialConverter` 保留的 AST/syntax metadata 做。

必须补一条 soundness rule：

```text
If term decomposition / syntactic product recognition is unavailable, AlgebraicIntegerReasoner must skip the rule, never guess.
```

否则很容易把不完整的结构识别当成证明。

---

# 5. Bounded solver 的 conflict over all active reasons 是 sound，但要写清楚是粗 conflict

这句：

```text
return {UnsatComplete, nullopt, allActiveReasonsConflict}
```

可以接受，因为完整搜索证明当前全部 active constraints 不可满足，返回所有 active reasons 的 negation 是 sound 的。

但建议写清楚：

```text
This is a coarse but sound conflict.
It must include every active normalized constraint reason used by the bounded solver.
```

如果 bounded solver 只用了部分 constraints，则 conflict 只能包含用到的那部分；最简单就是 always use all active reasons。
