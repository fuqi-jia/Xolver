建议 **继续 Revise**。这版仍然没有解决最关键的 `markEmitted` 问题，所以还不能 Approve。

大的方向没有问题：独立 `IncrementalLinearizer`、V1 只做 `x*y / x^2`、NIA cuts route 到 LIA、NRA cuts route 到 LRA、co-register LIA/LRA、mirror effective active linear bounds，这些都是对的。计划也已经明确如果不 mirror bounds，LIA 可以自由设置 `x=21,y=21`，导致 `0<=x,y<=10 ∧ x*y>200` 无法被 McCormick 证明 UNSAT，这个判断正确。

但还有两个 blocker。

---

# 1. `LinearizationResult` 仍然只有 `vector<TheoryLemma>`

现在结果类型仍然是：

```cpp
struct LinearizationResult {
    LinearizationStatus status;
    std::vector<TheoryLemma> lemmas;
};
```

但 cache API 需要：

```cpp
markEmitted(
    NonlinearTermKey key,
    SatLit nonlinearReason,
    vector<mpq_class> boundValues,
    int cutIndex
)
```

这两个对不上。`TheoryLemma` 本身不携带：

```text
NonlinearTermKey
nonlinearReason
boundValues
cutIndex
```

所以 adapter 无法正确调用 `markEmitted`。

而计划后面又写：

```text
Cache markEmitted: called by adapter AFTER lemma is enqueued in pending queue
```

这仍然缺少 metadata。

必须改成：

```cpp
struct CutCacheKey {
    NonlinearTermKey key;
    SatLit nonlinearReason;
    std::vector<mpq_class> boundValues;
    int cutIndex;
};

struct PendingLinearizationLemma {
    TheoryLemma lemma;
    std::optional<CutCacheKey> cacheKey;
};

struct LinearizationResult {
    LinearizationStatus status;
    std::vector<PendingLinearizationLemma> lemmas;
};
```

然后 adapter 才能写：

```cpp
for (auto& item : lr.lemmas) {
    if (!lemmaDb.contains(item.lemma)) {
        lemmaDb.insertIfNew(item.lemma);
        pendingLinLemmas_.push_back(std::move(item.lemma));

        if (item.cacheKey) {
            linAdapter_->markEmitted(*item.cacheKey);
        }
    }
}
```

`abstraction lemma` 可以没有 `cacheKey`；McCormick / square cut lemma 应该带 `cacheKey`。

---

# 2. D4 仍然有一句不安全表述

现在 D4 仍写：

```text
If bounds are unconditional (no reasons), omit bound guards.
```

这句话容易被实现成：

```cpp
if (reasons.empty()) omitGuard();
```

这是不安全的。空 reasons 可能只是 reason 丢失，不代表 bound 是 global。你前面已经多次确认过：**empty reasons does not mean unconditional**。

应该改成：

```text
Only omit bound guards when the bound is explicitly marked global.
Empty reason vectors alone do not imply unconditional bounds.
If finite bounds are not global and reasonComplete=false, skip the bounds-dependent cut in V1.
```

也就是说 V1 不做 coarse all-active guard，直接 skip incomplete-reason cuts，最安全。
