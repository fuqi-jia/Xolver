# 1. `terms()` 必须明确怎么表示常数项

当前 plan 说：

```cpp
struct MonomialTerm {
    mpz_class coefficient;
    std::vector<std::pair<std::string, int>> powers;
};
```

但又写：

```text
If polynomial is constant → return nullopt.
```

这会影响 multivar GCD，因为 GCD conflict 需要把：

```text
p = q + c
```

里的常数项 `c` 精确拿出来。

比如：

```text
2x + 4y - 3 = 0
```

`terms()` 必须返回类似：

```text
{2*x, 4*y, -3}
```

其中常数项可以表示为：

```cpp
MonomialTerm{ coefficient = -3, powers = {} }
```

否则 `checkGcdConflict` 没法可靠得到 `c`。

建议改成：

```text
terms(poly) returns all monomial terms, including constant term as powers.empty().
For a constant polynomial c, return vector{ MonomialTerm{c, {}} }, not nullopt.
Return nullopt only if decomposition fails, coefficients are not exact integers, or backend unsupported.
```

这样 multivar GCD 的实现才闭环。

---

# 2. 公共 interval 层不能依赖 `NiaReasoningKind` / `TheoryConflict`

当前 C4 里写：

```cpp
struct IntervalEvalResult {
    NiaReasoningKind kind;
    std::optional<TheoryConflict> conflict;
    std::vector<SatLit> usedReasons;
};
```

这会把 `src/theory/arith/interval/` 反向耦合到 NIA 和 theory conflict。既然这个目录是公共 interval/ICP skeleton，就应该返回通用结果，不要出现 `NiaReasoningKind`。

建议改成：

```cpp
enum class IntervalEvalStatus {
    NoChange,
    DefinitelyViolated
};

struct IntervalEvalResult {
    IntervalEvalStatus status;
    std::vector<SatLit> usedReasons;
};
```

然后由 NIA adapter 构造：

```cpp
if (ir.status == IntervalEvalStatus::DefinitelyViolated) {
    auto lits = violatedConstraintReason;
    append(lits, ir.usedReasons);
    return NiaReasoningResult::conflict(TheoryConflict{lits});
}
```

公共 interval 层只回答：

```text
这个 constraint 在这个 ReasonedBox 下是否 definitely violated；
这个判断用了哪些 bound reasons。
```

不要直接返回 NIA conflict。

---

# 3. GCD 公式建议写成 `g ∤ -c`

现在写的是：

```text
constant term c % g != 0 → UNSAT
```

数学上 `c % g != 0` 和 `(-c) % g != 0` 等价，但为了和定义一致，建议统一写：

```text
p = q + c
if g does not divide -c -> UNSAT
```

这样不会在实现里因为符号理解产生错位。
