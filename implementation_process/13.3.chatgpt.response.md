建议选 **[4] Revise**。这版已经把“角点采样不是 over-approx”的核心 soundness 问题修掉了，方向对：只做单变量 interval arithmetic，多变量 `NoChange`，无上下界变量 `NoChange`，不做反向传播、不做采样判断 UNSAT，这些都正确。

但还缺一个**关键正确性点**：**interval violation 返回 UNSAT/conflict 时，conflict 不能只包含当前非线性约束本身，还必须包含用于得到变量区间的 domain reasons。**

例如：

```smt2
(assert (>= x 0))
(assert (<= x 100000))
(assert (<= (+ (* x x x) 1) 0))
```

通过 interval 得到：

```text
x ∈ [0,100000]
x³ + 1 ∈ [1, 100000³+1]
```

因此 `x³+1<=0` definitely violated。

但这个 UNSAT 证明依赖三个条件：

```text
x >= 0
x <= 100000
x³ + 1 <= 0
```

不能只返回：

```text
¬(x³ + 1 <= 0)
```

因为单独的 `x³ + 1 <= 0` 是可满足的，例如 `x=-1`。

所以需要补一条：

```text
When IntervalEvaluator returns Conflict because a constraint is definitely violated under a domain interval, the conflict must include:
  1. the violated constraint reason;
  2. all DomainStore reasons used to justify the interval bounds of every variable involved.

If complete bound reasons cannot be collected, return NoChange/Unknown rather than Conflict.
```

第一版也可以用更粗但 sound 的方式：

```text
For Phase 3, interval UNSAT may return a coarse conflict containing all active normalized constraint reasons.
```

这样简单安全。
