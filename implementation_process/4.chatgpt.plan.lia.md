下面给你一套**可以直接用于 SMT 求解器实现的 LIA 求解算法方案**。我按你现在的 NLColver / GeneralSimplex 设计来写：**GeneralSimplex 做 LRA relaxation 内核，LIA solver 在外层做整数推理、cube test、cuts、branch-and-bound，并且所有 sat/unsat 都要有验证或解释**。

目标逻辑先限定为：

```text
QF_LIA:
  quantifier-free linear integer arithmetic

输入原子：
  a1*x1 + ... + an*xn <= c
  a1*x1 + ... + an*xn <  c
  a1*x1 + ... + an*xn >= c
  a1*x1 + ... + an*xn >  c
  a1*x1 + ... + an*xn =  c
  a1*x1 + ... + an*xn != c

变量：
  x_i ∈ Z
```

后面可以自然扩展到 mixed LIRA：

```text
x_i ∈ Z,  y_j ∈ Q
```

---

# 1. 总体架构

LIA 不应该单独重写一个 solver，而应该分成两层：

```text
LiaSolver
  ├── GeneralSimplex        // LRA relaxation
  ├── IntegerModelChecker   // 检查 rational model 是否 integral
  ├── DivisibilityReasoner  // gcd / modular / Diophantine reasoning
  ├── BoundTightener        // 整数上下界加强
  ├── CubeTester            // unit cube / largest cube
  ├── CutGenerator          // Gomory / MIR / gcd cuts
  ├── BranchSelector        // branch-and-bound split
  ├── LemmaBuilder          // 生成 guarded theory lemma
  └── LiaModelValidator     // 最终 exact model validation
```

主流程：

```text
当前 SAT assignment
  ↓
assert active LIA literals into GeneralSimplex
  ↓
GeneralSimplex.check()
  ↓
若 LRA UNSAT：返回 LIA conflict
  ↓
若 LRA SAT：得到 rational model β
  ↓
检查 integer variables 是否全为整数
  ↓
若全整数：model validation → SAT
  ↓
否则：
    divisibility reasoning
    bound tightening
    cube test
    cuts
    branch split
```

核心原则：

```text
1. LRA UNSAT ⇒ LIA UNSAT。
2. LRA SAT + integral model ⇒ LIA SAT。
3. cube test 成功 ⇒ LIA SAT。
4. cut 只能删除当前 fractional rational model，不能删除任何整数解。
5. branch split 必须是整数理论永真式。
6. 所有 theory lemma 都必须是 T-valid clause，不能随便加 unit cut。
7. 所有 SAT model 必须重新 exact validate。
```

## 2.4 LRA relaxation auxiliary variable

复用 GeneralSimplex 的设计。

对每个 linear form：

```text
p(x) = a1*x1 + ... + an*xn + c
```

创建辅助变量：

```text
s_p = p(x)
```

tableau row：

```text
s_p = c + a1*x1 + ... + an*xn
```

然后不同 relation 只是在 `s_p` 上加 bound：

```text
p <= 0   ⇒   s_p <= 0
p <  0   ⇒   s_p <= -δ
p >= 0   ⇒   s_p >= 0
p >  0   ⇒   s_p >= δ
p =  0   ⇒   s_p <= 0 and s_p >= 0
p != 0   ⇒   disequality obligation
```

注意：**aux var 应该按 LinearFormKey 复用，而不是按 AtomKey 复用。**

```cpp
struct LinearFormKey {
    std::vector<std::pair<VarId, mpz_class>> terms;
    mpz_class constant;
};

struct AtomKey {
    LinearFormKey form;
    Relation rel;
};
```

映射：

```cpp
std::unordered_map<LinearFormKey, GsVar> formToAux_;
std::unordered_map<AtomKey, LiaAtomInfo> atomInfo_;
```

# 3. `assertLit`：把 SAT literal 转成 LRA bound / disequality obligation

假设当前 SAT model 中某个 theory atom literal 为 true 或 false。

## 3.1 普通不等式

对 atom：

```text
s <= 0
```

如果 literal true：

```text
s <= 0
```

如果 literal false：

```text
s > 0
```

也就是：

```text
s >= δ
```

完整映射：

```text
atom        true literal             false literal
--------------------------------------------------
s <= 0      upper(s, 0)              lower(s, δ)
s <  0      upper(s, -δ)             lower(s, 0)
s >= 0      lower(s, 0)              upper(s, -δ)
s >  0      lower(s, δ)              upper(s, 0)
s =  0      lower(s,0), upper(s,0)   disequality obligation
s != 0      disequality obligation   lower(s,0), upper(s,0)
```

## 3.2 disequality obligation

`p(x) != 0` 不能直接转成 simplex bound。

处理方式：

```cpp
struct DiseqObligation {
    SatLit reasonLit; // 当前为真的 disequality literal
    GsVar aux;        // s_p
};
```

当前 LRA model β 下：

```text
如果 β(s_p) != 0：
    obligation satisfied.

如果 β(s_p) = 0：
    需要生成 split lemma:
        ¬(p != 0) ∨ p < 0 ∨ p > 0
```

内部用 δ 写成：

```text
¬lit_neq ∨ (s_p <= -δ) ∨ (s_p >= δ)
```

注意：这不是 conflict，而是 split lemma。

# 4. LIA 主算法

总入口：

```cpp
TheoryCheckResult LiaSolver::check(TheoryCheckLevel level) {
    // 1. LRA relaxation
    auto lra = simplex_.check();

    if (lra.unsat()) {
        return buildLraConflict();
    }

    Model beta = simplex_.model();

    // 2. Check disequality obligations
    if (auto lemma = checkDisequalities(beta)) {
        return TheoryCheckResult::lemma(*lemma);
    }

    // 3. Integral model?
    if (allIntegerVarsIntegral(beta)) {
        if (modelValidator_.validate(currentAtoms_, beta)) {
            return TheoryCheckResult::sat(beta);
        }
        // 若这里失败，是 solver bug，不应继续悄悄搜索
        return internalError("Integral model failed validation");
    }

    // 4. Cheap integer reasoning
    if (auto r = divisibilityReasoner_.check(currentAtoms_)) {
        return *r; // conflict or lemma
    }

    if (auto lemma = boundTightener_.tryTighten(currentAtoms_, beta)) {
        return TheoryCheckResult::lemma(*lemma);
    }

    // 5. Cube test: try to find integer point
    if (level >= FULL_EFFORT && shouldRunCubeTest(beta)) {
        auto cube = cubeTester_.unitCubeTest(currentLinearSystem_);
        if (cube.sat()) {
            if (modelValidator_.validate(currentAtoms_, cube.model())) {
                return TheoryCheckResult::sat(cube.model());
            }
            return internalError("Cube model failed validation");
        }
    }

    // 6. Generate cuts
    for (int i = 0; i < options_.maxCutsPerCheck; ++i) {
        auto cut = cutGenerator_.generate(simplex_.state(), beta);
        if (!cut) break;

        auto lemma = lemmaBuilder_.buildCutLemma(*cut);
        if (lemma.isUseful()) {
            return TheoryCheckResult::lemma(lemma);
        }
    }

    // 7. Branch-and-bound split
    BranchSplit split = branchSelector_.makeSplit(beta, integerVars_);
    return TheoryCheckResult::lemma(lemmaBuilder_.buildBranchLemma(split));
}
```

# 5. Integer model check

LRA simplex 得到的是 rational assignment：

```text
β(x_i) ∈ Q
```

对每个 integer var：

```cpp
bool isIntegral(const Rational& q) {
    return q.get_den() == 1;
}
```

实现：

```cpp
bool allIntegerVarsIntegral(const Model& beta) {
    for (VarId x : integerVars_) {
        if (!isIntegral(beta[x])) return false;
    }
    return true;
}
```

如果全部整数，仍然要完整验证：

```cpp
bool LiaModelValidator::validate(const std::vector<LiaAtom>& atoms,
                                 const Model& m) {
    for (VarId x : integerVars_) {
        if (!isIntegral(m[x])) return false;
    }

    for (const LiaAtom& atom : atoms) {
        mpq_class val = eval(atom.expr, m);

        if (!satisfies(val, atom.rel, 0)) {
            return false;
        }
    }

    return true;
}
```

# 6. Divisibility reasoning

这是 LIA 必须有的 cheap reasoning。

## 6.1 gcd equality conflict

对等式：

```text
a1*x1 + ... + an*xn = c
```

如果所有 `x_i ∈ Z`，那么左边一定是：

```text
g = gcd(a1,...,an)
```

的倍数。

如果：

```text
g ∤ c
```

则 UNSAT。

例子：

```text
6x + 10y = 7
```

因为：

```text
gcd(6,10)=2
2 ∤ 7
```

所以无整数解。

伪代码：

```cpp
optional<TheoryConflict> DivisibilityReasoner::checkEqGcd(
    const ActiveConstraint& c
) {
    if (c.rel != EQ) return none;
    if (!allVarsInteger(c.expr)) return none;

    mpz_class g = gcdAbs(c.expr.coeffs());
    mpz_class rhs = -c.expr.constant;

    if (rhs % g != 0) {
        return conflict({c.reasonLit});
    }

    return none;
}
```

对应 theory conflict clause：

```text
¬lit_eq
```

因为这个 active equality literal 本身不可能成立。

---

## 6.2 gcd normalization

如果：

```text
6x + 10y = 8
```

可以化简成：

```text
3x + 5y = 4
```

这不是 lemma，而是 preprocessing / normalization。

伪代码：

```cpp
LinearEquality normalizeGcd(LinearEquality eq) {
    mpz_class g = gcdAbs(eq.coeffs(), eq.rhs);
    divide all coeffs and rhs by g;
    normalize sign;
    return eq;
}
```

## 6.3 gcd inequality tightening

对 inequality：

```text
a1*x1 + ... + an*xn <= c
```

如果所有变量是整数，左边一定是 `g = gcd(a_i)` 的倍数。

所以可以加强：

```text
a·x <= g * floor(c/g)
```

例子：

```text
6x + 10y <= 23
```

左边是 2 的倍数，所以：

```text
6x + 10y <= 22
```

伪代码：

```cpp
optional<Cut> gcdTightenLeq(const ActiveConstraint& c) {
    if (c.rel != LEQ) return none;
    if (!allVarsInteger(c.expr)) return none;

    mpz_class g = gcdAbs(c.expr.coeffs());
    if (g <= 1) return none;

    mpz_class rhs = -c.expr.constant;
    mpz_class newRhs = g * floorDiv(rhs, g);

    if (newRhs < rhs) {
        LinearExpr tightened = c.expr;
        tightened.constant = -newRhs;
        return Cut::guardedBy({c.reasonLit}, tightened, LEQ);
    }

    return none;
}
```

注意：这是从 active literal 推出的，所以 lemma 不能直接是 unit，应该是 guarded：

```text
¬lit_original ∨ lit_tightened
```

如果 original 是 top-level assertion，SAT 层里它永远 true，那么也可以等价变成 unit；但通用 CDCL(T) 里必须 guarded。

# 7. Bound tightening

## 7.1 单变量 bound tightening

如果 active constraint 是：

```text
a*x <= c
```

且 `x ∈ Z`。

若 `a > 0`：

```text
x <= floor(c/a)
```

若 `a < 0`：

```text
x >= ceil(c/a)
```

例子：

```text
2x <= 11
```

推出：

```text
x <= 5
```

伪代码：

```cpp
optional<TheoryLemma> BoundTightener::singleVarTighten(
    const ActiveConstraint& c
) {
    if (!isSingleVariable(c.expr)) return none;

    VarId x = c.expr.singleVar();
    if (!isIntegerVar(x)) return none;

    mpz_class a = c.expr.coeff(x);
    mpz_class rhs = -c.expr.constant;

    if (c.rel == LEQ) {
        if (a > 0) {
            auto b = floorDiv(rhs, a);
            return guardedLemma({c.reasonLit}, mkLeq(x, b));
        } else {
            auto b = ceilDiv(rhs, a);
            return guardedLemma({c.reasonLit}, mkGeq(x, b));
        }
    }

    return none;
}
```

## 7.2 bound tightening 和 GeneralSimplex 的关系

这里有两种实现：

### 方案 A：返回 theory lemma

```text
¬lit_original ∨ x <= 5
```

优点：

```text
SAT/CDCL 能学习；
proof 清晰；
和其他 theory 一致。
```

### 方案 B：直接 assert bound 到 simplex

不推荐单独这么做，因为这个 bound 来自整数推理，不是原始 literal。如果不加入 SAT lemma，回溯、解释和 proof 都会混乱。

推荐：

```text
所有整数推导都通过 theory lemma 加回 SAT solver；
下一轮 SAT assignment 再 assert 到 simplex。
```

# 8. Cube Test

cube test 的用途：

```text
当 LRA relaxation 有 rational model，但不是 integral；
尝试直接证明 polyhedron 内存在 integer point。
```

它是：

```text
sound but incomplete
```

成功可以返回 SAT，失败只能返回 UNKNOWN，继续 cuts/branch。

## 8.1 Unit cube test

当前 active linear constraints：

```text
A x <= b
x ∈ Z^n
```

如果存在中心 `z`，使得边长 1 的 axis-parallel cube：

```text
C_1(z) = { y | |y_i - z_i| <= 1/2 }
```

完全包含于 polyhedron，则 `round(z)` 是整数解。

约束转化：

对每一行：

```text
a_i · x <= b_i
```

要求 cube 完全在里面：

```text
a_i · z + 1/2 * ||a_i||_1 <= b_i
```

等价于：

```text
a_i · z <= b_i - 1/2 * ||a_i||_1
```

所以 unit cube test 就是构造新 LRA 系统：

```text
A z <= b'
b'_i = b_i - 1/2 * ||a_i||_1
```

然后跑 simplex。

伪代码：

```cpp
CubeResult CubeTester::unitCubeTest(const LinearSystem& sys) {
    LinearSystem shifted;

    for (const Row& row : sys.rows) {
        mpq_class norm = 0;
        for (auto [x, a] : row.coeffs) {
            if (isIntegerVar(x)) {
                norm += abs(a);
            }
        }

        Row r = row;
        r.rhs = row.rhs - norm / 2;
        shifted.add(r);
    }

    auto r = internalSimplex.solve(shifted);

    if (r.unsat()) {
        return CubeResult::unknown();
    }

    Model center = r.model();
    Model candidate = center;

    for (VarId x : integerVars_) {
        candidate[x] = roundNearest(center[x]);
    }

    if (validator.validate(activeAtoms_, candidate)) {
        return CubeResult::sat(candidate);
    }

    // 理论上不该失败；失败说明转换或 round 有 bug
    return CubeResult::internalError();
}
```

## 8.2 Mixed LIRA cube test

如果有 integer vars 和 rational vars：

```text
x ∈ Z^n
y ∈ Q^m
```

只对 integer variables 方向缩小 bound：

```text
Sx + Ry <= b
```

转成：

```text
S z_x + R z_y <= b_i - 1/2 * ||S_i||_1
```

也就是只计算 integer 部分系数的 L1 norm。

## 8.3 什么时候跑 cube test

建议条件：

```text
1. integer vars 数量不太小；
2. 当前 rational model fractional vars 较多；
3. 约束不是大量 equality；
4. Boolean 0/1 变量不多；
5. 最近 cube test 没有连续失败太多次。
```

伪代码：

```cpp
bool shouldRunCubeTest(const Model& beta) {
    if (stats_.cubeFailuresInRow > 5) return false;
    if (numEqualities(activeSystem_) > numConstraints(activeSystem_) / 2) return false;
    if (numBinaryLikeVars() > integerVars_.size() / 2) return false;
    return true;
}
```

# 9. Cuts

cuts 的目标：

```text
删除当前 fractional rational model，
但保留所有 integer solutions。
```

每个 cut 都要满足：

```text
active_constraints ⇒ cut
```

所以 SMT lemma 应该是：

```text
¬premise_1 ∨ ... ∨ ¬premise_k ∨ cut
```

如果 cut 从当前 active constraints 和 tableau row 推出，premises 至少包括当前相关 bound 的 reason literals。

---

## 9.1 GCD cut

刚才的 gcd tightening 也可以看作 cut。

形式：

```text
a·x <= c
g = gcd(a_i)
```

推出：

```text
a·x <= g * floor(c/g)
```

lemma：

```text
¬lit(a·x <= c) ∨ lit(a·x <= g*floor(c/g))
```

---

## 9.2 Gomory fractional cut

从 simplex final tableau 中选一行：

```text
x_B = b + a1*y1 + ... + ak*yk
```

假设：

```text
x_B, y1,...,yk 都是 integer variables
y_j >= 0
b 不是整数
```

当前 simplex model 通常让 non-basic variables 在 bound 上，导致：

```text
x_B = b
```

若 `b` fractional，则生成 Gomory cut：

```text
Σ frac(a_j) * y_j >= frac(b)
```

它切掉当前 model，但所有整数解满足。

## 9.3 变量非负化

Gomory cut 常要求：

```text
y_j >= 0
```

如果变量有 lower bound：

```text
x_j >= l_j
```

可替换：

```text
y_j = x_j - l_j
```

如果变量在 upper bound：

```text
x_j <= u_j
```

可用：

```text
y_j = u_j - x_j
```

实际实现中，从当前 simplex 状态看 non-basic variable 在 lower 还是 upper：

```text
if x_j at lower:
    y_j = x_j - lower_j

if x_j at upper:
    y_j = upper_j - x_j
```

这样 `y_j >= 0`。

---

## 9.4 Pure Gomory cut 伪代码

第一版可以先实现 pure integer + lower-bound 版本：

```cpp
optional<Cut> CutGenerator::generatePureGomory(
    const SimplexState& st,
    const Model& beta
) {
    for (const TableauRow& row : st.rows()) {
        VarId xB = row.basicVar();

        if (!isIntegerVar(xB)) continue;
        if (isIntegral(beta[xB])) continue;

        // 只处理所有 non-basic 都是 integer 且有 lower bound 的简单情况
        bool ok = true;
        for (auto [y, a] : row.terms()) {
            if (!isIntegerVar(y)) {
                ok = false;
                break;
            }
            if (!st.lower(y).isFinite()) {
                ok = false;
                break;
            }
        }
        if (!ok) continue;

        // 将 y 替换成 y' = y - lower(y)
        // row: xB = b' + Σ a_j y'_j
        Rational bPrime = row.constantAfterLowerShift(st);

        Rational f0 = frac(bPrime);
        if (f0 == 0) continue;

        LinearExpr cutLhs;

        for (auto [y, a] : row.terms()) {
            Rational fj = frac(a);
            if (fj != 0) {
                cutLhs += fj * (y - st.lower(y).value);
            }
        }

        // cut: Σ frac(a_j)*(y_j - lower_j) >= frac(b')
        Cut cut;
        cut.expr = cutLhs;
        cut.rel = GEQ;
        cut.rhs = f0;
        cut.premises = collectBoundReasons(row, st);

        return cut;
    }

    return none;
}
```

需要注意：

```text
cut 中出现 rational coefficients，需要乘 lcm 转成 integer coefficients。
```

例如：

```text
1/2 y1 + 1/3 y2 >= 2/3
```

乘 6：

```text
3y1 + 2y2 >= 4
```

---

## 9.5 Bound-aware Gomory cut

更通用情况：

```text
non-basic var 有的在 lower bound，有的在 upper bound。
```

定义：

```text
if y at lower l:
    z = y - l >= 0

if y at upper u:
    z = u - y >= 0
```

把 tableau row 全部转成非负变量 `z` 后，再生成 fractional cut。

这个版本比 pure Gomory 更适合 SMT，因为 simplex 中 non-basic vars 经常在 upper bound。

---

## 9.6 MIR cut

Mixed Integer Rounding cut 用于 mixed LIRA：

```text
integer variables + rational variables
```

第一版 QF_LIA 可以不做 MIR。后续扩展到 mixed LIRA 再做。

阶段建议：

```text
V1: no cuts, only branch split
V2: gcd cuts / bound tightening
V3: pure Gomory
V4: bound-aware Gomory
V5: MIR cuts
```

# 10. Branch-and-bound

当：

```text
LRA SAT
model fractional
cube test failed
no useful cut generated
```

就 branch。

选一个 fractional integer var：

```text
β(x) = q, q ∉ Z
```

生成 split：

```text
x <= floor(q) ∨ x >= ceil(q)
```

这是整数理论永真式。

## 10.1 branch lemma

```cpp
TheoryLemma BranchSelector::makeSplit(const Model& beta) {
    VarId x = chooseFractionalVar(beta);
    mpq_class q = beta[x];

    mpz_class lo = floor(q);
    mpz_class hi = ceil(q);

    Atom left  = mkLeq(x, lo);
    Atom right = mkGeq(x, hi);

    return clause({left, right});
}
```

对应 SMT lemma：

```text
(x <= floor(q)) ∨ (x >= ceil(q))
```

它不需要 active premises，因为对整数变量永真。

---

## 10.2 branch variable 选择

策略：

```text
1. largest fractional distance:
   选 frac 最接近 1/2 的变量。

2. smallest domain:
   若变量有上下界，选 domain 最小的。

3. most active:
   出现在最多 active constraints 中。

4. conflict activity:
   类似 VSIDS，频繁出现在 LIA conflict/cut 中的变量优先。

5. pseudo-cost:
   记录对某变量 branch 后，左右分支分别带来的冲突/传播收益。
```

伪代码：

```cpp
VarId BranchSelector::chooseFractionalVar(const Model& beta) {
    VarId best = INVALID;
    Score bestScore = -inf;

    for (VarId x : integerVars_) {
        if (isIntegral(beta[x])) continue;

        Score s;
        s.fracDistance = distanceToNearestInteger(beta[x]);
        s.activity = activity_[x];
        s.domainScore = domainScore(x);
        s.total = 10*s.fracDistance + 2*s.activity + s.domainScore;

        if (s.total > bestScore) {
            best = x;
            bestScore = s.total;
        }
    }

    return best;
}
```

## 10.3 branch direction

SAT solver 会自己选 branch literal。但你可以给 phase hint：

```text
如果当前 β(x)=7.3：
    left:  x <= 7
    right: x >= 8
```

可以优先选择更可能保留 feasibility 的方向。

简单策略：

```text
如果 lower side 更宽，先 x <= floor(q)
如果 upper side 更宽，先 x >= ceil(q)
```

# 11. 完整性问题

必须诚实区分：

```text
soundness:
  不返回错误 sat/unsat。

completeness:
  一定能在有限时间内返回 sat/unsat。
```

对于第一版 SMT solver：

```text
simplex + model validation + branch split
```

是 sound 的，但在无界或复杂 LIA 上，可能搜索很差。

想要更接近 complete，需要：

```text
1. branch-and-cut with fair branching;
2. Gomory/MIR cuts;
3. divisibility reasoning;
4. resource fair scheduling;
5. 或者最后接 Cooper/Omega-style complete fallback。
```

我建议你设计两个模式：

```text
mode = practical:
    simplex + cube + cuts + branch
    resource exhausted 可返回 unknown

mode = complete-small:
    对小规模问题调用 Cooper/Omega elimination fallback
    牺牲性能换 completeness
```

---

# 12. Cooper / Omega fallback

作为严谨求解器，建议后续实现一个小规模 fallback。

## 12.1 目标

当 branch-and-cut 不收敛时，对变量数量较小的问题：

```text
exists x1...xn. A x <= b
```

用 Presburger elimination 思路消去变量。

## 12.2 简化版 Omega test

核心操作：

```text
1. 处理等式：
   a1x1 + ... + anxn = c
   用 gcd 检查和变量替换减少维度。

2. 对目标变量 x 收集上下界：
   lower_i <= x
   x <= upper_j

3. 通过 Fourier-Motzkin 风格组合上下界：
   lower_i <= upper_j

4. 对整数情况加 rounding / divisibility 修正。

5. 消去 x，得到低一维系统。
```

这个 fallback 不一定第一版做，但接口要留：

```cpp
class CompleteLiaFallback {
public:
    FallbackResult solve(const LinearSystem& sys,
                         const IntegerVarSet& intVars,
                         ResourceLimit limit);
};
```

主流程：

```cpp
if (options_.completeFallback && smallEnough(sys)) {
    auto r = cooperOmega_.solve(sys, integerVars_);
    if (r.sat()) return sat(r.model());
    if (r.unsat()) return conflictOrGlobalUnsat(r.certificate());
}
```

# 13. 与 CDCL(T) 的 lemma 交互

这是实现中最容易出错的地方。

## 13.1 LRA conflict

如果 simplex 发现 active bounds 冲突：

```text
lit1, lit2, ..., litk 不可同时成立
```

返回 conflict clause：

```text
¬lit1 ∨ ¬lit2 ∨ ... ∨ ¬litk
```

---

## 13.2 GCD conflict

如果 active equality：

```text
lit_eq: 6x + 10y = 7
```

不可能成立，返回：

```text
¬lit_eq
```

---

## 13.3 Tightening lemma

如果：

```text
lit: 6x + 10y <= 23
```

推出：

```text
6x + 10y <= 22
```

返回：

```text
¬lit ∨ lit_tightened
```

---

## 13.4 Cut lemma

如果 cut 从一组 active literals `P` 推出：

```text
P = {lit1, lit2, lit3}
P ⇒ cut
```

返回：

```text
¬lit1 ∨ ¬lit2 ∨ ¬lit3 ∨ lit_cut
```

不要直接加：

```text
lit_cut
```

除非它是全局 assertion 的后果。

---

## 13.5 Branch split lemma

整数 split：

```text
x <= k ∨ x >= k+1
```

是 theory-valid，不需要 premises：

```text
lit(x <= k) ∨ lit(x >= k+1)
```

---

## 13.6 Disequality split lemma

如果当前有：

```text
lit_neq: p != 0
```

但 model 里 `p = 0`，返回：

```text
¬lit_neq ∨ lit(p < 0) ∨ lit(p > 0)
```

用 δ 表示：

```text
¬lit_neq ∨ lit(p <= -δ) ∨ lit(p >= δ)
```

# 14. Scope / trail 设计

LIA solver 里至少有三种状态：

```text
1. registered constraints:
   tableau rows, aux vars, atom mapping
   随公式作用域变化。

2. active bounds:
   当前 SAT model 下的 bound assertions
   每次 theory check 可重建。

3. learned lemmas:
   加回 SAT solver，由 SAT solver 管理。
```

如果你现在 V1 是 semi-incremental，可以这样：

```cpp
void LiaSolver::beginCheckForSatModel() {
    simplex_.resetActiveBounds();
    diseqObligations_.clear();
    activeConstraintReasons_.clear();
}

void LiaSolver::assertLit(SatLit lit, LiaAtom atom, bool value) {
    auto assertions = translateLitToBoundsOrDiseq(lit, atom, value);

    for (auto& b : assertions.bounds) {
        simplex_.assertBound(b);
    }

    for (auto& d : assertions.diseqs) {
        diseqObligations_.push_back(d);
    }
}
```

注意：

```text
resetActiveBounds 不能清 tableau；
不能清 formToAux；
不能清 atomInfo；
只能清当前 active lower/upper 和 bound reason。
```

# 15. Proof / certificate 记录

你说这是“正经求解”，所以 proof object 要从第一版设计。

## 15.1 lemma reason 类型

```cpp
enum class LiaLemmaKind {
    LraConflict,
    DisequalitySplit,
    IntegerBranchSplit,
    GcdEqualityConflict,
    GcdInequalityTightening,
    SingleVarBoundTightening,
    GomoryCut,
    MirCut,
    CubeModelValidation
};

struct LiaProofHint {
    LiaLemmaKind kind;
    std::vector<SatLit> premises;
    std::vector<RowId> tableauRows;
    std::vector<BoundInfo> bounds;
    std::vector<VarId> integerVars;
};
```

## 15.2 sat result

SAT 不需要 proof，但需要 model certificate：

```text
每个 integer var 的值是整数；
每个 active atom exact evaluation 为 true；
每个 disequality exact evaluation 非零。
```

## 15.3 unsat result

UNSAT 来自 SAT solver 最终 UNSAT。每个 theory lemma 要有 proof hint。




这是最小可用 LIA。

# 18. 最终推荐的 LIA check 主循环

```cpp
TheoryCheckResult LiaSolver::check(TheoryCheckLevel level) {
    // LRA relaxation
    auto lra = simplex_.check();
    if (lra.unsat()) {
        return conflict(buildLraConflict());
    }

    Model beta = simplex_.model();

    // Disequality obligations
    if (auto lemma = checkDisequalityObligations(beta)) {
        return lemma(*lemma);
    }

    // Integral model
    if (allIntegerVarsIntegral(beta)) {
        if (validator_.validate(activeAtoms_, beta)) {
            return sat(beta);
        }
        return internalError("Invalid integral model");
    }

    // Cheap integer conflicts / lemmas
    if (auto r = divisibilityReasoner_.run(activeConstraints_)) {
        return *r;
    }

    if (auto lemma = boundTightener_.run(activeConstraints_, beta)) {
        return lemma(*lemma);
    }

    // Cube test
    if (level >= FULL_EFFORT && shouldRunCubeTest(beta)) {
        auto cube = cubeTester_.unitCubeTest(activeLinearSystem_);
        if (cube.sat()) {
            if (validator_.validate(activeAtoms_, cube.model())) {
                return sat(cube.model());
            }
            return internalError("Invalid cube model");
        }
    }

    // Cuts
    int cuts = 0;
    while (cuts < options_.maxCutsPerCheck) {
        auto cut = cutGenerator_.generate(simplex_.state(), beta);
        if (!cut) break;

        if (!isUsefulCut(*cut, beta)) break;

        TheoryLemma lemma = lemmaBuilder_.buildCutLemma(*cut);
        return TheoryCheckResult::lemma(lemma);
    }

    // Branch
    BranchSplit split = branchSelector_.makeSplit(beta, integerVars_);
    return lemma(lemmaBuilder_.buildBranchLemma(split));
}
```

# 19. 一句话总结

你这个 LIA solver 应该这样设计：

```text
GeneralSimplex 负责 LRA relaxation；
IntegerModelChecker 负责确认 rational model 是否已经是整数模型；
DivisibilityReasoner 和 BoundTightener 负责 cheap integer 推理；
CubeTester 负责快速找整数点；
CutGenerator 负责不分支地排除 fractional model；
BranchSelector 负责最后的整数 split；
LemmaBuilder 确保所有推导都以 guarded theory lemma 的形式返回给 CDCL；
ModelValidator 保证 sat 结果绝对正确。
```

第一版最小闭环：

```text
Simplex + integral check + disequality split + branch split + model validation
```

第二版实用闭环：

```text
Simplex + gcd reasoning + cube test + Gomory cuts + branch-and-bound
```

第三版严谨增强：

```text
branch-and-cut + proof hints + Cooper/Omega fallback + complete-small mode
```
