不建议这样做：

```text
如果两个分支都 Unsat，生成一个包含 disequality 的冲突
```

除非你能**严格合并两个分支的 conflict reason**。否则很容易生成一个不是 theory-valid 的 clause。为了少改接口而把 split lemma 伪装成 conflict，这个代价太大。

我建议：**现在就小改接口，支持 lemma 返回。这个改动其实不大，而且以后 cuts、branch-and-bound、Gomory cut、NIA split 都要用。**

# 1. 你的当前折中方案哪里危险

你说：

```text
保留内部分支尝试；
如果找到分支 Sat，使用正确 model；
如果两个分支都 Unsat，生成一个包含 disequality 的冲突。
```

前两条可以，第三条危险。

比如当前 assignment 里有：

```text
x = 0
x != 0
```

`x != 0` 被违反。你试：

```text
x < 0  → unsat
x > 0  → unsat
```

确实可以推出当前 assignment 不一致。但 conflict clause 不能只是：

```text
¬(x != 0)
```

因为 `x != 0` 本身并不矛盾。正确 conflict 至少应该是：

```text
¬(x = 0) ∨ ¬(x != 0)
```

更一般地，如果左右两个分支的 unsat reason 分别是：

```text
P_L ∧ (s < 0) unsat
P_R ∧ (s > 0) unsat
```

再加上 split：

```text
d ⇒ (s < 0 ∨ s > 0)
```

才能合成：

```text
¬d ∨ ¬P_L ∨ ¬P_R
```

也就是说，**两个分支都 unsat 之后的 conflict 不是“随便包含 disequality”就行，而是要 resolution 合并两个分支的 reason**。V1 不建议做这个。

# 2. 最正确的 V1：加 lemma 返回路径

这不是大改，是以后必须要有的接口。LIA 的这些东西全都需要返回 lemma：

```text
disequality split:
  ¬(p != 0) ∨ p < 0 ∨ p > 0

integer branch:
  x <= floor(v) ∨ x >= ceil(v)

GCD tightening:
  ¬lit(a·x <= c) ∨ lit(a·x <= c')

Gomory cut:
  ¬premise_1 ∨ ... ∨ ¬premise_k ∨ cut
```

所以现在改接口是对的。

建议定义：

```cpp
struct TheoryLemma {
    std::vector<SatLit> lits;
};

struct TheoryCheckResult {
    enum class Kind {
        Consistent,
        Conflict,
        Lemma,
        Unknown
    };

    Kind kind;
    std::optional<TheoryConflict> conflict;
    std::optional<TheoryLemma> lemma;

    static TheoryCheckResult consistent();
    static TheoryCheckResult conflict(TheoryConflict c);
    static TheoryCheckResult lemma(TheoryLemma l);
    static TheoryCheckResult unknown();
};
```

`TheorySolver` 接口改成：

```cpp
class TheorySolver {
public:
    virtual TheoryCheckResult check() = 0;
};
```

`TheoryManager::check` 聚合多个 theory：

```cpp
TheoryCheckResult TheoryManager::check(...) {
    for (auto& solver : solvers_) {
        auto r = solver->check();

        if (r.kind == TheoryCheckResult::Kind::Conflict) {
            return r;
        }

        if (r.kind == TheoryCheckResult::Kind::Lemma) {
            return r;
        }

        if (r.kind == TheoryCheckResult::Kind::Unknown) {
            return r;
        }
    }

    return TheoryCheckResult::consistent();
}
```

`Solver::checkSatInternal()` 里：

```cpp
while (true) {
    auto sr = sat->solve();

    if (sr == SatResult::Unsat) {
        return Result::Unsat;
    }

    theoryManager.beginCheckForSatModel();

    for (auto& atom : atomizer.atoms()) {
        bool value = sat->value(atom.satVar);
        theoryManager.assertLit(atom, value);
    }

    auto tr = theoryManager.check();

    if (tr.kind == TheoryCheckResult::Kind::Consistent) {
        return Result::Sat;
    }

    if (tr.kind == TheoryCheckResult::Kind::Conflict) {
        sat->addClause(tr.conflict->lits);
        continue;
    }

    if (tr.kind == TheoryCheckResult::Kind::Lemma) {
        sat->addClause(tr.lemma->lits);
        continue;
    }

    if (tr.kind == TheoryCheckResult::Kind::Unknown) {
        return Result::Unknown;
    }
}
```

如果你不想一次性改所有 solver，可以先保留旧接口，再加默认包装：

```cpp
virtual std::optional<TheoryLemma> lastLemma() const { return std::nullopt; }
```

但我更推荐直接改成 `TheoryCheckResult`。

# 3. disequality 的标准实现

`assertLit` 里不要分支，只记录 obligation：

```cpp
void SimplexSolver::assertEffectiveRelation(
    int aux,
    Relation rel,
    SatLit reason
) {
    switch (rel) {
        case Relation::LEQ:
            gs_.assertUpper(aux, BoundInfo::finite(delta(0, 0), reason));
            break;

        case Relation::LT:
            gs_.assertUpper(aux, BoundInfo::finite(delta(0, -1), reason));
            break;

        case Relation::GEQ:
            gs_.assertLower(aux, BoundInfo::finite(delta(0, 0), reason));
            break;

        case Relation::GT:
            gs_.assertLower(aux, BoundInfo::finite(delta(0, 1), reason));
            break;

        case Relation::EQ:
            gs_.assertLower(aux, BoundInfo::finite(delta(0, 0), reason));
            gs_.assertUpper(aux, BoundInfo::finite(delta(0, 0), reason));
            break;

        case Relation::NEQ:
            disequalities_.push_back({aux, reason});
            break;
    }
}
```

`check()`：

```cpp
TheoryCheckResult SimplexSolver::check() {
    auto r = gs_.check();

    if (r == GeneralSimplex::Result::Unsat) {
        return TheoryCheckResult::conflict(translateConflict());
    }

    auto model = gs_.model();

    for (const auto& d : disequalities_) {
        auto v = model.value(d.auxVar);

        if (v.isZero()) {
            return TheoryCheckResult::lemma(buildDiseqSplitLemma(d));
        }
    }

    currentModel_ = model;
    return TheoryCheckResult::consistent();
}
```

# 4. LRA disequality lemma

如果当前是 LRA：

```text
p(x) != 0
```

split lemma 是：

```text
¬lit(p != 0) ∨ lit(p < 0) ∨ lit(p > 0)
```

内部可以写成：

```text
¬lit(p != 0) ∨ lit(p <= -δ) ∨ lit(p >= δ)
```

代码：

```cpp
TheoryLemma SimplexSolver::buildDiseqSplitLemma(const DiseqInfo& d) {
    SatLit lt = getOrCreateBoundAtom(d.auxVar, Relation::LT, 0);
    SatLit gt = getOrCreateBoundAtom(d.auxVar, Relation::GT, 0);

    return TheoryLemma{
        .lits = {
            negate(d.lit),
            lt,
            gt
        }
    };
}
```

关键是：`lt` 和 `gt` 必须是 SAT solver 能认识的新 theory literal。也就是说，`getOrCreateBoundAtom` 要：

```text
1. 创建/复用 theory atom；
2. 创建/复用 SAT var；
3. 注册到 Atomizer / TheoryManager 映射；
4. 之后 SAT model 里如果该 literal 为 true，TheoryManager 能把它 assert 回 SimplexSolver。
```

# 5. LIA 的更强 disequality lemma

如果是 LIA，`p(x)` 是整数线性表达式：

```text
p(x) = a1*x1 + ... + an*xn + c
```

那么不需要 δ。更强的 split 是：

```text
p(x) <= -g ∨ p(x) >= g
```

其中：

```text
g = gcd(|a1|, ..., |an|)
```

但要先判断 `p(x)=0` 是否可能。

若：

```text
g ∤ -c
```

那么 `p(x)=0` 本来就不可能，`p(x)!=0` 恒真，不需要 split。

若：

```text
g | -c
```

则返回：

```text
¬lit(p != 0) ∨ lit(p <= -g) ∨ lit(p >= g)
```

代码逻辑：

```cpp
TheoryLemma SimplexSolver::buildLiaDiseqSplitLemma(const DiseqInfo& d) {
    LinearForm f = formOfAux(d.auxVar);

    mpz_class g = gcdAbs(f.coeffs);

    if (g == 0) {
        // constant != 0
        // 若 constant != 0，则 diseq 恒真；
        // 若 constant == 0，则 diseq 恒假，应返回 conflict: ¬d
    }

    if ((-f.constant) % g != 0) {
        // p(x)=0 impossible, disequality already satisfied.
        // 不应生成 lemma，直接 continue。
    }

    SatLit leNegG = getOrCreateLinearAtom(f, Relation::LEQ, -g);
    SatLit gePosG = getOrCreateLinearAtom(f, Relation::GEQ,  g);

    return TheoryLemma{
        .lits = {
            negate(d.lit),
            leNegG,
            gePosG
        }
    };
}
```

这比 LRA 的 `p < 0 ∨ p > 0` 更专业、更高效。

# 6. 如果暂时真的不想改 lemma 接口

最低限度也不要做：

```text
两个分支都 Unsat -> 返回随便构造的 conflict
```

可以这样：

```text
1. 内部尝试 s < 0。
2. 若 Sat，保存该分支 model，返回 Sat。
3. 否则尝试 s > 0。
4. 若 Sat，保存该分支 model，返回 Sat。
5. 若两个都 Unsat，返回 Unknown。
```

这个是 sound 的，但不完整，而且会损失很多性能。

如果你非要在两个分支都 unsat 时返回 conflict，至少要做一个保守版本：

```text
conflict = 当前所有 active theory literals 的 negation
```

也就是：

```text
¬lit1 ∨ ¬lit2 ∨ ... ∨ ¬litk
```

这在当前 active assignment 确实整体不可满足时是 sound 的，但非常粗，学习质量很差。并且你必须确保“两边 unsat”是在**同一组 active literals + disequality split**下得到的。

所以更推荐：**不要走这条路，直接加 lemma。**

# 8. 我的最终建议

这里不要妥协。**现在就加 lemma 返回路径。**

原因很简单：

```text
disequality split 要 lemma；
LIA branch-and-bound 要 lemma；
Gomory cut 要 lemma；
gcd tightening 要 lemma；
NIA split 也要 lemma。
```

你现在不加，后面每一步都会被卡住。

当前最小专业方案是：

```text
GeneralSimplex 仍只处理 conjunction of bounds；
SimplexSolver wrapper 处理 disequality obligation；
violated disequality 返回 guarded split lemma；
SAT solver 加 clause 后重新搜索。
```

这比内部临时分支更标准、更容易 debug、更适合 proof，也更符合你要做“正经 SMT 求解器”的目标。
