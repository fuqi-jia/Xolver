# Plan: Trail-Based CDCL(T) Refactor (Final Revision)

## Goal
Replace the current scan-all-atoms CDCL(T) loop with a unified, trail-based theory integration using CaDiCaL's native `ExternalPropagator` (IPASIR-UP). All theories — LRA, LIA, NRA — use exactly one integration path. No legacy scan-based path remains.

## Architecture Overview

```
CaDiCaL Solver
  -> ExternalPropagator (our CadicalTheoryPropagator)
     -> notifies observed theory variable assignments
     -> notifies backtracks
     -> asks theory to check complete models
     -> asks theory to propagate / add clauses
  -> TheoryAtomRegistry
     -> SatVar -> TheoryAtomRecord (O(1))
     -> canonical dynamic atom creation
  -> TheoryManager
     -> routes assignments to correct TheorySolver
     -> coordinates backtracking
     -> collects check results
  -> TheorySolver (abstract)
     -> LraSolver, LiaSolver, CdcacSolver
```

## Critical Design Decisions

1. **One integration path only.** No `useLegacy`, no `runLegacyCdclLoop`, no `Atomizer::atoms()` scan in the CDCL loop.
2. **CaDiCaL ExternalPropagator is the production backend.** The vendored CaDiCaL already exposes `connect_external_propagator`, `add_observed_var`, `notify_assignment`, `notify_backtrack`, `cb_check_found_model`, `cb_has_external_clause` / `cb_add_external_clause_lit`. We use these directly.
3. **All theory atoms in `TheoryAtomRegistry`.** Parsed atoms are registered during `Atomizer::atomize()`. Dynamic atoms are registered via `getOrCreate...`. `TheoryBridge` looks up **only** by `SatVar`.
4. **TheoryAtomRecord uses variant payload** for linear, polynomial, and future theory payloads.
5. **TheoryLemmaDatabase lives in `Solver::checkSatInternal`**, survives the entire check-sat call. Theory solvers query `contains()`; integration layer calls `insertIfNew()` only before adding a clause to CaDiCaL.
6. **Unknown logic returns `Unknown`.** No fallback registering multiple solvers.
7. **CdcacSolver implements the new interface.** If not fully migrated, `QF_NRA` returns `Unknown` during this refactor.

### 1.3 Assignment routing
`notify_assignment` receives signed literals for observed vars. Convert to `SatLit` and route:
```cpp
for (int lit : lits) {
    SatVar var = std::abs(lit);
    bool sign = lit > 0;
    const auto* atom = registry_.findBySatVar(var);
    if (!atom) continue;
    tm_.assertTheoryLit(*atom, SatLit{var, sign}, currentLevel_);
}
```

`notify_backtrack(new_level)`:
```cpp
tm_.backtrackToLevel(static_cast<int>(new_level));
```

`cb_check_found_model`:
```cpp
auto tr = tm_.check(lemmaDb_);
if (tr.kind == Consistent) return true;

if (tr.kind == Conflict) {
    pendingClause_ = tr.conflict.clause;
    hasPendingClause_ = true;
    return false;
}

if (tr.kind == Lemma) {
    if (lemmaDb_.insertIfNew(*tr.lemma)) {
        pendingClause_ = tr.lemma->lits;
        hasPendingClause_ = true;
    }
    return false;
}

return false; // Unknown
```

`cb_has_external_clause` / `cb_add_external_clause_lit`:
```cpp
int cb_has_external_clause() override {
    return hasPendingClause_ ? 1 : 0;
}

int cb_add_external_clause_lit() override {
    if (!hasPendingClause_ || pendingClause_.empty()) {
        hasPendingClause_ = false;
        return 0;
    }
    SatLit lit = pendingClause_.back();
    pendingClause_.pop_back();
    return lit.sign ? static_cast<int>(lit.var) : -static_cast<int>(lit.var);
}
```

## Phase 3: `TheorySolver` New Interface

```cpp
class TheorySolver {
public:
    virtual ~TheorySolver() = default;
    virtual TheoryId id() const = 0;

    // Scope management (for Solver::push/pop API, not CDCL loop)
    virtual void push() = 0;
    virtual void pop(uint32_t n) = 0;

    // Incremental assertion from SAT trail
    virtual void assertLit(
        const TheoryAtomRecord& atom,
        bool value,
        int level,
        SatLit reason
    ) = 0;

    // Backtrack to target decision level
    virtual void backtrackToLevel(int level) = 0;

    // Check current incremental state
    virtual TheoryCheckResult check(TheoryLemmaDatabase& lemmaDb) = 0;

    // Reset ONCE per fresh check-sat initialization
    virtual void reset() = 0;
};
```

## Phase 4: `TheoryLemmaDatabase`

```cpp
class TheoryLemmaDatabase {

    bool contains(const TheoryLemma& lemma) const;

```

**Ownership rules:**
- Theory solvers call `contains()` to skip duplicate candidates **before** returning.
- Integration layer (`CadicalTheoryPropagator`) calls `insertIfNew()` **only** when about to add the clause to CaDiCaL.
- If `insertIfNew` returns false (already known), the integration layer treats this as an error state (return `Unknown`) rather than looping.

## Phase 7: `LiaSolver` Incremental Trail

### 7.1 Trail entry
```cpp

    size_t activeAtomsSize;
    size_t disequalitiesSize;
};

std::vector<LiaTrailEntry> trail_;
```

### 7.2 `assertLit`
Store sizes **before** modification:
```cpp
void LiaSolver::assertLit(
    const TheoryAtomRecord& atom,
    bool value,
    int level,
    SatLit reason
) {
    ensureLevel(level);

    size_t oldActive = activeAtoms_.size();
    size_t oldDiseq  = disequalities_.size();

    const auto& payload = std::get<LinearAtomPayload>(atom.payload);
    int aux = manager_.getOrCreateAuxVar(gs_, payload.lhs, payload.rhs);

    if (isDisequality(payload.rel, value)) {
        disequalities_.push_back({aux, payload.lhs, payload.rhs, reason});
        trail_.push_back({level, oldActive, oldDiseq});
        return;
    }

    bool immediateConflict = manager_.assertBound(
        gs_, aux, payload.rel, value, reason, level);

    if (immediateConflict) {
        pendingConflict_ = PendingConflict{level, manager_.buildImmediateConflict(gs_)};
        return;
    }

    activeAtoms_.push_back({
        atom.exprId, aux, payload.rel, value,
        payload.lhs, payload.rhs, reason
    });
    trail_.push_back({level, oldActive, oldDiseq});
}
```

### 7.3 `backtrackToLevel`
```cpp
void LiaSolver::backtrackToLevel(int level) {
    while (!trail_.empty() && trail_.back().level > level) {
        activeAtoms_.resize(trail_.back().activeAtomsSize);
        disequalities_.resize(trail_.back().disequalitiesSize);
        trail_.pop_back();
    }

    if (pendingConflict_ && pendingConflict_->level > level) {
        pendingConflict_.reset();
    }

    gs_.backtrackToLevel(level);
}
```

### 7.4 `check(TheoryLemmaDatabase&)`
```cpp
TheoryCheckResult LiaSolver::check(TheoryLemmaDatabase& lemmaDb) {
    if (pendingConflict_) return mkConflict(pendingConflict_->conflict);

    auto r = gs_.check();
    if (r == Unsat) return mkConflict(manager_.translateConflict(gs_));
    if (r == Unknown) return unknown();

    auto dr = handleDisequalities(lemmaDb);
    if (dr.kind != Consistent) return dr;

    auto ir = checkIntegrality();
    if (ir.kind == Consistent) {
        if (validator_.validateLiaModel(...)) return consistent();
        return unknown();
    }

    // Fractional: cheap integer reasoning
    if (!activeAtoms_.empty()) {
        auto cr = integerReasoner_.run(activeAtoms_);
        if (cr) {
            if (cr->kind == Lemma && cr->lemmaOpt) {
                if (!lemmaDb.contains(*cr->lemmaOpt)) return *cr;
            } else {
                return *cr;
            }
        }
    }

    // Branch split
    if (ir.kind == Lemma && ir.lemmaOpt) {
        if (!lemmaDb.contains(*ir.lemmaOpt)) return ir;
    }

    return unknown();
}
```

### 7.5 `reset()`
```cpp
void LiaSolver::reset() {
    activeAtoms_.clear();
    disequalities_.clear();
    trail_.clear();
    pendingConflict_.reset();
    gs_.resetActiveBounds();
    manager_.resetBoundReasons();
}
```

Called **once** per fresh check-sat initialization.

## Phase 10: `Solver::checkSatInternal` Rewrite

```cpp
Result Solver::checkSatInternal() {
    if (!ir || ir->assertions().empty()) {
        return Result::Sat;
    }

    // Fresh SAT solver and registry per check-sat call
    auto sat = std::make_unique<CadicalBackend>();
    TheoryAtomRegistry registry;
    registry.setContext(sat.get(), nullptr); // Atomizer set later

    TheoryManager theoryManager;
    TheoryLemmaDatabase lemmaDb;

    // Register solvers by logic
    if (logic == "QF_LIA" || logic == "LIA") {
        auto lia = std::make_unique<LiaSolver>();
        lia->setRegistry(&registry);
        theoryManager.registerSolver(std::move(lia));
    } else if (logic == "QF_LRA" || logic == "LRA") {
        auto lra = std::make_unique<LraSolver>();
        lra->setRegistry(&registry);
        theoryManager.registerSolver(std::move(lra));
    } else if (logic == "QF_NRA" || logic == "NRA") {
        auto nra = std::make_unique<CdcacSolver>(createPolynomialKernel());
        theoryManager.registerSolver(std::move(nra));
    } else {
        return Result::Unknown;
    }

    // Atomizer registers parsed atoms into registry
    Atomizer atomizer(*sat);
    registry.setContext(sat.get(), &atomizer);

    for (ExprId a : ir->assertions()) {
        SatLit lit = atomizer.atomize(a, *ir);
        sat->addClause({lit});
    }

    // Mark all theory variables as observed
    for (const auto& rec : registry.records()) {
        sat->addObservedVar(rec.satVar);
    }

    // Connect theory propagator
    CadicalTheoryPropagator propagator(registry, theoryManager, lemmaDb);
    sat->connectPropagator(&propagator);

    auto result = sat->solve();

    if (result == SatSolver::SolveResult::Sat) return Result::Sat;
    if (result == SatSolver::SolveResult::Unsat) return Result::Unsat;
    return Result::Unknown;
}
```

**No `while (true)` loop.** CaDiCaL's `ExternalPropagator` handles the theory check internally during `solve()`. The propagator's `cb_check_found_model` is called when CaDiCaL finds a complete assignment. If the theory rejects it, the propagator adds a clause and CaDiCaL continues searching.

If the current CaDiCaL version requires the caller to manage the loop explicitly (some versions do), keep the loop but without scanning atoms:
```cpp
while (true) {
    auto result = sat->solve();
    if (result != Sat) break;

    auto tr = theoryManager.check(lemmaDb);
    if (tr.kind == Consistent) return Result::Sat;
    if (tr.kind == Conflict) { sat->addClause(tr.conflict.clause); continue; }
    if (tr.kind == Lemma) {
        if (lemmaDb.insertIfNew(*tr.lemma)) {
            sat->addClause(tr.lemma->lits);
            continue;
        }
        return Result::Unknown;
    }
    return Result::Unknown;
}
```

The exact loop structure depends on how `ExternalPropagator` integrates with `solve()`. Implement whichever matches the CaDiCaL version.

## Phase 11: No-Op Lemma Guards

All theory lemma generators must skip:

1. **Tautology**: `conclusion == premise` (produces `¬p ∨ p`).
2. **Self-tightening**: tightened atom is canonical-equivalent to original (same `satVar`).
3. **Duplicate**: `lemmaDb.contains(candidate)` is true.
4. **Already satisfied**: deferred until `SatAssignmentView` is available in theory solver.

Correct tautology guard:
```cpp
if (conclusionLit == premiseLit) return std::nullopt; // ¬p ∨ p
```

## Risks and Mitigations

| Risk | Mitigation |
|------|------------|
| CaDiCaL ExternalPropagator integration complexity | Start with `cb_check_found_model` only; add `cb_propagate` later |
| `CdcacSolver` migration breaks NRA tests | NRA returns `Unknown` until fully migrated; tests adjusted |
| Variant payload compilation overhead | `std::variant` is C++17 standard, no runtime cost for correct type |
| Registry lookup failure (SatVar not found) | Assert in debug, return `Unknown` in release |
| Duplicate lemma without new clause | `Solver` returns `Unknown`; theory solvers query `lemmaDb.contains()` |
