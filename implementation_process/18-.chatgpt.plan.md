我建议未来计划不要从最大的 MCSAT / Proof / OMT 直接开干，而是按这个顺序推进：

```text
先把 solver 变成稳定可用的产品内核；
再补强 NIA / DL / interval；
最后再做 MCSAT、Proof、OMT 这些上层大模块。
```

你现在的代码状态很清楚：Boolean、LRA、LIA、NRA、NIA-Core 已经有基础管道；但 MCSAT、Proof、OMT、Local Search、IncrementalLinearizer 还是 stage 级骨架，API 层也还有大量空实现。

---

# 我的未来路线建议

## Phase 0：当前 IDL/RDL 收尾

刚实现完 IDL/RDL，不要马上跳走。先把它变成稳定组件。

要做：

```text
1. 清理所有 debug 输出；
2. 固定 IDL/RDL regression；
3. 加 CLI 测试，不只跑 unit test；
4. 确认 unsupported atom 返回 Unknown，不死循环；
5. 确认 dynamic disequality split 不再因为重复 lemma 返回 Unknown；
6. getModel / getValue 未来过滤 "__ZERO__"。
```

这一步的目标是：

```text
QF_IDL / QF_RDL 可以作为可靠 fast path。
```

---

## Phase 1：API 层补全

这是我认为**最应该优先做**的事情。现在 API 空实现太多，包括：

```text
boolSort()
intSort()
realSort()
mkConst()
mkVar()
mkBool()
mkInt()
mkReal()
mkOp()
assertFormula()
getModel()
getValue()
```

这些如果不补，NLColver 很难作为真正的 solver library 使用。

目标：

```cpp
Solver s;
auto x = s.mkVar("x", s.intSort());
s.assertFormula(s.mkOp(Op::Leq, {x, s.mkInt(3)}));
auto r = s.checkSat();
auto m = s.getModel();
```

这一步完成后，你的 solver 才从“能跑 SMT2 文件”升级成：

```text
可嵌入、可测试、可供学习方法/上层工具调用的 solver API。
```

这对你后面的学习方法、MCSAT、OMT 都很关键。

---

## Phase 2：Model / Validator / getValue

API 做完后，马上补 model。

要统一：

```text
LRA model
LIA model
NRA model
NIA model
IDL model
RDL model
EUF future model
```

先实现：

```text
1. Model 对象；
2. getValue(term)；
3. 内部变量过滤，例如 "__ZERO__"；
4. FullModelValidator；
5. SAT 前强制 validate。
```

这是 correctness 生命线。

尤其 IDL/RDL/NIA 这种有内部变量、dynamic atom、split lemma 的理论，必须有 model validator。

---

## Phase 3：DL fast path 接入 LIA/LRA

现在已经有 IDL/RDL 了，下一步自然是做 fast path：

```text
QF_IDL -> IdlSolver
QF_RDL -> RdlSolver

QF_LIA:
  如果所有 atom 都是 difference logic -> IdlSolver
  否则 -> LiaSolver

QF_LRA:
  如果所有 atom 都是 difference logic -> RdlSolver
  否则 -> LraSolver
```

同时支持：

```text
AUTO:
  自动分类 IDL/RDL/LIA/LRA/NIA/NRA。
```

这样能显著提升简单线性差分约束的性能，也能让 solver 架构更清晰。

---

## Phase 4：NIA-Core 第二轮增强

NIA 是你的核心方向，应该继续补，但不要一上来搞大 MCSAT。

优先做这些：

```text
1. Factor lemma:
   xy = 0 -> x = 0 ∨ y = 0

2. Modular equality reasoning:
   x² + y² = 3 -> UNSAT by mod 4

3. BoundedNiaSolver 的大域策略：
   现在大范围 B&B 还是骨架，需要后续接 interval 框架。

4. NiaLocalSearch 从“少量候选”升级为真正 SAT finder。
```

注意原则：

```text
local search 只能找 SAT；
modular / gcd / finite enumeration 才能证明 UNSAT；
budget exhausted 一律 Unknown。
```

---

## Phase 5：公共 Interval / ICP 框架

你之前判断是对的：ICP 不应该私有塞进 NIA。

建议独立做：

```text
src/theory/arith/interval/
  IntervalZ / IntervalQ
  ReasonedBox
  IntervalEvaluator
  IntervalPruner
  ICP engine
```

然后：

```text
NIA 调用它；
NRA 调用它；
IncrementalLinearizer 调用它；
B&B 调用它。
```

这样以后：

```text
NIA Interval B&B
NRA ICP
McCormick linearization
local search domain pruning
```

都能复用。

---

## Phase 6：IncrementalLinearizer

等 interval 框架稳定后，再做 IncrementalLinearizer。

它应该负责：

```text
1. McCormick envelope；
2. product abstraction；
3. polynomial linear over-approx；
4. interval-dependent linear cuts；
5. dynamic lemma 生成。
```

例如：

```text
z = x*y
x ∈ [l_x,u_x]
y ∈ [l_y,u_y]
```

生成 McCormick inequalities。

但这一步依赖：

```text
DomainStore
Interval bounds
DynamicAtomRegistry
LIA/LRA cut handling
```

所以不应该太早做。

---

## Phase 7：LocalSearchAdvisor

Local Search 不要只作为 NIA 内部小模块，而应该变成 advisor：

```text
src/search/LocalSearchAdvisor.h
```

它给不同 theory 提供：

```text
1. candidate model；
2. variable activity；
3. branch suggestion；
4. restart suggestion；
5. cost landscape。
```

NIA local search 可以先做：

```text
Σ violation²
```

然后逐步加：

```text
tabu
random restart
large neighborhood search
gradient-like integer moves
learning-guided variable choice
```

这部分后面可以直接接你的学习方法。

---

## Phase 8：MCSAT

MCSAT 是大工程，不能现在立刻硬做完整。

我建议等下面这些稳定后再开：

```text
1. API 层稳定；
2. Model / getValue 稳定；
3. DynamicAtomRegistry 稳定；
4. IDL/RDL/LIA/LRA/NIA 基础 solver 稳定；
5. Interval framework 稳定；
6. Explanation / lemma pipeline 稳定。
```

然后 MCSAT 第一版只做：

```text
MCSAT Core + EUF + LRA
```

不要一上来 EUF/LIA/LRA/NIA/NRA 全支持。

路线：

```text
MCSAT-0:
  Trail + theory value + dynamic atom + lemma manager

MCSAT-1:
  EUF

MCSAT-2:
  LRA

MCSAT-3:
  LIA

MCSAT-4:
  NRA

MCSAT-5:
  NIA
```

这样才稳。
