## 0.2 修复 LinearNiaDomainReasoner 的整数除法

### 问题

GMP `mpz_class` 的 `/` 行为不是你需要的数学 floor/ceil。对整数域推理，必须显式实现：

```cpp
mpz_class floorDiv(mpz_class a, mpz_class b);
mpz_class ceilDiv(mpz_class a, mpz_class b);
```

要求：

```text
b > 0
floorDiv(a,b) = floor(a/b)
ceilDiv(a,b)  = ceil(a/b)
```

实现建议：

```cpp
mpz_class floorDiv(const mpz_class& a, const mpz_class& b) {
    assert(b > 0);
    mpz_class q = a / b;
    mpz_class r = a % b;
    if (r != 0 && a < 0) --q;
    return q;
}

mpz_class ceilDiv(const mpz_class& a, const mpz_class& b) {
    assert(b > 0);
    return -floorDiv(-a, b);
}
```

## 0.3 LinearNiaDomainReasoner 规则写死

输入 normalized constraint：

```text
a*x + c rel 0
```

只处理单变量一次线性形式。

### `a*x + c <= 0`

等价：

```text
a*x <= -c
```

若 `a > 0`：

```text
x <= floor((-c)/a)
```

若 `a < 0`：

```text
x >= ceil((-c)/a)
```

### `a*x + c >= 0`

等价：

```text
a*x >= -c
```

若 `a > 0`：

```text
x >= ceil((-c)/a)
```

若 `a < 0`：

```text
x <= floor((-c)/a)
```

### `a*x + c = 0`

```text
a*x = -c
```

若 `a | -c`：

```text
x = -c/a
```

否则：

```text
UNSAT conflict
```

### `a*x + c != 0`

若 `a | -c`：

```text
x != -c/a
```

否则：

```text
tautology, no domain update
```

## 0.4 修复 LibPolyKernel::variables()

当前不能只依赖 `main_variable`。目标是：

```text
variables(x*y)      -> {x,y}
variables(x + y^2)  -> {x,y}
variables(x^2)      -> {x}
variables(constant) -> {}
```

实现路径：

```text
优先：用 libpoly term iteration 收集每个 monomial 中出现的变量；
备选：在 PolynomialConverter 构造 PolyId 时同步记录变量集合；
禁止：只用 main_variable 判断是否 univariate。
```

建议新增缓存：

```cpp
std::unordered_map<PolyId, std::set<std::string>> polyVars_;
```

构造多项式时维护：

```cpp
vars(p+q) = vars(p) ∪ vars(q)
vars(p*q) = vars(p) ∪ vars(q)
vars(x)   = {x}
vars(c)   = {}
```

这样即使 libpoly term API 不好用，也能保证内部构造路径正确。

## 0.5 修复 UnivariateIntegerReasoner 的 root enumeration

### 正确原则

Rational Root Theorem 对整数多项式：

```text
a_n x^n + ... + a_0 = 0
```

若 `a_0 != 0`，所有整数根必须整除 `a_0`。

若 `a_0 == 0`，说明 `0` 是根，应先提取因子 `x^k`：

```text
p(x) = x^k * q(x)
```

然后：

```text
roots(p) = {0} ∪ roots(q)
```

不能因为 `a_0 == 0` 就只测试 `[-10,10]`，更不能标记 Complete。

### 算法

```cpp
IntegerRootResult solveUnivariateEq(poly p, var x) {
    coeffs = getIntegerCoefficients(p, x);
    if (!coeffs) return Incomplete;

    if (isZeroPolynomial(coeffs)) {
        return Complete{isZeroPolynomial=true};
    }

    std::set<mpz_class> roots;

    // coefficients from highest degree to constant
    while (constantTerm(coeffs) == 0) {
        roots.insert(0);
        divideByX(coeffs);
        if (isConstantNonZero(coeffs)) {
            return Complete{roots};
        }
    }

    mpz_class a0 = abs(constantTerm(coeffs));
    divisors = enumerateDivisors(a0);

    if (divisors.size() > maxDivisors) {
        return Incomplete{roots};  // 注意：有已知 root 0，但不能 complete
    }

    for (d : divisors) {
        test d and -d;
        if eval == 0 insert root;
    }

    return Complete{roots};
}
```

测试必须包括：

```text
x^2 - 5x + 6 = 0       -> {2,3}
x^3 - 1331 = 0         -> {11}
x*(x-11) = 0           -> {0,11}
2x^2 - 8 = 0           -> {-2,2}
x^2 - 2 = 0            -> no roots, Complete -> UNSAT
```

# Phase 1.1：DomainStore 做成可靠基础设施

## 数据结构

```cpp
struct BoundWithReason {
    mpz_class value;
    std::vector<SatLit> reasons;
};

struct IntDomain {
    bool hasLower = false;
    bool hasUpper = false;
    BoundWithReason lower;
    BoundWithReason upper;

    std::optional<std::set<mpz_class>> finiteValues;
    std::vector<SatLit> finiteSetReasons;

    std::map<mpz_class, std::vector<SatLit>> excludedValues;
};
```

## 操作语义

### `addLowerBound(x, lb, reason)`

```text
如果没有 lower：设置 lower=lb, reasons={reason}
如果 lb > oldLower：替换 lower=lb, reasons={reason}
如果 lb == oldLower：追加 reason
如果 lb < oldLower：忽略，因为旧 bound 更强
```

### `addUpperBound(x, ub, reason)`

对称处理。

### `restrictToFiniteSet(x, values, reason)`

```text
如果没有 finiteValues：finiteValues = values
如果已有 finiteValues：取交集
finiteSetReasons 追加 reason
```

### `excludeValue(x, v, reason)`

```text
excludedValues[v].push_back(reason)
```

## Empty domain 判定

```text
1. lower > upper
2. finiteValues 为空
3. finiteValues ∩ [lower, upper] 为空
4. finiteValues 中所有值都被 excluded
```

## Conflict 构造

### lower > upper

```text
conflict = ¬lowerReasons ∨ ¬upperReasons
```

### finiteValues 为空

```text
conflict = ¬finiteSetReasons
```

### finiteValues 与 bounds 无交集

```text
conflict = ¬finiteSetReasons ∨ ¬boundReasons
```

### finiteValues 全部被排除

```text
conflict = ¬finiteSetReasons ∨ ¬excludedValueReasons[v1] ∨ ...
```

先允许 coarse conflict，但必须 sound。

# Phase 1.3：handleSquareBound / SquareBoundReasoner

这个是下一步最值得做的 NIA 推理。不要等完整 ICP。

## 支持模式

### `x^2 <= c`

如果 `c < 0`：

```text
UNSAT
```

如果 `c >= 0`：

```text
x ∈ [-floorSqrt(c), floorSqrt(c)]
```

实现成：

```cpp
domains.addLowerBound(x, -floorSqrt(c), reason);
domains.addUpperBound(x,  floorSqrt(c), reason);
```

### `x^2 >= c`

暂时只处理显然 tautology：

```text
x^2 >= 0 -> NoChange / tautology
```

如果 `c > 0`，它对应：

```text
x <= -ceilSqrt(c) ∨ x >= ceilSqrt(c)
```

这是 disjunction。Phase 1 暂时不要做，后面作为 branch lemma。

### `x^2 = c`

如果 `c < 0`：

```text
UNSAT
```

如果 `c` 不是完全平方：

```text
UNSAT
```

如果 `c = 0`：

```text
x = 0
```

如果 `c = r^2`：

```text
x ∈ {-r, r}
```

### `x^2 != c`

如果 `c < 0`：

```text
tautology
```

如果 `c` 不是完全平方：

```text
tautology
```

如果 `c = r^2`：

```text
x != r
x != -r
```

---

## `floorSqrt` / `isSquare`

必须用 exact integer arithmetic：

```cpp
mpz_class floorSqrt(const mpz_class& n);
bool isPerfectSquare(const mpz_class& n, mpz_class* root);
```

GMP 有 `mpz_sqrt` / `mpz_perfect_square_p` 可用；否则自己实现 binary search。

# Phase 1.4：BoundedNiaSolver direct enumeration

## 触发条件

```text
所有变量 domain finite
totalSize <= threshold
```

注意：

```text
domain finite 可以来自：
1. finiteValues；
2. lower 和 upper 都存在，且区间大小有限；
3. finiteValues 与 bounds/exclusions 组合。
```

## 枚举域生成

对每个变量：

```cpp
std::vector<mpz_class> enumerateDomain(var);
```

规则：

```text
如果 finiteValues 存在：
  values = finiteValues
  过滤 bounds
  过滤 exclusions

否则：
  要求 hasLower && hasUpper
  values = [lower, upper]
  过滤 exclusions
```

如果任一变量 values 为空：

```text
DomainStore 应该已经 empty；返回 UNSAT conflict。
```

## 枚举算法

```cpp
BoundedSolveResult solve(const constraints, const domains) {
    vars = collectVars(constraints);

    if (!domains.allFinite(vars)) return UnknownUnsupported;

    if (domains.totalSize(vars) > threshold) return UnknownBudget;

    for assignment in cartesianProduct(domains.enumerate(var)):
        if (validator.validate(assignment, constraints)):
            return Sat(assignment);

    return UnsatComplete(allActiveReasonsConflict);
}
```

## UNSAT conflict

第一版使用 coarse conflict：

```text
所有 active normalized constraint reasons 的否定
```

这是 sound 的，因为完整枚举证明整组 active constraints 无解。

# Phase 2：最小 branch lemma

## 目标

当 domain 不完整、bounded solver 不能启动时，至少让 SAT/CDCL 能继续切整数空间。

## 只做全局有效 split

Phase 2 只允许：

```text
x <= k ∨ x >= k+1
```

不要先做：

```text
x = v1 ∨ x = v2 ∨ ...
```

因为 finite-domain split 是 domain-specific，需要 guarded reasons，容易复杂。

## 变量选择

优先级：

```text
1. 已有一个 finite lower/upper 的变量；
2. 当前 domain 最大的变量；
3. 出现在最多 constraints 中的变量；
4. fallback: 第一个变量。
```

## k 的选择

```text
如果 x 有 lower 和 upper：
  k = floor((lower + upper) / 2)

如果只有 lower：
  k = lower

如果只有 upper：
  k = upper - 1

如果无界：
  k = 0
```

## 动态 atom

Branch lemma 需要动态 polynomial atom：

```text
x <= k
x >= k+1
```

必须走：

```cpp
TheoryAtomRegistry::getOrCreatePolynomialAtom(...)
```

不允许 fake ExprId。

## 注意

NIA 的 polynomial atom 是：

```text
poly rel rhs
```

route 回 NiaSolver 时要转换成：

```text
(poly - rhs) rel 0
```

## 3.3 Interval evaluation

实现：

```cpp
IntervalZ evalInterval(poly, domains)
```

要求 over-approx：

```text
evalInterval(poly, box) 包含 poly 在 box 上所有可能值。
```

规则：

```text
add/sub/neg/mul/pow
```

乘法：

```text
[a,b] * [c,d] =
[min(ac,ad,bc,bd), max(ac,ad,bc,bd)]
```

偶次幂：

```text
[l,u]^2:
  if 0 in [l,u] -> [0, max(l^2,u^2)]
  else -> [min(l^2,u^2), max(l^2,u^2)]
```

奇次幂：

```text
[l,u]^k = [l^k, u^k]
```

---

## 3.4 Constraint interval status

对于 normalized constraint：

```text
p <= 0
```

若：

```text
interval.lo > 0
```

则 definitely violated。

若：

```text
interval.hi <= 0
```

则 definitely satisfied。

否则 unknown。

对于：

```text
p >= 0
```

若：

```text
interval.hi < 0
```

violated。

若：

```text
interval.lo >= 0
```

satisfied。

对于：

```text
p = 0
```

若：

```text
0 notin [lo,hi]
```

violated。

若 interval 是 `[0,0]`：

```text
satisfied
```

否则 unknown。

# Phase 4：Interval B&B

## 目标

当 domain finite 但 totalSize 太大时，不直接枚举，而是区间分裂。

## 入口

```text
domains.allFinite(vars) == true
totalSize > enumerationThreshold
```

## 算法

```cpp
solveBox(box):
    if budget exceeded:
        return UnknownBudget

    status = evaluate all constraints by interval

    if any definitely violated:
        return UnsatComplete for this box

    if box is singleton:
        exact validate
        if sat return Sat
        else return UnsatComplete for this box

    choose split variable
    split into left/right boxes
    solve left
    solve right

    if either Sat -> Sat
    if both UnsatComplete -> UnsatComplete
    else UnknownBudget/UnknownUnsupported
```

## Correctness

只有所有 boxes 都 closed，才返回 UnsatComplete。

任何 budget exhausted：

```text
UnknownBudget
```

# Phase 5：代数增强

等前面稳定后再做。

## 5.1 多变量 GCD conflict

对 normalized equality：

```text
p = 0
```

设：

```text
p = q + c
```

其中 `q` 没有常数项。

若所有 monomial integer-valued，且：

```text
g = gcd(coefficients of q)
g ∤ -c
```

则 UNSAT。

例子：

```text
6x + 9y = 5
g=3, 3 ∤ 5 -> UNSAT

2x^2 + 4yz = 3
g=2, 2 ∤ 3 -> UNSAT
```

## 5.2 Factor rules

对 syntactic product：

```text
p*q = 0
```

lemma：

```text
¬(p*q=0) ∨ (p=0) ∨ (q=0)
```

对：

```text
p*q != 0
```

lemmas：

```text
¬(p*q!=0) ∨ (p!=0)
¬(p*q!=0) ∨ (q!=0)
```

需要 dynamic polynomial atom route。

## 5.3 Modular reasoning

对 equalities：

```text
p1=0, ..., pk=0
```

选小模数：

```text
2,3,4,5,7,8,9,11
```

若没有 residue assignment 同时满足所有 equalities：

```text
UNSAT conflict = all equality reasons
```

# Phase 6：Local search 只做 SAT finder

Local search 第一版只作为 SAT finder，不参与 UNSAT。

## 输入

```text
normalized constraints
domains
```

## 初始点

```text
1. domain midpoint
2. small integers [-10,10]
3. univariate roots
4. random within bounds
```

## objective

对每个 constraint 计算 violation：

```text
p <= 0: max(0,p)
p >= 0: max(0,-p)
p = 0: abs(p)
p != 0: p==0 ? 1 : 0
```

总目标：

```text
Σ violation^2
```

若 objective 为 0：

```text
exact validate
return model
```

否则 budget 用完：

```text
No result
```
