# 1. `terms()` 对常数多项式的处理前后矛盾

前面已经写对了：

```text
For a constant polynomial c, return { MonomialTerm{c, {}} }.
```

也就是说常数项用：

```cpp
MonomialTerm{ coefficient = c, powers = {} }
```

表示。这个是对的，因为 multivar GCD 需要从：

```text
p = q + c
```

中拿到常数项 `c`。

但是后面 LibPolyKernel 实现部分又写：

```text
If C API fails, coefficient is non-integer, or polynomial is constant → return nullopt.
```

这里错了。常数多项式不应该 `nullopt`，应该返回一个 constant term。否则 `2x+4y-3` 的 `-3` 就可能拿不到。

必须改成：

```text
If C API fails, coefficient is non-integer, or backend unsupported → return nullopt.
If polynomial is constant c → return { MonomialTerm{c, {}} }.
```

# 2. IntervalEvaluator 的结果类型仍然残留 NIA 耦合

前面已经把公共 interval 层改成：

```cpp
enum class IntervalEvalStatus {
    NoChange,
    DefinitelyViolated
};

struct IntervalEvalResult {
    IntervalEvalStatus status;
    std::vector<SatLit> usedReasons;
};
```

这是对的。

但 C5 里还写：

```cpp
auto ir = intervalEvaluator_.run(normalized, box);
if (ir.kind == NiaReasoningKind::Conflict) { ... }
```

这又把公共 interval 返回值和 NIA 的 `NiaReasoningKind` 混在一起了。应该统一改成：

```cpp
auto ir = intervalEvaluator_.run(normalized, box);
if (ir.status == IntervalEvalStatus::DefinitelyViolated) {
    // NIA adapter builds TheoryConflict here
}
```

同时这句也要小心：

```cpp
auto lits = constraint.reason.negated();
```

如果 `constraint.reason` 是一个 `SatLit`，那 `lits` 应该是 vector：

```cpp
std::vector<SatLit> lits;
lits.push_back(constraint.reason.negated());
lits.insert(lits.end(), ir.usedReasons.begin(), ir.usedReasons.end());
return NiaReasoningResult::conflict(TheoryConflict{lits});
```
