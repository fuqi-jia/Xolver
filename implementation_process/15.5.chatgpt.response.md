建议选 **[4] Revise**。这版已经基本可以执行，但还有一个残留的接口矛盾，最好再修一下再 Approve。整体范围是合理的：本轮只做 `terms()`、多变量 GCD、公共 interval skeleton、迁移单变量 `IntervalEvaluator`，factor/modular 都暂缓。

主要问题是 **C5 仍然残留旧接口**。

前面 C4 已经改成了正确的公共接口：

```cpp
auto ir = intervalEvaluator_.run(c, box);
if (ir.status == IntervalEvalStatus::DefinitelyViolated) {
    ...
}
```

但 C5 里又写了：

```cpp
auto ir = intervalEvaluator_.run(normalized, box);
if (ir.status == IntervalEvalStatus::DefinitelyViolated) {
    std::vector<SatLit> lits;
    lits.push_back(constraint.reason.negated());
    ...
}
```

这里有两个问题：

```text
1. run(normalized, box) 和 run(c, box) 不一致；
2. constraint 变量在这段伪代码里没有定义。
```

应该统一成 NIA adapter 自己遍历每个 constraint：

```cpp
for (const auto& c : normalized) {
    auto ir = intervalEvaluator_.run(c, box);

    if (ir.status == IntervalEvalStatus::DefinitelyViolated) {
        std::vector<SatLit> lits;
        lits.push_back(c.reason.negated());
        lits.insert(lits.end(), ir.usedReasons.begin(), ir.usedReasons.end());
        return NiaReasoningResult::conflict(TheoryConflict{lits});
    }
}
```

另外有一句话建议改得更准确：

```text
No UNSAT decision may depend on terms() succeeding.
```

这句话容易误解。多变量 GCD 的 UNSAT 本来就依赖 `terms()` 成功并且精确。应该改成：

```text
If terms() fails or is incomplete, return NoChange.
UNSAT may be returned only when terms() succeeds and the decomposition is exact.
```
