建议 **还要 Revise**，不是大方向问题，而是这版仍残留一个之前反复提过的接口矛盾：`markEmitted(lemma)` 还是不对。整体方案已经非常接近可执行：co-register LIA/LRA、mirror effective active bounds、abstraction lemma / cut lemma 分离、square nonnegativity、canonical `NonlinearTermKey`、adapter member + pending queue 这些关键点都已经补上了。

但现在还有几个需要修的地方。

---

# 1. 最大残留问题：`markEmitted(lemma)` 仍然无法实现

计划前面定义的 cache API 是：

```cpp
markEmitted(
  NonlinearTermKey key,
  SatLit nonlinearReason,
  vector<mpq_class> boundValues,
  int cutIndex
)
```

但 `NiaSolver::check()` 伪代码里仍然写：

```cpp
pendingLinLemmas_.push_back(std::move(lemma));
linAdapter_->markEmitted(lemma);
```

这还是不匹配。`TheoryLemma` 本身没有 `NonlinearTermKey / nonlinearReason / boundValues / cutIndex`，所以 adapter 无法从一个 lemma 反推出 cache key。

必须改成下面之一。

**方案 A：返回带 metadata 的 pending item。**

```cpp
struct PendingLinearizationLemma {
    TheoryLemma lemma;
    std::optional<CutCacheKey> cacheKey;
};

struct LinearizationResult {
    LinearizationStatus status;
    std::vector<PendingLinearizationLemma> lemmas;
};
```

然后：

```cpp
for (auto& item : lr.lemmas) {
    if (!lemmaDb.contains(item.lemma)) {
        lemmaDb.insertIfNew(item.lemma);
        pendingLinLemmas_.push_back(std::move(item.lemma));
        if (item.cacheKey) {
            linAdapter_->markEmitted(*item.cacheKey);
        }
    }
}
```

**方案 B：在 `IncrementalLinearizer::run()` 内部只对真正返回的 lemma 标记 emitted。**

也就是说，`run()` 先生成候选 cut，应用 `maxLemmas` 截断，然后只把进入 `result.lemmas` 的 cut 标记 emitted。不要在 adapter 层标记。

我更建议方案 A，因为 abstraction lemma 可以没有 cache key，cut lemma 可以带 cache key，语义更清楚。

---

# 2. `BoundStore` 的策略前后有轻微冲突

前面 `D4` 写：

```text
If bounds are unconditional (no reasons), omit bound guards.
```

后面又写：

```text
Empty reasons does not mean unconditional.
If finite bound has no complete reason and is not global, skip the cut.
```

后者是正确的。

建议把 D4 改成：

```text
Only omit bound guards when the bound is explicitly marked global.
Empty reason vectors alone do not imply global/unconditional.
```

否则实现者可能看到 D4 后直接把 empty reasons 当成 unconditional，生成未 guard 的 McCormick cut。

---

# 3. “Missing bounds -> NoChange” 测试仍然不精确

测试表里仍然写：

```text
Missing bounds -> NoChange | x∈[0,10], y unbounded → NoChange
```

但完整 linearizer 即使缺少 product 的 finite complete bounds，也仍然可能生成：

```text
abstraction lemma: ¬C ∨ p_L rel 0
square nonnegativity: s >= 0
```

计划后文也明确说 “No finite complete bounds” 只是不发 bounds-dependent envelope cuts，abstraction lemma 和 square nonnegativity 仍允许。

所以这个测试应该改成：

```text
Missing product bounds:
  McCormickGenerator returns no envelope cuts;
  full IncrementalLinearizer may still emit abstraction lemmas.
```

如果要测 `NoChange`，只测 `McCormickGenerator`，不要测完整 `IncrementalLinearizer::run()`。

---

# 4. `LinearizationResult` 目前无法表达 mirror lemmas

计划说 NIA/NRA adapter 会生成 mirror lemmas，把 original active linear bounds 送到 LIA/LRA。这个设计是对的。

但现在 `IncrementalLinearizer::run()` 只吃 nonlinear constraints 和 bounds，不负责 active assignment 扫描。要明确 mirror lemmas 由谁返回：

```text
NiaLinearizationAdapter::run returns:
  mirror lemmas
  + linearization lemmas
```

或者：

```text
NiaSolver has a separate MirrorLinearBounds step before linearizer.
```

建议分开，结构更清晰：

```text
9.4 mirror effective active linear bounds to LIA/LRA;
9.5 run IncrementalLinearizer for nonlinear constraints.
```

否则 adapter 既扫 active assignments 又扫 normalized nonlinear constraints，职责容易混乱。

---

# 5. Co-register 需要一个硬回归

计划依赖：

```text
TheoryManager dispatches by TheoryAtomRecord.theory.
LIA/LRA solvers silently ignore polynomial atoms.
NIA/NRA solvers silently ignore linear atoms.
```

这个前提必须测试，因为一旦某个 solver 对非 owner atom 返回 Unknown，co-register 会直接把 QF_NIA 搞坏。计划里已有 routing tests，建议再具体一点：

```text
QF_NIA:
  original nonlinear atom (* x x)=4 must not be sent as unsupported to LiaSolver.
  generated LIA cut must not be sent as unsupported to NiaSolver.

QF_NRA:
  original nonlinear atom must not break LraSolver.
  generated LRA cut must be consumed by LraSolver.
```
