# Plan: NIA-Core — Univariate + Domain + Bounded Search Pipeline

## Goal
Upgrade `NiaSolver` from "trivial constants only" to a real nonlinear integer engine:
- **Univariate integer equality solving** (`x^2=4` → SAT, `x^2=2` → UNSAT)
- **Square / bound inference** (`x^2<=9` → finite domain)
- **Algebraic reasoning** (GCD equality conflict, square rules, factor rules)
- **Modular reasoning** (`x^2=2` → mod-4 UNSAT)
- **Domain store** (intervals + finite sets + exclusions)
- **Bounded complete solver** (finite-domain enumeration / interval B&B)
- **Exact integer validation** for every SAT result
- **Dynamic polynomial atom support** for lemmas

After NIA-Core, these must NOT be `Unknown`:
```smt2
(assert (= (* x x) 4))                          ; sat
(assert (= (* x x) 2))                          ; unsat
(assert (= (+ (* x x) 1) 0))                    ; unsat
(assert (and (>= x 0) (<= x 10) (= (* x x) 49))) ; sat
(assert (and (>= x 0) (<= x 10) (= (* x x) 50))) ; unsat
```

## NIA-Core Component Design

### 1. `NiaNormalizer` — `src/theory/arith/nia/NiaNormalizer.h/.cpp`

**Input**: `std::vector<ActiveNiaConstraint>` (from `NiaSolver` trail)
**Output**: `std::vector<NormalizedNiaConstraint>` or `nullopt` (unsupported)

**Critical preprocessing — effective literal**:
`ActiveNiaConstraint` carries a `SatLit reason` from the current SAT assignment. Before normalization, `NiaNormalizer` must compute the **effective relation**:
- If the SAT assignment made the atom **true** (positive literal asserted): use the original relation.
- If the SAT assignment made the atom **false** (negative literal asserted): negate the relation first:
  - `Eq` → `Neq`
  - `Neq` → `Eq`
  - `Leq` → `Gt` (since `¬(p <= 0)` ≡ `p > 0`)
  - `Geq` → `Lt` (since `¬(p >= 0)` ≡ `p < 0`)
  - `Lt` → `Geq`
  - `Gt` → `Leq`
Then apply strict inequality normalization to the effective relation.

**Rules**:
1. Compute effective relation from `SatLit` sign (negate if false-assigned)
2. `p rel_eff 0` form, where `rel_eff` preserves strictness (Gt/Lt are preserved, not mapped to Geq/Leq)
3. Clear denominators: multiply by positive LCM of all coefficients
4. Strict inequality normalization:
   - `Gt` (`p > 0`) → `p >= 1`
   - `Lt` (`p < 0`) → `p <= -1`
5. Output relations: `Eq`, `Neq`, `Leq`, `Geq` only

**Soundness note**: Never map `Leq false` directly to `Geq`. `¬(p <= 0)` is `p > 0`, which normalizes to `p >= 1`. Mapping to `p >= 0` would be a weaker constraint and unsound. Similarly for `Geq false` → `Lt` → `p <= -1`, not `Leq`.

```cpp
struct NormalizedNiaConstraint {
    PolyId poly;      // integer coefficients
    Relation rel;     // Eq, Neq, Leq, Geq
    SatLit reason;
};

class NiaNormalizer {
public:
    explicit NiaNormalizer(PolynomialKernel& kernel);
    std::optional<std::vector<NormalizedNiaConstraint>>
    normalize(const std::vector<ActiveNiaConstraint>& active);
private:
    PolynomialKernel& kernel_;
};
```

### 3. `DomainStore` — `src/theory/arith/nia/DomainStore.h/.cpp`

**Per-variable domain with reasons**:
```cpp
struct BoundWithReason {
    mpz_class value;
    std::vector<SatLit> reasons;  // all restrictions that established this bound
};

struct IntDomain {
    bool hasLower = false;
    bool hasUpper = false;
    BoundWithReason lower;
    BoundWithReason upper;

    std::optional<std::set<mpz_class>> finiteValues;
    std::vector<SatLit> finiteSetReasons;  // all restrictions that established finite set
    std::map<mpz_class, std::vector<SatLit>> excludedValues;  // value -> all exclusion reasons
};
```

**Operations** (all carry `SatLit reason`):
- `restrictToFiniteSet(var, values, reason)` — intersects with existing finite set, **appends reason** to `finiteSetReasons`
- `addLowerBound(var, lb, reason)` — takes max lower bound, **appends reason** to `lower.reasons`
- `addUpperBound(var, ub, reason)` — takes min upper bound, **appends reason** to `upper.reasons`
- `excludeValue(var, v, reason)` — inserts into `excludedValues[v]`, **appends reason**
- `isEmpty(var)` / `isEmpty()`
- `allFinite(vars)`
- `totalSize()` (product of domain sizes, for enumeration threshold)
- `TheoryConflict buildEmptyDomainConflict()` — collects **all reasons** that contributed to the emptiness:
  - If lower bound > upper bound: collect `lower.reasons` + `upper.reasons`
  - If finite set is empty after intersection: collect all `finiteSetReasons`
  - If finite set ∩ [lower, upper] is empty: collect `lower.reasons` + `upper.reasons` + `finiteSetReasons`
  - If every value in finite set is excluded: collect `finiteSetReasons` + all `excludedValues[value].reasons`
  Never emit conflict from an incomplete reason set.

### Common Result Type for Reasoners

All reasoning engines (Univariate, Algebraic, BoundInferer) use a shared result type that distinguishes domain updates from terminal results:

```cpp
enum class NiaReasoningKind {
    NoChange,      // not applicable / incomplete / skipped, continue
    DomainUpdated, // domains modified, continue to next engine
    Conflict,      // sound UNSAT proof found
    Lemma,         // theory lemma generated
    FatalUnknown   // fatal unsupported state, must terminate
};

struct NiaReasoningResult {
    NiaReasoningKind kind;
    std::optional<TheoryConflict> conflict;
    std::optional<TheoryLemma> lemma;
};
```

**Rule**: `DomainUpdated` does **not** return `TheoryCheckResult::consistent()`. It continues the pipeline so `BoundedNiaSolver` can use the updated domains. Only `Conflict`, `Lemma`, or `FatalUnknown` are terminal.

### 4. `UnivariateIntegerReasoner` — `src/theory/arith/nia/UnivariateIntegerReasoner.h/.cpp`

**Core capability**: Solve `p(x)=0` for integer roots.

**Algorithm**:

```cpp
enum class IntegerRootStatus { Complete, Incomplete };
struct IntegerRootResult {
    IntegerRootStatus status;
    std::set<mpz_class> roots;
    bool isZeroPolynomial = false;
};
```

1. Extract variable names via `kernel_.variables(poly)`
2. If `vars.size() == 1` → univariate
3. **Handle zero polynomial explicitly**:
   - If `kernel_.isZero(poly)`:
     - `Eq`/`Leq`/`Geq`: tautology (satisfied)
     - `Neq`: UNSAT conflict
4. For `Eq`:
   - Get exact coefficients via `getIntegerCoefficients()`
   - If coefficients unavailable → return **no-result** (continue to bounded/modular)
   - Factor out `x` while constant term is 0 (collect root 0)
   - Apply Rational Root Theorem: all integer roots divide `|a_0|`
   - **Complete enumeration required for UNSAT**: enumerate **all** divisors of `|a_0|`
   - If divisor count exceeds cap → return **Incomplete** (cannot prove UNSAT)
   - Test each divisor `d` and `-d` by `evalInteger`
   - If `status == Complete` and no roots → UNSAT conflict (use the equality's `reason`)
   - If roots found → `DomainStore::restrictToFiniteSet(x, roots, c.reason)`
5. For `Neq`:
   - Same coefficient extraction / divisor rules
   - If complete and no roots → tautology (ignore)
   - If roots found → `DomainStore::excludeValue(x, root, c.reason)` for each
6. For `Leq` / `Geq`:
   - `x^2 <= c` (c>=0) → `[-floor(sqrt(c)), floor(sqrt(c))]`
   - `x^2 >= 0` → tautology
   - `x^2 < 0` (already normalized to `x^2 <= -1`) → UNSAT

**Note**: RRT requires coefficient extraction. Since `LibPolyKernel` uses libpoly's internal representation, we need a way to extract coefficients for univariate polynomials. If libpoly doesn't expose this directly, we can evaluate at enough points and use finite differences / interpolation, or we can extend `PolynomialKernel` with `getCoefficients(poly)`.

**Required extensions to `PolynomialKernel`**:
```cpp
// Exact degree of polynomial with respect to a given variable.
// Returns nullopt if not univariate in x or unsupported.
virtual std::optional<int> degree(PolyId a, std::string_view var) const = 0;

// For univariate polynomials only. Returns coefficients from highest degree to constant.
// Returns nullopt if not univariate, unsupported, or coefficients unavailable.
virtual std::optional<std::vector<mpz_class>>
getIntegerCoefficients(PolyId a, std::string_view var) const = 0;
```

**Implementation priority**:
1. **LibPolyKernel**: Use libpoly term iteration if available; otherwise use evaluation+interpolation with **exact degree**.
2. **StubPolyKernel**: Return `nullopt`.

**Interpolation fallback must be conservative**:
```text
Only use interpolation if:
1. variables(poly) returns exactly one variable (confirmed univariate)
2. degree(poly, var) returns an exact finite degree d
3. Evaluate at d+1 distinct integer points
4. Solve for coefficients via exact rational interpolation
5. Verify: reconstruct polynomial and test equality at d+2 additional points,
   or compare degree-reconstructed polynomial against original via kernel subtraction
6. If any coefficient is not an integer → return nullopt
7. If verification fails → return nullopt
```
Never infer UNSAT from coefficient extraction that hasn't been verified.

**Soundness rule**: If `degree()` or `getIntegerCoefficients()` returns `nullopt`, `UnivariateIntegerReasoner` must return **no-result / NoChange**, never UNSAT. Only complete coefficient extraction supports RRT-based UNSAT proof.

### 5. `LinearNiaDomainReasoner` — `src/theory/arith/nia/LinearNiaDomainReasoner.h/.cpp`

Extracts domain updates from single-variable linear constraints. Must run before `BoundedNiaSolver`.

**Input**: `std::vector<NormalizedNiaConstraint>`
**Output**: Domain updates via `DomainStore`

**Rules** (only for constraints of form `a*x + c rel 0` where `a != 0`):
- `a*x + c <= 0`, `a > 0` → `x <= floor(-c/a)` → `addUpperBound(x, floor(-c/a), reason)`
- `a*x + c <= 0`, `a < 0` → `x >= ceil(-c/a)` → `addLowerBound(x, ceil(-c/a), reason)`
- `a*x + c >= 0`, `a > 0` → `x >= ceil(-c/a)` → `addLowerBound(x, ceil(-c/a), reason)`
- `a*x + c >= 0`, `a < 0` → `x <= floor(-c/a)` → `addUpperBound(x, floor(-c/a), reason)`
- `a*x + c = 0` → if `a` divides `c` exactly: `x = -c/a` → `restrictToFiniteSet(x, {-c/a}, reason)`; else UNSAT (no integer solution)
- `a*x + c != 0` → if `a` divides `c` exactly: `excludeValue(x, -c/a, reason)`

**Non-linear constraints are ignored** by this reasoner.

### 6. `AlgebraicIntegerReasoner` — `src/theory/arith/nia/AlgebraicIntegerReasoner.h/.cpp`

**Rules** (each returns `NiaReasoningResult`):

**A. Square rules**
- `x^2 >= 0`: tautology, ignore
- `x^2 < 0` (normalized to `x^2 <= -1`): UNSAT
- `x^2 <= 0`: implies `x = 0` → `DomainStore::restrictToFiniteSet(x, {0}, reason)`
- `x^2 + y^2 <= 0`: implies `x=0, y=0`
- `x^2 + y^2 < 0`: UNSAT

**B. Coefficient GCD equality conflict**
- Requires polynomial term decomposition to split `p = q + c` and extract coefficients
- If term decomposition unavailable → `NoChange` (skip, do not guess)
- For `p = 0` where all monomials are integer-valued:
  - Split `p = q + c` where `q` has no constant term
  - `g = gcd(|coefficients of q|)`
  - If `g != 0` and `(-c) % g != 0` → UNSAT
  - Conflict uses the equality's `reason`
  - Example: `2x^2 + 4yz = 3` → `q = 2x^2 + 4yz`, `c = -3`, `g = 2`, `3 % 2 = 1 != 0` → UNSAT

**C. Factor rules** (syntactic product recognition)
- Requires explicit term decomposition or AST-level product recognition
- If product structure unavailable → `NoChange` (skip, do not guess)
- `p*q = 0` → generate lemma: `¬(p*q=0) ∨ (p=0) ∨ (q=0)`
- `p*q != 0` → generate lemmas: `¬(p*q!=0) ∨ (p!=0)`, `¬(p*q!=0) ∨ (q!=0)`
- Uses `TheoryAtomRegistry::getOrCreatePolynomialAtom`

**Dynamic polynomial atom route must be complete**:
```cpp
SatLit factorLit = registry.getOrCreatePolynomialAtom(
    factorPoly, Relation::Eq, 0, TheoryId::NIA);
```
Requirements:
1. `factorPoly` must be a real `PolyId` from the same `PolynomialKernel` (not fake)
2. `getOrCreatePolynomialAtom` creates a real SAT var, registers in `TheoryAtomRegistry`
3. The new atom maps to a real `PolynomialAtomPayload` in `Atomizer`
4. When SAT selects this literal (true or false), `NiaSolver::assertLit` must receive it via `TheoryManager` dispatch and convert it to an `ActiveNiaConstraint`
5. No fake `ExprId` — dynamic atoms use polynomial lookup keys, not expression IDs

**D. Modular reasoning** (small modulus)
- For each active **normalized equality** `p(x)=0`:
  - Test mod `m ∈ {2,3,4,5,7,8,9,11}`
  - If `numVars <= 6`, enumerate all residue assignments
  - Check all equalities **simultaneously**: a residue must satisfy every equality
  - If no residue satisfies all equalities simultaneously → UNSAT
  - **Conflict must include all equality reasons**: `conflict = { ¬reason(e1), ¬reason(e2), ..., ¬reason(ek) }`
  - Example: `x^2 = 2` → mod 4: squares are 0 or 1, never 2 → UNSAT (single equality)
  - Example: `x^2 = 1, y^2 = 2` → mod 4: no residue satisfies both → UNSAT (both reasons in conflict)

**Restriction**: Modular reasoning only for equalities. Do **not** use it to prove `p != 0` false. Only check equalities whose coefficients have been cleared of denominators.

**Soundness rule for all algebraic rules**: If polynomial term decomposition, product recognition, or coefficient extraction is unavailable, the rule must return `NoChange` (skip). Never infer UNSAT from incomplete structural analysis.

### 7. `BoundedNiaSolver` — `src/theory/arith/nia/BoundedNiaSolver.h/.cpp`

**Trigger**: `DomainStore::allFinite()` is true.

```cpp
enum class BoundedSolveStatus {
    Sat,              // exact model found and validated
    UnsatComplete,    // complete enumeration / complete interval B&B: no model exists
    UnknownBudget,    // budget exhausted before complete search
    UnknownUnsupported // interval evaluation unsupported for some operator
};

struct BoundedSolveResult {
    BoundedSolveStatus status;
    std::optional<IntegerModel> model;      // only for Sat
    std::optional<TheoryConflict> conflict; // only for UnsatComplete
};
```

**Strategy**:
1. If `totalSize() <= threshold` (e.g., 10000): direct enumeration
2. Otherwise: interval branch-and-bound with pruning

**Direct enumeration** (always complete when triggered):
```cpp
for each assignment in cartesian product of finite domains:
    if validator.validate(assignment, constraints):
        return {Sat, model};
return {UnsatComplete, nullopt, allActiveReasonsConflict};
```
Direct enumeration is **complete**: if it runs, it covers every value in every finite domain.

**Interval branch-and-bound**:
- For each constraint, evaluate `IntervalZ` over current box
- If constraint definitely violated → prune branch (over-approx sound)
- If definitely satisfied → skip
- Choose split variable (largest domain or most active)
- Split at midpoint: `x <= mid` / `x >= mid+1`
- Recurse
- **Budget tracking**: track node count / recursion depth
- If budget exhausted before all branches are closed:
  - Return `{UnknownBudget}` — **never** return UnsatComplete from incomplete search
- If all branches are pruned or closed without finding a model:
  - Return `{UnsatComplete, conflict over all active reasons}`

**UnsatComplete conflict**: This is a coarse but sound conflict. Since the bounded solver performed a complete search over all finite domains, the entire set of active constraints is unsatisfiable. Returning all active normalized constraint reasons is sound. The simplest implementation is to always include all active reasons in the conflict.

**IntervalZ**:
```cpp
struct IntervalZ {
    mpz_class lo;
    mpz_class hi;
    bool empty = false;
};
```

Operations: `add`, `sub`, `neg`, `mul` with conservative interval arithmetic.

**Multiplication**:
```
[a,b] * [c,d] = [min(ac,ad,bc,bd), max(ac,ad,bc,bd)]
```

**Power** (conservative, must not under-approximate):
```
x^k, k even:
  if 0 ∈ [l,u]:   [0, max(l^k, u^k)]
  else:           [min(l^k, u^k), max(l^k, u^k)]
x^k, k odd:
  [l^k, u^k]
```

**Critical**: Interval pruning must be an **over-approximation**. Under-approximation causes false UNSAT.

### 7. `NiaLocalSearch` — `src/theory/arith/nia/NiaLocalSearch.h/.cpp` (skeleton for NIA-Core)

**Phase NIA-Core scope**: Minimal SAT finder only.

```cpp
class NiaLocalSearch {
public:
    explicit NiaLocalSearch(PolynomialKernel& kernel);
    std::optional<IntegerModel> tryFindModel(
        const std::vector<NormalizedNiaConstraint>& constraints,
        const DomainStore& domains);

```

**Algorithm**:
1. Generate initial assignments:
   - Domain midpoint
   - Univariate roots
   - Small integers in `[-10, 10]`
   - Random within bounds
2. Coordinate descent / small neighborhood search
3. Violation objective: sum of squared violations
4. If `F == 0` found → return model
5. Budget: max iterations / time

**Important**: Any returned model must be exact-validated by `IntegerModelValidator` before `NiaSolver` returns SAT.

## NiaSolver Refactoring

### New `NiaSolver` composition

```cpp
class NiaSolver : public TheorySolver {
private:
    // Infrastructure
    std::unique_ptr<PolynomialKernel> kernel_;
    std::unique_ptr<PolynomialConverter> converter_;

    // Trail state
    std::vector<ActiveNiaConstraint> active_;
    std::vector<NiaTrailEntry> trail_;
    std::optional<PendingConflict> pendingConflict_;
    std::optional<PendingUnknown> pendingUnknown_;

    // NIA-Core engines
    NiaNormalizer normalizer_;
    IntegerModelValidator validator_;
    DomainStore domains_;
    UnivariateIntegerReasoner univariate_;
    AlgebraicIntegerReasoner algebraic_;
    BoundedNiaSolver bounded_;
    NiaLocalSearch localSearch_;

    // Current model (if SAT found)
    std::optional<IntegerModel> currentModel_;

```

### New `check()` pipeline

```cpp
TheoryCheckResult NiaSolver::check(TheoryLemmaDatabase& lemmaDb) {
    if (pendingUnknown_) return unknown();
    if (pendingConflict_) return mkConflict(pendingConflict_->conflict);
    if (active_.empty()) return consistent();

    // 1. Normalize
    auto normalized = normalizer_.normalize(active_);
    if (!normalized) return unknown();

    // 2. Trivial constants
    if (auto r = checkTrivialConstants(*normalized)) return *r;

    // 3. Reset domains from normalized constraints
    domains_.reset();

    // 4. Linear domain inference
    auto lr = linearDomain_.run(*normalized, domains_);
    if (lr.kind == NiaReasoningKind::Conflict) return mkConflict(*lr.conflict);
    if (lr.kind == NiaReasoningKind::FatalUnknown) return unknown();
    // NoChange / DomainUpdated: continue

    // 5. Univariate reasoning (roots, square bounds)
    auto ur = univariate_.run(*normalized, domains_, lemmaDb);
    if (ur.kind == NiaReasoningKind::Conflict) return mkConflict(*ur.conflict);
    if (ur.kind == NiaReasoningKind::Lemma) return mkLemma(*ur.lemma);
    if (ur.kind == NiaReasoningKind::FatalUnknown) return unknown();
    // NoChange / DomainUpdated: continue

    // 6. Algebraic reasoning (square, GCD, factor, modular)
    auto ar = algebraic_.run(*normalized, domains_, lemmaDb);
    if (ar.kind == NiaReasoningKind::Conflict) return mkConflict(*ar.conflict);
    if (ar.kind == NiaReasoningKind::Lemma) return mkLemma(*ar.lemma);
    if (ar.kind == NiaReasoningKind::FatalUnknown) return unknown();
    // NoChange / DomainUpdated: continue

    // 7. Check if domains became empty
    if (domains_.isEmpty()) {
        return mkConflict(domains_.buildEmptyDomainConflict());
    }

    // 8. Bounded complete solver
    auto allVars = collectVars(*normalized);
    if (domains_.allFinite(allVars)) {
        auto br = bounded_.solve(*normalized, domains_, validator_, lemmaDb);
        if (br.status == BoundedSolveStatus::Sat) {
            currentModel_ = br.model;
            return consistent();
        }
        if (br.status == BoundedSolveStatus::UnsatComplete) {
            return mkConflict(*br.conflict);
        }
        // UnknownBudget / UnknownUnsupported: continue pipeline
    }

    // 9. Local search SAT finder (optional, budget-limited)
    if (auto model = localSearch_.tryFindModel(*normalized, domains_)) {
        if (validator_.validate(*model, *normalized)) {
            currentModel_ = *model;
            return consistent();
        }
    }

    // 10. Branch split or Unknown
    if (auto lemma = buildBranchLemma(*normalized, domains_, lemmaDb)) {
        return mkLemma(*lemma);
    }

    return unknown();
```

**`buildBranchLemma` specification**:
Phase NIA-Core should only emit **globally valid integer splits**:
```smt2
x <= k  ∨  x >= k+1
```
This is valid for any integer `k` regardless of current domains.

If a split is based on current domain information (e.g., finite values `{v1, v2}`):
```smt2
x = v1  ∨  x = v2  ∨  ...
```
This is **not globally valid** and must be guarded by domain reasons:
```smt2
¬reason1  ∨  ¬reason2  ∨  ...  ∨  (x = v1)  ∨  (x = v2)  ∨  ...
```
For Phase NIA-Core, prefer globally valid splits only.
}
```

## Correctness Red Lines

1. **Every SAT result must exact-validate** via `IntegerModelValidator`.
2. **UNSAT must come from sound proof**: constant contradiction, empty integer roots, modular contradiction, GCD contradiction, or finite-domain exhaustion.
3. **Local search failure is NOT UNSAT** — it gives no conclusion.
4. **Rational coefficients must be cleared** in `NiaNormalizer` before any reasoning.
5. **Dynamic atoms must register via `TheoryAtomRegistry`** — no fake ExprId routing.
6. **Conflict clauses must include all active reasons** involved in the proof.
7. **Modular reasoning only for equalities** — don't use it to prove `p != 0`.
8. **Factor split lemmas must be guarded** — `¬premise ∨ conclusion`.
9. **Branch lemmas must be globally valid** — `x <= floor(q) ∨ x >= ceil(q)` for integers.
10. **Unknown is acceptable** for general unbounded nonlinear integer cases.

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| `LibPolyKernel` cannot extract univariate coefficients | Use evaluation+interpolation fallback in `getIntegerCoefficients()` with verification |
| `LibPolyKernel::variables()` incomplete | Fix to iterate all terms; if libpoly API limited, track during construction |
| RRT divisor enumeration too large for large `a_0` | Cap divisor enumeration (e.g., max 1000 divisors); fallback to modular reasoning |
| Bounded solver enumeration explosion | Threshold-based: small domains enumerate, large domains use interval B&B |
| Interval arithmetic overflow | Use `mpz_class` intervals; no fixed-width overflow |
| Modular reasoning exponential for many vars | Limit to `numVars <= 6` and `modulus <= 11`; skip otherwise |
| Dynamic atom registration overhead | Lookup table dedup in `TheoryAtomRegistry`; no-op if atom already exists |
| Product recognition misses hidden factors | Syntactic recognition only; missed factors → Unknown (sound) |
| RRT divisor enumeration too large | Cap at 1000 divisors; return Incomplete (not UNSAT) |
| Bounded solver budget exhaustion | Budget returns Unknown, never UNSAT |
| Accidental fake evalInteger implementation | Unit tests enforce StubPolyKernel returns nullopt; libpoly backend exact |
