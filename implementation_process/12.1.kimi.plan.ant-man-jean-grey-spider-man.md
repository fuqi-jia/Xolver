# Phase 2: Minimal Integer Branch Lemma

## 目标

当 NIA domain 不完整（`!allFinite`）或 bounded solver 返回 UnknownBudget 时，通过 SAT/CDCL 整数分裂 `x ≤ k ∨ x ≥ k+1` 继续搜索。

## 现状

- `NiaSolver::buildBranchLemma` 返回 `nullopt`
- `NiaSolver` 没有 `TheoryAtomRegistry*` 指针（LiaSolver 有 `setRegistry`）
- `Solver.cpp` 中创建 NiaSolver 后没有调用 `setRegistry`
- `assertLit` 已有 `(poly - rhs) rel 0` 转换逻辑，dynamic atom 路由正确

## 设计

### 2.1 NiaSolver 获取 TheoryAtomRegistry

给 `NiaSolver` 添加：
```cpp
void setRegistry(TheoryAtomRegistry* reg) { registry_ = reg; }
```

在 `Solver.cpp` 中：
```cpp
auto nia = std::make_unique<NiaSolver>(std::move(polyKernel));
nia->setRegistry(&registry);
theoryManager.registerSolver(std::move(nia));
```

### 2.2 变量选择策略

按优先级选择分裂变量：
1. **有上下界的变量**（domain 区间明确，分裂最有效）
2. **domain 区间最大的变量**（按 `upper - lower`）
3. **出现在最多约束中的变量**
4. **fallback**: 第一个变量

若没有任何变量可选（理论上不会发生，因为 `hasNonConstant` 已保证），返回 `nullopt`。

### 2.3 k 值选择

```
有 lower 和 upper:   k = floor((lower + upper) / 2)
只有 lower:          k = lower
只有 upper:          k = upper - 1
无界:                k = 0
```

### 2.4 Branch Lemma 构造

对选定的变量 `x` 和值 `k`：

```cpp
// x <= k   =>  poly=x, rel=Leq, rhs=k
PolyId xPoly = kernel_->mkVar(x);
auto litLeq = registry_->getOrCreatePolynomialAtom(xPoly, Relation::Leq, mpq_class(k), TheoryId::NIA);

// x >= k+1 =>  poly=x, rel=Geq, rhs=k+1
auto litGeq = registry_->getOrCreatePolynomialAtom(xPoly, Relation::Geq, mpq_class(k + 1), TheoryId::NIA);

return TheoryLemma{{litLeq, litGeq}};
```

**关键验证**：`assertLit` 中已有 `diff = poly - rhs` 转换：
- `x <= k` → `x - k <= 0` ✓
- `x >= k+1` → `x - (k+1) >= 0` ✓

### 2.5 去重（Duplicate Suppression）

使用 `TheoryLemmaDatabase::contains` 和 `insertIfNew`：

```cpp
TheoryLemma lemma{{litLeq, litGeq}};
if (lemmaDb.contains(lemma)) {
    // 已被发射过，尝试其他变量/k 或返回 Unknown
    return std::nullopt;
}
lemmaDb.insertIfNew(lemma);
return lemma;
```

`TheoryLemmaDatabase::computeKey` 基于 clause 中所有 lit 的 hash，天然去重。

### 2.6 正确性保证

- `x ≤ k ∨ x ≥ k+1` 对所有整数 x 全局成立（与当前赋值无关）
- 两个分支都是多项式原子，通过 `getOrCreatePolynomialAtom` 注册为真实 SAT var
- SAT 求解器会 explore 两个分支，每个分支被 NiaSolver 的 domain reasoners 处理

## 接入点

在 `NiaSolver::buildBranchLemma` 中实现，取代 `return std::nullopt`。

## 测试

### 单元测试
- `buildBranchLemma` 在有界变量上生成分裂
- `buildBranchLemma` 在无界变量上选择 k=0
- 去重：同一 (var, k) 第二次返回 nullopt

### 端到端 SMT2
- `x² = 2`（无界，无整数根）→ 以前 Unknown，现在通过 branch 切空间 → unsat
- `x² = 50`（已有 SquareBound UNSAT，但可用于验证 SquareBound 优先）
- `x*y = 6, x+y = 5`（有界，已通）
- `x² + y² = 3`（无界多变量，需要 branch）→ unsat
