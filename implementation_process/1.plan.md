下面给你一套**可以长期演化成研究型 SMT/OMT 求解器平台**的方案。核心目标不是“写一个能跑的 solver demo”，而是做成：

**SOMTParser 前端 + 严格 typed DAG/rewriter + CDCL(T)/MCSAT 双内核 + 非线性 exact/hybrid 子系统 + local search 建议器 + proof/certificate/logging + learning-friendly API。**

SOMTParser 已经是 solver-independent 的 C++17 SMT/OMT 前端，支持 SMT-LIB/OMT 输入、typed DAG IR、type checking、traversal、rewriting 和 kind-based dispatch，这正好适合作为你的输入层，而不是把解析逻辑重新写进 solver 内核。([GitHub][1]) 你前面上传的调研里也已经把方向判断说得很清楚：非线性实数的主线应是 CAD/CAlC/NLSAT/MCSAT 与 incremental linearization/interval/local search 的混合，同时 learning 模块只能做候选建议，不能越过 exact solver 的验证回路。

# 0. 总体定位

我建议这个 solver 不要一开始叫“非线性求解器”，而要按 **SMT platform** 设计：

```text
SOMTParser
  ↓
Frontend Adapter
  ↓
Core IR / Rewriter / Preprocessor
  ↓
SAT/CDCL Layer  ───── Theory Manager ───── Proof Manager
  ↓                    ↓
Boolean search       Theory solvers
                     ├── EUF
                     ├── LRA / LIA
                     ├── BV / FP
                     ├── NRA exact: CAlC/CAC
                     ├── NRA MCSAT/NLSAT
                     ├── NIA hybrid
                     ├── Interval / ICP
                     ├── Incremental Linearization
                     ├── Local Search Advisor
                     └── OMT Layer
  ↓
Result Validator / Model Builder / Certificate Checker / Trace Exporter
```

核心设计原则：

1. **SAT/UNSAT 必须可验证。**
   `sat` 必须能通过独立 model validator 复查所有 asserted formula；`unsat` 至少要有 proof skeleton，最终逐步走向 Alethe/LFSC/LRAT/theory certificate。

2. **local search 只能给候选，不能给结论。**
   它可以给初始模型、branching value、repair move、restart seed，但不能直接返回 `sat`，除非 model validator 复查通过。

3. **learning 模块只能影响启发式，不能影响语义。**
   学习模块输出 `proposal`，核心 solver 负责 `check / accept / reject / fallback`。

4. **非线性实数以 exact 为主干。**
   CAlC/CAC 是第一主线；MCSAT/NLSAT 是第二主线；incremental linearization 与 interval 是加速层。cvc5 的公开系统描述就是把 CAlC 与 incremental linearization 放进 abstraction-refinement loop，并强调它仍是完整代数决策过程，可与 quantifier、integer variables 和其他 theories 结合。([gereon-kremer.de][2])

5. **非线性整数不要假装全判定。**
   QF_NIA 要做 hybrid：local search + MCSAT value proposal + branch-and-bound + bit-blasting fallback + bounded exact subroutine + unknown reason。NIA 的工程目标是强、稳、可解释，不是承诺不可能的全局完整性。

# 1. 外部 API：先设计成平台，不要设计成单算法入口

你需要两层 API：**用户 API** 和 **研究 API**。

## 1.1 用户 API

风格参考 Z3/cvc5，但保持你自己的简洁性：

```cpp
Solver solver;
solver.setLogic("QF_NRA");
solver.setOption("produce-models", true);
solver.setOption("produce-proofs", true);
solver.setOption("nl.engine", "cdcac");
solver.setOption("nl.local-search", true);

Sort real = solver.realSort();
Term x = solver.mkConst(real, "x");
Term y = solver.mkConst(real, "y");

Term f = solver.mkLeq(
    solver.mkAdd(solver.mkMul(x, x), solver.mkMul(y, y)),
    solver.mkReal(1)

solver.assertFormula(f);

Result r = solver.checkSat();

if (r.isSat()) {
    Model m = solver.getModel();
}
if (r.isUnsat()) {

}
```

# 2. 内部 IR：不要直接用 SOMTParser 节点做 reasoning

SOMTParser 负责 parse/typecheck/前端 DAG。solver 内部还需要自己的 **Reasoning IR**。

原因是：前端 IR 面向表达式结构；求解器 IR 面向：

* literal 管理；
* theory atom indexing；
* proof id；
* scope level；
* polynomial cache；
* abstraction literal；
* explanation dependency；
* e-graph/equality；
* model value；
* certificate object。

建议：

```text
SOMTParser::Node
  ↓ FrontendAdapter
CoreExpr
  ↓ Normalize/Rewriter
NormalizedExpr
  ↓ Atomizer
TheoryAtom + BooleanLiteral
  ↓ SAT Abstraction
ClauseDatabase
```

## 2.1 CoreExpr

```cpp
using ExprId = uint32_t;

struct Expr {
    Kind kind;
    SortId sort;
    SmallVector<ExprId, 4> children;
    Payload payload;          // symbol, rational, bv value, etc.
    Hash hash;
};
```

必须有：

```cpp
class ExprManager {
public:
    ExprId mk(Kind k, SortId s, Span<ExprId> children, Payload p);
    ExprId intern(...);       // hash-consing
    Expr get(ExprId id) const;
};
```

## 2.2 Literal / Atom 分离

```cpp
struct BoolLit {
    uint32_t var;
    bool sign;
};

struct TheoryAtom {
    AtomId id;
    TheoryId theory;
    ExprId original;
    AtomKind kind;

    // NRA example:
    PolyId lhs;
    Relation rel;   // =, !=, <, <=, >, >=
    Rational rhs;
};
```

这样 `x*x + y*y <= 1` 会被转成：

```text
Atom:
  theory = NRA
  poly = x^2 + y^2 - 1
  rel = <= 0
SAT literal:
  b_17 ↔ atom_42
```

## 2.3 Rewriter 必须 DAG-safe

你之前已经有 Rewriter 记忆：DAG-safe bottom-up rewrite + memo + optional fixpoint。这里继续沿用：

```cpp
class Rewriter {
public:
    ExprId rewrite(ExprId root, RewriteContext& ctx);
};
```

基础 rewrite：

```text
Boolean:
  not(not a) → a
  and true a → a
  or false a → a
  implies a b → or (not a) b
  xor normalize
  ite simplification

Arithmetic:
  (+ x 0) → x
  (* x 1) → x
  (* x 0) → 0
  (- x x) → 0
  normalize rational coefficients
  flatten associative ops
  sort commutative args
  merge monomials
  convert <=, >=, <, > to normalized poly relation

NRA:
  (* (+ ...) ...) → polynomial expansion when safe
  pow x k → monomial
  Horner form optionally for evaluation
  factor common rational coefficient
```

注意：**不要所有地方都强制展开 polynomial**。要同时保留：

```text
Expression DAG view: 适合 rewriting / proof / pretty print
Polynomial view: 适合 theory reasoning
Evaluation DAG view: 适合 local search incremental scoring
```

# 5. Theory Manager：支持 CDCL(T) 与 MCSAT 两套接口

你需要两类 theory solver interface。

## 5.1 CDCL(T) theory interface

```cpp
class TheorySolver {
public:
    virtual TheoryId id() const = 0;

    virtual void push() = 0;
    virtual void pop(uint32_t n) = 0;

    virtual void assertLit(SatLit lit, TheoryAtom atom) = 0;

    virtual TheoryCheckResult check(TheoryCheckLevel level) = 0;

    virtual std::vector<TheoryLemma> lemmas() = 0;
    virtual std::optional<TheoryConflict> conflict() = 0;

    virtual void collectModel(ModelBuilder&) = 0;
};
```

`TheoryCheckLevel`：

```text
PROPAGATION_ONLY
FULL_EFFORT
LAST_CALL
MODEL_VALIDATION
```

返回：

```cpp
enum class TheoryStatus {
    Sat,
    Conflict,
    LemmaGenerated,
    Unknown,
    NeedHigherEffort
};
```

## 5.2 MCSAT theory interface

MCSAT 要能操作 theory trail：

```cpp
struct TheoryAssignment {
    VarId var;
    AlgebraicValue value;
    DecisionLevel level;
    Reason reason;
};

class McsatSolver {
public:
    virtual McsatCheckResult checkPartial(const McsatTrail&) = 0;
    virtual std::vector<ValueProposal> proposeValues(const McsatSnapshot&) = 0;
    virtual Explanation explainConflict(const McsatConflict&) = 0;

```

MCSAT/NLSAT 的优势在于 theory-level decision，即直接给实变量/整数变量赋值；但工程上比传统 CDCL(T) 更难与多 theory 组合。cvc5 的系统描述也明确提到，MCSAT/NLSAT 与 theory-aware core 结合紧，但难以集成到 CDCL(T) theory combination；而 CAlC + incremental linearization 更自然嵌入传统 CDCL(T)。([gereon-kremer.de][2])

所以我建议：

```text
主路径：CDCL(T) + CAlC/CAC
研究路径：MCSAT/NLSAT engine
桥接层：LocalSearch / Learning 共享 proposal API
```

# 6. Preprocessing：必须做，但要 proof-aware

模块：

```text
Input normalization
Boolean simplification
ITE elimination
NNF/CNF optional
Theory atom extraction
Arithmetic normalization
Polynomial canonicalization
Bound inference
Variable elimination
Ackermannization for UF when needed
BV/FP lowering when needed
```

## 6.1 Arithmetic preprocessing

对每个 arithmetic atom：

```text
p(x1,...,xn) ⋈ q(x1,...,xn)
```

转成：

```text
p - q ⋈ 0
```

关系统一：

```text
= 0
!= 0
< 0
<= 0
> 0
>= 0
```

建议内部只保留：

```text
= 0
!= 0
< 0
<= 0
```

因为 `>` 和 `>=` 可以翻转符号。

## 6.2 Nonlinear term abstraction

incremental linearization 需要把 nonlinear term 抽象成 fresh variable：

```text
z_xy = x * y
z_x2 = x * x
```

然后先交给 linear solver：

```text
linear abstraction:
  z_xy, z_x2 as free real vars
```

如果 linear model 违反乘法语义，再加 lemma。

cvc5 的公开系统描述就是先使用 linear solver 找 abstraction model，再通过 nonlinear check/refinement 生成 lemmas 或 conflicts。([gereon-kremer.de][2])

# 8. NRA exact 主线：CAlC / CAC / CDCAC

这是核心。

## 8.1 输入形式

CAlC/CAC 子系统处理当前 Boolean assignment 下为真的 NRA atoms：

```text
A = { p_i(x) ⋈ 0 }
```

目标：

```text
判断 A 是否可满足；
若 sat，返回 algebraic model；
若 unsat，返回 conflict clause / covering certificate。
```

## 8.2 核心对象

```cpp
struct Cell {
    VarOrder order;
    uint32_t level;
    std::vector<SectionOrSector> coordinates;
    std::vector<PolyId> delineating_polys;
};

struct Covering {
    std::vector<Cell> cells;
    Box projected_box;
    Explanation explanation;
};

struct SamplePoint {
    std::vector<AlgebraicNumber> values;
};
```

`SectionOrSector`：

```text
sector: root_i < x < root_{i+1}
section: x = root_i
unbounded sector: -∞ < x < root_1, root_k < x < +∞
```

## 8.3 CAlC 工作流

```text
Input: conjunction of polynomial constraints under current SAT assignment

1. Select variable order.
2. Choose sample point incrementally.
3. Evaluate constraints at sample.
4. If all constraints satisfied:
     return SAT with algebraic model.
5. If some constraint violated:
     choose conflict set C.
6. Construct local cell around sample where C remains violated.
7. Project cell to lower dimension.
8. Add cell to covering.
9. If covering covers whole base:
     return UNSAT with explanation.
10. Else choose next uncovered sample and repeat.
```

伪代码：

```cpp
NraResult CdcacSolver::check(AtomSet atoms) {
    VarOrder ord = orderManager.choose(atoms);
    Covering cov(ord);

    while (true) {
        auto sample = sampleManager.nextUncovered(cov, atoms);
        if (!sample) {
            return unsat(cov.toConflictClause(), cov.toCertificate());
        }

        EvalResult er = evaluator.evaluate(atoms, *sample);
        if (er.allSatisfied()) {
            return sat(Model::fromSample(*sample));
        }

        ConflictSet cs = conflictSelector.select(er);
        Cell cell = cellBuilder.construct(ord, *sample, cs);
        Certificate cert = cellCertifier.certify(cell, cs);

        cov.add(cell, cert);
        trace.onNraCellGenerated(cell, cs, cert);
    }
}
```

## 8.4 conflict selector

不要只选第一个 violated atom。要支持策略：

```text
first violated
minimum degree
maximum degree
smallest projection cost
largest excluded interval
strict constraint first
equality first
learned selector

```

2026 年已有工作把 CDCAC 冲突泛化中的 set-covering 问题形式化为关键优化任务，说明这里是很好的研究切入点。([arXiv][5])

## 8.7 Correctness invariant

CAlC/CAC 每次生成 cell 必须满足：

```text
对于 cell 中所有点，至少有一个当前 assignment 要求为真的 atom 被违反。
```

因此该 cell 可以安全排除。

每个 cell certificate 至少包括：

```text
sample point
conflict atoms
projection polynomials
root ordering
sign-invariance evidence
violated relation evidence
lifting chain
```

# 9. Incremental Linearization：作为 CDCL(T) 内的轻量第一攻击

## 9.1 抽象

例如：

```text
x*y + z <= 3
```

抽象：

```text
m_xy + z <= 3
```

并注册：

```text
m_xy = x*y
```

## 9.2 Lemma schema

基础 lemma：

```text
x > 0 ∧ y > 0  ⇒  x*y > 0
x > 0 ∧ y < 0  ⇒  x*y < 0
x = a ∧ y = b  ⇒  m_xy = a*b
x ∈ [l,u], y ∈ [p,q] ⇒ m_xy ∈ interval_product([l,u],[p,q])
McCormick envelope for bounded variables
tangent/secant lemmas for convex pieces
```

cvc5 的 nonlinear solver 会在 linear abstraction model 之后检查是否满足 nonlinear semantics，并用 lemmas/refinements 去排除 spurious model；其系统描述中也说明 incremental linearization 和 coverings 在 abstraction-refinement loop 中协作。([gereon-kremer.de][2])

## 9.3 与 CAlC 的协作策略

```text
1. 先跑 LRA abstraction。
2. 如果 LRA unsat：直接返回 theory conflict。
3. 如果 LRA sat：
   3.1 检查 nonlinear terms 是否满足。
   3.2 若明显违反，生成 cheap linearization lemma。
   3.3 若 cheap lemmas 连续失败或进入循环，调用 CAlC。
4. CAlC 返回：
   - sat model：验证并返回；
   - unsat covering：生成 theory conflict clause。
```

```cpp
nl.inlin.enable_tangent
nl.inlin.fallback_to_cdcac
```

# 10. MCSAT / NLSAT 内核：作为第二条主线

Z3 的 arithmetic solver 已经经历重写，并强调 arithmetic 是 SMT solver 的核心组件。([Springer][6]) 你不需要照抄 Z3，但要借鉴它的 waterfall 思路：

```text
cheap propagation
linear relaxation
Groebner/equality simplification
incremental linearization
NLSAT/MCSAT endgame
```

## 10.1 MCSAT trail

```cpp
struct McsatTrailEntry {
    enum Kind { BoolAssign, TheoryAssign };
    VarOrLit target;
    Value value;
    DecisionLevel level;
    Reason reason;
};
```

## 10.2 Value selection

这是 MCSAT 的核心瓶颈。要支持：

```text
rational sample
algebraic sample
interval midpoint
root-adjacent sample
local-search suggested value
feasible-set jumping value
learned value proposal
```

2025 年 NIA local-search + MCSAT 工作明确把 local search 找到的 assignment values 用来引导 MCSAT，并提出 accelerated hill-climbing 与 feasible-sets jumping。([arXiv][7])

## 10.3 Clause-level feasible set

NLSAT 改进方向也值得直接吸收：不是只看 literal，而是看 clause-level feasible set；已有工作提出 clause-level feasible-set look-ahead、clause-level propagation，并把它集成进 branching heuristic。([arXiv][8])

接口：

```cpp
struct FeasibleSet {
    ClauseId clause;
    VarId variable;
    IntervalSet allowed_values;
};

class ClauseLevelAnalyzer {
public:
    std::vector<FeasibleSet> analyze(const McsatSnapshot&);
```

# 11. Local Search：做成 Advisor，不做成 Solver

local search 对 sat-heavy 实例非常重要。NRA local search 已经被用于完整 nonlinear real arithmetic 的 sat 实例搜索，并提出 incremental variable score、临时放松 equality constraints、候选 moves 和 look-ahead 等机制。([arXiv][9])

## 11.1 Local search state

```cpp
struct LSAssignment {
    std::unordered_map<VarId, NumericValue> values;
};

struct ConstraintScore {
    AtomId atom;
    double violation;
    double weight;
};

struct LSState {
    LSAssignment assignment;
    double total_cost;
    std::vector<ConstraintScore> scores;
};
```

## 11.2 Cost function

对 atom：

```text
p = 0      cost = |p|
p != 0     cost = 0 if |p| > eps else penalty
p <= 0     cost = max(0, p)
p < 0      cost = max(0, p + margin)
```

总成本：

```text
cost = Σ weight_i * violation_i
```

但注意 exact solver 不信 `eps`，它只把 local search 结果当 candidate。

## 11.3 Moves

NRA：

```text
single variable rational move
root move: solve p as univariate in x
gradient-like move
critical move from most violated atom
random restart
interval midpoint move
look-ahead move
equality relaxation move
```

NIA：

```text
x := x + 1
x := x - 1
x := sampled integer
x := nearest root rounded
feasible-set jumping
accelerated hill-climbing
large neighborhood move
```

## 11.4 与主 solver 的交互

local search 输出：

```cpp
struct LocalSearchProposal {
    enum Kind {
        FullModelCandidate,
        PartialAssignment,
        BranchValue,
        VariablePriority,
        RestartSeed
    };

    Assignment assignment;
    double cost;
};
```

主 solver 处理：

```text
FullModelCandidate:
  → exact model validator
  → pass: sat
  → fail: use as warm-start only

BranchValue:
  → MCSAT value decision candidate

PartialAssignment:
  → CDCL polarity / phase saving / NRA sample point hint

VariablePriority:
  → branching heuristic bias
```

# 12. NIA hybrid：不要追求单一算法

QF_NIA 的平台设计：

```text
NIA Engine
  ├── normalization / polynomial integer view
  ├── bound inference
  ├── local search
  ├── LIA relaxation
  ├── branch-and-bound
  ├── modular reasoning
  ├── bit-blasting for bounded variables
  ├── MCSAT value proposal
  ├── Groebner over Z/Q for equalities
  └── fallback unknown with reason
```

## 12.1 NIA workflow

```text
1. Normalize integer polynomial atoms.
2. Infer bounds from linear and simple nonlinear constraints.
3. Run local search for candidate integer model.
4. If candidate found, exact integer model validator checks.
5. Run LIA relaxation.
6. If LIA unsat, return conflict.
7. If bounded enough, bit-blast selected variables.
8. Else branch-and-bound on selected variable.
9. Use MCSAT/local-search proposals for value selection.
10. If resource exhausted, return unknown with reason.
```

## 12.2 Bounded bit-blasting

你之前 BLAN 的经验可以放这里：

```text
integer x with bound [l,u]
  → choose bit-width W
  → encode x = offset + signed/unsigned bit-vector
  → polynomial multiplication as BV/integer circuit
  → SAT solve
  → model decode
  → exact validate
```

注意：

```text
bit-blasting result sat：必须 decode 后在 integer semantics 下验证；
bit-blasting result unsat：只对当前 finite box unsat，除非 box 覆盖所有可能值。
```

## 12.3 Modular filtering

对 NIA equality：

```text
p(x) = 0 over Z
```

可加 cheap modular constraint：

```text
p(x) ≡ 0 mod m
```

用于剪枝，但不能单独证明原问题 sat。可以证明 unsat 的情况：

```text
∀x mod m, p(x) ≠ 0 mod m
⇒ p(x)=0 over Z impossible
```

## 12.4 Groebner

用途：

```text
integer/real polynomial equalities simplification
derive implied equalities
detect inconsistent equalities over Q/R
reduce polynomial atoms modulo equality ideal
```

但 Groebner over rationals 的 unsat 结论不总是直接等价于 integer unsat/sat，需要 theory-specific 解释。

# 13. Interval / ICP：作为快速剪枝层

ICP 不作为最终 exact NRA 判定主线，但非常有用：

```text
bounded NRA
transcendental constraints
local search warm-start
pre-CAlC pruning
NIA bound tightening
OMT bound narrowing
```

## 13.1 Interval domain

```cpp
struct Interval {
    Rational lower;
    Rational upper;
    BoundType lower_type; // open/closed
    BoundType upper_type;

```

支持：

```text
interval add/mul/div/pow
interval polynomial evaluation
contractor for p = 0
contractor for p <= 0
branch box
prune box
```

## 13.2 ICP result semantics

```text
box eliminated:
  safe if interval proof shows no solution.

small box remains:
  candidate only, not sat.

δ-sat:
```

如果支持 transcendental，可以单独给：


否则不要混淆 exact SMT 和 δ-SMT。

# 14. OMT 层：建立在 SMT core 之上

你后续研究 OMT，这个必须预留。

## 14.1 API

```cpp
Optimize opt;
Handle h = opt.minimize(term);
Result r = opt.check();
OptResult v = opt.getValue(h);
```

## 14.2 单目标 NRA OMT

流程：

```text
1. Check feasibility.
2. If unsat: return unsat.
3. Obtain feasible model with objective value c.
4. Add bound obj < c, repeat improvement.
5. When no better model:
   - exact attained optimum
   - non-attained infimum
   - unbounded
```

你自己的 CDCL(OCAC) / GOMT 语义可以接进来。

## 14.3 OMT certificate

必须区分：

```text
optimum attained:
  model M, obj(M)=v, proof no obj<v

infimum not attained:
  no model at v, for all eps>0 exists model obj<v+eps, proof no obj<v

unbounded:
  for every bound B exists model obj<B
```

内部结果类型不要只有 `OptSat`：

```cpp
enum class OptimumKind {
    Attained,
    InfimumNotAttained,
    Unbounded,
    Unknown
};
```

# 15. Proof / Certificate：从 day 0 设计，不然后面补不上

cvc5 支持 proof output，Alethe 是基于 SMT-LIB 的灵活 proof format，包含 coarse/fine-grained steps，并已有 Isabelle/HOL、Coq/SMTCoq、Rust checker 等检查路径。([cvc5][10])

建议分三层：

```text
Level 0: internal proof skeleton
Level 1: independently checkable certificates for theory lemmas
Level 2: export Alethe/LFSC/LRAT-compatible proof
```

## 15.1 SAT proof

用 CaDiCaL：

```text
SAT clause proof:
  DRAT / LRAT / FRAT / VeriPB
```

## 15.2 Theory lemma proof obligations

每条 theory lemma：

```cpp
struct TheoryLemma {
    Clause clause;
    TheoryId theory;
    ProofObligation obligation;
};
```

例如：

```text
LRA lemma:
  Farkas coefficients

NRA linearization lemma:
  algebraic inequality proof / interval proof / sign proof

CAlC conflict:
  cell certificate + sign-invariance + covering proof

NIA bounded bit-blast:
  encoding correctness + SAT proof for finite abstraction
```

## 15.3 NRA certificate

CAlC unsat certificate：

```text
1. List cells.
2. For each cell:
   - sample point
   - conflict atom
   - projection polynomial set
   - root isolation data
   - sign-invariance proof
   - violated relation proof
3. Covering proof:
   - cells cover all possible assignments under current projection.
4. Conflict clause:
   - not all current assignment literals can hold.
```

## 15.4 Model validator

`SAT` 最低要求：

```cpp
class ModelValidator {
public:
    ValidationResult validate(
        const AssertionSet& assertions,
        const Model& model
    );
};
```

对每类值精确验证：

```text
Bool: exact
Int: exact mpz
Real rational: exact mpq
Real algebraic: signOf(poly) exact
BV: bit-precise
FP: IEEE exact or bit-blasted exact
```

# 17. Strategy / Portfolio 层

不要把策略写死在 solver 里。做成 tactic pipeline：

```cpp
Tactic t =
    simplify()
  | splitClauses()
  | tryFor(localSearch(), Budget::conflicts(1000))
  | tryFor(incrementalLinearization(), Budget::lemmas(200))
  | cdcac()
  | fallback(mcsat());
```

配置：

```yaml
logic: QF_NRA
pipeline:
  - rewrite
  - lra_abstraction
  - local_search:
      mode: advisor
      budget_conflicts: 500
  - incremental_linearization:
      max_rounds: 200
  - cdcac:
      variable_order: learned_or_gmods
      projection: lazard
  - mcsat:
      enabled: fallback
```

学习策略调度可以只改 pipeline 参数，不碰 soundness。

```text
1. User API:
```

# 23. 我建议的主线选择

最终建议非常明确：

```text
主干：
  CDCL(T) + CAlC/CAC + incremental linearization

第二内核：
  MCSAT/NLSAT with clause-level feasible sets

加速层：
  local search + interval + cheap linear lemmas

NIA：
  local search + MCSAT value proposal + LIA relaxation + branch-and-bound + bounded bit-blasting

正确性：
  model validator + proof skeleton + theory certificates

学习接口：
  advisor/proposal/trace/replay 四件套
```

不要一开始把所有算法都写成并列模块。应该形成一个优先级：

```text
必须正确：
  Core IR / SAT / LRA / Polynomial / CAlC / Validator

必须可扩展：
  TheoryManager / Advisor / Trace / Strategy

必须可研究：
  variable order / conflict generalization / value selection / proof certificate

可以后置：
  full MCSAT
  full NIA
  OMT
  quantifiers
  transcendental
  complete proof export
```

这样做的好处是：你不是在写一个“论文原型”，而是在搭一个长期能承载 **非线性 SMT + OMT + learning-guided solving + proof-producing solving** 的平台。
