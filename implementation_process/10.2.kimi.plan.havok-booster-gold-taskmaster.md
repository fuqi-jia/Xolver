# 全面 NIA 测试套件 — 发现 Bug 计划

## 目标

设计并实施一套覆盖 NIA-Core 所有引擎和边界的测试，正确发现当前实现中隐藏的 bug。重点测试单元测试（直接调用引擎）和端到端测试（通过 Solver API）两条路径。

## 已确认的关键 Bug

### Bug 1: LinearNiaDomainReasoner — 负系数整数除法错误
**位置**: `src/theory/arith/nia/LinearNiaDomainReasoner.cpp` 第 70-91 行
**根因**: `a < 0` 时 `Leq` 和 `Geq` 分支的 ceiling/floor 计算未正确处理 GMP `mpz_class` 向零截断的除法语义。
**示例**:
- `-3x + 5 <= 0` → 正确: `x >= 2`（整数），代码可能给出 `x >= 1`
- `-3x + 5 >= 0` → 正确: `x <= 1`，代码可能给出 `x <= 0`
**现有覆盖**: 无。所有现有测试使用正系数。

### Bug 2: LibPolyKernel::variables() — 多变量多项式漏变量
**位置**: `src/theory/arith/poly/LibPolyKernel.cpp` 第 92-113 行
**根因**: 只检查 `poly::main_variable(p) == var`，对 `x*y`、`x+y^2` 等多变量多项式只返回一个变量。
**影响**: `BoundedNiaSolver` 漏枚举变量，`collectVars` 不完整，多变量问题无法求解。
**现有覆盖**: 无。所有现有测试是单变量。

### Bug 3: UnivariateIntegerReasoner — incomplete root enumeration incorrectly marked Complete
**位置**: `src/theory/arith/nia/UnivariateIntegerReasoner.cpp` 第 64-76 行
**根因**: `a0 == 0` 时只测试 `[-10, 10]`，但返回 `status == Complete`。高次零常数项多项式如 `x*(x-11) = 0`（根 `{0, 11}`）会被错误判定为仅 `{0}` 且 Complete。
**现有覆盖**: 无。

## 测试类别设计

### 类别 A: LinearNiaDomainReasoner 边界测试（直接单元测试）
新建 `test_linear_nia_domain.cpp`，直接构造 `LinearNiaDomainReasoner` + `DomainStore`，测试以下场景：

1. **正系数 Leq**: `2x + 5 <= 0` → `x <= -3`
2. **正系数 Geq**: `2x - 5 >= 0` → `x >= 3`
3. **负系数 Leq**: `-3x + 5 <= 0` → `x >= 2` ❌ 预计失败（Bug 1）
4. **负系数 Geq**: `-3x + 5 >= 0` → `x <= 1` ❌ 预计失败（Bug 1）
5. **负系数 Eq**: `-2x + 4 = 0` → `x = 2`
6. **负系数 Neq**: `-2x + 4 != 0` → `x != 2`
7. **Bound 冲突**: `2x + 5 <= 0` + `2x - 1 >= 0` → empty domain
8. **大系数**: `1000000x + 999999 <= 0` → `x <= -1`

### 类别 B: DomainStore 组合推理测试（直接单元测试）
新建 `test_domain_store.cpp`，直接测试 `DomainStore`：

1. **Bounds 冲突**: `x >= 5` + `x <= 3` → empty domain
2. **Finite set + bounds 冲突**: `x ∈ {1,2,3}` + `x >= 4` → empty domain
3. **Exclusions + bounds 冲突**: `x ≠ 0,1,2` + `0 <= x <= 2` → empty domain
4. **Finite set 交集**: `x ∈ {1,2,3}` + `x ∈ {2,3,4}` → `{2,3}`
5. **重复 bound 聚合 reason**: `x >= 0` (reason A) + `x >= 3` (reason B) → lower=3, reasons={B}
6. **相同 bound 多 reason**: `x >= 5` (reason A) + `x >= 5` (reason B) → lower=5, reasons={A,B}
7. **Conflict clause 完整性**: 验证 `buildEmptyDomainConflict()` 返回的 clause 包含所有相关 reason 的否定

### 类别 C: UnivariateIntegerReasoner 边界测试
扩展 `test_nia_core.cpp` 或新建 `test_univariate_reasoner.cpp`：

1. **RRT 完整根**: `x^2 - 5x + 6 = 0` → roots `{2, 3}`
2. **RRT 非零常数项高次**: `x^3 - 1331 = 0` → root `{11}`
3. **零常数项一次**: `2x = 0` → root `{0}`
4. **零常数项高次越界根**: `x*(x-11) = 0` → roots `{0, 11}` ❌ 预计失败（Bug 3）
5. **Neq 排除根**: `x^2 - 4 != 0` → exclusions `{2, -2}`
6. **Neq 零多项式**: `0 != 0` → UNSAT
7. **大除数截断**: `x - p# = 0` 其中 p# 是前 100 个素数的乘积（除数超过 1000）→ Incomplete，无结论
8. **非首一多项式**: `2x^2 - 8 = 0` → roots `{2, -2}`

### 类别 D: AlgebraicIntegerReasoner 边界测试
扩展 `test_nia_core.cpp`：

1. **Square rule UNSAT**: `x^2 + 1 <= 0` → UNSAT
2. **Square rule domain**: `x^2 <= 0` → `x = 0`
3. **GCD conflict 非线性**: `4x^2 - 2 = 0` → UNSAT
4. **GCD conflict 多变量**: `2x^2 + 4yz - 3 = 0` → UNSAT
5. **GCD no conflict 多变量**: `6x + 9y = 15` → NoChange / continue
6. **Modular 2-var**: `x^2 + y^2 = 3` → UNSAT（mod 4: 平方只能是 0 或 1，和最大为 2）

### 类别 E: 多变量 Bounded 枚举测试
扩展 `test_nia_core.cpp`：

1. **2D SAT**: `0<=x<=3, 0<=y<=3, x*y=6` → sat (x=2,y=3 或 x=3,y=2) ❌ 预计失败/Unknown（Bug 2）
2. **2D UNSAT**: `0<=x<=2, 0<=y<=2, x*y=6` → unsat
3. **2D 线性+非线性 SAT**: `x+y=5, x*y=6` → sat (x=2,y=3 或 x=3,y=2)
4. **3D SAT**: `0<=x<=2, 0<=y<=2, 0<=z<=2, x+y+z=5` → sat (x=1,y=2,z=2 等)

### 类别 F: 严格不等式 End-to-End
扩展 `test_nia.cpp`：

1. **Lt 转换**: `(assert (< x 5))` → `x <= 4`，SAT（如 x=4）
2. **Gt 转换**: `(assert (> x 0))` → `x >= 1`，SAT（如 x=1）
3. **Lt + Gt 直接冲突**: `(assert (< x 3))` + `(assert (> x 3))`
   - normalize: `x <= 2` + `x >= 4` → DomainStore 直接 empty domain → UNSAT
4. **严格非线性**: `(assert (< (* x x) 0))` → UNSAT

### 类别 G: 布尔 + NIA 混合（CDCL(T) 交互）
扩展 `test_nia_core.cpp`：

1. **Bool + NIA SAT**: `(or p (= (* x x) 4))` → sat
2. **Bool guard NIA UNSAT**:
   ```smt2
   (assert (or (not p) (= (* x x) 2)))
   (assert p)
   ```
   → unsat（布尔分支强制启用不可满足的 NIA 约束）
3. **SAT 分支覆盖 NIA**: `(or (= (* x x) 4) (= (* x x) 2))` → sat（SAT 引擎应找到 x^2=4 分支）

### 类别 H: False literal effective relation 回归
扩展 `test_nia.cpp`：

1. **Leq false → Gt → >=1**:
   ```smt2
   (assert (not (<= x 0)))
   (assert (<= x 0))
   ```
   → unsat。如果错误把 `not (x<=0)` 变成 `x>=0`，就会错误 SAT with x=0。

### 类别 I: Dynamic polynomial atom route 回归
扩展 `test_nia_core.cpp`：

1. **Factor lemma dynamic atom**:
   ```smt2
   (assert (= (* x y) 0))
   (assert (not (= x 0)))
   (assert (not (= y 0)))
   ```
   → unsat。确保 `x*y=0` factor lemma 生成的新多项式原子是真正注册在 `TheoryAtomRegistry` 中的 SAT var，能回传到 `NiaSolver::assertLit`。

1. `nia_001_sat_x2_eq_4.smt2` — 基本 sat

3. `nia_003_sat_bounded_xy.smt2` — 2D 有界 sat

6. `nia_006_sat_bool_nia.smt2` — bool + nia

8. `nia_008_unsat_modular_2var.smt2` — 2-var 模运算
