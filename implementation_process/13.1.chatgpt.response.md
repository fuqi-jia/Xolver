建议选 **[4] Revise**，而且这版里有一个非常关键的正确性误判：**角点采样不是 over-approx，不能用来证明 definitely violated，也不能用来返回 UNSAT**。

计划里一方面说 “Interval evaluation 必须是 over-approx”，另一方面又说用角点采样取 min/max；还声称“角点采样可能 under-approx，但不会导致 false UNSAT”。这句话是错的。文件里明确写到对多变量多项式用角点采样，且把 `min > 0`、`max < 0`、`0∉[min,max]` 用作 violated 判断，这是 soundness 风险。

# 核心问题：角点采样会导致 false UNSAT

例子：

```text
p(x) = x^2 - 1
domain: x ∈ [-2, 2]
constraint: p <= 0
```

角点采样：

```text
p(-2)=3
p(2)=3
min=max=3
```

如果按计划：

```text
p <= 0 且 min > 0 -> definitely violated
```

就会返回 UNSAT。

但实际上：

```text
x = 0 时 p(0) = -1
```

约束是可满足的。因此这是 **false UNSAT**。

二维也一样：

```text
p(x,y)=x^2+y^2-1
x,y∈[-2,2]
```

四个角点都是正数，但中心 `(0,0)` 满足 `p<=0`。

所以：

```text
角点采样不是 over-approx；
角点采样只能作为 local-search / candidate finding / heuristic；
不能用于 UNSAT pruning；
不能用于 definitely violated。
```

---

# 正确方案：Phase 3 只做真正 over-approx interval arithmetic

不要用角点采样。你应该让 Kimi 改成：

```text
IntervalEvaluator 必须基于表达式结构做 interval arithmetic。
如果无法获得表达式结构或无法 soundly over-approx，返回 NoChange。
```

也就是说：

```text
能 soundly evalInterval(poly, domain) -> 用它判断 violated/satisfied；
不能 soundly evalInterval -> skip；
绝不采样后返回 conflict。
```

# 怎么实现才安全？

## 方案 1：基于表达式 DAG 实现 interval evaluator

这是最推荐的。

你前端有 CoreExpr / DAG。直接对 normalized polynomial 的原始表达式或 polynomial AST 做 interval evaluation：

```text
const        -> [c,c]
var x        -> domain(x)
add          -> interval add
sub          -> interval sub
mul          -> interval mul
neg          -> interval neg
pow(k)       -> interval pow
```

这样得到的区间是 over-approx。

例如：

```text
x ∈ [-2,2]
x^2 -> [0,4]
x^2 - 1 -> [-1,3]
```

然后对于 `x^2 - 1 <= 0`，因为 interval `[-1,3]` 不能证明 violated，所以返回 NoChange，正确。

---

## 方案 2：扩展 PolynomialKernel 暴露 term/AST 结构

如果你想基于 PolyId 做，可以扩展 kernel：

```cpp
struct PolyTerm {
    mpz_class coeff;
    std::vector<std::pair<std::string, int>> powers;
};

std::optional<std::vector<PolyTerm>>
PolynomialKernel::terms(PolyId p);
```

然后对每个 monomial 做 interval arithmetic：

```text
coeff * x^a * y^b * ...
```

每个 monomial 得到 interval，再求和。

注意：逐项 interval arithmetic 会宽，但 sound。

---

## 方案 3：只处理已经可分解的模式

最保守也可以：

```text
只处理：
  linear form
  single-variable polynomial
  sum-of-squares
  syntactic add/mul/pow 能分解的表达式

其他返回 NoChange。
```

不要为了覆盖更多而采样。

# 另一个问题：单变量也不能只采端点

计划里的单变量测试 `x³+1<=0, x∈[0,5]` 用端点没问题，因为 `x^3` 单调。但一般单变量多项式也不能只看端点：

```text
p(x) = x^2 - 1
x ∈ [-2,2]
p(-2)=3, p(2)=3
```

端点全违反，但中间满足。所以单变量也要做 interval arithmetic，不能只端点采样。

对 pow2 的 interval rule 能正确得到：

```text
x^2 -> [0,4]
x^2-1 -> [-1,3]
```

因此不能证明 violated。
