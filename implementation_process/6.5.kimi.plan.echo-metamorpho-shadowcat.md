# Plan: Trail-Based CDCL(T) Refactor (Final Revision)

## Goal
Replace the current scan-all-atoms CDCL(T) loop with a unified, trail-based theory integration using CaDiCaL's native `ExternalPropagator` (IPASIR-UP). All theories â€” LRA, LIA, NRA â€” use exactly one integration path. No legacy scan-based path remains.

## Architecture Overview

```
CaDiCaL Solver
  -> ExternalPropagator (our CadicalTheoryPropagator)
     -> notifies observed theory variable assignments
     -> notifies backtracks
     -> asks theory to check complete models
     -> asks theory to add external clauses
  -> TheoryAtomRegistry
     -> SatVar -> TheoryAtomRecord (O(1))
     -> canonical dynamic atom creation
  -> TheoryManager
     -> routes assignments to correct TheorySolver
     -> coordinates backtracking
     -> collects check results
  -> TheorySolver (abstract)
     -> LraSolver, LiaSolver, CdcacSolver
```

## Critical Design Decisions

1. **One integration path only.** No `useLegacy`, no `runLegacyCdclLoop`, no `Atomizer::atoms()` scan in the CDCL loop.
2. **CaDiCaL ExternalPropagator is the production backend.** The vendored CaDiCaL already exposes `connect_external_propagator`, `add_observed_var`, `notify_assignment`, `notify_backtrack`, `cb_check_found_model`, `cb_has_external_clause` / `cb_add_external_clause_lit`. We use these directly. **Implementation must match the exact names and signatures in `third_party/cadical/src/cadical.hpp`.**
3. **All theory atoms in `TheoryAtomRegistry`.** Parsed atoms are registered during `Atomizer::atomize()`. Dynamic atoms are registered via `getOrCreate...`. `TheoryBridge` looks up **only** by `SatVar`. Synthetic `ExprId` is diagnostic only, never used for routing.
4. **TheoryAtomRecord uses variant payload** for linear, polynomial, and future theory payloads.
5. **TheoryLemmaDatabase lives in `Solver::checkSatInternal`**, survives the entire check-sat call. Theory solvers query `contains()`; integration layer calls `insertIfNew()` only before adding a clause to CaDiCaL.
6. **Unknown logic returns `Unknown`.** No fallback registering multiple solvers.
7. **CdcacSolver implements the new interface.** If not fully migrated, `QF_NRA` returns `Unknown` during this refactor.
8. **`connectPropagator` must happen before any `addObservedVar`.** CaDiCaL requires an external propagator to be connected before variables can be observed.
9. **Dynamic atoms must be observed immediately upon creation.** `TheoryAtomRegistry::getOrCreateLinearBoundAtom` calls `sat_->addObservedVar(v)` right after creating the SatVar.
10. **No `return false` without a pending external clause.** If theory returns `Unknown` or a duplicate lemma, the propagator must terminate solving (not reject the model silently).

### 1.5 Model check (`cb_check_found_model`)
```cpp
bool CadicalTheoryPropagator::cb_check_found_model(const std::vector<int>& model) {
    if (abortWithUnknown_) {
        terminateSolve();
        return true; // allow SAT solver to exit with whatever status
    }

    auto tr = tm_.check(lemmaDb_);

    if (tr.kind == Consistent) return true;

    if (tr.kind == Conflict) {
        setPendingClause(tr.conflict.clause);
        return false;
    }

    if (tr.kind == Lemma) {
        if (lemmaDb_.insertIfNew(*tr.lemma)) {
            setPendingClause(*tr.lemma);
            return false;
        }
        // Duplicate lemma without new clause â†’ internal error
        abortWithUnknown_ = true;
        terminateSolve();
        return true;
    }

    // Unknown
    abortWithUnknown_ = true;
    terminateSolve();
    return true;
}
```

**Rule**: `return false` is only valid when a pending external clause is queued. If there is no clause to add, we must terminate and make `CadicalBackend::solve()` return `Unknown`.

### 1.6 External clause addition
```cpp
void CadicalTheoryPropagator::setPendingClause(const std::vector<SatLit>& lits) {
    pendingClause_ = lits;
    pendingClausePos_ = 0;
    hasPendingClause_ = true;
}

void CadicalTheoryPropagator::setPendingClause(const TheoryLemma& lemma) {
    setPendingClause(lemma.lits);
}

int CadicalTheoryPropagator::cb_has_external_clause() override {
    return hasPendingClause_ ? 1 : 0;
}

int CadicalTheoryPropagator::cb_add_external_clause_lit() override {
    if (!hasPendingClause_) return 0;
    if (pendingClausePos_ == pendingClause_.size()) {
        hasPendingClause_ = false;
        pendingClause_.clear();
        pendingClausePos_ = 0;
        return 0;
    }
    SatLit lit = pendingClause_[pendingClausePos_++];
    return lit.sign ? static_cast<int>(lit.var) : -static_cast<int>(lit.var);
}
```

Index-based buffer preserves clause order and avoids state confusion.

### 1.7 Termination
```cpp
void CadicalTheoryPropagator::terminateSolve() {
    backend_.requestTerminate();
}
```

`CadicalBackend::solve()` must give `terminateRequested_` priority over CaDiCaL's returned status:

```cpp
SolveResult CadicalBackend::solve() {
    terminateRequested_ = false;
    int res = solver_->solve();

    if (terminateRequested_) {
        terminateRequested_ = false;
        return SolveResult::Unknown;
    }

    return mapCadicalResult(res);
}
```

Even if CaDiCaL returns `SAT` after the propagator requested termination, the backend must override to `Unknown`. This preserves correctness when the theory solver signals `Unknown` or encounters a duplicate lemma without a new clause to add.

### 2.4 `Atomizer::atomizeRec()` registers theory atoms
For every theory atom detected during atomization:
```cpp
SatVar v = freshVar();
result = SatLit::positive(v);

bool isTheory = (e.kind == Kind::Eq || e.kind == Kind::Distinct ||
                 e.kind == Kind::Lt || e.kind == Kind::Leq ||
                 e.kind == Kind::Gt || e.kind == Kind::Geq);

if (isTheory) {
    LinearAtomPayload linearPayload;
    PolynomialAtomPayload polyPayload;

    if (extractLinearConstraint(eid, ir, linearPayload.lhs,
                                linearPayload.rhs, linearPayload.rel)) {
        TheoryId theory = detectLinearTheory(linearPayload.lhs);
        registry_->registerParsedTheoryAtom(
            v, eid, theory, linearPayload);
    }
    else if (extractPolynomialConstraint(eid, ir, polyPayload.poly,
                                         polyPayload.rhs, polyPayload.rel)) {
        registry_->registerParsedTheoryAtom(
            v, eid, TheoryId::NRA, polyPayload);
    }
    else {
        // Unsupported theory atom â€” mark and later force Unknown
        registry_->setUnsupportedTheorySeen();
    }
}
```

`Atomizer::atoms_` may remain for debugging but is **not used** in the CDCL loop.

### 2.6 Unsupported theory atom policy
If `extractLinearConstraint` and `extractPolynomialConstraint` both fail for a theory-looking atom (`Eq`, `Distinct`, `Lt`, `Leq`, `Gt`, `Geq`), the atom is **unsupported**.

`Atomizer::atomizeRec()` must mark the registry:
```cpp
void TheoryAtomRegistry::setUnsupportedTheorySeen() {
    unsupportedTheorySeen_ = true;
}
```

`checkSatInternal()` checks **after atomization** and before solving:
```cpp
if (registry.hasUnsupportedTheoryAtom()) {
    return Result::Unknown;
}
```

**Rationale:** An unregistered theory atom would be treated as a pure boolean variable by the SAT solver. The theory solver would never see it, yet a SAT model could assign it arbitrarily, leading to an incorrect `sat` answer. The safe policy is to abort with `Unknown` immediately.

### 2.7 Eq / Distinct routing
`Eq` and `Distinct` are **not** unconditionally arithmetic. They become arithmetic theory atoms **only if** a supported payload is successfully extracted:

```cpp
bool isArithmeticTheory = false;

if (extractLinearConstraint(eid, ir, linearPayload.lhs,
                            linearPayload.rhs, linearPayload.rel)) {
    isArithmeticTheory = true;
    // register as linear ...
}
else if (extractPolynomialConstraint(eid, ir, polyPayload.poly,
                                     polyPayload.rhs, polyPayload.rel)) {
    isArithmeticTheory = true;
    // register as polynomial ...
}

if (!isArithmeticTheory && (e.kind == Kind::Eq || e.kind == Kind::Distinct)) {
    // Non-arithmetic equality/disequality (Boolean, UF, etc.)
    // Route to future theory solver, or mark unsupported
    registry_->setUnsupportedTheorySeen();
}
```

Do not let non-arithmetic `Eq`/`Distinct` silently become pure SAT atoms.

### 7.2 `assertLit`
Store sizes **before** modification:
```cpp
void LiaSolver::assertLit(
    const TheoryAtomRecord& atom,
    bool value,
    int level,
    SatLit reason
) {
    ensureLevel(level);

    size_t oldActive = activeAtoms_.size();
    size_t oldDiseq  = disequalities_.size();

    const auto& payload = std::get<LinearAtomPayload>(atom.payload);
    int aux = manager_.getOrCreateAuxVar(gs_, payload.lhs, payload.rhs);

    if (isDisequality(payload.rel, value)) {
        disequalities_.push_back({aux, payload.lhs, payload.rhs, reason});
        trail_.push_back({level, oldActive, oldDiseq});
        return;
    }

    bool immediateConflict = manager_.assertBound(
        gs_, aux, payload.rel, value, reason, level);

    if (immediateConflict) {
        pendingConflict_ = PendingConflict{level, manager_.buildImmediateConflict(gs_)};
        return;
    }

    activeAtoms_.push_back({
        atom.exprId, aux, payload.rel, value,
        payload.lhs, payload.rhs, reason
    });
    trail_.push_back({level, oldActive, oldDiseq});
}
```

### 7.3 `backtrackToLevel`
```cpp
void LiaSolver::backtrackToLevel(int level) {
    while (!trail_.empty() && trail_.back().level > level) {
        activeAtoms_.resize(trail_.back().activeAtomsSize);
        disequalities_.resize(trail_.back().disequalitiesSize);
        trail_.pop_back();
    }

    if (pendingConflict_ && pendingConflict_->level > level) {
        pendingConflict_.reset();
    }

    gs_.backtrackToLevel(level);
}
```

### 7.4 `check(TheoryLemmaDatabase&)`
```cpp
TheoryCheckResult LiaSolver::check(TheoryLemmaDatabase& lemmaDb) {
    if (pendingConflict_) return mkConflict(pendingConflict_->conflict);

    auto r = gs_.check();
    if (r == Unsat) return mkConflict(manager_.translateConflict(gs_));
    if (r == Unknown) return unknown();

    auto dr = handleDisequalities(lemmaDb);
    if (dr.kind != Consistent) return dr;

    auto ir = checkIntegrality();
    if (ir.kind == Consistent) {
        if (validator_.validateLiaModel(...)) return consistent();
        return unknown();
    }

    // Fractional: cheap integer reasoning
    if (!activeAtoms_.empty()) {
        auto cr = integerReasoner_.run(activeAtoms_);
        if (cr) {
            if (cr->kind == Lemma && cr->lemmaOpt) {
                if (!lemmaDb.contains(*cr->lemmaOpt)) return *cr;
            } else {
                return *cr;
            }
        }
    }

    // Branch split
    if (ir.kind == Lemma && ir.lemmaOpt) {
        if (!lemmaDb.contains(*ir.lemmaOpt)) return ir;
    }

    return unknown();
}
```

## Phase 9: `CdcacSolver` Migration

`CdcacSolver` must implement the new `TheorySolver` interface.

### 9.1 `assertLit`
```cpp
void CdcacSolver::assertLit(
    const TheoryAtomRecord& atom,
    bool value,
    int level,
    SatLit reason
) {
    ensureLevel(level);

    size_t oldActive = activeConstraints_.size();

    const auto& payload = std::get<PolynomialAtomPayload>(atom.payload);

    // Apply false-literal negation to get effective relation
    Relation effectiveRel = value ? payload.rel : negateRelation(payload.rel);

    activeConstraints_.push_back({
        payload.poly, effectiveRel, payload.rhs, reason
    });
    trail_.push_back({level, oldActive});
}
```

If CDCAC does not support disequality or strict relations, `check()` returns `Unknown`.

Phase 1 implementation policy:
- If the effective relation is unsupported by `CdcacSolver`, return `Unknown`.
- Do **not** silently drop the atom.
- Do **not** approximate strict as non-strict.
- Do **not** approximate disequality as two inequalities unless a sound split lemma is generated.

This prevents NRA from becoming unsound due to incomplete constraint handling.

### 9.2 `backtrackToLevel`
```cpp
void CdcacSolver::backtrackToLevel(int level) {
    while (!trail_.empty() && trail_.back().level > level) {
        activeConstraints_.resize(trail_.back().sizeBefore);
        trail_.pop_back();
    }
}
```

### 9.3 `check(TheoryLemmaDatabase&)`
Run existing CDCAC/NRA algorithm over `activeConstraints_`.
If not fully migrated, return `TheoryCheckResult::unknown()`.

### 9.4 Policy
- `QF_NRA` registers `CdcacSolver`.
- If `CdcacSolver::check()` returns `Unknown`, the overall solver returns `Unknown`.
- No legacy scan-based path.

## Phase 10: `Solver::checkSatInternal` Rewrite

```cpp
Result Solver::checkSatInternal() {
    if (!ir || ir->assertions().empty()) {
        return Result::Sat;
    }

    // Fresh SAT solver per check-sat call
    auto sat = std::make_unique<CadicalBackend>();

    // Fresh registry per SAT solver instance
    TheoryAtomRegistry registry;

    TheoryManager theoryManager;
    TheoryLemmaDatabase lemmaDb;

    // Register solvers by logic
    if (logic == "QF_LIA" || logic == "LIA") {
        auto lia = std::make_unique<LiaSolver>();
        lia->setRegistry(&registry);
        theoryManager.registerSolver(std::move(lia));
    } else if (logic == "QF_LRA" || logic == "LRA") {
        auto lra = std::make_unique<LraSolver>();
        lra->setRegistry(&registry);
        theoryManager.registerSolver(std::move(lra));
    } else if (logic == "QF_NRA" || logic == "NRA") {
        auto nra = std::make_unique<CdcacSolver>(createPolynomialKernel());
        theoryManager.registerSolver(std::move(nra));
    } else {
        return Result::Unknown;
    }

    // Connect propagator FIRST (required before addObservedVar)
    CadicalTheoryPropagator propagator(registry, theoryManager, lemmaDb, *sat);
    sat->connectPropagator(&propagator);

    // Atomizer registers parsed atoms into registry (which calls addObservedVar)
    Atomizer atomizer(*sat);
    registry.setContext(sat.get(), &atomizer);

    for (ExprId a : ir->assertions()) {
        SatLit lit = atomizer.atomize(a, *ir);
        sat->addClause({lit});
    }

    // Early abort if any theory atom could not be classified
    if (registry.hasUnsupportedTheoryAtom()) {
        return Result::Unknown;
    }

    auto result = sat->solve();

    if (result == SatSolver::SolveResult::Sat) return Result::Sat;
    if (result == SatSolver::SolveResult::Unsat) return Result::Unsat;
    return Result::Unknown;
}
```

**No explicit `while (true)` SAT-theory loop.** CaDiCaL's `ExternalPropagator` handles theory checks internally during `solve()`. If the vendored CaDiCaL does not support the required `ExternalPropagator` API, upgrade CaDiCaL rather than reintroducing the old loop.

## Phase 11: No-Op Lemma Guards

All theory lemma generators must skip:

1. **Tautology**: `conclusionLit == premiseLit` (produces `Â¬p âˆ¨ p`).
2. **Self-tightening**: tightened atom is canonical-equivalent to original (same `satVar`).
3. **Duplicate**: `lemmaDb.contains(candidate)` is true.
4. **Already satisfied**: deferred until `SatAssignmentView` is available in theory solver.

Correct tautology guard:
```cpp
if (conclusionLit == premiseLit) return std::nullopt; // Â¬p âˆ¨ p
```

## Risks and Mitigations

| Risk | Mitigation |
|------|------------|
| CaDiCaL ExternalPropagator integration complexity | Start with `cb_check_found_model` only; add `cb_propagate` later; exact API from vendored header |
| `CdcacSolver` migration breaks NRA tests | NRA returns `Unknown` until fully migrated; tests adjusted |
| Variant payload compilation overhead | `std::variant` is C++17 standard, no runtime cost for correct type |
| Registry lookup failure (SatVar not found) | Assert in debug, return `Unknown` in release |
| Duplicate/Unknown without pending clause | Propagator terminates solve; `CadicalBackend` returns `Unknown` |
