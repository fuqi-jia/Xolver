# Plan: IncrementalLinearizer V1 — McCormick + Square Cuts

## Goal

Implement `IncrementalLinearizer` as an **independent arithmetic component** (not inside NIA/NRA) that:

1. Abstracts nonlinear subterms (`x*y`, `x^2`) into auxiliary variables.
2. Generates **sound linear cuts** (McCormick envelope, square secant/tangent) from current bounds.
3. Returns **guarded dynamic linear lemmas** via `TheoryAtomRegistry::getOrCreateLinearBoundAtom`.
4. Routes NIA cuts → `TheoryId::LIA`, NRA cuts → `TheoryId::LRA`.

**V1 scope**: `x*y` and `x^2` only. No `x*y*z`, no `x^k` for `k>2`, no ICP, no local search, no MCSAT.

### Critical: Linear solver must be co-registered

Current `Solver.cpp` registers **only** NiaSolver for QF_NIA, only NraSolver for QF_NRA. If linearizer cuts route to `TheoryId::LIA/LRA`, no solver receives them.

**Fix**: Co-register LIA/LRA solver:

```cpp
// QF_NIA
auto polyKernel = createPolynomialKernel();
polyKernelRaw = polyKernel.get();
auto nia = std::make_unique<NiaSolver>(std::move(polyKernel));
nia->setRegistry(&registry);
theoryManager.registerSolver(std::move(nia));
auto lia = std::make_unique<LiaSolver>();
lia->setRegistry(&registry);
theoryManager.registerSolver(std::move(lia));  // NEW

// QF_NRA
auto polyKernel = createPolynomialKernel();
polyKernelRaw = polyKernel.get();
theoryManager.registerSolver(std::make_unique<NraSolver>(std::move(polyKernel)));
theoryManager.registerSolver(std::make_unique<LraSolver>());  // NEW
```

`TheoryManager` dispatches by `TheoryAtomRecord.theory`. LIA/LRA solvers silently ignore polynomial atoms; NIA/NRA solvers silently ignore linear atoms. No routing conflict.

### D5a: Bound mirror lemmas

Linearizer-generated McCormick cuts alone are **not sufficient** for LIA/LRA to derive conflict. LIA/LRA must also see the original active linear bounds.

**Strategy**: Mirror **effective active linear bounds** from NIA/NRA to LIA/LRA via dynamic lemmas.

For each original linear atom assigned by SAT:

```cpp
// 1. Compute effective relation
Relation effRel = value ? payload->rel : negateRelation(payload->rel);

// 2. Create a mirror linear atom for LIA/LRA
SatLit bLia = registry->getOrCreateLinearBoundAtom(
    b.lhs, effRel, b.rhs,
    TheoryId::LIA  // or TheoryId::LRA for NRA
);

// 3. Mirror lemma: originalLit -> B_LIA, i.e. ¬originalLit ∨ B_LIA
TheoryLemma mirrorLemma{{reason.negated(), bLia}};
```

**Important**: Mirror the **effective** constraint, not the raw syntactic atom. If the original atom is assigned `false`, the active constraint is its negation (e.g., `x>3` becomes `x>=4` in NIA). The mirror must reflect this effective relation.

**Who emits mirror lemmas?**

The NIA/NRA adapter scans active assignments and generates mirror lemmas alongside linearizer cuts.

**Routing clarification**:

```text
QF_NIA:
  Original polynomial atoms           -> NIA
  Original linear bound atoms         -> NIA
  Mirror of effective linear bounds   -> LIA (via ¬B ∨ B_LIA lemma)
  Linearizer-generated cuts           -> LIA

QF_NRA:
  Original polynomial atoms           -> NRA
  Original linear bound atoms         -> NRA
  Mirror of effective linear bounds   -> LRA (via ¬B ∨ B_LRA lemma)
  Linearizer-generated cuts           -> LRA
```

Without bound mirroring, tests like `0<=x,y<=10 and x*y>200` cannot be proved UNSAT, because LIA would be free to set `x=21,y=21`.

## Design Decisions

### D1: Aux var naming

Internal only, filtered from public model in P2:

```cpp
inline constexpr std::string_view NL_AUX_PREFIX = "__nl_aux_";
inline bool isInternalAuxVar(std::string_view name) {
    return name.substr(0, NL_AUX_PREFIX.size()) == NL_AUX_PREFIX;
}
```

### D2: `NonlinearTermKey` canonicalization

Product keys sort children to ensure `x*y == y*x`:

```cpp
struct NonlinearTermKey {
    NonlinearKind kind;
    std::vector<std::pair<std::string, int>> powers; // sorted by var name
    // Example:
    //   x*y  -> {("x",1),("y",1)}
    //   x^2  -> {("x",2)}
    bool operator==(const NonlinearTermKey&) const;
};
```

**Why not PolyId children?** `PolyId` is kernel-internal and may vary across polynomial reconstructions. Variable names from `terms()` are stable canonical identifiers.

### D2a: Strict relation normalization for NIA

For `owner == TheoryId::NIA`, strict linear atoms must be integer-normalized before registration:

```cpp
// Before registering via getOrCreateLinearBoundAtom:
if (owner == TheoryId::NIA) {
    // General form: lhs > rhs -> lhs >= rhs + 1
    if (rel == Relation::Gt) { rel = Relation::Geq; rhs += 1; }
    // General form: lhs < rhs -> lhs <= rhs - 1
    if (rel == Relation::Lt) { rel = Relation::Leq; rhs -= 1; }
}
// Then register with normalized rel/rhs
```

This ensures LIA solver receives non-strict integer relations it can handle.

### D2b: Aux variable sort

```cpp
SortKind auxSort = (owner == TheoryId::NIA) ? SortKind::Int : SortKind::Real;
```

NIA aux vars must be Int (for LIA strict normalization to be valid). NRA aux vars must be Real.

### D3: Dynamic atom routing

Cuts are registered as **linear atoms** (not polynomial atoms) because they are linear after abstraction:

```cpp
SatLit cutLit = registry->getOrCreateLinearBoundAtom(
    linearLhs, rel, rhs,
    owner == TheoryId::NIA ? TheoryId::LIA : TheoryId::LRA
);
```

This routes cut assertions to LIA/LRA solver, not back to NIA/NRA.

### D4: Lemma guarding

Every cut is a **guarded clause**:

```
¬nonlinear_reason ∨ ¬bound_reason_1 ∨ ... ∨ ¬bound_reason_n ∨ cut_lit
```

If bounds are unconditional (no reasons), omit bound guards.

### D5: Soundness boundary

Linearizer is a **relaxation** (over-approx). It can prove UNSAT (relaxation UNSAT → original UNSAT) but cannot prove SAT. It never returns `consistent()`.

**Important**: Linearization lemmas are **conservative extension lemmas** over fresh aux variables. They preserve satisfiability but are NOT ordinary theory-valid lemmas over arbitrary aux assignments. Therefore:

1. Final SAT must validate **original nonlinear constraints** (aux vars are internal).
2. ProofManager (P10) must treat them as definitional/extension lemmas.
3. Aux vars must never appear in public model.

### Step 3: `NonlinearTermAbstraction.h/.cpp`

**Responsibility**: Scan each monomial in `terms(poly)`, identify nonlinear subterms, create/reuse aux variables, build linearized polynomial.

```cpp
struct AbstractionResult {
    PolyId linearizedPoly;      // polynomial with aux vars replacing nonlinear monomials
    std::vector<AuxTerm> auxTerms;
    bool unsupported = false;
};

class NonlinearTermAbstraction {
public:
    explicit NonlinearTermAbstraction(PolynomialKernel& kernel);

    // Scan each monomial, replace nonlinear ones with aux vars
    AbstractionResult abstract(PolyId poly);

    // Get or create aux var for a canonical key
    AuxTerm getOrCreateAux(const NonlinearTermKey& key);

private:
    PolynomialKernel& kernel_;
    std::unordered_map<NonlinearTermKey, AuxTerm, NonlinearTermKeyHash> auxCache_;
    uint32_t nextAuxId_ = 0;
};
```

**Per-monomial detection rules**:

```
For each monomial term in terms(poly):
  if powers = {x:1, y:1}:        → Product, replace coeff*x*y by coeff*aux_xy
  if powers = {x:2}:             → Square, replace coeff*x^2 by coeff*aux_x2
  if powers is linear/constant:  → keep original
  otherwise:                     → V1 unsupported
```

Example:
```
x*y + z <= 5     →  aux_xy + z <= 5
x^2 + y^2 <= 1   →  aux_x2 + aux_y2 <= 1
3*x*y + 2*z <= 7 →  3*aux_xy + 2*z <= 7
```

### Step 4: `McCormickGenerator.h/.cpp`

**Responsibility**: Given `t = x*y` and **finite complete bounds**, generate 4 McCormick envelope cuts.

```cpp
class McCormickGenerator {
public:
    // Returns empty if any bound missing or incomplete.
    std::vector<LinearCut> generate(
        const AuxTerm& t,
        const std::string& x, const std::string& y,
        const BoundInfo& xBounds,
        const BoundInfo& yBounds,
        SatLit nonlinearReason);
};
```

**Precondition**: `xBounds.hasFiniteCompleteBounds() && yBounds.hasFiniteCompleteBounds()`.

**Cuts** (all rewritten to `... <= 0` form):

```
-t + lx*y + ly*x - lx*ly <= 0   (lower 1)
-t + ux*y + uy*x - ux*uy <= 0   (lower 2)
 t - ux*y - ly*x + ux*ly <= 0   (upper 1)
 t - lx*y - uy*x + lx*uy <= 0   (upper 2)
```

Where `lx,ux,ly,uy` are `mpq_class` bounds.

**Reasons** for each cut:

```cpp
reasons = {nonlinearReason}
         + xBounds.lowerReasons + xBounds.upperReasons
         + yBounds.lowerReasons + yBounds.upperReasons
```

### Step 5: `SquareCutGenerator.h/.cpp`

**Responsibility**: Given `s = x^2`, generate up to 3 cuts:

```cpp
class SquareCutGenerator {
public:
    std::vector<LinearCut> generate(
        const AuxTerm& s,
        const std::string& x,
        const BoundInfo& xBounds,
        SatLit nonlinearReason,
        const std::optional<mpq_class>& modelX, // for tangent point
        bool emitNonneg,
        bool emitSecant,
        bool emitTangent);
};
```

**Cut 1 — Nonnegativity** (always, no finite bounds required):

```
s >= 0
→ -s <= 0
```

Reasons: `{nonlinearReason}` only (no bound reasons).

**Cut 2 — Secant upper bound** (requires `xBounds.hasFiniteCompleteBounds()`):

```
s <= (l+u)*x - l*u
→ s - (l+u)*x + l*u <= 0
```

Reasons: `{nonlinearReason} + xBounds.lowerReasons + xBounds.upperReasons`.

**Cut 3 — Tangent lower bound** at point `a` (requires `xBounds.hasFiniteCompleteBounds()`):

```
s >= 2a*x - a^2
→ -s + 2a*x - a^2 <= 0
```

If `modelX` provided, use `a = *modelX`. Otherwise use midpoint `a = (l+u)/2`.

Reasons: `{nonlinearReason} + xBounds.lowerReasons + xBounds.upperReasons`.

### Step 6: `LinearizationCache.h/.cpp`

**Responsibility**: Prevent duplicate lemmas for identical (term, nonlinear reason, bounds) combinations.

```cpp
class LinearizationCache {
public:
    bool hasEmitted(const NonlinearTermKey& key,
                    SatLit nonlinearReason,
                    const std::vector<mpq_class>& boundValues,
                    int cutIndex);
    void markEmitted(const NonlinearTermKey& key,
                     SatLit nonlinearReason,
                     const std::vector<mpq_class>& boundValues,
                     int cutIndex);
    void clear();
private:
    std::unordered_set<CutCacheKey> emitted_;
};
```

**Cache key** includes: `kind`, `aux name`, **`nonlinearReason`**, `all bound values`, `cut index`.

**CRITICAL: Mark timing**. `markEmitted` must be called **only after** the lemma is actually enqueued in `pendingLinLemmas_` or returned to the propagator. Do NOT mark during generation, or cuts may be lost due to `maxLemmas` filtering.

Rationale: The same `x*y` under constraint `C1: x*y+z<=5` and `C2: x*y-w>=3` needs **separate guarded lemmas** (`¬C1 ∨ ...` vs `¬C2 ∨ ...`). If cache ignores `nonlinearReason`, the second constraint's guard is lost.

### Step 7: `IncrementalLinearizer.h/.cpp`

**Responsibility**: Orchestrate abstraction + cut generation + lemma assembly.

```cpp
class IncrementalLinearizer {
public:
    explicit IncrementalLinearizer(PolynomialKernel& kernel,
                                   TheoryAtomRegistry* registry);

    LinearizationResult run(
        const std::vector<NormalizedArithConstraint>& constraints,
        const BoundStore& bounds,
        TheoryId owner,
        const LinearizationConfig& config);

private:
    PolynomialKernel& kernel_;
    TheoryAtomRegistry* registry_;
    NonlinearTermAbstraction abstraction_;
    McCormickGenerator mcGen_;
    SquareCutGenerator sqGen_;
    LinearizationCache cache_;
};
```

**Config**:

```cpp
struct LinearizationConfig {
    bool emitAllMcCormick = true;   // V1: always emit all envelope cuts
    bool emitSquareSecant = true;
    bool emitSquareTangent = true;
    bool emitSquareNonneg = true;   // s = x^2 -> s >= 0 (no bounds needed)
    size_t maxLemmas = 10;
    size_t maxCutsPerTerm = 4;
};
```

Note: `emitAllMcCormick = true` because `run()` currently has no model parameter. Violated-cuts-only mode deferred to V1.5 when model is available.

**Run flow**:

```
1. For each constraint:
   a. Abstract nonlinear subterms → aux vars
   b. Build linearized polynomial p_L
   c. Generate abstraction lemma: ¬reason ∨ (p_L rel 0)
      (no bound guards — linearized constraint is definitional)

2. For each unique nonlinear term:
   a. Look up bounds from BoundStore
   b. If missing/incomplete bounds → skip envelope cuts; nonnegativity still OK
   c. Generate cuts via McCormickGenerator / SquareCutGenerator
   d. For each generated cut, build cut lemma: ¬reason ∨ ¬bound_reasons... ∨ cut
   e. If not in cache, add to result lemmas (respecting maxLemmas)

3. Register all cut lits via getOrCreateLinearBoundAtom
4. Return LinearizationResult with lemmas
5. Cache markEmitted: called by adapter AFTER lemma is enqueued in pending queue
```

**Lemma assembly — two kinds**:

```cpp
// A. Abstraction lemma: ¬C ∨ p_L rel 0
//    No bound guards. Linearized constraint is a definitional extension.
TheoryLemma buildAbstractionLemma(SatLit nonlinearReason, SatLit linearizedLit) {
    return TheoryLemma{{nonlinearReason.negated(), linearizedLit}};
}

// B. Cut/envelope lemma: ¬C ∨ ¬bound_reasons... ∨ cut
//    Bound guards required for bounds-dependent cuts.
TheoryLemma buildCutLemma(
    SatLit nonlinearReason,
    const std::vector<SatLit>& boundReasons,
    SatLit cutLit) {

    std::vector<SatLit> lits;
    lits.push_back(nonlinearReason.negated());
    for (auto r : boundReasons) lits.push_back(r.negated());
    lits.push_back(cutLit);
    return TheoryLemma{std::move(lits)};
}
```

### Step 8: `BoundStore` adapter

`DomainStore` (NIA) and `ReasonedBox` (interval) have different bound formats. Create a unified read-only interface:

```cpp
// src/theory/arith/linearizer/BoundStore.h
```cpp
struct BoundInfo {
    mpq_class lower;
    mpq_class upper;
    std::vector<SatLit> lowerReasons;
    std::vector<SatLit> upperReasons;

    bool hasLower = false;
    bool hasUpper = false;

    bool lowerIsGlobal = false;        // e.g. x^2 >= 0 is globally true
    bool upperIsGlobal = false;
    bool lowerReasonComplete = false;  // all contributing reasons are known
    bool upperReasonComplete = false;

    bool isFinite() const { return hasLower && hasUpper; }

    bool hasFiniteCompleteBounds() const {
        return isFinite() &&
               ((lowerIsGlobal || lowerReasonComplete) &&
                (upperIsGlobal || upperReasonComplete));
    }
};

class BoundStore {
public:
    virtual std::optional<BoundInfo> get(const std::string& var) const = 0;
};
```

**Guard policy for empty reasons**: Empty `reasons` does **not** mean unconditional. If a finite bound has no complete reason and is not marked global, either:
- Skip the cut, or
- Use a **coarse all-active guard** (all currently asserted SAT literals).

Never emit unguarded McCormick cuts with unverified bounds.

Adapters:
- `DomainStoreBoundStore` — wraps `DomainStore`
- `ReasonedBoxBoundStore` — wraps `ReasonedBox`

**Integration point**: `NiaSolver` holds adapter and pending lemma queue as **members** (not local variables):

```cpp
class NiaSolver : public TheorySolver {
    // ... existing members ...
    std::unique_ptr<NiaLinearizationAdapter> linAdapter_;
    std::deque<TheoryLemma> pendingLinLemmas_;
};
```

In `NiaSolver::check()`, after interval evaluation (step 9) and before bounded solver:

```cpp
// NiaSolver::check()
// ... steps 1-9 ...

// 9.5: Incremental linearization
if (pendingLinLemmas_.empty() && registry_ && linAdapter_) {
    auto lr = linAdapter_->run(normalized, domains_, lemmaDb);
    if (lr.status == LinearizationStatus::Lemma) {
        for (auto& lemma : lr.lemmas) {
            if (!lemmaDb.contains(lemma)) {
                lemmaDb.insertIfNew(lemma);
                pendingLinLemmas_.push_back(std::move(lemma));
                // Cache is marked as emitted ONLY after successful enqueue
                linAdapter_->markEmitted(lemma);
            }
        }
    }
}

if (!pendingLinLemmas_.empty()) {
    auto lemma = std::move(pendingLinLemmas_.front());
    pendingLinLemmas_.pop_front();
    return TheoryCheckResult::mkLemma(lemma);
}

// 10. Bounded complete solver
// ...
```

**CRITICAL: Cache mark timing**. `markEmitted` must only be called after the lemma enters `pendingLinLemmas_`. If `maxLemmas` truncates the result, un-enqueued cuts must NOT be marked, or they are lost forever.

**Routing rule**: In QF_NIA, original polynomial atoms route to NIA; original linear bounds route to NIA **and are mirrored to LIA** via `¬B ∨ B_LIA` lemmas. Only linearizer-generated cuts route directly to LIA. This gives LIA both the variable bounds and the relaxation cuts needed to derive conflict.

## Correctness Red Lines

```
1. No finite COMPLETE bounds → do not emit bounds-dependent envelope cuts.
   Abstraction lemmas and square nonnegativity remain allowed.
2. Bounds-dependent cuts must carry nonlinear_reason + bound_reasons as guards.
3. Dynamic atoms route to LIA (NIA) or LRA (NRA), never back to NIA/NRA.
4. Original linear bounds must be mirrored to LIA/LRA via ¬B ∨ B_LIA/LRA lemmas.
5. Never return SAT from Linearizer.
6. Never use sampling to prove UNSAT.
7. Aux vars (__nl_aux_*) filtered from public model.
8. Budget exhaustion → NoChange, not crash.
9. Cache prevents duplicate lemmas for identical (term, nonlinearReason, bounds).
10. If finite bounds have incomplete reasons and are not global, skip the cut.
```
