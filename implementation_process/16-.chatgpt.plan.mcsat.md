下面给你一版**终极版 MCSAT 实现 plan**。我会直接按“要做一个严肃 SMT 求解器”的标准设计，而不是临时补丁。目标理论是：

```text
EUF + LIA + LRA + NIA + NRA
```

并且要支持它们的组合，例如：

```text
QF_UFLIA
QF_UFLRA
QF_UFNIA
QF_UFNRA
QF_NIRA
QF_UFNIRA
```

核心判断先说清楚：**不要把 MCSAT 做成某个算术 solver 的附属模块**。MCSAT 应该成为 NLColver 的主求解框架，上面挂 EUF、linear arithmetic、nonlinear arithmetic、integer reasoning、model repair、lemma generation、dynamic atom registry。MCSAT 本质是把 CDCL 从“只赋布尔值”提升到“同时构造 theory model”，这也是原始 MCSAT 设计的核心思想；Jovanović、Barrett、de Moura 的 MCSAT 设计论文明确将其描述为对 CDCL-style SAT 求解器思想向 SMT 的推广，并作为整合模型构造型理论过程的共同框架。([德克萨斯大学奥斯汀分校计算机科学][1])

---

# 0. 总体目标

你最终要做的是这个架构：

```text
SOMTParser
  ↓
Typed Core IR / DAG
  ↓
Atomization + Term Registration
  ↓
MCSAT Core
  ├── Boolean CDCL layer
  ├── Theory-value trail
  ├── Dynamic atom registry
  ├── Explanation / Lemma manager
  ├── Model object
  ├── Plugin scheduler
  │
  ├── EUF Plugin
  ├── LRA Plugin
  ├── LIA Plugin
  ├── NRA Plugin
  ├── NIA Plugin
  └── Combination Layer
```

MCSAT 的核心不是简单替代 CDCL(T)，而是维护一条混合 trail：

```text
Boolean assignment:
  A := true
  B := false

Theory-value assignment:
  x := 3
  y := 1/2
  z := α   // algebraic real root
  f(a) := v1
  a = b
```

当当前 partial model 与某些 theory constraints 不兼容时，理论插件返回一个 **incompatibility / lemma**：

```text
¬l1 ∨ ¬l2 ∨ ... ∨ new_atom
```

或者：

```text
¬assignment_1 ∨ ¬assignment_2 ∨ ...
```

然后由统一的 CDCL/MCSAT 内核做 conflict analysis、backjump、learn。

# 1. 顶层原则

## 1.1 先支持 QF，再考虑 quantifier

本 plan 只覆盖：

```text
QF_UF
QF_LIA
QF_LRA
QF_NIA
QF_NRA
QF_UFLIA
QF_UFLRA
QF_UFNIA
QF_UFNRA
QF_NIRA
QF_UFNIRA
```

不做量词、不做数组、不做字符串、不做 bit-vector。

## 1.2 所有 SAT 必须 exact validate

每个 theory plugin 给出的 model 只能作为 candidate。最终返回 SAT 前必须执行：

```text
FullModelValidator
```

检查：

```text
1. 每个原始 asserted formula 为 true；
2. 所有 let / quantifier-free structure 正确展开；
3. EUF congruence closure 成立；
4. LRA/LIA/NRA/NIA constraints 在模型下精确满足；
5. Int 变量取整数；
6. Real 变量取 rational 或 algebraic real；
7. Dynamic atoms 与其定义一致。
```

不能因为 local search 或 interval pruning 找到 candidate 就直接 SAT。

## 1.3 不完整推理只能返回 Unknown / Lemma，不能 UNSAT

下面这些不能直接返回 UNSAT：

```text
local search failure
sampling failure
incomplete root search
budget exhaustion
partial interval pruning
failed factorization
failed polynomial decomposition
unsupported theory combination
```

UNSAT 只能来自：

```text
Boolean CDCL conflict
EUF congruence conflict
LRA exact Farkas / simplex conflict
LIA complete integer conflict under stated conditions
NRA CAD/NLSAT/CAC complete conflict
NIA complete finite search / sound modular / sound gcd / complete MCSAT proof
```

## 1.4 Dynamic atoms 必须是一等公民

MCSAT 必须能创建新 atom：

```text
x <= 3
p(x,y) = 0
p(x,y) < 0
a = b
f(a) = f(b)
x ∈ cell_i
```

所有新 atom 必须走：

```cpp
TheoryAtomRegistry::getOrCreate(...)
```

不允许 fake ExprId，不允许 solver-local literal。

# 2. MCSAT Core 设计

## 2.1 Trail

传统 CDCL trail 只有 boolean literal。MCSAT trail 要扩展为：

```cpp
enum class TrailEntryKind {
    BoolLit,
    TheoryValue,
    Equality,
    Disequality,
    Bound,
    DynamicAtomDecision,
    Propagation
};

struct TrailEntry {
    TrailEntryKind kind;

    DecisionLevel level;
    bool isDecision;

    std::optional<SatLit> lit;

    // Theory assignment
    TheoryVar var;
    TheoryValue value;

    // For explanation
    Reason reason;
};
```

其中 `TheoryValue` 支持：

```cpp
struct TheoryValue {
    enum Kind {
        Bool,
        Rational,
        Integer,
        AlgebraicReal,
        EUFClass,
        UninterpretedFresh
    };

    Kind kind;
    mpq_class rational;
    mpz_class integer;
    AlgebraicReal algebraic;
    EClassId eclass;
};
```

NRA 必须支持 algebraic real。NLSAT/MCSAT for nonlinear real arithmetic 一般需要在 CAD/NLSAT 风格的 cell 或 algebraic sample point 上构造模型；Jovanović 和 de Moura 的 NLSAT 工作就是用 CDCL-style search 并结合 CAD projection 来避开冲突区域。([Leonardo de Moura][2])

---

## 2.2 Decision levels

每个 decision level 允许：

```text
1. Boolean decision；
2. Theory value decision；
3. Split lemma 后的 Boolean branch；
4. Semantic theory decision。
```

例如 NRA 中可以决策：

```text
x := 0
```

或者选择一个 cell：

```text
x ∈ (-∞, root_1)
```

LIA 中可以决策：

```text
x <= k  ∨  x >= k+1
```

EUF 中可以决策：

```text
a = b  ∨  a ≠ b
```

---

## 2.3 Explanation object

统一 explanation：

```cpp
struct Explanation {
    std::vector<SatLit> literals;
    std::vector<TrailEntryId> trailDeps;
    TheoryId sourceTheory;
    std::string debug;
};
```

生成的 learned clause 必须只包含 SAT 层能理解的 literal：

```text
¬A ∨ ¬B ∨ C
```

如果 explanation 依赖 theory value assignment，需要先转成 dynamic atom，例如：

```text
x := 3
```

对应可解释为：

```text
x = 3
```

或者 branch atom：

```text
x <= 3
x >= 4
```

第一版建议：**learned lemmas 只使用 Boolean/theory atoms，不直接学习 raw value assignment**。这样 easier to integrate with current CDCL.

## 2.4 MCSAT plugin API

定义统一 theory plugin：

```cpp
class McsatTheory {
public:
    virtual TheoryId id() const = 0;

    virtual void reset() = 0;

    virtual void registerTerm(TermId t, Sort sort) = 0;
    virtual void registerAtom(AtomId a) = 0;

    virtual TheoryCheckResult assertLit(SatLit lit, const Atom& atom) = 0;

    virtual TheoryCheckResult propagate(const McsatTrail& trail) = 0;

    virtual std::optional<TheoryDecision>
    nextTheoryDecision(const McsatTrail& trail, const PartialModel& model) = 0;

    virtual TheoryCheckResult checkPartial(const PartialModel& model) = 0;

    virtual TheoryCheckResult checkFull(const PartialModel& model) = 0;

    virtual void collectModel(TheoryModel& out) = 0;
};
```

返回：

```cpp
enum class TheoryResultKind {
    NoChange,
    Propagation,
    Lemma,
    Conflict,
    ModelCompleted,
    Unknown
};

struct TheoryCheckResult {
    TheoryResultKind kind;
    std::vector<SatLit> propagated;
    std::optional<TheoryLemma> lemma;
    std::optional<TheoryConflict> conflict;
};
```

# 3. MCSAT 主循环

伪代码：

```cpp
Result McsatSolver::check() {
    initialize();

    while (true) {
        // 1. Boolean propagation
        auto bp = sat_.propagate();
        if (bp.conflict) {
            if (!analyzeAndBackjump(bp.conflict)) return Unsat;
            continue;
        }

        // 2. Assert current Boolean literals to theories
        for (auto lit : newlyAssignedTheoryLits()) {
            auto r = theoryManager_.assertLit(lit);
            if (handleTheoryResult(r)) continue;
        }

        // 3. Theory propagation
        bool changed = false;
        for (auto* th : theories_) {
            auto r = th->propagate(trail_);
            if (handleTheoryResult(r)) {
                changed = true;
                break;
            }
        }
        if (changed) continue;

        // 4. Theory partial check
        for (auto* th : theories_) {
            auto r = th->checkPartial(model_);
            if (handleTheoryResult(r)) {
                changed = true;
                break;
            }
        }
        if (changed) continue;

        // 5. Decide
        if (sat_.hasUnassignedBoolAtom()) {
            decideBoolean();
            continue;
        }

        if (auto d = theoryManager_.nextTheoryDecision(trail_, model_)) {
            applyTheoryDecision(*d);
            continue;
        }

        // 6. Full theory check
        for (auto* th : theories_) {
            auto r = th->checkFull(model_);
            if (handleTheoryResult(r)) {
                changed = true;
                break;
            }
        }
        if (changed) continue;

        // 7. Validate final model
        if (validator_.validate(model_)) return Sat;
        return Unknown; // should be debug failure
    }
}
```

# 4. EUF Plugin

## 4.1 数据结构

EUF 需要：

```text
Term DAG
Congruence closure
Union-find
Use-lists
Explanation forest
Disequality set
Function application table
```

核心结构：

```cpp
struct ENode {
    TermId term;
    Symbol symbol;
    std::vector<TermId> args;
};

class EGraph {
    UnionFind uf;
    std::unordered_map<AppKey, TermId> congruenceTable;
    std::vector<EqualityReason> mergeReasons;
};
```

---

## 4.2 Assert equality / disequality

```text
a = b true:
  merge(a,b)

a = b false:
  assert disequality(a,b)
```

如果 merge 后发现某个 disequality 两边同类：

```text
conflict
```

conflict reason：

```text
path reasons proving a=b
+
disequality reason
```

---

## 4.3 EUF propagation

如果已有：

```text
a = b
```

则可推出：

```text
f(a) = f(b)
```

但在 SMT 中通常不需要主动生成所有 congruence equalities。更好做法：

```text
只在相关 atom 出现时 propagate。
```

例如 atom `f(a)=f(b)` 已存在，EUF 可以 propagate true。

---

## 4.4 EUF theory decision

如果两个 EUF terms 未确定相等关系，且某个 formula 需要区分，可以决策：

```text
a = b ∨ a ≠ b
```

但第一版不主动做大量 EUF split，避免 blow-up。

---

## 4.5 EUF model

EUF model 包含：

```text
1. 每个 equivalence class 一个 fresh value；
2. 每个 function symbol 的 finite interpretation；
3. 未定义输入用 default value。
```

# 5. LRA Plugin

## 5.1 使用 GeneralSimplex

你已有 `GeneralSimplex`。MCSAT 中 LRA 插件应成为 exact rational simplex plugin。

输入 normalized linear constraints：

```text
a·x <= c
a·x >= c
a·x = c
a·x != c
```

---

## 5.2 Strict inequality

对 LRA，strict inequality 不能像 LIA 一样转成 `<= c-1`。需要：

```text
Delta rational
```

例如：

```text
x < c
```

内部表示：

```text
x <= c - δ
```

`DeltaRational = a + bδ`，其中 `δ > 0` infinitesimal。

---

## 5.3 Disequality

LRA disequality：

```text
p != 0
```

如果当前 model 满足则 NoChange。

如果当前 model 违反：

```text
p = 0
```

返回 split lemma：

```text
p < 0 ∨ p > 0
```

guarded：

```text
¬lit(p != 0) ∨ lit(p < 0) ∨ lit(p > 0)
```

---

## 5.4 Conflict

LRA conflict 必须是 Farkas-style explanation：

```text
Σ λ_i constraint_i => 0 <= -1
```

第一版可以从 `GeneralSimplex` 的 bound reasons 生成 coarse conflict，但最终要做 Farkas proof object。

---

## 5.5 LRA model decision

如果所有 asserted constraints consistent，simplex 给 rational model。

MCSAT theory-value decision 可以从：

```text
unassigned real variable x
```

选择：

```text
x := current simplex beta(x)
```

或者延迟到 full model construction。

第一版：LRA 不主动 theory decision，交给 simplex model。

---

# 6. LIA Plugin

LIA 插件不应继承 LRA solver，而是：

```text
GeneralSimplex relaxation
+
integer reasoning
+
branch-and-cut
```

你之前已经做了独立 `LiaSolver`，这个方向是对的。

---

## 6.1 LIA check sequence

```text
1. Assert active linear atoms to GeneralSimplex.
2. LRA relaxation check.
3. If LRA UNSAT → conflict.
4. Check integrality of all Int vars.
5. If integral → SAT candidate.
6. Disequality gcd split.
7. GCD equality conflict.
8. GCD inequality tightening.
9. Single-var bound tightening.
10. Unit cube test.
11. Gomory cut.
12. Branch split.
```

---

## 6.2 Integer branch

全局有效：

```text
x <= floor(v) ∨ x >= ceil(v)
```

对 fractional `v`。

如果 current LRA model：

```text
x = 5/2
```

branch：

```text
x <= 2 ∨ x >= 3
```

---

## 6.3 Gomory / cuts

LIA 插件可以复用你前面设计的 bound-aware Gomory。

关键原则：

```text
1. cut 必须保留所有整数解；
2. cut 必须排除当前 fractional relaxation model；
3. cut 必须 guarded；
4. cut atom 必须通过 registry 注册；
5. contradiction cut 返回 guarded conflict。
```

---

## 6.4 LIA model

整数变量必须 denominator 为 1。

如果 relaxation model 非整数，不能 SAT。

---

# 7. NRA Plugin

NRA 是整个系统最核心也最难的部分。建议采用 **NLSAT / MCSAT-CAD hybrid**。

NLSAT 本身就是针对 nonlinear real arithmetic 的 CDCL-style search，使用 CAD projection operator 引导搜索避开冲突区域。([Leonardo de Moura][2]) MCSAT 中的 NRA plugin 可以把 NLSAT/CAC 作为 theory-level explanation engine。

cvc5 近年的 nonlinear real arithmetic 求解也使用了 cylindrical algebraic coverings 相关技术，说明 CAC/CAD-style covering 对现代 NRA solver 是实际可行路线。([Gereon Kremer][3])

---

## 7.1 NRA 输入

Normalized polynomial constraints：

```text
p(x1,...,xn) < 0
p <= 0
p = 0
p != 0
p >= 0
p > 0
```

系数：

```text
rational
```

变量：

```text
real
```

---

## 7.2 AlgebraicReal

必须实现：

```cpp
class AlgebraicReal {
public:
    Polynomial minimalPolynomial;
    Interval isolatingInterval;
    int rootIndex;

    int signAt(const Polynomial& p) const;
    int compare(const AlgebraicReal& other) const;
};
```

需要支持：

```text
rational sample
algebraic sample
root isolation
polynomial sign evaluation at algebraic point
```

---

## 7.3 Variable ordering

NRA MCSAT 必须有 variable order：

```text
x1 ≺ x2 ≺ ... ≺ xn
```

这个 order 用于：

```text
1. projection;
2. model construction;
3. conflict explanation;
4. cell decomposition.
```

初始版本：

```text
input order / degree heuristic
```

后续可接学习策略。

---

## 7.4 Model construction

NRA plugin 逐变量赋值：

```text
x1 := sample_1
x2 := sample_2
...
xn := sample_n
```

每次赋值后，把已赋变量代入未满足 constraints，得到 lower-dimensional polynomials。

---

## 7.5 Conflict detection

当当前 partial assignment 使某个 constraint 不可扩展时，生成 CAD/CAC-style explanation。

例如当前：

```text
x = a
```

导致关于 `y` 的 constraints 无解。

NRA plugin 构造一个 cell：

```text
x ∈ I
```

使得在这个 cell 中都无解，然后返回 lemma：

```text
x ∉ I
```

或者转成 boundary atoms：

```text
x <= l ∨ x >= r
```

如果 cell 由 algebraic roots 定义，则生成 dynamic root-bound atoms。

---

## 7.6 Projection

需要实现 projection operator：

```text
discriminant
resultant
leading coefficient
content
```

第一版可以只支持 NLSAT-style local projection：

```text
按冲突约束局部投影，不全量 CAD。
```

数据结构：

```cpp
struct ProjectionSet {
    std::vector<Polynomial> polynomialsByLevel[level];
};
```

---

## 7.7 Sample point selection

对于一维 variable `x`：

```text
收集所有 relevant polynomial roots
排序成 cells
选择满足 constraints 的 cell sample
```

cells：

```text
(-∞, r1)
{r1}
(r1, r2)
{r2}
...
(rk, +∞)
```

sample：

```text
rational if interval cell
algebraic if root cell
```

---

## 7.8 NRA explanation lemmas

基本 lemma 类型：

```text
1. Sign-invariant cell exclusion:
   current assignment falls into cell C; C is infeasible;
   learn ¬C.

2. Root ordering lemma:
   x < root(p,i)
   x = root(p,i)
   x > root(p,i)

3. Polynomial sign lemma:
   p(x) < 0 / p(x) = 0 / p(x) > 0

4. Disequality split:
   p != 0 → p < 0 ∨ p > 0
```

所有 root-bound atoms 必须 dynamic register。

---

## 7.9 NRA completeness target

对 QF_NRA，目标是 complete。

第一阶段可以调用 internal NLSAT/CAC engine 对 full NRA block 求 complete result，然后逐步把 explanation 接入 MCSAT。

最终版本：

```text
NRA plugin produces MCSAT-compatible conflict lemmas,
not black-box SAT/UNSAT only.
```

# 8. NIA Plugin

NIA 是最难完整做的理论之一。NIA MCSAT 可以采用：

```text
NRA relaxation
+
integer model search
+
modular reasoning
+
branch-and-bound
+
finite-domain enumeration
+
cuts
+
local search
```

Jovanović 2017 的 NIA MCSAT 工作就是用 MCSAT 框架处理 nonlinear integer arithmetic，并通过 real geometry/NLSAT 来枚举潜在整数解。([Yices][4])

---

## 8.1 NIA check sequence

```text
1. Normalize integer polynomial constraints.
2. Linear domain reasoning.
3. Square bound / sum-of-squares bound.
4. Univariate integer roots.
5. Multivar GCD conflict.
6. Modular equality reasoning.
7. Factor lemmas.
8. Bounded enumeration.
9. NRA relaxation via NRA plugin.
10. Integer candidate extraction.
11. Local search SAT finder.
12. Branch lemma.
13. Unknown if incomplete budget exhausted.
```

---

## 8.2 NIA as MCSAT over integers

Theory decisions:

```text
x := k
x <= k ∨ x >= k+1
x ≡ r mod m
```

First version only use:

```text
x <= k ∨ x >= k+1
```

Later add congruence atoms:

```text
x ≡ r mod m
```

---

## 8.3 Real relaxation

For NIA constraints, drop integrality:

```text
Int x -> Real x
```

Use NRA plugin to find real cells.

If real relaxation UNSAT:

```text
NIA UNSAT
```

with same conflict.

If real relaxation SAT but no integer point found:

```text
generate integer split / modular conflict / cube test
```

---

## 8.4 Finite-domain exact enumeration

If DomainStore makes all variables finite and size below threshold:

```text
enumerate
exact validate
SAT or UNSAT
```

This gives sound UNSAT.

---

## 8.5 Modular reasoning

For equalities:

```text
p1=0,...,pk=0
```

For modulus `m`, enumerate residues.

If no residue assignment:

```text
conflict over equality reasons
```

Do not use modular reasoning to prove disequality false unless formally designed.

---

## 8.6 NIA local search

Use as SAT finder only:

```text
objective = Σ violation²
```

If objective becomes 0:

```text
exact validate
SAT candidate
```

Failure:

```text
NoChange / Unknown
```

Never UNSAT.

# 9. Combination: EUF + arithmetic

For EUF + arithmetic, use Nelson-Oppen-style equality sharing but inside MCSAT.

Shared variables are terms that occur in both EUF and arithmetic, e.g.:

```text
f(x) = y
y + 1 > 0
```

Need a shared equality manager.

---

## 9.1 Shared terms

Define:

```cpp
struct SharedTerm {
    TermId term;
    Sort sort;
    std::vector<TheoryId> owners;
};
```

If EUF discovers:

```text
a = b
```

and `a,b` are arithmetic terms or shared variables, notify arithmetic plugin.

If arithmetic discovers:

```text
x = y
```

and `x,y` are EUF terms, notify EUF plugin.

---

## 9.2 Equality propagation

Central equality engine:

```text
EqualityManager
```

Responsibilities:

```text
1. Maintain global equalities/disequalities for shared terms.
2. Route equality propagation to EUF and arithmetic.
3. Generate explanation over source theory reasons.
```

Architecture:

```text
EUFPlugin
   ↘
    EqualityManager
   ↗
ArithmeticPlugin
```

---

## 9.3 Purification

Before solving mixed formulas, purify:

```text
f(x+1) = y
```

Introduce:

```text
t = x+1
f(t) = y
```

Arithmetic owns `t = x+1`; EUF owns `f(t)=y`.

SOMTParser/Core IR should produce purified form for combination.

# 12. Explanation correctness

Every theory lemma must satisfy:

```text
Current trail entails lemma.
```

Before adding lemma to SAT, debug mode should run:

```cpp
LemmaValidator::validate(lemma, currentTrail, plugin);
```

Validators:

```text
EUF lemma validator
LRA Farkas validator
LIA cut/branch validator
NRA cell-exclusion validator
NIA modular/gcd/finite-search validator
```

If validator unavailable:

```text
debug assert disabled only for trusted mature module
```

For a serious solver, keep validators on during development.

# Final architecture summary

最终 NLColver 的 MCSAT 形态应该是：

```text
MCSAT Core:
  CDCL + theory-value trail + dynamic atoms + explanation manager

EUF:
  congruence closure + equality propagation + finite model

LRA:
  simplex + delta rationals + Farkas explanation

LIA:
  LRA relaxation + integer cuts + branch-and-bound

NRA:
  NLSAT/CAD/CAC-style algebraic model construction

NIA:
  NIA algebraic reasoning + modular + finite search + NRA relaxation + branch

Combination:
  purification + shared equality manager
```

这才是能长期支撑 EUF、LIA、LRA、NIA、NRA 以及混合理论的“正经求解器”架构。
