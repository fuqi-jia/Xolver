# IDL / RDL Difference-Logic Solver Implementation Plan

## Objective
Implement dedicated IDL (Integer Difference Logic) and RDL (Real Difference Logic) solvers using graph-based negative-cycle detection (Bellman-Ford / SPFA) instead of Simplex. These solvers will be registered for `QF_IDL` / `QF_RDL` logics and will reuse the existing `LinearAtomPayload` infrastructure.

## Key Design Decisions

1. **TheoryId**: Add `IDL` and `RDL` to the `TheoryId` enum in `src/expr/types.h`. This allows `TheoryManager` to route atoms directly without confusing them with general LIA/LRA atoms.
2. **Atomizer integration**: Minimal change. In `Solver.cpp`, when logic is `QF_IDL`/`QF_RDL`, set `Atomizer::defaultTheory_` to `IDL`/`RDL`. The existing `extractLinearConstraint` path in `Atomizer` already produces `LinearAtomPayload`, which is sufficient. The IDL/RDL solvers will self-filter difference constraints from the payload.
3. **Unsupported atoms**: If an IDL/RDL solver receives a non-difference linear constraint, it sets an internal `unsupported_` flag and returns `Unknown` from `check()`. The existing `hasUnsupportedTheoryAtom()` mechanism is handled at the registry level; for IDL/RDL we will add a solver-level unsupported check that the caller can query, or simply return `Unknown` and let the SAT solver abort.
4. **Dynamic atoms for disequality split**: Reuse `TheoryAtomRegistry::getOrCreateLinearBoundAtom`. The split lemmas generate new `LinearAtomPayload` atoms with the same `TheoryId` (IDL or RDL).
5. **Model validation**: Each solver implements a private `validateModel()` that checks all active constraints before returning `Sat`.

## Phase 1: DL Common Infrastructure

### 1.1 DlWeight.h
- `using IdlWeight = mpz_class;`
- `struct RdlWeight { mpq_class c; int deltaCoeff; };` with lexicographic comparison and addition/subtraction operators.

### 1.2 DifferenceGraph.h/.cpp
- `template<typename Weight> class DifferenceGraph`
- Nodes indexed densely (`int nodeId`).
- `int getOrCreateNode(const std::string& varName)` — maps variable name → dense id. Special variable `"__ZERO__"` gets node 0.
- `EdgeId addEdge(int from, int to, Weight w, SatLit reason)`
- `void deactivateAfter(size_t marker)` / `size_t marker() const` — for backtracking.
- `const std::vector<DlEdge<Weight>>& edges() const`
- `const std::vector<std::vector<EdgeId>>& outgoing() const`

### 1.3 BellmanFord.h/.cpp
- `template<typename Weight> class BellmanFord`
- `struct Result { bool negativeCycle; std::vector<EdgeId> cycle; std::vector<Weight> dist; };`
- `Result runFull(const DifferenceGraph<Weight>& graph)` — standard Bellman-Ford with all-zero initialization.
- `Result runIncremental(const DifferenceGraph<Weight>& graph, const std::vector<EdgeId>& newEdges, std::vector<Weight>& currentDist)` — SPFA-style propagation from newly added edges.
- Cycle extraction: walk `pred[]` back `n` steps then loop to collect cycle edges.

### 1.4 DlModel.h/.cpp
- `template<typename Weight> struct DlModel { std::unordered_map<std::string, mpq_class> values; };`
- `DlModel<IdlWeight> buildIdlModel(const std::vector<Weight>& dist, int zeroNode)` — `value = dist[x] - dist[ZERO]` (pure integer).
- `DlModel<RdlWeight> buildRdlModel(const std::vector<RdlWeight>& dist, int zeroNode)` — choose `ε = 1` (or compute upper bound from active strict constraints), instantiate `δ = ε`, produce rational values.

### 1.5 DlExplanation.h/.cpp
- `template<typename Weight> TheoryConflict buildConflict(const std::vector<EdgeId>& cycle, const DifferenceGraph<Weight>& graph)` — collect unique `SatLit` reasons, negate each, form clause.

## Phase 2: IDL Solver

### 2.1 IdlSolver.h/.cpp
Inherits `TheorySolver`.

**State:**
```cpp
struct ActiveAssignment { int level; SatLit lit; TheoryAtomRecord atom; bool value; };
std::vector<ActiveAssignment> activeAssignments_;
DifferenceGraph<mpz_class> graph_;
BellmanFord<mpz_class> bf_;
std::vector<EdgeId> newEdges_;       // edges added since last check
bool graphDirty_ = true;
TheoryAtomRegistry* registry_ = nullptr;
std::optional<TheoryConflict> pendingConflict_;
std::optional<TheoryLemma> pendingLemma_;
```

**`assertLit`**: Store assignment. If `LinearAtomPayload`, attempt to extract difference constraint:
- 1 var: treat `x` as `x - ZERO`, coefficient +1.
- 2 vars: require coefficients `{+1, -1}` or `{-1, +1}`.
- Convert relation to canonical `x - y <= c` form.
- Apply IDL normalization (floor for `<=`, `ceil-1` for `<`, etc.).
- Add edge to graph, record `EdgeId` in `newEdges_`.
- For `Eq`: add two edges (`<=` and `>=`).
- For `Neq`: store disequality for later split (do not add edge).

**`backtrackToLevel`**: Remove assignments with `level > target`. Mark `graphDirty_ = true`. Clear `newEdges_`.

**`check`**:
```cpp
if (graphDirty_) {
    rebuild graph from all active assignments;
    auto r = bf_.runFull(graph_);
} else {
    auto r = bf_.runIncremental(graph_, newEdges_, currentDist_);
}
newEdges_.clear();

if (r.negativeCycle)
    return TheoryCheckResult::mkConflict(buildConflict(r.cycle, graph_));

// Disequalities
for each stored diseq (x - y != c):
    if current model has x - y == c:
        return splitLemma(diseq)

buildModel();
return TheoryCheckResult::consistent();
```

**Disequality split** (IDL):
```
¬(x-y != c) ∨ (x-y <= c-1) ∨ (y-x <= -(c+1))
```
Use `registry_->getOrCreateLinearBoundAtom` to create the two branch literals.

**Model validation**: Verify every active edge `dist[to] <= dist[from] + weight`, every active equality, every active disequality.

### 2.2 test_idl.cpp
- SAT: `x - y <= 3 ∧ y - x <= 2`
- UNSAT (negative cycle): `x - y <= 3 ∧ y - z <= 4 ∧ z - x <= -8`
- UNSAT (strict): `x - y < 3 ∧ x - y >= 3`
- UNSAT (disequality): `x = y ∧ x != y`
- SAT (single var): `x <= 5 ∧ x >= 3`

## Phase 3: RDL Solver

### 3.1 RdlSolver.h/.cpp
Nearly identical structure to `IdlSolver`, but uses `RdlWeight`.

**Normalization differences:**
- `x - y < c` → edge weight `(c, -1)` (strict)
- `x - y <= c` → edge weight `(c, 0)`
- No integer rounding needed.

**Negative cycle**: A cycle with total weight `(c, d)` is negative iff `c < 0 || (c == 0 && d < 0)`. This captures the strict-zero cycle (`0 - δ`).

**Model construction**:
1. Run Bellman-Ford to get potentials `dist[v] = a_v + b_v·δ`.
2. Collect all active strict constraints: each gives an upper bound on `ε` (the concrete value of `δ`).
3. Choose `ε = 1` if unconstrained, otherwise `ε = minPositiveUpperBound / 2`.
4. Instantiate: `model(v) = (a_v - a_ZERO) + (b_v - b_ZERO)·ε`.

**Disequality split** (RDL):
```
¬(x-y != c) ∨ (x-y < c) ∨ (y-x < -c)
```
Use `registry_->getOrCreateLinearBoundAtom` with strict relations. Note: `getOrCreateLinearBoundAtom` currently takes `Relation` (`Leq`, `Geq`, etc.). For RDL strict splits we may need `<` and `>`. Since `LinearAtomPayload` supports `Relation::Lt`/`Gt`, we can pass those directly.

### 3.2 test_rdl.cpp
- UNSAT (strict zero cycle): `x - y < 1 ∧ y - x <= -1`
- SAT: `x - y < 1 ∧ y - x < -1`  (strict + strict)
- UNSAT (disequality): `x = 0 ∧ x != 0`
- SAT: `x - y <= 2.5 ∧ y - x <= -1.5`

```cpp
else if (logic == "QF_IDL" || logic == "IDL") {
    auto idl = std::make_unique<IdlSolver>();
    idl->setRegistry(&registry);
    theoryManager.registerSolver(std::move(idl));

```

Add atomizer default theory:
```cpp
else if (logic == "QF_IDL" || logic == "IDL") {
    atomizer.setDefaultTheory(TheoryId::IDL);
} else if (logic == "QF_RDL" || logic == "RDL") {
    atomizer.setDefaultTheory(TheoryId::RDL);
}
```

### 4.3 Unsupported atom handling
In `IdlSolver::check()` / `RdlSolver::check()`, if any active assignment is not a difference constraint, return `TheoryCheckResult::unknown()`. This causes the CDCL(T) loop to abort with `Unknown` (same pattern as NIA's incomplete cases).

## Complexity Summary

| Component        | Time      | Space |
|------------------|-----------|-------|
| Bellman-Ford     | O(VE)     | O(V+E)|
| SPFA (average)   | O(E)      | O(V+E)|
| Model validation | O(E)      | O(V)  |
| Diseq split      | O(1)      | O(1)  |

## Risks & Mitigations

1. **RDL epsilon selection**: If upper-bound computation is buggy, strict constraints may be violated. Mitigation: always run `validateModel()` before returning `Sat`; if it fails, return `Unknown`.
2. **Disequality lemma loops**: Split lemmas could be re-learned. Mitigation: deduplicate via `TheoryLemmaDatabase::contains()` before returning a lemma.
3. **Backtracking correctness**: Full rebuild on backtrack is simplest and correct. Mitigation: Phase 1 uses `graphDirty_` full-rebuild pattern (same as LraSolver/LiaSolver's lazy replay).
