# 1. V1 到底是否使用 incremental SPFA 仍然矛盾

前面 `BellmanFord` 仍然设计了：

```cpp
Result runIncremental(...)
```

而 `check()` 伪代码也写了：

```cpp
if (graphDirty_) {
    rebuild graph from all active assignments;
    auto r = bf_.runFull(graph_);
} else {
    auto r = bf_.runIncremental(graph_, newEdges_, currentDist_);
}
```

但后面 Out-of-Scope 又写：

```text
Incremental SPFA optimization
V1 uses full Bellman-Ford rebuild every check; incremental SPFA is V2
```

这两处冲突。

我建议第一版严格做：

```text
V1: 每次 check 都从 activeAssignments_ 重建图，然后 runFull Bellman-Ford。
不使用 runIncremental，不维护 currentDist_，不依赖 newEdges_。
```

这样实现更稳，backtrack 正确性也简单。IDL/RDL 本来就轻量，第一版全量 Bellman-Ford 足够。

可以把 `runIncremental` 留到接口 TODO，但不要在 V1 的 `check()` 里调用。

---

# 2. `Eq` 非整数 immediate conflict 要写明 conflict reason

IDL 中：

```text
x - y = 1/2
```

确实立即 UNSAT，因为整数差不可能等于非整数。但实现时不能只设置 `unsupported` 或直接返回空 conflict，必须生成：

```text
¬reason(x-y = 1/2)
```

同理，常量冲突、归一化阶段发现的 impossible constraint，都要有当前 literal 的 negated reason。

建议补一句：

```text
Any normalization-time contradiction must return a TheoryConflict containing the negation of the active literal reason.
```
