# Plan: Trail-Based CDCL(T) Refactor

## Goal
Replace the current scan-all-atoms CDCL(T) loop with a trail-based, incrementally-driven theory integration architecture.

## Current State
- `TheoryManager::check()` scans `atomizer.atoms()` every SAT model, calls `solver->reset()`, then re-asserts all theory literals from scratch.
- `LiaSolver::reset()` clears `activeAtoms_`, `disequalities_`, and calls `gs_.resetActiveBounds()`.
- `GeneralSimplex` has `push()/pop()` and an internal `trail_` for bounds, but no decision-level aware backtracking.
- `SatSolver` interface is opaque: no assignment callbacks, only `value(SatVar)` after `solve()`.
- `TheoryAtomRegistry` can look up by synthetic `ExprId` (linear scan), but not by `SatVar`.

## Why This Matters
1. **Performance**: O(#models × #atoms) scan grows with dynamic atoms (branch splits, cuts).
2. **Correctness**: Repeated `reset()` + rebuild loses lemma deduplication state and causes repeated lemma generation.
3. **Dynamic atom round-trip**: The current scan approach *happens* to include dynamic atoms because `Atomizer::registerDynamicAtom` appends to `atoms_`, but this is fragile.

## Approach: Interface-First, Simulate Until Backend Supports It

We design the final trail-based interface now, but implement SAT-side notifications via model-delta simulation inside `CadicalBackend::solve()` until CaDiCaL supports true assignment callbacks.

### Phase 1: New Interfaces (SatSolver, TheoryManager, TheorySolver)

#### 1.1 `SatAssignmentListener` + `SatSolver::setAssignmentListener`
```cpp
class SatAssignmentListener {
public:
    virtual ~SatAssignmentListener() = default;
    virtual void onAssign(SatLit lit, int decisionLevel) = 0;
    virtual void onBacktrack(int targetLevel) = 0;
};
```

`SatSolver` gains:
```cpp
virtual void setAssignmentListener(SatAssignmentListener* listener) = 0;
```

#### 1.2 `CadicalBackend` Simulation
Since CaDiCaL C++ API does not expose fine-grained assignment callbacks in our version, `CadicalBackend::solve()` will:
1. Call `solver_->solve()`.
2. On SAT: compare the new model with `prevModel_` (a `std::vector<int8_t>` indexed by `SatVar`).
3. If a var flipped from unassigned/false → true or true → false, emit:
   - `listener_->onBacktrack(0)` (simplified: we notify full reset then replay)
   - Then for every var that is true in the new model: `listener_->onAssign(positive(var), level)`
   - For every var that is false: `listener_->onAssign(negative(var), level)`

**Note**: This is still O(#vars) per solve, but the complexity moves from `TheoryManager` into the SAT wrapper, and the *interface* is correct. When CaDiCaL is upgraded or we switch to a callback-capable backend, only `CadicalBackend` changes.

#### 1.3 `TheoryBridge` (new class)
Implements `SatAssignmentListener`.
```cpp
class TheoryBridge : public SatAssignmentListener {
public:
    TheoryBridge(TheoryAtomRegistry& registry, TheoryManager& tm)
        : registry_(registry), tm_(tm) {}

    void onAssign(SatLit lit, int level) override {
        const auto* atom = registry_.findBySatVar(lit.var());
        if (!atom) return; // not a theory atom
        tm_.assertTheoryLit(*atom, lit, level);
    }

    void onBacktrack(int targetLevel) override {
        tm_.backtrackToLevel(targetLevel);
    }
};
```

```cpp
struct TheoryAtomRecord {
    SatVar satVar;

    ExprId exprId;

std::vector<TheoryAtomRecord> records_;           // dense, indexed by internal id
```

Update `getOrCreateLinearBoundAtom` to populate both maps.
Add:


#### 1.5 `TheoryManager` New Interface
Remove:
```cpp
TheoryCheckResult check(const CoreIr& ir,
                        const std::vector<Atomizer::AtomRecord>& atoms,
                        const SatSolver& sat);
```

Add:
```cpp
void registerSolver(std::unique_ptr<TheorySolver> solver); // keep
void clearSolvers(); // keep

void assertTheoryLit(const TheoryAtomRecord& atom, SatLit assignedLit, int level);
void backtrackToLevel(int level);
TheoryCheckResult check();
```

`assertTheoryLit` routes to the correct solver by `atom.theory`.
`backtrackToLevel` calls `solver->backtrackToLevel(level)` for all solvers.
`check()` calls `solver->check()` for all solvers, returns first non-consistent result.

#### 1.6 `TheorySolver` New Interface
Replace old interface:
```cpp
class TheorySolver {
public:
    virtual ~TheorySolver() = default;
    virtual TheoryId id() const = 0;

    // Scope management (keep for push/pop API, not used in CDCL loop)
    virtual void push() = 0;
    virtual void pop(uint32_t n) = 0;

    // Incremental assertion from SAT trail
    virtual void assertLit(const TheoryAtomRecord& atom,
                           bool value,
                           int level,
                           SatLit reason) = 0;

    // Backtrack to target decision level
    virtual void backtrackToLevel(int level) = 0;

    // Check current incremental state
    virtual TheoryCheckResult check() = 0;

    // Reset for a fresh check-sat call (clears all incremental state)
    virtual void reset() = 0;
};
```

### Phase 2: LiaSolver Incremental Refactor

`LiaSolver` must stop rebuilding from scratch every check round.

#### 2.1 Trail-Based Internal State
```cpp
struct TrailEntry {
    int level;
    enum class Kind { Bound, Disequality, ActiveAtom } kind;
    // Data needed to undo:
    int auxVar;
    bool wasLower; // for bound undo
    BoundInfo oldBound;
};

std::vector<TrailEntry> trail_;
std::vector<size_t> levelLimits_; // trail_.size() at each level
```

#### 2.2 `assertLit` (incremental)
```cpp
void LiaSolver::assertLit(const TheoryAtomRecord& atom, bool value,
                          int level, SatLit reason) {
    ensureLevel(level);

    int aux = manager_.getOrCreateAuxVar(gs_, atom.lhs, atom.rhs);

    if (isDisequality(atom.rel, value)) {
        disequalities_.push_back({aux, atom.lhs, atom.rhs, reason});
        trail_.push_back({level, TrailEntry::Kind::Disequality, aux, false, {}});
        return;
    }

    manager_.assertBound(gs_, aux, atom.rel, value, reason);
    // Record old bound for undo
    bool isLower = ...;
    BoundInfo old = isLower ? gs_.getLower(aux) : gs_.getUpper(aux);
    trail_.push_back({level, TrailEntry::Kind::Bound, aux, isLower, old});

    activeAtoms_.push_back({atom.exprId, aux, atom.rel, value,
                            atom.lhs, atom.rhs, reason});
    trail_.push_back({level, TrailEntry::Kind::ActiveAtom, aux, false, {}});
}
```

#### 2.3 `backtrackToLevel`
```cpp
void LiaSolver::backtrackToLevel(int level) {
    while (!trail_.empty() && trail_.back().level > level) {
        undo(trail_.back());
        trail_.pop_back();
    }
}
```

`undo(entry)`:
- `Bound`: restore `vars_[aux].lower` or `vars_[aux].upper` to `oldBound`
- `Disequality`: pop from `disequalities_`
- `ActiveAtom`: pop from `activeAtoms_`

#### 2.4 `check()` (no parameters)
```cpp
TheoryCheckResult LiaSolver::check() {
    auto r = gs_.check();
    if (r == Unsat) return mkConflict(...);
    if (r == Unknown) return unknown();

    // disequality, integrality, cheap reasoning, validation
    // Same logic as today, but no `reset()` called before this.
}
```

#### 2.5 `reset()` (for fresh check-sat call)
Clear everything: `activeAtoms_`, `disequalities_`, `trail_`, `levelLimits_`, `emittedLemmas_`, `gs_.resetActiveBounds()`.

### Phase 3: GeneralSimplex Level-Aware Backtracking

Current `GeneralSimplex` has:
- `push()` / `pop()` using `trail_` vector of vectors
- `resetActiveBounds()` clears all bounds

We need `backtrackToLevel(int level)`.

Options:

**Option A: Reuse existing `push()/pop()`**
- `TheoryManager` calls `solver->push()` at every decision level increase.
- `TheoryManager` calls `solver->pop()` to backtrack.
- This matches the existing `GeneralSimplex::push()/pop()` design.
- **Pros**: Minimal changes to GeneralSimplex.
- **Cons**: `push()/pop()` are stack-based; if SAT backtracks multiple levels, we need multiple `pop()` calls. Also, `push()` creates a new vector on every level, which may be expensive for deep searches.

**Option B: Add `backtrackToLevel` directly**
- GeneralSimplex stores `levelLimits_` mapping decision level → `trail_.size()`.
- `assertLower/Upper` record the current decision level in the `TrailEntry`.
- `backtrackToLevel(target)` pops `trail_` until `trail_.back().level <= target`.
- **Pros**: Single call for multi-level backtrack. No per-level vector allocation.
- **Cons**: Requires modifying `assertLower/Upper` to accept level, and `TrailEntry` to store level.

**Recommendation: Option B** for GeneralSimplex, because it aligns with the final architecture. The existing `push()/pop()` can be deprecated or kept for the legacy `Solver::push()/pop()` API.

Changes to `GeneralSimplex`:
1. `TrailEntry` gains `int level`.
2. `assertLower/Upper` gain `int level` parameter.
3. `backtrackToLevel(int level)` method added.
4. `push()/pop()` can call `backtrackToLevel` for backward compatibility.

### Phase 4: Solver::checkSatInternal Rewrite

```cpp
Result checkSatInternal() {
    if (!ir || ir->assertions().empty()) return Result::Sat;

    sat = createSatSolver();
    Atomizer atomizer(*sat);
    atomRegistry_.setContext(sat.get(), &atomizer);

    // Register solvers
    theoryManager_.clearSolvers();
    registerSolversByLogic();

    // Set up bridge
    TheoryBridge bridge(atomRegistry_, theoryManager_);
    sat->setAssignmentListener(&bridge);

    // Atomize assertions
    for (ExprId a : ir->assertions()) {
        SatLit lit = atomizer.atomize(a, *ir);
        sat->addClause({lit});
    }

    // Lemma database (global for this check-sat call)
    TheoryLemmaDatabase lemmaDb;

    while (true) {
        auto sr = sat->solve();
        if (sr == Unsat) return Result::Unsat;
        if (sr == Unknown) return Result::Unknown;

        auto tr = theoryManager_.check();

        if (tr.kind == Consistent) return Result::Sat;

        if (tr.kind == Conflict) {
            sat->addClause(tr.conflict.clause);
            continue;
        }

        if (tr.kind == Lemma) {
            if (lemmaDb.insertIfNew(*tr.lemma)) {
                sat->addClause(tr.lemma->lits);
            }
            // Even if duplicate, we continue (SAT solver may derive new info)
            continue;
        }

        return Result::Unknown;
    }
}
```

### Phase 5: No-Op Lemma Guards

All theory lemma generators must skip:
1. **Tautology**: conclusion literal == negated premise literal.
2. **Self-tightening**: tightened atom is identical to original atom.
3. **Already satisfied**: conclusion is already true in current SAT assignment. *(Deferred: requires SAT view in theory solver.)*
4. **Duplicate**: already in `TheoryLemmaDatabase`.

### Phase 6: LraSolver and CdcacSolver Migration

Both must implement the new `TheorySolver` interface:
- `assertLit(const TheoryAtomRecord&, bool, int, SatLit)`
- `backtrackToLevel(int)`
- `check()`

For `LraSolver`: similar refactor to `LiaSolver` (incremental bounds, no per-check reset).
For `CdcacSolver`: currently scan-based; migrate to incremental assertion of polynomial constraints.

## Risks and Mitigations

| Risk | Mitigation |
|------|------------|
| CadicalBackend model simulation is O(#vars) | Acceptable for now; interface is correct for future true-callback backend |
| GeneralSimplex `backtrackToLevel` may have bugs | Extensive unit tests on bound trail correctness |
| LiaSolver incremental state may diverge from SAT model | Model validation runs before every `consistent()` return |
| LraSolver / CdcacSolver migration breaks NRA | Keep fallback scan mode if needed, or defer NRA migration |
