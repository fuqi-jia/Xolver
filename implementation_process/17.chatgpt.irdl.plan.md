下面给你一版 **IDL / RDL 专用求解器的完整实现设计**。核心思想很明确：**不要用 Simplex 解 IDL/RDL**。IDL/RDL 是差分约束，最合适的是图算法：

```text
x - y <= c
⇔
edge y -> x with weight c
```

可满足性等价于：**差分约束图中不存在负环**。

其中：

```text
IDL: integer difference logic
RDL: real/rational difference logic
```

二者共享大部分图算法，但在 **strict inequality、disequality、常数规约、模型构造** 上不同。

# 2. 差分约束形式

统一内部约束形式：

```text
x - y <= c
```

对应图边：

```text
y -> x  weight c
```

含义：

```text
value(x) <= value(y) + c
```

单变量约束使用特殊变量：

```text
ZERO
```

例如：

```text
x <= c
⇔ x - ZERO <= c
⇔ ZERO -> x, weight c

x >= c
⇔ ZERO - x <= -c
⇔ x -> ZERO, weight -c
```

最终模型归一化：

```text
model(x) = potential(x) - potential(ZERO)
```

这样 `ZERO` 的值永远解释为 0。

# 3. DL 权重设计

## 3.1 IDL 权重

IDL 只需要整数权重：

```cpp
using IdlWeight = mpz_class;
```

因为所有 strict/非 strict 都可以归约到非 strict integer bound。

例如：

```text
x - y < c
```

由于 `x-y` 是整数：

```text
x - y <= ceil(c) - 1
```

---

## 3.2 RDL 权重

RDL 必须支持 strict inequality。

建议用 infinitesimal delta：

```cpp
struct RdlWeight {
    mpq_class c;     // rational part
    int deltaCoeff;  // usually 0 or -1
};
```

解释为：

```text
c + deltaCoeff * δ
```

其中：

```text
δ > 0, infinitesimal
```

编码：

```text
x - y <= c  -> weight (c, 0)
x - y <  c  -> weight (c, -1)
```

比较采用 lexicographic order：

```text
(a, da) < (b, db)
iff
a < b
or
a == b and da < db
```

负权判断：

```text
weight < 0
iff
c < 0
or
c == 0 and deltaCoeff < 0
```

因此 RDL 里的这种组合会被判 UNSAT：

```text
x - y < 1
y - x <= -1
```

两条相加：

```text
0 < 0
```

也就是 strict zero negative cycle。

# 4. IDL 约束归一化

IDL 变量是整数。输入关系：

```text
x - y rel c
```

其中 `c` 可以是 rational，但最后必须转成整数差分约束。

令：

```text
d = x - y ∈ Z
```

归一化规则：

## 4.1 `d <= c`

```text
d <= floor(c)
```

生成：

```text
x - y <= floor(c)
```

---

## 4.2 `d < c`

```text
d <= ceil(c) - 1
```

例如：

```text
x - y < 3
=> x - y <= 2

x - y < 3.2
=> x - y <= 3
```

---

## 4.3 `d >= c`

```text
d >= ceil(c)
⇔
y - x <= -ceil(c)
```

---

## 4.4 `d > c`

```text
d >= floor(c) + 1
⇔
y - x <= -(floor(c) + 1)
```

---

## 4.5 `d = c`

如果 `c` 不是整数：

```text
UNSAT
```

如果 `c` 是整数：

```text
x - y <= c
y - x <= -c
```

---

## 4.6 `d != c`

如果 `c` 不是整数：

```text
tautology
```

因为整数不可能等于非整数。

如果 `c` 是整数：

```text
d != c
⇔
d <= c-1 ∨ d >= c+1
```

也就是 split lemma：

```text
x - y <= c-1
∨
y - x <= -(c+1)
```

guarded form：

```text
¬lit(x-y != c) ∨ lit(x-y <= c-1) ∨ lit(y-x <= -(c+1))
```

---

# 5. RDL 约束归一化

RDL 变量取 rational/real。

## 5.1 `x - y <= c`

```text
edge y -> x, weight (c, 0)
```

## 5.2 `x - y < c`

```text
edge y -> x, weight (c, -1)
```

## 5.3 `x - y >= c`

等价于：

```text
y - x <= -c
```

edge：

```text
x -> y, weight (-c, 0)
```

## 5.4 `x - y > c`

等价于：

```text
y - x < -c
```

edge：

```text
x -> y, weight (-c, -1)
```

## 5.5 `x - y = c`

生成两条边：

```text
x - y <= c
y - x <= -c
```

## 5.6 `x - y != c`

不能直接加入图。

如果当前 model 违反 disequality，即：

```text
x - y = c
```

则生成 split lemma：

```text
x - y < c ∨ x - y > c
```

guarded：

```text
¬lit(x-y != c) ∨ lit(x-y < c) ∨ lit(x-y > c)
```

# 8. 核心算法：负环检测

IDL/RDL 的可满足性：

```text
无负环 ⇔ SAT
存在负环 ⇔ UNSAT
```

---

## 8.1 基础 Bellman-Ford

初始化：

```text
dist[v] = 0 for all v
```

然后 relax 所有 active edges。

伪代码：

```cpp
CheckResult bellmanFord(graph):
    for v in V:
        dist[v] = 0
        pred[v] = none

    for i in 1..|V|:
        changed = false

        for edge e: u -> v weight w:
            if dist[v] > dist[u] + w:
                dist[v] = dist[u] + w
                pred[v] = e
                changed = true

                if i == |V|:
                    cycle = extractCycle(v, pred)
                    return NegativeCycle(cycle)

        if !changed:
            return Feasible(dist)

    return Feasible(dist)
```

注意：

```text
这里 dist 是 potential。
最终 model(x) = dist[x] - dist[ZERO]。
```

复杂度：

```text
O(VE)
```

对 IDL/RDL 通常很快，比 Simplex 简洁很多。

---

## 8.2 SPFA / Incremental Bellman-Ford

为了 CDCL(T) 增量：

当新增边：

```text
u -> v weight w
```

如果：

```text
dist[v] <= dist[u] + w
```

则无需传播。

否则：

```text
dist[v] = dist[u] + w
```

从 `v` 开始沿 outgoing edges 传播 relax。

伪代码：

```cpp
incrementalAddEdge(e):
    if dist[e.to] <= dist[e.from] + e.weight:
        return Feasible

    dist[e.to] = dist[e.from] + e.weight
    pred[e.to] = e

    queue.push(e.to)

    while queue not empty:
        u = queue.pop()

        for edge f in outgoing(u):
            if dist[f.to] > dist[u] + f.weight:
                dist[f.to] = dist[u] + f.weight
                pred[f.to] = f
                relaxCount[f.to]++

                if relaxCount[f.to] > numVars:
                    cycle = extractCycle(f.to, pred)
                    return NegativeCycle(cycle)

                queue.push(f.to)

    return Feasible
```

建议实现策略：

```text
如果没有 backtrack：用 incremental SPFA；
如果发生 backtrack：标记 dirty，下次 check 重新 Bellman-Ford rebuild。
```

这样正确性简单，性能也足够。

# 9. Negative cycle explanation

当发现负环：

```text
v0 -> v1 -> ... -> vk -> v0
```

其中每条边来自一个 active literal。

冲突 clause：

```text
¬reason(e1) ∨ ¬reason(e2) ∨ ... ∨ ¬reason(ek)
```

例如：

```text
x - y <= 3
y - z <= 4
z - x <= -8
```

三条相加：

```text
0 <= -1
```

负环冲突：

```text
¬l1 ∨ ¬l2 ∨ ¬l3
```

RDL strict zero cycle 也一样：

```text
x - y < 1
y - x <= -1
```

路径权重和：

```text
0 - δ
```

是负，因此冲突。

---

## 9.1 Cycle extraction

当 Bellman-Ford 第 `n` 轮还能 relax `v`：

```cpp
int x = v;
for i in 0..n-1:
    x = pred[x].from;

cycleStart = x;
do:
    e = pred[x];
    cycle.push_back(e);
    x = e.from;
while x != cycleStart;
```

然后去重 reason：

```cpp
std::vector<SatLit> lits;
for edge in cycle:
    lits.push_back(edge.reason.negated());
```

# 10. DBM Closure：传播用

Bellman-Ford 擅长冲突检测和模型构造，但如果你想做 theory propagation，需要 shortest path closure。

DBM:

```text
D[i][j] = currently known upper bound for value(j) - value(i)
```

也就是边：

```text
i -> j weight c
```

表示：

```text
j - i <= c
```

初始化：

```text
D[i][i] = 0
D[i][j] = INF
```

新增边：

```text
a -> b weight w
```

若：

```text
w < D[a][b]
```

更新：

```text
D[a][b] = w
```

增量闭包：

```cpp
for i in V:
  for j in V:
    D[i][j] = min(D[i][j], D[i][a] + w + D[b][j])
```

复杂度：

```text
O(n^2) per edge
```

如果：

```text
D[i][i] < 0
```

则 UNSAT。

---

## 10.1 DBM 适用场景

建议用双引擎：

```text
small n / propagation enabled:
  DBMClosure

large n / conflict only:
  IncrementalBellmanFord
```

阈值：

```text
n <= 256 使用 DBM
n > 256 使用 Bellman-Ford/SPFA
```

DBM 用于：

```text
1. 快速查询 implied bound；
2. theory propagation；
3. all-pairs shortest path；
4. small IDL/RDL 高性能求解。
```

Bellman-Ford 用于：

```text
1. 大规模稀疏图；
2. 模型构造；
3. negative cycle explanation。
```

# 11. Theory propagation

已有 atom：

```text
A := (x - y <= c)
```

如果当前图已经推出：

```text
x - y <= c
```

则可以 propagate `A=true`。

因为：

```text
x - y <= c
⇔
y -> x <= c
```

所以检查：

```text
D[y][x] <= c
```

如果成立：

```text
propagate A
```

reason 是 `y -> x` 的最短路径上的 edge reasons。

---

## 11.1 Positive / negative propagation

对于每个 registered difference atom `A`：

```text
A: x - y rel c
```

要检查：

```text
current graph entails A
current graph entails ¬A
```

例如 IDL：

```text
A: x - y <= c
```

则：

```text
A true entailed iff D[y][x] <= c

A false means x - y > c
           means x - y >= c+1
           means y - x <= -(c+1)
           entailed iff D[x][y] <= -(c+1)
```

RDL：

```text
A false means x - y > c
           means y - x < -c
```

所以需要 strict weight query。

---

## 11.2 Propagation lemma

如果 path `P` entails atom `A`：

```text
reason(P) => A
```

learn:

```text
¬r1 ∨ ¬r2 ∨ ... ∨ A
```

如果 entails `¬A`：

```text
¬r1 ∨ ¬r2 ∨ ... ∨ ¬A
```

# 12. Disequality handling

IDL/RDL disequality 不能直接作为边加入图。

---

## 12.1 IDL disequality

Active literal:

```text
d := (x - y != c)
```

若 `c` 非整数：

```text
tautology
```

若 `c` 整数，生成 guarded split:

```text
¬d ∨ (x - y <= c-1) ∨ (x - y >= c+1)
```

第二个分支：

```text
x - y >= c+1
⇔
y - x <= -(c+1)
```

即：

```text
¬d ∨ lit(x-y <= c-1) ∨ lit(y-x <= -(c+1))
```

这个 lemma 可以在 disequality active 时立即生成，也可以等当前 model 违反 disequality 时生成。

建议：

```text
第一版：当 model 违反 disequality 时生成 split lemma。
第二版：可配置 eager split。
```

---

## 12.2 RDL disequality

```text
d := (x - y != c)
```

如果当前 model 满足：

```text
x - y != c
```

NoChange。

如果当前 model 违反：

```text
x - y = c
```

生成：

```text
¬d ∨ (x - y < c) ∨ (x - y > c)
```

也就是：

```text
¬d ∨ lit(x-y < c) ∨ lit(y-x < -c)
```

# 13. Model construction

## 13.1 IDL model

Bellman-Ford 得到 `dist[v] ∈ Z`。

归一化：

```text
model(x) = dist[x] - dist[ZERO]
```

所有变量都是整数。

未出现变量：

```text
model(x) = 0
```

---

## 13.2 RDL model

Bellman-Ford 得到：

```text
dist[v] = a + bδ
```

先得到 delta model：

```text
modelδ(x) = dist[x] - dist[ZERO]
```

然后选择一个具体 rational `ε > 0`，将：

```text
δ = ε
```

得到 rational model。

选择方法：

对所有 active constraints 计算约束：

```text
A + Bε <= 0
```

收集对 `ε` 的 upper bounds。

若 `A < 0`：

```text
constraint already has rational slack
```

若 `A == 0`：

```text
需要 B <= 0
```

若某个约束要求：

```text
ε <= q
```

收集 `q`。

最终取：

```text
ε = min(1, all_positive_q) / 2
```

如果没有 upper bound：

```text
ε = 1
```

最终 validate 全部 RDL atoms。

# 16. Incremental / backtracking design

## 16.1 Simple correct version

At each theory check:

```text
reset graph
assert all active literals
run Bellman-Ford
handle disequalities
produce model
```

优点：

```text
最简单，正确性最好。
```

缺点：

```text
重复构图。
```

---

## 16.2 Production version

Maintain edge trail:

```cpp
struct EdgeTrailEntry {
    EdgeId edge;
    DecisionLevel level;
};
```

On SAT assignment:

```text
add edge incrementally
```

On backtrack:

```text
mark edges above level inactive
mark graph dirty
```

Next check:

```text
if dirty:
    rebuild graph and BF distances
else:
    incremental relax from newly added edges
```

This gives correctness + decent performance.

# 17. IDL/RDL check 流程

```cpp
TheoryCheckResult DlSolver::check() {
    if (graphDirty_) {
        graph_.rebuildActive();
        auto r = bf_.runFull(graph_);
        if (r.negativeCycle) {
            return conflictFromCycle(r.cycle);
        }
    } else {
        auto r = bf_.runIncremental(newEdges_);
        if (r.negativeCycle) {
            return conflictFromCycle(r.cycle);
        }
    }

    if (enableDbm_) {
        auto r = dbm_.update(newEdges_);
        if (r.negativeCycle) {
            return conflictFromDbm(r);
        }

        if (auto prop = findPropagation()) {
            return TheoryCheckResult::lemmaOrPropagation(*prop);
        }
    }

    if (auto lemma = handleDisequalities()) {
        return TheoryCheckResult::lemma(*lemma);
    }

    buildModel();
    return TheoryCheckResult::consistent();
}
```

# 21. Debug / validation

必须实现：

```cpp
DlModelValidator
```

检查：

```text
1. 每条 active edge 对应的约束成立；
2. 每个 active equality 两方向成立；
3. 每个 active disequality 成立；
4. IDL model 值都是整数；
5. RDL strict constraints 在 rational ε 实例化后成立；
6. dynamic atoms 与原始约束一致。
```

每次返回 SAT 前：

```cpp
assert(validator.validate(model, activeAtoms));
```

release 模式下：

```text
如果 validate 失败，返回 Unknown，不返回 SAT。
```

# 23. 最终建议

你的 IDL/RDL 不应该再走 Simplex。最终实现应该是：

```text
IDL/RDL:
  差分约束图
  + Bellman-Ford / SPFA 负环检测
  + DBM closure 做 propagation
  + dynamic atom 做 disequality split
  + exact model validation
```

最小可用版本：

```text
IDL:
  normalization + graph + Bellman-Ford + negative cycle explanation + model

RDL:
  delta weight + graph + strict negative cycle + epsilon model
```

增强版本：

```text
incremental SPFA
DBM propagation
disequality split
LIA/LRA difference fast path
```

这样设计比 Simplex 更简洁、更快，也更容易解释 UNSAT。
