# Phase 2：Cheap Integer Reasoning

目标：加入低成本、强收益、容易验证的整数推理。

Phase 2 的顺序：

```text
1. gcd equality conflict
2. gcd inequality tightening
3. single-variable bound tightening
4. equality normalization
5. branch selector 改进
```

---

## 1. GCD equality conflict

处理：

```text
a1*x1 + ... + an*xn = c
```

若：

```text
g = gcd(a1, ..., an)
g ∤ c
```

则该 equality 不可满足。

例子：

```smt2
(set-logic QF_LIA)
(declare-const x Int)
(declare-const y Int)
(assert (= (+ (* 6 x) (* 10 y)) 7))
(check-sat)
```

因为：

```text
gcd(6,10)=2
2 ∤ 7
```

所以 `unsat`。

实现：

```cpp
std::optional<TheoryConflict>
IntegerReasoner::checkGcdEqualityConflict(const ActiveLinearAtom& atom);
```

返回：

```text
¬lit_eq
```

---

## 2. GCD inequality tightening

对于：

```text
a·x <= c
```

若所有变量是整数，则左边是：

```text
g = gcd(a_i)
```

的倍数，所以可加强为：

```text
a·x <= g * floor(c / g)
```

例子：

```text
6x + 10y <= 23
```

加强为：

```text
6x + 10y <= 22
```

返回 guarded lemma：

```text
¬lit(6x+10y<=23) ∨ lit(6x+10y<=22)
```

对于：

```text
a·x >= c
```

加强为：

```text
a·x >= g * ceil(c / g)
```

---

## 3. 单变量 bound tightening

例如：

```text
2x <= 11
```

推出：

```text
x <= 5
```

例如：

```text
-3x <= 10
```

等价于：

```text
x >= ceil(-10/3)
```

实现：

```cpp
std::optional<TheoryLemma>
IntegerReasoner::singleVarBoundTighten(const ActiveLinearAtom& atom);
```

---

## 4. Equality normalization

例如：

```text
6x + 10y = 8
```

归约为：

```text
3x + 5y = 4
```

如果不能整除，直接 gcd equality conflict。

---

## 5. Branch selector 改进

当前如果是选第一个 fractional variable，Phase 2 改成：

```text
1. 优先选有 finite domain 且 domain 最小的变量；
2. 否则选 fractional part 最接近 1/2 的变量；
3. 再按变量 activity；
4. 最后按 var id。
```

至少先实现：

```cpp
score(x) = distanceToNearestInteger(beta[x])
```

选 score 最大者。

# Phase 3：Unit Cube Test

目标：在 branch/cut 前尝试直接构造 integer model。

位置：

```text
GeneralSimplex.check()
  ↓
disequality check
  ↓
integrality check
  ↓
cheap integer reasoning
  ↓
unit cube test
  ↓
Gomory cut
  ↓
branch split
```

Unit cube test：

给定：

```text
A x <= b
```

若存在中心 `z`，使边长为 1 的 cube 完全包含在 polyhedron 内，则 `round(z)` 是整数解。

变换：

```text
a_i · z <= b_i - 1/2 * ||a_i||_1
```

实现：

```cpp
class UnitCubeTester {
public:
    CubeResult tryFindIntegerModel(
        const std::vector<ActiveLinearAtom>& atoms,
        const std::unordered_set<int>& integerVars
    );
};
```

返回：

```text
sat(model)  或 unknown
```

注意：

```text
cube test 失败不能返回 unsat。
```

# Phase 4：Gomory Cut

现在重点展开 Phase 4。

目标：当 LRA relaxation 给出 fractional integer model 时，不只是 branch，而是从 simplex tableau 中生成 cutting plane，排除当前 fractional model，同时保留所有整数解。

Phase 4 先做：

```text
Pure / bound-aware Gomory fractional cut
```

暂时不做 MIR cut。

---

## 4.1 需要 GeneralSimplex 暴露的状态

Gomory 需要读取 tableau row，所以 `GeneralSimplex` 需要提供只读 snapshot：

```cpp
struct TableauEntry {
    int var;
    mpq_class coeff;
};

struct TableauRowView {
    int basicVar;
    mpq_class rhs;
    std::vector<TableauEntry> entries;
};

struct BoundView {
    bool hasLower;
    DeltaRational lower;
    std::optional<SatLit> lowerReason;

    bool hasUpper;
    DeltaRational upper;
    std::optional<SatLit> upperReason;
};

struct SimplexSnapshot {
    std::vector<TableauRowView> rows;
    std::vector<BoundView> bounds;
    std::vector<DeltaRational> beta;
    std::vector<bool> isBasic;
    std::vector<bool> isIntegerValued;
};
```

新增：

```cpp
SimplexSnapshot GeneralSimplex::snapshot() const;
```

必须保证 snapshot 是只读，不允许 cut generator 修改 simplex 内部状态。

---

## 4.2 integer-valued variable tracking

Gomory cut 不只要求原始变量是 Int，还要求 tableau row 里出现的所有变量都是 integer-valued。

需要在 `LiaSolver` 中维护：

```cpp
std::unordered_set<int> integerValuedVars_;
```

包含：

```text
1. 原始 Int 变量；
2. 由整数线性形式创建的 aux var；
3. dynamic branch bound 对应的 aux var，如果 lhs-rhs 是 integer-valued。
```

例如：

```text
s = 2x + 3y - 5
```

若 `x,y` 是 Int，则 `s` 也是 integer-valued。

但如果未来 mixed LIRA 有 Real 变量，包含 Real 变量的 aux 不能加入 `integerValuedVars_`。

Phase 4 暂时只做 QF_LIA，所以所有 arithmetic 变量和 linear-form aux 都可视为 integer-valued。但接口要按 `integerValuedVars_` 写，别写死。

---

## 4.3 Gomory cut 的 tableau 形式

GeneralSimplex row 语义是：

```text
x_B = b + Σ a_j x_j
```

其中：

```text
x_B 是 basic variable
x_j 是 non-basic variable
```

若：

```text
x_B 是 integer-valued；
当前 beta[x_B] 是 fractional；
row 中所有 x_j 都是 integer-valued；
```

则该行可作为 Gomory candidate。

但是为了生成标准 cut，需要把 non-basic variables 转成非负变量。

---

## 4.4 Bound-aware nonnegative transformation

对于 row：

```text
x_B = b + Σ a_j x_j
```

对每个 non-basic `x_j`：

如果当前在 lower bound：

```text
x_j = l_j + y_j
y_j >= 0
```

如果当前在 upper bound：

```text
x_j = u_j - y_j
y_j >= 0
```

若某个 `x_j` 既不在 finite lower，也不在 finite upper，则 Phase 4 先跳过该 row。

转换后：

```text
x_B = b' + Σ a'_j y_j
```

其中：

```text
if x_j at lower:
  b' += a_j * l_j
  a'_j = a_j

if x_j at upper:
  b' += a_j * u_j
  a'_j = -a_j
```

要求：

```text
l_j / u_j 必须是 finite rational；
若要保证 y_j integer，bound value 必须是整数；
bound 必须有 reason literal，方便生成 guarded lemma。
```

若这些条件不满足，跳过该 row。

---

## 4.5 Gomory fractional cut 公式

对于：

```text
x_B = b' + Σ a'_j y_j
```

且所有变量为 integer，`y_j >= 0`，若 `b'` 非整数，则：

```text
Σ frac(a'_j) * y_j >= frac(b')
```

其中：

```text
frac(q) = q - floor(q)
```

该 cut：

```text
1. 对所有整数解有效；
2. 当前 y_j = 0 的 relaxation model 违反；
3. 因此可以排除当前 fractional model。
```

---

## 4.6 把 cut 转回原变量

cut：

```text
Σ frac(a'_j) * y_j >= frac(b')
```

其中：

```text
if x_j at lower:
  y_j = x_j - l_j

if x_j at upper:
  y_j = u_j - x_j
```

所以 cut 变成：

```text
Σ_{L} f_j * (x_j - l_j)
+
Σ_{U} f_j * (u_j - x_j)
>= f0
```

展开：

```text
Σ_L f_j*x_j - Σ_U f_j*x_j
>=
f0 + Σ_L f_j*l_j - Σ_U f_j*u_j
```

其中：

```text
f_j = frac(a'_j)
f0 = frac(b')
```

最后把所有有理系数乘以 lcm，转成整数系数：

```text
A·x >= C
```

再做 gcd normalization。

---

## 4.7 生成 guarded lemma

Gomory cut 依赖当前 non-basic variables 的 active bounds。

如果用到：

```text
x_j at lower l_j
```

则 premise 是：

```text
lit(x_j >= l_j)
```

如果用到：

```text
x_j at upper u_j
```

则 premise 是：

```text
lit(x_j <= u_j)
```

最终 lemma：

```text
¬premise_1 ∨ ... ∨ ¬premise_k ∨ lit(cut)
```

其中 `lit(cut)` 是通过 `TheoryAtomRegistry::getOrCreateLinearBoundAtom` 注册的新动态 atom。

注意：

```text
不能直接 add unit cut。
必须 guarded。
```

实现：

```cpp
TheoryLemma buildGuardedCutLemma(
    const std::vector<SatLit>& premises,
    SatLit cutLit
) {
    TheoryLemma lemma;
    for (SatLit p : premises) {
        lemma.lits.push_back(p.negated());
    }
    lemma.lits.push_back(cutLit);
    return lemma;
}
```

## 4.9 Candidate row selection

不要每次扫所有 row 生成很多 cut。Phase 4 先做一个 cut：

```text
每次 theory check 最多生成 1 个 Gomory cut。
```

选择策略：

```text
1. basicVar 必须 integer-valued；
2. beta[basicVar] 必须 fractional；
3. row 中所有 non-basic vars 必须 integer-valued；
4. row 中每个 non-basic var 必须在 finite integer lower/upper bound；
5. cut 必须违反当前 beta；
6. 选择 row nnz 最小者；
7. tie-break 选 fractional part 最接近 1/2 的 basic var；
8. 再 tie-break by basic var id。
```

这样能减少 fill-in 和大 cut。

## 4.10 Cut validation

Gomory cut 生成后必须做两层 validation：

### A. 当前 model 必须违反 cut

```cpp
assert(evaluate(cut, beta) == false);
```

如果当前 model 不违反，跳过。

### B. Debug-only Gomory lemma validator

检查：

```text
1. cut 的 premises 都有合法 SatLit；
2. cutLit 是动态注册 atom；
3. cut 的 lhs/rhs 经过 lcm 清理，系数是整数；
4. cut 来自一行合法 tableau row；
5. 所有参与变量是 integer-valued；
6. all y_j are nonnegative under premise bounds。
```

## 4.11 LiaSolver check 顺序加入 Gomory

最终 `LiaSolver::check()` 顺序：

```cpp
TheoryCheckResult LiaSolver::check(const CoreIr& ir) {
    auto r = gs_.check();
    if (r == Unsat) {
        return conflict(manager_.translateConflict(gs_, boundReasons_));
    }

    auto beta = gs_.model();

    // 1. Disequality obligations
    for (const auto& d : disequalities_) {
        if (!gs_.value(d.auxVar).isZero()) continue;

        auto dr = checkIntegerDisequality(d);
        if (dr.kind == Satisfied) continue;
        if (dr.kind == Conflict) return conflict(*dr.conflict);
        if (dr.kind == Lemma) return lemma(*dr.lemma);
    }

    // 2. Integral model check
    bool allIntegral = true;
    int fractionalVar = -1;
    DeltaRational fractionalValue;

    for (int v : integerVars_) {
        auto val = gs_.value(v);
        if (val.b != 0 || val.a.get_den() != 1) {
            allIntegral = false;
            fractionalVar = v;
            fractionalValue = val;
            break;
        }
    }

    if (allIntegral) {
        if (!validateModel(beta, activeAtoms_, disequalities_)) {
            return unknown();
        }
        return consistent();
    }

    // 3. Cheap integer reasoning
    if (auto lemmaOrConflict = integerReasoner_.run(...)) {
        return *lemmaOrConflict;
    }

    // 4. Unit cube test
    if (auto cube = cubeTester_.tryFindIntegerModel(...)) {
        if (cube->sat && validateModel(cube->model)) {
            setModel(cube->model);
            return consistent();
        }
    }

    // 5. Gomory cut
    if (options_.enableGomoryCuts) {
        auto snapshot = gs_.snapshot();
        auto cut = gomory_.generate(snapshot, integerValuedVars_);

        if (cut) {
            SatLit cutLit = registry_.getOrCreateLinearBoundAtom(
                cut->lhs, cut->rel, cut->rhs
            );

            TheoryLemma lemma =
                buildGuardedCutLemma(cut->premises, cutLit);

            assert(lemmaValidator_.validateGomoryCut(lemma));

            return TheoryCheckResult::mkLemma(lemma);
        }
    }

    // 6. Branch split fallback
    return lemma(buildBranchSplitLemma(fractionalVar, fractionalValue));
}
```
