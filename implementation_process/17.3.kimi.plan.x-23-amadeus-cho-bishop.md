# IDL / RDL Difference-Logic Solver Implementation Plan

## Objective
Implement dedicated IDL (Integer Difference Logic) and RDL (Real Difference Logic) solvers using graph-based negative-cycle detection (Bellman-Ford / SPFA) instead of Simplex. These solvers will be registered for `QF_IDL` / `QF_RDL` logics and will reuse the existing `LinearAtomPayload` infrastructure.

## Key Design Decisions

1. **TheoryId**: Add `IDL` and `RDL` to the `TheoryId` enum in `src/expr/types.h`. This allows `TheoryManager` to route atoms directly without confusing them with general LIA/LRA atoms.
2. **Atomizer integration**: Minimal change. In `Solver.cpp`, when logic is `QF_IDL`/`QF_RDL`, set `Atomizer::defaultTheory_` to `IDL`/`RDL`. The existing `extractLinearConstraint` path in `Atomizer` already produces `LinearAtomPayload`, which is sufficient. The IDL/RDL solvers will self-filter difference constraints from the payload.
3. **Unsupported atoms**: If an IDL/RDL solver receives a non-difference linear constraint, it sets an internal `unsupported_` flag and returns `Unknown` from `check()`. The existing `hasUnsupportedTheoryAtom()` mechanism is handled at the registry level; for IDL/RDL we will add a solver-level unsupported check that the caller can query, or simply return `Unknown` and let the SAT solver abort.
4. **Dynamic atoms for disequality split**: Reuse `TheoryAtomRegistry::getOrCreateLinearBoundAtom`. The split lemmas generate new `LinearAtomPayload` atoms with the same `TheoryId` (IDL or RDL).
5. **Model validation**: Each solver implements a private `validateModel()` that checks all active constraints before returning `Sat`.

## Phase 2: IDL Solver

### 2.1 IdlSolver.h/.cpp
Inherits `TheorySolver`.

**State (V1):**
```cpp
struct ActiveAssignment { int level; SatLit lit; TheoryAtomRecord atom; bool value; };
std::vector<ActiveAssignment> activeAssignments_;
DifferenceGraph<mpz_class> graph_;
BellmanFord<mpz_class> bf_;
TheoryAtomRegistry* registry_ = nullptr;
std::optional<TheoryConflict> pendingConflict_;
std::optional<TheoryLemma> pendingLemma_;
// Note: V1 does NOT use incremental SPFA. Graph is rebuilt from scratch every check().
```

**`assertLit`**: Store assignment. If `LinearAtomPayload`, first compute **effective relation**:
```
value=true  -> original relation
value=false -> negated relation

Negation:
  <= -> >
  <  -> >=
  >= -> <
  >  -> <=
  =  -> !=
  != -> =
```

Then attempt to extract difference constraint from the effective atom:
- Extract linear form `a*x + b*y + const rel 0`.
- **1 var**:
  - `a = +1` â†’ `x - ZERO rel -const`
  - `a = -1` â†’ `ZERO - x rel -const`
  - otherwise â†’ unsupported.
- **2 vars**: require `{a, b} = {+1, -1}`.
  - `+1*x + (-1)*y + const rel 0` â†’ `x - y rel -const`
  - `-1*x + (+1)*y + const rel 0` â†’ `y - x rel -const`
- Convert to canonical `x - y <= c` form (negate and swap vars if needed).
- Apply IDL normalization (floor for `<=`, `ceil-1` for `<`, etc.).
- Add edge to graph, record `EdgeId` in `newEdges_`.
- For `Eq`: add two edges (`<=` and `>=`).
- For `Neq`: store disequality for later split (do not add edge).

**`backtrackToLevel`**: Remove assignments with `level > target`. Mark `graphDirty_ = true`. Clear `newEdges_`.

**`check` (V1)**:
```cpp
// V1: always full rebuild + full Bellman-Ford. No incremental SPFA.
graph_.clear();
for each active assignment:
    extract difference constraint, normalize, add edge to graph_

auto r = bf_.runFull(graph_);

if (r.negativeCycle)
    return TheoryCheckResult::mkConflict(buildConflict(r.cycle, graph_));

// Disequalities
for each stored diseq (x - y != c):
    if current model has x - y == c:
        return splitLemma(diseq)

buildModel();
return TheoryCheckResult::consistent();
```

**Normalization-time conflicts**: Any atom that is discovered to be impossible during normalization must return a `TheoryConflict` containing the **negation of the active literal's reason**.
- Example: `x - y = 1/2` in IDL â†’ `TheoryConflict{{reason.negated()}}`.
- Example: `x - y != 1/2` in IDL â†’ tautology, no edge added, no conflict.
- This applies to all immediate contradictions detected before graph construction.

**IDL rational RHS rules** (must be explicit):
```
x - y = c:
  if c not integer -> immediate conflict (UNSAT)
  else add both bounds (x-y <= c) and (y-x <= -c)

x - y != c:
  if c not integer -> tautology / NoChange (integer can never equal non-integer)
  else store disequality for split

x - y <= c -> x - y <= floor(c)
x - y <  c -> x - y <= ceil(c) - 1
x - y >= c -> y - x <= -ceil(c)
x - y >  c -> y - x <= -(floor(c) + 1)
```

**Model validation**: Verify every active edge `dist[to] <= dist[from] + weight`, every active equality, every active disequality.

## Phase 3: RDL Solver

### 3.1 RdlSolver.h/.cpp
Nearly identical structure to `IdlSolver`, but uses `RdlWeight`.

**Normalization differences:**
- `x - y < c` â†’ edge weight `(c, -1)` (strict)
- `x - y <= c` â†’ edge weight `(c, 0)`
- No integer rounding needed.

**Negative cycle**: A cycle with total weight `(c, d)` is negative iff `c < 0 || (c == 0 && d < 0)`. This captures the strict-zero cycle (`0 - Î´`).

**Model construction**:
1. Run Bellman-Ford to get potentials `dist[v] = a_v + b_vÂ·Î´`.
2. Collect all active strict constraints: each gives an upper bound on `Îµ` (the concrete value of `Î´`).
3. Choose `Îµ = 1` if unconstrained, otherwise `Îµ = minPositiveUpperBound / 2`.
4. Instantiate: `model(v) = (a_v - a_ZERO) + (b_v - b_ZERO)Â·Îµ`.

**Disequality split** (RDL):
```
Â¬(x-y != c) âˆ¨ (x-y < c) âˆ¨ (y-x < -c)
```
Use `registry_->getOrCreateLinearBoundAtom` with strict relations, passing **`TheoryId::RDL`** so new literals route back to `RdlSolver`. `LinearAtomPayload` supports `Relation::Lt`/`Gt`, so strict splits can use those directly.

### 4.3 Unsupported atom handling
In `IdlSolver::check()` / `RdlSolver::check()`, if any active assignment is not a difference constraint, return `TheoryCheckResult::unknown()`. This causes the CDCL(T) loop to abort with `Unknown` (same pattern as NIA's incomplete cases).

### 4.3 Unsupported atom handling
In `IdlSolver::check()` / `RdlSolver::check()`, if any active assignment is not a difference constraint, return `TheoryCheckResult::unknown()`. `TheoryManager::check()` short-circuits on first non-Consistent result, and `Solver::checkSatInternal` maps `Unknown` to top-level `Result::Unknown`. ## Risks & Mitigations

1. **RDL epsilon selection**: If upper-bound computation is buggy, strict constraints may be violated. Mitigation: always run `validateModel()` before returning `Sat`; if it fails, return `Unknown`.
2. **Disequality lemma loops**: Split lemmas could be re-learned. Mitigation: deduplicate via `TheoryLemmaDatabase::contains()` before returning a lemma.
3. **Backtracking correctness**: Full rebuild on backtrack is simplest and correct. Mitigation: Phase 1 uses `graphDirty_` full-rebuild pattern (same as LraSolver/LiaSolver's lazy replay).
