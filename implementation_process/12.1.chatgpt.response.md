对，这个计划确实太小了。它基本只做了 **Phase 2: minimal branch lemma**，主要内容是给 `NiaSolver` 加 `TheoryAtomRegistry`、实现 `x <= k ∨ x >= k+1`、加去重和少量测试。作为一个小任务可以，但不适合作为“下一阶段 NIA 主线计划”。

而且当前计划里还有一个关键判断不严谨：

```text

```

这个不应该作为 branch lemma 的预期效果。`x²=2` 应该由 **univariate root / modular reasoning** 证明 UNSAT；单靠 `x <= k ∨ x >= k+1` 对无界整数域分裂，理论上可能无限分支，不能指望它完成 UNSAT 证明。

# Phase 1：Bounded-first 能力补强

这个阶段解决“有界 NIA 问题稳定 SAT/UNSAT”。

## 1.1 SquareBoundReasoner

新建：

```text
src/theory/arith/nia/SquareBoundReasoner.h/.cpp
```

处理 normalized constraint：

```text
x² - c <= 0
x² - c = 0
x² - c != 0
x² - c >= 0
```

支持：

```text
x² - c <= 0:
  c < 0  -> UNSAT
  c >= 0 -> x ∈ [-floorSqrt(c), floorSqrt(c)]

x² - c = 0:
  c < 0 -> UNSAT
  c not square -> UNSAT
  c = 0 -> x = 0
  c = r² -> x ∈ {-r, r}

x² - c != 0:
  c < 0 -> tautology
  c not square -> tautology
  c = 0 -> x != 0
  c = r² -> x != r, x != -r

x² - c >= 0:
  Phase 1 暂时 NoChange
```

注意：

```text
输入是 p rel 0，不是 x² rel c。
```

所以必须识别 `p = x² - c`。

测试：

```text
x² <= 4       -> domain [-2,2]
x² <= -1      -> UNSAT
x² = 49       -> domain {-7,7}
x² = 50       -> UNSAT
x² != 4       -> exclude {-2,2}
x² != 0       -> exclude {0}
```

# Phase 2：Branch Lemma + CDCL(T) 搜索闭环

这个阶段不是为了直接证明所有无界 UNSAT，而是让 SAT/CDCL 能探索整数空间。

## 2.1 NiaSolver 注册 TheoryAtomRegistry

添加：

```cpp
void NiaSolver::setRegistry(TheoryAtomRegistry* reg);
```

在 `Solver.cpp` 中注册：

```cpp
auto nia = std::make_unique<NiaSolver>(std::move(polyKernel));
nia->setRegistry(&registry);
theoryManager.registerSolver(std::move(nia));
```

## 2.2 Branch lemma

只做全局有效 split：

```text
x <= k ∨ x >= k+1
```

构造：

```cpp
PolyId xPoly = kernel_->mkVar(x);

SatLit le = registry_->getOrCreatePolynomialAtom(
    xPoly, Relation::Leq, mpq_class(k), TheoryId::NIA
);

SatLit ge = registry_->getOrCreatePolynomialAtom(
    xPoly, Relation::Geq, mpq_class(k + 1), TheoryId::NIA
);

return TheoryLemma{{le, ge}};
```

必须保证 dynamic atom 回来时：

```text
(poly rel rhs) -> (poly - rhs) rel 0
```

例如：

```text
x <= k   -> x-k <= 0
x >= k+1 -> x-(k+1) >= 0
```

## 2.3 Duplicate suppression

维护：

```cpp
struct BranchSplitKey {
    std::string var;
    mpz_class k;
};
```

如果同一个 `(var,k)` 已生成过：

```text
尝试另一个变量；
尝试另一个 k；
否则返回 Unknown。
```

不能无限重复加同一条 lemma。

## 2.4 变量选择

优先：

```text
1. 有 lower 和 upper 的最大区间变量；
2. 只有 lower 或 upper 的变量；
3. 出现频率最高的变量；
4. fallback: 第一个变量。
```

## 2.5 k 选择

```text
有 lower 和 upper:
  k = floor((lower + upper)/2)

只有 lower:
  k = lower

只有 upper:
  k = upper - 1

无界:
  k = 0
```

## 2.6 测试

不要写：

```text
x²=2 通过 branch 证明 UNSAT
```

这个预期不对。

Branch 测试应该是：

```text
1. buildBranchLemma 生成 x<=k ∨ x>=k+1
2. dynamic atoms 正确注册
3. SAT 选择分支后 NiaSolver 能收到 dynamic atom
4. duplicate split 不重复生成
5. bounded problem 在 branch 后 domain 收紧
```

端到端可以用：

```text
x >= 0, x <= 100000, x² = 49
```

branch 不是必须证明 UNSAT，而是应该帮助缩小搜索或让 CDCL 继续探索。

# Phase 3：Interval Evaluation + 基础 ICP

这个阶段做 **安全的 over-approx interval reasoning**，不做复杂反向乘法传播。

## 3.1 IntervalZ

```cpp
struct IntervalZ {
    mpz_class lo;
    mpz_class hi;

    bool isEmpty() const { return lo > hi; }
};
```

## 3.2 IntervalEvaluator

支持：

```text
add
sub
neg
mul
pow2
pow3
general pow
```

规则：

```text
[a,b] + [c,d] = [a+c, b+d]
[a,b] - [c,d] = [a-d, b-c]
-[a,b] = [-b, -a]

[a,b] * [c,d]
= [min(ac,ad,bc,bd), max(ac,ad,bc,bd)]

even power:
  if 0 in [l,u] -> [0, max(l^k,u^k)]
  else -> [min(l^k,u^k), max(l^k,u^k)]

odd power:
  [l^k,u^k]
```

必须保证：

```text
永远 over-approx，不允许 under-approx。
```

## 3.3 Constraint interval status

对 normalized `p rel 0`：

```text
p <= 0:
  violated if interval.lo > 0
  satisfied if interval.hi <= 0

p >= 0:
  violated if interval.hi < 0
  satisfied if interval.lo >= 0

p = 0:
  violated if 0 notin [lo,hi]
  satisfied if [lo,hi] == [0,0]

p != 0:
  satisfied if 0 notin [lo,hi]
  otherwise unknown
```

## 3.4 Sum-of-squares propagation

支持：

```text
x² + y² <= c
```

若 `c < 0`：

```text
UNSAT
```

若 `c >= 0`：

```text
x,y ∈ [-floorSqrt(c), floorSqrt(c)]
```

推广：

```text
Σ xi² <= c
```

对每个变量收紧同样边界。

## 3.5 禁止 Product reverse pruning

这一阶段明确不做：

```text
xy=c 反推 x/y bounds
```

原因是跨 0、负数、除法边界、不等式方向都容易写错。只做 interval evaluation 和 safe pruning。

# Phase 4：Interval B&B

这个阶段解决：

```text
finite domains 但 totalSize 很大
```

## 4.1 入口

```text
domains.allFinite(vars) == true
totalSize > enumerationThreshold
```

## 4.2 内部状态

内部结果不要叫 `UnsatComplete`，避免和顶层 conflict 混淆。

```cpp
enum class BoxResult {
    Sat,
    Closed,
    Unknown
};
```

## 4.3 算法

```cpp
BoxResult solveBox(Box box, int depth) {
    if (budget exceeded) return Unknown;

    auto status = evaluateConstraintsByInterval(box);

    if (any definitely violated) {
        return Closed;
    }

    if (box is singleton) {
        if (validator.validate(exactAssignment)) return Sat;
        return Closed;
    }

    auto var = chooseSplitVar(box);
    auto [left, right] = split(box, var);

    auto r1 = solveBox(left, depth+1);
    if (r1 == Sat) return Sat;

    auto r2 = solveBox(right, depth+1);
    if (r2 == Sat) return Sat;

    if (r1 == Closed && r2 == Closed) return Closed;
    return Unknown;
}
```

顶层：

```cpp
auto r = solveBox(rootBox);

if (r == Sat) return Sat;
if (r == Closed) return UnsatComplete(allActiveReasonsConflict);
if (r == Unknown) return UnknownBudget;
```

## 4.4 正确性

```text
Closed 只表示该 box 无解或被完全关闭；
只有 root box Closed 才能变成 UNSAT。
```

# Phase 5：代数增强

等 Phase 1-4 稳定后再做。

## 5.1 多变量 GCD conflict

对 normalized equality：

```text
p = 0
p = q + c
q 无常数项
g = gcd(coefficients(q))
```

若：

```text
g ∤ -c
```

返回 UNSAT。

例子：

```text
6x + 9y = 5 -> UNSAT
2x² + 4yz = 3 -> UNSAT
```

## 5.2 Factor rules

```text
p*q = 0
=> p=0 ∨ q=0

p*q != 0
=> p!=0
=> q!=0
```

用 guarded lemmas：

```text
¬(pq=0) ∨ (p=0) ∨ (q=0)

¬(pq!=0) ∨ (p!=0)
¬(pq!=0) ∨ (q!=0)
```

所有新 atom 必须走：

```cpp
TheoryAtomRegistry::getOrCreatePolynomialAtom
```

## 5.3 Modular reasoning 扩展

扩展：

```text
evaluateMod(poly, assignment_mod_m, m)
```

支持更多变量、更通用的模运算检查。

# Phase 6：Local Search SAT Finder

Local search 只作为 SAT finder。

## 6.1 初始点

```text
domain midpoint
small integers [-10,10]
univariate roots
random within bounds
```

## 6.2 Objective

```text
p <= 0: max(0,p)
p >= 0: max(0,-p)
p = 0: abs(p)
p != 0: p==0 ? 1 : 0
```

总目标：

```text
Σ violation²
```

## 6.3 Move

```text
单变量 ±1
随机重启
按 violation 最大的约束挑变量
```

## 6.4 正确性

```text
找到 candidate -> exact validate -> SAT
失败 -> NoChange / Unknown
永远不能 UNSAT
```
